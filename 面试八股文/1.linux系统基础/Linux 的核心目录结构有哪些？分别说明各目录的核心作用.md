#### 核心标准答案

Linux 采用**树形目录结构**，所有目录都从根目录`/`开始，遵循 FHS（文件系统层次结构标准），核心目录及作用如下：

表格

|目录名|核心作用|关键说明|
|:--|:--|:--|
|**/（根目录）**|整个 Linux 文件系统的起点，所有目录、文件、设备都挂载在根目录下|根目录必须单独分区，所有其他目录都是根目录的子目录，根目录的权限必须严格控制，禁止普通用户修改|
|**/boot**|系统启动相关的文件，包括 Linux 内核、引导程序、启动配置文件|存放 vmlinuz（内核镜像）、initramfs（内存文件系统）、grub（引导程序）目录，系统启动必须的目录，通常单独分区，大小 200MB-1GB 即可|
|**/etc**|系统和应用的配置文件目录，所有系统级、服务级的配置都存放在这里|核心目录，存放 hosts、passwd、shadow、fstab、sysctl.conf、nginx、mysql 等服务的配置文件，所有配置文件基本都是文本格式，修改配置无需重启系统，仅需重启对应服务|
|**/home**|普通用户的家目录，每个普通用户都有一个以用户名命名的子目录|比如用户 testuser 的家目录是`/home/testuser`，用户的文档、下载、配置文件都存放在这里，用户对自己的家目录有完全的控制权，通常单独分区，避免用户数据占满系统盘|
|**/root**|root 超级用户的家目录|区别于普通用户的家目录，不在 /home 下，直接在根目录下，只有 root 用户有权限访问，root 用户的默认家目录，系统单用户模式下，只有 /root 目录会被挂载|
|**/usr**|系统的应用程序、资源、文档目录，相当于 Windows 的 Program Files|只读目录，存放系统安装的软件、库文件、头文件、文档、二进制命令，是 Linux 中最大的目录之一，通常单独分区|
|**/usr/bin**|普通用户可执行的二进制命令，系统安装的软件命令都在这里|比如 ls、cd、cat、grep、awk 等基础命令，以及 nginx、mysql 等软件的命令，所有用户都可以执行，默认在 PATH 环境变量中|
|**/usr/sbin**|root 超级用户可执行的系统管理命令|比如 fdisk、ifconfig、systemctl、iptables 等系统管理命令，只有 root 用户有权限执行，默认在 root 的 PATH 环境变量中|
|**/usr/lib**|系统和应用程序的共享库文件、内核模块|存放程序运行需要的动态链接库（.so 文件），相当于 Windows 的 dll 文件，32 位系统是 /usr/lib，64 位系统还有 /usr/lib64|
|**/usr/local**|用户手动编译安装的软件目录，区别于系统自带的软件|手动编译安装的软件默认安装在这里，比如`./configure --prefix=/usr/local/nginx`，有独立的 bin、sbin、lib、etc 目录，不会和系统自带的软件冲突，环境变量通常会包含 /usr/local/bin|
|**/var**|可变数据目录，存放系统运行过程中不断变化的文件|核心目录，存放频繁写入、变化的文件，通常单独分区，避免日志、缓存文件占满系统盘|
|**/var/log**|系统和应用的日志文件目录，所有日志都存放在这里|核心目录，存放系统日志（messages、secure、dmesg）、应用日志（nginx、mysql、tomcat 等），是故障排查、安全审计的核心目录，生产环境必须配置日志轮转，避免日志占满磁盘|
|**/var/lib**|应用程序运行时需要的持久化数据目录|比如 mysql 的数据库文件、docker 的镜像和容器数据、redis 的持久化文件，都存放在这里，是应用数据的核心存储目录，生产环境通常单独分区，保证数据安全|
|**/var/run**|系统运行时的临时文件，存放进程的 PID 文件、socket 文件|比如 nginx.pid、sshd.pid，存放进程的 PID，用于进程管理，系统重启后会清空，挂载为内存文件系统 tmpfs|
|**/tmp**|系统临时文件目录，所有用户都可以读写，存放临时文件|系统和应用程序产生的临时文件都存放在这里，默认 10 天自动清理，系统重启后会清空，挂载为内存文件系统 tmpfs，读写速度快，禁止存放重要数据|
|**/dev**|设备文件目录，存放所有硬件设备的文件|Linux 中一切皆文件，硬件设备都以文件的形式存放在这里，比如 /dev/sda（硬盘）、dev/null（空设备）、dev/random（随机数设备）、dev/pts（终端设备），应用程序通过读写设备文件，和硬件交互|
|**/proc**|虚拟文件系统，不占用磁盘空间，存放内核和进程的实时状态信息|核心目录，内存中的映射，存放系统内核、进程、CPU、内存、网络的实时状态信息，比如 /proc/cpuinfo（CPU 信息）、/proc/meminfo（内存信息）、/proc/net/tcp（网络连接信息）、/proc/[pid]/（进程信息），是系统监控、故障排查的核心目录|
|**/sys**|虚拟文件系统，存放内核、硬件设备的驱动和状态信息|和 /proc 类似，也是内存映射，更偏向于硬件设备和驱动的管理，存放设备树、驱动参数、电源管理等信息，是 Linux 设备模型的核心目录|
|**/mnt**|临时挂载目录，用于临时挂载文件系统、硬盘、U 盘、网络共享|系统管理员临时挂载外部设备的目录，比如挂载 U 盘、移动硬盘、NFS 共享，区别于永久挂载的 /media 目录|
|**/opt**|可选的第三方大型软件安装目录|用于安装大型的商业软件、第三方应用，比如 Oracle、SAP 等，区别于 /usr/local，通常是单个软件一个独立的目录，方便管理和卸载|
|**/media**|可移动介质的挂载目录|系统自动挂载 U 盘、光盘、移动硬盘等可移动设备的目录，区别于手动挂载的 /mnt 目录|

#### 专家级拓展

- 生产环境分区最佳实践：
    
    生产环境服务器，必须对核心目录单独分区，避免单个目录占满磁盘导致系统崩溃，推荐分区方案：
    
    1. `/boot`：单独分区，大小 1GB；
    2. `/`：根目录，单独分区，大小 50-100GB；
    3. `/home`：单独分区，大小根据用户数量调整；
    4. `/var`：单独分区，大小 50-200GB，避免日志占满系统盘；
    5. `/var/lib`：如果有数据库、docker 等应用，单独分区，保证数据安全；
    6. `/tmp`：单独分区，挂载为 tmpfs，提升性能，限制最大大小。
    
- 核心目录权限规范：
    
    1. 根目录`/`的权限必须是 755，所有者 root:root，禁止修改；
    2. `/etc`目录权限 755，配置文件权限 644，敏感文件比如`/etc/shadow`权限 000，只有 root 可访问；
    3. `/var/log`目录权限 755，日志文件权限 644，禁止普通用户修改删除；
    4. `/boot`目录权限 755，只有 root 可写，避免启动文件被篡改。
    

#### 面试避坑指南

- 严禁只零散罗列目录名称，不说核心作用，会被面试官判定为对 Linux 目录结构理解不全面；
- 避免混淆`/bin`和`/usr/bin`、`/sbin`和`/usr/sbin`，现在主流 Linux 发行版中，`/bin`是`/usr/bin`的软链接，`/sbin`是`/usr/sbin`的软链接，统一放在 /usr 目录下，要说明这个细节；
- 不要遗漏`/proc`和`/sys`虚拟文件系统，这是 Linux 的核心特色，也是故障排查、监控的核心目录，提到会大幅加分。
