#### 核心标准答案

整个流程是一个分层递进的过程，从应用层到网络层、数据链路层，再到服务端处理，最终返回数据并渲染页面，完整流程分为 8 个核心阶段：

1. **URL 解析与合法性校验**
    
    浏览器首先解析输入的 URL，拆分出协议（HTTP/HTTPS）、域名、端口、路径、查询参数、锚点；同时校验 URL 的合法性，不合法则直接调用搜索引擎搜索；如果是 HTTPS 协议，默认端口 443，HTTP 默认端口 80。
2. **DNS 域名解析，获取服务器 IP 地址**
    
    浏览器需要将域名转换为服务器的 IP 地址，完整解析流程：
    
    1. 浏览器缓存：先查浏览器自身的 DNS 缓存，是否有该域名的解析记录；
    2. 系统缓存：查操作系统的 hosts 文件、系统 DNS 缓存；
    3. 本地 DNS 服务器：向本地配置的 LDNS（比如运营商 DNS、公共 DNS 114.114.114.114）发起查询；
    4. 迭代查询：LDNS 向根 DNS 服务器、顶级域名服务器、权威域名服务器依次发起迭代查询，最终获取域名对应的服务器 IP 地址，返回给浏览器。
    
3. **TCP 连接建立**
    
    浏览器拿到服务器 IP 后，根据端口号，与服务器建立 TCP 连接，通过 TCP 三次握手完成连接初始化：
    
    1. 客户端发送 SYN 报文，发起连接请求；
    2. 服务端回复 SYN+ACK 报文，确认连接请求；
    3. 客户端回复 ACK 报文，连接建立成功，开始传输数据。
        
        如果是 HTTPS 协议，TCP 连接建立后，还会完成 TLS 四次握手，协商加密密钥，建立安全加密通道。
    
4. **HTTP/HTTPS 请求发送**
    
    浏览器构建 HTTP 请求报文，包括请求行、请求头、请求体，通过建立好的 TCP 连接，发送给服务器；请求头中包含 Cookie、User-Agent、Accept、Host 等核心信息。
5. **服务端处理请求，返回响应**
    
    服务端的 Web 服务器（Nginx/Apache）收到请求后，解析请求报文，根据路径匹配对应的业务服务，处理请求（查询数据库、渲染页面、调用接口等），构建 HTTP 响应报文，包括状态行、响应头、响应体，通过 TCP 连接返回给浏览器。
6. **TCP 连接关闭**
    
    数据传输完成后，通过 TCP 四次挥手关闭连接：
    
    1. 客户端发送 FIN 报文，请求关闭连接；
    2. 服务端回复 ACK 报文，确认收到关闭请求；
    3. 服务端发送 FIN 报文，通知客户端服务端也准备关闭连接；
    4. 客户端回复 ACK 报文，连接正式关闭。
        
        如果是 HTTP/1.1，默认开启长连接`Connection: keep-alive`，会复用 TCP 连接，传输多个请求 / 响应，不会立即关闭连接。
    
7. **浏览器解析响应，渲染页面**
    
    浏览器收到响应后，首先判断状态码：200 成功则继续处理，3xx 则重定向，4xx/5xx 则展示错误页面。
    
    成功响应的渲染流程：
    
    1. 解析 HTML，生成 DOM 树；
    2. 解析 CSS，生成 CSSOM 规则树；
    3. 结合 DOM 树和 CSSOM 树，生成渲染树（Render Tree）；
    4. 布局（Layout）：计算渲染树中每个节点的位置和大小；
    5. 绘制（Paint）：将节点绘制到屏幕上；
    6. 合成（Composite）：将页面分层合成，提升渲染性能。
    
8. **静态资源加载**
    
    渲染过程中，遇到 HTML 中的 JS、CSS、图片、视频等静态资源，会再次发起 HTTP 请求，重复上述 DNS、TCP、请求流程，加载完成后更新页面渲染。

#### 专家级拓展

- 进阶细节补充：
    
    1. ARP 地址解析：拿到服务器 IP 后，操作系统会通过 ARP 协议，将网关的 IP 地址转换为 MAC 地址，封装到以太网帧中，实现二层数据传输；
    2. CDN 加速：如果域名配置了 CDN，DNS 解析会返回离用户最近的 CDN 边缘节点 IP，请求会先到 CDN 节点，静态资源直接从 CDN 返回，无需回源站，提升访问速度；
    3. 负载均衡：请求到达服务端后，会先经过负载均衡器（LVS/Nginx），转发到后端的应用服务器，实现高可用和负载均衡；
    4. 浏览器缓存：请求静态资源时，会先查浏览器的强缓存（Cache-Control/Expires），如果缓存有效，直接使用本地缓存，不发起网络请求；如果强缓存失效，发起协商缓存请求（If-Modified-Since/If-None-Match），服务端返回 304，则继续使用本地缓存。
    
- 生产环境优化方向：
    
    1. DNS 预解析，减少 DNS 解析耗时；
    2. 开启 TCP 长连接、HTTP/2，减少 TCP 握手开销；
    3. 静态资源 CDN 加速，配置合理的缓存策略；
    4. 页面渲染优化，减少 DOM 节点、优化 CSS/JS 加载顺序，避免阻塞渲染。
    

#### 面试避坑指南

- 严禁只零散说几个步骤，不说完整的分层流程，会被面试官判定为对网络分层模型理解不透彻；
- 避免遗漏 DNS 解析的完整流程、HTTPS 的 TLS 握手、浏览器渲染流程，这是面试的核心加分细节；
- 不要把 TCP 三次握手和四次挥手的顺序说反，这是基础扣分点。
