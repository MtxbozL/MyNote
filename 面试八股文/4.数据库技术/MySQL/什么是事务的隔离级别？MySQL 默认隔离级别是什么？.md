#### 核心标准答案

事务的隔离级别，是数据库为了平衡「并发性能」和「数据一致性」，定义的多个隔离等级，用于控制并发事务之间的隔离程度，解决并发事务带来的脏读、不可重复读、幻读问题。SQL 标准定义了 4 个隔离级别，从低到高依次为：

1. **读未提交（Read Uncommitted）**：最低隔离级别，一个事务可以读取到另一个事务未提交的修改。
    
    - 存在问题：脏读、不可重复读、幻读
    - 生产环境几乎不使用。
    
2. **读已提交（Read Committed，RC）**：一个事务只能读取到另一个事务已经提交的修改，未提交的修改不可见。
    
    - 解决问题：脏读
    - 存在问题：不可重复读、幻读
    - 是 Oracle、SQL Server 等大多数数据库的默认隔离级别。
    
3. **可重复读（Repeatable Read，RR）**：保证同一个事务内，多次读取同一个字段的结果始终一致，不受其他事务的修改影响，不管其他事务是否提交。
    
    - 解决问题：脏读、不可重复读
    - 存在问题：幻读（MySQL InnoDB 引擎通过 MVCC + 间隙锁，在 RR 级别解决了幻读问题）
    - **是 MySQL InnoDB 引擎的默认隔离级别**。
    
4. **串行化（Serializable）**：最高隔离级别，所有事务串行执行，完全禁止并发，一个事务执行完成后，才能执行下一个事务。
    
    - 解决问题：脏读、不可重复读、幻读
    - 存在问题：并发性能极差，生产环境几乎不使用。
    

补充：三个并发问题的定义

- 脏读：事务 A 读取到了事务 B 未提交的修改，事务 B 回滚后，事务 A 读取到的数据就是脏数据。
- 不可重复读：同一个事务内，多次读取同一个字段，结果不一致，原因是其他事务对该字段做了修改并提交。
- 幻读：同一个事务内，多次执行同一个范围查询，返回的行数不一致，原因是其他事务插入 / 删除了符合查询条件的数据并提交。

#### 专家级拓展

- MySQL InnoDB 对隔离级别的优化：
    
    1. RR 级别解决幻读：InnoDB 在 RR 隔离级别下，通过**MVCC（多版本并发控制）** 实现快照读的幻读解决，通过**间隙锁（Gap Lock）+ 临键锁（Next-Key Lock）** 实现当前读的幻读解决，是所有数据库中唯一在 RR 级别就解决了幻读问题的引擎。
    2. 两种读模式：
        
        - 快照读：普通的 SELECT 查询，不加锁，通过 MVCC 读取数据的历史版本，不会加锁，也不会被阻塞，完全解决幻读。
        - 当前读：加锁的查询，比如`SELECT ... FOR UPDATE`、`INSERT`、`UPDATE`、`DELETE`，通过临键锁锁定查询范围，禁止其他事务在范围内插入 / 删除数据，解决幻读。
        
    
- 生产环境选型建议：
    
    1. 绝大多数场景，使用 MySQL 默认的 RR 隔离级别即可，平衡了并发性能和数据一致性。
    2. 对于对账、财务等对数据一致性要求极高的场景，可使用 RC 隔离级别，避免间隙锁导致的锁范围过大，提升并发性能，同时通过业务逻辑保证数据一致性。
    3. 严禁使用读未提交和串行化隔离级别，前者数据一致性无法保障，后者并发性能极差。
    
- 查看与修改隔离级别：
    
    sql
    
    ```
    -- 查看当前会话隔离级别
    SELECT @@transaction_isolation;
    -- 查看全局隔离级别
    SELECT @@global.transaction_isolation;
    -- 修改当前会话隔离级别
    SET SESSION TRANSACTION ISOLATION LEVEL REPEATABLE READ;
    -- 修改全局隔离级别
    SET GLOBAL TRANSACTION ISOLATION LEVEL READ COMMITTED;
    ```
    

#### 面试避坑指南

- 严禁把 MySQL 的默认隔离级别说成读已提交（RC），这是面试最高频的错误，RC 是 Oracle 的默认级别，MySQL InnoDB 默认是可重复读（RR）；
- 避免混淆四个隔离级别解决的问题和存在的问题，顺序不能说反，隔离级别越高，解决的问题越多，并发性能越差；
- 不要说「MySQL RR 级别无法解决幻读」，这是过时的知识，InnoDB 已经通过 MVCC + 间隙锁解决了幻读问题，面试说出来能大幅加分。
