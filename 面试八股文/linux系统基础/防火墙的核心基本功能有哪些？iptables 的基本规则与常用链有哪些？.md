#### 核心标准答案

##### 一、防火墙的核心基本功能

Linux 防火墙（netfilter/iptables）是工作在网络层的安全防护系统，核心作用是**根据预设的规则，对进出服务器 / 集群的网络数据包进行过滤、转发、修改，实现访问控制、安全防护、地址转换等能力**，核心基本功能如下：

1. **访问控制（核心功能）**
    
    基于源 IP、目的 IP、源端口、目的端口、协议类型、网卡接口等维度，设置允许 / 拒绝规则，实现精细化的访问控制。比如只允许指定 IP 访问服务器的 22 端口，禁止公网访问数据库的 3306 端口，这是防火墙最基础、最核心的功能。
    
1. **网络地址转换（NAT）**
    
    实现 SNAT（源地址转换）和 DNAT（目的地址转换），解决公网 IP 不足的问题，实现内网设备共享公网 IP 上网，以及公网端口映射到内网服务，这是之前讲过的 NAT 功能，防火墙是 Linux 实现 NAT 的核心载体。
    
1. **安全防护与攻击防御**
    
    抵御常见的网络攻击，比如 SYN 洪水攻击、端口扫描、ICMP 洪水攻击、DOS/DDOS 攻击等；通过限制连接数、限制数据包速率、过滤异常数据包，提升服务器的安全性。
    
1. **数据包修改与标记**
    
    对数据包的 TTL、TOS、DSCP 等字段进行修改，实现 QoS 流量控制；对数据包进行 MARK 标记，配合路由策略、流量控制，实现精细化的流量管理。
    
1. **端口转发与流量转发**
    
    将收到的数据包转发到其他 IP / 端口，实现端口映射、反向代理、流量中转的功能，无需额外的反向代理服务。
    
1. **日志审计**
    
    对匹配规则的数据包进行日志记录，记录源 IP、目的 IP、端口、协议等信息，用于安全审计、故障排查、攻击溯源。

##### 二、iptables 的基本规则与常用链

iptables 是 Linux 内核 netfilter 框架的用户态管理工具，通过规则表和规则链，管理防火墙规则，核心是**四表五链**结构。

1. **四表（规则表，按优先级从高到低）**
    
    每个表对应不同的功能场景，包含对应的规则链：
    
    表格
    
    |表名|核心作用|包含的链|
    |:--|:--|:--|
    |raw 表|最高优先级，用于处理数据包的状态跟踪，关闭连接跟踪|PREROUTING、OUTPUT|
    |mangle 表|用于修改数据包的内容，比如 TTL、TOS、MARK 标记|所有 5 个链|
    |nat 表|用于网络地址转换，SNAT/DNAT/ 端口映射|PREROUTING、POSTROUTING、OUTPUT、INPUT|
    |filter 表|默认表，用于数据包过滤，允许 / 拒绝数据包，是最常用的表|INPUT、FORWARD、OUTPUT|
    
2. **五链（规则链，数据包流经的节点）**
    
    每个链对应数据包生命周期中的一个节点，数据包到达对应节点时，会匹配链中的规则，按规则执行动作：
    
    表格
    
    |链名|触发时机|核心作用|
    |:--|:--|:--|
    |PREROUTING|数据包进入网卡后，路由判断之前|DNAT、端口映射、修改数据包|
    |INPUT|路由判断后，目标是本机的数据包，进入用户空间之前|过滤进入本机的数据包，控制入站访问|
    |FORWARD|路由判断后，目标不是本机，需要转发的数据包|过滤转发的数据包，控制内网转发|
    |OUTPUT|本机产生的数据包，发出之前|过滤本机发出的数据包，控制出站访问|
    |POSTROUTING|数据包发出网卡之前|SNAT、地址伪装、修改数据包|
    
3. **iptables 的基本规则语法**
    
    基本命令格式：
    
    bash
    
    运行
    
    ```
    iptables [-t 表名] 动作 链名 匹配条件 -j 执行动作
    ```
    
    - 常用动作：
        
        - `-A`：在链的末尾添加一条规则；
        - `-I`：在链的开头插入一条规则；
        - `-D`：删除一条规则；
        - `-L`：列出链中的所有规则；
        - `-F`：清空链中的所有规则；
        - `-P`：设置链的默认策略。
        
    - 常用匹配条件：
        
        - `-s`：源 IP 地址 / 网段；
        - `-d`：目的 IP 地址 / 网段；
        - `-p`：协议类型（tcp/udp/icmp）；
        - `--dport`：目的端口；
        - `--sport`：源端口；
        - `-i`：入站网卡；
        - `-o`：出站网卡。
        
    - 常用执行动作：
        
        - `ACCEPT`：允许数据包通过；
        - `DROP`：直接丢弃数据包，不返回任何响应；
        - `REJECT`：拒绝数据包，返回拒绝响应；
        - `SNAT`：源地址转换；
        - `DNAT`：目的地址转换；
        - `MASQUERADE`：地址伪装；
        - `LOG`：记录数据包日志。
        
    
4. **常用规则示例**
    
    bash
    
    运行
    
    ```
    # 1. 允许本地回环地址访问
    iptables -A INPUT -i lo -j ACCEPT
    # 2. 允许已建立的、相关的连接通过
    iptables -A INPUT -m state --state ESTABLISHED,RELATED -j ACCEPT
    # 3. 允许指定IP段访问22端口（SSH）
    iptables -A INPUT -s 192.168.1.0/24 -p tcp --dport 22 -j ACCEPT
    # 4. 允许公网访问80、443端口
    iptables -A INPUT -p tcp --dport 80 -j ACCEPT
    iptables -A INPUT -p tcp --dport 443 -j ACCEPT
    # 5. 禁止其他所有入站访问（默认策略，必须放在最后）
    iptables -P INPUT DROP
    # 6. 配置SNAT，内网网段共享公网IP上网
    iptables -t nat -A POSTROUTING -s 192.168.1.0/24 -o eth0 -j MASQUERADE
    # 7. 配置DNAT，公网8080端口转发到内网192.168.1.100:80
    iptables -t nat -A PREROUTING -p tcp --dport 8080 -j DNAT --to-destination 192.168.1.100:80
    ```
    

#### 专家级拓展

- 生产环境最佳实践：
    
    1. 遵循最小权限原则，默认拒绝所有访问，只开放必要的端口和 IP，白名单机制是最安全的策略；
    2. 规则顺序很重要，iptables 按顺序匹配规则，匹配到第一条符合的规则就会执行，所以要把高频、精细的规则放在前面，通用规则放在后面；
    3. 配置规则前，先确保 22 端口的访问规则已配置，避免远程连接被断开，无法登录服务器；
    4. 规则配置完成后，通过`iptables-save`保存规则，避免服务器重启后规则丢失；
    5. 开启连接跟踪模块，通过`--state`匹配已建立的连接，大幅提升规则效率，减少不必要的匹配；
    6. 生产环境优先使用 firewalld（CentOS）/ufw（Ubuntu），它们是 iptables 的前端封装工具，配置更简单，安全性更高，避免直接操作 iptables 出现错误。
    
- 进阶知识点：
    
    1. 扩展匹配模块：iptables 支持丰富的扩展模块，比如`limit`模块限制数据包速率，防御 DOS 攻击；`connlimit`模块限制单个 IP 的最大连接数，防御 CC 攻击；`string`模块匹配数据包的内容，过滤恶意请求；
    2. IPv6 防火墙：iptables 仅支持 IPv4，IPv6 需要使用 ip6tables，规则语法和 iptables 一致；
    3. 新一代防火墙：nftables 是 iptables 的替代方案，已成为 Linux 主流发行版的默认防火墙，语法更简洁，性能更高，功能更丰富。
    

#### 面试避坑指南

- 严禁混淆四表五链的作用，尤其是 filter 表和 nat 表对应的链，不能说反，这是面试高频错误；
- 避免只说规则语法，不说防火墙的核心功能，面试问这个问题，核心是考察你对防火墙的理解，而不只是背命令；
- 不要遗漏默认策略的配置，默认拒绝所有访问是防火墙安全的核心，必须放在规则的最后，否则会导致所有规则失效。
