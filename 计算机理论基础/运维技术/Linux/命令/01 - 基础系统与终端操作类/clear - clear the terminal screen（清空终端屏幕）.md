## 一、核心定位

Linux 基础终端操作命令，属于「基础系统与终端操作类」，核心作用是清空当前终端屏幕的所有内容，将光标移至屏幕顶部，保持终端界面整洁，方便后续命令的输入与输出查看，是日常终端操作的高频辅助命令。

## 二、语法格式

```bash
clear [可选参数]
# 日常使用几乎不需要参数，直接执行命令即可
```

## 三、高频核心参数

clear 命令设计极简，**无日常高频使用的参数**，核心功能仅为清空屏幕，直接执行 `clear` 即可满足 99.9% 的使用场景。

> 补充：极少数场景下可使用 `clear -x`（仅清空当前可见屏幕，保留回滚缓冲区内容），但日常几乎无需关注。

## 四、实操示例

### 1. 基础入门示例

```bash
# 示例1：直接清空当前终端屏幕
clear
# 执行效果：终端屏幕瞬间清空，之前的所有命令输出被清除，光标回到屏幕左上角，终端界面恢复整洁
```

### 2. 高频场景示例

```bash
# 场景1：执行完大量命令后，屏幕内容杂乱，清空后继续操作
# 先模拟执行几个命令
ls -la
ps -ef
df -h
# 此时屏幕内容较多，直接清空
clear
# 执行效果：快速清理屏幕，为接下来的系统巡检或日志查看提供干净的界面
```

## 五、避坑提示 & 注意事项

1. **仅清空视觉内容，不删除历史命令**：`clear` 只是 “隐藏” 了之前的屏幕输出，并非真正删除，通过终端的「向上滚动」或 `Shift+PageUp` 仍可查看之前的命令与输出；若需彻底清空历史，需配合 `history -c`（清空命令历史记录）。
2. 与快捷键 `Ctrl+L` 功能完全一致：`Ctrl+L` 是 `clear` 的终端快捷键，效果完全相同，日常操作可根据习惯选择（快捷键更高效，命令更适合写在脚本中）。
3. 脚本中谨慎使用：若在 Shell 脚本中使用 `clear`，会清空执行脚本时的所有前置输出，可能影响问题排查；除非脚本明确需要清屏展示结果，否则不建议在脚本中滥用。

## 六、同类命令对比 & 拓展

|命令 / 快捷键|核心作用|适用场景|
|:--|:--|:--|
|`clear`|清空终端屏幕，保留回滚缓冲区|日常操作，保持界面整洁，最常用|
|`Ctrl+L`|与 `clear` 完全一致，终端快捷键|日常快速清屏，效率更高|
|`reset`|重置终端（彻底清空 + 恢复终端默认设置）|终端出现乱码、显示异常时的 “救命命令”，日常不常用|

## 七、课后练习任务

1. 连续执行 `ls -la`、`whoami`、`date` 三个命令，然后用 `clear` 清空屏幕，再尝试向上滚动终端，确认仍能看到之前的命令输出。
2. 对比使用 `clear` 命令和 `Ctrl+L` 快捷键，感受两者的效果差异（实际无差异，仅操作方式不同）。
