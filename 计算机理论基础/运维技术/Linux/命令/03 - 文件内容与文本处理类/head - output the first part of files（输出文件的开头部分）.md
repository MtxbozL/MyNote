## 一、核心定位

`head` 是 Linux 系统中**最基础、最高频的文件开头预览命令**，属于「文本处理类」，无任何权限要求，普通用户可直接查看自己有权限的文件。核心作用是**快速查看文件的前 N 行（默认 10 行）、指定字节数的内容、查看多个文件的开头、配合其他命令做初步筛选**，是大文件预览、日志开头查看、文件格式确认的必备基础命令，也是 Linux 入门必学的文本查看命令。

> 补充说明：`head` 与 `tail` 是黄金搭档，`head` 查看开头，`tail` 查看结尾，两者结合可快速预览文件的首尾内容。

## 二、语法格式

bash

运行

```
# 核心语法1：查看单个文件的开头
head [可选参数] 文件

# 核心语法2：查看多个文件的开头
head [可选参数] 文件1 文件2 文件3 ...
```

> 关键说明：
> 
> 1. 无参数时，默认查看文件的**前 10 行**；
> 2. 无文件参数时，默认从 ** 标准输入（stdin）** 读取内容，即管道输入；
> 3. 查看多个文件时，默认会在每个文件前显示文件名（`==> 文件名 <==`）。

## 三、高频核心参数

表格

|参数|核心作用|高频使用场景|
|:--|:--|:--|
|无参数（直接执行`head 文件`）|查看文件的前 10 行|日常快速预览文件开头，最核心、最常用的用法|
|`-n, --lines=NUM`|指定查看的**行数**，NUM 为正整数；支持`-NUM`简写（如`-5`等价于`-n 5`）|灵活指定查看的行数，快速预览文件开头|
|`-c, --bytes=NUM`|指定查看的**字节数**，NUM 支持单位：`b`=512 字节、`k`=1KB、`M`=1MB、`G`=1GB；支持`-NUM`简写|查看指定大小的内容、二进制文件预览|
|`-q, --quiet, --silent`|查看多个文件时**不显示文件名**|合并多个文件的开头、避免文件名干扰|
|`-v, --verbose`|查看单个文件时也**显示文件名**|明确标识查看的文件、多文件对比|
|`-z, --zero-terminated`|以 NULL 字符（`\0`）作为行分隔符，而非换行符|处理带换行符的文件名、特殊格式文件|
|`--help`|输出 head 帮助信息|快速查看参数用法|
|`--version`|输出 head 版本信息|确认版本兼容性|

## 四、实操示例

### 1. 基础入门示例（新手必练，零门槛上手）

bash

运行

```
# 示例1：最基础用法，查看文件的前10行
head test.txt
# 执行效果：输出test.txt的前10行
```

bash

运行

```
# 示例2：指定查看的行数（两种写法等价）
# 写法1：-n参数
head -n 5 test.txt
# 写法2：简写-5
head -5 test.txt
# 执行效果：输出test.txt的前5行
```

bash

运行

```
# 示例3：指定查看的字节数
# 查看前100字节
head -c 100 test.txt
# 查看前1KB
head -c 1k test.txt
# 查看前1MB
head -c 1M test.txt
```

bash

运行

```
# 示例4：查看多个文件的开头
head file1.txt file2.txt file3.txt
# 执行效果示例：
# ==> file1.txt <==
# file1的前10行
#
# ==> file2.txt <==
# file2的前10行
```

### 2. 高频日常场景示例

#### 场景 1：查看多个文件时不显示文件名

bash

运行

```
# 查看多个文件的前5行，不显示文件名
head -q -n 5 file1.txt file2.txt file3.txt
# 执行效果：直接输出所有文件的前5行，无文件名标识
```

#### 场景 2：查看单个文件时也显示文件名

bash

运行

```
# 查看单个文件的前10行，强制显示文件名
head -v test.txt
# 执行效果示例：
# ==> test.txt <==
# 前10行内容
```

#### 场景 3：结合其他命令（管道输入）

bash

运行

```
# 结合ls，查看当前目录下前10个文件
ls -l | head -10

# 结合ps，查看前10个进程
ps aux | head -10

# 结合df，查看前5个分区
df -h | head -5
```

#### 场景 4：快速确认文件格式

bash

运行

```
# 查看压缩文件的前几个字节，确认文件类型
head -c 10 file.tar.gz
# 执行效果：输出压缩文件的魔数，确认文件类型
```

### 3. 进阶奇妙用法

#### 奇妙用法 1：查看除了最后 N 行的所有内容

bash

运行

```
# 查看除了最后5行的所有内容（结合tail）
head -n -5 test.txt
# 执行效果：输出test.txt除了最后5行的所有内容
```

#### 奇妙用法 2：查看文件的中间部分（结合 head 和 tail）

bash

运行

```
# 查看文件的第10行到第20行（先取前20行，再取后11行）
head -n 20 test.txt | tail -n 11
# 执行效果：输出第10行到第20行
```

#### 奇妙用法 3：批量查看目录下所有文件的开头

bash

运行

```
# 批量查看当前目录下所有.txt文件的前3行
head -n 3 *.txt
```

#### 奇妙用法 4：实时查看日志的开头（结合 tail -f）

bash

运行

```
# 先查看日志的前10行，再实时跟踪日志结尾
head -n 10 /var/log/syslog && tail -f /var/log/syslog
```

#### 奇妙用法 5：查看二进制文件的开头（结合 hexdump）

bash

运行

```
# 查看二进制文件的前100字节，以十六进制显示
head -c 100 binary_file | hexdump -C
```

## 五、避坑提示 & 注意事项

1. **新手最高频踩坑：默认是前 10 行，不是前 10 个字符**
    
    - `head` 默认查看的是**前 10 行**，不是前 10 个字符或字节；
    - 若需查看指定字节数，必须加`-c`参数；
    - 典型错误：`head test.txt`以为是查看前 10 个字符，实际是前 10 行。
    
2. **`-n`和`-c`的区别**
    
    - `-n`：按**行数**查看，以换行符为分隔；
    - `-c`：按**字节数**查看，不考虑换行符；
    - 注意：一个中文字符在 UTF-8 编码下占 3 个字节，用`-c`查看时可能会截断中文字符，导致乱码。
    
3. **大文件的处理**
    
    - `head` 查看大文件的开头时性能很高，因为只需读取文件的开头部分，无需读取整个文件；
    - 推荐：大文件预览开头用`head`，预览结尾用`tail`，完整查看用`less`。
    
4. **二进制文件的处理**
    
    - `head` 直接查看二进制文件会输出乱码，甚至输出控制字符改变终端设置；
    - 解决方法：查看二进制文件的开头需结合`hexdump -C`、`od -t x1`等工具；
    - 若终端已乱码，可执行`reset`命令恢复。
    
5. **权限不足的坑**
    
    - 查看系统文件（如`/root`、`/etc/shadow`、`/var/log/syslog`）时，普通用户会报错`head: xxx: Permission denied`；
    - 解决方法：查看系统文件需加`sudo`，但必须确认文件内容可查看，避免泄露敏感信息。
    
6. **符号链接的处理**
    
    - `head` 默认查看符号链接指向的**目标文件内容**，而非符号链接本身；
    - 若需查看符号链接本身的信息（如指向路径），需用`ls -l`或`readlink`，而非 head。
    
7. **`-n -NUM`的特殊用法**
    
    - `head -n -NUM`（注意是负号）表示查看**除了最后 NUM 行的所有内容**；
    - 这是一个非常实用的功能，但容易被忽略；
    - 示例：`head -n -5 test.txt`输出除了最后 5 行的所有内容。
    
8. **查看多个文件时的文件名显示**
    
    - 查看多个文件时，默认会在每个文件前显示文件名（`==> 文件名 <==`）；
    - 若需合并多个文件的开头，避免文件名干扰，必须加`-q`参数；
    - 若需查看单个文件时也显示文件名，必须加`-v`参数。
    

## 六、同类命令对比 & 拓展

表格

|命令|核心作用|与 head 的区别 & 适用场景|
|:--|:--|:--|
|`tail`|查看文件的结尾部分|head 查看开头，tail 查看结尾；两者是黄金搭档，快速预览文件首尾，tail 还支持实时跟踪（-f）|
|`cat`|查看文件的完整内容|优势：查看完整内容；劣势：大文件会刷屏。小文件完整查看用 cat，大文件预览开头用 head|
|`less`|分页查看文件，支持滚动、搜索、跳转|优势：功能强大，支持分页、搜索、跳转；劣势：查看开头不如 head 简洁。完整查看大文件用 less，快速预览开头用 head|
|`more`|分页查看文件，仅支持向下滚动|优势：简单易用；劣势：功能弱于 less。已逐步被 less 替代，简单分页查看可用 more|
|`sed`|流编辑器，用于文本替换、删除、插入|优势：可查看指定行范围、文本替换；劣势：查看开头不如 head 简洁。查看指定行范围用 sed，快速预览开头用 head|
|`awk`|模式驱动的文本处理语言|优势：支持模式匹配、字段提取、数据计算；劣势：查看开头不如 head 简洁。复杂文本处理用 awk，快速预览开头用 head|
|`cut`|简单字段提取|优势：简单快速，仅用于提取字段；劣势：无法查看开头。字段提取用 cut，查看开头用 head|

## 七、课后练习任务

1. 基础查看练习：在当前目录下创建一个`test.txt`文件，写入 20 行内容，用`head`查看前 10 行。
2. 指定行数练习：用`head -n 5`和`head -5`两种方式查看`test.txt`的前 5 行，对比效果。
3. 指定字节数练习：用`head -c 100`查看`test.txt`的前 100 字节，用`head -c 1k`查看前 1KB。
4. 查看多个文件练习：创建`file1.txt`、`file2.txt`、`file3.txt`三个文件，分别写入不同内容，用`head`查看它们的前 3 行，观察文件名显示。
5. 不显示文件名练习：用`head -q -n 3`查看三个文件的前 3 行，对比与上一个练习的差异。
6. 结合管道练习：用`ls -l | head -10`查看当前目录下前 10 个文件，用`ps aux | head -5`查看前 5 个进程。
7. 查看除了最后 N 行练习：用`head -n -5`查看`test.txt`除了最后 5 行的所有内容。
8. 查看中间部分练习：用`head -n 15 test.txt | tail -n 6`查看`test.txt`的第 10 行到第 15 行。
9. 快速确认文件类型练习：找一个压缩文件（如`.tar.gz`），用`head -c 10`查看其前 10 个字节，确认文件类型。
10. 结合 tail 练习：先查看`test.txt`的前 5 行，再查看后 5 行，对比两者的输出。
