## 一、核心定位

`less` 是 Linux 系统中**最强大、最常用的分页文件查看工具**，属于「文本处理类」，无任何权限要求，普通用户可直接查看自己有权限的文件。核心作用是**分页查看大文件，支持向上 / 向下滚动、正则表达式搜索、行号跳转、位置标记、实时跟踪日志、多个文件切换、过滤显示等功能**，是大文件查看、日志分析、代码阅读的首选工具，也是 Linux 入门必学的高级文本查看命令。

> 补充说明：`less` 名称来源于 “less is more”（少即是多），功能比传统的`more`更强大，支持双向滚动，是查看文件的终极工具。

## 二、语法格式

bash

运行

```
# 核心语法1：查看单个文件
less [可选参数] 文件

# 核心语法2：查看多个文件
less [可选参数] 文件1 文件2 文件3 ...

# 核心语法3：查看标准输入（管道输入）
命令 | less [可选参数]
```

> 关键说明：
> 
> 1. 无文件参数时，默认从 ** 标准输入（stdin）** 读取内容，即管道输入；
> 2. `less` 是交互式工具，进入后需使用键盘命令操作，按`q`退出；
> 3. 查看多个文件时，支持在文件间切换。

## 三、高频核心参数与内置命令

### 1. 核心启动参数

表格

|参数|核心作用|高频使用场景|
|:--|:--|:--|
|无参数（直接执行`less 文件`）|基本分页查看，支持所有内置命令|日常查看文件，最核心、最常用的用法|
|`-N, --LINE-NUMBERS`|显示**行号**|定位问题行、代码查看、日志分析|
|`-i, --ignore-case`|搜索时**忽略大小写**|模糊搜索，不区分大小写差异|
|`-S, --chop-long-lines`|长行**不换行**，截断显示，需左右滚动查看|查看宽表格、避免长行刷屏|
|`-m, --long-prompt`|显示**百分比进度**和文件名|查看大文件时了解当前位置|
|`-M, --LONG-PROMPT`|显示更详细的提示（行号、百分比、文件名）|更全面的位置信息|
|`-n, --no-line-numbers`|不显示行号（默认行为）|提升大文件查看性能|
|`-r, --raw-control-chars`|显示**原始控制字符**（如颜色）|查看带颜色的日志、保留终端颜色|
|`-X, --no-init`|退出时**不清屏**，保留文件内容在终端|查看后仍需参考文件内容|
|`-F, --quit-if-one-screen`|内容少于一屏时**自动退出**|小文件自动退出，无需按 q|
|`--help`|输出 less 帮助信息|快速查看参数与内置命令用法|
|`-V, --version`|输出 less 版本信息|确认版本兼容性|

### 2. 核心内置导航命令（进入 less 后使用）

表格

|命令|核心作用|高频使用场景|
|:--|:--|:--|
|`Space` / `Page Down` / `f` / `Ctrl+F`|向下翻一页|快速向下浏览|
|`b` / `Page Up` / `Ctrl+B`|向上翻一页|快速向上浏览|
|`j` / `Ctrl+N` / `↓`|向下滚动一行|逐行向下查看|
|`k` / `Ctrl+P` / `↑`|向上滚动一行|逐行向上查看|
|`g` / `Home` / `<`|跳转到**文件开头**|快速回到文件第一行|
|`G` / `End` / `>`|跳转到**文件结尾**|快速跳到文件最后一行|
|`:n` / `n`|跳转到**第 n 行**（如`:100`跳转到第 100 行）|精准定位到指定行|
|`n%`|跳转到**n% 位置**（如`50%`跳转到中间）|快速定位到文件的百分比位置|
|`:n`|查看**下一个文件**（多文件时）|多文件间切换|
|`:p`|查看**上一个文件**（多文件时）|多文件间切换|
|`:d`|从列表中**删除当前文件**（多文件时）|减少文件列表|

### 3. 核心内置搜索命令

表格

|命令|核心作用|高频使用场景|
|:--|:--|:--|
|`/模式`|向下搜索**模式**（支持正则表达式）|从当前位置向下搜索|
|`?模式`|向上搜索**模式**（支持正则表达式）|从当前位置向上搜索|
|`n`|跳转到**下一个匹配项**|继续搜索下一个|
|`N`|跳转到**上一个匹配项**|返回搜索上一个|
|`&模式`|仅显示**匹配模式的行**（过滤显示）|过滤无关内容，专注查看匹配行|
|`Esc+u`|清除搜索高亮|取消高亮显示|

### 4. 核心内置其他命令

表格

|命令|核心作用|高频使用场景|
|:--|:--|:--|
|`q` / `Q` / `:q` / `ZZ`|退出 less|查看完成后退出|
|`F`|实时跟踪文件结尾（类似`tail -f`），按`Ctrl+C`停止|实时查看日志更新|
|`ma` / `'a`|标记当前位置为`a`（可用任意小写字母） / 跳转到标记`a`|临时标记位置，快速跳转|
|`''`（两个单引号）|跳转到**上一个位置**|返回之前的位置|
|`=` / `Ctrl+G`|显示当前位置信息（文件名、行号、字节数、百分比）|查看当前详细位置|
|`-N` / `-S` / `-i`等|在 less 内部**切换启动参数**（如`-N`切换行号显示）|动态调整显示选项|
|`h` / `H`|显示 less 的**帮助信息**|查看所有内置命令|
|`!命令`|执行**Shell 命令**（如`!ls`），按回车返回 less|不退出 less 执行临时命令|
|`v`|用**默认编辑器**（通常是 vim）打开当前文件|需要编辑时快速打开编辑器|

### 1. 基础入门示例（新手必练，零门槛上手）

bash

运行

```
# 示例1：最基础用法，分页查看文件
less test.txt
# 执行效果：进入less界面，显示文件内容，按q退出
```

bash

运行

```
# 示例2：显示行号查看文件
less -N test.txt
# 执行效果：每行前显示行号
```

bash

运行

```
# 示例3：显示百分比进度查看文件
less -m test.txt
# 执行效果：底部显示当前位置的百分比
```

bash

运行

```
# 示例4：退出时不清屏，保留内容
less -X test.txt
# 执行效果：按q退出后，文件内容仍保留在终端
```

bash

运行

```
# 示例5：查看多个文件
less file1.txt file2.txt file3.txt
# 执行效果：进入less查看file1.txt，按:n切换到file2.txt，按:p切换回file1.txt
```

## 五、避坑提示 & 注意事项

1. **新手最高频认知：大文件必须用 less，不要用 cat**
    
    - `cat` 会一次性输出文件的完整内容，查看大文件（如几百 MB、GB 级的日志）时会导致终端严重刷屏，无法查看；
    - `less` 仅加载当前屏幕的内容，性能极高，支持双向滚动、搜索、跳转；
    - 强制要求：查看大文件**必须用 less**，小文件（<100 行）可用 cat。
    
2. **搜索时的特殊字符转义**
    
    - less 的搜索支持正则表达式，若需匹配特殊字符（如`.`、`*`、`?`、`+`、`|`、`()`、`[]`、`^`、`$`），**必须用反斜杠`\`转义**；
    - 错误示例：搜索`error.log`时输入`/error.log`（`.`会匹配任意字符）；
    - 正确示例：输入`/error\.log`。
    
3. **退出时不清屏的参数`-X`**
    
    - 默认情况下，less 退出时会清屏，文件内容会从终端消失；
    - 若需查看后仍参考文件内容，必须加`-X`参数，即`less -X 文件`；
    - 进阶技巧：可在`~/.bashrc`中添加`alias less='less -X'`，让 less 默认不清屏。
    
4. **实时跟踪`F`的使用**
    
    - 按`F`进入实时跟踪模式后，会持续显示文件的新内容，类似`tail -f`；
    - 按`Ctrl+C`停止跟踪，返回普通查看模式，不要直接按`q`，否则会退出 less；
    - 推荐：实时跟踪日志优先用`less +F 日志文件`，比`tail -f`更强大，可随时停止跟踪并向上查看历史内容。
    
5. **权限不足的坑**
    
    - 查看系统文件（如`/root`、`/etc/shadow`、`/var/log/syslog`）时，普通用户会报错`less: xxx: Permission denied`；
    - 解决方法：查看系统文件需加`sudo`，但必须确认文件内容可查看，避免泄露敏感信息。
    
6. **二进制文件的处理**
    
    - `less` 直接查看二进制文件会输出乱码，甚至输出控制字符改变终端设置；
    - 解决方法：查看二进制文件需结合`hexdump -C`、`od -t x1`等工具；
    - 若终端已乱码，可执行`reset`命令恢复。
    
7. **行号显示的性能问题**
    
    - 查看超大文件（如 GB 级）时，加`-N`显示行号会降低性能，因为 less 需要计算每行的行号；
    - 解决方法：查看超大文件时不加`-N`，需要时用`:n`跳转到指定行。
    
8. **多个文件的切换**
    
    - 查看多个文件时，用`:n`切换到下一个文件，用`:p`切换到上一个文件，用`:d`删除当前文件；
    - 用`:args`查看当前文件列表。
    
9. **帮助信息的查看**
    
    - less 的内置命令非常多，记不住时按`h`或`H`查看帮助信息；
    - 帮助信息也用 less 查看，按`q`退出帮助。
    

## 六、同类命令对比 & 拓展

表格

|命令|核心作用|与 less 的区别 & 适用场景||
|:--|:--|:--|---|
|`more`|分页查看文件，仅支持向下滚动|优势：简单易用，系统默认；劣势：仅支持向下滚动，功能弱于 less。已逐步被 less 替代，简单分页查看可用 more，复杂查看必须用 less||
|`cat`|查看文件的完整内容|优势：简单快速，查看完整内容；劣势：大文件会刷屏。小文件完整查看用 cat，大文件必须用 less||
|`head`|查看文件的开头部分|优势：快速查看开头；劣势：仅能查看开头。快速预览开头用 head，完整查看用 less||
|`tail`|查看文件的结尾部分，支持实时跟踪（-f）|优势：快速查看结尾、实时跟踪；劣势：仅能查看结尾。快速预览结尾用 tail，实时跟踪可用 tail -f 或 less +F，完整查看用 less||
|`grep`|按模式匹配行|优势：速度极快，仅用于模式匹配；劣势：无法查看完整内容。简单模式匹配用 grep，匹配后查看上下文用 grep -C，或用 grep|less 分页查看|
|`vim` / `vi`|文本编辑器|优势：功能强大，支持编辑；劣势：仅查看不如 less 简洁。编辑文件用 vim，仅查看用 less，less 中按 v 可快速打开 vim||
|`nano`|简单文本编辑器|优势：简单易用，适合新手；劣势：功能弱于 vim，仅查看不如 less 简洁。新手编辑可用 nano，仅查看用 less||
|`most`|另一个分页查看工具|优势：支持彩色、多窗口；劣势：非系统默认，需安装。功能与 less 类似，less 是系统默认，优先用 less||

## 七、课后练习任务

1. 基础查看练习：找一个大文件（或创建一个包含 100 行的`test.txt`），用`less`查看，按`q`退出。
2. 显示行号练习：用`less -N`查看`test.txt`，观察行号显示。
3. 导航练习：用`less`查看`test.txt`，练习以下操作：
    
    - 按`Space`向下翻页
    - 按`b`向上翻页
    - 按`j`向下滚动一行
    - 按`k`向上滚动一行
    - 按`g`跳转到开头
    - 按`G`跳转到结尾
    - 按`:50`跳转到第 50 行
    
4. 搜索练习：在`test.txt`中写入几行包含 "test"、"Test"、"TEST" 的内容，用`less`查看：
    
    - 输入`/test`向下搜索
    - 按`n`跳转到下一个匹配项
    - 按`N`跳转到上一个匹配项
    - 用`less -i`查看，输入`/test`，验证是否忽略大小写
    
5. 过滤显示练习：用`less`查看`test.txt`，输入`&test`，仅显示包含 "test" 的行，再次输入`&`清除过滤。
6. 多个文件练习：创建`file1.txt`、`file2.txt`、`file3.txt`三个文件，用`less file1.txt file2.txt file3.txt`查看：
    
    - 按`:n`切换到下一个文件
    - 按`:p`切换到上一个文件
    
7. 实时跟踪练习：找一个日志文件（或用`tail -f /dev/null > test.log &`创建一个测试日志），用`less +F test.log`实时跟踪，按`Ctrl+C`停止跟踪。
8. 标记位置练习：用`less`查看`test.txt`：
    
    - 滚动到第 30 行，输入`ma`标记
    - 滚动到第 70 行，输入`mb`标记
    - 输入`'a`跳转到标记 a
    - 输入`'b`跳转到标记 b
    - 输入`''`跳转到上一个位置
    
9. 结合管道练习：用`ps aux | less`分页查看所有进程，用`df -h | less`分页查看磁盘使用情况。
10. 退出练习：用`less test.txt`查看，按`q`退出；再用`less -X test.txt`查看，按`q`退出，对比两者的区别（是否清屏）。
11. 帮助练习：用`less`查看任意文件，按`h`查看帮助信息，浏览所有内置命令，按`q`退出帮助。
