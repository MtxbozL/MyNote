
## 一、核心定位

`uniq` 是 Linux 系统中**去除相邻重复行的核心文本处理工具**，属于「文本处理类」，无任何权限要求，普通用户可直接处理自己有权限的文件。核心作用是**去除相邻的重复行、仅显示重复行、仅显示唯一行、显示重复次数、忽略大小写、跳过指定字段 / 字符比较**，是文本去重、重复数据统计、日志清洗的必备基础命令，也是 Linux 入门必学的文本处理命令。

> 补充说明：`uniq` 仅对**相邻的重复行**有效，非相邻的重复行无法去除，因此通常需先配合`sort`排序，使所有重复行相邻后再去重；与`sort`是黄金搭档，`sort -u`可直接实现排序后去重。

## 二、语法格式

bash

运行

```
# 核心语法1：处理文件，输出到标准输出
uniq [可选参数] [输入文件 [输出文件]]

# 核心语法2：处理标准输入（管道输入）
命令 | uniq [可选参数]
```

> 关键说明：
> 
> 1. 无参数时，默认去除**相邻的重复行**，仅输出唯一行；
> 2. 无输入文件时，默认从 ** 标准输入（stdin）** 读取内容，即管道输入；
> 3. **必须相邻重复行才有效**，非相邻重复行需先排序；
> 4. 若指定输出文件，结果会写入输出文件，而非标准输出。

## 三、高频核心参数（按功能分类）

### 1. 基础去重类

表格

|参数|核心作用|高频使用场景|
|:--|:--|:--|
|无参数（直接执行`uniq 文件`）|去除**相邻的重复行**，仅输出唯一行|日常简单去重，最基础、最常用的用法|
|`-d, --repeated`|仅输出**重复的行**（每组重复行仅输出一次）|查找重复数据、定位重复行|
|`-D, --all-repeated[=METHOD]`|输出**所有重复的行**（每组重复行全部输出），METHOD 可选：`none`（默认，无分隔）、`prepend`（每组前加分隔）、`separate`（每组间加分隔）|查看所有重复行、完整重复数据统计|
|`-u, --unique`|仅输出**唯一的行**（从未重复的行）|提取唯一数据、去除所有重复行|
|`-c, --count`|在每行前**显示重复次数**（重复次数为该行出现的次数）|统计重复次数、数据频率分析|

### 2. 比较规则类

表格

|参数|核心作用|高频使用场景|
|:--|:--|:--|
|`-i, --ignore-case`|比较时**忽略大小写**，将小写字母视为大写字母比较|模糊去重、不区分大小写的重复行去除|
|`-f, --skip-fields=N`|比较时**跳过前 N 个字段**，从第 N+1 个字段开始比较|表格数据去重、忽略前几列的差异|
|`-s, --skip-chars=N`|比较时**跳过前 N 个字符**，从第 N+1 个字符开始比较|忽略行首的差异、特定位置开始比较|
|`-w, --check-chars=N`|比较时**仅比较前 N 个字符**，忽略 N 个字符之后的内容|仅比较行首的差异、部分内容去重|

### 3. 输出类

表格

|参数|核心作用|高频使用场景|
|:--|:--|:--|
|`-z, --zero-terminated`|以 NULL 字符（`\0`）作为行分隔符，而非换行符|处理带换行符的文件名、特殊格式文件|
|`--help`|输出 uniq 帮助信息|快速查看参数与用法|
|`--version`|输出 uniq 版本信息|确认版本兼容性|

## 四、实操示例

### 1. 基础入门示例（新手必练，零门槛上手）

bash

运行

```
# 示例1：最基础用法，去除相邻重复行（注意：仅相邻有效）
uniq test.txt
# 执行效果：输出test.txt的内容，去除相邻的重复行
```

bash

运行

```
# 示例2：先排序再去重（去除所有重复行，非仅相邻）
sort test.txt | uniq
# 执行效果：先排序使所有重复行相邻，再去除所有重复行
```

bash

运行

```
# 示例3：显示重复次数
sort test.txt | uniq -c
# 执行效果示例：
#       3 apple
#       2 banana
#       1 cherry
```

bash

运行

```
# 示例4：仅显示重复的行
sort test.txt | uniq -d
# 执行效果示例：
# apple
# banana
```

bash

运行

```
# 示例5：仅显示唯一的行
sort test.txt | uniq -u
# 执行效果示例：
# cherry
```

### 2. 高频日常场景示例

#### 场景 1：忽略大小写去重

bash

运行

```
# 忽略大小写，去除重复行
sort -f test.txt | uniq -i
# 执行效果：不区分大小写，将Apple和apple视为相同
```

#### 场景 2：跳过前 N 个字段比较

bash

运行

```
# 处理表格数据（空格分隔），跳过前1个字段，从第2个字段开始比较去重
sort -k 2 test.txt | uniq -f 1
# 执行效果：仅比较第2个字段及之后的内容，忽略第1个字段的差异
```

#### 场景 3：跳过前 N 个字符比较

bash

运行

```
# 跳过前2个字符，从第3个字符开始比较去重
sort test.txt | uniq -s 2
# 执行效果：仅比较第3个字符及之后的内容，忽略前2个字符的差异
```

#### 场景 4：仅比较前 N 个字符

bash

运行

```
# 仅比较前5个字符，忽略之后的内容
sort test.txt | uniq -w 5
# 执行效果：仅比较前5个字符，5个字符之后的内容不影响去重
```

#### 场景 5：结合其他命令（管道输入）

bash

运行

```
# 结合ps，去除重复的进程名
ps aux | awk '{print $11}' | sort | uniq

# 结合ls，去除重复的文件扩展名
ls -l | awk '{print $NF}' | awk -F '.' '{print $NF}' | sort | uniq

# 结合grep，先过滤再去重
grep 'error' test.log | sort | uniq -c
```

```
# 统计重复次数，按次数降序排序，取Top10
```

```
# 直接排序后去重，等价于sort | uniq

```

## 五、避坑提示 & 注意事项

1. **新手最高频致命坑：仅对相邻重复行有效，必须先排序**
    
    - `uniq` 仅对**相邻的重复行**有效，非相邻的重复行无法去除；
    - 强制要求：去除所有重复行**必须先排序**，即`sort test.txt | uniq`；
    - 典型错误：`uniq test.txt`（非相邻重复行无法去除）；
    - 正确示例：`sort test.txt | uniq`或`sort -u test.txt`。
    
2. **`-d`和`-u`的区别**
    
    - `-d`：仅输出**重复的行**（每组重复行仅输出一次）；
    - `-u`：仅输出**唯一的行**（从未重复的行）；
    - 注意：`-d`和`-u`是互斥的，不能同时使用；
    - 典型错误：`uniq -d -u test.txt`（语法错误）。
    
3. **`-c`的计数规则**
    
    - `-c` 显示的重复次数是该行**出现的总次数**，不是重复的次数（即出现 1 次显示 1，出现 3 次显示 3）；
    - 注意：计数是在排序后统计的，所以必须先排序；
    - 典型错误：`uniq -c test.txt`（未排序，计数错误）；
    - 正确示例：`sort test.txt | uniq -c`。
    
4. **字段跳过的参数**
    
    - `-f` 跳过的是**字段**，默认字段分隔符是任意空白字符；
    - `-s` 跳过的是**字符**，从行首开始计数；
    - 注意：字段和字符是不同的概念，不要混淆；
    - 典型错误：`uniq -f 2 test.txt`（未排序，字段跳过无效）；
    - 正确示例：`sort -k 3 test.txt | uniq -f 2`。
    
5. **大文件的性能优化**
    
    - 处理大文件时，`sort | uniq` 性能很高，但需注意避免不必要的操作；
    - 尽量用`sort -u`替代`sort | uniq`，一步完成，性能更好；
    - 结合`awk`按字段去重无需排序，性能更高，如`awk '!seen[$2]++' test.txt`。
    
6. **二进制文件的处理**
    
    - `uniq` 是文本处理工具，直接处理二进制文件会导致文件损坏；
    - 解决方法：不要用 uniq 处理二进制文件，二进制文件处理用`hexdump`、`dd`等工具。
    
7. **权限不足的坑**
    
    - 处理系统文件（如`/root`、`/etc/shadow`、`/var/log/syslog`）时，普通用户会报错`uniq: cannot open 'xxx': Permission denied`；
    - 解决方法：处理系统文件需加`sudo`，但必须确认文件内容可处理，避免泄露敏感信息。
    
8. **符号链接的处理**
    
    - `uniq` 默认处理符号链接指向的**目标文件内容**，而非符号链接本身；
    - 若需处理符号链接本身的信息（如指向路径），需用`ls -l`或`readlink`，而非 uniq。
    
9. **`-D`的分隔选项**
    
    - `-D` 输出所有重复的行，支持`prepend`（每组前加分隔）、`separate`（每组间加分隔）；
    - 注意：`-D`和`-d`、`-u`是互斥的，不能同时使用。
    

## 六、同类命令对比 & 拓展

表格

|命令|核心作用|与 uniq 的区别 & 适用场景|||
|:--|:--|:--|---|---|
|`sort -u`|排序后直接去重|优势：一步完成，性能更好；劣势：仅能去重，无法显示重复次数。简单去重优先用 sort -u，需显示重复次数用 sort|uniq -c||
|`awk`|模式驱动的文本处理语言|优势：支持按任意字段去重、无需排序、保留整行、功能更强大；劣势：学习曲线更陡。简单去重用 uniq，复杂去重用 awk|||
|`sed`|流编辑器，用于文本替换、删除、插入|优势：可简单去除相邻重复行；劣势：无法统计重复次数、无法处理非相邻重复行。简单相邻去重用 sed，复杂去重用 uniq|||
|`tr`|字符转换、删除、压缩|优势：简单快速，仅用于字符级操作；劣势：无法行级去重。字符级操作用 tr，行级去重用 uniq|||
|`head`/`tail`|取前 N 行 / 后 N 行|优势：简单快速，仅用于取首尾；劣势：无法去重。取去重后的前 N 行 / 后 N 行用 sort|uniq|head/tail|
|`grep`|按模式匹配行|优势：速度极快，仅用于模式匹配；劣势：无法去重。模式匹配后去重用 grep|sort|uniq|

## 七、课后练习任务

1. 基础去重练习：创建一个`test.txt`文件，写入几行包含相邻重复和非相邻重复的内容（如 apple、apple、banana、apple、cherry），用`uniq test.txt`去重，观察仅相邻重复行被去除；再用`sort test.txt | uniq`去重，观察所有重复行被去除。
2. 显示重复次数练习：用`sort test.txt | uniq -c`统计重复次数，观察计数规则。
3. 仅显示重复行练习：用`sort test.txt | uniq -d`仅显示重复的行，验证效果。
4. 仅显示唯一行练习：用`sort test.txt | uniq -u`仅显示唯一的行，验证效果。
5. 忽略大小写练习：在`test.txt`中添加几行包含大小写字母的重复内容（如 Apple、apple、Banana、banana），用`sort -f test.txt | uniq -i`忽略大小写去重，验证效果。
6. 跳过字段练习：创建一个表格数据文件`data.txt`（空格分隔），包含 3 列（ID、姓名、城市），用`sort -k 2 data.txt | uniq -f 1`跳过前 1 个字段去重，验证效果。
7. 跳过字符练习：在`test.txt`中添加几行前 2 个字符不同但之后内容相同的行（如 01apple、02apple、03banana、04banana），用`sort test.txt | uniq -s 2`跳过前 2 个字符去重，验证效果。
8. 仅比较前 N 个字符练习：在`test.txt`中添加几行前 5 个字符相同但之后内容不同的行（如 apple123、apple456、banana789、banana012），用`sort test.txt | uniq -w 5`仅比较前 5 个字符去重，验证效果。
9. 结合 sort -u 练习：用`sort -u test.txt`直接排序后去重，对比与`sort test.txt | uniq`的效果。
10. 统计重复次数最多的 Top10 练习：用`sort test.txt | uniq -c | sort -nr | head -10`统计重复次数最多的 Top10。
11. 仅显示重复次数大于 N 的行练习：用`sort test.txt | uniq -c | awk '$1 > 2'`仅显示重复次数大于 2 的行。
12. 输出所有重复的行练习：用`sort test.txt | uniq -D --separate`输出所有重复的行，每组间加分隔。
13. 结合 awk 按字段去重练习：用`awk '!seen[$2]++' data.txt`按第 2 个字段去重，保留第一次出现的整行，对比与`sort -k 2 data.txt | uniq -f 1`的效果。
14. 结合其他命令练习：用`ps aux | awk '{print $11}' | sort | uniq`去除重复的进程名，用`ls -l | awk '{print $NF}' | awk -F '.' '{print $NF}' | sort | uniq`去除重复的文件扩展名。
15. 帮助练习：用`uniq --help`查看帮助信息，浏览所有参数。
