## 一、核心定位

`cat` 是 Linux 系统中**最基础、最高频的文件查看与内容拼接命令**，属于「文本处理类」，无任何权限要求，普通用户可直接查看自己有权限的文件。核心作用是**查看单个 / 多个文本文件的内容、按顺序拼接多个文件、结合重定向创建小文件、非交互式写入内容、显示控制字符**，是文本文件查看、内容合并、脚本初始化的必备基础命令，也是 Linux 入门必学的第一个文本查看命令。

> 补充说明：`cat` 的名称来源于 “concatenate”（拼接），其核心功能是拼接文件，查看文件是其最常用的附带功能。

## 二、语法格式

```bash
# 核心语法1：查看单个/多个文件的内容
cat [可选参数] 文件1 文件2 文件3 ...

# 核心语法2：拼接多个文件并输出到标准输出
cat [可选参数] 文件1 文件2 > 拼接后的文件

# 核心语法3：结合重定向创建小文件或写入内容
cat > 新文件 << 'EOF'
文件内容
EOF
```

> 关键说明：
> 
> 1. 无文件参数时，默认从 ** 标准输入（stdin）** 读取内容，按`Ctrl+D`结束输入；
> 2. 拼接多个文件时，按命令中文件的**顺序**依次合并；
> 3. `>` 是覆盖重定向，会清空目标文件后写入；`>>` 是追加重定向，会在目标文件末尾追加内容，不会清空原文件。

## 三、高频核心参数

|参数|核心作用|高频使用场景|
|:--|:--|:--|
|无参数（直接执行`cat 文件`）|查看文件的完整内容，无任何修饰|日常快速查看小文本文件，最核心、最常用的用法|
|`-n, --number`|显示**所有行的行号**，包括空行|查看带行号的文件内容、定位问题行、代码查看|
|`-b, --number-nonblank`|仅显示**非空行的行号**，空行不编号|查看代码或文本时，仅对有效内容编号，避免空行干扰|
|`-s, --squeeze-blank`|压缩连续的空行，将多个连续空行合并为一个|查看格式混乱的文件、压缩冗余空行、美化输出|
|`-E, --show-ends`|在每行的**末尾显示`$`符号**|查看行尾是否有隐藏的空格、定位换行问题|
|`-T, --show-tabs`|将**制表符（Tab）显示为`^I`**|区分制表符和空格、定位缩进问题|
|`-A, --show-all`|显示**所有控制字符**，等价于`-vET`|查看文件中的所有隐藏字符、调试格式问题|
|`-v, --show-nonprinting`|显示除制表符和换行符外的**非打印字符**，用`^`或`M-`前缀表示|查看文件中的隐藏控制字符|
|`--help`|输出命令帮助信息|快速查看参数用法|
|`--version`|输出命令版本信息|确认命令兼容性|

## 四、实操示例

### 1. 基础入门示例（新手必练，零门槛上手）

```bash
# 示例1：最基础用法，查看单个文件的内容
cat test.txt
# 执行效果：输出test.txt的完整内容到终端
```

```bash
# 示例2：查看多个文件的内容（按顺序拼接）
cat file1.txt file2.txt file3.txt
# 执行效果：依次输出file1.txt、file2.txt、file3.txt的内容
```

```bash
# 示例3：显示所有行的行号
cat -n test.txt
# 执行效果示例：
#      1  第一行内容
#      2
#      3  第三行内容
```

```bash
# 示例4：仅显示非空行的行号
cat -b test.txt
# 执行效果示例：
#      1  第一行内容
#
#      2  第三行内容
```

```bash
# 示例5：压缩连续的空行
cat -s test.txt
# 执行效果：将多个连续空行合并为一个
```

### 2. 高频日常场景示例

#### 场景 1：拼接多个文件为一个新文件

```bash
# 方式1：覆盖拼接，清空目标文件后写入
cat file1.txt file2.txt file3.txt > merged.txt
# 执行效果：将file1-3的内容按顺序合并为merged.txt，原merged.txt会被清空

# 方式2：追加拼接，在目标文件末尾追加
cat file4.txt >> merged.txt
# 执行效果：将file4.txt的内容追加到merged.txt的末尾，原内容保留
```

#### 场景 2：结合重定向创建小文件（非交互式写入）

```bash
# 方式1：覆盖创建新文件，输入内容后按Ctrl+D结束
cat > new_file.txt
# 输入内容...
# 按Ctrl+D结束

# 方式2：Here Document方式，无需Ctrl+D，指定结束标记（如EOF）
cat > new_file.txt << 'EOF'
第一行内容
第二行内容
第三行内容
EOF
# 执行效果：自动创建new_file.txt并写入内容，遇到EOF结束
```

#### 场景 3：追加内容到已存在的文件

```bash
# 方式1：交互式追加，按Ctrl+D结束
cat >> existing_file.txt
# 输入追加内容...
# 按Ctrl+D结束

# 方式2：Here Document方式追加
cat >> existing_file.txt << 'EOF'
追加的第一行
追加的第二行
EOF
```

#### 场景 4：结合其他命令（如 grep、awk）

```bash
# 结合grep，先查看文件再过滤
cat access.log | grep "error"

# 结合awk，先查看文件再提取字段
cat data.txt | awk '{print $1, $3}'

# 结合sort，先查看文件再排序
cat test.txt | sort
```

### 3. 进阶奇妙用法

#### 奇妙用法 1：查看文件中的隐藏字符

```bash
# 显示所有控制字符，包括行尾$、制表符^I、其他非打印字符
cat -A test.txt
# 执行效果示例：
# 第一行内容$
# ^I第二行内容（制表符显示为^I）
# 第三行内容$
```

#### 奇妙用法 2：查看非文本文件（结合 hexdump 或 od）

```bash
# 直接用cat查看二进制文件会乱码，需结合hexdump或od
# 方式1：结合hexdump，以十六进制和ASCII显示
cat binary_file | hexdump -C

# 方式2：结合od，以八进制或十六进制显示
cat binary_file | od -t x1
```

#### 奇妙用法 3：反向查看文件（结合 tac）

```bash
# tac是cat的反向，从最后一行开始显示
cat test.txt | tac
# 或直接用tac
tac test.txt
```

#### 奇妙用法 4：实时查看日志文件（结合 tail -f）

```bash
# 虽然tail -f更常用，但结合cat也可以实现类似效果（不推荐，仅作示例）
# 推荐直接用tail -f
tail -f /var/log/syslog
```

#### 奇妙用法 5：批量查看文件（结合 find）

```bash
# 查找所有.txt文件并批量查看内容
find . -name "*.txt" -exec cat {} \;

# 查找所有.txt文件并拼接为一个文件
find . -name "*.txt" -exec cat {} > all_txt_merged.txt \;
```

#### 奇妙用法 6：清空文件内容（不删除文件）

```bash
# 方式1：用cat清空文件
cat /dev/null > test.txt

# 方式2：更简洁的方式（> test.txt）
> test.txt
```

## 五、避坑提示 & 注意事项

1. **新手最高频踩坑：大文件直接用 cat 会刷屏**
    
    - `cat` 会一次性输出文件的**完整内容**，查看大文件（如几百 MB、GB 级的日志）时会导致终端严重刷屏，无法查看；
    - 解决方法：查看大文件**必须用`less`或`more`**，支持分页、滚动、搜索；
    - 推荐：小文件（<100 行）用 cat，大文件用 less。
    
2. **二进制文件用 cat 会乱码**
    
    - `cat` 是文本查看工具，直接查看二进制文件（如图片、视频、可执行程序、压缩包）会导致终端乱码，甚至输出控制字符改变终端设置；
    - 解决方法：查看二进制文件需结合`hexdump -C`、`od -t x1`、`file`等工具；
    - 若终端已乱码，可执行`reset`命令恢复。
    
3. **`>`覆盖文件无提示，无法恢复**
    
    - `>` 覆盖重定向会**直接清空目标文件**，无任何提示，且 Linux 默认没有回收站，覆盖后数据几乎无法恢复；
    - 强制要求：使用`>`前，**必须先确认目标文件不存在或不重要**，或先用`ls`确认；
    - 进阶技巧：可在`~/.bashrc`中添加`alias cp='cp -i'`、`alias mv='mv -i'`，但`>`无法通过别名保护，需格外谨慎。
    
4. **拼接文件时顺序错误**
    
    - `cat` 拼接文件时，严格按照**命令中文件的顺序**依次合并；
    - 典型错误：`cat file2.txt file1.txt > merged.txt`会导致合并后的文件顺序颠倒；
    - 解决方法：拼接前先确认文件顺序，或用`ls -v`按自然顺序排序后再拼接。
    
5. **特殊字符显示问题**
    
    - 默认情况下，`cat` 不会显示行尾空格、制表符、其他非打印字符；
    - 若需查看隐藏字符，必须加`-A`（显示所有）、`-E`（显示行尾 $）、`-T`（显示制表符 ^I）参数；
    - 典型应用：排查代码缩进问题（区分 Tab 和空格）、定位行尾多余空格。
    
6. **权限不足的坑**
    
    - 查看系统文件（如`/root`、`/etc/shadow`、`/var/log/syslog`）时，普通用户会报错`cat: xxx: Permission denied`；
    - 解决方法：查看系统文件需加`sudo`，但必须确认文件内容可查看，避免泄露敏感信息。
    
7. **符号链接的处理**
    
    - `cat` 默认查看符号链接指向的**目标文件内容**，而非符号链接本身；
    - 若需查看符号链接本身的信息（如指向路径），需用`ls -l`或`readlink`，而非 cat。
    
8. **Here Document 的结束标记**
    
    - 使用 Here Document（`<< 'EOF'`）时，结束标记`EOF`必须**单独占一行，且前后不能有任何字符（包括空格）**；
    - 若结束标记不满足要求，cat 会一直等待输入，不会结束；
    - 解决方法：结束标记`EOF`单独写一行，前后无空格。
    

## 六、同类命令对比 & 拓展

表格

|命令|核心作用|与 cat 的区别 & 适用场景|
|:--|:--|:--|
|`less`|分页查看文件，支持滚动、搜索、跳转|优势：支持分页、不刷屏、功能强大（搜索、跳转、行号）；劣势：查看小文件不如 cat 简洁。小文件用 cat，大文件 / 需搜索用 less，less 是查看文件的首选工具|
|`more`|分页查看文件，仅支持向下滚动|优势：简单易用；劣势：功能弱于 less，仅支持向下滚动。已逐步被 less 替代，简单分页查看可用 more|
|`head`|查看文件的前 N 行（默认 10 行）|优势：快速查看文件开头；劣势：仅能查看开头。查看文件开头用 head，与 tail 互补|
|`tail`|查看文件的后 N 行（默认 10 行），支持实时跟踪（-f）|优势：快速查看文件结尾、实时跟踪日志；劣势：仅能查看结尾。查看文件结尾用 tail，实时查看日志用 tail -f，与 head 互补|
|`tac`|反向查看文件，从最后一行开始显示|是 cat 的反向，仅用于反向查看；cat 正向查看，tac 反向查看，两者互补|
|`nl`|显示文件行号，功能类似 cat -n|优势：行号格式更灵活；劣势：功能单一。仅需显示行号可用 nl，通用查看用 cat -n|
|`od`/`hexdump`|查看二进制文件，以十六进制 / 八进制显示|专门用于查看二进制文件；cat 用于文本文件，二进制文件用 od/hexdump|
|`grep`|按模式匹配行|优势：速度极快，仅用于模式匹配；劣势：无法查看完整内容。简单查看用 cat，模式匹配用 grep，两者常配合使用|

## 七、课后练习任务

1. 基础查看练习：在当前目录下创建一个`test.txt`文件，写入几行内容（包含空行），用`cat`查看完整内容。
2. 显示行号练习：用`cat -n`和`cat -b`两种方式查看`test.txt`，对比两者的行号显示差异。
3. 拼接文件练习：创建`file1.txt`、`file2.txt`、`file3.txt`三个文件，分别写入不同内容，用`cat`将它们拼接为`merged.txt`，查看拼接后的内容。
4. 创建文件练习：用 Here Document 方式创建一个`new_file.txt`，写入 3 行内容，查看创建的文件。
5. 结合 grep 练习：创建一个`access.log`文件，写入几行包含 "error" 和 "info" 的日志，用`cat access.log | grep "error"`过滤出包含 "error" 的行。
6. 压缩空行练习：在`test.txt`中添加几个连续的空行，用`cat -s`查看，对比压缩前后的差异。
7. 结合 find 练习：用`find`查找当前目录下所有`.txt`文件，用`cat`批量查看它们的内容，然后拼接为`all_txt.txt`。# cat - concatenate files and print on the standard output（拼接文件并打印到标准输出）
