## 一、核心定位

`more` 是 Linux/Unix 系统中**经典、兼容性极强的轻量分页文本查看工具**，属于「文本处理类」，无任何权限要求，普通用户可直接查看自己有权限的文件。核心作用是按屏分页查看文本文件，支持基础的向下滚动、关键词搜索、行号跳转，是 POSIX 标准命令，所有类 Unix 系统均原生支持。

它是比`less`更早出现的分页工具，名称来源于 “more”（查看更多内容），功能更精简、资源占用更低，适合极简环境、跨平台脚本、新手入门的轻量分页查看，复杂场景下已逐步被功能更强大的`less`替代。

## 二、语法格式

bash

运行

```
# 核心语法1：查看单个文件
more [可选参数] 文件

# 核心语法2：查看多个文件
more [可选参数] 文件1 文件2 文件3 ...

# 核心语法3：查看标准输入（管道输入）
命令 | more [可选参数]
```

> 关键说明：
> 
> 1. 无文件参数时，默认从 ** 标准输入（stdin）** 读取内容，即管道输入；
> 2. `more` 是交互式工具，进入后需使用键盘命令操作，按`q`退出；
> 3. 默认每页显示行数与终端高度一致，底部会显示当前阅读进度百分比；
> 4. 仅本地文件支持向上回翻，管道流式输入不支持向上回翻。

## 三、高频核心参数与内置命令

### 1. 核心启动参数

表格

|参数|核心作用|高频使用场景|
|:--|:--|:--|
|无参数（直接执行`more 文件`）|基础分页查看，默认以终端高度为一页|日常简单分页查看文件，最核心、最常用的用法|
|`-num`|指定每页显示的行数（如`-5`代表每页 5 行）|小屏终端、自定义预览行数、精准控制显示范围|
|`-d, --silent`|显示新手友好的操作提示（Press space to continue, 'q' to quit.），按错键无蜂鸣、有引导|新手入门使用，避免误操作|
|`-s, --squeeze-blank`|压缩连续的空行，将多个连续空行合并为 1 行|美化格式混乱的文件、减少无效翻页|
|`-c, --print-over`|清屏后显示内容，不滚动刷新，直接在当前页顶部重绘内容|避免滚动干扰、提升阅读体验|
|`-p, --clean-print`|清屏后显示内容，清除当前页之前的内容，与`-c`功能类似，用于兼容不同发行版|跨平台脚本、适配不同终端环境|
|`+/pattern`|打开文件后直接跳转到第一个匹配 pattern 的行，支持基础正则|日志排查、快速定位目标关键词|
|`+num`|从第 num 行开始显示文件内容，跳过前 num 行|从指定位置开始查看、跳过文件头部冗余内容|
|`--help`|输出 more 帮助信息|快速查看参数与命令用法|
|`-V, --version`|输出 more 版本信息|确认跨平台兼容性|

### 2. 核心内置交互命令（进入 more 后使用）

表格

|命令|核心作用|高频使用场景|
|:--|:--|:--|
|`Space` / `f` / `Ctrl+F` / `Ctrl+V`|向下翻一页|快速向下浏览内容|
|`Enter` / `j` / `Ctrl+N` / `↓`|向下滚动 1 行|逐行查看细节内容|
|`b` / `Ctrl+B`|向上翻一页（仅本地文件有效，管道输入无效）|回看之前的内容|
|`/pattern`|向下搜索关键词 pattern，支持基础正则表达式|定位目标内容|
|`n`|跳转到下一个匹配的搜索结果|继续查找下一个关键词|
|`:n`|查看下一个文件（多文件查看时）|多文件间顺序切换|
|`:p`|查看上一个文件（多文件查看时）|多文件间回退切换|
|`=` / `Ctrl+G`|显示当前行号、文件名、阅读进度|确认当前阅读位置|
|`h` / `?`|显示 more 的帮助信息|查看所有内置命令|
|`q` / `Q` / `:q`|退出 more|结束查看|
|`!命令`|临时执行 Shell 命令，按回车返回 more|不退出 more 执行临时操作|

## 四、实操示例

### 1. 基础入门示例（新手必练，零门槛上手）

bash

运行

```
# 示例1：最基础用法，分页查看文件
more test.txt
# 执行效果：进入more界面，显示第一页内容，底部显示阅读百分比，按Space翻页，q退出
```

bash

运行

```
# 示例2：指定每页显示的行数
more -5 test.txt
# 执行效果：每页仅显示5行，按Space向下翻5行，按Enter向下翻1行
```

bash

运行

```
# 示例3：从指定行开始查看文件
more +20 test.txt
# 执行效果：直接跳过前19行，从第20行开始显示内容
```

bash

运行

```
# 示例4：打开文件直接跳转到匹配关键词的位置
more +/error test.log
# 执行效果：打开test.log后，直接跳转到第一个包含error的行，高亮匹配内容
```

bash

运行

```
# 示例5：同时查看多个文件
more file1.txt file2.txt file3.txt
# 执行效果：先显示file1.txt，按:n切换到file2.txt，按:p切换回file1.txt，每个文件开头会显示文件名
```

### 2. 高频日常场景示例

#### 场景 1：新手友好模式，显示操作引导提示

bash

运行

```
# 显示友好操作提示，按错键有引导，无蜂鸣
more -d test.txt
# 执行效果：底部固定显示「Press space to continue, 'q' to quit.」，误操作会显示帮助提示
```

#### 场景 2：压缩连续空行，美化文件显示

bash

运行

```
# 将多个连续空行合并为1行，避免无效翻页
more -s messy_file.txt
```

#### 场景 3：结合管道，分页查看命令输出

bash

运行

```
# 分页查看系统所有进程
ps aux | more

# 分页查看磁盘使用情况
df -h | more

# 先过滤日志，再分页查看
grep 'warning' test.log | more
```

#### 场景 4：清屏显示，避免滚动干扰

bash

运行

```
# 每次翻页先清屏，在当前页顶部重绘内容，无滚动刷新
more -c test.txt
```

### 3. 进阶奇妙用法

#### 奇妙用法 1：查看带行号的文件内容（结合 nl）

bash

运行

```
# 给文件添加行号后分页显示，方便定位问题行
nl test.txt | more
```

#### 奇妙用法 2：查看二进制文件的可读内容（结合 strings）

bash

运行

```
# 提取二进制文件中的可读字符串，分页查看，避免乱码
strings binary_file | more
```

#### 奇妙用法 3：在 more 内部临时执行 Shell 命令

bash

运行

```
# 进入more后，不退出执行ls命令
more test.txt
# 输入!ls，执行完成后按回车返回more
```

#### 奇妙用法 4：批量查看目录下所有日志文件

bash

运行

```
# 按顺序分页查看当前目录下所有.log文件
ls *.log | xargs more
```

#### 奇妙用法 5：先预览文件开头，再分页查看完整内容

bash

运行

```
# 先显示前20行，再分页查看完整日志
head -n 20 /var/log/syslog && more /var/log/syslog
```

## 五、避坑提示 & 注意事项

1. **新手最高频踩坑：管道输入无法向上回翻**
    
    - `more` 的`b`键向上翻页，**仅对本地文件有效**；管道输入（如`ps aux | more`）是流式传输，无内容缓存，无法回翻，按`b`会无响应；
    - 解决方法：需要双向滚动的场景，优先用`less`；仅需单向向下查看可用`more`。
    
2. **功能局限性：核心仅支持单向向下滚动**
    
    - `more` 仅支持基础的分页、逐行滚动、简单搜索，不支持长行左右滚动、位置标记、实时跟踪日志、高级正则搜索等功能；
    - 适用场景：极简环境、跨平台脚本、简单分页查看；大文件分析、日志排查、代码阅读等复杂场景优先用`less`。
    
3. **长行处理的格式混乱坑**
    
    - `more` 默认会将长行自动换行显示，无原生的长行截断、左右滚动功能，会导致宽表格、超长行的内容格式完全混乱；
    - 解决方法：需要查看长行不换行的场景，必须用`less -S`。
    
4. **二进制文件的乱码风险**
    
    - 直接用`more`查看二进制文件（图片、压缩包、可执行程序）会输出乱码，甚至触发终端控制字符，导致终端设置异常；
    - 解决方法：查看二进制文件需结合`strings`、`hexdump -C`等工具；若终端已乱码，执行`reset`命令恢复。
    
5. **权限不足的坑**
    
    - 查看系统文件（如`/root`、`/etc/shadow`、`/var/log/syslog`）时，普通用户会报错`more: cannot open 'xxx': Permission denied`；
    - 解决方法：查看系统文件需加`sudo`，但必须确认文件内容可查看，避免泄露敏感信息。
    
6. **搜索的特殊字符转义**
    
    - `more` 的搜索支持基础正则表达式，匹配`.`、`*`、`?`、`^`、`$`等特殊字符时，**必须用反斜杠`\`转义**；
    - 错误示例：搜索`test.txt`输入`/test.txt`（`.`会匹配任意字符）；
    - 正确示例：输入`/test\.txt`。
    
7. **跨平台兼容性差异**
    
    - `more` 是 POSIX 标准命令，所有类 Unix 系统均支持，但 GNU more（Linux）和 BSD more（MacOS、FreeBSD）的参数略有差异；
    - 跨平台脚本中，仅使用`-num`、`-d`、`-s`、`+/pattern`、`+num`等通用参数，避免使用发行版专属参数。
    

## 六、同类命令对比 & 拓展

表格

|命令|核心作用|与 more 的区别 & 适用场景||
|:--|:--|:--|---|
|`less`|双向分页查看文件，功能更强大|more 是经典轻量分页工具，仅支持基础向下滚动；less 支持双向滚动、长行左右滚动、位置标记、实时跟踪日志、高级搜索，功能远强于 more。简单分页查看、跨平台脚本可用 more，复杂场景、大文件查看必须用 less||
|`cat`|查看文件完整内容|优势：简单快速，输出完整内容；劣势：大文件会刷屏。小文件完整查看用 cat，大文件分页查看用 more||
|`head`|查看文件开头部分|优势：快速预览开头；劣势：仅能查看开头。快速预览开头用 head，分页查看完整内容用 more||
|`tail`|查看文件结尾部分，支持实时跟踪|优势：快速预览结尾、实时跟踪日志；劣势：仅能查看结尾。快速预览结尾用 tail，分页查看完整内容用 more||
|`grep`|按模式匹配行|优势：快速过滤匹配内容；劣势：无法查看完整上下文。简单过滤用 grep，过滤后分页查看用 `grep 模式|more`|
|`vim`/`vi`|文本编辑器|优势：支持编辑、强大的搜索和导航；劣势：仅查看不如 more 简洁。编辑文件用 vim，仅分页查看用 more||

## 七、课后练习任务

1. 基础查看练习：创建一个包含 50 行内容的`test.txt`文件，用`more`分页查看，练习按 Space 翻页、按 q 退出。
2. 指定每页行数练习：用`more -5 test.txt`查看文件，验证每页仅显示 5 行，练习按 Space 翻页、按 Enter 逐行滚动。
3. 从指定行开始练习：用`more +20 test.txt`查看文件，验证是否从第 20 行开始显示内容。
4. 关键词跳转练习：在`test.txt`中写入几行包含 "test" 的内容，用`more +/test test.txt`打开文件，验证是否直接跳转到第一个匹配 "test" 的行。
5. 新手友好模式练习：用`more -d test.txt`查看文件，观察底部的操作提示，按 h 查看帮助信息。
6. 压缩空行练习：在`test.txt`中添加多个连续空行，用`more -s test.txt`查看，验证连续空行是否被压缩为 1 行。
7. 结合管道练习：用`ps aux | more`分页查看系统进程，尝试按 b 键回翻，验证管道输入无法回翻的特性。
8. 搜索练习：用`more`打开`test.txt`，输入`/test`向下搜索 "test"，按 n 跳转到下一个匹配项。
9. 多文件查看练习：创建`file1.txt`、`file2.txt`、`file3.txt`三个文件，用`more`同时查看，练习按`:n`切换到下一个文件，按`:p`切换到上一个文件。
10. 带行号查看练习：用`nl test.txt | more`查看带行号的文件内容，验证行号显示与分页效果。
