## 一、核心定位

`grep` 是 Linux 系统中**最核心、最高频的模式匹配文本搜索工具**，属于「文本处理类」，无任何权限要求，普通用户可直接搜索自己有权限的文件。核心作用是**按固定字符串或正则表达式匹配文本行，支持忽略大小写、反向匹配、递归搜索、显示上下文、计数、颜色高亮等功能**，是日志分析、文本过滤、代码查找、配置排查的核心工具，也是 Linux 入门必学的高级文本处理命令。

> 补充说明：`grep` 名称来源于 “Global Regular Expression Print”（全局正则表达式打印）；Linux 系统中默认支持`grep`（基本正则）、`egrep`（扩展正则，等价于`grep -E`）、`fgrep`（固定字符串，等价于`grep -F`）。

## 二、语法格式

```bash
# 核心语法1：搜索文件
grep [可选参数] '模式' 文件1 文件2 ...

# 核心语法2：搜索标准输入（管道输入）
命令 | grep [可选参数] '模式'
```

> 关键语法说明：
> 
> 1. **模式（Pattern）**：必选，用于匹配行的字符串或正则表达式，**必须用单引号包裹**，避免 Shell 解析其中的`$`、`*`、`?`等特殊字符；
> 2. **文件**：可选，指定要搜索的文件，支持通配符`*`、`?`；
> 3. **无文件时**：默认从 ** 标准输入（stdin）** 读取内容，即管道输入；
> 4. **默认行为**：打印匹配模式的**整行内容**，支持颜色高亮（现代终端默认开启）。

## 三、高频核心参数

| 参数                             | 核心作用                                      | 高频使用场景                      |
| :----------------------------- | :---------------------------------------- | :-------------------------- |
| 无参数（直接执行`grep '模式' 文件`）        | 基本匹配，打印包含模式的整行                            | 日常快速搜索文本，最核心、最常用的用法         |
| `-i, --ignore-case`            | 忽略大小写匹配                                   | 模糊搜索，不区分大小写差异               |
| `-v, --invert-match`           | 反向匹配，打印**不包含**模式的行                        | 过滤不需要的行、排除干扰内容              |
| `-n, --line-number`            | 显示匹配行的**行号**                              | 定位问题行、代码查找、日志分析             |
| `-c, --count`                  | 仅打印**匹配的行数**，不打印具体内容                      | 统计匹配次数、快速评估数量               |
| `-l, --files-with-matches`     | 仅打印**包含匹配行的文件名**，不打印具体内容                  | 批量查找包含目标内容的文件               |
| `-L, --files-without-match`    | 仅打印**不包含匹配行的文件名**                         | 批量查找不包含目标内容的文件              |
| `-w, --word-regexp`            | 匹配**整词**，模式必须作为独立单词出现（前后为非单词字符）           | 精确匹配单词、避免部分匹配               |
| `-x, --line-regexp`            | 匹配**整行**，模式必须与整行完全一致                      | 精确匹配整行内容                    |
| `-r, -R, --recursive`          | 递归搜索目录及其所有子目录下的文件                         | 批量搜索目录树、代码库查找               |
| `-A NUM, --after-context=NUM`  | 显示匹配行**后 NUM 行**的上下文                      | 查看匹配行后的内容、日志分析              |
| `-B NUM, --before-context=NUM` | 显示匹配行**前 NUM 行**的上下文                      | 查看匹配行前的内容、日志分析              |
| `-C NUM, --context=NUM`        | 显示匹配行**前后各 NUM 行**的上下文，等价于`-A NUM -B NUM` | 查看完整上下文、日志分析首选              |
| `--color=auto/always/never`    | 颜色高亮匹配内容，`auto`为终端自动开启（默认）                | 突出显示匹配内容、提升可读性              |
| `-E, --extended-regexp`        | 使用**扩展正则表达式**（ERE），等价于`egrep`             | 复杂正则匹配、支持 ``、`+`、`?`、`()` 等 |
| `-F, --fixed-strings`          | 使用**固定字符串**匹配，不解析正则表达式，等价于`fgrep`         | 快速搜索包含特殊字符的字符串、性能更好         |
| `-f FILE, --file=FILE`         | 从**文件中读取模式**，每行一个模式                       | 批量匹配多个模式、模式列表管理             |
| `--exclude=GLOB`               | 递归搜索时**排除**匹配 GLOB 的文件                    | 排除特定类型文件、提升搜索性能             |
| `--exclude-dir=GLOB`           | 递归搜索时**排除**匹配 GLOB 的目录                    | 排除特定目录、提升搜索性能               |
| `--help`                       | 输出 grep 帮助信息                              | 快速查看参数与正则表达式用法              |
| `--version`                    | 输出 grep 版本信息                              | 确认版本兼容性                     |

## 四、实操示例

### 1. 基础入门示例（新手必练，零门槛上手）

```bash
# 示例1：最基础用法，搜索包含"error"的行
grep 'error' test.log
# 执行效果：打印test.log中所有包含"error"的行
```

```bash
# 示例2：忽略大小写匹配
grep -i 'error' test.log
# 执行效果：匹配"error"、"Error"、"ERROR"等所有大小写变体
```

```bash
# 示例3：反向匹配，打印不包含"error"的行
grep -v 'error' test.log
# 执行效果：打印test.log中所有不包含"error"的行
```

```bash
# 示例4：显示匹配行的行号
grep -n 'error' test.log
# 执行效果示例：
# 10:  error message 1
# 25:  error message 2
```

```bash
# 示例5：仅统计匹配的行数
grep -c 'error' test.log
# 执行效果示例：5
```

### 2. 高频日常场景示例

#### 场景 1：递归搜索目录树

```bash
# 递归搜索当前目录及其所有子目录下的.txt文件，查找包含"TODO"的行
grep -r 'TODO' --include='*.txt' .

# 递归搜索时排除.git目录和.log文件
grep -r 'function' --exclude-dir='.git' --exclude='*.log' /data/code
```

#### 场景 2：显示匹配行的上下文

```bash
# 显示匹配行前后各3行的上下文
grep -C 3 'error' test.log

# 仅显示匹配行后5行
grep -A 5 'error' test.log

# 仅显示匹配行前2行
grep -B 2 'error' test.log
```

#### 场景 3：匹配整词，避免部分匹配

```bash
# 匹配整词"test"，避免匹配"testing"、"test123"等
grep -w 'test' test.txt
```

#### 场景 4：从文件中读取多个模式批量匹配

```bash
# 创建模式文件patterns.txt，每行一个模式
cat > patterns.txt << 'EOF'
error
warning
critical
EOF

# 从patterns.txt读取模式，批量匹配
grep -f patterns.txt test.log
```

#### 场景 5：结合其他命令（管道输入）

```bash
# 结合ps，查找包含"nginx"的进程
ps aux | grep 'nginx'

# 结合df，查找使用率超过80%的分区
df -h | grep -E '[8-9][0-9]%|100%'

# 结合ls，查找包含"log"的文件
ls -l | grep 'log'
```

### 3. 进阶奇妙用法

#### 奇妙用法 1：扩展正则表达式（grep -E）

```bash
# 匹配"error"或"warning"
grep -E 'error|warning' test.log

# 匹配以"2026"开头的行
grep -E '^2026' test.log

# 匹配以".txt"结尾的行
grep -E '\.txt$' test.log

# 匹配包含3个连续数字的行
grep -E '[0-9]{3}' test.log
```

#### 奇妙用法 2：统计每个模式的匹配次数

```bash
# 统计"error"、"warning"、"info"各自的匹配次数
for pattern in error warning info; do
    count=$(grep -c "$pattern" test.log)
    echo "$pattern: $count"
done
```

#### 奇妙用法 3：结合 xargs 批量处理匹配的文件

```bash
# 查找所有包含"TODO"的.txt文件，批量复制到/tmp/todo目录
mkdir -p /tmp/todo
grep -rl 'TODO' --include='*.txt' . | xargs -I {} cp {} /tmp/todo/
```

#### 奇妙用法 4：颜色高亮（默认已开启，可强制开启）

```bash
# 强制开启颜色高亮，突出显示匹配内容
grep --color=always 'error' test.log
```

#### 奇妙用法 5：仅显示匹配的部分，而非整行（结合 - o）

```bash
# 仅显示匹配的"error"单词，而非整行
grep -o 'error' test.log

# 结合正则，仅显示匹配的IP地址
grep -E -o '[0-9]+\.[0-9]+\.[0-9]+\.[0-9]+' access.log
```

## 五、避坑提示 & 注意事项

1. **新手最高频踩坑：模式必须用单引号包裹**
    
    - 模式中若包含`$`、`*`、`?`、`()`、`|`等特殊字符，**必须用单引号包裹**，否则 Shell 会先解析这些字符，导致 grep 接收到的模式错误；
    - 错误示例：`grep error$ test.log`（Shell 会解析`$`为变量结尾，导致模式错误）；
    - 正确示例：`grep 'error$' test.log`。
    
2. **正则表达式的特殊字符转义**
    
    - 若需匹配正则表达式的特殊字符（如`.`、`*`、`?`、`+`、`|`、`()`、`[]`、`^`、`$`），**必须用反斜杠`\`转义**；
    - 错误示例：`grep 'error.log' test.log`（`.`会匹配任意字符，导致匹配 "errorxlog" 等）；
    - 正确示例：`grep 'error\.log' test.log`。
    
3. **大文件搜索的性能优化**
    
    - 搜索大文件时，直接用`grep '模式' 文件`比`cat 文件 | grep '模式'`性能更好，因为避免了管道的额外开销；
    - 递归搜索大目录时，用`--exclude`和`--exclude-dir`排除不需要的文件和目录，可大幅提升性能；
    - 若仅需匹配固定字符串，用`grep -F`（fgrep）比普通 grep 性能更好，因为无需解析正则表达式。
    
4. **`-w`和`-x`的区别**
    
    - `-w`：匹配整词，模式作为独立单词出现，前后为非单词字符（字母、数字、下划线为单词字符）；
    - `-x`：匹配整行，模式必须与整行完全一致；
    - 典型错误：用`-w`匹配整行，导致匹配失败；
    - 解决方法：根据需求选择正确的参数，整词匹配用`-w`，整行匹配用`-x`。
    
5. **二进制文件的处理**
    
    - `grep` 默认会搜索二进制文件，但可能会输出乱码或控制字符；
    - 若需排除二进制文件，可加`-I`参数（忽略二进制文件）；
    - 示例：`grep -rI 'error' .`。
    
6. **递归搜索的目录路径**
    
    - 递归搜索时，目录路径`.`代表当前目录，`/`代表根目录（需谨慎，搜索根目录会很慢）；
    - 若需搜索特定目录，直接指定目录路径，如`grep -r 'error' /var/log`。
    
7. **颜色高亮的配置**
    
    - 现代 Linux 系统默认`grep`已开启`--color=auto`，终端会自动高亮匹配内容；
    - 若需永久强制开启颜色高亮，可在`~/.bashrc`中添加`alias grep='grep --color=auto'`。
    
8. **扩展正则表达式的使用**
    
    - 基本正则（BRE）和扩展正则（ERE）的语法略有不同，ERE 支持`|`（或）、`+`（一次或多次）、`?`（零次或一次）、`()`（分组）、`{n,m}`（重复次数）等，无需转义；
    - 若需使用这些功能，必须加`-E`参数（或用`egrep`）。
    

## 六、同类命令对比 & 拓展

|命令|核心作用|与 grep 的区别 & 适用场景|
|:--|:--|:--|
|`awk`|模式驱动的文本处理语言|优势：支持字段提取、数据计算、格式化输出；劣势：简单模式匹配不如 grep 快。简单模式匹配用 grep，复杂文本处理用 awk，两者常配合使用|
|`sed`|流编辑器，用于文本替换、删除、插入|优势：擅长文本替换、行编辑；劣势：模式匹配能力弱于 grep。模式匹配用 grep，文本替换用 sed，两者常配合使用|
|`egrep`|扩展正则表达式 grep|等价于`grep -E`，功能完全相同；仅为历史遗留命令，推荐直接用`grep -E`|
|`fgrep`|固定字符串 grep|等价于`grep -F`，功能完全相同；仅为历史遗留命令，推荐直接用`grep -F`|
|`ag`/`ack`/`rg`|更快的代码搜索工具|优势：速度远快于 grep，自动忽略版本控制目录（如.git），支持更多语言特性；劣势：非系统默认工具，需安装。代码库搜索优先用 ag/ack/rg，通用文本搜索用 grep|
|`find`|在目录树中查找文件|优势：按文件名、类型、大小、时间查找文件；劣势：无法搜索文件内容。查找文件用 find，搜索文件内容用 grep，两者常配合使用（find -exec grep）|
|`cut`|简单字段提取|优势：简单快速，仅用于提取字段；劣势：无法模式匹配。简单字段提取用 cut，模式匹配用 grep|

## 七、课后练习任务

1. 基础匹配练习：创建一个`test.log`文件，写入几行包含 "error"、"Error"、"warning"、"info" 的日志，用`grep`搜索包含 "error" 的行。
2. 忽略大小写练习：用`grep -i`搜索包含 "error" 的行，验证是否匹配所有大小写变体。
3. 反向匹配练习：用`grep -v`搜索不包含 "error" 的行。
4. 显示行号练习：用`grep -n`搜索包含 "error" 的行，查看行号。
5. 计数练习：用`grep -c`分别统计 "error"、"warning"、"info" 的匹配次数。
6. 递归搜索练习：在当前目录下创建几个子目录和`.txt`文件，写入 "TODO"，用`grep -r`递归搜索包含 "TODO" 的文件。
7. 显示上下文练习：用`grep -C 2`搜索包含 "error" 的行，查看前后各 2 行的上下文。
8. 匹配整词练习：在`test.txt`中写入 "test"、"testing"、"test123"，用`grep -w`搜索 "test"，验证是否仅匹配整词。
9. 结合管道练习：用`ps aux | grep 'bash'`查找包含 "bash" 的进程。
10. 扩展正则练习：用`grep -E`搜索以 "2026" 开头或以 ".log" 结尾的行。
