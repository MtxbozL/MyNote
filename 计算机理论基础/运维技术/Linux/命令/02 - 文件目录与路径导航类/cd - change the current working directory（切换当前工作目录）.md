路径：绝对路径和相对路径
## 一、核心定位

`cd` 是 Linux 系统中**最基础、最高频的目录导航命令**，属于「02 - 文件目录与路径导航类」，无任何权限要求，普通用户可直接执行。核心作用是**切换当前 Shell 会话的工作目录**，是终端文件操作、路径跳转、脚本目录管理的核心基础，也是 Linux 入门必学的第一个文件操作命令。

> 补充说明：`cd` 是 Shell 内置命令，而非外部二进制程序，可通过 `type cd` 验证；其行为受 Shell 环境变量`CDPATH`影响，可实现快速目录跳转。

## 二、语法格式

bash

运行

```
# 核心语法（极简设计，日常99%场景直接使用）
cd [可选参数] [目标路径]
```

> 关键说明：
> 
> 1. 无目标路径时，默认切换到**当前用户的家目录**（等价于`cd ~`）；
> 2. 目标路径支持 4 种类型：
> 
> - **绝对路径**：以`/`开头，从根目录开始的完整路径（如`/home/ubuntu/linux_practice`）；
> - **相对路径**：不以`/`开头，从当前目录开始的路径（如`../backup`、`./doc`）；
> - **特殊符号路径**：`~`（家目录）、`.`（当前目录）、`..`（上一级目录）、`-`（上一次目录）；
> - **符号链接路径**：指向其他目录的软链接，可通过参数控制是否跟随。

## 三、高频核心参数

表格

|参数|核心作用|高频使用场景|
|:--|:--|:--|
|无参数（直接执行`cd`）|切换到当前用户的家目录|快速回到家目录，日常操作最常用|
|`-L, --logical`|跟随符号链接（默认行为），切换到符号链接指向的目标目录|日常通过软链接快速跳转，默认无需显式指定|
|`-P, --physical`|不跟随符号链接，直接切换到符号链接本身所在的物理目录|排查软链接问题、修改软链接本身属性时使用|
|`-`（短横线，非参数）|切换到**上一次所在的工作目录**，并输出目标目录路径|两个目录间快速来回切换，效率提升神器|
|`--help`|输出命令帮助信息|快速查看参数用法，解决符号链接相关疑问|

### 2. 高频日常场景示例

#### 场景 1：脚本中安全切换目录，避免后续命令在错误目录执行

脚本中直接使用`cd`切换目录，若目录不存在会导致后续命令在错误目录执行，必须做安全校验：

bash

运行

```
#!/bin/bash
# 脚本中安全切换目录的标准写法
TARGET_DIR="/data/backup"

# 方式1：先判断目录是否存在，再切换
if [ -d "$TARGET_DIR" ]; then
    cd "$TARGET_DIR" || { echo -e "\e[1;31m 切换目录失败！\e[0m"; exit 1; }
    echo "已切换到目录：$(pwd)"
    # 后续在目标目录执行的命令
    ls -la
else
    echo -e "\e[1;31m 目录 $TARGET_DIR 不存在！\e[0m"
    exit 1
fi

# 方式2：一行命令安全切换，失败则退出
cd "$TARGET_DIR" || exit 1
echo "已切换到目录：$(pwd)"
```

#### 场景 2：结合`CDPATH`环境变量，实现常用目录快速跳转

配置`CDPATH`后，无需输入完整路径，直接输入目录名即可快速跳转到常用目录：

bash

运行

```
# 1. 临时配置CDPATH（当前会话生效）
export CDPATH=.:~:/data:/home/ubuntu/workspace

# 2. 测试快速跳转：直接输入目录名，无需完整路径
# 假设/data目录下有backup子目录，直接输入
cd backup
# 执行效果：自动跳转到/data/backup，无需输入完整路径

# 3. 永久配置CDPATH（写入~/.bashrc）
echo 'export CDPATH=.:~:/data:/home/ubuntu/workspace' >> ~/.bashrc
source ~/.bashrc
```

#### 场景 3：批量目录切换与操作

bash

运行

```
#!/bin/bash
# 批量在多个目录中执行操作
DIRS=("/data/backup" "/home/ubuntu/doc" "/var/log")

# 保存初始目录，操作完成后切回
INIT_DIR=$(pwd)

for DIR in "${DIRS[@]}"; do
    echo "正在处理目录：$DIR"
    cd "$DIR" || continue  # 切换失败则跳过该目录，继续下一个
    # 在当前目录执行操作，比如查看文件数量
    echo "当前目录文件数量：$(ls -1 | wc -l)"
done

# 操作完成后切回初始目录
cd "$INIT_DIR"
echo "已回到初始目录：$(pwd)"
```

### 3. 进阶奇妙用法

#### 奇妙用法 1：目录栈操作（pushd/popd/dirs），实现多目录快速切换

`cd` 只能在两个目录间切换，结合目录栈命令可实现多目录快速跳转：

bash

运行

```
# 1. pushd：切换到目标目录，并将当前目录压入栈
cd ~
pushd /data/backup  # 切换到/data/backup，家目录入栈
pushd /var/log       # 切换到/var/log，/data/backup入栈
pushd /home/ubuntu/doc  # 切换到/doc，/var/log入栈

# 2. dirs：查看当前目录栈
dirs -v  # 带编号查看目录栈
# 执行效果示例：
# 0  /home/ubuntu/doc
# 1  /var/log
# 2  /data/backup
# 3  /home/ubuntu

# 3. popd：弹出栈顶目录，并切换到该目录
popd  # 切换到/var/log，/doc出栈
popd +2  # 切换到栈中第2个目录（/data/backup），并删除该目录

# 4. 目录栈快速切换：pushd +N 切换到栈中第N个目录
pushd +1  # 切换到栈中第1个目录
```

#### 奇妙用法 2：为常用目录设置别名，一键跳转

bash

运行

```
# 1. 临时设置别名（当前会话生效）
alias cdb='cd /data/backup'
alias cdl='cd /var/log'
alias cdw='cd ~/workspace'

# 2. 测试别名：直接输入别名即可跳转
cdb  # 一键跳转到/data/backup

# 3. 永久设置别名（写入~/.bashrc）
echo 'alias cdb="cd /data/backup"' >> ~/.bashrc
echo 'alias cdl="cd /var/log"' >> ~/.bashrc
echo 'alias cdw="cd ~/workspace"' >> ~/.bashrc
source ~/.bashrc
```

#### 奇妙用法 3：一行命令切换到深层目录，无需逐层输入

bash

运行

```
# 场景：需要切换到/home/ubuntu/workspace/project/src/main/java/com/example
# 方式1：利用大括号扩展，快速创建并切换到深层目录
mkdir -p /home/ubuntu/workspace/project/src/main/java/com/example
cd $_  # $_ 代表上一条命令的最后一个参数，即目标目录

# 方式2：一行命令直接切换到深层目录
cd /home/ubuntu/workspace/project/src/main/java/com/example

# 方式3：结合CDPATH，直接输入最后一级目录名跳转
export CDPATH=.:/home/ubuntu/workspace/project/src/main/java/com
cd example
```

#### 奇妙用法 4：符号链接目录的跟随与不跟随切换

bash

运行

```
# 1. 创建一个符号链接
ln -s /data/backup ~/link_to_backup

# 2. 默认跟随符号链接（-L）
cd ~/link_to_backup
pwd  # 输出：/data/backup（跟随到目标目录）

# 3. 不跟随符号链接，直接进入物理目录（-P）
cd -P ~/link_to_backup
pwd  # 输出：/home/ubuntu（符号链接本身所在的物理目录）
```

## 五、避坑提示 & 注意事项

1. **新手最高频踩坑：相对路径 vs 绝对路径**
    
    - 绝对路径：以`/`开头，从根目录开始，**永远不会出错**，脚本中优先使用；
    - 相对路径：不以`/`开头，从当前目录开始，**依赖当前工作目录**，当前目录变化后会失效；
    - 典型错误：脚本中用相对路径`cd backup`，若脚本从其他目录执行，会切换到错误的目录，甚至导致`rm -rf`误删数据。
    
2. **脚本中 cd 失败的致命坑**
    
    - 脚本中直接写`cd 目录`，若目录不存在或无权限，`cd`会执行失败，但**脚本会继续执行后续命令**，导致后续命令在错误的目录执行，极易引发数据灾难；
    - 强制要求：脚本中所有`cd`命令必须加错误处理，如`cd 目录 || exit 1`，或先判断目录是否存在再切换。
    
3. **空格与特殊字符的处理**
    
    - 目录名包含空格、`&`、`!`、`()`等特殊字符时，必须用**双引号包裹**，或用反斜杠转义；
    - 错误示例：`cd My Documents` 会被识别为两个参数，导致报错；
    - 正确示例：`cd "My Documents"` 或 `cd My\ Documents`。
    
4. **`cd .` 的无用性**
    
    - `cd .` 切换到当前目录，没有任何实际意义，日常操作和脚本中绝对不要使用。
    
5. **`CDPATH` 的副作用**
    
    - 配置`CDPATH`后，`cd`会优先在`CDPATH`指定的目录中查找目标目录，可能导致切换到意料之外的目录；
    - 建议`CDPATH`的第一个路径为`.`（当前目录），优先在当前目录查找，避免意外跳转。
    
6. **符号链接的`pwd`输出差异**
    
    - 跟随符号链接切换目录后，`pwd`默认输出符号链接的逻辑路径，`pwd -P`输出物理路径；
    - 脚本中获取真实物理路径，必须使用`pwd -P`，避免依赖符号链接的逻辑路径。
    

## 六、同类命令对比 & 拓展

表格

|命令|核心作用|与 cd 的区别 & 适用场景|
|:--|:--|:--|
|`pwd`|查看当前工作目录的完整路径|与 cd 互补，cd 用于切换目录，pwd 用于确认当前目录，两者是目录操作的黄金搭档|
|`ls`|列出目录下的文件与子目录|与 cd 配合使用，cd 切换目录后，用 ls 查看目录内容，是日常文件操作的标准流程|
|`pushd`/`popd`/`dirs`|目录栈操作，多目录快速切换|cd 仅能在两个目录间切换，目录栈可管理多个目录，实现多目录快速跳转，适合需要频繁在多个目录间切换的场景|
|`ln`|创建符号链接 / 硬链接|与 cd 配合使用，通过软链接简化长路径的输入，实现常用目录的快速跳转|
|`mkdir -p`|递归创建多级目录|与 cd 配合使用，`mkdir -p`创建深层目录后，用`cd $_`快速切换到该目录|

## 七、课后练习任务

1. 用 5 种不同的方式切换到你的家目录：直接`cd`、`cd ~`、`cd $HOME`、通过绝对路径、通过相对路径（假设当前在`/tmp`目录）。
2. 练习两个目录间的快速切换：先切换到`/var/log`，再切换到`/tmp`，然后用`cd -`在两个目录间来回切换 3 次，记录每次的输出结果。
3. 编写一个安全切换目录的脚本：目标目录为`/data/test`，先判断目录是否存在，若不存在则创建该目录，然后切换到该目录，输出当前目录路径，最后切回初始目录。
4. 练习目录栈操作：用`pushd`依次切换到`/tmp`、`/var/log`、`/data`，用`dirs -v`查看目录栈，然后用`popd`依次弹出目录，观察目录栈的变化。
5. 创建一个带空格的目录`My Project`，练习用双引号包裹和反斜杠转义两种方式切换到该目录，验证两种方式的效果。
