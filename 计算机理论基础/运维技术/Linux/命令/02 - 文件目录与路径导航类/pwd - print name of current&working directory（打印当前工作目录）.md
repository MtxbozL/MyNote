## 一、核心定位

`pwd` 是 Linux 系统中**最基础、最高频的工作路径确认命令**，属于「02 - 文件目录与路径导航类」，无任何权限要求，普通用户可直接执行。核心作用是**输出当前 Shell 会话所在工作目录的绝对路径**，是终端目录导航、脚本路径校验、操作位置确认、相对路径转绝对路径的核心基础命令，也是 Linux 入门必学的第一个路径导航命令。

> 补充说明：`pwd` 是 Shell 内置命令（bash/zsh 均原生支持），同时系统也提供`/usr/bin/pwd`外部二进制程序，两者功能基本一致，内置命令优先级更高。

## 二、语法格式

```bash
# 核心语法（99%的使用场景直接无参数执行）
pwd [可选参数]
```

> 关键说明：
> 
> 1. 无参数时，默认使用`-L`（逻辑路径）模式，输出当前工作目录的逻辑绝对路径；
> 2. 输出的永远是**绝对路径**，不会输出相对路径；
> 3. 核心差异点在于对符号链接的处理，`-L`跟随符号链接显示逻辑路径，`-P`显示物理真实路径。

## 三、高频核心参数

表格

|参数|核心作用|高频使用场景|
|:--|:--|:--|
|无参数（直接执行`pwd`）|等价于`pwd -L`，输出当前工作目录的逻辑绝对路径|日常快速确认当前所在位置，最核心、最常用的用法|
|`-L, --logical`|逻辑路径模式，跟随符号链接，输出符号链接对应的逻辑路径（默认行为）|日常路径查看、保持与 cd 命令的路径一致性，默认无需显式指定|
|`-P, --physical`|物理路径模式，不跟随符号链接，输出目录所在的真实物理绝对路径|排查符号链接问题、确认目录真实物理位置、脚本中获取真实路径|
|`--help`|输出命令帮助信息|快速查看参数用法|
|`--version`|输出命令版本信息|确认命令兼容性|

## 四、实操示例

### 1. 基础入门示例（新手必练，零门槛上手）

```bash
# 示例1：最基础用法，查看当前工作目录的绝对路径
pwd
# 执行效果示例：/home/ubuntu/linux_practice
```

```bash
# 示例2：逻辑路径vs物理路径（符号链接场景核心对比）
# 步骤1：创建一个真实目录和指向它的符号链接
mkdir -p /data/real_dir
ln -s /data/real_dir ~/link_dir

# 步骤2：进入符号链接目录，查看逻辑路径（默认）
cd ~/link_dir
pwd
# 执行效果：/home/ubuntu/link_dir（逻辑路径，与cd的路径一致）

# 步骤3：查看真实物理路径
pwd -P
# 执行效果：/data/real_dir（目录的真实物理位置）
```

```bash
# 示例4：脚本中获取当前工作目录，保存到变量
CURRENT_DIR=$(pwd)
echo "当前工作目录是：$CURRENT_DIR"
# 执行效果：打印当前工作目录的绝对路径
```

### 2. 高频日常场景示例

#### 场景 1：脚本中安全校验当前目录，避免在错误路径执行操作

```bash
#!/bin/bash
# 脚本前置路径校验，避免误操作
EXPECTED_DIR="/data/backup"
CURRENT_DIR=$(pwd)

# 判断当前目录是否符合预期
if [ "$CURRENT_DIR" != "$EXPECTED_DIR" ]; then
    echo -e "\e[1;31m [错误] 请先切换到 $EXPECTED_DIR 目录再执行此脚本！\e[0m"
    echo "当前目录：$CURRENT_DIR"
    exit 1
fi

# 校验通过，执行核心逻辑
echo -e "\e[1;32m 目录校验通过，开始执行备份任务...\e[0m"
# 后续业务逻辑
```

#### 场景 2：切换目录后，记录原路径并自动返回

```bash
#!/bin/bash
# 记录原路径，切换目录执行操作后自动返回
# 保存执行脚本时的初始目录
INIT_DIR=$(pwd)
echo "初始目录：$INIT_DIR"

# 切换到目标目录执行操作
cd /var/log || exit 1
echo "已切换到：$(pwd)"
echo "当前目录日志文件数量：$(ls -1 *.log | wc -l)"

# 操作完成后切回初始目录
cd "$INIT_DIR" || exit 1
echo "已回到初始目录：$(pwd)"
```

#### 场景 3：生成带绝对路径的文件列表，避免相对路径歧义

```bash
# 生成当前目录下所有.txt文件的绝对路径列表
for file in *.txt; do
    echo "$(pwd)/$file" >> file_abs_path.list
done

# 查看生成的绝对路径列表
cat file_abs_path.list
```

### 3. 进阶奇妙用法

#### 奇妙用法 1：一行命令将当前路径复制到系统剪贴板

```bash
# 适用于带图形界面的Linux桌面系统（需安装xclip）
pwd | xclip -selection clipboard
# 执行效果：当前工作目录的绝对路径已复制到剪贴板，可直接粘贴使用

# 适用于MacOS系统
pwd | pbcopy
```

#### 奇妙用法 2：处理被删除的当前目录，自动恢复到合法路径

```bash
#!/bin/bash
# 处理当前目录被删除的异常场景
CURRENT_DIR=$(pwd)

# 检查当前目录是否存在
if [ ! -d "$CURRENT_DIR" ]; then
    echo -e "\e[1;33m [警告] 当前目录已被删除，自动切换到用户家目录\e[0m"
    cd ~ || exit 1
    echo "已切换到：$(pwd)"
fi
```

#### 奇妙用法 3：在 Shell 提示符中永久显示当前路径

```bash
# 编辑~/.bashrc，修改PS1环境变量，让终端提示符永久显示当前工作目录
echo 'export PS1="[\u@\h \w]\\$ "' >> ~/.bashrc

# 使配置立即生效
source ~/.bashrc
# 执行效果：终端提示符变为 [用户名@主机名 当前绝对路径]$ ，无需反复执行pwd
```

#### 奇妙用法 4：结合 find，查找文件并输出完整绝对路径

```bash
# 查找当前目录下所有.log文件，输出完整绝对路径
find "$(pwd)" -type f -name "*.log"
# 执行效果：输出每个.log文件的完整绝对路径，而非相对路径
```

#### 奇妙用法 5：获取当前目录的上级目录绝对路径

```bash
# 获取当前目录的上级目录绝对路径
PARENT_DIR=$(dirname "$(pwd)")
echo "当前目录：$(pwd)"
echo "上级目录：$PARENT_DIR"

# 获取上两级目录绝对路径
GRAND_PARENT_DIR=$(dirname "$(dirname "$(pwd)")")
echo "上两级目录：$GRAND_PARENT_DIR"
```

## 五、避坑提示 & 注意事项

1. **新手最高频踩坑：符号链接的逻辑路径与物理路径混淆**
    
    - `cd` 进入符号链接目录后，默认`pwd`输出的是**逻辑路径**（与 cd 的路径一致），而非目录的真实物理位置；
    - 若需确认目录的真实物理路径，**必须加`-P`参数**，即`pwd -P`；
    - 典型错误：在符号链接目录中用相对路径执行`rm -rf ../`，可能会误删真实物理目录的上级文件。
    
2. **子 Shell 中的 pwd 不影响父 Shell**
    
    - 在管道、`$( )`、`( )`子 Shell 中执行`cd`和`pwd`，仅在子 Shell 中生效，不会改变父 Shell 的当前工作目录；
    - 错误示例：`cd /tmp | pwd` 执行后，父 Shell 的当前目录不会变化；
    - 正确示例：需在同一个 Shell 中执行`cd /tmp && pwd`。
    
3. **当前目录被删除的异常场景**
    
    - 若当前所在的目录被其他进程 / 用户删除，执行`pwd`会输出`pwd: error retrieving current directory: getcwd: cannot access parent directories: No such file or directory`；
    - 解决方法：执行`cd ~`或`cd /`切换到合法目录，再重新执行 pwd。
    
4. **权限不足的坑**
    
    - 若当前目录的父目录无执行权限（`x`权限），即使当前目录有权限，执行`pwd`也可能报错；
    - 解决方法：给父目录添加执行权限`sudo chmod +x 父目录路径`。
    
5. **内置命令与外部命令的差异**
    
    - Shell 内置的`pwd`和外部二进制`/usr/bin/pwd`的默认行为可能存在差异：内置`pwd`默认`-L`，外部`pwd`默认`-P`；
    - 脚本中为保证兼容性，建议显式指定`-L`或`-P`参数，避免环境差异导致的异常。
    
6. **带空格 / 特殊字符的路径处理**
    
    - 若当前路径包含空格、特殊字符，`$(pwd)`必须用双引号包裹，否则会被 Shell 拆分为多个参数，导致脚本执行异常；
    - 错误示例：`cd $(pwd)/sub_dir`（路径带空格时会报错）；
    - 正确示例：`cd "$(pwd)/sub_dir"`。
    

## 六、同类命令对比 & 拓展

表格

|命令|核心作用|与 pwd 的区别 & 适用场景|
|:--|:--|:--|
|`cd`|切换当前工作目录|与 pwd 形成黄金搭档，cd 用于切换目录，pwd 用于确认当前目录，两者是路径导航的基础组合|
|`dirs`|显示目录栈中的目录列表|配合 pushd/popd 使用，可查看多个历史目录；pwd 仅查看当前工作目录，多目录切换用 dirs，当前路径确认用 pwd|
|`readlink -f`|输出文件 / 目录的真实绝对路径|可获取任意文件 / 目录的真实物理路径，无需 cd 到该目录；pwd 仅能查看当前工作目录，获取任意目录的真实路径用 readlink -f|
|`realpath`|输出文件 / 目录的规范化绝对路径|等价于`readlink -f`，专门用于路径规范化，支持相对路径转绝对路径；pwd 仅针对当前目录，批量路径转换用 realpath|
|`dirname`|提取路径中的目录部分|用于从完整路径中剥离文件名，获取上级目录路径；pwd 输出当前目录完整路径，路径拆分用 dirname|
|`basename`|提取路径中的文件名 / 目录名|用于从完整路径中提取最后一级目录名 / 文件名；pwd 输出完整路径，路径末尾名称提取用 basename|
|`echo $PWD`|输出环境变量 PWD 的值|与`pwd -L`完全等价，是 Shell 内置的环境变量，实时更新当前工作目录，脚本中可直接使用，无需执行 pwd 命令|

## 七、课后练习任务

1. 基础查看练习：分别在你的家目录、`/tmp`目录、`/var/log`目录执行`pwd`，记录输出的绝对路径。
2. 符号链接练习：创建一个指向`/tmp`的符号链接`link_to_tmp`，cd 进入该链接目录，分别执行`pwd`和`pwd -P`，对比两者的输出差异，理解逻辑路径与物理路径的区别。
3. 脚本路径校验练习：编写一个脚本，要求仅能在`/tmp`目录下执行，若在其他目录执行则报错退出，校验通过后打印当前目录路径。
4. 路径记录练习：编写一个脚本，记录初始目录，切换到`/usr/local`目录查看文件数量，然后自动切回初始目录，打印切换前后的路径。
5. 绝对路径生成练习：用`pwd`结合循环，生成当前目录下所有文件的绝对路径列表，保存到`abs_path.txt`文件中。
6. 上级目录获取练习：用`pwd`结合`dirname`，分别获取当前目录的上级、上两级、上三级目录的绝对路径，打印输出。
7. 提示符配置练习：修改`~/.bashrc`文件，让终端提示符永久显示当前工作目录的绝对路径，验证配置生效。
