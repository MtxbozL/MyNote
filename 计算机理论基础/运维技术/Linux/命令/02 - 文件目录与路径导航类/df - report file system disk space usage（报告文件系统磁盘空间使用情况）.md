## 一、核心定位

`df` 是 Linux 系统中**最核心、最高频的磁盘空间监控命令**，属于「02 - 文件目录与路径导航类」，无任何权限要求，普通用户可直接执行。核心作用是**查看系统已挂载文件系统的总容量、已用空间、可用空间、挂载点、文件系统类型、inode 使用情况等核心磁盘指标**，是系统日常巡检、磁盘空间预警、挂载点管理、文件系统选型的必备基础命令，也是 Linux 入门必学的磁盘管理工具。

> 补充说明：`df` 查看的是**文件系统层面的整体空间使用情况**，与`du`（查看目录 / 文件的实际占用）互补，两者结合可全面排查磁盘空间问题。

## 二、语法格式

```bash
# 核心语法1：查看所有已挂载文件系统的磁盘空间
df [可选参数]

# 核心语法2：查看指定文件/目录所在文件系统的磁盘空间
df [可选参数] 文件/目录路径
```

> 关键说明：
> 
> 1. 无参数时，默认查看所有已挂载的非伪文件系统（如 tmpfs、sysfs 等默认不显示）；
> 2. 指定文件 / 目录路径时，仅显示该路径所在文件系统的磁盘空间；
> 3. 输出单位默认是 1K 块，需配合`-h`参数转换为人类可读单位。

## 三、高频核心参数

表格

|参数|核心作用|高频使用场景|
|:--|:--|:--|
|`-h, --human-readable`|以人类可读的单位（K/M/G/T/P）显示磁盘空间|日常快速查看磁盘空间，最核心、最常用的参数|
|`-H, --si`|以 1000 为进制的人类可读单位（而非 1024）|硬盘厂商容量对比、商业场景容量统计|
|`-T, --print-type`|显示每个文件系统的类型（如 ext4、xfs、tmpfs）|查看文件系统类型、选型对比、挂载点管理|
|`-i, --inodes`|显示 inode 的使用情况，而非磁盘块|排查 inode 满导致的 “磁盘空间不足” 问题|
|`-x, --exclude-type=TYPE`|排除指定类型的文件系统，不显示|过滤 tmpfs、sysfs 等伪文件系统，专注查看真实磁盘|
|`-a, --all`|显示所有文件系统，包括伪文件系统|排查系统挂载问题、查看完整挂载列表|
|`-l, --local`|仅显示本地文件系统，排除网络文件系统（如 NFS）|专注查看本地磁盘，避免网络文件系统卡顿|
|`-t, --type=TYPE`|仅显示指定类型的文件系统|筛选查看特定类型的文件系统，如仅看 xfs 或 ext4|
|`--output=FIELD1,FIELD2`|自定义输出字段，按需显示|脚本提取特定字段、生成定制化报告|
|`-P, --portability`|使用 POSIX 标准输出格式，避免换行问题|脚本解析输出，保证格式一致性|

### 2. 高频日常场景示例

#### 场景 1：系统日常巡检，查看所有本地磁盘空间

```bash
# 查看所有本地文件系统，排除伪文件系统和网络文件系统，显示人类可读单位和文件系统类型
df -hlT
# 执行效果：仅显示真实的本地磁盘，专注查看核心存储的使用情况
```

#### 场景 2：排查 “磁盘空间不足” 但实际还有空间的问题（inode 满）

```bash
# 1. 先查看磁盘块空间，确认还有空间
df -h
# 2. 再查看inode使用情况，确认是否inode满了
df -hi
# 执行效果：若inode使用率为100%，说明是小文件过多导致inode耗尽，需清理小文件
```

#### 场景 3：过滤特定类型的文件系统

```bash
# 仅查看xfs类型的文件系统
df -h -t xfs

# 仅查看ext4类型的文件系统
df -h -t ext4

# 排除tmpfs伪文件系统
df -h -x tmpfs
```

#### 场景 4：脚本中提取磁盘使用率，实现自动告警

```bash
#!/bin/bash
# 磁盘空间自动告警脚本
THRESHOLD=80  # 告警阈值：80%

# 遍历所有已挂载的本地文件系统，提取使用率
df -hl -x tmpfs -x devtmpfs | grep -vE "^Filesystem" | while read LINE; do
    USAGE=$(echo $LINE | awk '{print $5}' | sed 's/%//')
    MOUNT_POINT=$(echo $LINE | awk '{print $6}')
    FILESYSTEM=$(echo $LINE | awk '{print $1}')
    
    # 判断是否超过阈值
    if [ $USAGE -ge $THRESHOLD ]; then
        echo -e "\e[1;31m [紧急告警] 文件系统 $FILESYSTEM ($MOUNT_POINT) 使用率已达 ${USAGE}%，超过阈值 ${THRESHOLD}%！\e[0m"
    else
        echo -e "\e[1;32m [正常] 文件系统 $FILESYSTEM ($MOUNT_POINT) 使用率为 ${USAGE}%\e[0m"
    fi
done
```

```bash
0 * * * * echo "=== $(date +%F %T) ===" >> /var/log/disk_usage.log && df -h >> /var/log/disk_usage.log

# 3. 查看记录的磁盘空间日志

```

#### 奇妙用法 3：结合 watch 命令，实时监控磁盘空间变化

```bash
# 每2秒刷新一次磁盘空间，实时监控
watch -n 2 df -h
# 执行效果：动态刷新磁盘空间，适合观察大文件复制时的空间变化
```

#### 奇妙用法 4：排查已删除文件但空间未释放的问题

```bash
# 场景：df显示空间已用很多，但du统计的文件总大小远小于df显示的已用空间
# 原因：有进程仍在占用已删除的文件，导致空间未释放
# 解决：用lsof查找占用已删除文件的进程
sudo lsof | grep deleted
# 执行效果：列出所有占用已删除文件的进程，重启对应进程即可释放空间
```

#### 奇妙用法 5：POSIX 标准输出，保证脚本解析兼容性

```bash
# 使用-P参数，输出POSIX标准格式，避免文件名过长导致的换行问题
df -P -h
# 执行效果：保证每行一个文件系统，脚本解析更可靠
```

## 五、避坑提示 & 注意事项

1. **最大认知误区：df 显示的已用空间 ≠ du 统计的文件总大小**
    
    - `df`：文件系统层面的空间统计，包含已删除但仍被进程占用的文件、文件系统元数据、保留空间；
    - `du`：文件层面的空间统计，仅统计实际存在的文件大小；
    - 典型差异场景：删除了大文件但进程未重启，`df`显示空间未释放，`du`统计不到该文件；
    - 解决方法：用`lsof | grep deleted`查找占用已删除文件的进程，重启进程释放空间。
    
2. **新手高频踩坑：默认单位是 1K 块，不是字节**
    
    - 无参数时，`df` 默认以 1K 块为单位显示空间，新手极易误读；
    - 强制要求：日常查看**必须加`-h`参数**，转换为人类可读的 K/M/G/T 单位，避免误读。
    
3. **文件系统保留空间的坑**
    
    - ext4/xfs 等文件系统默认会保留 5% 的空间给 root 用户使用，防止磁盘满了导致系统无法启动；
    - `df` 显示的 “可用空间” 已扣除保留空间，普通用户无法使用这部分空间；
    - 调整保留空间：`sudo tune2fs -m 1 /dev/sda1`（将保留空间调整为 1%，数据盘可调整为 0%）。
    
4. **inode 满了但磁盘还有空间的坑**
    
    - 磁盘空间充足，但提示 “磁盘空间不足”，大概率是 inode 耗尽了；
    - inode 是文件系统中存储文件元数据的结构，每个文件占用一个 inode，小文件过多会导致 inode 耗尽；
    - 解决方法：用`df -i`确认 inode 使用率，清理小文件或重新格式化文件系统（调整 inode 比例）。
    
5. **伪文件系统的干扰**
    
    - `df` 默认会显示 tmpfs、sysfs、devtmpfs 等伪文件系统，这些是内存中的文件系统，不占用真实磁盘空间；
    - 日常查看真实磁盘空间，必须加`-x tmpfs -x devtmpfs`或`-l`参数，过滤伪文件系统，避免误判。
    
6. **网络文件系统的卡顿**
    
    - 若系统挂载了 NFS 等网络文件系统，网络不通时，直接执行`df`会导致终端卡顿；
    - 排查网络文件系统问题时，优先用`-l`参数仅查看本地文件系统，避免卡顿。
    
7. **自定义输出字段的兼容性**
    
    - `--output`参数是 GNU df 的特性，主流 Linux 发行版支持，但老旧 Unix 系统可能不支持；
    - 跨平台脚本需避免使用`--output`，改用`awk`提取字段。
    

## 六、同类命令对比 & 拓展

| 命令               | 核心作用             | 与 df 的区别 & 适用场景                                                  |
| :--------------- | :--------------- | :--------------------------------------------------------------- |
| `du`             | 查看目录 / 文件的实际磁盘占用 | df 查看文件系统整体空间，du 查看具体文件 / 目录的占用；df 用于整体空间监控，du 用于定位大文件 / 目录，两者互补 |
| `lsblk`          | 查看块设备的物理结构与挂载关系  | 专注于物理磁盘、分区的结构展示，不显示空间使用率；df 专注于文件系统的空间使用，两者互补                    |
| `fdisk`/`parted` | 磁盘分区工具           | 用于磁盘分区、格式化，不显示空间使用率；df 用于分区后的空间监控，两者配合使用                         |
| `mount`          | 查看 / 管理文件系统挂载    | 查看完整的挂载信息、挂载参数；df 专注于空间使用，两者互补                                   |
| `lsof`           | 查看进程打开的文件        | 用于排查已删除文件但空间未释放的问题；df 用于发现空间异常，lsof 用于定位原因，两者配合使用                |
| `tune2fs`        | 调整 ext 文件系统参数    | 用于调整文件系统保留空间、inode 比例等；df 用于监控空间使用，tune2fs 用于优化文件系统配置            |
| `ncdu`           | 交互式磁盘空间分析工具      | 是 du 的进阶替代方案，支持交互式浏览、排序、删除大文件，比 df 更直观，适合深度排查大文件                 |

## 七、课后练习任务

1. 基础查看练习：分别用`df`、`df -h`、`df -hT`三种方式查看磁盘空间，对比输出结果的差异，记录每个字段的含义。
2. 指定目录查看练习：查看你的家目录、`/tmp`目录、`/var/log`目录所在文件系统的磁盘空间，确认它们是否在同一个文件系统。
3. inode 查看练习：用`df -hi`查看 inode 使用情况，记录 inode 的总量、已用、可用、使用率。
4. 过滤文件系统练习：用`df -hlT -x tmpfs -x devtmpfs`查看所有本地真实磁盘空间，过滤伪文件系统。
5. 脚本告警练习：完善磁盘空间告警脚本，当使用率超过 70% 时，不仅打印告警，还将告警信息写入`/var/log/disk_warn.log`日志文件。
6. 实时监控练习：用`watch -n 1 df -h`实时监控磁盘空间变化，同时在另一个终端复制一个大文件，观察磁盘空间的变化。
