## 一、核心定位

`chmod` 是 Linux 系统中**最核心、最基础的文件 / 目录权限管理命令**，属于「文件权限管理类」，普通用户仅能修改自己有权限的文件 / 目录，root 用户可修改所有文件 / 目录。核心作用是**修改文件 / 目录的访问权限，支持两种模式：符号模式（u/g/o/a +/-/= r/w/x）和数字模式（4/2/1 三位数字），支持递归修改、显示详细过程、参考其他文件权限、设置特殊权限（SUID/SGID/ 粘滞位）**，是文件权限管理、安全配置、目录访问控制的必备基础命令，也是 Linux 入门必学的权限管理命令。

> 补充说明：`chmod` 名称来源于 “change mode”（修改权限模式）；Linux 文件权限分为三级：用户（u）、组（g）、其他（o），每级有读（r）、写（w）、执行（x）三种权限；与`chown`（改所有者）、`chgrp`（改组）并称为 Linux 权限管理三剑客。

## 二、语法格式

```bash
# 核心语法1：符号模式（灵活，适合单次修改）
chmod [可选参数] [ugoa][+-=][rwxXst] 文件1 文件2 ...

# 核心语法2：数字模式（简洁，适合批量设置）
chmod [可选参数] 三位数字 文件1 文件2 ...
```

> 关键语法说明：
> 
> 1. **符号模式**：
> 
> - `u`（user）：文件所有者；`g`（group）：文件所属组；`o`（others）：其他用户；`a`（all）：所有用户（u+g+o，默认）；
> - `+`：添加权限；`-`：移除权限；`=`：设置权限（覆盖原有权限）；
> - `r`（read）：读；`w`（write）：写；`x`（execute）：执行；`X`：仅当文件是目录或已有执行权限时才设置执行；`s`（SUID/SGID）：特殊权限；`t`（粘滞位）：特殊权限；
> 
> 2. **数字模式**：
> 
> - 三位数字分别对应用户（u）、组（g）、其他（o）；
> - 每级权限值：`r=4`、`w=2`、`x=1`，相加得到该级权限值；
> - 常用组合：`755`（rwxr-xr-x，目录 / 可执行文件）、`644`（rw-r--r--，普通文件）、`700`（rwx------，私密文件）；
> 
> 3. **可选参数**：如`-R`（递归）、`-v`（详细）等。

## 三、高频核心参数（按功能分类）

### 1. 符号模式权限类

|符号|核心作用|高频使用场景|
|:--|:--|:--|
|`u`|文件**所有者**（user）|修改文件所有者的权限|
|`g`|文件**所属组**（group）|修改文件所属组的权限|
|`o`|**其他用户**（others）|修改其他用户的权限|
|`a`|**所有用户**（all，u+g+o，默认）|修改所有用户的权限|
|`+`|**添加**权限|给指定用户添加权限|
|`-`|**移除**权限|给指定用户移除权限|
|`=`|**设置**权限（覆盖原有）|给指定用户设置精确权限|
|`r`|**读**权限（read，4）|允许读文件 / 列目录|
|`w`|**写**权限（write，2）|允许写文件 / 修改目录|
|`x`|**执行**权限（execute，1）|允许执行文件 / 进入目录|
|`X`|条件执行权限（仅目录或已有执行权限时设置）|批量给目录添加执行权限，不影响普通文件|
|`s`|SUID/SGID 特殊权限（4000/2000）|可执行文件提权（需谨慎）|
|`t`|粘滞位特殊权限（1000）|目录仅允许所有者删除自己的文件（如 /tmp）|

### 2. 数字模式权限类

|数字|权限组合|核心作用|高频使用场景|
|:--|:--|:--|:--|
|`7`|`rwx`（4+2+1）|读、写、执行|目录、可执行文件的所有者权限|
|`6`|`rw-`（4+2）|读、写|普通文件的所有者权限|
|`5`|`r-x`（4+1）|读、执行|目录、可执行文件的组 / 其他权限|
|`4`|`r--`（4）|只读|普通文件的组 / 其他权限|
|`3`|`-wx`（2+1）|写、执行|特殊场景|
|`2`|`-w-`（2）|只写|特殊场景|
|`1`|`--x`（1）|只执行|特殊场景|
|`0`|`---`（0）|无权限|禁止访问|
|`755`|`rwxr-xr-x`|所有者读写执行，组 / 其他读执行|公共目录、可执行文件|
|`644`|`rw-r--r--`|所有者读写，组 / 其他只读|普通文本文件、配置文件|
|`700`|`rwx------`|所有者读写执行，组 / 其他无权限|私密文件、个人目录|
|`600`|`rw-------`|所有者读写，组 / 其他无权限|私密配置文件、密钥文件|

### 3. 通用操作类

|参数|核心作用|高频使用场景|
|:--|:--|:--|
|`-R, --recursive`|**递归**修改目录及其所有子目录 / 文件的权限|批量修改目录树权限、目录结构权限统一|
|`-v, --verbose`|显示**详细**修改过程，输出每个被修改的文件 / 目录|监控修改进度、调试权限问题|
|`-c, --changes`|仅显示**被修改**的文件 / 目录，不显示未修改的|仅关注权限变化、减少冗余输出|
|`--reference=RFILE`|参考**指定文件**的权限，将目标文件 / 目录的权限设置为与参考文件一致|批量同步权限、保持权限一致性|
|`--help`|输出 chmod 帮助信息|快速查看参数与用法|
|`--version`|输出 chmod 版本信息|确认版本兼容性|

## 四、实操示例

### 1. 基础入门示例（新手必练，零门槛上手）

```bash
# 示例1：符号模式，给文件所有者添加执行权限
chmod u+x test.sh
# 执行效果：test.sh的所有者权限添加x
```

```bash
# 示例2：符号模式，给文件所属组移除写权限
chmod g-w test.txt
# 执行效果：test.txt的所属组权限移除w
```

```bash
# 示例3：符号模式，给所有用户设置只读权限
chmod a=r test.txt
# 执行效果：test.txt的所有用户权限设置为r--r--r--
```

```bash
# 示例4：数字模式，设置为755（rwxr-xr-x）
chmod 755 test.sh
# 执行效果：test.sh的权限设置为rwxr-xr-x
```

```bash
# 示例5：数字模式，设置为644（rw-r--r--）
chmod 644 test.txt
# 执行效果：test.txt的权限设置为rw-r--r--
```

```bash
# 示例6：数字模式，设置为700（rwx------）
chmod 700 private_dir
# 执行效果：private_dir的权限设置为rwx------
```

---

### 2. 高频日常场景示例

#### 场景 1：递归修改目录权限

```bash
# 递归修改目录及其所有子目录/文件的权限
chmod -R 755 /data/www
# 执行效果：/data/www及其所有子目录/文件的权限设置为755
```

```bash
# 更安全的递归修改：目录755，文件644
# 先递归给所有目录添加执行权限
find /data/www -type d -exec chmod 755 {} \;
# 再递归给所有普通文件设置644
find /data/www -type f -exec chmod 644 {} \;
```

#### 场景 2：显示详细修改过程

```bash
# 显示详细修改过程
chmod -v 644 test.txt
# 执行效果示例：
# mode of 'test.txt' changed from 0755 (rwxr-xr-x) to 0644 (rw-r--r--)
```

```bash
# 仅显示被修改的文件
chmod -c 644 test.txt
# 执行效果：仅当权限变化时显示，无变化则无输出
```

#### 场景 3：参考其他文件权限

```bash
# 参考reference.txt的权限，将test.txt的权限设置为一致
chmod --reference=reference.txt test.txt
# 执行效果：test.txt的权限与reference.txt完全一致
```

#### 场景 4：设置粘滞位（如 /tmp 目录）

```bash
# 给/tmp目录设置粘滞位（1777）
chmod 1777 /tmp
# 执行效果：/tmp目录权限为rwxrwxrwt，仅允许所有者删除自己的文件
```

---

### 3. 进阶奇妙用法

#### 奇妙用法 1：批量给目录添加执行权限，不影响普通文件

```bash
# 用X权限，仅给目录添加执行权限，普通文件不影响
chmod -R a+X /data/www
# 执行效果：所有目录添加x权限，普通文件保持不变
```

#### 奇妙用法 2：结合 find，批量修改特定类型文件的权限

```bash
# 批量给所有.sh脚本添加执行权限
find /data/script -type f -name "*.sh" -exec chmod +x {} \;
```

#### 奇妙用法 3：结合 ls -l，查看权限修改前后的对比

```bash
# 查看修改前的权限
ls -l test.txt
# 修改权限
chmod 644 test.txt
# 查看修改后的权限
ls -l test.txt
```

#### 奇妙用法 4：设置 SUID/SGID（需谨慎，仅用于可执行文件）

```bash
# 给可执行文件设置SUID（4755）
chmod 4755 /usr/bin/sudo
# 执行效果：文件权限为rwsr-xr-x，执行时以所有者身份运行
```

## 五、避坑提示 & 注意事项

1. **新手最高频致命坑：数字模式的三位顺序不能错**
    
    - 数字模式的三位数字**严格对应用户（u）、组（g）、其他（o）**，顺序不能颠倒；
    - 强制要求：记住顺序`u-g-o`，不要写成`o-g-u`；
    - 典型错误：`chmod 557 test.txt`（以为是 755，实际是 r-xr-xrwx，其他用户有读写执行，高风险）；
    - 正确示例：`chmod 755 test.txt`（rwxr-xr-x）。
    
2. **`-R`递归修改的高危风险**
    
    - `-R` 会递归修改目录及其所有子目录 / 文件的权限，无任何提示，一旦执行无法恢复；
    - 强制要求：
        
        - 使用`-R`前，**必须先不加`-R`执行一遍，确认目标目录正确**；
        - 重要目录先备份；
        - 优先用`find`分别修改目录和文件，避免普通文件被错误设置执行权限；
        
    - 典型错误：`chmod -R 777 /`（会清空整个系统的权限，导致系统崩溃，绝对禁止尝试）。
    
3. **目录和文件的权限区别**
    
    - 目录的`x`权限是**进入目录**的权限，没有`x`权限无法进入目录，即使有`r`权限也无法列目录；
    - 普通文件的`x`权限是**执行**的权限，普通文件不需要`x`权限；
    - 解决方法：批量修改时，优先用`find -type d`修改目录，`find -type f`修改文件，避免普通文件被错误设置`x`权限。
    
4. **特殊权限的风险**
    
    - SUID/SGID 特殊权限有极高的安全风险，可导致提权攻击，仅用于必要的系统可执行文件（如`sudo`、`passwd`）；
    - 粘滞位仅用于公共目录（如`/tmp`），避免其他用户删除自己的文件；
    - 强制要求：不要给普通文件设置 SUID/SGID，仅给必要的系统可执行文件设置。
    
5. **权限不足的坑**
    
    - 修改系统文件（如`/root`、`/etc/shadow`、`/usr/bin`）的权限时，普通用户会报错`chmod: cannot access 'xxx': Permission denied`；
    - 解决方法：修改系统文件需加`sudo`，但必须极其谨慎，确认修改正确，避免系统无法启动。
    
6. **符号链接的处理**
    
    - `chmod` 默认**不跟随符号链接**，仅修改符号链接本身的权限（符号链接的权限通常无意义，默认 777）；
    - 若需修改符号链接指向的目标文件的权限，需直接修改目标文件，或用`chmod -h`（部分系统支持）；
    - 注意：不要尝试修改符号链接的权限，无实际意义。
    
7. **`--reference`的参考文件**
    
    - `--reference` 会将目标文件的权限设置为与参考文件**完全一致**，包括特殊权限；
    - 解决方法：使用`--reference`前，先确认参考文件的权限正确。
    

## 六、同类命令对比 & 拓展

表格

|命令|核心作用|与 chmod 的区别 & 适用场景|
|:--|:--|:--|
|`chown`|修改文件 / 目录的所有者|chmod 修改权限，chown 修改所有者；两者常配合使用，权限管理用 chmod，所有者管理用 chown|
|`chgrp`|修改文件 / 目录的所属组|chmod 修改权限，chgrp 修改所属组；两者常配合使用，权限管理用 chmod，组管理用 chgrp|
|`umask`|设置 / 查看默认权限掩码|umask 控制新创建文件 / 目录的默认权限；chmod 修改已有文件 / 目录的权限，默认权限控制用 umask，已有权限修改用 chmod|
|`ls -l`|查看文件 / 目录的权限|优势：简单快速，查看权限；劣势：无法修改权限。查看权限用 ls -l，修改权限用 chmod，两者常配合使用|
|`stat`|查看文件 / 目录的详细元数据|优势：查看详细权限（包括八进制）、时间、所有者；劣势：无法修改权限。查看详细权限用 stat，修改权限用 chmod|

## 七、课后练习任务

1. 基础符号模式练习：创建一个`test.sh`文件，用`chmod u+x`给所有者添加执行权限，用`chmod g-w`给所属组移除写权限，用`chmod a=r`给所有用户设置只读权限，用`ls -l`查看每次修改后的权限。
2. 基础数字模式练习：创建一个`test.txt`文件，用`chmod 755`设置为 755，用`chmod 644`设置为 644，用`chmod 700`设置为 700，用`ls -l`查看每次修改后的权限。
3. 递归修改练习：创建一个`test_dir`目录，并在其中创建子目录和文件，用`chmod -R 755 test_dir`递归修改权限，用`ls -lR`查看修改后的权限；再用`find test_dir -type d -exec chmod 755 {} \;`和`find test_dir -type f -exec chmod 644 {} \;`分别修改目录和文件，对比效果。
4. 显示详细过程练习：用`chmod -v 644 test.txt`显示详细修改过程，用`chmod -c 644 test.txt`仅显示被修改的文件，对比两者的差异。
5. 参考文件练习：创建一个`reference.txt`文件，设置为 600，用`chmod --reference=reference.txt test.txt`将`test.txt`的权限设置为一致，用`ls -l`验证。
6. 批量给脚本添加执行权限练习：创建几个`.sh`文件，用`find . -name "*.sh" -exec chmod +x {} \;`批量添加执行权限，验证效果。
7. 批量给目录添加执行权限练习：用`chmod -R a+X test_dir`批量给目录添加执行权限，不影响普通文件，验证效果。
8. 查看权限修改前后对比练习：用`ls -l test.txt`查看修改前的权限，修改权限后再用`ls -l test.txt`查看，对比差异。
9. 权限不足练习：尝试用普通用户修改`/etc/shadow`的权限，观察权限不足的报错；再用`sudo chmod 600 /etc/shadow`修改，验证效果（仅在测试环境操作）。
10. 帮助练习：用`chmod --help`查看帮助信息，浏览所有参数。
