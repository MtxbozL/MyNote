## 一、核心定位

`su` 是 Linux 系统中**最核心、最基础的用户身份完全切换命令**，属于「用户身份切换类」，普通用户切换到其他用户需输入目标用户密码，root 用户切换到任何用户无需密码。核心作用是**完全切换到目标用户的身份，支持登录 / 非登录 shell 切换、执行单个命令、保留环境变量、指定 shell**，是用户身份切换、长时间以其他用户身份操作、系统管理的必备基础命令，也是 Linux 入门必学的用户身份管理命令。

> 补充说明：`su` 名称来源于 “switch user”（切换用户）；与`sudo`（执行单个命令，无需完全切换身份）并称为 Linux 用户身份切换两大工具；`su -`（登录 shell 切换）是最常用的形式，会重置环境变量并切换到目标用户的家目录。

## 二、语法格式

bash

运行

```
# 核心语法1：切换到目标用户（无用户名默认切换到root）
su [可选参数] [用户名]

# 核心语法2：切换到目标用户并执行单个命令
su [可选参数] -c "命令" [用户名]
```

> 关键语法说明：
> 
> 1. **用户名**：可选，无用户名时默认切换到 root；
> 2. **登录 shell 与非登录 shell**：
> 
> - `su`（无`-`）：非登录 shell，保留当前目录和部分环境变量，不执行目标用户的登录脚本；
> - `su -`/`su -l`：登录 shell，重置环境变量，切换到目标用户的家目录，执行目标用户的登录脚本（如`.bash_profile`、`.profile`）；
> 
> 3. **可选参数**：如`-`（登录 shell）、`-c`（执行命令）等。

## 三、高频核心参数（按功能分类）

### 1. 基础切换类

表格

|参数|核心作用|高频使用场景|
|:--|:--|:--|
|无参数（直接执行`su`）|普通用户切换到 root，非登录 shell，保留当前目录和部分环境变量|临时切换到 root，短时间操作|
|无参数（直接执行`su 用户名`）|切换到指定用户，非登录 shell|临时切换到其他普通用户|
|`-, -l, --login`|切换到**登录 shell**，重置环境变量，切换到目标用户的家目录，执行登录脚本|完全切换身份，长时间以目标用户身份操作，**最常用的形式**|
|`-h, --help`|输出 su 帮助信息|快速查看参数与用法|
|`-v, --version`|输出 su 版本信息|确认版本兼容性|

### 2. 命令执行类

表格

|参数|核心作用|高频使用场景|
|:--|:--|:--|
|`-c, --command=COMMAND`|切换到目标用户**执行单个命令**，执行完返回原用户|临时以目标用户身份执行单个命令，无需完全切换|

### 3. 环境与 shell 类

表格

|参数|核心作用|高频使用场景|
|:--|:--|:--|
|`-m, -p, --preserve-environment`|**保留**当前用户的环境变量，不重置|保留当前环境变量，临时切换|
|`-s, --shell=SHELL`|指定使用的**shell**，而非目标用户的默认 shell|切换到默认 shell 为 /sbin/nologin 的系统用户|

## 四、实操示例

### 1. 基础入门示例（新手必练，零门槛上手）

bash

运行

```
# 示例1：普通用户切换到root（非登录shell）
su
# 执行效果：
# Password: （输入root密码，不回显）
# [root@hostname currentdir]# （提示符变为root，保留当前目录）
```

bash

运行

```
# 示例2：普通用户切换到root（登录shell，最常用）
su -
# 执行效果：
# Password: （输入root密码）
# [root@hostname ~]# （提示符变为root，切换到/root家目录，环境变量重置）
```

bash

运行

```
# 示例3：root切换到普通用户（无需密码）
su - user1
# 执行效果：
# [user1@hostname ~]$ （直接切换到user1的家目录，无需密码）
```

bash

运行

```
# 示例4：切换到目标用户并执行单个命令
su -c "ls /root" root
# 执行效果：
# Password: （输入root密码）
# （输出/root目录的内容，执行完返回原用户）
```

### 2. 高频日常场景示例

#### 场景 1：登录 shell 与非登录 shell 的对比

bash

运行

```
# 非登录shell：保留当前目录
pwd
# 输出：/home/user1
su
pwd
# 输出：/home/user1（保留当前目录）

# 登录shell：切换到目标用户的家目录
pwd
# 输出：/home/user1
su -
pwd
# 输出：/root（切换到/root家目录）
```

#### 场景 2：保留当前环境变量

bash

运行

```
# 保留当前环境变量切换
echo $MYVAR
# 输出：myvalue
su -m user1
echo $MYVAR
# 输出：myvalue（保留当前环境变量）
```

#### 场景 3：切换到默认 shell 为 /sbin/nologin 的系统用户

bash

运行

```
# 创建系统用户sysuser（默认shell为/sbin/nologin）
useradd -r sysuser
# 直接切换会失败
su - sysuser
# 输出：This account is currently not available.
# 指定shell切换
su -s /bin/bash sysuser
# 执行效果：成功切换到sysuser，使用/bin/bash
```

## 五、避坑提示 & 注意事项

1. **新手最高频踩坑：登录 shell 与非登录 shell 的区别**
    
    - `su`（无`-`）：非登录 shell，**保留当前目录和部分环境变量**，不执行目标用户的登录脚本（如`.bash_profile`）；
    - `su -`/`su -l`：登录 shell，**重置环境变量，切换到目标用户的家目录，执行登录脚本**；
    - 强制要求：完全切换身份、长时间操作**必须用`su -`**，避免环境变量和目录的混淆；
    - 典型错误：`su`切换到 root 后，发现家目录还是当前用户的家目录，环境变量不对；
    - 正确示例：`su -`切换到 root 的登录 shell。
    
2. **普通用户切换需输入目标用户密码**
    
    - 普通用户切换到其他用户（包括 root）**必须输入目标用户的密码**，root 用户切换到任何用户无需密码；
    - 强制要求：普通用户切换到 root 需输入 root 密码，不要尝试猜测；
    - 典型错误：普通用户执行`su`后忘记输入 root 密码，或输入错误；
    - 正确示例：普通用户执行`su -`，输入正确的 root 密码。
    
3. **不要滥用`su`切换到 root**
    
    - `su`完全切换到 root 身份，风险极高，容易误操作（如`rm -rf /`）；
    - 强制要求：推荐用`sudo`执行单个命令，审计更方便，风险更低；仅在需要长时间以 root 身份操作时用`su -`；
    - 典型错误：生产环境一直用`su -`切换到 root 操作，误操作导致系统崩溃；
    - 正确示例：单个命令用`sudo 命令`，长时间操作用`su -`，操作完及时退出。
    
4. **系统用户默认 shell 为 /sbin/nologin**
    
    - 系统用户（如`sysuser`、`daemon`）的默认 shell 通常为`/sbin/nologin`，直接切换会失败；
    - 解决方法：切换系统用户需指定 shell，如`su -s /bin/bash sysuser`；
    - 典型错误：`su - sysuser`报错`This account is currently not available.`；
    - 正确示例：`su -s /bin/bash sysuser`。
    
5. **密码输入不回显**
    
    - `su`输入密码时**不回显**，这是正常的安全机制，不是故障；
    - 解决方法：正常输入密码，输完按回车即可；
    - 典型错误：以为输入没反应，反复输入；
    - 正确示例：正常输入密码，输完按回车。
    
6. **环境变量的保留**
    
    - 默认`su -`会**重置环境变量**，需保留当前环境变量用`-m`或`-p`；
    - 解决方法：需保留环境变量时用`su -m 用户名`；
    - 典型错误：`su -`后发现当前环境变量丢失；
    - 正确示例：`su -m user1`保留环境变量。
    
7. **不要在生产环境滥用`su root`**
    
    - 生产环境滥用`su root`会导致审计困难，风险极高；
    - 强制要求：生产环境推荐用`sudo`，审计更方便，风险更低；
    - 典型错误：生产环境所有操作都用`su -`切换到 root；
    - 正确示例：单个命令用`sudo`，长时间操作用`su -`，操作完及时退出。
    

## 六、同类命令对比 & 拓展

表格

|命令|核心作用|与 su 的区别 & 适用场景|
|:--|:--|:--|
|`sudo`|以目标用户身份执行单个命令，无需完全切换身份|su 完全切换身份，适合长时间操作；sudo 执行单个命令，审计方便，风险更低。单个命令用 sudo，长时间操作用 su，两者常配合使用|
|`sudo su -`|结合 sudo 和 su，无需输入 root 密码切换到 root|su 需输入 root 密码；sudo su - 无需输入 root 密码（需 sudo 权限）。需切换到 root 且不想输入 root 密码用 sudo su -|
|`runuser`|root 切换到普通用户，更安全，无需密码|su root 切换到普通用户也无需密码；runuser 更安全，仅 root 可用。root 切换到普通用户可用 su 或 runuser|

## 七、课后练习任务

**重要提示：所有练习仅在测试环境操作，绝对不要在生产环境滥用`su root`！**

1. 基础切换练习：用普通用户身份执行`su`切换到 root，输入 root 密码，观察提示符和当前目录；再执行`su -`切换到 root 的登录 shell，对比两者的家目录和环境变量（用`pwd`和`echo $PATH`查看）。
2. root 切换到普通用户练习：用 root 身份执行`su - user1`切换到普通用户，观察无需密码的效果。
3. 执行单个命令练习：用普通用户身份执行`su -c "ls /root" root`，输入 root 密码，观察执行单个命令的效果。
4. 保留环境变量练习：设置一个环境变量`export MYVAR=myvalue`，执行`su -m user1`，用`echo $MYVAR`验证环境变量是否保留。
5. 切换到系统用户练习：创建一个系统用户`sysuser`（`useradd -r sysuser`），尝试直接`su - sysuser`，观察报错；再用`su -s /bin/bash sysuser`指定 shell 切换，验证效果。
6. 结合 sudo 练习：用普通用户身份执行`sudo su -`（需当前用户有 sudo 权限），观察无需输入 root 密码切换到 root 的效果。
7. 权限不足练习：用普通用户身份执行`su user2`（user2 是另一个普通用户），观察需输入 user2 密码的要求；尝试不输入看报错。
8. 帮助练习：用`su --help`查看帮助信息，浏览所有参数。
