## 一、核心定位

`passwd` 是 Linux 系统中**最核心、最基础的用户密码管理与密码策略配置命令**，属于「用户管理类」，普通用户仅能修改自己的密码，root 用户可修改所有用户的密码、锁定 / 解锁用户、删除密码、强制密码过期、配置密码时效策略。核心作用是**修改用户登录密码、管理用户账户状态、配置密码使用时效**，是用户安全管理、账户控制、密码策略实施的必备基础命令，也是 Linux 入门必学的用户管理命令。

> 补充说明：`passwd` 名称来源于 “password”（密码）；修改的密码信息会加密存储在`/etc/shadow`文件（仅 root 可读）；与`chage`（修改密码时效）、`usermod`（用户账户修改）并称为 Linux 用户密码与账户管理工具。

## 二、语法格式

bash

运行

```
# 核心语法1：普通用户修改自己的密码
passwd

# 核心语法2：root用户修改指定用户的密码
passwd [可选参数] 用户名
```

> 关键语法说明：
> 
> 1. **普通用户**：仅能修改自己的密码，需输入旧密码验证，且需满足系统密码复杂度要求；
> 2. **root 用户**：可修改所有用户的密码，无需输入旧密码，可直接设置新密码；
> 3. **可选参数**：如`-l`（锁定）、`-u`（解锁）、`-d`（删除密码）、`-e`（强制过期）、`-S`（查看状态）等；
> 4. **默认行为**：交互式输入密码，密码不回显。

## 三、高频核心参数（按功能分类）

### 1. 基础密码修改类

表格

|参数|核心作用|高频使用场景|
|:--|:--|:--|
|无参数（普通用户直接执行`passwd`）|普通用户修改自己的密码，需输入旧密码验证|日常用户修改自己的登录密码，最基础、最常用的用法|
|无参数（root 直接执行`passwd`）|root 修改当前用户的密码，无需输入旧密码|root 修改自己的密码|
|用户名（root 执行`passwd 用户名`）|root 修改指定用户的密码，无需输入旧密码|管理员重置用户密码、新用户设置初始密码|
|`--stdin`|从**标准输入**读取密码（非交互式），适合脚本批量设置|脚本批量设置用户密码、自动化配置|
|`-h, --help`|输出 passwd 帮助信息|快速查看参数与用法|
|`-v, --version`|输出 passwd 版本信息|确认版本兼容性|

### 2. 账户状态管理类

表格

|参数|核心作用|高频使用场景|
|:--|:--|:--|
|`-l, --lock`|**锁定**用户账户，禁止登录（在密码前加`!`）|临时禁止用户登录、账户安全控制|
|`-u, --unlock`|**解锁**用户账户，恢复登录（移除密码前的`!`）|恢复用户登录、解除账户锁定|
|`-d, --delete`|**删除**用户密码，用户可无密码登录（高风险）|特殊场景、临时无密码访问（需谨慎）|
|`-e, --expire`|**强制**用户密码过期，下次登录必须修改密码|新用户首次登录强制改密、定期密码重置|

### 3. 密码状态查看类

表格

|参数|核心作用|高频使用场景|
|:--|:--|:--|
|`-S, --status`|查看用户的**密码状态**（账户状态、最后修改时间、最小 / 最大 / 警告 / 非活动天数）|查看用户账户信息、密码策略检查|

### 4. 密码时效策略类

表格

|参数|核心作用|高频使用场景|
|:--|:--|:--|
|`-n, --minimum DAYS`|设置密码**最小使用天数**（DAYS 天内不能修改密码）|防止用户频繁修改密码|
|`-x, --maximum DAYS`|设置密码**最大使用天数**（DAYS 天后必须修改密码）|强制用户定期修改密码|
|`-w, --warning DAYS`|设置密码**过期警告天数**（过期前 DAYS 天开始警告）|提前提醒用户修改密码|
|`-i, --inactive DAYS`|设置密码**过期非活动天数**（过期后 DAYS 天仍可登录，之后账户锁定）|密码过期后缓冲期|

passwd

bash

运行

```
# 示例2：root用户修改指定用户的密码
passwd user1
# 执行效果：
# Changing password for user user1.
# New password: （输入新密码）
# Retype new password: （再次输入新密码）
# passwd: all authentication tokens updated successfully.
```

### 2. 高频日常场景示例

#### 场景 1：锁定 / 解锁用户账户

bash

运行

```
# 锁定用户账户，禁止登录
passwd -l user1
# 执行效果：user1的密码前加!，无法登录

# 解锁用户账户，恢复登录
passwd -u user1
# 执行效果：移除user1密码前的!，恢复登录
```

#### 场景 2：删除用户密码（无密码登录，高风险）

bash

运行

```
# 删除用户密码，用户可无密码登录
passwd -d user1
# 执行效果：user1的密码字段为空，可无密码登录（需谨慎）
```

#### 场景 3：强制用户密码过期（下次登录必须改）

bash

运行

```
# 强制用户密码过期，下次登录必须修改
passwd -e user1
# 执行效果：user1下次登录时会提示必须修改密码
```

#### 场景 4：查看用户密码状态

bash

运行

```
# 查看用户的密码状态
passwd -S user1
# 执行效果示例：
# user1 PS 2026-03-15 0 99999 7 -1 (Password set, SHA512 crypt.)
# 解释：用户名、状态、最后修改时间、最小天数、最大天数、警告天数、非活动天数
```

### 3. 进阶奇妙用法

#### 奇妙用法 1：设置密码时效策略

bash

运行

```
# 设置密码最小使用天数为7天（7天内不能改）
passwd -n 7 user1

# 设置密码最大使用天数为90天（90天后必须改）
passwd -x 90 user1

# 设置密码过期警告天数为7天（过期前7天开始警告）
passwd -w 7 user1

# 设置密码过期非活动天数为30天（过期后30天仍可登录）
passwd -i 30 user1

# 同时设置所有策略
passwd -n 7 -x 90 -w 7 -i 30 user1
```

#### 奇妙用法 2：结合 chage 查看更详细的密码时效

bash

运行

```
# 用passwd -S查看基础状态，用chage查看更详细的时效
passwd -S user1
chage -l user1
# 执行效果：chage显示更详细的密码时效信息
```

#### 奇妙用法 3：批量设置用户密码（结合循环）

bash

运行

```
# 批量设置user1到user5的密码为newpassword
for i in {1..5}; do
    echo "newpassword" | passwd --stdin user$i
done
```

## 五、避坑提示 & 注意事项

1. **新手最高频踩坑：普通用户只能改自己的密码**
    
    - 普通用户**仅能修改自己的密码**，需输入旧密码验证，且需满足系统密码复杂度要求；
    - 强制要求：修改其他用户的密码必须加`sudo`，或在 root 用户下操作；
    - 典型错误：普通用户执行`passwd user1`（报错`passwd: Only root can specify a user name.`）；
    - 正确示例：`sudo passwd user1`。
    
2. **密码复杂度要求**
    
    - 系统通常有密码复杂度要求（长度、大小写、数字、特殊字符），普通用户修改密码时需满足；
    - 解决方法：设置密码时遵循复杂度要求，root 用户可绕过（但不推荐）；
    - 典型错误：普通用户设置简单密码（如`123456`）报错`passwd: password is too simple`；
    - 正确示例：设置符合复杂度的密码（如`Abc123!@#`）。
    
3. **`-l`锁定用户的影响**
    
    - `-l`锁定用户后，用户无法通过密码登录，但仍可通过 SSH 密钥登录（若配置了密钥）；
    - 解决方法：若需完全禁止用户登录，可同时修改用户的 shell 为`/sbin/nologin`（`usermod -s /sbin/nologin user1`）；
    - 典型错误：仅用`passwd -l user1`以为完全禁止登录，实际用户仍可通过密钥登录；
    - 正确示例：`passwd -l user1 && usermod -s /sbin/nologin user1`。
    
4. **`-d`删除密码的高风险**
    
    - `-d`删除用户密码后，用户可无密码登录，极高风险，仅用于特殊测试环境；
    - 强制要求：不要在生产环境使用`-d`，仅在测试环境使用；
    - 典型错误：生产环境执行`passwd -d user1`（用户可无密码登录，高风险）；
    - 正确示例：仅在测试环境使用`-d`，且及时恢复密码。
    
5. **`-e`强制密码过期的使用**
    
    - `-e`强制用户密码过期后，用户下次登录必须修改密码，适合新用户首次登录；
    - 解决方法：新用户创建后用`passwd -e 用户名`强制首次改密；
    - 典型错误：新用户创建后不强制改密，用户一直使用初始密码；
    - 正确示例：`useradd user1 && passwd user1 && passwd -e user1`。
    
6. **密码时效策略的合理设置**
    
    - 密码时效策略（`-n`/`-x`/`-w`/`-i`）需根据安全需求合理设置，避免过严或过松；
    - 推荐设置：最小 7 天、最大 90 天、警告 7 天、非活动 30 天；
    - 典型错误：设置最大天数为 1 天（用户每天必须改密，过严）；
    - 正确示例：`passwd -n 7 -x 90 -w 7 -i 30 user1`。
    
7. **不要修改系统用户的密码**
    
    - 不要修改系统已存在的重要用户（如 root、bin、daemon）的密码或状态，否则会导致系统服务异常；
    - 强制要求：仅修改自己创建的普通用户的密码，不要修改系统用户；
    - 典型错误：`passwd -l root`（锁定 root 用户，高风险，绝对禁止尝试）；
    - 正确示例：仅修改自己创建的普通用户，如`passwd user1`。
    
8. **`--stdin`的安全性**
    
    - `--stdin`从标准输入读取密码，密码会在命令历史中留下记录（`history`），不安全；
    - 解决方法：脚本中使用`--stdin`后，及时清除命令历史（`history -c`），或用更安全的方式；
    - 典型错误：`echo "newpassword" | passwd --stdin user1`（密码在 history 中可见）；
    - 正确示例：`echo "newpassword" | passwd --stdin user1 && history -c`（仅在测试环境）。
    
9. **密码存储在 /etc/shadow，不要直接编辑**
    
    - 密码信息加密存储在`/etc/shadow`文件（仅 root 可读），不要直接编辑该文件；
    - 解决方法：仅用`passwd`、`chage`、`usermod`等命令修改密码和账户信息；
    - 典型错误：直接编辑`/etc/shadow`（可能导致文件损坏，用户无法登录）；
    - 正确示例：仅用`passwd`等命令修改，不要直接编辑`/etc/shadow`。
    

## 六、同类命令对比 & 拓展

表格

|命令|核心作用|与 passwd 的区别 & 适用场景|
|:--|:--|:--|
|`chage`|修改用户密码时效策略|passwd 可修改密码、锁定 / 解锁、删除密码、强制过期、查看状态、设置基础时效；chage 专门用于修改更详细的密码时效策略。基础密码管理用 passwd，详细时效配置用 chage，两者常配合使用|
|`usermod`|修改用户账户信息|passwd 可锁定 / 解锁用户（`-l`/`-u`）；usermod 也可锁定 / 解锁用户（`-L`/`-U`），还可修改用户的 shell、主组、附加组等。密码管理用 passwd，账户信息修改用 usermod，两者常配合使用|
|`useradd`|创建新用户|passwd 修改已有用户的密码；useradd 创建新用户，创建后需用 passwd 设置初始密码。创建用户用 useradd，设置密码用 passwd，两者常配合使用|
|`cat /etc/shadow`|查看密码信息（仅 root）|优势：查看加密的密码信息；劣势：无法修改，且不安全。查看密码状态用 passwd -S，不要直接查看 /etc/shadow 的密码字段|
|`id`|查看用户账户信息|优势：查看用户的 UID、GID、所属组；劣势：无法查看密码状态。查看用户基本信息用 id，查看密码状态用 passwd -S|

## 七、课后练习任务

**重要提示：所有练习仅在测试环境操作，绝对不要修改系统已存在的重要用户（如 root）！**

1. 基础修改密码练习：创建一个测试用户`user1`，用普通用户身份执行`passwd`修改自己的密码，观察旧密码验证和复杂度要求；再用`sudo passwd user1`修改`user1`的密码，观察无需旧密码的效果。
2. 从标准输入读取密码练习：用`echo "newpassword" | sudo passwd --stdin user1`直接设置`user1`的密码，验证效果（仅在测试环境，且及时清除 history）。
3. 锁定 / 解锁用户练习：用`sudo passwd -l user1`锁定`user1`，尝试登录验证；再用`sudo passwd -u user1`解锁，验证恢复登录。
4. 删除密码练习：用`sudo passwd -d user1`删除`user1`的密码，尝试无密码登录（仅在测试环境，且及时恢复密码）。
5. 强制密码过期练习：用`sudo passwd -e user1`强制`user1`密码过期，尝试登录验证是否提示必须修改密码。
6. 查看密码状态练习：用`sudo passwd -S user1`查看`user1`的密码状态，观察各项信息的含义。
7. 设置密码时效策略练习：用`sudo passwd -n 7 -x 90 -w 7 -i 30 user1`设置`user1`的密码时效策略，用`sudo passwd -S user1`验证设置是否生效。
8. 结合 chage 练习：用`sudo passwd -S user1`查看基础状态，用`sudo chage -l user1`查看更详细的时效信息，对比两者的差异。
9. 批量设置密码练习：用循环批量设置`user1`到`user5`的密码（仅在测试环境）。
10. 权限不足练习：尝试用普通用户修改其他用户的密码，观察权限不足的报错；再用`sudo`修改，验证效果。
11. 帮助练习：用`passwd --help`查看帮助信息，浏览所有参数。
