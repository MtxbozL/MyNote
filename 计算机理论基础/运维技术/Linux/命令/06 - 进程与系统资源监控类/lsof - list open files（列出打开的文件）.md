## 一、核心定位

`lsof` 是 Linux 上**最强大、最全面的 “打开文件” 查看工具**，属于「系统监控与故障排查类」，普通用户可查看自己进程的打开文件，root 可查看所有。核心作用是**利用 “Linux 一切皆文件” 的特性，查看进程打开的普通文件、目录、网络连接（TCP/UDP）、管道、设备、套接字等**，是端口占用排查、进程文件关联分析、网络连接监控的首选高级工具，也是 Linux 进阶必学的系统工具。

> 补充说明：`lsof` 名称来源于 “list open files”；多数发行版默认**不安装**，需手动安装（Debian/Ubuntu：`sudo apt install lsof`，RHEL/CentOS：`sudo yum install lsof`）；功能极丰富，本文仅覆盖最常用的部分。

---

## 二、语法格式

```bash
lsof [选项]
```

> 关键说明：
> 
> 1. **无参数**：默认列出**所有进程**打开的**所有文件**（输出较多，需配合筛选）。
> 2. **筛选条件**：可通过用户、PID、进程名、文件路径、网络连接等多种条件筛选。

---

## 三、高频核心选项

### 1. 基础进程筛选类

| 选项                      | 核心作用                   | 高频使用场景                       |
| :---------------------- | :--------------------- | :--------------------------- |
| `-u, --user USERNAME`   | 仅显示**指定用户**的进程打开的文件    | 查看特定用户的文件 / 网络活动             |
| `-p, --pid PID`         | 仅显示**指定 PID** 的进程打开的文件 | 查看特定进程的文件关联                  |
| `-c, --command COMMAND` | 仅显示**指定进程名**的进程打开的文件   | 查看特定服务（如 nginx、sshd）的文件 / 网络 |

### 2. 文件 / 目录筛选类

| 选项                    | 核心作用                    | 高频使用场景     |
| :-------------------- | :---------------------- | :--------- |
| 直接跟文件路径               | 查看**指定文件**被哪些进程打开       | 排查文件被占用的问题 |
| `+D, --directory DIR` | 递归查看**指定目录**下的文件被哪些进程打开 | 排查目录被占用的问题 |

### 3. 网络连接筛选类（最常用）

| 选项                                      | 核心作用                                  | 高频使用场景                |
| :-------------------------------------- | :------------------------------------ | :-------------------- |
| `-i, --internet [PROTOCOL][:PORT][@IP]` | 查看**网络连接**（TCP/UDP），可加协议 / 端口 / IP 筛选 | **排查端口占用、查看网络连接的首选！** |
| `-n, --no-names`                        | 不解析**IP 为域名**，直接显示 IP                 | 避免 DNS 延迟，更快          |
| `-P, --no-port-names`                   | 不解析**端口为服务名**（如 80→http），直接显示端口号      | 避免服务名解析，更快            |

### 4. 其他实用类

| 选项            | 核心作用               | 高频使用场景             |
| :------------ | :----------------- | :----------------- |
| `-t, --terse` | 仅输出**PID**，不显示其他信息 | 适合管道传递给 `kill` 等命令 |
| `-h, --help`  | 输出帮助信息             | 快速查看参数             |

## 四、核心示例

### 1. 基础进程筛选

```bash
# 查看所有进程打开的所有文件（输出较多，仅测试）
lsof

# 仅查看当前用户的进程打开的文件
lsof -u $USER

# 仅查看 PID 为 1234 的进程打开的文件
lsof -p 1234

# 仅查看 nginx 进程打开的文件
lsof -c nginx
```

### 2. 文件 / 目录筛选

```bash
# 查看 /etc/passwd 被哪些进程打开
lsof /etc/passwd

# 递归查看 /var/log 目录下的文件被哪些进程打开
lsof +D /var/log
```

### 3. 网络连接筛选（最常用！）

bash

运行

```
# 查看所有网络连接（TCP/UDP）
lsof -i

# 仅查看 TCP 连接
lsof -i tcp

# 仅查看 UDP 连接
lsof -i udp

# 查看 80 端口被哪些进程占用（排查端口占用必备！）
lsof -i :80

# 查看 22 端口的连接，不解析 IP/端口（更快）
lsof -i :22 -nP

# 查看与 192.168.1.10 的连接
lsof -i @192.168.1.10
```

### 4. 其他实用

bash

运行

```
# 仅输出 nginx 进程的 PID
lsof -t -c nginx

# 结合 kill，杀死所有 nginx 进程（仅测试环境！）
kill -9 $(lsof -t -c nginx)
```

## 五、避坑提示与注意事项

1. **普通用户只能看自己的进程，看所有需 root**
    
    - 普通用户运行 `lsof` 只能看到自己进程的打开文件，看不到其他用户的；
    - 解决方法：看所有进程需加 `sudo`，如 `sudo lsof -i :80`。
    
2. **排查端口占用用 `-i :端口 -nP`，最快最常用**
    
    - `-i :端口` 筛选端口，`-nP` 不解析名称，避免 DNS 延迟，速度快；
    - 典型错误：`lsof -i :80` 解析慢；
    - 正确示例：`sudo lsof -i :80 -nP`。
    
3. **“一切皆文件”，网络连接、设备、管道都是文件**
    
    - Linux 下网络连接、设备、管道、套接字都被视为文件，所以 `lsof` 都能查看；
    - 解决方法：灵活利用这一特性，排查各种资源占用。
    
4. **`-t` 仅输出 PID，适合管道**
    
    - `-t` 输出简洁，仅 PID，可直接传递给 `kill` 等命令；
    - 典型错误：手动复制 PID 再 kill；
    - 正确示例：`kill -9 $(sudo lsof -t -i :80)`（仅测试环境）。

## 六、同类对比

表格

|工具|核心作用|与 `lsof` 的区别|
|:--|:--|:--|
|`ss`/`netstat`|仅查看网络连接|`lsof` 可看网络连接 + 文件 + 设备；`ss`/`netstat` 更专注网络，速度更快。网络连接用 `ss`，全面查看用 `lsof`。|
|`fuser`|仅查看文件被哪些进程用|`lsof` 功能更全；`fuser` 更简单，仅看文件占用。简单文件占用用 `fuser`，全面用 `lsof`。|

## 七、课后练习

1. **安装 lsof**：如果未安装，用 `sudo apt install lsof` 或 `sudo yum install lsof` 安装。
2. **基础查看**：`lsof -u $USER` 查看当前用户的进程打开的文件。
3. **指定 PID**：`sudo lsof -p 1` 查看 init/systemd 进程打开的文件。
4. **指定进程名**：`sudo lsof -c sshd` 查看 sshd 进程打开的文件。
5. **查看文件占用**：`lsof /etc/passwd` 查看 /etc/passwd 被哪些进程打开。
6. **查看目录占用**：`sudo lsof +D /var/log` 递归查看 /var/log 目录的文件占用。
7. **查看网络连接**：

- `sudo lsof -i -nP` 查看所有网络连接，不解析名称。
- `sudo lsof -i :22 -nP` 查看 22 端口的连接。
- `sudo lsof -i tcp -nP` 仅查看 TCP 连接。

8. **仅输出 PID**：`sudo lsof -t -c sshd` 仅输出 sshd 的 PID。
9. **同类对比**：同时用 `sudo lsof -i :22 -nP` 和 `sudo ss -tuln` 查看 22 端口，对比两者的输出。
