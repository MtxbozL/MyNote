一、核心定位

`vmstat` 是 Linux 系统中**最核心、最基础的虚拟内存、分页、交换、块 IO、CPU、系统调用综合动态统计工具**，属于「系统监控类」，无任何权限要求，普通用户可直接执行。核心作用是**查看系统的虚拟内存使用、分页 / 交换活动、块设备 IO、CPU 使用率、系统中断 / 上下文切换的动态变化**，支持平均统计、持续刷新、指定单位显示，是系统内存监控、IO 瓶颈排查、CPU 性能分析的必备基础命令，也是 Linux 入门必学的系统监控命令。

> 补充说明：`vmstat` 名称来源于 “virtual memory statistics”（虚拟内存统计）；与`free`（仅内存快照）不同，`vmstat`显示的是**动态变化的统计信息**；默认无参数显示系统启动以来的平均统计，加延迟参数显示实时刷新；信息来源于`/proc`文件系统（`/proc/meminfo`、`/proc/stat`、`/proc/vmstat`等）。

## 二、语法格式

```bash
# 核心语法1：查看系统启动以来的平均统计
vmstat [可选参数]

# 核心语法2：持续刷新显示（延迟N秒，可选刷新M次后退出）
vmstat [可选参数] 延迟 [次数]
```

> 关键语法说明：
> 
> 1. **无文件参数**：直接运行即可，无需指定文件；
> 2. **延迟**：可选，正整数，单位秒，持续刷新的间隔；
> 3. **次数**：可选，正整数，配合延迟使用，刷新指定次数后自动退出；
> 4. **可选参数**：如`-a`（活跃内存）、`-d`（磁盘统计）、`-s`（事件统计）、`-t`（时间戳）、`-S`（单位）等；
> 5. **默认行为**：无参数显示系统启动以来的平均统计，第一行是平均，后续（加延迟）是实时。

## 三、高频核心参数（按功能分类）

### 1. 基础统计类

表格

|参数|核心作用|高频使用场景|
|:--|:--|:--|
|无参数（直接执行`vmstat`）|显示系统启动以来的**平均统计**|快速查看系统整体运行情况，最基础用法|
|`延迟`（如`vmstat 2`）|持续**刷新显示**，间隔 N 秒（如 2 秒）|实时监控内存 / IO/CPU 变化|
|`延迟 次数`（如`vmstat 2 5`）|持续刷新显示，间隔 N 秒，刷新 M 次后**自动退出**|定时监控、自动停止|

### 2. 显示内容类

表格

|参数|核心作用|高频使用场景|
|:--|:--|:--|
|`-a, --active`|显示**活跃 / 非活跃内存**（inact/active 列）|详细区分内存使用状态|
|`-d, --disk`|显示**磁盘统计**（读 / 写 / IOPS 等）|磁盘 IO 性能排查|
|`-p, --partition DEVICE`|显示**指定分区**的统计（DEVICE 为分区名，如`/dev/sda1`）|特定分区 IO 监控|
|`-s, --stats`|显示**内存事件统计**（分页、交换、中断等累计数）|系统事件累计分析|
|`-t, --timestamp`|显示**时间戳**（每行前加当前时间）|监控时记录时间|
|`-S, --unit UNIT`|指定**显示单位**（UNIT 为`k`/`K`/`m`/`M`，默认 KB）|更直观的单位显示|
|`-h, --help`|输出 vmstat 帮助信息|快速查看参数与用法|
|`-V, --version`|输出 vmstat 版本信息|确认版本兼容性|

## 四、核心列含义说明（默认显示）

表格

|区域|列名|含义|说明|
|:--|:--|:--|:--|
|**procs**（进程）|`r`|运行队列长度|等待 CPU 的进程数，高说明 CPU 瓶颈|
||`b`|不可中断睡眠队列长度|等待 IO 的进程数，高说明 IO 瓶颈|
|**memory**（内存）|`swpd`|已用交换空间|单位 KB，高说明物理内存不足|
||`free`|空闲物理内存|单位 KB|
||`buff`|缓冲区（文件系统元数据）|单位 KB，可回收|
||`cache`|缓存区（文件内容）|单位 KB，可回收|
|**swap**（交换）|`si`|交换空间换入|单位 KB / 秒，高说明内存不足|
||`so`|交换空间换出|单位 KB / 秒，高说明内存不足|
|**io**（块 IO）|`bi`|块设备读入|单位块 / 秒，高说明读 IO 繁忙|
||`bo`|块设备写出|单位块 / 秒，高说明写 IO 繁忙|
|**system**（系统）|`in`|中断次数|单位次 / 秒|
||`cs`|上下文切换次数|单位次 / 秒，高说明进程切换频繁|
|**cpu**（CPU）|`us`|用户态 CPU 使用率|百分比|
||`sy`|内核态 CPU 使用率|百分比|
||`id`|CPU 空闲率|百分比|
||`wa`|IO 等待 CPU 使用率|百分比，高说明 IO 瓶颈|
||`st`|被偷 CPU 时间（虚拟机）|百分比，虚拟机被宿主机偷的时间|

## 六、避坑提示 & 注意事项

1. **新手最高频踩坑：无参数是启动以来的平均，要看实时加延迟**
    
    - 无参数显示的是**系统启动以来的平均统计**，不是当前实时；
    - 强制要求：看实时必须加延迟，如`vmstat 2`；
    - 典型错误：`vmstat`以为是当前实时；
    - 正确示例：`vmstat 2`看实时。
    
2. **`si`/`so`高说明物理内存不足，系统在频繁用交换空间**
    
    - `si`（换入）和`so`（换出）高，说明物理内存不足，系统在频繁读写交换空间（硬盘），性能会大幅下降；
    - 解决方法：增加物理内存，或优化内存使用；
    - 典型错误：忽略`si`/`so`高，以为系统正常；
    - 正确示例：关注`si`/`so`列，高时排查内存问题。
    
3. **`wa`（IO 等待）高说明 IO 瓶颈**
    
    - `wa`（IO 等待 CPU 使用率）高，说明 CPU 在等待块设备 IO，IO 是瓶颈；
    - 解决方法：结合`iostat`排查磁盘 IO；
    - 典型错误：忽略`wa`高；
    - 正确示例：关注`wa`列，高时用`iostat`进一步排查。
    
4. **`buff`/`cache`是可回收的，不是浪费**
    
    - `buff`（缓冲区）和`cache`（缓存区）是 Linux 为了提升性能而使用的，会在应用需要时自动释放；
    - 解决方法：不要试图手动清除缓存（除非特殊测试），系统会自动管理；
    - 典型错误：以为`buff`/`cache`高是内存泄漏，手动清除；
    - 正确示例：让系统自动管理`buff`/`cache`。
    
5. **可以和`iostat`、`mpstat`结合使用**
    
    - `vmstat`看综合，`iostat`看详细 IO，`mpstat`看详细 CPU；
    - 解决方法：综合使用多个工具；
    - 典型错误：只用`vmstat`，不看其他；
    - 正确示例：`vmstat`+`iostat`+`mpstat`结合排查。
    
6. **不要直接编辑`/proc`，用`vmstat`看**
    
    - 信息来源于`/proc`文件系统，不要直接编辑；
    - 解决方法：仅用`vmstat`等命令查看；
    - 典型错误：直接编辑`/proc`；
    - 正确示例：用`vmstat`查看。
    

## 七、同类命令对比 & 拓展

表格

|命令|核心作用|与 vmstat 的区别 & 适用场景|
|:--|:--|:--|
|`free`|仅显示内存快照|vmstat 看内存 + IO+CPU 动态；free 仅看内存快照。仅看内存用 free，综合监控用 vmstat|
|`iostat`|仅显示块 IO 统计|vmstat 看综合 IO；iostat 看详细磁盘 IO。综合用 vmstat，详细 IO 用 iostat|
|`mpstat`|仅显示 CPU 统计|vmstat 看综合 CPU；mpstat 看每个 CPU 核心的详细统计。综合用 vmstat，详细 CPU 用 mpstat|
|`top`/`htop`|实时系统监控 + 进程|vmstat 看系统级统计；top/htop 看进程 + 系统。系统级统计用 vmstat，进程监控用 top/htop|
|`sar`|系统活动报告 + 历史数据|vmstat 看当前 / 平均；sar 看历史数据。当前用 vmstat，历史用 sar|

## 八、课后练习任务

1. 基础统计练习：用`vmstat`查看系统启动以来的平均统计，理解各列的含义（procs/memory/swap/io/system/cpu）。
2. 持续刷新练习：用`vmstat 2`持续刷新，每 2 秒一次，按 Ctrl+C 停止；再用`vmstat 2 5`刷新 5 次后自动退出。
3. 活跃内存练习：用`vmstat -a 2`查看活跃 / 非活跃内存，对比与默认的区别。
4. 磁盘统计练习：用`vmstat -d`查看磁盘统计。
5. 指定分区练习：用`vmstat -p /dev/sda1`（替换为你的分区）查看指定分区统计。
6. 事件统计练习：用`vmstat -s`查看内存事件统计。
7. 时间戳练习：用`vmstat -t 2`查看时间戳。
8. 指定单位练习：用`vmstat -S M 2`指定单位为 MB，对比与默认 KB 的区别。
9. 关注 si/so 练习：观察 swap 列的 si/so，若有使用，思考是否内存不足。
10. 关注 wa 练习：观察 cpu 列的 wa，若高，思考是否 IO 瓶颈。
11. 结合 watch 练习：用`watch -n 2 vmstat`持续监控，观察高亮变化。
12. 同类对比练习：同时运行`vmstat`、`free`、`iostat`（如果已安装），对比各工具的信息。
13. 帮助练习：用`vmstat --help`查看帮助信息，浏览所有参数。
