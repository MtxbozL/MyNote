这份配置整合了负载均衡、反向代理、动静分离、WebSocket 代理和安全加固，可直接用于生产环境部署，只需按实际环境调整关键参数。

---

## 一、全局配置

```nginx
user nginx;
worker_processes auto;
error_log /var/log/nginx/error.log warn;
pid /var/run/nginx.pid;

events {
    worker_connections 10240;
    use epoll;
    multi_accept on;
}
```

## 二、HTTP 核心模块

```nginx
http {
    include /etc/nginx/mime.types;
    default_type application/octet-stream;

    # 日志格式
    log_format main '$remote_addr - $remote_user [$time_local] "$request" '
                    '$status $body_bytes_sent "$http_referer" '
                    '"$http_user_agent" "$http_x_forwarded_for" '
                    '$request_time $upstream_response_time $upstream_addr';
    access_log /var/log/nginx/access.log main;

    # 性能调优
    sendfile on;
    tcp_nopush on;
    tcp_nodelay on;
    keepalive_timeout 65;
    keepalive_requests 1000;

    # Gzip 压缩
    gzip on;
    gzip_comp_level 6;
    gzip_min_length 256;
    gzip_types text/plain text/css application/json application/javascript 
               text/xml application/xml application/xml+rss text/javascript
               application/vnd.ms-fontobject application/x-font-ttf font/opentype
               image/svg+xml image/x-icon;
    gzip_proxied any;
    gzip_vary on;

    # -------------------------- 负载均衡后端集群 --------------------------
    # 1. 应用服务集群（最小连接 + 权重）
    upstream app_servers {
        least_conn;
        server 127.0.0.1:8080 weight=5 max_fails=3 fail_timeout=30s;
        server 127.0.0.1:8081 weight=3 max_fails=3 fail_timeout=30s;
        server 127.0.0.1:8082 backup;
    }

    # 2. WebSocket 服务集群
    upstream ws_servers {
        least_conn;
        server 127.0.0.1:8090 weight=2 max_fails=3 fail_timeout=30s;
    }

    # 3. 子请求/微服务集群
    upstream micro_servers {
        server 127.0.0.1:9000;
        server 127.0.0.1:9001;
    }
```

## 三、虚拟主机核心配置

```nginx
    server {
        listen 80;
        server_name your-domain.com www.your-domain.com;
        # 生产环境建议强制 HTTPS
        # return 301 https://$host$request_uri;

        # -------------------------- 1. 动静分离：静态资源 --------------------------
        # 网站图标（精确匹配）
        location = /favicon.ico {
            root /var/www/static/icons;
            expires 30d;
            access_log off;
        }

        # 静态资源目录（前缀匹配）
        location ^~ /static/ {
            root /var/www;
            expires 7d;
            add_header Cache-Control "public, max-age=604800";
            access_log off;
        }

        # 媒体文件（正则匹配）
        location ~* \.(jpg|jpeg|png|gif|webp|mp4|avi|mp3)$ {
            root /var/www/media;
            expires 30d;
            access_log off;
        }

        # 样式/脚本文件（正则匹配）
        location ~* \.(css|js|json|svg|woff|woff2)$ {
            root /var/www/assets;
            expires 7d;
            add_header Cache-Control "public, max-age=604800";
            access_log off;
        }

        # -------------------------- 2. WebSocket 代理 --------------------------
        location /ws/ {
            proxy_pass http://ws_servers;
            proxy_http_version 1.1;
            proxy_set_header Upgrade $http_upgrade;
            proxy_set_header Connection "upgrade";
            proxy_set_header Host $host;
            proxy_set_header X-Real-IP $remote_addr;
            proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
            proxy_read_timeout 86400s;
            proxy_send_timeout 86400s;
            proxy_buffering off;
        }

        # -------------------------- 3. 子请求（接口聚合） --------------------------
        location /api/user {
            default_type application/json;
            rewrite ^ /_internal/user/info break;
            proxy_pass http://micro_servers;
            echo_subrequest GET /_internal/user/permission @permission;
            echo '{"user_info": $echo_response_body, "permission": $echo_subrequest_response_body}';
        }

        location /_internal/ {
            internal;
            proxy_pass http://micro_servers;
            proxy_set_header Host $host;
            proxy_set_header X-Real-IP $remote_addr;
            proxy_connect_timeout 10s;
            proxy_read_timeout 10s;
        }

        # -------------------------- 4. 反向代理：动态请求 --------------------------
        location / {
            proxy_pass http://app_servers;
            proxy_set_header Host $host;
            proxy_set_header X-Real-IP $remote_addr;
            proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
            proxy_set_header X-Forwarded-Proto $scheme;
            proxy_connect_timeout 30s;
            proxy_read_timeout 60s;
            proxy_send_timeout 30s;
            proxy_buffer_size 4k;
            proxy_buffers 8 4k;
            proxy_busy_buffers_size 8k;
        }

        # -------------------------- 5. 安全加固 --------------------------
        # 禁止访问敏感文件
        location ~* \.(conf|bak|sql|log|sh)$ {
            deny all;
        }

        # 限制请求方法
        if ($request_method !~ ^(GET|POST|HEAD|PUT|DELETE|OPTIONS)$) {
            return 403;
        }

        # 错误页面
        error_page 404 /404.html;
        error_page 500 502 503 504 /50x.html;
        location = /50x.html {
            root /var/www/static/error;
        }
    }
```

## 四、HTTPS 配置（生产建议开启）

```nginx
    server {
        listen 443 ssl http2;
        server_name your-domain.com www.your-domain.com;

        ssl_certificate /etc/nginx/ssl/your-domain.crt;
        ssl_certificate_key /etc/nginx/ssl/your-domain.key;

        ssl_protocols TLSv1.2 TLSv1.3;
        ssl_prefer_server_ciphers on;
        ssl_ciphers ECDHE-RSA-AES256-GCM-SHA512:DHE-RSA-AES256-GCM-SHA512;
        ssl_session_timeout 10m;
        ssl_session_cache shared:SSL:10m;
        ssl_session_tickets off;

        # 复用 HTTP 配置
        location = /favicon.ico { ... }
        location ^~ /static/ { ... }
        location /ws/ { ... }
        location /api/user { ... }
        location /_internal/ { ... }
        location / { ... }
    }
}
```

## 五、部署与验证

1. **创建目录结构**

  ```bash
    mkdir -p /var/www/{static/icons,media,assets,static/error}
    mkdir -p /var/log/nginx
    mkdir -p /etc/nginx/ssl
    ```

2. **关键配置调整**
    
    - `server_name`：替换为你的实际域名。
    - `upstream` 节点：替换为你的后端服务地址。
    - `root` 路径：确保与服务器静态资源路径一致。
    - SSL 证书路径：启用 HTTPS 时替换为你的证书路径。
    
3. **验证与重载**

```bash
    nginx -t
    systemctl reload nginx
    ```

## 六、策略选择建议

|业务场景|推荐策略|
|:--|:--|
|通用 Web 服务|轮询（默认）或最小连接|
|后端处理能力差异大|最小连接 + 权重|
|会话一致性（登录 / 购物车）|IP 哈希|
|缓存服务器集群|一致性哈希|
|WebSocket 实时通信|最小连接 + 长超时|
