## 一、设置匹配规则（路由规则）- location

`location` 指令用于根据客户端请求的 URI 来匹配不同的处理规则，是实现动静分离的核心。

### 基本语法


```nginx
location [修饰符] pattern {
    ... 配置项 ...
}
```

### 动静分离典型配置示例


```nginx
server {
    listen 80;
    server_name example.com;

    # 静态资源：图片、样式、脚本等
    location ~* \.(jpg|jpeg|png|gif|css|js|ico|svg)$ {
        root /var/www/static;       # 静态文件根目录
        expires 7d;                  # 浏览器缓存7天
        add_header Cache-Control "public, max-age=604800";
    }

    # 动态请求：转发给后端应用服务器
    location / {
        proxy_pass http://127.0.0.1:8080;  # 转发到Tomcat/Node.js等
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }
}
```

---

## 二、匹配符

Nginx 提供了多种修饰符来改变 `location` 的匹配行为，优先级从高到低如下：

|修饰符|含义|匹配规则|
|:--|:--|:--|
|`=`|精确匹配|必须与请求 URI 完全一致，匹配后立即停止搜索|
|`^~`|前缀匹配|匹配 URI 前缀，匹配后立即停止搜索，不再检查正则|
|`~`|区分大小写的正则匹配|按正则表达式匹配 URI|
|`~*`|不区分大小写的正则匹配|按正则表达式匹配 URI|
|无修饰符|普通前缀匹配|匹配 URI 前缀，所有正则匹配失败后才会使用|

### 示例


```nginx
# 精确匹配，优先级最高
location = /favicon.ico { ... }

# 前缀匹配，匹配后不再检查正则
location ^~ /static/ { ... }

# 不区分大小写的正则匹配，常用于静态资源
location ~* \.(jpg|png|css|js)$ { ... }

# 普通前缀匹配，作为最后兜底
location / { ... }
```

---

## 三、优先级

Nginx 匹配 `location` 的顺序和优先级非常关键，直接影响请求的处理方式：

1. **精确匹配 (`=`)**：优先级最高。如果找到完全匹配，立即使用。
2. **前缀匹配 (`^~`)**：找到最长匹配的前缀后，立即停止搜索，不再检查正则表达式。
3. **正则匹配 (`~` / `~*`)**：按配置文件中出现的顺序进行检查，找到第一个匹配的正则后立即使用。
4. **普通前缀匹配（无修饰符）**：如果以上都未匹配，选择最长的前缀匹配。

### 优先级总结

`=` > `^~` > `~` / `~*` > 普通前缀匹配

## 四、完整动静分离配置示例


```nginx
http {
    server {
        listen 80;
        server_name www.mydomain.com;

        # 精确匹配网站图标
        location = /favicon.ico {
            root /var/www/images;
            expires 30d;
        }

        # 前缀匹配静态资源目录，优先级高于正则
        location ^~ /static/ {
            root /var/www;
            expires 1d;
        }

        # 正则匹配图片、视频等媒体文件
        location ~* \.(jpg|jpeg|png|gif|mp4|webm)$ {
            root /var/www/media;
            expires 7d;
        }

        # 正则匹配样式和脚本
        location ~* \.(css|js)$ {
            root /var/www/assets;
            expires 1d;
        }

        # 所有其他请求转发给后端应用
        location / {
            proxy_pass http://backend_server;
            proxy_set_header Host $host;
            proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        }
    }
}
```
