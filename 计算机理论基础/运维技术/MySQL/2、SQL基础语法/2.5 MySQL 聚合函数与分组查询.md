## 一、开篇：聚合与分组的作用

聚合函数与分组查询是 DQL（数据查询语言）中用于**数据统计与分析**的核心组合：

- **聚合函数**：对一组数据进行计算（如求和、平均值、计数），返回单个统计结果；
- **分组查询（GROUP BY）**：将数据按指定字段分组，再对每组分别使用聚合函数，实现「分组统计」（如统计每个用户的订单数、每个月的销售额）。

本笔记基于 `user_info`（用户表）和 `order_info`（订单表）演示，覆盖核心语法、常见场景与易错点。

---

## 二、聚合函数：数据统计的基础

聚合函数对**一组行**进行计算，忽略 `NULL` 值（除 `COUNT(*)` 外），返回单个结果。

### 2.1 常用聚合函数详解

|函数名|作用|语法|示例|关键说明|
|:--|:--|:--|:--|:--|
|**COUNT()**|统计行数|`COUNT(*)` / `COUNT(字段)`|`SELECT COUNT(*) AS '总用户数' FROM user_info;`|- `COUNT(*)`：统计**所有行**（包括 `NULL` 值行）；<br><br>- `COUNT(字段)`：统计该字段**非 NULL** 的行数。|
|**SUM()**|求和（仅数值型）|`SUM(字段)`|`SELECT SUM(order_amount) AS '总订单金额' FROM order_info;`|忽略 `NULL` 值，仅对数值型字段有效。|
|**AVG()**|求平均值（仅数值型）|`AVG(字段)`|`SELECT AVG(age) AS '平均年龄' FROM user_info;`|忽略 `NULL` 值，自动排除 `NULL` 行后计算。|
|**MAX()**|求最大值|`MAX(字段)`|`SELECT MAX(order_amount) AS '最高订单金额' FROM order_info;`|可用于数值型、日期型、字符串型（按字典序）。|
|**MIN()**|求最小值|`MIN(字段)`|`SELECT MIN(create_time) AS '最早下单时间' FROM order_info;`|同 `MAX()`，支持多种数据类型。|

### 2.2 聚合函数的关键注意事项

1. **NULL 值的处理**：
    
    - 除 `COUNT(*)` 外，所有聚合函数都**忽略 `NULL` 值**；
    - 示例：若 `age` 字段有 3 行 `NULL`，`COUNT(age)` 会统计非 `NULL` 的行数，`AVG(age)` 会自动排除这 3 行。
    
2. **聚合函数不能直接用于 WHERE 子句**：
    
    - 错误示例：`SELECT * FROM user_info WHERE age > AVG(age);`（语法报错）；
    - 原因：`WHERE` 执行在聚合函数之前，此时还未计算出聚合结果，需用子查询或 `HAVING` 解决。

## 三、分组查询（GROUP BY）：分组统计的核心

`GROUP BY` 用于将数据**按一个或多个字段分组**，分组后配合聚合函数，对每组分别统计。

### 3.1 核心语法

```sql
SELECT 分组字段1, 分组字段2, ..., 聚合函数(字段)
FROM 表名
[WHERE 条件]
GROUP BY 分组字段1, 分组字段2, ...;
```

### 3.2 常见示例

|需求|示例代码|说明|
|:--|:--|:--|
|**按性别分组，统计每组的用户数和平均年龄**|`SELECT gender, COUNT(*) AS '用户数', AVG(age) AS '平均年龄' FROM user_info GROUP BY gender;`|分组字段为 `gender`，对每组分别计数、求平均。|
|**按用户 ID 分组，统计每个用户的订单数和总金额**|`SELECT user_id, COUNT(*) AS '订单数', SUM(order_amount) AS '总金额' FROM order_info GROUP BY user_id;`|分组字段为 `user_id`，统计每个用户的订单情况。|
|**多字段分组：按性别 + 年龄分组，统计用户数**|`SELECT gender, age, COUNT(*) AS '用户数' FROM user_info GROUP BY gender, age;`|先按 `gender` 分组，再在每组内按 `age` 细分。|

### 3.3 GROUP BY 的关键注意事项

1. **SELECT 字段的限制**：
    
    - `SELECT` 中的**非聚合字段必须出现在 `GROUP BY` 中**（MySQL 5.7+ 默认开启 `ONLY_FULL_GROUP_BY` 模式，违反会报错）；
    - 错误示例：`SELECT gender, age, COUNT(*) FROM user_info GROUP BY gender;`（`age` 不在 `GROUP BY` 中）；
    - 正确示例：`SELECT gender, age, COUNT(*) FROM user_info GROUP BY gender, age;`（多字段分组）。
    
2. **NULL 值的分组**：
    
    - `GROUP BY` 会将所有 `NULL` 值分为**同一组**；
    - 示例：若 `gender` 字段有 2 行 `NULL`，分组后这 2 行会被统计为一组。
    
3. **WHERE 与 GROUP BY 的配合**：
    
    - `WHERE` 执行在 `GROUP BY` 之前，先过滤数据，再分组，可减少分组的数据量；
    - 示例：`SELECT gender, COUNT(*) FROM user_info WHERE age > 25 GROUP BY gender;`（先筛选年龄 > 25 的用户，再按性别分组）。

## 四、HAVING 子句：分组后的过滤

`HAVING` 用于**过滤分组后的结果**，通常与 `GROUP BY` 配合使用（也可单独使用，但场景极少）。

### 4.1 核心语法

```sql
SELECT 分组字段, 聚合函数(字段)
FROM 表名
[WHERE 条件]
GROUP BY 分组字段
HAVING 分组后条件;
```

### 4.2 关键区别：WHERE vs HAVING

这是最容易混淆的知识点，需重点掌握：

|对比维度|WHERE|HAVING|
|:--|:--|:--|
|**作用阶段**|分组前（过滤原始行）|分组后（过滤分组结果）|
|**过滤对象**|表中的原始行|分组后的统计结果|
|**可使用的内容**|原始字段、普通条件|分组字段、聚合函数结果|
|**示例**|`WHERE age > 25`|`HAVING AVG(age) > 25`|

### 4.3 常见示例

|需求|示例代码|说明|
|:--|:--|:--|
|**统计平均年龄大于 25 的性别分组**|`SELECT gender, AVG(age) AS '平均年龄' FROM user_info GROUP BY gender HAVING AVG(age) > 25;`|先分组求平均，再过滤平均年龄 > 25 的组。|
|**统计订单数≥2 的用户**|`SELECT user_id, COUNT(*) AS '订单数' FROM order_info GROUP BY user_id HAVING COUNT(*) >= 2;`|先分组统计订单数，再筛选订单数≥2 的用户。|
|**WHERE + GROUP BY + HAVING 配合**|`SELECT user_id, SUM(order_amount) AS '总金额' FROM order_info WHERE order_status = '已支付' GROUP BY user_id HAVING SUM(order_amount) > 500;`|先筛选已支付订单，再分组统计总金额，最后过滤总金额 > 500 的用户。|

## 五、综合实操演练

基于 `user_info` 和 `order_info` 表，完成以下需求：

1. 统计用户表的总人数、最大年龄、最小年龄；
2. 按订单状态分组，统计每种状态的订单数和平均金额；
3. 统计每个用户的订单数，筛选出订单数≥3 的用户；
4. 先筛选年龄在 20-35 岁之间的用户，再按性别分组，统计每组的平均年龄，筛选平均年龄 > 28 的组。

### 参考答案

```sql
-- 1. 统计总人数、最大年龄、最小年龄
SELECT 
  COUNT(*) AS '总人数', 
  MAX(age) AS '最大年龄', 
  MIN(age) AS '最小年龄' 
FROM user_info;

-- 2. 按订单状态分组统计
SELECT 
  order_status, 
  COUNT(*) AS '订单数', 
  AVG(order_amount) AS '平均金额' 
FROM order_info 
GROUP BY order_status;

-- 3. 统计订单数≥3的用户
SELECT 
  user_id, 
  COUNT(*) AS '订单数' 
FROM order_info 
GROUP BY user_id 
HAVING COUNT(*) >= 3;

-- 4. WHERE + GROUP BY + HAVING 配合
SELECT 
  gender, 
  AVG(age) AS '平均年龄' 
FROM user_info 
WHERE age BETWEEN 20 AND 35 
GROUP BY gender 
HAVING AVG(age) > 28;
```

## 六、核心要点总结

1. **聚合函数**：
    
    - `COUNT(*)` 统计所有行，`COUNT(字段)` 统计非 NULL 行；
    - 聚合函数忽略 NULL 值（除 `COUNT(*)`）；
    - 聚合函数不能直接用于 WHERE 子句。
    
2. **GROUP BY**：
    
    - 用于分组，配合聚合函数实现分组统计；
    - SELECT 中的非聚合字段必须出现在 GROUP BY 中；
    - NULL 值会被分为同一组。
    
3. **HAVING vs WHERE**：
    
    - WHERE 分组前过滤原始行，HAVING 分组后过滤统计结果；
    - HAVING 可使用聚合函数，WHERE 不行；
    - 优先用 WHERE 过滤数据，减少分组量。
    
4. **完整查询顺序**：
    
    `SELECT` → `FROM` → `WHERE` → `GROUP BY` → `HAVING` → `ORDER BY` → `LIMIT`。
