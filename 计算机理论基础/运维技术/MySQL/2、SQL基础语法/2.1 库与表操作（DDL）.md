## 一、DDL 基础定义

**DDL（Data Definition Language，数据定义语言）**：用于定义和管理数据库对象（库、表、视图、索引等）的 SQL 语言，核心关键字为 `CREATE`（创建）、`ALTER`（修改）、`DROP`（删除）、`TRUNCATE`（清空），操作会直接修改数据库结构，**执行前需谨慎**。

---

## 二、库的操作（Database）

### 1. 核心语法与示例

表格

|操作|语法|示例|说明|
|:--|:--|:--|:--|
|**创建库**|`CREATE DATABASE [IF NOT EXISTS] 库名 [DEFAULT CHARACTER SET 字符集] [COLLATE 排序规则];`|`CREATE DATABASE IF NOT EXISTS study_mysql DEFAULT CHARACTER SET utf8mb4 COLLATE utf8mb4_general_ci;`|`IF NOT EXISTS`避免重复创建报错；`utf8mb4`是推荐字符集（支持 emoji）|
|**删除库**|`DROP DATABASE [IF EXISTS] 库名;`|`DROP DATABASE IF EXISTS study_mysql;`|`IF EXISTS`避免删除不存在的库报错；**删除后数据不可恢复**|
|**使用库**|`USE 库名;`|`USE study_mysql;`|切换当前操作的数据库，后续表操作默认针对该库|
|**查看库列表**|`SHOW DATABASES;`|`SHOW DATABASES;`|查看当前 MySQL 实例下的所有数据库|
|**查看库创建语句**|`SHOW CREATE DATABASE 库名;`|`SHOW CREATE DATABASE study_mysql;`|查看库的创建语句（含字符集、排序规则等配置）|

## 三、表的操作之创建（CREATE TABLE）

### 3.1 常用数据类型（核心必记）

数据类型定义了字段的存储规则，是建表的基础，以下是最常用的类型：

表格

|类型分类|类型名|说明|适用场景|
|:--|:--|:--|:--|
|**数值型**|`INT`|普通整数，占 4 字节，范围约 - 21 亿到 21 亿|用户 ID、年龄、数量等常规整数|
||`BIGINT`|大整数，占 8 字节，范围约 - 922 亿亿到 922 亿亿|订单 ID、流水号等超大整数|
||`DECIMAL(M,D)`|精确小数，M 为总位数，D 为小数位数|金额、价格等需精确计算的数值（如`DECIMAL(10,2)`表示最多 10 位，其中 2 位小数）|
|**字符串型**|`VARCHAR(M)`|可变长度字符串，M 为最大长度（最多 65535 字节）|用户名、地址、描述等长度不固定的文本|
||`CHAR(M)`|固定长度字符串，M 为固定长度（最多 255 字节）|手机号、身份证号等长度固定的文本|
||`TEXT`|长文本，最多 65535 字节|文章内容、备注等长文本|
|**日期时间型**|`DATETIME`|日期时间，格式`YYYY-MM-DD HH:MM:SS`，范围 1000-01-01 到 9999-12-31|订单创建时间、用户注册时间等通用日期时间|
||`TIMESTAMP`|时间戳，格式同`DATETIME`，范围 1970-01-01 到 2038-01-19，**自动更新时区**|记录最后修改时间（可设置自动更新）|
||`DATE`|日期，格式`YYYY-MM-DD`|生日、下单日期等仅需日期的场景|

### 3.2 核心约束（保障数据完整性的关键）

约束是对字段的强制规则，避免脏数据，以下是 6 个核心约束：

表格

|约束名|关键字|作用|语法示例|
|:--|:--|:--|:--|
|**主键约束**|`PRIMARY KEY`|唯一标识一条记录，非空、不重复，一张表只能有一个主键|`user_id INT PRIMARY KEY AUTO_INCREMENT COMMENT '用户ID，主键，自增'`（`AUTO_INCREMENT`表示自增，仅适用于整数型主键）|
|**非空约束**|`NOT NULL`|字段值不能为空|`user_name VARCHAR(50) NOT NULL COMMENT '用户名，非空'`|
|**唯一约束**|`UNIQUE`|字段值不能重复（但可以为空）|`phone VARCHAR(11) UNIQUE NOT NULL COMMENT '手机号，唯一且非空'`|
|**默认值约束**|`DEFAULT`|字段未赋值时使用默认值|`create_time DATETIME DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间，默认当前时间'`（`CURRENT_TIMESTAMP`表示当前时间）|
|**检查约束**|`CHECK`|限制字段的取值范围（MySQL 8.0 + 原生支持）|`age TINYINT CHECK (age >= 0 AND age <= 150) COMMENT '年龄，0-150之间'`|
|**外键约束**|`FOREIGN KEY`|建立两张表的关联关系，维护参照完整性|`FOREIGN KEY (user_id) REFERENCES user_info(user_id)`（当前表的`user_id`关联`user_info`表的`user_id`）|

### 3.3 完整建表示例（含所有核心约束）

sql

```
-- 先切换到目标库
USE study_mysql;

-- 创建用户表（主表）
CREATE TABLE IF NOT EXISTS user_info (
    user_id INT PRIMARY KEY AUTO_INCREMENT COMMENT '用户ID，主键，自增',
    user_name VARCHAR(50) NOT NULL COMMENT '用户名，非空',
    phone VARCHAR(11) UNIQUE NOT NULL COMMENT '手机号，唯一且非空',
    age TINYINT CHECK (age >= 0 AND age <= 150) COMMENT '年龄，0-150之间',
    gender ENUM('男', '女', '未知') DEFAULT '未知' COMMENT '性别，枚举类型，默认未知',
    create_time DATETIME DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间，默认当前时间',
    update_time TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间，默认当前时间，自动更新'
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='用户信息表';

-- 创建订单表（从表，含外键）
CREATE TABLE IF NOT EXISTS order_info (
    order_id INT PRIMARY KEY AUTO_INCREMENT COMMENT '订单ID，主键，自增',
    order_no VARCHAR(32) UNIQUE NOT NULL COMMENT '订单编号，唯一且非空',
    user_id INT NOT NULL COMMENT '下单用户ID',
    order_amount DECIMAL(10,2) NOT NULL CHECK (order_amount > 0) COMMENT '订单金额，大于0',
    order_status ENUM('待支付', '已支付', '已发货', '已收货', '已取消') DEFAULT '待支付' COMMENT '订单状态',
    create_time DATETIME DEFAULT CURRENT_TIMESTAMP COMMENT '下单时间',
    -- 外键约束：关联用户表的user_id
    FOREIGN KEY (user_id) REFERENCES user_info(user_id)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='订单信息表';
```

> 说明：`ENGINE=InnoDB`指定存储引擎为 InnoDB（MySQL 默认，支持事务、外键、行级锁），是推荐的存储引擎。

## 四、表的操作之修改（ALTER TABLE）

### 4.1 字段操作

表格

|操作|语法|示例|
|:--|:--|:--|
|**添加字段**|`ALTER TABLE 表名 ADD COLUMN 字段名 数据类型 [约束] [FIRST/AFTER 已有字段];`|`ALTER TABLE user_info ADD COLUMN email VARCHAR(100) UNIQUE AFTER phone;`（在 phone 字段后添加 email 字段）|
|**修改字段数据类型 / 约束**|`ALTER TABLE 表名 MODIFY COLUMN 字段名 新数据类型 [新约束];`|`ALTER TABLE user_info MODIFY COLUMN age TINYINT NOT NULL;`（将 age 字段改为非空）|
|**修改字段名 / 数据类型**|`ALTER TABLE 表名 CHANGE COLUMN 旧字段名 新字段名 新数据类型 [新约束];`|`ALTER TABLE user_info CHANGE COLUMN gender sex ENUM('男', '女', '未知') DEFAULT '未知';`（将 gender 字段改名为 sex）|
|**删除字段**|`ALTER TABLE 表名 DROP COLUMN 字段名;`|`ALTER TABLE user_info DROP COLUMN email;`（删除 email 字段）|

### 4.2 约束操作

表格

|操作|语法|示例|
|:--|:--|:--|
|**添加主键约束**|`ALTER TABLE 表名 ADD PRIMARY KEY (字段名);`|`ALTER TABLE user_info ADD PRIMARY KEY (user_id);`|
|**删除主键约束**|`ALTER TABLE 表名 DROP PRIMARY KEY;`|`ALTER TABLE user_info DROP PRIMARY KEY;`（自增字段需先删除自增属性才能删除主键）|
|**添加唯一约束**|`ALTER TABLE 表名 ADD UNIQUE KEY 约束名 (字段名);`|`ALTER TABLE user_info ADD UNIQUE KEY uk_phone (phone);`|
|**删除唯一约束**|`ALTER TABLE 表名 DROP INDEX 约束名;`|`ALTER TABLE user_info DROP INDEX uk_phone;`|
|**添加外键约束**|`ALTER TABLE 表名 ADD FOREIGN KEY (字段名) REFERENCES 关联表名(关联字段名);`|`ALTER TABLE order_info ADD FOREIGN KEY (user_id) REFERENCES user_info(user_id);`|
|**删除外键约束**|`ALTER TABLE 表名 DROP FOREIGN KEY 外键约束名;`|`ALTER TABLE order_info DROP FOREIGN KEY order_info_ibfk_1;`（需先通过`SHOW CREATE TABLE order_info;`查看外键约束名）|

## 五、表的操作之删除与清空

### 核心区别（极易混淆，必记）

表格

|操作|关键字|作用|是否可恢复|执行速度|
|:--|:--|:--|:--|:--|
|**删除表**|`DROP TABLE`|删除整个表结构和所有数据|不可恢复|快|
|**清空表**|`TRUNCATE TABLE`|清空表中所有数据，但保留表结构|不可恢复|极快（相当于删除并重建表）|
|**删除数据**|`DELETE FROM`|删除表中部分或全部数据，保留表结构|可通过事务回滚（需在事务中执行）|慢（逐行删除）|

### 语法示例

sql

```
-- 删除表
DROP TABLE IF EXISTS order_info;

-- 清空表
TRUNCATE TABLE order_info;

-- 删除数据（不带WHERE条件则删除全部）
DELETE FROM order_info WHERE order_status = '已取消';
```

## 八、核心要点总结

1. DDL 操作直接修改数据库结构，**执行前务必确认**，建议先在测试环境练习；
2. 建库推荐用 `utf8mb4` 字符集，建表推荐用 `InnoDB` 存储引擎；
3. 主键是表的灵魂，建议用自增整数型主键；
4. 区分 `DROP`（删表）、`TRUNCATE`（清空）、`DELETE`（删数据）的区别；
5. 外键约束会降低性能，互联网场景下通常**不建议使用外键**，而是在应用层维护关联关系。
