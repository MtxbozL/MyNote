## 一、DQL 基础定义

**DQL（Data Query Language，数据查询语言）**：专门用于从数据库表中**查询和检索数据**的 SQL 语言，核心关键字为 `SELECT`，是日常开发中使用频率最高、场景最复杂的 SQL 类型。

本笔记聚焦**基础查询**，覆盖从单表简单查询到分组、关联的核心操作，基于之前创建的 `user_info`（用户表）和 `order_info`（订单表）演示。

---

## 二、基础查询（SELECT 入门）

### 2.1 核心语法结构

```sql
SELECT [DISTINCT] 字段1, 字段2, ... 
FROM 表名;
```

### 2.2 常见基础操作

| 操作类型       | 语法                                   | 示例                                                          | 说明                               |
| :--------- | :----------------------------------- | :---------------------------------------------------------- | :------------------------------- |
| **查询所有字段** | `SELECT * FROM 表名;`                  | `SELECT * FROM user_info;`                                  | `*` 表示所有字段，**生产环境不推荐**（效率低、可读性差） |
| **查询指定字段** | `SELECT 字段1, 字段2, ... FROM 表名;`      | `SELECT user_id, user_name, phone FROM user_info;`          | 只查询需要的字段，效率更高                    |
| **字段别名**   | `SELECT 字段 AS 别名 FROM 表名;`（`AS` 可省略） | `SELECT user_id AS '用户ID', user_name '用户名' FROM user_info;` | 让查询结果更易读，支持中文别名（需加引号）            |
| **查询结果去重** | `SELECT DISTINCT 字段 FROM 表名;`        | `SELECT DISTINCT gender FROM user_info;`                    | 去除重复的查询结果，仅保留唯一值                 |
| **字段计算**   | `SELECT 字段 运算符 数值 FROM 表名;`          | `SELECT user_name, age, age + 1 AS '明年年龄' FROM user_info;`  | 支持 `+`、`-`、`*`、`/` 等运算符          |

## 三、条件查询（WHERE 子句）

用于从表中**筛选符合条件的数据**，是查询的核心过滤手段。

### 3.1 条件类型与示例

| 条件类型     | 关键字 / 符号                                                     | 示例                                                         | 说明                                                                   |
| :------- | :----------------------------------------------------------- | :--------------------------------------------------------- | :------------------------------------------------------------------- |
| **比较条件** | `=`（等于）、`<>`/`!=`（不等于）、`>`（大于）、`<`（小于）、`>=`（大于等于）、`<=`（小于等于） | `SELECT * FROM user_info WHERE age > 25;`                  | 最基础的条件判断                                                             |
| **逻辑条件** | `AND`（且，多个条件同时满足）、`OR`（或，多个条件满足一个即可）、`NOT`（非，取反）             | `SELECT * FROM user_info WHERE age > 25 AND gender = '男';` | 可组合多个条件，`AND` 优先级高于 `OR`                                             |
| **范围条件** | `BETWEEN ... AND ...`（在范围内，包含边界）                             | `SELECT * FROM user_info WHERE age BETWEEN 20 AND 30;`     | 等价于 `age >= 20 AND age <= 30`                                        |
|          | `IN`（在指定值列表中）                                                | `SELECT * FROM user_info WHERE age IN (22, 25, 28);`       | 等价于 `age = 22 OR age = 25 OR age = 28`                               |
|          | `NOT IN`（不在指定值列表中）                                           | `SELECT * FROM user_info WHERE age NOT IN (22, 25, 28);`   | 排除指定值                                                                |
| **模糊查询** | `LIKE`（匹配字符串模式）                                              | `SELECT * FROM user_info WHERE user_name LIKE '张%';`       | `%` 匹配**任意多个字符**（包括 0 个），`_` 匹配**单个字符**（如 `'张_'` 匹配 “张三” 但不匹配 “张三四”） |
| **空值查询** | `IS NULL`（字段为空）                                              | `SELECT * FROM user_info WHERE email IS NULL;`             | **注意**：不能用 `= NULL`，必须用 `IS NULL`                                    |
|          | `IS NOT NULL`（字段不为空）                                         | `SELECT * FROM user_info WHERE email IS NOT NULL;`         | 排除空值数据                                                               |

## 四、排序查询（ORDER BY）

用于对查询结果**按指定字段排序**，默认升序。

### 4.1 核心语法与示例

```sql
-- 语法：ORDER BY 字段1 [ASC/DESC], 字段2 [ASC/DESC], ...
-- ASC：升序（默认，可省略）；DESC：降序

-- 1. 单字段升序（按年龄从小到大）
SELECT * FROM user_info ORDER BY age ASC;

-- 2. 单字段降序（按用户ID从大到小）
SELECT * FROM user_info ORDER BY user_id DESC;

-- 3. 多字段排序（先按年龄升序，年龄相同再按用户ID降序）
SELECT * FROM user_info ORDER BY age ASC, user_id DESC;
```

## 五、分页查询（LIMIT）

用于从查询结果中**截取指定范围的数据**，是分页功能的核心。

### 5.1 核心语法与示例

```sql
-- 语法：LIMIT 偏移量, 每页条数
-- 偏移量 = (当前页 - 1) * 每页条数

-- 示例1：查询第1页，每页10条数据（偏移量为0）
SELECT * FROM user_info ORDER BY user_id LIMIT 0, 10;

-- 示例2：查询第2页，每页10条数据（偏移量为10）
SELECT * FROM user_info ORDER BY user_id LIMIT 10, 10;

-- 示例3：查询前5条数据（可省略偏移量）
SELECT * FROM user_info ORDER BY user_id LIMIT 5;
```

## 六、聚合函数（统计查询）

用于对一组数据进行**统计计算**，返回单个结果，常与分组查询配合使用。

### 6.1 常用聚合函数

| 函数名       | 作用   | 示例                                                     | 说明                                            |
| :-------- | :--- | :----------------------------------------------------- | :-------------------------------------------- |
| `COUNT()` | 统计行数 | `SELECT COUNT(*) AS '总用户数' FROM user_info;`            | `COUNT(*)`：统计所有行（包括空值）；`COUNT(字段)`：统计该字段非空的行数 |
| `SUM()`   | 求和   | `SELECT SUM(order_amount) AS '总订单金额' FROM order_info;` | 仅对数值型字段有效                                     |
| `AVG()`   | 求平均值 | `SELECT AVG(age) AS '平均年龄' FROM user_info;`            | 仅对数值型字段有效，自动忽略空值                              |
| `MAX()`   | 求最大值 | `SELECT MAX(age) AS '最大年龄' FROM user_info;`            | 可对数值型、日期型字段有效                                 |
| `MIN()`   | 求最小值 | `SELECT MIN(age) AS '最小年龄' FROM user_info;`            | 可对数值型、日期型字段有效                                 |

## 七、分组查询（GROUP BY + HAVING）

用于将数据**按指定字段分组**，再对每组进行聚合统计。

### 7.1 核心语法与示例

```sql
-- 语法：SELECT 分组字段, 聚合函数 FROM 表名 GROUP BY 分组字段 [HAVING 分组后条件];

-- 1. 基础分组：按性别分组，统计每组的用户数和平均年龄
SELECT gender, COUNT(*) AS '用户数', AVG(age) AS '平均年龄' 
FROM user_info 
GROUP BY gender;

-- 2. HAVING 过滤分组：统计平均年龄大于25的性别分组
SELECT gender, COUNT(*) AS '用户数', AVG(age) AS '平均年龄' 
FROM user_info 
GROUP BY gender 
HAVING AVG(age) > 25;
```

### 7.2 关键区别：WHERE vs HAVING

| 对比维度       | WHERE            | HAVING                 |
| :--------- | :--------------- | :--------------------- |
| **作用阶段**   | 分组前（过滤行）         | 分组后（过滤分组）              |
| **可使用的内容** | 表中原始字段           | 分组字段、聚合函数结果            |
| **示例**     | `WHERE age > 25` | `HAVING AVG(age) > 25` |

## 八、基础关联查询

用于从**多张表**中联合查询数据，基于表之间的关联关系（如外键）。

### 8.1 常用关联类型

| 关联类型                | 语法                                         | 说明                                        | 示例                                                                                                                                |
| :------------------ | :----------------------------------------- | :---------------------------------------- | :-------------------------------------------------------------------------------------------------------------------------------- |
| **内连接（INNER JOIN）** | `SELECT 字段 FROM 表1 INNER JOIN 表2 ON 关联条件;` | 只返回两张表中**关联条件匹配**的数据                      | `SELECT u.user_name, o.order_no, o.order_amount FROM user_info u INNER JOIN order_info o ON u.user_id = o.user_id;`（`u`、`o` 是表别名） |
| **左连接（LEFT JOIN）**  | `SELECT 字段 FROM 表1 LEFT JOIN 表2 ON 关联条件;`  | 返回左表（表 1）的**所有数据**，右表（表 2）匹配不到的字段填 `NULL` | `SELECT u.user_name, o.order_no, o.order_amount FROM user_info u LEFT JOIN order_info o ON u.user_id = o.user_id;`（会返回没有订单的用户）    |

## 十、核心要点总结

1. **SELECT 执行顺序**：`FROM` → `WHERE` → `GROUP BY` → `HAVING` → `SELECT` → `ORDER BY` → `LIMIT`；
2. **条件查询注意**：空值用 `IS NULL`，模糊查询用 `LIKE`（`%` 匹配多字符，`_` 匹配单字符）；
3. **分页计算**：偏移量 = (当前页 - 1) × 每页条数；
4. **聚合函数区别**：`COUNT(*)` 统计所有行，`COUNT(字段)` 统计非空行；
5. **WHERE vs HAVING**：分组前过滤用 `WHERE`，分组后过滤用 `HAVING`；
6. **关联查询选择**：需要匹配数据用 `INNER JOIN`，需要左表所有数据用 `LEFT JOIN`。
