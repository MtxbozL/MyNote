## 一、DML 基础定义

**DML（Data Manipulation Language，数据操作语言）**：用于对数据库表中的数据进行**增（INSERT）、删（DELETE）、改（UPDATE）、查（SELECT）** 操作的 SQL 语言，是日常开发中使用频率最高的 SQL 类型。

---

## 二、数据插入（INSERT）

### 2.1 核心语法与示例

基于之前创建的 `user_info` 表，演示插入操作：

表格

|操作类型|语法|示例|说明|
|:--|:--|:--|:--|
|**插入单条完整数据**|`INSERT INTO 表名 VALUES (值1, 值2, ...);`|`INSERT INTO user_info VALUES (1, '张三', '13800138000', 25, '男', '2026-03-09 10:00:00', '2026-03-09 10:00:00');`|需按表字段顺序赋值，自增主键可填 `NULL` 或 `DEFAULT` 让其自动生成|
|**插入单条指定字段数据**|`INSERT INTO 表名 (字段1, 字段2, ...) VALUES (值1, 值2, ...);`|`INSERT INTO user_info (user_name, phone, age) VALUES ('李四', '13800138001', 28);`|未指定的字段会使用默认值（如 `create_time` 自动填当前时间，`gender` 默认为 `未知`）|
|**插入多条数据**|`INSERT INTO 表名 (字段1, 字段2, ...) VALUES (值1, 值2, ...), (值1, 值2, ...), ...;`|`INSERT INTO user_info (user_name, phone, age) VALUES ('王五', '13800138002', 22), ('赵六', '13800138003', 30);`|一次插入多条，效率高于多次单条插入|
|**插入查询结果**|`INSERT INTO 表名 (字段1, 字段2, ...) SELECT 字段1, 字段2, ... FROM 另一表名 [WHERE 条件];`|`INSERT INTO user_info_backup (user_name, phone) SELECT user_name, phone FROM user_info WHERE age > 25;`|将一张表的查询结果插入到另一张表（需先创建 `user_info_backup` 表）|

### 2.2 关键注意事项

- 自增主键（`AUTO_INCREMENT`）：插入时可省略该字段，或填 `NULL`/`DEFAULT`，数据库会自动生成唯一递增的值；
- 字段与值的对应：指定字段插入时，值的顺序必须与字段顺序一致；
- 约束冲突：插入数据违反唯一约束、非空约束、检查约束时会报错，需确保数据符合规则。

## 三、数据更新（UPDATE）

### 3.1 核心语法与示例

sql

```
-- 基础语法
UPDATE 表名 SET 字段1 = 值1, 字段2 = 值2, ... [WHERE 条件];

-- 示例1：更新单条数据的单个字段（必须加 WHERE 条件！）
UPDATE user_info SET age = 26 WHERE user_id = 1;

-- 示例2：更新单条数据的多个字段
UPDATE user_info SET user_name = '张三三', gender = '男' WHERE user_id = 1;

-- 示例3：更新多条数据（符合条件的都会被更新）
UPDATE user_info SET age = age + 1 WHERE age < 25;
```

### 3.2 关键注意事项（重中之重！）

- **必须加 WHERE 条件**：否则会更新**全表所有数据**，造成不可逆的严重后果；
- 执行前建议先测试：用 `SELECT` 语句验证 `WHERE` 条件是否正确，确保只更新目标数据；
- 事务保护：重要更新操作建议在事务中执行（`START TRANSACTION;` → 执行更新 → 确认无误后 `COMMIT;`，有误则 `ROLLBACK;`）。

## 四、数据删除（DELETE）

### 4.1 核心语法与示例

sql

```
-- 基础语法
DELETE FROM 表名 [WHERE 条件];

-- 示例1：删除单条数据（必须加 WHERE 条件！）
DELETE FROM user_info WHERE user_id = 4;

-- 示例2：删除多条数据（符合条件的都会被删除）
DELETE FROM user_info WHERE age > 30;
```

### 4.2 关键注意事项（与 UPDATE 同等重要！）

- **必须加 WHERE 条件**：否则会删除**全表所有数据**，造成不可逆的严重后果；
- 与 `TRUNCATE` 的区别：
    
    - `DELETE`：逐行删除，可加 `WHERE` 条件，支持事务回滚，速度慢；
    - `TRUNCATE`：直接清空表并重建，不可加 `WHERE` 条件，不可回滚，速度极快；
    
- 外键约束影响：若表有外键关联，需先删除从表数据或解除外键约束，否则主表数据无法删除。

## 五、数据查询（SELECT）—— 核心中的核心

`SELECT` 是 DML 中最复杂、使用频率最高的操作，以下是完整的查询语法结构（按执行顺序排列）：

sql

```
SELECT [DISTINCT] 字段1, 字段2, ...  -- 5. 选择字段，去重
FROM 表1  -- 1. 确定数据源
[JOIN 表2 ON 关联条件]  -- 2. 关联表
[WHERE 条件]  -- 3. 过滤行
[GROUP BY 分组字段]  -- 4. 分组
[HAVING 分组后条件]  -- 6. 过滤分组
[ORDER BY 排序字段 ASC/DESC]  -- 7. 排序
[LIMIT 偏移量, 条数];  -- 8. 分页
```

### 5.1 基础查询

sql

```
-- 1. 查询所有字段（不推荐生产环境使用，效率低）
SELECT * FROM user_info;

-- 2. 查询指定字段
SELECT user_id, user_name, phone FROM user_info;

-- 3. 字段别名（AS 可省略）
SELECT user_id AS '用户ID', user_name '用户名', phone '手机号' FROM user_info;

-- 4. 查询结果去重（DISTINCT）
SELECT DISTINCT gender FROM user_info;

-- 5. 字段计算（如年龄+1）
SELECT user_name, age, age + 1 AS '明年年龄' FROM user_info;
```

### 5.2 条件查询（WHERE）

表格

|条件类型|关键字 / 符号|示例|说明|
|:--|:--|:--|:--|
|**比较条件**|`=`, `<>`, `!=`, `>`, `<`, `>=`, `<=`|`SELECT * FROM user_info WHERE age > 25;`|`<>` 和 `!=` 都表示 “不等于”|
|**范围条件**|`BETWEEN ... AND ...`|`SELECT * FROM user_info WHERE age BETWEEN 20 AND 30;`|包含边界值（20 ≤ age ≤ 30）|
||`IN`|`SELECT * FROM user_info WHERE age IN (22, 25, 28);`|匹配指定值列表|
||`NOT IN`|`SELECT * FROM user_info WHERE age NOT IN (22, 25, 28);`|排除指定值列表|
|**模糊查询**|`LIKE`|`SELECT * FROM user_info WHERE user_name LIKE '张%';`|`%` 匹配任意多个字符，`_` 匹配单个字符（如 `'张_'` 匹配 “张三” 但不匹配 “张三四”）|
|**空值查询**|`IS NULL`|`SELECT * FROM user_info WHERE email IS NULL;`|查询字段为 NULL 的数据（注意不能用 `= NULL`）|
||`IS NOT NULL`|`SELECT * FROM user_info WHERE email IS NOT NULL;`|查询字段不为 NULL 的数据|
|**逻辑条件**|`AND`|`SELECT * FROM user_info WHERE age > 25 AND gender = '男';`|多个条件同时满足|
||`OR`|`SELECT * FROM user_info WHERE age < 22 OR age > 30;`|多个条件满足一个即可|
||`NOT`|`SELECT * FROM user_info WHERE NOT age > 25;`|取反|

### 5.3 排序查询（ORDER BY）

sql

```
-- 1. 单字段排序（ASC 升序，默认；DESC 降序）
SELECT * FROM user_info ORDER BY age ASC;

-- 2. 多字段排序（先按 age 升序，age 相同再按 user_id 降序）
SELECT * FROM user_info ORDER BY age ASC, user_id DESC;
```

### 5.4 聚合函数（统计查询）

聚合函数用于对一组数据进行统计计算，返回单个结果：

表格

|函数名|作用|示例|
|:--|:--|:--|
|`COUNT()`|统计行数|`SELECT COUNT(*) AS '总用户数' FROM user_info;`（`COUNT(*)` 统计所有行，`COUNT(字段)` 统计非 NULL 行）|
|`SUM()`|求和|`SELECT SUM(age) AS '年龄总和' FROM user_info;`|
|`AVG()`|求平均值|`SELECT AVG(age) AS '平均年龄' FROM user_info;`|
|`MAX()`|求最大值|`SELECT MAX(age) AS '最大年龄' FROM user_info;`|
|`MIN()`|求最小值|`SELECT MIN(age) AS '最小年龄' FROM user_info;`|

### 5.5 分组查询（GROUP BY + HAVING）

分组查询用于将数据按指定字段分组，再对每组进行聚合统计：

sql

```
-- 1. 基础分组：按性别分组，统计每组的用户数和平均年龄
SELECT gender, COUNT(*) AS '用户数', AVG(age) AS '平均年龄' 
FROM user_info 
GROUP BY gender;

-- 2. HAVING 过滤分组：统计平均年龄大于25的性别分组
SELECT gender, COUNT(*) AS '用户数', AVG(age) AS '平均年龄' 
FROM user_info 
GROUP BY gender 
HAVING AVG(age) > 25;
```

> 关键区别：`WHERE` 过滤行（分组前），`HAVING` 过滤分组（分组后）。

### 5.6 关联查询（JOIN）

关联查询用于从多张表中获取数据，基于 `user_info`（主表）和 `order_info`（从表）演示：

表格

|关联类型|语法|说明|示例|
|:--|:--|:--|:--|
|**内连接（INNER JOIN）**|`SELECT 字段 FROM 表1 INNER JOIN 表2 ON 关联条件;`|只返回两张表中**关联条件匹配**的数据|`SELECT u.user_name, o.order_no, o.order_amount FROM user_info u INNER JOIN order_info o ON u.user_id = o.user_id;`（`u` 和 `o` 是表别名）|
|**左连接（LEFT JOIN）**|`SELECT 字段 FROM 表1 LEFT JOIN 表2 ON 关联条件;`|返回左表（表 1）的**所有数据**，右表（表 2）匹配不到的字段填 NULL|`SELECT u.user_name, o.order_no, o.order_amount FROM user_info u LEFT JOIN order_info o ON u.user_id = o.user_id;`（会返回没有订单的用户）|
|**右连接（RIGHT JOIN）**|`SELECT 字段 FROM 表1 RIGHT JOIN 表2 ON 关联条件;`|返回右表（表 2）的**所有数据**，左表（表 1）匹配不到的字段填 NULL|`SELECT u.user_name, o.order_no, o.order_amount FROM user_info u RIGHT JOIN order_info o ON u.user_id = o.user_id;`（会返回没有对应主表用户的异常订单）|

### 5.7 子查询

子查询是嵌套在另一个 SQL 语句中的查询，用于先获取中间结果：

sql

```
-- 1. WHERE 子查询：查询年龄大于平均年龄的用户
SELECT * FROM user_info WHERE age > (SELECT AVG(age) FROM user_info);

-- 2. FROM 子查询：将子查询结果作为临时表
SELECT temp.gender, temp.avg_age 
FROM (SELECT gender, AVG(age) AS avg_age FROM user_info GROUP BY gender) temp 
WHERE temp.avg_age > 25;
```

### 5.8 分页查询（LIMIT）

sql

```
-- 语法：LIMIT 偏移量, 每页条数（偏移量 = (当前页 - 1) * 每页条数）
-- 示例：查询第2页，每页10条数据
SELECT * FROM user_info ORDER BY user_id LIMIT 10, 10;
```

## 七、核心要点总结

1. **增删改操作的铁律**：`UPDATE` 和 `DELETE` 必须加 `WHERE` 条件，执行前建议用 `SELECT` 验证；
2. **SELECT 执行顺序**：`FROM` → `JOIN` → `WHERE` → `GROUP BY` → `HAVING` → `SELECT` → `ORDER BY` → `LIMIT`；
3. **关联查询的选择**：需要匹配数据用 `INNER JOIN`，需要左表所有数据用 `LEFT JOIN`；
4. **NULL 值的处理**：查询 NULL 用 `IS NULL`，不能用 `= NULL`；
5. **聚合函数的使用**：`COUNT(*)` 统计所有行，`COUNT(字段)` 统计非 NULL 行。
