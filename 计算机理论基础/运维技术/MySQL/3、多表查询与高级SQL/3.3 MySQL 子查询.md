## 一、开篇：子查询的核心定义与作用

### 1. 什么是子查询

子查询（Subquery），又称嵌套查询，是指**嵌套在另一个 SQL 语句（SELECT/INSERT/UPDATE/DELETE）内部的查询语句**，用于先获取一个「中间结果集」，再供外层 SQL 使用。

简单来说：子查询就是「先查一步，把结果当条件 / 临时表用」，是解决复杂查询需求的核心手段。

### 2. 为什么需要子查询

很多复杂需求无法通过单表查询或简单的连接查询完成，比如：

- 需求 1：查询「年龄大于平均年龄」的用户（需要先算出平均年龄，再筛选）；
- 需求 2：查询「有订单的用户」（需要先查出所有下单的用户 ID，再匹配用户表）；
- 需求 3：查询「每个用户的最新订单」（需要先按用户分组查出最新订单时间，再关联订单表）。

这些需求的核心特点是：**需要先获取一个中间结果，再基于这个结果做进一步查询**，子查询就是为了解决这类问题而生的。

---

## 二、子查询的分类

子查询有两种常用的分类方式，分别对应不同的使用场景：

|分类维度|分类结果|核心说明|
|:--|:--|:--|
|**按子查询的位置分**|WHERE 子查询|子查询写在 WHERE 子句中，作为过滤条件|
||FROM 子查询|子查询写在 FROM 子句中，作为临时表使用|
||SELECT 子查询|子查询写在 SELECT 字段列表中，作为计算字段|
||HAVING 子查询|子查询写在 HAVING 子句中，作为分组后的过滤条件|
|**按子查询的结果集分**|标量子查询（Scalar Subquery）|返回**单个值**（一行一列），如平均年龄、最大订单金额|
||列子查询（Column Subquery）|返回**一列值**（多行一列），如所有下单的用户 ID|
||行子查询（Row Subquery）|返回**一行值**（一行多列），如某个用户的年龄 + 性别|
||表子查询（Table Subquery）|返回**多行多列**（一张临时表），如按用户分组的订单统计结果|

## 三、核心子查询类型详解（附示例）

基于之前的 `user_info`（用户表）和 `order_info`（订单表）演示所有核心子查询类型。

### 3.1 标量子查询（最常用）

#### 核心定义

返回**单个值**（一行一列）的子查询，通常配合 `=`、`>`、`<`、`>=`、`<=` 等比较运算符使用。

#### 常见位置

WHERE 子句、SELECT 字段列表、HAVING 子句。

#### 示例 1：WHERE 子句中的标量子查询

需求：查询「年龄大于平均年龄」的用户

```sql
-- 先子查询算出平均年龄（返回单个值），再外层筛选年龄大于该值的用户
SELECT user_id, user_name, age
FROM user_info
WHERE age > (SELECT AVG(age) FROM user_info);
```

#### 示例 2：SELECT 字段列表中的标量子查询

需求：查询所有用户，以及他们的「订单总金额」（没有订单的显示 NULL）

```sql
SELECT 
  u.user_id, u.user_name,
  -- 子查询：对每个用户，查出其订单总金额（返回单个值）
  (SELECT SUM(order_amount) FROM order_info o WHERE o.user_id = u.user_id) AS '总订单金额'
FROM user_info u;
```

> 注意：这是一个「关联子查询」（子查询依赖外层查询的 `u.user_id`），会对每个用户执行一次子查询，数据量大时性能较差，可优先用左连接代替。

---

### 3.2 列子查询

#### 核心定义

返回**一列值**（多行一列）的子查询，通常配合 `IN`、`NOT IN`、`ANY`、`ALL` 等关键字使用。

#### 常用关键字说明

|关键字|作用|示例|
|:--|:--|:--|
|`IN`|外层值在子查询结果集中存在|`WHERE user_id IN (子查询)`|
|`NOT IN`|外层值不在子查询结果集中|`WHERE user_id NOT IN (子查询)`|
|`ANY`|外层值满足子查询结果集中的**任意一个**|`WHERE age > ANY (子查询)`（大于任意一个即可）|
|`ALL`|外层值满足子查询结果集中的**所有**|`WHERE age > ALL (子查询)`（大于所有才行）|

#### 示例 1：IN 关键字的列子查询

需求：查询「有订单的用户」

```sql
-- 先子查询查出所有下单的用户ID（返回一列），再外层匹配用户表
SELECT user_id, user_name, phone
FROM user_info
WHERE user_id IN (SELECT DISTINCT user_id FROM order_info);
```

#### 示例 2：NOT IN 关键字的列子查询

需求：查询「没有订单的用户」

```sql
SELECT user_id, user_name, phone
FROM user_info
WHERE user_id NOT IN (SELECT DISTINCT user_id FROM order_info);
```

#### 示例 3：ALL 关键字的列子查询

需求：查询「年龄比所有 25 岁以下用户都大」的用户

```sql
SELECT user_id, user_name, age
FROM user_info
WHERE age > ALL (SELECT age FROM user_info WHERE age < 25);
```

---

### 3.3 行子查询

#### 核心定义

返回**一行值**（一行多列）的子查询，通常配合 `=`、`IN` 等关键字使用，需用括号包裹多个字段进行匹配。

#### 示例

需求：查询「和用户 ID 为 1 的用户，同年龄、同性别」的用户

```sql
-- 先子查询查出用户1的年龄+性别（返回一行两列），再外层匹配
SELECT user_id, user_name, age, gender
FROM user_info
WHERE (age, gender) = (SELECT age, gender FROM user_info WHERE user_id = 1);
```

---

### 3.4 表子查询（FROM 子查询）

#### 核心定义

返回**多行多列**（一张临时表）的子查询，必须写在 FROM 子句中，且**必须给临时表加别名**。

#### 适用场景

需要先对数据做分组 / 聚合统计，再基于统计结果做进一步查询。

#### 示例

需求：查询「每个用户的最新订单」（假设订单 ID 越大，下单时间越晚）

```sql
-- 第一步：子查询按用户分组，查出每个用户的最大订单ID（返回临时表）
-- 第二步：外层关联订单表，获取完整订单信息
SELECT 
  o.order_id, o.order_no, o.user_id, o.order_amount, o.create_time
FROM order_info o
INNER JOIN (
  -- 子查询：每个用户的最大订单ID
  SELECT user_id, MAX(order_id) AS max_order_id
  FROM order_info
  GROUP BY user_id
) temp ON o.user_id = temp.user_id AND o.order_id = temp.max_order_id;
```

> 说明：子查询 `temp` 就是一张临时表，包含每个用户的最大订单 ID，外层通过关联获取完整订单信息。

## 四、关联子查询 vs 非关联子查询

这是子查询的另一种重要分类，核心区别在于**子查询是否依赖外层查询的字段**。

### 4.1 非关联子查询

#### 核心定义

子查询**独立执行**，不依赖外层查询的任何字段，只执行一次，结果集固定。

#### 示例

```sql
-- 子查询不依赖外层，先执行一次算出平均年龄，再外层筛选
SELECT user_id, user_name, age
FROM user_info
WHERE age > (SELECT AVG(age) FROM user_info);
```

#### 特点

- 执行效率高，只执行一次；
- 逻辑清晰，容易理解。

---

### 4.2 关联子查询

#### 核心定义

子查询**依赖外层查询的字段**，会对外层查询的每一行数据，都执行一次子查询，性能通常较差。

#### 示例

```sql
-- 子查询依赖外层的 u.user_id，对每个用户都执行一次子查询
SELECT 
  u.user_id, u.user_name,
  (SELECT SUM(order_amount) FROM order_info o WHERE o.user_id = u.user_id) AS '总订单金额'
FROM user_info u;
```

#### 特点

- 执行效率低，外层有多少行，子查询就执行多少次；
- 数据量大时，性能会急剧下降；
- 很多场景可优先用「连接查询」代替，性能更好。

---

## 五、子查询 vs 连接查询：如何选型？

很多需求既可以用子查询，也可以用连接查询，以下是核心选型原则：

|对比维度|子查询|连接查询|
|:--|:--|:--|
|**逻辑清晰度**|高，适合分步处理复杂需求（先查 A，再查 B）|中，多表关联时逻辑较绕|
|**执行效率**|关联子查询效率低，非关联子查询效率高|通常更高，尤其是多表关联时|
|**适用场景**|1. 需要先获取聚合结果（如平均年龄、最大 ID）再筛选；<br><br>2. 逻辑上需要分步处理的需求|1. 需要从多张表获取字段；<br><br>2. 数据量大的关联查询；<br><br>3. 可代替关联子查询的场景|

#### 示例：子查询 vs 连接查询的等价转换

需求：查询「有订单的用户」

```sql
-- 写法1：子查询（逻辑清晰）
SELECT user_id, user_name
FROM user_info
WHERE user_id IN (SELECT DISTINCT user_id FROM order_info);

-- 写法2：内连接（性能通常更好）
SELECT DISTINCT u.user_id, u.user_name
FROM user_info u
INNER JOIN order_info o ON u.user_id = o.user_id;
```

## 六、子查询的常见坑与避坑指南

### 1. 标量子查询返回多个值

#### 错误示例

```sql
-- 错误：子查询返回了多个用户ID，但外层用了 = 比较
SELECT * FROM user_info WHERE user_id = (SELECT user_id FROM order_info);
```

#### 错误原因

标量子查询必须返回**单个值**，如果返回多个值，会报错 `Subquery returns more than 1 row`。

#### 解决方法

- 如果需要匹配多个值，改用 `IN` 关键字；
- 确保子查询逻辑正确，只返回单个值（如加 `LIMIT 1` 或用聚合函数）。

---

### 2. NOT IN 子查询中的 NULL 值

#### 错误示例

```sql
-- 需求：查询没有订单的用户
SELECT * FROM user_info 
WHERE user_id NOT IN (SELECT user_id FROM order_info);
```

#### 错误风险

如果子查询结果中包含 `NULL` 值，整个 `NOT IN` 条件会返回 `UNKNOWN`，导致**查询结果为空**（没有数据返回）。

#### 解决方法

- 确保子查询结果中没有 `NULL` 值（如加 `WHERE user_id IS NOT NULL`）；
- 优先用 `LEFT JOIN + IS NULL` 代替 `NOT IN`，更安全：

```sql
    -- 安全写法：左连接 + IS NULL
    SELECT u.* FROM user_info u
    LEFT JOIN order_info o ON u.user_id = o.user_id
    WHERE o.user_id IS NULL;
```

---

### 3. 关联子查询的性能问题

#### 问题

关联子查询会对外层每一行都执行一次，数据量大时性能极差。

#### 解决方法

- 优先用「连接查询」代替关联子查询；
- 给关联字段加索引，提升子查询的执行效率；
- 用「表子查询（FROM 子查询）」代替关联子查询，先聚合再关联。

## 七、综合实操演练

基于 `user_info` 和 `order_info` 表，完成以下需求：

1. 查询「订单金额大于所有订单平均金额」的订单；
2. 查询「每个用户的订单数」，并筛选出订单数大于平均订单数的用户；
3. 查询「年龄最大的用户」的信息；
4. 查询「每个分类下价格最高的商品」（需结合 `product` 和 `category` 表）。

### 参考答案

```sql
-- 1. 查询订单金额大于平均金额的订单
SELECT * FROM order_info
WHERE order_amount > (SELECT AVG(order_amount) FROM order_info);

-- 2. 筛选订单数大于平均订单数的用户
SELECT user_id, COUNT(*) AS '订单数'
FROM order_info
GROUP BY user_id
HAVING COUNT(*) > (SELECT AVG(order_count) FROM (
  SELECT COUNT(*) AS order_count FROM order_info GROUP BY user_id
) temp);

-- 3. 查询年龄最大的用户
SELECT * FROM user_info
WHERE age = (SELECT MAX(age) FROM user_info);

-- 4. 查询每个分类下价格最高的商品
SELECT p.* FROM product p
INNER JOIN (
  SELECT category_id, MAX(price) AS max_price FROM product GROUP BY category_id
) temp ON p.category_id = temp.category_id AND p.price = temp.max_price;
```

## 八、核心要点总结

1. 子查询的核心是**嵌套在 SQL 内部的查询**，用于先获取中间结果，再供外层使用；
2. 按结果集分类：标量子查询（单值）、列子查询（单列）、行子查询（单行）、表子查询（多行列）；
3. 按位置分类：WHERE、FROM、SELECT、HAVING 子查询，其中 FROM 子查询必须加别名；
4. 关联子查询依赖外层字段，性能差，优先用连接查询代替；
5. 常见坑：标量子查询返回多值、NOT IN 中的 NULL 值、关联子查询性能问题；
6. 选型原则：逻辑分步用子查询，多表关联用连接查询，性能优先选连接查询。
