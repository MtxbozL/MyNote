## 一、开篇：存储过程与函数的核心定义与作用

### 1. 什么是存储过程与函数

存储过程（Stored Procedure）和存储函数（Stored Function）是**存储在 MySQL 服务器端的一组预编译 SQL 语句集合**，封装了复杂的业务逻辑，可通过名称调用执行，类似编程语言中的「函数」或「方法」。

简单对比：

- **普通 SQL**：单条语句，每次执行都要解析、编译，逻辑分散在应用代码中；
- **存储过程 / 函数**：预编译的 SQL 集合，逻辑封装在数据库端，一次编译多次执行。

### 2. 为什么要用存储过程与函数

|优势|核心说明|
|:--|:--|
|**性能提升**|预编译后存储在服务器，无需每次解析编译，执行效率更高|
|**代码复用**|复杂业务逻辑封装后，可被多个应用 / 多个场景调用，避免重复开发|
|**减少网络传输**|只需传递调用命令和参数，无需传输多条 SQL 语句，降低网络开销|
|**安全性提升**|可通过权限控制，只允许用户调用存储过程，不直接操作表，提升数据安全|
|**业务逻辑封装**|将复杂的事务、流程控制、异常处理封装在数据库端，保证业务逻辑的一致性|

### 3. 存储过程 vs 存储函数的核心区别（重中之重）

这是新手最容易混淆的知识点，先明确核心差异：

|对比维度|存储过程（Stored Procedure）|存储函数（Stored Function）|
|:--|:--|:--|
|**返回值**|无直接返回值，可通过 OUT/INOUT 参数返回多个结果|必须有且仅有一个返回值（RETURN 语句）|
|**参数类型**|支持 IN（输入）、OUT（输出）、INOUT（输入输出）三种参数|仅支持 IN（输入）参数，无 OUT/INOUT|
|**调用方式**|用 `CALL` 关键字调用：`CALL 存储过程名();`|用在 SQL 语句中，像普通函数一样调用：`SELECT 函数名();`|
|**是否可在 SQL 中使用**|不能直接用在 SELECT/UPDATE 等 SQL 语句中|可以直接用在 SELECT/UPDATE/INSERT 等 SQL 语句中|
|**结果集**|可以返回多个结果集（多个 SELECT）|不能返回结果集，只能返回单个值|
|**核心用途**|处理复杂业务逻辑、批量操作、事务控制|计算单个值、数据转换、简单逻辑处理|

## 二、前置必备知识（衔接流程控制笔记）

在学习存储过程与函数前，必须先掌握以下内容（详细可参考之前的「流程控制」笔记）：

### 2.1 分隔符 DELIMITER 修改

MySQL 默认以分号 `;` 作为语句结束符，而存储过程内部的 SQL 语句也会用到分号，会导致 MySQL 提前解析结束。因此**创建存储过程 / 函数前，必须先修改分隔符**，创建完成后再改回分号。

```SQL
-- 1. 修改分隔符为 //（也可自定义为 $$ 等）
DELIMITER //

-- 2. 中间写存储过程/函数创建语句，内部用 ; 结尾
CREATE PROCEDURE 存储过程名()
BEGIN
  -- 内部 SQL 语句，用 ; 结尾
  SELECT * FROM user_info;
END //

-- 3. 改回默认分隔符 ;
DELIMITER ;
```

### 2.2 BEGIN...END 代码块

用于包裹一组 SQL 语句，形成一个逻辑代码块，类似编程语言中的 `{}`，存储过程 / 函数的主体逻辑必须放在 `BEGIN...END` 中。

### 2.3 变量的三大类型

存储过程 / 函数中会用到三种变量，需明确区分：

|变量类型|声明方式|作用域|示例|
|:--|:--|:--|:--|
|**局部变量**|`DECLARE 变量名 数据类型 [DEFAULT 默认值];`|仅在当前 `BEGIN...END` 块中有效，必须写在块的最开头|`DECLARE user_count INT DEFAULT 0;`|
|**用户变量**|直接用 `@变量名` 赋值，无需 DECLARE|当前会话（连接）有效|`SET @total = 100;`|
|**系统变量**|用 `@@变量名` 访问|全局或会话级，MySQL 内置|`SELECT @@version;`（查看 MySQL 版本）|

## 三、第一部分：存储过程（Stored Procedure）

### 3.1 创建存储过程的核心语法

```SQL
DELIMITER //
CREATE PROCEDURE 存储过程名(
  -- 参数列表：可选，多个参数用逗号分隔
  [IN/OUT/INOUT] 参数名1 数据类型,
  [IN/OUT/INOUT] 参数名2 数据类型,
  ...
)
[COMMENT '存储过程的注释说明']
BEGIN
  -- 1. 声明局部变量（必须写在最开头）
  DECLARE 局部变量名 数据类型 [DEFAULT 默认值];
  
  -- 2. 主体逻辑：SQL 语句、流程控制（IF/WHILE/CASE）、异常处理
  -- ...
  
END //
DELIMITER ;
```

### 3.2 参数的三大类型（重点）

存储过程支持三种参数类型，分别用于输入、输出、输入输出，是存储过程的核心能力。

#### 3.2.1 IN 参数：输入参数（默认类型）

- **作用**：接收调用者传入的值，仅用于输入，参数值在存储过程内部修改后，不会影响调用者的变量。
- **语法**：`IN 参数名 数据类型`（`IN` 可省略，默认就是 IN）。
- **示例**：根据用户 ID 查询用户信息

```SQL
DELIMITER //
CREATE PROCEDURE get_user_by_id(IN in_user_id INT)
BEGIN
  SELECT user_id, user_name, phone, age 
  FROM user_info 
  WHERE user_id = in_user_id;
END //
DELIMITER ;

-- 调用：传入用户 ID 1
CALL get_user_by_id(1);
```

#### 3.2.2 OUT 参数：输出参数

- **作用**：用于向调用者返回值，存储过程内部可修改该参数值，修改后会传递给调用者的变量。
- **语法**：`OUT 参数名 数据类型`。
- **示例**：根据用户 ID，返回该用户的总订单金额

```SQL
DELIMITER //
CREATE PROCEDURE get_user_order_total(
  IN in_user_id INT,
  OUT out_total DECIMAL(10,2)
)
BEGIN
  -- 查询总订单金额，赋值给 OUT 参数
  SELECT IFNULL(SUM(order_amount), 0) INTO out_total 
  FROM order_info 
  WHERE user_id = in_user_id;
END //
DELIMITER ;

-- 调用：先声明用户变量接收结果
SET @total = 0;
CALL get_user_order_total(1, @total);
-- 查看返回结果
SELECT @total AS 用户总订单金额;
```

#### 3.2.3 INOUT 参数：输入输出参数

- **作用**：兼具 IN 和 OUT 的功能，既接收调用者传入的值，又可在内部修改后返回给调用者。
- **语法**：`INOUT 参数名 数据类型`。
- **示例**：传入一个金额，给金额打 8 折后返回

```SQL
DELIMITER //
CREATE PROCEDURE apply_discount(INOUT inout_amount DECIMAL(10,2))
BEGIN
  -- 打 8 折
  SET inout_amount = inout_amount * 0.8;
END //
DELIMITER ;

-- 调用：先赋值，再传入
SET @amount = 100;
CALL apply_discount(@amount);
-- 查看打折后的结果
SELECT @amount AS 打折后金额;
```

### 3.3 存储过程的查看、修改、删除

#### 3.3.1 查看存储过程

```SQL
-- 1. 查看所有存储过程
SHOW PROCEDURE STATUS;

-- 2. 查看指定数据库的存储过程
SHOW PROCEDURE STATUS WHERE Db = '数据库名';

-- 3. 查看存储过程的创建语句（核心，可看完整代码）
SHOW CREATE PROCEDURE 存储过程名;
```

#### 3.3.2 修改存储过程

MySQL 不支持直接修改存储过程的逻辑，只能先删除再重新创建。

```SQL
-- 先删除
DROP PROCEDURE IF EXISTS 存储过程名;
-- 再重新创建
DELIMITER //
CREATE PROCEDURE 存储过程名()
BEGIN
  -- 新的逻辑
END //
DELIMITER ;
```

#### 3.3.3 删除存储过程

```SQL
DROP PROCEDURE IF EXISTS 存储过程名;
```

### 3.4 存储过程实战案例：批量插入测试数据

整合流程控制（WHILE 循环）、变量、字符串函数，实现批量插入指定数量的测试用户数据。

```SQL
DELIMITER //
CREATE PROCEDURE batch_insert_test_user(IN insert_count INT)
BEGIN
  -- 声明局部变量
  DECLARE i INT DEFAULT 1;
  DECLARE user_name VARCHAR(50);
  DECLARE phone VARCHAR(11);
  DECLARE age INT;
  
  -- 循环插入
  insert_loop: WHILE i <= insert_count DO
    -- 生成测试数据
    SET user_name = CONCAT('测试用户', i);
    SET phone = CONCAT('138', LPAD(i, 8, '0')); -- LPAD 补零，生成 11 位手机号
    SET age = FLOOR(RAND() * 50 + 18); -- 生成 18-67 岁的随机年龄
    
    -- 插入数据
    INSERT INTO user_info (user_name, phone, age) 
    VALUES (user_name, phone, age);
    
    -- 计数器 +1
    SET i = i + 1;
  END WHILE insert_loop;
  
  -- 返回结果
  SELECT CONCAT('成功插入 ', insert_count, ' 条测试用户数据') AS result;
END //
DELIMITER ;

-- 调用：插入 100 条测试数据
CALL batch_insert_test_user(100);
```

## 四、第二部分：存储函数（Stored Function）

### 4.1 创建存储函数的核心语法

```SQL
DELIMITER //
CREATE FUNCTION 函数名(
  -- 参数列表：仅支持 IN 参数，多个参数用逗号分隔
  参数名1 数据类型,
  参数名2 数据类型,
  ...
)
RETURNS 返回值数据类型 -- 必须指定返回值类型
[COMMENT '函数的注释说明']
[DETERMINISTIC] -- 可选：表示相同输入总是返回相同输出，优化性能
BEGIN
  -- 1. 声明局部变量
  DECLARE 局部变量名 数据类型;
  
  -- 2. 主体逻辑
  
  -- 3. 必须有 RETURN 语句，返回单个值
  RETURN 返回值;
END //
DELIMITER ;
```

### 4.2 存储函数的核心要求

1. **必须有 `RETURNS` 子句**：指定返回值的数据类型；
2. **必须有 `RETURN` 语句**：在函数体内返回单个值，且返回值类型必须与 `RETURNS` 子句一致；
3. **仅支持 IN 参数**：无 OUT/INOUT 参数；
4. **不能返回结果集**：不能有 `SELECT` 语句（除非是 `SELECT ... INTO` 赋值给变量）。

### 4.3 存储函数实战案例

#### 案例 1：根据用户总订单金额，计算用户等级

```SQL
DELIMITER //
CREATE FUNCTION get_user_level(user_id INT)
RETURNS VARCHAR(20)
DETERMINISTIC
BEGIN
  DECLARE order_total DECIMAL(10,2) DEFAULT 0;
  DECLARE user_level VARCHAR(20);
  
  -- 查询用户总订单金额
  SELECT IFNULL(SUM(order_amount), 0) INTO order_total 
  FROM order_info 
  WHERE order_info.user_id = user_id;
  
  -- 分支判断等级
  CASE
    WHEN order_total >= 1000 THEN SET user_level = 'VIP用户';
    WHEN order_total >= 500 THEN SET user_level = '高级用户';
    WHEN order_total > 0 THEN SET user_level = '普通用户';
    ELSE SET user_level = '新用户';
  END CASE;
  
  -- 返回等级
  RETURN user_level;
END //
DELIMITER ;

-- 调用：在 SELECT 中直接使用，像普通函数一样
SELECT user_id, user_name, get_user_level(user_id) AS 用户等级 FROM user_info;
```

#### 案例 2：计算订单折扣（金额 > 100 打 8 折，> 50 打 9 折，否则不打折）

```SQL
DELIMITER //
CREATE FUNCTION calc_order_discount(order_amount DECIMAL(10,2))
RETURNS DECIMAL(10,2)
DETERMINISTIC
BEGIN
  DECLARE discount_amount DECIMAL(10,2);
  
  IF order_amount > 100 THEN
    SET discount_amount = order_amount * 0.8;
  ELSEIF order_amount > 50 THEN
    SET discount_amount = order_amount * 0.9;
  ELSE
    SET discount_amount = order_amount;
  END IF;
  
  RETURN discount_amount;
END //
DELIMITER ;

-- 调用：在查询订单时直接计算折扣后金额
SELECT order_id, order_amount, calc_order_discount(order_amount) AS 折扣后金额 FROM order_info;
```

```SQL
-- 2. 查看指定数据库的存储函数

-- 3. 查看存储函数的创建语句

```

```SQL

```

```SQL

```

## 五、存储过程 vs 存储函数：详细对比总结

| 对比维度        | 存储过程                                        | 存储函数                            |
| :---------- | :------------------------------------------ | :------------------------------ |
| **返回值**     | 无直接返回值，通过 OUT/INOUT 参数返回多个结果                | 必须有且仅有一个 RETURN 返回值             |
| **参数类型**    | IN、OUT、INOUT                                | 仅 IN                            |
| **调用方式**    | `CALL 存储过程名();`                             | `SELECT 函数名();` 或直接用在 SQL 中     |
| **结果集**     | 可返回多个结果集                                    | 不可返回结果集                         |
| **SQL 中使用** | 不能直接用在 SELECT/UPDATE 等语句中                   | 可直接用在 SELECT/UPDATE/INSERT 等语句中 |
| **事务控制**    | 可在内部使用事务（START TRANSACTION/COMMIT/ROLLBACK） | 不建议在内部使用事务                      |
| **核心用途**    | 复杂业务逻辑、批量操作、事务处理、多结果返回                      | 单个值计算、数据转换、简单逻辑、SQL 中嵌入使用       |

## 六、常见坑与避坑指南

### 1. 分隔符忘记修改或改回

- **坑**：创建存储过程时未修改 DELIMITER，导致内部分号被提前解析，报错；或创建完成后忘记改回分号，后续 SQL 无法执行。
- **避坑**：严格遵循「修改分隔符 → 创建 → 改回分号」的流程，创建完成后立即执行一条简单 SQL（如 `SELECT 1;`）验证。

### 2. 局部变量声明位置错误

- **坑**：在 `BEGIN...END` 块的中间声明局部变量，报错 `Variable declaration must come first`。
- **避坑**：`DECLARE` 声明局部变量必须写在 `BEGIN...END` 块的**最开头**，所有 SQL 语句、流程控制之前。

### 3. 局部变量与用户变量搞混

- **坑**：把局部变量写成用户变量（加 `@`），或反之，导致变量作用域错误、值未生效。
- **避坑**：
    
    - 仅在当前块用的变量：用 `DECLARE` 声明局部变量（无 `@`）；
    - 需在会话中传递的变量：用用户变量（加 `@`）。
    

### 4. 存储函数中使用 SELECT 返回结果集

- **坑**：在存储函数中写 `SELECT * FROM 表名;`，报错 `Not allowed to return a result set from a function`。
- **避坑**：存储函数只能返回单个值，不能返回结果集，查询数据需用 `SELECT ... INTO` 赋值给变量。

### 5. 存储函数中未处理 NOT FOUND 异常

- **坑**：在存储函数中用 `SELECT ... INTO` 赋值，但查询无结果时，会触发 `NOT FOUND` 异常，导致函数执行中断。
- **避坑**：在函数中声明 `DECLARE CONTINUE HANDLER FOR NOT FOUND` 处理异常，或用 `IFNULL()` 兜底。

### 6. 过度使用存储过程 / 函数

- **坑**：把所有业务逻辑都封装在存储过程中，导致业务逻辑耦合在数据库端，难以维护、迭代、测试，且数据库性能压力大。
- **避坑**：
    
    - 简单逻辑：放在应用代码中；
    - 复杂批量操作、事务控制、性能敏感的高频逻辑：可考虑用存储过程；
    - 单个值计算、SQL 中嵌入的简单逻辑：可考虑用存储函数。
    

### 7. 权限问题

- **坑**：创建存储过程 / 函数时报错 `Access denied; you need (at least one of) the CREATE ROUTINE privilege(s)`。
- **避坑**：确保当前用户有 `CREATE ROUTINE`（创建）、`ALTER ROUTINE`（修改 / 删除）、`EXECUTE`（执行）权限。

## 七、综合实战案例：订单处理存储过程

整合事务、流程控制、异常处理、参数，实现一个完整的订单处理逻辑：创建订单、扣减库存、更新用户等级。

```SQL
-- 先准备库存表
CREATE TABLE IF NOT EXISTS product_stock (
    product_id INT PRIMARY KEY,
    product_name VARCHAR(100) NOT NULL,
    stock INT NOT NULL DEFAULT 0 COMMENT '库存数量'
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;

-- 插入测试库存数据
INSERT INTO product_stock (product_id, product_name, stock) VALUES 
(1, '商品A', 100),
(2, '商品B', 50);

DELIMITER //
CREATE PROCEDURE create_order(
  IN in_user_id INT,
  IN in_product_id INT,
  IN in_product_num INT,
  IN in_order_amount DECIMAL(10,2),
  OUT out_result VARCHAR(100),
  OUT out_order_id INT
)
BEGIN
  -- 声明变量
  DECLARE current_stock INT DEFAULT 0;
  DECLARE EXIT HANDLER FOR SQLEXCEPTION
  BEGIN
    -- 异常处理：回滚事务，返回失败信息
    ROLLBACK;
    SET out_result = '订单创建失败：系统异常';
    SET out_order_id = 0;
  END;
  
  -- 开启事务
  START TRANSACTION;
  
  -- 1. 查询商品库存
  SELECT stock INTO current_stock FROM product_stock WHERE product_id = in_product_id FOR UPDATE; -- 加行锁，防止并发超卖
  
  -- 2. 判断库存是否充足
  IF current_stock < in_product_num THEN
    ROLLBACK;
    SET out_result = '订单创建失败：库存不足';
    SET out_order_id = 0;
  ELSE
    -- 3. 扣减库存
    UPDATE product_stock SET stock = stock - in_product_num WHERE product_id = in_product_id;
    
    -- 4. 创建订单
    INSERT INTO order_info (user_id, order_no, order_amount, order_status) 
    VALUES (in_user_id, CONCAT('ORD', DATE_FORMAT(NOW(), '%Y%m%d%H%i%s'), FLOOR(RAND()*1000)), in_order_amount, '待支付');
    
    -- 5. 获取生成的订单ID
    SET out_order_id = LAST_INSERT_ID();
    
    -- 6. 提交事务
    COMMIT;
    SET out_result = '订单创建成功';
  END IF;
END //
DELIMITER ;

-- 调用存储过程
SET @result = '';
SET @order_id = 0;
CALL create_order(1, 1, 2, 199.98, @result, @order_id);
-- 查看结果
SELECT @result AS 结果, @order_id AS 订单ID;
-- 验证库存
SELECT * FROM product_stock;
```

## 八、核心要点总结

1. **存储过程与函数的核心定位**：
    
    - 存储过程：处理复杂业务逻辑、批量操作、事务控制，返回多个结果；
    - 存储函数：计算单个值、数据转换，嵌入 SQL 中使用。
    
2. **参数类型**：
    
    - 存储过程：IN（输入）、OUT（输出）、INOUT（输入输出）；
    - 存储函数：仅 IN。
    
3. **调用方式**：
    
    - 存储过程：`CALL 存储过程名();`；
    - 存储函数：`SELECT 函数名();` 或直接用在 SQL 中。
    
4. **变量区分**：
    
    - 局部变量：`DECLARE` 声明，仅当前 `BEGIN...END` 有效；
    - 用户变量：`@变量名`，当前会话有效；
    - 系统变量：`@@变量名`，MySQL 内置。
    
5. **避坑核心**：
    
    - 严格遵循分隔符修改流程；
    - 局部变量声明在块的最开头；
    - 存储函数不能返回结果集，必须有 RETURN；
    - 不要过度使用，避免业务逻辑耦合在数据库。
