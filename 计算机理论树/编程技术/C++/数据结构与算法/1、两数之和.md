## 一、题目核心要求拆解

1. 输入：整数数组 `nums`、整数目标值 `target`
2. 输出：数组中**和为 target 的两个整数对应的下标**
3. 约束条件：
    
    - 每组输入仅有一个有效答案
    - 不能重复使用数组中同一个元素（不能同一个下标用两次）
    - 下标返回顺序无要求

---

## 二、解法 1：暴力枚举法（入门理解）

### 核心思路

通过两层循环枚举数组中所有不重复的元素对，判断两数之和是否等于 target，找到后直接返回对应下标。

### 详细执行流程

1. 获取数组长度 `n`，确定循环边界
2. 外层循环：遍历数组每个元素，下标 `i` 从 `0` 到 `n-2`（内层需要从 i 之后开始，避免重复配对）
3. 内层循环：遍历 `i` 之后的所有元素，下标 `j` 从 `i+1` 到 `n-1`
4. 配对判断：若 `nums[i] + nums[j] == target`，直接返回 `{i, j}`
5. 题目保证有唯一解，无需处理无结果的异常情况

### 完整 C++ 代码

```cpp
// 引入头文件
#include <iostream>
#include <vector>  // 数组容器所需头文件

using namespace std;

class Solution {
public:
    vector<int> twoSum(vector<int>& nums, int target) {
        int n = nums.size();  // 获取数组长度
        // 外层循环：遍历每个元素
        for (int i = 0; i < n; ++i) {
            // 内层循环：只遍历i之后的元素，避免重复使用同一元素
            for (int j = i + 1; j < n; ++j) {
                // 判断两数之和是否等于目标值
                if (nums[i] + nums[j] == target) {
                    return {i, j};  // 找到结果，直接返回下标
                }
            }
        }
        return {};  // 题目保证有解，此处仅为语法兼容
    }
};

// 测试代码
int main() {
    Solution solution;
    // 测试用例1
    vector<int> nums1 = {2,7,11,15};
    int target1 = 9;
    vector<int> res1 = solution.twoSum(nums1, target1);
    cout << "测试用例1结果：[" << res1[0] << "," << res1[1] << "]" << endl;

    // 测试用例2
    vector<int> nums2 = {3,2,4};
    int target2 = 6;
    vector<int> res2 = solution.twoSum(nums2, target2);
    cout << "测试用例2结果：[" << res2[0] << "," << res2[1] << "]" << endl;

    return 0;
}
```

### 复杂度分析

- 时间复杂度：`O(n²)`，n 为数组长度，两层循环最坏情况下需要遍历所有元素对
- 空间复杂度：`O(1)`，仅使用了常数级的额外空间

## 三、解法 2：哈希表法（最优解，面试必用）

### 核心思路

利用哈希表（`unordered_map`）**O (1) 平均时间复杂度的查找和插入特性**，遍历数组时记录已经访问过的数值和对应下标。对于当前元素，只需要计算「目标值 - 当前值」的补数，直接在哈希表中查找是否存在这个补数：

- 存在：直接得到两个下标，返回结果
- 不存在：将当前数值和下标存入哈希表，继续遍历

### 详细执行流程

1. 创建`unordered_map<int, int>`哈希表，key 存储数组的数值，value 存储数值对应的下标
2. 遍历数组，下标`i`从`0`到`n-1`，依次访问每个元素`nums[i]`
3. 计算补数：`complement = target - nums[i]`（补数就是和当前数相加等于 target 的目标数）
4. 哈希表查找：
    
    - 若哈希表中存在 key 为`complement`的元素：说明之前已经遍历过这个补数，直接返回`{哈希表中补数对应的下标, i}`
    - 若不存在：将当前数值`nums[i]`和下标`i`存入哈希表，继续下一轮循环
    
5. 题目保证有唯一解，无需处理无结果的情况

> 关键注意点：**先查找补数，再存入当前元素**，避免出现「当前元素和自己配对」的错误（比如示例 3 的`[3,3]`，target=6，若先存再找会找到自己，返回错误下标`[0,0]`）

### 完整 C++ 代码

```cpp
// 引入头文件
#include <iostream>
#include <vector>
#include <unordered_map>  // 哈希表所需头文件

using namespace std;

class Solution {
public:
    vector<int> twoSum(vector<int>& nums, int target) {
        unordered_map<int, int> numMap;  // 哈希表：key=数值，value=下标
        int n = nums.size();

        for (int i = 0; i < n; ++i) {
            int complement = target - nums[i];  // 计算需要的补数
            // 查找补数是否存在于哈希表中
            if (numMap.find(complement) != numMap.end()) {
                // 找到补数，返回两个下标
                return {numMap[complement], i};
            }
            // 没找到补数，将当前数值和下标存入哈希表
            numMap[nums[i]] = i;
        }
        return {};  // 题目保证有解，此处仅为语法兼容
    }
};

// 测试代码
int main() {
    Solution solution;
    // 测试用例1
    vector<int> nums1 = {2,7,11,15};
    int target1 = 9;
    vector<int> res1 = solution.twoSum(nums1, target1);
    cout << "测试用例1结果：[" << res1[0] << "," << res1[1] << "]" << endl;

    // 测试用例2
    vector<int> nums2 = {3,2,4};
    int target2 = 6;
    vector<int> res2 = solution.twoSum(nums2, target2);
    cout << "测试用例2结果：[" << res2[0] << "," << res2[1] << "]" << endl;

    // 测试用例3（重复元素场景）
    vector<int> nums3 = {3,3};
    int target3 = 6;
    vector<int> res3 = solution.twoSum(nums3, target3);
    cout << "测试用例3结果：[" << res3[0] << "," << res3[1] << "]" << endl;

    return 0;
}
```

### 代码逐行核心解释

1. `unordered_map<int, int> numMap;`：创建哈希表，用于缓存已经遍历过的数值和下标，避免重复遍历
2. `int complement = target - nums[i];`：计算补数，把「两数之和等于 target」的问题，转化为「查找补数是否存在」的问题
3. `numMap.find(complement) != numMap.end()`：哈希表的查找操作，`find`方法如果找不到元素会返回`end()`迭代器，找到则返回对应元素的迭代器
4. `numMap[nums[i]] = i;`：将当前遍历的数值和下标存入哈希表，供后续元素查找使用

### 复杂度分析

- 时间复杂度：`O(n)`，n 为数组长度，每个元素仅遍历一次，哈希表的查找和插入操作平均时间复杂度都是`O(1)`
- 空间复杂度：`O(n)`，最坏情况下需要存储数组中所有的元素到哈希表中
