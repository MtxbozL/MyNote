## 一、概述

### 1. 为什么需要时间复杂度？

衡量一个算法的 “快慢”，不能只看 “运行时间（秒数）”—— 因为运行时间受硬件、编程语言、编译器等外部因素影响极大（比如同一算法在超级计算机和手机上的运行时间天差地别）。

**时间复杂度**是一种**不依赖外部环境**的、**数学化的**算法效率衡量方式，它关注的是：**随着输入数据规模 `n` 的增大，算法的执行时间的「增长趋势」**。

### 2. 核心定义

时间复杂度描述的是：**算法中「基本操作的执行次数」与「输入数据规模 `n`」之间的函数关系**，用 **大 O 符号（Big-O Notation）** 表示（如 `O(n)`、`O(n²)`、`O(logn)` 等）。

---

## 二、核心概念：大 O 符号（Big-O Notation）

### 1. 大 O 的含义

大 O 符号表示的是 **「渐进上界」**（Asymptotic Upper Bound），简单理解就是：**当输入规模 `n` 足够大时，算法的执行时间「最多」增长得有多快**。

它只关注**增长趋势的 “量级”**，不关心具体的常数系数和低阶项（因为当 `n` 很大时，常数和低阶项对增长趋势的影响可以忽略不计）。

### 2. 大 O 的简化规则（关键！必须记牢）

在计算时间复杂度时，我们需要对 “基本操作的执行次数” 进行**数学简化**，只保留**最高阶的项**，并忽略：

1. **常数项**：如 `O(100)` → 简化为 `O(1)`；
2. **常数系数**：如 `O(2n)` → 简化为 `O(n)`，`O(3n²)` → 简化为 `O(n²)`；
3. **低阶项**：如 `O(n² + n)` → 简化为 `O(n²)`，`O(nlogn + n)` → 简化为 `O(nlogn)`。

---

## 三、时间复杂度的计算步骤（新手友好）

计算时间复杂度，只需按以下 4 步走：

### 第 1 步：找出算法中的「基本操作」

基本操作是算法中**执行次数最多、对运行时间影响最大**的操作，通常是：

- 循环体中的核心语句（如赋值、比较、算术运算）；
- 递归调用（递归算法中）。

### 第 2 步：计算基本操作的「执行次数」

统计基本操作在算法中一共执行了多少次，用输入规模 `n` 的函数表示（记为 `T(n)`）。

### 第 3 步：对 `T(n)` 进行简化

应用大 O 的简化规则，忽略常数项、常数系数和低阶项，只保留最高阶项。

### 第 4 步：用大 O 符号表示结果

将简化后的最高阶项用 `O(...)` 包裹，得到最终的时间复杂度。

---

## 四、常见时间复杂度类型（从优到劣排序）

### 1. `O(1)`：常数时间（Constant Time）

#### 含义

基本操作的执行次数是**固定的常数**，与输入规模 `n` 无关（无论 `n` 是 10 还是 10000，执行次数都一样）。

#### 典型代码示例

```CPP
// 示例1：访问数组的第 i 个元素
int getElement(int arr[], int i) {
    return arr[i]; // 基本操作：数组索引访问，执行1次，与n无关
}

// 示例2：简单的算术运算
int add(int a, int b) {
    return a + b; // 基本操作：加法，执行1次
}
```

---

### 2. `O(logn)`：对数时间（Logarithmic Time）

#### 含义

基本操作的执行次数随 `n` 增大而**对数级增长**（通常以 2 为底，即 `log₂n`），增长极慢（效率极高）。

#### 典型代码示例：二分查找（Binary Search）

cpp

运行

```
// 二分查找：在有序数组中查找目标值 target
int binarySearch(int arr[], int n, int target) {
    int left = 0, right = n - 1;
    while (left <= right) {
        int mid = left + (right - left) / 2; // 基本操作：计算 mid
        if (arr[mid] == target) {
            return mid;
        } else if (arr[mid] < target) {
            left = mid + 1;
        } else {
            right = mid - 1;
        }
    }
    return -1;
}
```

#### 计算过程

- 每次循环，搜索范围都会**缩小一半**（从 `n` → `n/2` → `n/4` → ... → `1`）；
- 设循环执行了 `k` 次，则 `n / 2^k = 1` → `k = log₂n`；
- 简化后时间复杂度：`O(logn)`。

---

### 3. `O(n)`：线性时间（Linear Time）

#### 含义

基本操作的执行次数随 `n` 增大而**线性增长**（`n` 增大 10 倍，执行次数也增大 10 倍）。

#### 典型代码示例：遍历数组

cpp

运行

```
// 遍历数组，求所有元素的和
int sumArray(int arr[], int n) {
    int sum = 0;
    for (int i = 0; i < n; i++) { // 循环执行 n 次
        sum += arr[i]; // 基本操作：加法，执行 n 次
    }
    return sum;
}
```

#### 计算过程

- 循环执行 `n` 次，基本操作执行 `n` 次；
- 简化后时间复杂度：`O(n)`。

---

### 4. `O(nlogn)`：线性对数时间（Linearithmic Time）

#### 含义

基本操作的执行次数随 `n` 增大而**线性对数级增长**（比 `O(n)` 稍慢，但远快于 `O(n²)`）。

#### 典型代码示例：快速排序（Quick Sort）、归并排序（Merge Sort）

以归并排序为例（核心思想是 “分治”）：

cpp

运行

```
// 归并排序的时间复杂度是 O(nlogn)
void mergeSort(int arr[], int left, int right) {
    if (left < right) {
        int mid = left + (right - left) / 2;
        mergeSort(arr, left, mid);      // 递归左半部分
        mergeSort(arr, mid + 1, right); // 递归右半部分
        merge(arr, left, mid, right);   // 合并两个有序数组（O(n) 操作）
    }
}
```

#### 计算过程

- 递归深度为 `log₂n`（每次将数组分成两半）；
- 每一层递归的合并操作总时间为 `O(n)`；
- 总时间复杂度：`O(n) × O(logn) = O(nlogn)`。

---

### 5. `O(n²)`：平方时间（Quadratic Time）

#### 含义

基本操作的执行次数随 `n` 增大而**平方级增长**（`n` 增大 10 倍，执行次数增大 100 倍），效率较低，通常对应**嵌套循环**。

#### 典型代码示例：两数之和（暴力解法）

cpp

运行

```
// 两数之和暴力解法：两层嵌套循环
vector<int> twoSum(vector<int>& nums, int target) {
    int n = nums.size();
    for (int i = 0; i < n; i++) {        // 外层循环执行 n 次
        for (int j = i + 1; j < n; j++) { // 内层循环平均执行 n/2 次
            if (nums[i] + nums[j] == target) { // 基本操作：加法+比较
                return {i, j};
            }
        }
    }
    return {};
}
```

#### 计算过程

- 外层循环执行 `n` 次，内层循环平均执行 `n/2` 次；
- 总执行次数：`n × n/2 = n²/2`；
- 简化后时间复杂度：`O(n²)`。

### 6. 时间复杂度优劣对比图

表格

|时间复杂度|名称|增长速度|适用场景|
|---|---|---|---|
|`O(1)`|常数时间|极慢|数组索引访问、哈希表查找|
|`O(logn)`|对数时间|很慢|二分查找、平衡树操作|
|`O(n)`|线性时间|慢|遍历数组、链表|
|`O(nlogn)`|线性对数时间|中等|高效排序（快排、归并）|
|`O(n²)`|平方时间|快|暴力解法、冒泡排序|
|`O(2ⁿ)`|指数时间|极快|递归求解斐波那契（暴力）|

## 五、最好、最坏、平均情况时间复杂度

有些算法的时间复杂度**受输入数据的具体内容影响**（比如排序算法，输入已经有序和输入完全逆序的执行时间可能不同），因此需要区分三种情况：

### 1. 最好情况时间复杂度（Best Case）

输入数据处于**最理想状态**时的时间复杂度（执行次数最少）。

- 示例：在数组中查找目标值，最好情况是 “第一个元素就是目标值”，此时时间复杂度为 `O(1)`。

### 2. 最坏情况时间复杂度（Worst Case）

输入数据处于**最糟糕状态**时的时间复杂度（执行次数最多）。

- 示例：在数组中查找目标值，最坏情况是 “目标值在最后一个位置或不存在”，此时时间复杂度为 `O(n)`。
- **重要性**：通常我们最关注**最坏情况时间复杂度**，因为它保证了算法的 “上限”—— 无论输入什么数据，算法的运行时间都不会超过这个值。

### 3. 平均情况时间复杂度（Average Case）

输入数据处于**所有可能情况的平均状态**时的时间复杂度（需要考虑所有输入的概率分布）。

- 示例：在数组中查找目标值，平均情况是 “目标值在每个位置的概率相等”，此时平均查找次数为 `n/2`，时间复杂度为 `O(n)`。

## 六、结合之前算法题的实例分析

### 实例 1：两数之和

#### 解法 1：暴力解法（两层嵌套循环）

cpp

运行

```
vector<int> twoSum(vector<int>& nums, int target) {
    int n = nums.size();
    for (int i = 0; i < n; i++) {
        for (int j = i + 1; j < n; j++) {
            if (nums[i] + nums[j] == target) {
                return {i, j};
            }
        }
    }
    return {};
}
```

- **基本操作**：内层循环的加法和比较；
- **执行次数**：`n²/2`；
- **时间复杂度**：`O(n²)`。

#### 解法 2：哈希表解法（一层循环）

cpp

运行

```
vector<int> twoSum(vector<int>& nums, int target) {
    unordered_map<int, int> hashMap;
    for (int i = 0; i < nums.size(); i++) {
        int complement = target - nums[i];
        if (hashMap.find(complement) != hashMap.end()) {
            return {hashMap[complement], i};
        }
        hashMap[nums[i]] = i;
    }
    return {};
}
```

- **基本操作**：哈希表的 `find()` 和插入（均为 `O(1)` 平均时间）；
- **执行次数**：`n`；
- **时间复杂度**：`O(n)`（平均情况）。

### 实例 2：字母异位词分组

#### 解法 1：排序法

cpp

运行

```
vector<vector<string>> groupAnagrams(vector<string>& strs) {
    unordered_map<string, vector<string>> hashMap;
    for (const string& s : strs) {
        string key = s;
        sort(key.begin(), key.end()); // 排序的时间复杂度是 O(klogk)，k是字符串最大长度
        hashMap[key].push_back(s);
    }
    vector<vector<string>> result;
    for (auto& pair : hashMap) {
        result.push_back(pair.second);
    }
    return result;
}
```

- **基本操作**：字符串排序；
- **执行次数**：设 `n` 是字符串数量，`k` 是字符串最大长度，则总时间为 `n × klogk`；
- **时间复杂度**：`O(nklogk)`。

## 七、新手避坑指南

### 1. 不要只看循环层数，要看循环的实际执行次数

- 错误认知：“一层循环就是 `O(n)`，两层循环就是 `O(n²)`”；
- 反例：二分查找只有一层循环，但时间复杂度是 `O(logn)`（不是 `O(n)`）。

### 2. 递归算法的时间复杂度不是 `O(1)`

递归算法的时间复杂度需要结合**递归深度**和**每一层的操作次数**来计算（如归并排序是 `O(nlogn)`）。

### 3. 哈希表的操作是「平均」`O(1)`

`unordered_map` 的 `find()`、`insert()` 等操作的**平均**时间复杂度是 `O(1)`，但**最坏情况**（哈希冲突严重）是 `O(n)`（不过实际工程中通常可以忽略最坏情况）。

---

## 八、总结

1. **核心作用**：时间复杂度是衡量算法效率的数学工具，关注执行时间随输入规模 `n` 的增长趋势；
2. **大 O 符号**：表示渐进上界，需忽略常数项、系数和低阶项；
3. **计算步骤**：找基本操作 → 算执行次数 → 简化 → 用大 O 表示；
4. **常见类型**：`O(1)` < `O(logn)` < `O(n)` < `O(nlogn)` < `O(n²)`；
5. **关注重点**：通常优先关注**最坏情况时间复杂度**，它保证了算法的性能上限。
