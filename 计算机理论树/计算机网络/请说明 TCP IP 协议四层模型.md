#### 核心标准答案

TCP/IP 四层模型是互联网的核心通信架构，**从下到上依次为网络接口层、网络层、传输层、应用层**，每层职责清晰、协议固定，共同完成端到端的数据通信。

1. **网络接口层**：对应 OSI 模型的物理层 + 数据链路层，负责硬件设备间的比特流传输，处理以太网帧、MAC 地址寻址，核心协议为 ARP、RARP。
2. **网络层**：负责跨网络的寻址与路由转发，定义了统一的 IP 地址格式，核心协议为 IP、ICMP、IGMP，核心设备为路由器、三层交换机。
3. **传输层**：负责两台主机的端到端通信，提供流量控制、可靠传输、差错校验能力，核心协议为 TCP（面向连接、可靠）、UDP（无连接、不可靠）。
4. **应用层**：为用户应用程序提供通信服务，定义了各类业务场景的通信规则，核心协议为 HTTP/HTTPS、DNS、FTP、SSH、SMTP 等。

#### 专家级拓展

- 与 OSI 七层模型的核心差异：TCP/IP 是工业落地标准，合并了会话层、表示层到应用层，简化了层级，更适配实际网络架构；OSI 是理论参考模型，多用于教学与问题分层排查。
- 岗位落地场景：运维排查网络问题时，严格遵循「从下到上」的分层思路 —— 先 ping 通 IP 确认网络层正常，再 telnet 端口确认传输层正常，最后 curl 接口确认应用层正常，快速定位故障层级。

#### 面试避坑指南

- 严禁把 TCP/IP 四层模型和 OSI 七层模型混淆、层级顺序说反；
- 避免只说层级名称，不说核心职责与核心协议，会被面试官判定为「死记硬背，无实际应用能力」。
