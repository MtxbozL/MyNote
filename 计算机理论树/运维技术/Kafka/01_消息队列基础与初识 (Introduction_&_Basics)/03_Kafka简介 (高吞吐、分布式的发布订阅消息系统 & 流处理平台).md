## 1.3 Kafka 简介

Apache Kafka 是由 Apache 软件基金会维护的开源分布式系统，是一款兼具高吞吐、低延迟、高可用、可扩展特性的发布 - 订阅模式消息队列系统，同时也是一套完整的分布式流处理平台。该项目最初由 LinkedIn 公司研发，2011 年开源并进入 Apache 孵化器，2012 年正式成为 Apache 顶级项目，目前已成为分布式实时数据处理领域的基础设施。

### 1.3.1 Kafka 核心设计目标

1. **极致高吞吐**：通过分区并行、顺序写盘、零拷贝等核心机制，以毫秒级延迟支持每秒数十万级的消息读写，满足海量数据流的处理需求；
2. **持久化与可靠性**：基于磁盘的顺序持久化存储，配合多副本冗余机制、ISR 同步副本策略，保障消息不丢失与数据一致性；
3. **高可用无单点**：分布式集群架构，支持节点故障自动转移、Leader 副本自动选举，集群无单点故障，服务可用性可达 99.99% 以上；
4. **线性水平扩展**：基于分区的并行设计，支持集群节点与分区数的动态扩容，吞吐量随集群规模线性增长；
5. **全生态兼容**：与 Flink、Spark、Hadoop、Elasticsearch 等大数据生态组件深度适配，提供 Kafka Connect、Kafka Streams 等原生生态工具，覆盖数据集成、流处理全场景。

### 1.3.2 Kafka 核心应用场景

1. **日志采集与传输**：作为海量服务日志的集中传输枢纽，实现日志的采集、聚合与转发，是 ELK/EFK 日志架构的核心传输层组件；
2. **实时大数据流处理**：作为 Flink、Spark Streaming 等流计算引擎的核心数据源与数据汇（Sink），支撑实时数仓、实时风控、实时推荐、实时监控等业务场景；
3. **微服务异步通信**：在微服务架构中作为核心消息中间件，实现服务解耦、业务异步化、流量削峰，提升分布式系统的稳定性与扩展性；
4. **事件驱动架构（EDA）**：作为分布式事件总线，支撑事件溯源、CQRS（命令查询职责分离）等架构模式的落地，实现业务系统的全链路事件化；
5. **异构数据同步管道**：通过 Kafka Connect 组件，实现数据库、数据仓库、搜索引擎、对象存储等异构数据源之间的实时数据同步与集成。
