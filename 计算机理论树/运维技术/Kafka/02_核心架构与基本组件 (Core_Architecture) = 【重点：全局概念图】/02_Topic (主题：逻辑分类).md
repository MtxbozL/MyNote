## 2.3 Topic（主题：逻辑分类）

Topic 是 Kafka 中消息的**逻辑分类容器**，是发布 - 订阅模型的核心抽象，实现了不同业务消息的逻辑隔离与路由管理，是生产者与消费者的核心交互载体。

### 2.3.1 核心定义与特性

1. **逻辑抽象，无物理存储**：Topic 本身是纯逻辑概念，不直接存储任何消息数据，消息实际存储在 Topic 下属的 Partition（分区）中，Topic 的存储能力、并行处理能力完全由下属分区决定；
2. **全局唯一性**：在同一个 Kafka 集群内，Topic 名称必须全局唯一，作为消息路由的核心标识，生产者通过 Topic 名称指定消息发送目标，消费者通过 Topic 名称订阅消息；
3. **多订阅者模型**：同一个 Topic 可被任意数量的消费组、任意数量的消费者订阅，不同消费组之间的消费偏移量、消费行为完全隔离，互不影响，天然支持单播与多播两种消费模式；
4. **配置隔离与优先级**：Topic 级别可独立配置副本因子、分区数量、日志保留时间、清理策略、最小 ISR 大小、时间戳类型等参数，优先级高于集群全局配置，可针对不同业务场景的差异化需求定制配置；
5. **生命周期全管控**：支持 Topic 的创建、查询、修改、删除全生命周期管理，通过 CLI 工具、Admin API 可完成全量操作，Topic 删除时会同步删除其下属所有分区的副本数据与元数据。
