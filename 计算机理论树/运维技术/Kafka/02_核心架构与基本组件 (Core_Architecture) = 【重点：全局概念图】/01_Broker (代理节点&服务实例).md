本章基于 Kafka 分布式架构的全局拓扑，系统性拆解其核心基础组件的设计原理、核心职责与交互逻辑，构建 Kafka 的完整概念体系，是后续深入学习生产消费机制、数据一致性、高性能存储等核心内容的理论基础。本章核心重点为 Kafka 全局架构概念视图，所有组件均围绕该视图的数据流与控制流展开。

## 2.1 Kafka 全局架构总览

Kafka 是一套分布式、多副本、基于发布 - 订阅模型的消息系统与流处理平台，其核心架构分为**数据平面**与**控制平面**两大维度，核心角色与交互逻辑如下：

1. **数据平面**：负责消息的生产、存储、消费全链路数据流处理，核心角色为生产者（Producer）、Broker 代理节点、消费者 / 消费组（Consumer/Consumer Group）；
2. **控制平面**：负责集群元数据管理、选主、故障转移、配置管理等集群控制逻辑，核心载体为元数据管理系统（ZooKeeper/KRaft 控制器集群）；
3. **核心数据流**：生产者将业务数据封装为消息，发送至 Broker 节点的指定主题（Topic），Broker 完成消息的持久化存储与副本同步；消费者通过订阅主题，从 Broker 拉取消息完成业务处理，不同消费组之间的消费行为完全隔离；
4. **核心控制流**：控制平面统一管理集群所有元数据，向 Broker 节点同步集群状态变更，触发分区 Leader 选举、故障转移、副本重分配等核心操作，保障集群的高可用与数据一致性。

## 2.2 Broker（代理节点 / 服务实例）

Broker 是 Kafka 集群的核心服务进程实例，是数据平面的核心载体，单个物理 / 虚拟机节点通常部署一个 Broker 进程，多个 Broker 节点共同组成 Kafka 分布式集群，实现集群的水平扩展与高可用。

### 2.2.1 核心职责

1. **消息存储与读写服务**：接收生产者发送的消息，基于顺序写盘机制完成消息的持久化存储；响应消费者的拉取请求，从本地磁盘读取指定消息并返回，是集群唯一的消息存储载体；
2. **副本管理**：作为分区副本的宿主节点，承担 Leader 副本或 Follower 副本角色：Leader 副本负责处理该分区的所有读写请求，Follower 副本负责与 Leader 副本同步数据，为故障转移提供冗余保障；
3. **集群协调与控制**：在 KRaft 架构中，指定 Broker 可承担控制器（Controller）角色，组成控制平面集群，负责集群元数据管理、分区 Leader 选举、故障转移等核心控制逻辑；在 ZooKeeper 架构中，单个 Broker 会被选举为集群 Controller，承担控制平面职责；
4. **请求处理与流量管控**：基于 Reactor 多线程模型处理客户端的网络请求，实现高并发 IO 处理；原生支持基于请求速率、网络带宽的配额管控，避免单客户端流量过载影响集群稳定性；
5. **数据生命周期管理**：执行 Topic 配置的日志清理策略，包括基于时间 / 大小的日志删除、日志压缩，完成过期数据的自动清理，管理磁盘存储空间。

### 2.2.2 核心设计特性

- **对等节点设计**：Kafka 集群中所有 Broker 节点在网络拓扑上是对等的，客户端可连接任意 Broker 节点获取集群全量元数据，无需依赖中心转发节点，简化了客户端接入逻辑，避免了中心节点瓶颈；
- **故障自愈能力**：Broker 节点故障时，控制平面可快速感知，自动触发该节点上 Leader 副本的重选举，将故障分区的 Leader 角色切换至健康 Broker 节点的同步 Follower 副本，实现服务的自动故障转移，无需人工干预；
- **无状态服务特性**：Broker 的核心状态均存储于本地磁盘副本数据与集群元数据系统中，节点重启后可自动从元数据系统恢复集群状态，从 Leader 副本同步缺失数据，快速重新加入集群。



### 2.3.2 Topic 分类

Kafka 将 Topic 分为两大类，分别承担不同的功能：

1. **用户业务 Topic**：由用户创建，用于承载业务消息数据，是生产消费的核心载体，配置与生命周期完全由用户管控；
2. **系统内部 Topic**：Kafka 集群内置的 Topic，用于实现集群核心功能，用户不可随意修改或删除，核心包括：
    
    - `__consumer_offsets`：存储消费组的消费偏移量数据，替代了旧版本 ZooKeeper 的偏移量存储功能；
    - `__transaction_state`：存储 Kafka 事务的状态数据，用于实现事务消息与 Exactly-Once 语义；
    - `__cluster_metadata`：KRaft 架构内置元数据主题，存储集群全量元数据日志，基于 Raft 协议实现强一致性。
