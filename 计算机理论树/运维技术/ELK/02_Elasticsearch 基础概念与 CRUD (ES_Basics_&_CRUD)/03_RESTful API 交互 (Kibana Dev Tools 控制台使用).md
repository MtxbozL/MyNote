
ES 完全基于**RESTful 架构风格**设计对外交互接口，所有操作均通过标准 HTTP 协议实现，采用 HTTP 方法定义操作语义，JSON 作为请求与响应的标准数据格式，具备无状态、跨语言、易调试的核心优势，是所有 ES 操作的唯一标准入口。

### 2.3.1 RESTful API 基础规范

#### 标准 URL 格式

8.x + 版本的标准 API 端点格式为：

```txt
http://<es_host>:<es_port>/<索引名>/_doc/<文档ID>
```

其中`_doc`为固定文档操作端点，不再具备 Type 语义，所有文档操作均基于该格式展开。

#### 标准 HTTP 方法语义

ES 严格遵循 HTTP 方法的 RESTful 语义，核心方法与对应操作如下：

|HTTP 方法|核心语义|幂等性|典型使用场景|
|---|---|---|---|
|GET|数据读取与查询，无任何副作用|严格幂等|单文档查询、搜索、集群状态查询、索引信息查询|
|PUT|创建或全量替换资源，目标资源位置由客户端指定|严格幂等|指定 ID 的文档创建、全量替换、索引创建与配置更新|
|POST|资源创建、更新或操作执行，服务端可处理动态逻辑|非严格幂等|自动生成 ID 的文档创建、局部更新、批量操作、搜索、脚本执行|
|DELETE|资源删除|严格幂等|文档删除、索引删除|
|HEAD|资源存在性检查，仅返回状态码，无响应体|严格幂等|索引、文档的存在性校验|

### 2.3.2 Kibana Dev Tools 控制台

Kibana Dev Tools 是 ES 官方提供的原生 API 调试与开发工具，是学习与生产环境调试的首选工具，相较于 curl 等命令行工具，具备语法高亮、自动补全、API 文档跳转、请求格式化、历史记录留存等核心优势。

#### 核心使用规范

1. **入口路径**：Kibana 控制台 → 左侧导航栏 → Analytics → Dev Tools（8.x 版本）；
2. **请求语法**：每行请求格式为`<HTTP方法> <请求路径>`，换行后编写 JSON 请求体，支持多请求批量编写与执行；
3. **执行快捷键**：Windows/Linux 为`Ctrl+Enter`，Mac 为`Cmd+Enter`；
4. **基础示例**：

    ```bash
    # 集群健康状态查询
    GET /_cluster/health
    
    # 索引列表查询
    GET /_cat/indices?v
    
    # 单文档写入
    PUT /user_info/_doc/1
    {
      "username": "zhangsan",
      "age": 25,
      "email": "zhangsan@example.com"
    }
    ```
