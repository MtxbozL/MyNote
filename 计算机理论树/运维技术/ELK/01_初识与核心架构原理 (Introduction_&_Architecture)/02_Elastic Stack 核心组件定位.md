
Elastic Stack 的四大核心组件各司其职，形成了松耦合、高内聚的全链路架构，各组件的核心定位与技术特性如下：

### 1.2.1 Elasticsearch

Elasticsearch 是整个技术栈的**核心分布式搜索与分析引擎**，基于 Apache Lucene 封装实现，承担全量数据的存储、检索、聚合计算核心职责，是整个技术栈的算力与存储底座。

其核心技术特性包括：

- 分布式原生架构，支持水平扩展，可实现 PB 级数据的分布式存储与并行计算；
- 近实时（Near Real-Time, NRT）搜索与分析能力，数据写入后可在秒级内被检索；
- 基于 RESTful API 的标准化交互接口，兼容多语言客户端接入；
- 完善的多租户能力、数据生命周期管理能力与容灾备份机制；
- 丰富的全文检索查询语法与多维度聚合分析能力，支撑复杂的业务检索与统计需求。

### 1.2.2 Logstash

Logstash 是技术栈的**ETL（Extract-Transform-Load）数据处理管道**，基于 JVM 开发，核心定位是重量级数据清洗、格式转换与路由分发，解决多源异构数据的结构化处理需求。

其核心处理流为`Input→Filter→Output`三段式架构：

- Input 层：支持 Beats、Kafka、文件、数据库、TCP/UDP 等多源数据的接入；
- Filter 层：核心能力层，提供正则抽取、时间解析、字段转换、地理信息解析等丰富的处理插件，实现非结构化数据向结构化数据的转换；
- Output 层：支持将处理后的数据路由至 Elasticsearch、Kafka、文件、对象存储等多个目标终端，支持动态索引路由与数据备份。

### 1.2.3 Kibana

Kibana 是技术栈的**前端可视化与集群管控平台**，与 Elasticsearch 深度原生集成，是整个技术栈的人机交互入口，核心实现数据价值的可视化落地与集群的全生命周期管控。

其核心能力包括：

- 数据视图管理，通过索引模式关联 Elasticsearch 中的数据，建立统一的数据访问入口；
- 日志实时检索，通过 KQL 语法实现海量日志的快速检索与过滤；
- 可视化与仪表盘搭建，支持拖拽式生成折线图、柱状图、饼图、地图、时序数据图表等可视化组件，实现业务监控、日志分析等场景的大盘搭建；
- 集群管控能力，支持在前端界面完成索引管理、生命周期配置、用户权限管控、安全配置、告警规则设置等运维操作。

### 1.2.4 Beats

Beats 是技术栈的**边缘轻量级数据采集探针家族**，基于 Go 语言开发，无 JVM 依赖，资源占用极低，核心定位是边缘节点的原位数据采集，解决传统 ELK 架构采集层的资源瓶颈问题。

Beats 家族针对不同采集场景提供了专用采集器，核心包括：

- Filebeat：最核心的采集组件，面向日志文件场景，支持日志文件的实时采集、多行合并、断点续传，是日志分析场景的首选采集工具；
- Metricbeat：面向系统与服务指标采集，支持 CPU、内存、磁盘、网络等主机指标，以及 Nginx、MySQL、Redis 等中间件服务的指标采集；
- 其他专用采集器：Packetbeat（网络流量采集）、Auditbeat（审计数据采集）、Winlogbeat（Windows 事件日志采集）、Heartbeat（服务可用性监控采集）等。
    
    Beats 支持数据直接输出至 Elasticsearch，也可输出至 Logstash 做进一步清洗，或输出至 Kafka 做流量削峰，具备极强的架构适配性。
