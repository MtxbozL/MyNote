## 本章导读

本章是 Elasticsearch 除全文检索外的第二大核心能力模块，承接第四章的 Query DSL 查询体系，聚焦海量数据的结构化统计分析能力。核心目标是建立 ES 聚合分析的完整认知框架，掌握分桶聚合、指标聚合、管道聚合三大核心聚合类型的原理、语法与适用场景，精通时序数据统计、多维度分组分析、二次聚合计算的生产级实现方案。本章内容是日志监控、业务数据统计、Kibana 可视化大屏的核心底层支撑，是日常开发与面试考察的重点内容，为后续企业级可观测性架构设计奠定数据分析基础。

聚合分析的核心本质是：**对 Query DSL 查询命中的文档数据集，执行结构化的数值计算与分组统计，将非结构化 / 半结构化的原始数据转化为可量化、可可视化的业务指标**，区别于全文检索 “查找匹配文档” 的核心目标，聚合分析的核心目标是 “从数据中提取统计价值”。

## 5.1 聚合分析的核心概念

### 5.1.1 核心定义与执行逻辑

ES 聚合分析是一套分布式的统计计算引擎，基于 Lucene 的列式存储 Doc Values 实现，针对海量数据实现低延迟的多维度统计分析。其核心执行逻辑严格遵循 **“先过滤，后聚合”** 的执行流程：

1. 执行 Query DSL 查询，通过过滤上下文优先缩小目标数据集，得到聚合的原始文档集；
2. 协调节点将聚合请求分发到对应分片，各分片在本地执行并行聚合计算，生成分片级聚合结果；
3. 协调节点收集所有分片的聚合结果，执行全局合并计算，生成最终的聚合统计结果返回给客户端。

该分布式执行架构保证了 ES 在 PB 级数据下的聚合分析性能，远超传统关系型数据库的单节点计算模式，典型应用场景包括：

- 日志分析场景：百万级日志中计算接口平均响应时间、错误状态码占比、TOP N 异常接口；
- 业务统计场景：电商平台的商品销量排行、用户地域分布、销售额趋势变化、转化率统计；
- 监控运维场景：服务器指标的时序趋势分析、容器资源使用率统计、告警阈值计算；
- 可视化场景：Kibana Dashboard 的所有图表、大盘，底层均基于 ES 聚合分析实现。

### 5.1.2 聚合的标准语法结构

ES 聚合分析通过 RESTful API 的`_search`端点实现，与查询语句无缝集成，标准语法结构如下：

```bash
GET /<索引名称>/_search
{
  "size": 0,  // 关闭原始文档返回，仅获取聚合结果，大幅提升性能
  "query": {
    // Query DSL查询语句，先过滤出聚合的目标数据集
  },
  "aggs": {  // 聚合声明，简写为aggs，与aggregations完全等价
    "<聚合名称>": {  // 自定义聚合名称，用于标识聚合结果，全局唯一
      "<聚合类型>": {
        // 该聚合类型的核心参数配置
        "field": "<聚合字段名>",
        "<参数名>": "<参数值>"
      },
      "aggs": {
        // 嵌套子聚合，实现多维度的分层统计
        "<子聚合名称>": {
          "<子聚合类型>": {
            // 子聚合参数配置
          }
        }
      }
    }
  }
}
```

#### 核心语法规则

1. `size: 0`：生产环境聚合查询的强制规范，当无需查看原始文档、仅需聚合结果时，必须设置`size: 0`，避免返回全量文档导致的网络开销与性能损耗；
2. 聚合名称：用户自定义的唯一标识，用于在响应结果中定位聚合数据，支持字母、数字、下划线，禁止使用特殊字符；
3. 聚合嵌套：ES 聚合支持无限层级的嵌套，分桶聚合内可嵌套子分桶聚合与指标聚合，实现 “先分组，再分组，最后统计” 的多维度分析，这是 ES 聚合最强大的核心特性之一；
4. 字段约束：所有聚合字段必须开启 Doc Values（keyword、数值、date 类型默认开启，text 类型默认关闭），否则无法执行聚合计算。

### 5.1.3 聚合的三大核心类型

ES 聚合体系分为三大核心类型，覆盖全场景统计分析需求，各自的核心定位与能力边界如下：

|聚合类型|核心定位|核心能力|类比 SQL|
|---|---|---|---|
|Bucket Aggregations（分桶聚合）|数据分组|按照指定规则将文档分配到不同的桶中，实现数据的分组归类|`GROUP BY`|
|Metric Aggregations（指标聚合）|数值计算|对文档集执行数值统计计算，输出统计指标值|`COUNT`/`SUM`/`AVG`/`MAX`/`MIN`/`COUNT(DISTINCT)`|
|Pipeline Aggregations（管道聚合）|二次聚合|对其他聚合的输出结果进行二次计算，实现复杂的趋势分析|基于 SQL 查询结果的二次计算、窗口函数|

三大聚合类型通常协同使用：先通过分桶聚合实现数据分组，再通过指标聚合对每个桶内的文档执行统计计算，最后通过管道聚合对统计结果做二次趋势分析，构成完整的数据分析链路。
