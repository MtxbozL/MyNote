## 2.5 Web 前端中文乱码问题解决方案

### 2.5.1 乱码问题根因

Zabbix Web 前端中文乱码分为两类，核心根因如下：

1. **页面文本乱码**：前端 CSS 默认调用的字体不包含中文字形，导致页面中文显示为方框（豆腐块）；
2. **监控图形乱码**：PHP 的 GD 库用于生成监控图形，默认使用的 DejaVuSans 字体无中文字形支持，导致图形内的中文标签、注释显示为乱码。

两类乱码的核心解决方案为 **替换兼容中文的合规字体** ，修改前端字体配置，实现全场景中文正常渲染。

### 2.5.2 标准化解决方案

1. **合规中文字体选型**：生产环境优先使用开源无版权风险的中文字体，如思源黑体（Noto Sans SC）、文泉驿正黑体，测试环境可使用微软雅黑字体，禁止使用无授权的商业字体；
2. **字体文件部署**：将选中的中文字体文件（TTF 格式）上传至 Zabbix Web 前端的字体目录，不同版本默认路径如下：
    
    - 6.0 LTS：`/usr/share/zabbix/assets/fonts/`
    - 7.0 LTS：`/usr/share/zabbix/assets/fonts/`、`/usr/share/zabbix/include/fonts/`
    
3. **前端字体配置修改**：修改 Zabbix Web 前端的字体定义文件，将默认字体名称替换为新部署的中文字体名称，配置文件路径为`/usr/share/zabbix/include/defines.inc.php`，修改`ZBX_GRAPH_FONT_NAME`、`ZBX_FONT_NAME`两个常量为新字体名称；
4. **生效验证**：重启 Web 服务器与 PHP-FPM 服务，清理浏览器缓存，重新访问 Zabbix Web 前端，验证页面文本与监控图形的中文正常显示，无乱码问题。
