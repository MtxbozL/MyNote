## 1.5 核心术语与语义映射

Zabbix 监控体系基于一套标准化的术语体系构建，各术语之间存在明确的逻辑依赖关系，构成完整的监控对象模型，核心术语的定义与语义映射如下：

### 1.5.1 Host（主机）

- 严谨定义：Zabbix 监控体系中的 **核心监控载体** ，是所有监控配置的挂载对象，对应物理服务器、虚拟机、交换机、路由器、容器、应用实例等被监控对象；
- 核心属性：包含主机名称、IP 地址、所属主机组、关联的模板、代理节点、权限配置等元数据；
- 语义定位：监控体系的一级对象，所有监控项、触发器等二级对象均必须挂载在 Host 之上。

### 1.5.2 Item（监控项）

- 严谨定义：挂载在 Host 之上的 **最小监控单元** ，定义了单一指标的采集规则，是 Zabbix 数据采集的核心载体；
- 核心属性：包含采集 Key、采集周期、数据类型、存储策略、历史数据保留周期、所属主机 / 模板等配置；
- 语义定位：监控体系的最小数据单元，所有的分析、告警、可视化均基于 Item 采集到的指标数据实现，一个 Host 可挂载多个 Item。

### 1.5.3 Trigger（触发器）

- 严谨定义：基于 Item 采集数据的 **异常判断规则** ，通过逻辑表达式定义指标的异常阈值与判断条件，是 Zabbix 异常事件生成的核心依据；
- 核心属性：包含触发器表达式、异常等级、依赖关系、所属主机 / 模板等配置；
- 语义定位：监控体系的异常决策单元，基于 Item 的时序数据生成异常事件，一个 Trigger 可关联一个或多个 Item，一个 Item 可被多个 Trigger 关联。

### 1.5.4 Action（动作）

- 严谨定义：基于 Trigger 生成的异常事件的 **处置执行规则**，定义了异常事件触发后的全流程处置逻辑，是 Zabbix 告警闭环与自动化处置的核心载体；
- 核心属性：包含触发条件、告警操作、告警升级策略、恢复操作、远程命令执行规则等配置；
- 语义定位：监控体系的处置执行单元，基于 Trigger 生成的事件触发告警通知与自动处置，是实现告警闭环的核心环节。

### 1.5.5 Template（模板）

- 严谨定义：Zabbix 监控配置的 **标准化复用单元**，是一组预定义的 Item、Trigger、图形、发现规则、动作等监控配置的集合；
- 核心属性：包含模板名称、关联的主机 / 子模板、继承关系、宏定义、监控配置集合等元数据；
- 语义定位：监控体系的配置复用核心，解决大规模监控场景下配置重复开发、标准化程度低、运维成本高的问题，模板可直接关联到多个 Host，实现监控配置的批量下发与统一管理。

### 1.5.6 核心术语的逻辑链路

完整的监控逻辑链路为：**Template → Host → Item → Trigger → Action**，即通过模板实现监控配置的标准化复用，将模板关联到主机，主机通过监控项采集指标数据，触发器基于指标数据判断异常，动作基于异常事件触发告警与处置，形成完整的监控闭环。
