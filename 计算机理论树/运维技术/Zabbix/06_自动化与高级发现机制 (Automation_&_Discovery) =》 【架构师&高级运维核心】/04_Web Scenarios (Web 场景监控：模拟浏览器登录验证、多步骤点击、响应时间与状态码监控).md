
## 6.5 Web 场景监控（Web Scenarios）

### 6.5.1 核心定义与工作原理

Web 场景监控是 Zabbix 原生内置的、无代理的 Web 业务全流程可用性监控能力，支持多步骤 HTTP/HTTPS 请求链式执行，模拟用户浏览器的访问行为与业务操作流程，实现对 Web 网站、API 接口、Web 业务系统的全链路可用性、响应性能、内容正确性的监控，是业务层监控的核心能力。

核心工作原理：Zabbix Server/Proxy 的 HTTP Poller 进程，按照 Web 场景预设的更新周期，依次执行场景内的所有步骤，每一步发送对应的 HTTP 请求，自动保持 Cookie 与会话状态，执行预设的校验规则；任何一步校验失败，整个场景标记为失败，触发对应告警；所有步骤执行完成后，生成全流程与单步骤的性能指标，实现从可用性到性能的全维度监控。

### 6.5.2 核心能力与适用场景

#### 核心能力

1. 多步骤链式请求：支持无限级步骤配置，模拟用户完整的业务操作流程，如 “访问首页→用户登录→浏览商品→提交订单→退出登录”；
2. 全 HTTP 协议支持：原生支持 GET/POST/PUT/DELETE/PATCH 等所有标准 HTTP 方法，支持自定义请求头、请求体、Cookie、表单提交；
3. 多维度校验能力：支持响应状态码校验、响应内容关键字校验（必须包含 / 必须不包含）、响应时间阈值校验、SSL 证书有效期校验；
4. 会话保持能力：自动管理 Cookie 与会话状态，步骤之间共享会话，完美模拟用户登录后的业务操作；
5. 全认证体系兼容：原生支持 Basic Auth、Digest Auth、NTLM Auth、OAuth2 Bearer Token、表单登录等所有主流认证方式；
6. 动态变量传递：支持从响应内容中提取变量，传递到后续步骤的请求中，实现带动态参数的 API 链式调用；
7. 代理与 SSL 适配：支持 HTTP/HTTPS/SOCKS 代理，适配跨网络隔离场景；原生支持 SSL 证书校验、客户端证书认证。

#### 核心适用场景

1. Web 业务系统全流程可用性监控，模拟用户真实操作，确保核心业务流程正常可用；
2. API 接口的链式调用监控，如微服务接口的全链路调用可用性与性能监控；
3. 网站页面可用性与响应性能监控，包括静态页面、动态页面；
4. SSL/TLS 证书有效期监控，提前告警证书即将过期，避免业务中断；
5. 网站内容篡改检测，校验页面核心关键字，避免页面被篡改或挂马；
6. 第三方接口可用性监控，如支付接口、短信接口、地图接口等第三方依赖服务的监控。

### 6.5.3 核心配置体系

Web 场景监控分为两大核心配置模块：场景全局配置、步骤配置，同时原生生成对应的监控项与触发器，无需手动创建。

#### 6.5.3.1 场景全局配置

定义 Web 场景的全局规则，适用于场景内的所有步骤，关键配置项如下：

表格

|配置项|核心作用|生产环境配置规范|
|---|---|---|
|名称|Web 场景的语义化标识|必须明确监控的业务流程，如 “电商平台用户下单全流程监控”|
|主机|场景所属的主机|通常为 Web 业务系统对应的主机，也可使用专用的 Web 监控主机|
|更新周期|场景的执行周期|核心业务流程 1-5min，非核心业务 5-15min，禁止 < 1min 的高频执行|
|重试次数|单步骤执行失败后的重试次数|建议 1-2 次，避免网络波动导致的误告警|
|客户端类型|HTTP 请求的 User-Agent|模拟浏览器 / 移动端客户端，避免被 WAF 拦截|
|SSL 验证|是否开启 SSL 证书校验|生产环境必须开启，避免中间人攻击|
|跟随重定向|是否自动跟随 HTTP 3xx 重定向|生产环境建议开启|
|HTTP 代理|场景使用的代理服务器地址|适配跨网络隔离场景|

#### 6.5.3.2 步骤配置

步骤是 Web 场景的核心执行单元，定义了单个 HTTP 请求的规则与校验逻辑，步骤按照序号顺序执行，前一步执行成功后才会执行下一步，关键配置如下：

1. **基础请求配置**
    
    - 步骤名称：语义化名称，明确对应的用户操作，如 “用户登录请求”；
    - URL：请求的目标 URL，必须包含 http/https 协议头；
    - 请求方法：GET/POST/PUT/DELETE 等标准 HTTP 方法；
    - 超时时间：单步骤请求的最大超时时间，生产环境建议 10-30s；
    - 跟随重定向：单独为该步骤配置重定向跟随规则，默认继承全局配置。
    
2. **请求内容配置**
    
    - 请求头：自定义 HTTP 请求头，如`Content-Type`、`Authorization`、`Accept`等；
    - 表单字段：模拟表单提交的键值对，用于用户登录等场景，自动编码为`application/x-www-form-urlencoded`格式；
    - 请求体：POST/PUT 请求的内容，支持 JSON、XML、表单格式，支持变量填充。
    
3. **核心校验规则（生产环境必须配置）**
    
    所有规则都满足才判定为步骤执行成功，支持多规则组合：
    
    - 响应状态码：要求的 HTTP 状态码，如`200`、`200,302`，支持多个状态码逗号分隔；
    - 必需字符串：响应内容中必须包含的字符串，用于校验页面内容正确性，是避免页面报错但状态码 200 的核心配置；
    - 必需不包含字符串：响应内容中必须不包含的字符串，用于校验错误信息，如 “系统异常”、“登录失败”；
    - 响应时间阈值：步骤的最大响应时间，超过阈值则判定为失败，用于性能异常告警。
    
4. **变量提取配置**
    
    支持从响应头、响应内容、Cookie 中提取变量，使用正则表达式或 JSONPath 提取，保存为场景内的全局变量，可在后续步骤的请求头、请求体、URL 中使用，实现动态参数传递。

#### 6.5.3.3 原生自动生成的监控项

Web 场景创建后，Zabbix 会自动生成对应的监控项，无需手动创建，核心监控项包括：

1. 场景级核心监控项：
    
    - `web.test.fail[场景名称]`：场景执行失败的步骤编号，0 为全部成功，非 0 为对应步骤失败，是核心告警监控项；
    - `web.test.error[场景名称]`：场景执行的错误信息，无错误则为空；
    - `web.test.time[场景名称]`：场景全流程的总执行时间，单位为秒；
    
2. 步骤级监控项：
    
    - `web.test.rspcode[场景名称,步骤名称]`：对应步骤的响应状态码；
    - `web.test.time[场景名称,步骤名称]`：对应步骤的执行时间，单位为秒；
    
3. SSL 证书监控项：针对 HTTPS URL，自动生成 SSL 证书剩余有效期监控项，单位为天，可用于证书过期告警。

#### 6.5.3.4 标准化触发器配置

基于自动生成的监控项，可创建对应的触发器实现异常告警，核心标准示例如下：

1. **业务流程执行失败告警**
    
    表达式：`last("web.test.fail[电商平台下单流程]") <> 0`
    
    严重性：High
    
    含义：场景执行失败，任意步骤校验不通过，触发告警。
2. **全流程响应超时告警**
    
    表达式：`avg("web.test.time[电商平台下单流程]",3m) > 5`
    
    严重性：Warning
    
    含义：最近 3 分钟内，场景总执行时间平均超过 5 秒，触发性能告警。
3. **SSL 证书即将过期告警**
    
    表达式：`last("web.test.instances[电商平台首页,ssl_cert_expire_days]") < 30`
    
    严重性：Warning
    
    含义：SSL 证书剩余有效期不足 30 天，提前触发告警。
4. **单步骤响应超时告警**
    
    表达式：`max("web.test.time[电商平台下单流程,提交订单]",5m) > 3`
    
    严重性：Warning
    
    含义：最近 5 分钟内，提交订单步骤的最大响应时间超过 3 秒，触发性能告警。

### 6.5.4 进阶实战场景

#### 6.5.4.1 带表单登录的 Web 业务全流程监控

**场景**：模拟用户登录电商平台，浏览商品，提交订单的完整业务流程，核心步骤配置：

1. 步骤 1：访问平台首页
    
    - URL：`https://mall.example.com/`
    - 方法：GET
    - 校验：状态码 200，必需包含字符串 “电商平台首页”
    
2. 步骤 2：用户登录提交
    
    - URL：`https://mall.example.com/login`
    - 方法：POST
    - 表单字段：`username=test_user`、`password=TestPass123`
    - 校验：状态码 302，必需包含字符串 “登录成功”，开启跟随重定向
    
3. 步骤 3：访问商品详情页
    
    - URL：`https://mall.example.com/goods/123`
    - 方法：GET
    - 校验：状态码 200，必需包含字符串 “商品详情”
    
4. 步骤 4：提交订单请求
    
    - URL：`https://mall.example.com/order/submit`
    - 方法：POST
    - 请求体：`{"goods_id":123,"num":1}`
    - 请求头：`Content-Type: application/json`
    - 校验：状态码 200，必需包含字符串 “订单提交成功”
    
5. 步骤 5：退出登录
    
    - URL：`https://mall.example.com/logout`
    - 方法：GET
    - 校验：状态码 200，必需包含字符串 “已退出登录”
    

整个场景自动保持会话 Cookie，模拟用户完整操作流程，任何一步失败都会触发告警，确保核心业务流程可用。

#### 6.5.4.2 带 Token 认证的 API 链式调用监控

**场景**：监控微服务 API 接口的链式调用，从登录接口获取 Token，传递到后续业务接口的请求头中，核心步骤配置：

1. 步骤 1：获取 Access Token
    
    - URL：`https://api.example.com/auth/login`
    - 方法：POST
    - 请求体：`{"username":"api_user","password":"ApiPass123"}`
    - 请求头：`Content-Type: application/json`
    - 变量提取：使用 JSONPath `$.data.access_token` 提取 Token，保存为变量`{ACCESS_TOKEN}`
    - 校验：状态码 200，必需包含字符串`access_token`
    
2. 步骤 2：调用用户信息接口
    
    - URL：`https://api.example.com/user/info`
    - 方法：GET
    - 请求头：`Authorization: Bearer {ACCESS_TOKEN}`
    - 校验：状态码 200，必需包含字符串`username`
    
3. 步骤 3：调用订单列表接口
    
    - URL：`https://api.example.com/order/list`
    - 方法：GET
    - 请求头：`Authorization: Bearer {ACCESS_TOKEN}`
    - 校验：状态码 200，必需包含字符串`order_id`
    

通过变量提取与传递，实现带动态 Token 的 API 全链路监控，确保微服务接口的可用性与正确性。

### 6.5.5 生产环境最佳实践与安全约束

1. **内容校验强制配置**：所有步骤必须配置内容校验规则，禁止仅校验响应状态码，避免页面报错、内容篡改但状态码 200 导致的漏监控；
2. **监控账号最小权限**：Web 场景使用的登录账号，必须配置最小权限的测试账号，禁止使用管理员账号，避免对生产数据造成影响；
3. **敏感信息保护**：请求体、表单中的密码、API 密钥等敏感信息，使用 Zabbix 用户自定义加密宏存储，禁止明文写入配置；
4. **白名单配置**：在 Web 业务系统的 WAF、防火墙中，将 Zabbix Server/Proxy 的 IP 地址加入白名单，避免被误拦截为爬虫或攻击流量；
5. **步骤拆分规范**：每个用户操作对应一个独立步骤，禁止一个步骤中包含多个业务操作，便于故障定位，明确异常环节；
6. **性能阈值合理设置**：响应时间阈值基于业务正常性能设置，避免阈值过高导致的性能异常漏告警，或阈值过低导致的误告警；
7. **SSL 证书校验强制开启**：生产环境必须开启 SSL 证书校验，禁止关闭校验导致的中间人攻击风险。

### 6.5.6 常见故障排查

1. 步骤执行失败，但手动访问 URL 正常：核心原因是 Zabbix 的请求被 WAF / 防火墙拦截，或 User-Agent 被限制，检查服务器访问日志，配置 IP 白名单；
2. 登录步骤成功，但后续步骤提示未登录：核心原因是会话 Cookie 未正常保持，检查是否开启了跟随重定向，或步骤之间的会话共享配置；
3. 响应内容包含必需字符串，但步骤校验失败：核心原因是字符串大小写不匹配，或字符串包含特殊字符，或响应内容是 JS 动态渲染的（Zabbix 无法执行 JS，仅能获取原始 HTML 内容）；
4. SSL 证书校验失败：核心原因是 SSL 证书过期、证书链不完整、域名不匹配，或使用了自签名证书，需导入 CA 证书到 Zabbix Server 的信任库；
5. 变量提取失败：核心原因是 JSONPath / 正则表达式错误，或响应内容格式不符合预期，手动验证响应内容与提取规则的匹配性。
