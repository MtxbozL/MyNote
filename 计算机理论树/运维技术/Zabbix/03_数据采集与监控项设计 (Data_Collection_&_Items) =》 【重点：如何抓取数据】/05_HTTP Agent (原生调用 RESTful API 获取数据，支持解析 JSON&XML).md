
## 3.6 HTTP Agent

### 3.6.1 核心定义与原理

HTTP Agent 是 Zabbix 原生内置的**HTTP/HTTPS 协议采集能力**，无需额外插件、脚本或代理，可直接调用 RESTful API、Web 接口，获取 JSON/XML/HTML 格式的响应数据，并通过原生解析能力提取目标指标，是 Web 服务、API 接口、云服务、物联网设备监控的核心无代理方案。

工作原理：Zabbix Server/Proxy 的`HTTP Poller`进程按照监控项配置的周期，向目标 URL 发送 HTTP 请求，接收并校验响应数据，按照预设的解析规则（JSONPath/XPath/ 正则）提取目标指标值，将结果存入数据库，整个流程完全原生实现，无外部依赖。

### 3.6.2 核心特性与适用场景

#### 3.6.2.1 核心特性

- 原生支持 GET/POST/PUT/DELETE/PATCH 等所有标准 HTTP 方法；
- 支持自定义请求头、请求体、Cookie、代理服务器、跟随重定向；
- 原生支持多种认证方式：Basic Auth、Digest Auth、Bearer Token、API Key、NTLM 认证；
- 原生支持 SSL/TLS，可配置 SSL 证书校验、客户端证书认证；
- 内置 JSONPath、XPath 解析引擎，可直接从 JSON/XML 响应中提取嵌套数据，无需额外脚本；
- 支持将响应状态码、响应时间、响应大小作为独立指标提取。

#### 3.6.2.2 核心适用场景

1. RESTful API 接口监控：接口可用性、响应时间、返回码校验、业务指标提取；
2. Web 服务健康检查：页面可用性、响应时间、页面内容一致性校验；
3. 云服务 API 指标采集：云厂商资源监控 API、账单 API、业务开放 API 的数据获取；
4. 物联网设备监控：支持 HTTP 接口的 IoT 设备的指标采集与状态监控；
5. 第三方业务系统监控：提供 HTTP API 的业务系统、SaaS 服务的指标对接。

### 3.6.3 核心配置规范

HTTP Agent 监控项的核心配置分为六大模块，配置要点如下：

1. **基础通信配置**：选择监控项类型为 “HTTP Agent”，配置目标 URL、HTTP 方法、更新周期、超时时间、重试次数；
2. **请求配置**：自定义请求头、请求体（支持 JSON / 表单 / XML 格式）、Cookie、跟随重定向次数、代理服务器地址；
3. **认证配置**：选择认证类型，配置对应的认证参数，敏感认证信息需通过 Zabbix 宏存储，禁止明文写入配置；
4. **SSL 配置**：配置 SSL 证书校验规则、客户端证书、CA 证书，生产环境必须开启 SSL 证书校验；
5. **响应解析配置**：根据响应数据格式选择解析方式：
    
    - JSON 响应：使用 JSONPath 表达式提取目标值，示例：`$.data.total`、`$.items[0].value`；
    - XML/HTML 响应：使用 XPath 表达式提取目标值；
    - 文本响应：使用正则表达式提取目标值；
    
6. **数据类型配置**：配置提取结果的数据类型，与解析得到的数值类型匹配。

### 3.6.4 进阶用法与最佳实践

1. **多指标批量提取**：通过 HTTP Agent 主请求获取响应数据，结合相关依赖项，从同一份响应中提取多个指标，避免重复发送 HTTP 请求，降低目标接口压力；
2. **会话保持**：通过 Cookie 存储实现多步骤会话保持，模拟用户登录后的接口访问，结合 Web 场景监控实现复杂业务流程监控；
3. **安全最佳实践**：生产环境必须使用 HTTPS 协议，避免敏感信息明文传输；API 密钥、认证密码通过用户自定义宏存储，开启宏加密，避免泄露；
4. **性能约束**：合理配置请求频率与超时时间，避免对目标接口造成过大压力，大规模使用时需调整 Server 的`StartHTTPPollers`参数，避免请求阻塞。
