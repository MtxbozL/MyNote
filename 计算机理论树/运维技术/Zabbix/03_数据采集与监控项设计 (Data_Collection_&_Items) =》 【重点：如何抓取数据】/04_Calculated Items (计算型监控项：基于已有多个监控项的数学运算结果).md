## 3.5 Calculated Items（计算型监控项）

### 3.5.1 核心定义与原理

计算型监控项是 Zabbix 原生支持的 **衍生指标构建能力** ，基于已有一个或多个监控项的历史数据，通过数学表达式、逻辑运算生成新的监控指标，所有计算逻辑完全在 Zabbix Server 端执行，无需额外脚本或采集组件，是复合指标、聚合指标、业务指标构建的核心方式。

工作原理：Server 按照监控项配置的更新周期，读取表达式中引用的监控项的历史数据，执行预设的计算逻辑，将计算结果作为该监控项的数值存入数据库，计算生成的指标与普通采集指标完全等效，可用于触发器、图形、仪表盘、告警等所有场景。

### 3.5.2 表达式语法规范

#### 3.5.2.1 基础语法格式

计算型监控项表达式的核心单元为监控项引用函数，标准格式为：

```go
func(<host>,"<key>",<parameter>)
```

- `func`：Zabbix 内置的数值统计函数，支持`last`、`avg`、`max`、`min`、`sum`、`count`等，与触发器表达式函数体系完全兼容（详见 4.2 节）；
- `<host>`：引用的监控项所属的主机名，引用当前主机的监控项可省略主机名；
- `<key>`：引用的监控项 Key 名称，必须用双引号包裹；
- `<parameter>`：函数的参数，通常为统计时间窗口，例如`5m`（5 分钟）、`1h`（1 小时）。

#### 3.5.2.2 运算能力

- 支持加减乘除四则运算、括号优先级、取模、幂运算等数学运算；
- 支持大于、小于、等于、不等于等比较运算，返回 1（真）或 0（假）；
- 支持 AND、OR 等逻辑运算，适用于状态类复合指标；
- 支持多主机、多监控项的联合计算，可实现跨主机的集群聚合指标。

### 3.5.3 核心适用场景与示例

1. **基础衍生指标计算**
    
    示例：CPU 空闲率计算，基于 CPU 利用率指标生成
    
    ```shell
    100 - last("system.cpu.util[,idle]")
    ```
    
    示例：磁盘使用率计算，基于磁盘总容量与已用容量生成
    
    plaintext
    
    ```
    (last("vfs.fs.size[/,used]") / last("vfs.fs.size[/,total]")) * 100
    ```
    
2. **业务成功率指标计算**
    
    示例：接口请求成功率，基于成功请求数与总请求数生成
    
    plaintext
    
    ```
    (last("api.request.success") / last("api.request.total")) * 100
    ```
    
3. **集群聚合指标计算**
    
    示例：集群总出口流量，跨多台服务器的网卡流量聚合
    
    plaintext
    
    ```
    last("web-server-01","net.if.out[eth0,bytes]") + last("web-server-02","net.if.out[eth0,bytes]") + last("web-server-03","net.if.out[eth0,bytes]")
    ```
    
4. **趋势衍生指标计算**
    
    示例：5 分钟内磁盘空间增长率
    
    plaintext
    
    ```
    (last("vfs.fs.size[/,used]") - avg("vfs.fs.size[/,used]",5m,1)) / avg("vfs.fs.size[/,used]",5m,1) * 100
    ```
    

### 3.5.4 配置约束与注意事项

1. 监控项类型必须选择 “Calculated”，数据类型必须与计算结果的类型严格匹配；
2. 禁止循环引用，计算型监控项的表达式中不得引用自身，避免无限循环计算；
3. 避免使用过于复杂的嵌套表达式与大规模跨主机引用，防止 Server 计算性能过载；
4. 表达式中引用的监控项若没有有效数据，计算结果将变为 “不支持” 状态，需做好异常处理与兜底逻辑；
5. 更新周期需与引用的监控项的采集周期匹配，避免计算结果出现时间偏差。
