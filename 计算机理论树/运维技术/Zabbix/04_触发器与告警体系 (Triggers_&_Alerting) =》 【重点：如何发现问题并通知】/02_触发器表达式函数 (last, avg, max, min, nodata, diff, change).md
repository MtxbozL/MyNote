
## 4.3 触发器表达式与函数体系

触发器表达式是触发器的核心灵魂，是异常判断逻辑的标准化载体，Zabbix 提供完整的表达式语法体系与内置函数库，支持多维度、多场景的异常判断逻辑构建。

### 4.3.1 表达式基础语法规范

#### 4.3.1.1 核心语法结构

触发器表达式的最小执行单元为**函数调用语句**，标准格式为：

plaintext

```
<function>(<host>,"<key>",<parameter1>,<parameter2>,...)
```

- 与计算型监控项函数语法完全兼容，`function`为 Zabbix 内置触发器函数；
- `<host>`为关联监控项所属主机名，引用当前主机的监控项可省略主机名；
- `<key>`为关联监控项的 Key 名称，必须用双引号包裹，大小写敏感；
- `<parameter>`为函数入参，用于定义统计时间窗口、偏移量、阈值等参数。

完整的触发器表达式由多个函数调用语句通过**运算符**连接构成，最终计算结果为布尔值（TRUE/FALSE），示例：

plaintext

```
avg("system.cpu.util[,user]",5m) > 80 and max("system.load.1",3m) > 10
```

- 表达式结果为 TRUE 时，触发器切换为 PROBLEM 状态；
- 表达式结果为 FALSE 时，触发器保持 / 切换为 OK 状态。

#### 4.3.1.2 运算符与优先级

Zabbix 触发器表达式支持完整的运算体系，优先级从高到低排序如下：

1. 括号`()`：用于调整运算优先级，强制优先执行括号内的逻辑；
2. 一元运算符：`-`（负号）、`not`（逻辑非）；
3. 乘法类运算符：`*`（乘）、`/`（除）、`mod`（取模）；
4. 加法类运算符：`+`（加）、`-`（减）；
5. 关系运算符：`<`（小于）、`>`（大于）、`<=`（小于等于）、`>=`（大于等于）；
6. 相等性运算符：`=`（等于）、`<>`（不等于）；
7. 逻辑与：`and`；
8. 逻辑或：`or`。

**生产环境硬性规范**：复杂表达式必须使用括号明确运算优先级，禁止依赖默认优先级顺序，避免逻辑歧义导致的误触发 / 漏触发。

#### 4.3.1.3 时间窗口参数规范

时间窗口是触发器函数的核心入参，用于定义指标统计的时间范围，支持两种标准化格式：

1. **相对时间格式**：使用时间单位后缀，`s`（秒）、`m`（分钟）、`h`（小时）、`d`（天）、`w`（周），示例：`5m`（最近 5 分钟）、`1h`（最近 1 小时）；
2. **数值格式**：纯数字，代表监控项采集周期的个数，示例：监控项采集周期为 30s，参数`5`代表最近 5 个采集周期（150s）。

**生产环境最佳实践**：优先使用相对时间格式，避免因监控项采集周期变更导致的统计逻辑异常，保证表达式的稳定性。

### 4.3.2 核心内置函数分类与详解

Zabbix 触发器函数分为六大类，覆盖全场景异常判断需求，核心高频函数的语法、功能与适用场景如下：

#### 4.3.2.1 瞬时值获取类函数

用于获取监控项的最新采集值，适用于硬阈值类异常判断，核心函数：

1. **last()**
    
    - 标准语法：`last(<host>,"<key>",<#offset>,<time_shift>)`
    - 核心功能：获取监控项的最新采集值，`#offset`参数用于获取历史偏移值，示例`#2`代表倒数第二个采集值；`time_shift`参数用于获取历史时间点的值，示例`1d`代表 24 小时前的最新值。
    - 高频示例：`last("vfs.fs.size[/,pused]") > 90`，磁盘使用率最新值超过 90% 触发异常。
    - 适用场景：硬阈值告警、状态类指标（端口监听状态、进程存活状态）判断。
    
2. **prev()**
    
    - 标准语法：`prev(<host>,"<key>")`
    - 核心功能：获取监控项的上一个采集值，等价于`last(#2)`。
    - 高频示例：`prev("net.if.out[eth0,bytes]")`，用于流量变化对比。
    - 适用场景：指标变化量判断、相邻周期数据对比。
    

#### 4.3.2.2 统计聚合类函数

用于计算指定时间窗口内监控项的聚合统计值，是避免指标抖动误告警的核心函数，生产环境优先使用，核心函数：

1. **avg()**
    
    - 标准语法：`avg(<host>,"<key>",<time_window>,<time_shift>)`
    - 核心功能：计算指定时间窗口内监控项的算术平均值。
    - 高频示例：`avg("system.cpu.util[,user]",5m) > 85`，最近 5 分钟 CPU 平均使用率超过 85% 触发异常。
    - 适用场景：资源使用率告警、性能指标异常判断，避免单次峰值导致的误告警。
    
2. **max()/min()**
    
    - 标准语法：`max(<host>,"<key>",<time_window>,<time_shift>)` / `min(...)`
    - 核心功能：计算指定时间窗口内监控项的最大值 / 最小值。
    - 高频示例：`min("icmppingsec",10m) > 0.5`，最近 10 分钟内 ICMP 延迟最小值超过 500ms，触发链路质量异常。
    - 适用场景：峰值 / 谷值判断、最低性能保障、链路质量监控。
    
3. **sum()**
    
    - 标准语法：`sum(<host>,"<key>",<time_window>,<time_shift>)`
    - 核心功能：计算指定时间窗口内监控项的数值总和。
    - 高频示例：`sum("api.request.error",5m) > 100`，最近 5 分钟内 API 错误请求总数超过 100 触发异常。
    - 适用场景：累计值告警、错误总数统计、业务量异常判断。
    
4. **count()**
    
    - 标准语法：`count(<host>,"<key>",<time_window>,<pattern>,<operator>)`
    - 核心功能：统计指定时间窗口内，满足指定条件的采集值个数。
    - 高频示例：`count("icmpping",3m,"0","eq") > 2`，最近 3 分钟内 ICMP 不可达的次数超过 2 次，触发主机存活异常。
    - 适用场景：异常次数统计、抖动判断、可用性指标计算。
    

#### 4.3.2.3 时间与存活类函数

用于监控数据采集的连续性、主机存活状态、无数据场景判断，核心函数：

1. **nodata()**
    
    - 标准语法：`nodata(<host>,"<key>",<time_window>)`
    - 核心功能：判断指定时间窗口内是否无监控数据采集，返回 1（无数据）/0（有数据）。
    - 高频示例：`nodata("agent.ping",5m) = 1`，最近 5 分钟内无 Agent 心跳数据，触发 Agent 离线异常。
    - 适用场景：Agent 存活监控、采集中断判断、设备离线告警，是生产环境必配的基础监控项。
    
2. **date()/time()/now()**
    
    - 核心功能：分别返回当前日期、当前时间、当前时间戳，用于时间相关的条件判断。
    - 高频示例：`avg("system.cpu.util",5m) > 90 and time() between 09:00:00 and 18:00:00`，仅在工作时间触发 CPU 高负载告警。
    - 适用场景：分时段告警阈值调整、业务高峰期专属告警规则。
    

#### 4.3.2.4 变化与差异类函数

用于判断监控项数值的变化量、变化率、状态切换，适用于突变类异常、状态变更类场景，核心函数：

1. **diff()**
    
    - 标准语法：`diff(<host>,"<key>")`
    - 核心功能：对比最新采集值与上一个采集值，返回 1（值不同）/0（值相同）。
    - 高频示例：`diff("system.uname") = 1`，系统内核版本发生变化，触发配置变更告警。
    - 适用场景：配置变更检测、系统版本变更、关键参数修改告警。
    
2. **change()**
    
    - 标准语法：`change(<host>,"<key>")`
    - 核心功能：计算最新采集值与上一个采集值的绝对差值。
    - 高频示例：`abs(change("vfs.fs.size[/,used]")) > 10G`，单次采集周期内磁盘使用量变化超过 10GB，触发异常。
    - 适用场景：指标突变检测、磁盘突增 / 突降、流量突发告警。
    
3. **delta()**
    
    - 标准语法：`delta(<host>,"<key>",<time_window>)`
    - 核心功能：计算指定时间窗口内，监控项最大值与最小值的差值。
    - 高频示例：`delta("mysql.connections",10m) > 500`，10 分钟内数据库连接数变化超过 500，触发连接突增异常。
    - 适用场景：长周期指标波动检测、业务量突变判断。
    

#### 4.3.2.5 字符串与正则类函数

用于字符型监控项的内容匹配、关键字检测，适用于日志监控、版本号校验、状态文本判断，核心函数：

1. **str()/regexp()**
    
    - 标准语法：`str(<host>,"<key>",<pattern>,<time_shift>)` / `regexp(...)`
    - 核心功能：`str()`用于检测监控项值中是否包含指定字符串，`regexp()`支持正则表达式匹配，匹配成功返回 1，失败返回 0。
    - 高频示例：`str("system.hostname","prod") = 0`，主机名不包含 prod 关键字，触发环境配置异常。
    - 适用场景：日志关键字告警、配置合规性检测、文本内容校验。
    
2. **strlen()**
    
    - 标准语法：`strlen(<host>,"<key>")`
    - 核心功能：返回监控项字符串值的长度。
    - 高频示例：`strlen("api.response") = 0`，API 响应内容为空，触发接口异常。
    - 适用场景：空值检测、响应内容完整性校验。
    

#### 4.3.2.6 逻辑判断类函数

用于多条件复合判断、异常场景的逻辑组合，核心函数：

1. **abs()**：返回数值的绝对值，用于变化量的正负无关判断；
2. **bitand()/bitor()**：位运算函数，适用于状态位、标志位的复合判断；
3. **fuzzytime()**：检测监控项时间戳与系统时间的偏差，适用于时间同步校验。

### 4.3.3 恢复表达式与防抖动设计

告警抖动（Flapping）是生产环境的高频痛点，指监控指标在阈值附近频繁波动，导致触发器在 OK 与 PROBLEM 状态之间反复切换，产生大量重复告警。Zabbix 通过**恢复表达式**机制实现阈值分离，从根本上解决告警抖动问题。

#### 4.3.3.1 恢复表达式核心原理

恢复表达式是独立于触发表达式的恢复判断逻辑，仅当触发器处于 PROBLEM 状态时生效，实现 “高阈值触发，低阈值恢复” 的滞后判断机制。

标准设计示例：

- 触发表达式：`avg("system.cpu.util[,user]",5m) > 85`（5 分钟平均 CPU 使用率超过 85% 触发异常）
- 恢复表达式：`avg("system.cpu.util[,user]",5m) < 75`（5 分钟平均 CPU 使用率低于 75% 才恢复正常）

通过 10% 的阈值缓冲带，避免指标在 85% 附近频繁波动导致的告警抖动，是生产环境的硬性规范。

#### 4.3.3.2 生产环境防抖动最佳实践

1. 所有数值型指标的触发器必须配置恢复表达式，禁止使用与触发表达式完全相反的恢复逻辑；
2. 优先使用时间窗口聚合函数（avg ()/max ()/min ()），禁止仅使用 last () 单次值作为触发条件，避免指标瞬时峰值导致的误告警；
3. 对于抖动性强的指标（网络流量、CPU 使用率），使用 count () 函数实现 “N 个周期内 M 次满足条件才触发” 的逻辑，示例：`count("system.cpu.util",5m,"85","gt") >= 3`，5 分钟内有 3 个采集周期 CPU 使用率超过 85% 才触发告警；
4. 配置异常持续时间参数，仅当异常状态持续指定时间后才生成 PROBLEM 事件，进一步过滤瞬时抖动。
