## 一、开篇：流程控制的核心定义与分类

### 1. 什么是 MySQL 流程控制

MySQL 流程控制，是用于**在 SQL 中实现逻辑判断、循环执行、分支跳转、异常处理**的语法体系，是把 SQL 从「单条查询语句」升级为「复杂业务逻辑程序」的核心能力。

### 2. 两大分类（必须先分清，90% 的新手报错都源于此）

|分类|适用场景|可使用位置|核心作用|
|:--|:--|:--|:--|
|**查询级流程控制函数**|日常单条 SQL 查询、数据处理|普通 SELECT/UPDATE/INSERT 语句中均可使用|实现简单的二分支、多分支判断、空值处理|
|**程序级流程控制语句**|复杂业务逻辑、批量数据处理|仅能在「存储过程 / 存储函数 / 触发器 / 事件」的`BEGIN...END`代码块中使用|实现完整的分支、循环、跳转、异常处理，类似编程语言的逻辑控制|

> 注意：之前常用函数里的`IF()`、`CASE WHEN`，属于第一类「查询级函数」，普通 SQL 可直接用；而`IF...THEN`、`WHILE`循环等，属于第二类「程序级语句」，必须在存储程序中使用。

---

## 二、第一部分：查询级流程控制函数（高频日常使用）

这部分是日常开发中 90% 场景会用到的，普通 SQL 语句可直接使用，无需存储过程，衔接之前的常用函数笔记，做系统深化。

### 2.1 IF () 函数：二分支判断

#### 核心功能

实现简单的「如果... 就... 否则...」逻辑，类似编程语言中的三元表达式。

#### 语法

```sql
IF(判断条件, 条件成立返回值, 条件不成立返回值)
```

#### 示例（基于 order_info 订单表）

```sql
-- 需求：判断订单是否已支付，返回对应状态说明
SELECT 
  order_id, order_no, order_amount,
  IF(order_status = '已支付', '交易完成', '待处理') AS order_desc
FROM order_info;

-- 进阶：嵌套IF，实现三分支
SELECT 
  order_id, order_amount,
  IF(order_amount > 100, '高客单', IF(order_amount > 50, '中客单', '低客单')) AS amount_level
FROM order_info;
```

#### 注意事项

- 条件成立的判断规则：条件返回 TRUE（非 0、非 NULL）则走第一个值；返回 FALSE 或 NULL 则走第二个值；
- 嵌套 IF 最多支持 255 层，但超过 3 层建议用 CASE WHEN，可读性更好。

---

### 2.2 IFNULL () 函数：空值兜底处理

#### 核心功能

专门处理 NULL 值，给 NULL 值设置默认兜底值，避免 NULL 值导致的计算异常。

#### 语法

```sql
IFNULL(待判断的值, 兜底默认值)
```

#### 示例（基于 user_info 用户表）

```sql
-- 需求：用户邮箱为NULL时，显示「未填写」
SELECT 
  user_id, user_name,
  IFNULL(email, '未填写') AS email,
  IFNULL(age, 0) AS age
FROM user_info;

-- 进阶：避免拼接字符串时出现NULL
SELECT CONCAT(user_name, '：', IFNULL(phone, '无手机号')) AS user_contact FROM user_info;
```

#### 关键区别：IFNULL () vs IS NULL

- `IS NULL`：是条件运算符，用于 WHERE 子句中过滤 NULL 值，返回布尔值；
- `IFNULL()`：是函数，用于给 NULL 值替换默认值，返回替换后的结果。

---

### 2.3 NULLIF () 函数：等值判断返回 NULL

#### 核心功能

判断两个值是否相等，相等则返回 NULL，不相等则返回第一个值，常用于避免「除零错误」等异常场景。

#### 语法

```sql
NULLIF(value1, value2)
```

#### 示例

```sql
-- 核心场景：避免除零错误
SELECT 
  order_id, 
  order_amount / NULLIF(paid_amount, 0) AS discount_rate -- 实付金额为0时，返回NULL，不会报除零错误
FROM order_info;

-- 基础用法：两个值相等返回NULL
SELECT NULLIF('张三', '张三') AS result; -- 返回NULL
SELECT NULLIF('张三', '李四') AS result; -- 返回张三
```

---

### 2.4 CASE 表达式：多分支判断（最灵活）

#### 核心功能

实现多分支逻辑判断，类似编程语言中的`if-else if-else`或`switch-case`，可读性强，支持复杂条件，是多分支场景的首选。

#### 两种写法

##### 写法 1：简单 CASE（等值判断，类似 switch-case）

###### 语法

```sql
CASE 待判断的表达式
  WHEN 匹配值1 THEN 返回结果1
  WHEN 匹配值2 THEN 返回结果2
  ...
  ELSE 兜底默认值 -- 可选，不写的话无匹配返回NULL
END
```

###### 示例

```sql
-- 需求：把订单状态编码转换为中文说明
SELECT 
  order_id, order_no,
  CASE order_status
    WHEN '待支付' THEN '请尽快付款'
    WHEN '已支付' THEN '已安排发货'
    WHEN '已发货' THEN '运输中'
    WHEN '已收货' THEN '交易完成'
    WHEN '已取消' THEN '订单已关闭'
    ELSE '未知状态'
  END AS status_desc
FROM order_info;
```

##### 写法 2：搜索 CASE（条件判断，最灵活，类似 if-else if）

###### 语法

```sql
CASE
  WHEN 条件1 THEN 返回结果1
  WHEN 条件2 THEN 返回结果2
  ...
  ELSE 兜底默认值 -- 可选
END
```

###### 示例

```sql
-- 需求：根据用户年龄划分年龄段
SELECT 
  user_id, user_name, age,
  CASE
    WHEN age < 18 THEN '未成年'
    WHEN age BETWEEN 18 AND 35 THEN '青年'
    WHEN age BETWEEN 36 AND 60 THEN '中年'
    WHEN age > 60 THEN '老年'
    ELSE '年龄未填写'
  END AS age_group
FROM user_info;

-- 进阶：聚合函数中嵌套CASE，实现分组统计
SELECT 
  COUNT(*) AS total_order,
  SUM(CASE WHEN order_status = '已支付' THEN 1 ELSE 0 END) AS paid_order,
  SUM(CASE WHEN order_status = '待支付' THEN 1 ELSE 0 END) AS unpaid_order
FROM order_info;
```

#### 关键选型：IF () vs CASE WHEN

- 简单二分支：用 IF ()，代码更简洁；
- 超过 2 个分支：优先用 CASE WHEN，可读性、可维护性更强；
- 复杂条件判断：必须用搜索 CASE，支持多字段、多条件组合。

## 三、第二部分：程序级流程控制语句（进阶复杂逻辑）

这部分是 MySQL 实现复杂业务逻辑的核心，**仅能在存储过程 / 存储函数 / 触发器 / 事件的 BEGIN...END 代码块中使用**，普通 SQL 语句无法直接使用。

### 3.1 前置必备知识

#### 3.1.1 分隔符 DELIMITER 修改

MySQL 默认以分号`;`作为语句结束符，而存储过程内部的流程控制语句也会用到分号，会导致 MySQL 提前解析结束，报错。因此**创建存储过程前，必须先修改分隔符**，创建完成后再改回分号。

```sql
-- 1. 修改分隔符为 //（可自定义，比如$$）
DELIMITER //

-- 2. 中间写存储过程创建语句，内部用;结尾
CREATE PROCEDURE 存储过程名()
BEGIN
  -- 内部SQL语句，用;结尾
  SELECT * FROM user_info;
END //

-- 3. 改回默认分隔符;
DELIMITER ;
```

#### 3.1.2 BEGIN...END 代码块

用于包裹一组 SQL 语句，形成一个逻辑代码块，类似编程语言中的`{}`，存储过程的主体逻辑必须放在 BEGIN...END 中。

#### 3.1.3 局部变量声明

流程控制中会用到变量，用`DECLARE`声明局部变量，仅在当前 BEGIN...END 块中有效，必须写在代码块的最开头。

```sql
-- 声明变量语法
DECLARE 变量名 数据类型 [DEFAULT 默认值];

-- 示例
DECLARE user_count INT DEFAULT 0; -- 声明int类型变量，默认值0
DECLARE user_name VARCHAR(50); -- 声明varchar类型变量
```

---

### 3.2 分支结构语句

用于实现多条件逻辑判断，分为 IF 语句和 CASE 语句，和之前的查询级函数不同，这是完整的语句块，支持多行逻辑。

#### 3.2.1 IF 语句（多分支完整逻辑）

##### 核心功能

实现完整的`if-else if-else`逻辑，支持多行 SQL 语句，是分支结构的首选。

##### 语法

```sql
IF 条件1 THEN
  -- 条件1成立时执行的SQL语句
ELSEIF 条件2 THEN
  -- 条件2成立时执行的SQL语句
...
ELSE
  -- 所有条件都不成立时执行的SQL语句
END IF;
```

##### 示例（存储过程）

```sql
DELIMITER //
-- 需求：根据用户ID，判断用户的订单等级
CREATE PROCEDURE get_user_order_level(IN in_user_id INT)
BEGIN
  -- 声明变量
  DECLARE order_total DECIMAL(10,2) DEFAULT 0;
  
  -- 查询用户的总订单金额
  SELECT SUM(order_amount) INTO order_total 
  FROM order_info 
  WHERE user_id = in_user_id;
  
  -- IF分支判断
  IF order_total >= 1000 THEN
    SELECT 'VIP用户' AS user_level, order_total;
  ELSEIF order_total >= 500 THEN
    SELECT '高级用户' AS user_level, order_total;
  ELSEIF order_total > 0 THEN
    SELECT '普通用户' AS user_level, order_total;
  ELSE
    SELECT '新用户（无订单）' AS user_level, order_total;
  END IF;
END //
DELIMITER ;

-- 调用存储过程
CALL get_user_order_level(1);
```

#### 3.2.2 CASE 语句（分支语句）

##### 核心功能

和 CASE 表达式功能类似，但是是完整的语句块，支持多行 SQL 执行，分为简单 CASE 和搜索 CASE 两种写法。

##### 语法 1：简单 CASE 语句

```sql
CASE 待判断的表达式
  WHEN 匹配值1 THEN
    -- 匹配成功执行的SQL
  WHEN 匹配值2 THEN
    -- 匹配成功执行的SQL
  ...
  ELSE
    -- 无匹配执行的SQL
END CASE;
```

##### 语法 2：搜索 CASE 语句

```sql
CASE
  WHEN 条件1 THEN
    -- 条件成立执行的SQL
  WHEN 条件2 THEN
    -- 条件成立执行的SQL
  ...
  ELSE
    -- 无匹配执行的SQL
END CASE;
```

##### 示例

```sql
DELIMITER //
-- 需求：根据订单状态，执行不同的统计逻辑
CREATE PROCEDURE stat_order_by_status(IN in_status VARCHAR(20))
BEGIN
  CASE in_status
    WHEN '已支付' THEN
      SELECT COUNT(*) AS 已支付订单数, SUM(order_amount) AS 总营收 FROM order_info WHERE order_status = '已支付';
    WHEN '待支付' THEN
      SELECT COUNT(*) AS 待支付订单数, SUM(order_amount) AS 待支付金额 FROM order_info WHERE order_status = '待支付';
    ELSE
      SELECT COUNT(*) AS 其他状态订单数 FROM order_info WHERE order_status = in_status;
  END CASE;
END //
DELIMITER ;

-- 调用
CALL stat_order_by_status('已支付');
```

##### 关键区别：CASE 语句 vs CASE 表达式

- CASE 表达式：用在 SELECT 等查询语句中，返回一个值，结尾是`END`；
- CASE 语句：用在存储程序的 BEGIN...END 中，执行 SQL 语句块，结尾是`END CASE`。

---

### 3.3 循环结构语句

用于重复执行一组 SQL 语句，MySQL 支持 3 种循环结构：WHILE、REPEAT、LOOP，均需配合标签使用。

#### 3.3.1 循环标签（Label）

用于给循环 / 代码块命名，是跳转语句 LEAVE、ITERATE 的必备前提，语法：

```sql
标签名: WHILE 条件 DO
  -- 循环体
END WHILE 标签名;
```

#### 3.3.2 WHILE 循环（先判断后执行，最常用）

##### 核心功能

先判断循环条件，条件成立才执行循环体，类似编程语言中的`while`循环，条件不成立则直接退出循环。

##### 语法

```sql
标签名: WHILE 循环条件 DO
  -- 循环体：要重复执行的SQL语句
END WHILE 标签名;
```

##### 示例：批量插入 100 条测试用户数据

```sql
DELIMITER //
CREATE PROCEDURE batch_insert_user(IN insert_count INT)
BEGIN
  DECLARE i INT DEFAULT 1; -- 循环计数器
  
  -- WHILE循环：i小于等于插入数量时，循环执行
  insert_loop: WHILE i <= insert_count DO
    -- 插入测试数据
    INSERT INTO user_info (user_name, phone, age) 
    VALUES (CONCAT('测试用户', i), CONCAT('1380000', LPAD(i, 4, '0')), FLOOR(RAND() * 50 + 18));
    -- 计数器+1
    SET i = i + 1;
  END WHILE insert_loop;
  
  SELECT CONCAT('成功插入', insert_count, '条数据') AS result;
END //
DELIMITER ;

-- 调用：插入100条测试数据
CALL batch_insert_user(100);
```

#### 3.3.3 REPEAT 循环（先执行后判断，类似 do-while）

##### 核心功能

先执行一次循环体，再判断退出条件，条件不成立则继续循环，条件成立则退出循环，类似编程语言中的`do-while`循环。**无论条件是否成立，循环体至少执行一次**。

##### 语法

```sql
标签名: REPEAT
  -- 循环体
UNTIL 退出条件 -- 注意：这里没有分号
END REPEAT 标签名;
```

##### 示例

```sql
DELIMITER //
CREATE PROCEDURE repeat_demo()
BEGIN
  DECLARE i INT DEFAULT 1;
  DECLARE total INT DEFAULT 0;
  
  sum_loop: REPEAT
    SET total = total + i;
    SET i = i + 1;
  UNTIL i > 10 -- 当i>10时，退出循环
  END REPEAT sum_loop;
  
  SELECT total AS 1到10的和;
END //
DELIMITER ;

-- 调用
CALL repeat_demo();
```

#### 3.3.4 LOOP 循环（无条件循环，需手动退出）

##### 核心功能

无条件的死循环，没有内置的循环条件，必须通过`LEAVE`语句手动退出循环，灵活性最高，适合复杂的循环逻辑。

##### 语法

```sql
标签名: LOOP
  -- 循环体
  -- 必须有LEAVE语句退出循环，否则会死循环
END LOOP 标签名;
```

##### 示例

```sql
DELIMITER //
CREATE PROCEDURE loop_demo()
BEGIN
  DECLARE i INT DEFAULT 1;
  DECLARE total INT DEFAULT 0;
  
  calc_loop: LOOP
    -- 退出条件：i>10时，退出循环
    IF i > 10 THEN
      LEAVE calc_loop; -- 退出循环
    END IF;
    
    SET total = total + i;
    SET i = i + 1;
  END LOOP calc_loop;
  
  SELECT total AS 1到10的和;
END //
DELIMITER ;

-- 调用
CALL loop_demo();
```

#### 三种循环的选型对比

|循环类型|执行顺序|核心特点|适用场景|
|:--|:--|:--|:--|
|**WHILE**|先判断，后执行|条件不成立则一次都不执行|绝大多数循环场景，最常用|
|**REPEAT**|先执行，后判断|至少执行一次|必须先执行一次逻辑，再判断是否继续的场景|
|**LOOP**|无条件执行，手动退出|灵活性最高，可自定义退出时机|复杂循环逻辑、多退出条件的场景|

### 3.4 循环跳转控制语句

用于控制循环的执行流程，分为`LEAVE`（退出）和`ITERATE`（跳过），必须配合循环标签使用。

#### 3.4.1 LEAVE 语句：退出循环 / 代码块

##### 核心功能

退出指定标签的循环或代码块，类似编程语言中的`break`。

##### 语法

```sql
LEAVE 标签名;
```

##### 示例：循环中满足条件直接退出

```sql
DELIMITER //
CREATE PROCEDURE leave_demo()
BEGIN
  DECLARE i INT DEFAULT 1;
  
  test_loop: WHILE i <= 10 DO
    -- 当i=5时，直接退出循环
    IF i = 5 THEN
      LEAVE test_loop;
    END IF;
    SELECT i;
    SET i = i + 1;
  END WHILE test_loop;
END //
DELIMITER ;
```

#### 3.4.2 ITERATE 语句：跳过本次循环

##### 核心功能

跳过本次循环剩余的代码，直接进入下一次循环的条件判断，类似编程语言中的`continue`。

##### 语法

```sql
ITERATE 标签名;
```

##### 示例：循环中跳过偶数，只输出奇数

```sql
DELIMITER //
CREATE PROCEDURE iterate_demo()
BEGIN
  DECLARE i INT DEFAULT 0;
  
  test_loop: WHILE i < 10 DO
    SET i = i + 1;
    -- 偶数时，跳过本次循环，不执行后续的SELECT
    IF i MOD 2 = 0 THEN
      ITERATE test_loop;
    END IF;
    SELECT i AS 奇数;
  END WHILE test_loop;
END //
DELIMITER ;
```

### 3.5 异常处理（DECLARE HANDLER）

#### 核心功能

用于捕获并处理存储程序执行过程中出现的 SQL 异常，避免程序直接中断退出，类似编程语言中的`try-catch`异常处理。

#### 语法

```sql
DECLARE 处理类型 HANDLER FOR 异常条件
  -- 异常发生时执行的处理语句
```

#### 核心参数说明

1. **处理类型**：
    
    - `CONTINUE`：异常处理完成后，继续执行后续的 SQL 语句；
    - `EXIT`：异常处理完成后，退出当前的 BEGIN...END 代码块；
    - `UNDO`：MySQL 不支持，忽略。
    
2. **异常条件**：
    
    - `SQLEXCEPTION`：捕获所有 SQL 异常；
    - `SQLWARNING`：捕获所有 SQL 警告；
    - `NOT FOUND`：捕获「未找到数据」的异常（如 SELECT INTO 无结果）；
    - 具体的错误码，如`1062`（唯一键冲突）。
    

#### 示例 1：处理 NOT FOUND 异常

```sql
DELIMITER //
-- 需求：根据用户ID查询用户名，无数据时返回默认值
CREATE PROCEDURE get_user_name(IN in_user_id INT, OUT out_user_name VARCHAR(50))
BEGIN
  -- 声明异常处理：未找到数据时，给out_user_name赋值默认值，继续执行
  DECLARE CONTINUE HANDLER FOR NOT FOUND
    SET out_user_name = '用户不存在';
  
  -- 查询用户名，赋值给输出变量
  SELECT user_name INTO out_user_name FROM user_info WHERE user_id = in_user_id;
END //
DELIMITER ;

-- 调用
SET @name = '';
CALL get_user_name(999, @name);
SELECT @name;
```

#### 示例 2：处理唯一键冲突异常

```sql
DELIMITER //
-- 需求：插入用户，手机号重复时捕获异常，返回提示
CREATE PROCEDURE insert_user_safe(
  IN in_user_name VARCHAR(50),
  IN in_phone VARCHAR(11),
  OUT out_result VARCHAR(100)
)
BEGIN
  -- 声明异常处理：唯一键冲突（错误码1062）时，赋值提示，退出
  DECLARE EXIT HANDLER FOR 1062
    SET out_result = '插入失败：手机号已存在';
  
  -- 插入数据
  INSERT INTO user_info (user_name, phone) VALUES (in_user_name, in_phone);
  SET out_result = '插入成功';
END //
DELIMITER ;

-- 调用
SET @result = '';
CALL insert_user_safe('测试用户', '13800138000', @result);
SELECT @result;
```

## 四、综合实战案例

### 案例：用户订单统计存储过程

整合分支、循环、异常处理，实现：批量统计每个用户的订单数据，生成用户等级报表。

```sql
DELIMITER //
CREATE PROCEDURE stat_user_order_level()
BEGIN
  -- 声明变量
  DECLARE total_user INT DEFAULT 0;
  DECLARE i INT DEFAULT 1;
  DECLARE current_user_id INT;
  DECLARE order_total DECIMAL(10,2) DEFAULT 0;
  DECLARE user_level VARCHAR(20);
  
  -- 异常处理
  DECLARE EXIT HANDLER FOR SQLEXCEPTION
    SELECT '程序执行异常，已终止' AS error_msg;
  
  -- 1. 获取总用户数
  SELECT COUNT(*) INTO total_user FROM user_info;
  
  -- 2. 创建结果表（不存在则创建）
  CREATE TABLE IF NOT EXISTS user_level_report (
    report_id INT PRIMARY KEY AUTO_INCREMENT,
    user_id INT UNIQUE NOT NULL,
    user_name VARCHAR(50) NOT NULL,
    order_total DECIMAL(10,2) DEFAULT 0,
    user_level VARCHAR(20) NOT NULL,
    create_time DATETIME DEFAULT CURRENT_TIMESTAMP
  ) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;
  
  -- 3. 清空历史数据
  TRUNCATE TABLE user_level_report;
  
  -- 4. 循环遍历每个用户
  stat_loop: WHILE i <= total_user DO
    -- 获取当前用户ID
    SELECT user_id INTO current_user_id FROM user_info LIMIT i-1, 1;
    
    -- 查询用户总订单金额
    SELECT IFNULL(SUM(order_amount), 0) INTO order_total 
    FROM order_info 
    WHERE user_id = current_user_id;
    
    -- 分支判断用户等级
    CASE
      WHEN order_total >= 1000 THEN SET user_level = 'VIP用户';
      WHEN order_total >= 500 THEN SET user_level = '高级用户';
      WHEN order_total > 0 THEN SET user_level = '普通用户';
      ELSE SET user_level = '新用户';
    END CASE;
    
    -- 插入报表数据
    INSERT INTO user_level_report (user_id, user_name, order_total, user_level)
    SELECT current_user_id, user_name, order_total, user_level 
    FROM user_info 
    WHERE user_id = current_user_id;
    
    -- 计数器+1
    SET i = i + 1;
  END WHILE stat_loop;
  
  -- 5. 返回结果
  SELECT '统计完成' AS result, COUNT(*) AS 统计用户数 FROM user_level_report;
END //
DELIMITER ;

-- 调用存储过程
CALL stat_user_order_level();
-- 查看报表
SELECT * FROM user_level_report;
```

## 五、常见坑与避坑指南

1. **混淆查询级函数和程序级语句**
    
    - 坑：在普通 SELECT 语句中写`IF...THEN...END IF`，直接报错；
    - 避坑：普通 SQL 只能用 IF ()、CASE 表达式，IF 语句、循环只能在存储过程的 BEGIN...END 中使用。
    
2. **存储过程分隔符未修改**
    
    - 坑：创建存储过程时，未修改 DELIMITER，导致 MySQL 把内部的分号当成语句结束，报错；
    - 避坑：创建前必须`DELIMITER //`，创建完成后`DELIMITER ;`。
    
3. **循环死循环**
    
    - 坑：WHILE 循环中忘记给计数器 + 1，LOOP 循环中忘记写 LEAVE 语句，导致死循环；
    - 避坑：循环必须有明确的退出条件，写完先在测试环境执行，加 LIMIT 限制。
    
4. **CASE 表达式和 CASE 语句结尾搞混**
    
    - 坑：查询中的 CASE 表达式结尾写`END CASE`，存储过程中的 CASE 语句结尾写`END`，报错；
    - 避坑：查询级 CASE 结尾`END`，程序级 CASE 语句结尾`END CASE`。
    
5. **NULL 值导致的分支判断异常**
    
    - 坑：IF 条件中出现 NULL 值，判断结果为 UNKNOWN，会走 ELSE 分支，导致逻辑错误；
    - 避坑：分支判断前，先用 IFNULL () 处理 NULL 值，明确判断规则。
    
6. **DECLARE 变量的位置错误**
    
    - 坑：在 BEGIN...END 块的中间声明变量，报错；
    - 避坑：DECLARE 声明变量必须写在 BEGIN...END 块的最开头，所有 SQL 语句之前。

## 六、核心要点总结

1. MySQL 流程控制分为两大类：
    
    - **查询级函数**：IF ()、IFNULL ()、NULLIF ()、CASE 表达式，普通 SQL 可用，日常高频使用；
    - **程序级语句**：IF 语句、CASE 语句、循环、跳转、异常处理，仅在存储程序的 BEGIN...END 中使用，处理复杂业务逻辑。
    
2. 分支选型：
    
    - 简单二分支用 IF () 函数；
    - 多分支优先用 CASE 表达式 / CASE 语句，可读性更强；
    - 复杂多行逻辑用 IF 语句。
    
3. 循环选型：
    
    - 绝大多数场景用 WHILE 循环（先判断后执行）；
    - 必须先执行一次用 REPEAT；
    - 复杂多退出条件用 LOOP。
    
4. 跳转控制：LEAVE 退出循环（break），ITERATE 跳过本次循环（continue），必须配合标签使用。
    
5. 异常处理：用 DECLARE HANDLER 捕获异常，避免程序中断，提升存储过程的健壮性。
