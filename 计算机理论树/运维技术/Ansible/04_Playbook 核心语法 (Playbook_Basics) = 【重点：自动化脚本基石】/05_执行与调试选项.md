
Ansible 提供了全链路的 Playbook 执行与调试工具，覆盖语法校验、变更预演、交互式执行、问题定位全场景，是 Playbook 编写、测试、生产落地的核心配套能力，本节完整覆盖目录指定的 4 类核心调试能力。

### 4.5.1 语法检查（--syntax-check）

- **核心作用**：仅对 Playbook 文件进行 YAML 语法与 Ansible 规范校验，不连接被控节点、不执行任何任务，是编写 Playbook 后的第一步校验，可提前发现缩进错误、关键字错误、缺少必填字段、模块不存在等问题；
- **执行命令**：

    ```bash
    ansible-playbook --syntax-check nginx_deploy.yml
    ```
    
- **结果说明**：无任何报错输出则语法完全合规；若存在错误，会精准定位错误的行号与错误原因，大幅降低调试成本。

### 4.5.2 模拟执行 / 干跑（--check /-C）

- **核心作用**：完全模拟 Playbook 的全量执行流程，包括连接被控节点、采集 Facts、执行模块的前置状态校验，但**绝对不修改被控节点的任何系统状态**。基于模块的幂等性，预判执行后的变更情况，输出哪些 Task 会发生状态变更、哪些会被跳过，是生产环境变更前的核心预校验工具；
- **执行命令**：

    ```bash
    # 基础干跑
    ansible-playbook --check nginx_deploy.yml
    
    # 干跑+显示文件变更差异
    ansible-playbook --check --diff nginx_deploy.yml
    ```
    
- **补充说明**：
    
    - `--diff`参数可配合`--check`使用，显示 copy、template 等模块的文件内容行级变更差异，直观预览配置修改内容；
    - 干跑能力依赖模块的内置幂等性，command、shell 等无内置幂等性的模块，干跑时不会执行，无法准确预判执行结果。
    

### 4.5.3 逐步执行（--step）

- **核心作用**：交互式逐步执行 Playbook 的每个 Task，执行每个 Task 前会弹出交互式提示，用户可选择执行、跳过、继续无提示执行、退出，适合调试复杂 Playbook、精准控制执行流程、定位故障 Task；
- **执行命令**：

    ```bash
    ansible-playbook --step nginx_deploy.yml
    ```
    
- **交互式选项说明**：
    
    - `y`：执行当前 Task，进入下一个 Task 的提示；
    - `n`：跳过当前 Task，进入下一个 Task 的提示；
    - `c`：继续执行剩余所有 Task，不再弹出提示；
    - `q`：立即退出 Playbook 执行。
    

### 4.5.4 详细输出与深度调试（-v /-vvv 系列）

- **核心作用**：控制 Ansible 执行时的日志输出详细程度，是定位执行失败、排查异常问题的核心工具，不同级别对应不同粒度的输出内容；
- **级别划分与使用场景**：

| 级别  | 命令      | 输出内容                                          | 适用场景                  |
| :-- | :------ | :-------------------------------------------- | :-------------------- |
| 1 级 | `-v`    | 每个 Task 的执行结果摘要，包括 changed 状态、核心返回值           | 日常执行、基础结果查看           |
| 2 级 | `-vv`   | 在 1 级基础上，增加 Task 输入参数、Inventory 主机匹配信息、变量解析结果 | 进阶调试、参数校验             |
| 3 级 | `-vvv`  | 在 2 级基础上，增加 SSH 连接细节、模块完整执行逻辑、远程命令全量输出        | 深度调试、Task 执行失败排查，最常用  |
| 4 级 | `-vvvv` | 开启 SSH 连接的全量 Debug 日志，包括密钥认证、握手、会话建立的完整细节     | 网络连通性、SSH 认证、连接超时问题排查 |
    
- **执行命令**：

    ```bash
    # 最常用的3级详细输出
    ansible-playbook -vvv nginx_deploy.yml
    
    # SSH连接调试4级输出
    ansible-playbook -vvvv nginx_deploy.yml
    ```
    

### 4.5.5 补充高频执行选项

|选项|核心作用|
|:--|:--|
|`-i <path>`|指定 Inventory 资产清单文件 / 目录路径|
|`-u <username>`|覆盖 Playbook 中的 remote_user，指定远程连接用户名|
|`-b`|覆盖 Playbook 配置，强制开启提权|
|`-K`|执行时提示输入提权密码|
|`--limit <host-pattern>`|限制 Playbook 仅在指定的主机 / 分组上执行，避免全量误操作|
|`--list-hosts`|仅列出 Playbook 匹配的目标主机列表，不执行任何操作，用于范围校验|
|`--list-tasks`|仅列出 Playbook 中的所有 Task 列表，不执行任何操作，用于流程校验|
