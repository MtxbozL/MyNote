Ansible_Learning_Roadmap/
│
├── 01_初识与核心架构 (Introduction_&_Architecture)
│   ├── 01_什么是 Ansible (IT 自动化、配置管理、应用部署工具)
│   ├── 02_与其他工具对比 (Ansible vs SaltStack vs Puppet vs Chef)
│   ├── 03_核心特性 (Agentless 无代理架构、Push 推送模型、幂等性 Idempotency)
│   ├── 04_工作原理与架构 (Control Node 控制节点、Managed Node 被控节点、基于 SSH/WinRM)
│   └── 05_安装与配置 (Yum/Apt 安装、Pip 安装、ansible.cfg 基础配置解析)
│
├── 02_资产清单与主机连通 (Inventory_&_Connection) => 【重点：如何定义你的机器】
│   ├── 01_静态 Inventory (ini 与 yaml 格式的 hosts 文件编写)
│   ├── 02_主机分组与嵌套 (定义 webservers, dbservers, 以及 children 组)
│   ├── 03_内置连接参数 (ansible_ssh_user, ansible_ssh_port, ansible_ssh_private_key_file)
│   ├── 04_免密登录配置 (ssh-keygen 与 ssh-copy-id 的批量分发原理)
│   ├── 05_动态 Inventory (结合公有云 AWS EC2 / 阿里云 或 CMDB 动态生成主机列表)
│   └── 06_连通性测试 (ansible all -m ping 命令底层解析)
│
├── 03_核心模块与 Ad-Hoc 命令 (Modules_&_Ad-Hoc) => 【重点：日常敲打命令】
│   ├── 01_Ad-Hoc 概念 (什么是临时命令，何时使用它)
│   ├── 02_命令执行模块 (command, shell, script —— 区别与使用场景)
│   ├── 03_文件与存储模块 (file, copy, fetch, template, synchronize)
│   ├── 04_软件管理模块 (yum, apt, pip, get_url)
│   ├── 05_系统与用户模块 (user, group, cron, hostname, sysctl)
│   ├── 06_服务管理模块 (service, systemd)
│   ├── 07_网络与安全模块 (firewalld, iptables, selinux)
│   └── 08_使用帮助文档 (ansible-doc -s module_name 查询模块参数神器)
│
├── 04_Playbook 核心语法 (Playbook_Basics) => 【重点：自动化脚本基石】
│   ├── 01_YAML 语法速成 (缩进、列表、字典、大小写敏感特性)
│   ├── 02_Playbook 结构 (Play、Task、Module 的层级关系)
│   ├── 03_编写第一个剧本 (指定 hosts, remote_user, tasks, become 提权)
│   ├── 04_提权机制 (become: yes, become_method: sudo, become_user)
│   ├── 05_执行与调试选项 
│   │   ├── 语法检查 (--syntax-check)
│   │   ├── 模拟执行/干跑 (--check / -C)
│   │   ├── 逐步执行 (--step)
│   │   └── 详细输出 (-v, -vvv 调试神器)
│   └── 06_Idempotency 幂等性实战 (确保多次执行结果一致，不产生副作用)
│
├── 05_变量、事实与模板 (Variables_Facts_&_Jinja2) => 【重点：让脚本动起来】
│   ├── 01_变量作用域与优先级 (Extra vars > Task vars > Host/Group vars > Default vars)
│   ├── 02_定义变量的方式 (vars 块、group_vars 目录、host_vars 目录、命令传入 -e)
│   ├── 03_Register 机制 (将任务执行结果捕获并存储到变量中供后续使用)
│   ├── 04_Facts 系统信息采集 (setup 模块、ansible_hostname/ansible_default_ipv4 等内置变量)
│   ├── 05_魔法变量 (Magic Variables: hostvars, groups, inventory_hostname)
│   ├── 06_Jinja2 模板渲染 (Template 模块实战)
│   │   ├── 基础变量替换 ( {{ variable_name }} )
│   │   ├── 控制结构 ( {% for %} 循环、{% if %} 条件判断动态生成 Nginx 配置 )
│   │   └── 过滤器 ( Filters: default, json_query, regex_replace )
│   └── 07_关闭/优化 Facts 采集 (gather_facts: no 提升执行速度)
│
├── 06_流程控制与错误处理 (Control_Flow_&_Error_Handling) => 【重点：复杂业务逻辑】
│   ├── 01_Conditionals 条件判断 (when 语句、基于 Facts 和注册变量跳过任务)
│   ├── 02_Loops 循环机制 (loop, with_items, with_dict 批量创建用户/文件)
│   ├── 03_Handlers 触发器机制 => 【必考特性】
│   │   └── notify 与 handlers 配合 (配置变更后，仅在改变时才触发 Nginx/MySQL 重启)
│   ├── 04_Tags 标签运行 (给 Task 打标签，使用 --tags 或 --skip-tags 只执行部分任务)
│   ├── 05_错误处理机制
│   │   ├── 忽略错误 (ignore_errors: yes 继续往下走)
│   │   ├── 强制处理程序执行 (force_handlers: yes)
│   │   └── Block/Rescue/Always (类似 Try-Catch-Finally 的异常捕获逻辑)
│   └── 06_暂停与交互 (pause 模块、wait_for 端口检测)
│
├── 07_Roles 角色与模块化 (Roles_&_Ansible_Galaxy) => 【重点：代码重用与工程化】
│   ├── 01_为什么需要 Roles？ (解决单个 yaml 文件过长、难以复用的痛点)
│   ├── 02_Roles 标准目录结构解析 (tasks, handlers, templates, files, vars, defaults, meta)
│   ├── 03_创建与调用角色 (ansible-galaxy init, import_role, include_role)
│   ├── 04_Include 与 Import 的核心区别 (静态加载 vs 动态加载的影响)
│   ├── 05_Ansible Galaxy 社区 (查找、下载、使用开源角色：如 Geerlingguy 的 MySQL/Nginx 角色)
│   └── 06_依赖管理 (meta/main.yml 定义角色执行前置依赖)
│
├── 08_高级特性与性能调优 (Advanced_Features_&_Tuning) => 【高级架构师必备】
│   ├── 01_安全与加密 (Ansible Vault 机制)
│   │   └── 创建加密文件、加密字符串、执行时解密 (--ask-vault-pass)
│   ├── 02_任务委派与本地操作 (delegate_to, local_action，适用于 F5 负载均衡器下线操作)
│   ├── 03_异步与轮询 (Async & Poll)
│   │   └── 处理耗时极长的任务 (如大数据备份、编译安装，防止 SSH 连接超时)
│   ├── 04_执行策略 (Strategy: 默认的 Linear 线性执行 vs Free 自由并发执行)
│   ├── 05_并发与网络调优 (极大缩短执行时间)
│   │   ├── 增加并发数 (forks = 50)
│   │   ├── 开启 Pipelining (减少 SSH 连接次数，大幅提升传输效率)
│   │   ├── ControlMaster / SSH 复用 (修改 ssh_args 加速握手)
│   │   └── 替换 JSON 引擎 (使用 Mitogen 插件提速)
│   └── 06_插件开发初探 (自定义 Callback 回调插件输出自定义日志格式)
│
├── 09_自动化实战场景 (Practical_Scenarios)
│   ├── 01_操作系统基线初始化 (批量修改内核参数、关闭 SElinux、同步时区、配置 Yum 源)
│   ├── 02_LAMP/LEMP 架构一键部署 (结合 Roles 完整搭建 Linux+Nginx+MySQL+PHP)
│   ├── 03_安全加固与补丁批量下发 (针对特定漏洞批量升级 OpenSSH 版本)
│   ├── 04_数据灾备自动化 (定时拉取多台服务器的 DB 备份文件至本地中控机)
│   └── 05_公有云资源编排 (使用 AWS/Aliyun 模块通过 API 自动创建虚拟机并初始化)
│
└── 10_企业级生态与 CI/CD 演进 (Enterprise_Ecosystem_&_DevOps)
    ├── 01_Ansible AWX / Red Hat Ansible Tower
    │   ├── Web 图形化界面管理
    │   ├── RBAC 权限控制 (研发只能点发布，不能改脚本)
    │   ├── 审计日志与执行历史追踪
    │   └── 定时任务 (取代系统的 Cron)
    ├── 02_GitOps 与版本控制整合 (剧本存入 GitLab，通过 Webhook 触发执行)
    ├── 03_与 Jenkins 结合 (Jenkins Pipeline + Ansible 实现应用的一键构建与发布)
    └── 04_Ansible 在 Kubernetes 中的应用 (k8s 模块管理集群资源，operator-sdk 编写 Operator)
