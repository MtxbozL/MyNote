
Nginx 主配置文件`nginx.conf`采用**层级嵌套式**语法结构，所有配置指令均归属不同作用域块，块与块之间为父子继承关系，指令生效范围由所在块层级决定，核心层级从外到内依次为：**main 全局块 → events 块 → http 块 → server 块 → location 块**，是 Nginx 配置的核心骨架。

### 2.1.1 层级核心特性

1. **继承性**：子块可继承父块的所有配置指令，若子块定义同名指令，将**覆盖**父块配置，实现精细化的配置粒度控制；
2. **唯一性**：`main`块为最顶层根块，`events`块、`http`块为`main`块的直接子块，一个配置文件中仅能有一个`events`块和一个`http`块；
3. **多实例性**：`http`块内可定义多个`server`块（虚拟主机），单个`server`块内可定义多个`location`块（URI 路由），实现多站点、多路由的独立配置；
4. **指令生效规则**：指令仅能在指定作用域块中生效，如`worker_processes`仅能在`main`块配置，`listen`仅能在`server`块配置，违规配置会导致语法校验失败。

### 2.1.2 层级作用域与核心职责

|配置块|所属父块|核心作用域|核心职责|
|---|---|---|---|
|main 全局块|根块|整个 Nginx 进程|配置进程级全局参数，如运行用户、进程数、日志路径，启动时加载且部分不支持热重载|
|events 块|main 全局块|所有 Worker 进程的连接处理|配置事件驱动模型、并发连接数，优化网络连接处理性能|
|http 块|main 全局块|所有 HTTP/HTTPS 服务|配置 HTTP 协议通用参数，如 MIME 类型、日志格式、全局压缩，为所有 server 块提供默认配置|
|server 块|http 块|单个虚拟主机（站点）|配置站点独立参数，如监听端口、域名、根目录，实现多站点的隔离部署|
|location 块|server 块|单个 URI 路由|配置 URI 匹配规则与请求处理逻辑，如反向代理、静态资源映射、重写规则|

### 2.1.3 配置语法基础规则

1. **指令格式**：所有配置指令均为`指令名 参数值;`格式，**分号结尾为强制要求**，缺失会导致语法错误；
2. **块指令格式**：块指令以`指令名 { ... }`表示，大括号内为子指令或子块，大括号与指令名之间需有空格，且大括号必须成对出现；
3. **注释规则**：以`#`开头的行为注释行，Nginx 解析时会忽略，注释不可嵌套，且不可写在指令行内；
4. **空格与换行**：配置中多个空格、制表符、换行符均视为单个分隔符，可通过格式化提升配置可读性；
5. **文件引入**：通过`include 路径/文件名;`指令引入外部配置文件，支持通配符`*`，可实现配置的模块化拆分与复用，如`include conf.d/*.conf;`。
