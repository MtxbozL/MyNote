
Nginx 内置变量是预定义的**动态变量**，由 Nginx 在处理请求时自动赋值，存储了客户端请求、服务器状态、连接信息等核心数据，可嵌入配置指令（如 log_format、rewrite、proxy_pass）中使用，是 Nginx 配置灵活性的核心，所有内置变量均以`$`开头。

### 2.6.1 核心内置变量详解

1. **$host**
    
    - 取值规则：优先取客户端请求的 Host 头值，若 Host 头为空或非法，取`server_name`的第一个域名，若未配置`server_name`，取监听的服务器 IP 地址；
    - 作用：获取请求的有效域名，用于日志记录、反向代理、重写规则等场景，避免 Host 头为空导致的配置异常；
    - 示例：`proxy_set_header Host $host;`（反向代理时透传有效域名至后端服务）。
    
2. **$remote_addr**
    
    - 取值：客户端的 IP 地址，即 Nginx 直接接收到的 TCP 连接的源 IP 地址；
    - 作用：获取客户端原始 IP，用于访问控制、日志记录、IP 哈希负载均衡等场景；
    - 注意：若客户端与 Nginx 之间存在代理（如 CDN、负载均衡器），`$remote_addr`将为代理服务器的 IP，而非客户端真实 IP，需通过`X-Real-IP`/`X-Forwarded-For`请求头透传。
    
3. **$request_uri**
    
    - 取值：客户端发起的**原始完整请求 URI**，包含路径与查询参数，且为只读，不受 Nginx 内部重写规则影响；
    - 示例：请求`http://example.com/api/user?id=100`，`$request_uri`值为`/api/user?id=100`；
    - 作用：反向代理时原封不动透传客户端请求至后端，或记录原始请求地址，是最常用的 URI 变量。
    
4. **$uri** / **$document_uri**
    
    - 取值：Nginx**内部处理后的 URI**，不含查询参数，若请求经过 rewrite 重写规则，将为重写后的 URI；
    - 示例：请求`http://example.com/api/user?id=100`经 rewrite 重写为`/v1/user`，则`$uri`值为`/v1/user`；
    - 作用：用于 Nginx 内部的路径匹配、文件查找，如拼接根目录查找静态文件。
    
5. **$http_user_agent**
    
    - 取值：客户端的 User-Agent 请求头值，包含客户端浏览器、操作系统、设备信息；
    - 作用：实现基于客户端的差异化处理，如为移动端、PC 端返回不同页面，或屏蔽恶意爬虫；
    - 示例：`if ($http_user_agent ~* "Spider") { return 403; }`（屏蔽爬虫请求）。
    
6. **$status**
    
    - 取值：Nginx 返回给客户端的 HTTP 响应状态码，如 200、404、502；
    - 作用：用于访问日志记录，统计不同状态码的请求数，实现异常请求监控。
    

### 2.6.2 通用请求头变量规则

Nginx 提供**通用请求头变量**规则：`$http_请求头名`，将请求头名的大写字母转为小写，横线`-`转为下划线`_`，即可获取对应请求头的值。

- 示例：获取`X-Real-IP`请求头值：`$http_x_real_ip`；获取`Accept-Language`请求头值：`$http_accept_language`；
- 作用：灵活获取任意自定义请求头或标准请求头，为个性化配置、请求透传提供支持。

### 2.6.3 内置变量使用注意事项

- 内置变量为**动态变量**，仅在请求处理阶段生效，不可在全局块、events 块中使用；
- `$request_uri`保留原始请求信息，适合反向代理与日志记录；`$uri`适合内部路径处理，需根据场景选择；
- 正则匹配内置变量时，注意使用`~`（区分大小写）或`~*`（不区分大小写），避免匹配失败。
