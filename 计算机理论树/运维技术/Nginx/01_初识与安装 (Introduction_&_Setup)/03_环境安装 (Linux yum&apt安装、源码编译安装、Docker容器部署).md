Nginx 的安装分为三类主流方式，分别适配不同的部署场景与运维需求，具体如下：

### 1.3.1 Linux 发行版预编译包安装（yum/apt）

该方式通过系统包管理器直接安装官方预编译的 Nginx 安装包，适配快速部署、测试环境、无自定义模块需求的生产环境。

1. **RPM 系发行版（CentOS/RHEL/Rocky Linux）yum 安装**
    
    - 步骤 1：配置 Nginx 官方 yum 源，解决系统自带源版本过低问题；
    - 步骤 2：执行`yum install -y nginx`完成安装；
    - 步骤 3：执行`systemctl start nginx`启动服务，`systemctl enable nginx`设置开机自启；
    - 步骤 4：通过`curl http://127.0.0.1`验证服务可用性。
    
2. **Debian 系发行版（Debian/Ubuntu）apt 安装**
    
    - 步骤 1：导入 Nginx 官方 GPG 密钥，配置官方 apt 源；
    - 步骤 2：执行`apt update && apt install -y nginx`完成安装；
    - 步骤 3：执行`systemctl start nginx`启动服务，`systemctl enable nginx`设置开机自启；
    - 步骤 4：验证服务可用性。
    
3. **方案特性**：优点为操作简单、自动解决依赖、支持系统级安全更新；缺点为版本固定、无法自定义编译模块、配置路径与功能受预编译包限制。

### 1.3.2 源码编译安装

该方式通过 Nginx 官方源码包手动编译安装，是生产环境定制化部署的主流方式，适配需要添加第三方模块、性能极致优化、指定特定版本、功能裁剪的场景。

1. **核心安装流程**
    
    - 步骤 1：安装编译依赖环境，包括 gcc、make、openssl-devel、pcre-devel、zlib-devel 等基础库；
    - 步骤 2：从 Nginx 官方下载指定版本源码包，解压至编译目录；
    - 步骤 3：执行`configure`脚本，配置安装路径、编译参数、启用 / 禁用模块、第三方模块集成等核心选项；
    - 步骤 4：执行`make`完成源码编译，生成二进制可执行文件；
    - 步骤 5：执行`make install`完成安装，将二进制文件、配置文件、依赖资源安装至指定路径；
    - 步骤 6：配置系统服务与环境变量，实现服务的系统级管理。
    
2. **方案特性**：优点为完全自定义功能模块、编译参数、安装路径，可实现与服务器硬件、业务场景的最优适配；缺点为操作复杂度高、需手动解决依赖冲突、版本更新与模块变更需重新编译。

### 1.3.3 Docker 容器化部署

该方式基于 Docker 容器运行时部署 Nginx 服务，适配容器化集群、云原生环境、快速多版本测试、跨平台部署场景。

1. **核心部署流程**
    
    - 步骤 1：服务器安装 Docker 与 docker-compose 容器编排工具；
    - 步骤 2：执行`docker pull nginx:指定版本`拉取官方 Nginx 镜像；
    - 步骤 3：编写 docker-compose.yml 配置文件，定义容器端口映射、配置文件挂载、静态资源目录挂载、日志目录挂载、网络模式等核心配置；
    - 步骤 4：执行`docker-compose up -d`启动 Nginx 容器；
    - 步骤 5：验证容器服务可用性。
    
2. **方案特性**：优点为开箱即用、环境强隔离、可移植性强、支持容器编排与弹性扩缩容；缺点为需部署容器运行时环境、自定义模块需重新构建镜像、网络配置需适配容器网络模型。
