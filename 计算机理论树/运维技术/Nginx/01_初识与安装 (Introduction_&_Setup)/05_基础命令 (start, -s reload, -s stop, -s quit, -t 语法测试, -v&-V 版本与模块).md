
Nginx 的所有命令行操作均基于其二进制可执行文件，默认路径为`安装根目录/sbin/nginx`，命令分为启动类、运行时信号控制类、配置校验类、信息查询类四大类，是 Nginx 日常运维的核心基础。

### 1.5.1 服务启动命令

- 基础启动命令：`./nginx`，读取默认路径的主配置文件启动 Nginx，创建 Master 进程与 Worker 进程，完成服务初始化；
- 指定配置文件启动：`./nginx -c /path/to/nginx.conf`，指定自定义主配置文件启动，适配多实例部署、配置文件测试场景；
- 临时全局参数启动：`./nginx -g "全局配置指令"`，临时覆盖主配置文件中的全局块参数，无需修改配置文件，适配临时调优、测试场景。

### 1.5.2 运行时信号控制命令

通过`-s`参数向 Master 进程发送管理信号，实现服务运行时的无中断管控，是生产环境核心运维命令，具体如下：

- `./nginx -s reload`：**配置热加载**，生产环境核心命令。执行流程为：Master 进程首先校验配置文件语法，语法合法则启动新的 Worker 进程，向旧 Worker 进程发送优雅关闭信号；旧 Worker 进程停止接收新连接，处理完已建立的所有请求后正常退出，实现无业务中断的配置更新。
- `./nginx -s quit`：**优雅停止服务**，Master 进程向所有 Worker 进程发送 QUIT 信号，Worker 进程停止接收新连接，处理完所有已建立的请求后退出，Master 进程随后终止，无请求丢失，是生产环境推荐的服务停止方式。
- `./nginx -s stop`：**强制停止服务**，Master 进程向所有 Worker 进程发送 TERM 信号，所有进程立即终止，丢弃正在处理的请求，仅适用于服务异常、紧急停止场景。
- `./nginx -s reopen`：**重新打开日志文件**，关闭当前所有日志文件句柄，重新打开配置中指定的日志文件，配合 logrotate 工具实现日志切割与归档，避免单日志文件过大。

### 1.5.3 配置校验命令

- `./nginx -t`：配置语法测试，读取主配置文件与所有 include 引入的子配置文件，校验全量配置的语法合法性，输出校验结果与错误信息。**修改配置后执行 reload 前必须执行该命令**，避免配置错误导致服务中断。
- `./nginx -T`：全量配置校验与打印，除完成语法校验外，打印所有 include 引入的配置文件全量内容，用于排查配置继承、参数覆盖、路径错误等复杂配置问题。

### 1.5.4 信息查询命令

- `./nginx -v`：打印 Nginx 版本号，用于快速确认服务版本。
- `./nginx -V`：打印 Nginx 版本号、编译时执行的 configure 全量参数、所有已编译的模块信息，是排查模块是否启用、编译参数是否正确、功能异常定位的核心命令，为生产环境故障排查必用。
