
生命周期标签是 Prometheus 中以`__`双下划线开头的系统保留标签，核心特性是：**仅在 Relabeling 阶段有效，Relabeling 执行完成后，未被手动保留的生命周期标签会被自动丢弃，不会写入 TSDB 存储**。生命周期标签是 Relabeling 机制的核心操作对象，分为通用生命周期标签与服务发现专属元数据标签两类。

### 6.5.1 通用生命周期标签

通用生命周期标签适用于所有抓取目标，核心控制抓取行为，是`relabel_configs`最常用的操作对象：

|标签名|核心含义|可修改性|核心作用|
|---|---|---|---|
|`__address__`|抓取目标的地址，格式为`host:port`|是|定义 Prometheus 发起抓取请求的目标地址，默认值为服务发现的实例地址 + 端口|
|`__metrics_path__`|抓取请求的 URI 路径|是|定义指标抓取的路径，默认值为`/metrics`|
|`__scheme__`|抓取请求的协议|是|定义抓取请求的协议，可选`http`/`https`，默认值为`http`|
|`__param_<name>`|抓取请求的 URL 参数|是|定义抓取请求的查询参数，`<name>`为参数名，标签值为参数值|
|`__scrape_interval__`|该目标的抓取间隔|是|覆盖全局 / Job 级的抓取间隔，Prometheus 2.47 + 支持|
|`__scrape_timeout__`|该目标的抓取超时时间|是|覆盖全局 / Job 级的抓取超时时间，Prometheus 2.47 + 支持|
|`__name__`|指标名称|是|指标名称的内置标签，仅在`metric_relabel_configs`中可用，可用于过滤、修改指标名称|

### 6.5.2 服务发现专属元数据标签

服务发现专属元数据标签是各类 SD 机制生成的、以`__meta_`为前缀的生命周期标签，包含了服务发现实例的所有元数据，不同 SD 机制生成的标签不同，核心包括：

- Consul SD：`__meta_consul_`前缀的标签，包含服务名称、ID、地址、端口、标签、健康状态等；
- Kubernetes SD：`__meta_kubernetes_`前缀的标签，包含命名空间、资源名称、IP、标签、注解等；
- EC2 SD：`__meta_ec2_`前缀的标签，包含实例 ID、IP、标签、可用区、实例类型等；
- File SD：`__meta_filepath`标签，包含目标文件的路径。

### 6.5.3 核心使用规范

1. **元数据必须手动保留**：`__meta_`前缀的标签，必须通过`replace`或`labelmap`动作转换为非双下划线开头的永久标签，否则会被自动丢弃，无法存储和查询；
2. **抓取参数修改时机**：`__address__`、`__metrics_path__`等抓取控制标签，仅在`relabel_configs`中修改有效，`metric_relabel_configs`中修改无任何作用；
3. **禁止修改系统核心标签**：除明确可修改的通用生命周期标签外，禁止修改其他系统内置的双下划线标签，可能导致抓取行为异常；
4. **临时标签命名规范**：自定义临时标签必须以`__tmp_`为前缀，确保 Relabeling 后自动丢弃，避免污染永久标签体系。
