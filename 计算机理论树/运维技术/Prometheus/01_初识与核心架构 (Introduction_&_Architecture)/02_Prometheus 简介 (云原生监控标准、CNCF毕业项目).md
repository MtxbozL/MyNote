
Prometheus 是一款开源的系统监控与告警工具包，是云原生时代监控领域的事实标准，同时也是云原生计算基金会（CNCF）的毕业项目。

### 1.2.1 项目发展与生态地位

Prometheus 最初由 SoundCloud 公司于 2012 年基于 Go 语言开发，用于解决云原生架构下的动态服务监控问题；2015 年正式开源，2016 年加入 CNCF 成为继 Kubernetes 之后的第二个孵化项目，2018 年正式毕业，具备完整的社区生态、标准化的设计规范与企业级的稳定性保障。

### 1.2.2 核心设计理念与核心特性

Prometheus 的核心设计围绕**时序数据处理**与**云原生动态环境适配**两大核心，其核心特性包括：

1. 基于维度标签的时序数据模型，支持多维度指标的灵活存储与查询；
2. 原生内置 PromQL 查询语言，支持时序数据的多维度聚合、计算与分析；
3. 以 HTTP Pull（拉取）模型为核心的指标采集机制，同时兼容 Push（推送）模型的补充场景；
4. 无分布式存储依赖，单机即可实现完整的监控能力，同时支持联邦集群与远程存储扩展；
5. 支持基于文件、Consul、Kubernetes 等多种机制的动态服务发现，适配云原生环境下服务实例动态漂移的场景；
6. 内置告警规则管理能力，可通过 AlertManager 组件实现灵活的告警路由、聚合与通知。
