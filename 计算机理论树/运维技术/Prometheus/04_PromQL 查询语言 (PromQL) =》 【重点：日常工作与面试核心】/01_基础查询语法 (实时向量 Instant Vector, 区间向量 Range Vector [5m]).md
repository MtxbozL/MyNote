## 4.1 PromQL 核心定位与基础查询语法

PromQL（Prometheus Query Language）是 Prometheus 专为时序数据设计的**声明式多维查询语言**，是 Prometheus 生态的核心能力载体。区别于传统 SQL 的关系型数据查询模型，PromQL 完全围绕时序数据的维度标签与时间序列特性设计，支持对时序数据进行过滤、聚合、计算、关联与可视化，是日常监控运维、告警规则配置、故障排查、性能分析的核心工具。

PromQL 查询的核心执行单元为**向量（Vector）**，根据返回的数据结构，分为两大核心类型：**瞬时向量（Instant Vector）** 与 **区间向量（Range Vector）**，二者构成了 PromQL 语法的基础。

### 4.1.1 瞬时向量（Instant Vector）

#### 数学定义

瞬时向量是一组时序数据点的集合，其中**每个时序仅包含单个数据点**，所有数据点共享完全相同的时间戳（默认为查询执行的时间戳），每个数据点由全局唯一的时序标识（指标名称 + 标签集）与对应的 float64 数值构成。

#### 核心特性

1. 仅包含单个时间戳的数值，可直接用于二元运算、聚合运算、告警规则表达式与可视化渲染；
2. 是 PromQL 中最基础的查询结果类型，绝大多数函数的输入与输出均为瞬时向量；
3. 时间戳可通过时间位移操作、绝对时间修饰符修改，默认与查询执行时间对齐。

#### 基础语法与示例

瞬时向量的基础查询为**指标选择器**，通过指标名称 + 标签匹配器过滤目标时序，语法如下：

```bash
<metric_name>{<label_matchers>}
```

标准示例：

```bash
# 基础瞬时向量查询：查询所有实例的HTTP请求总数
http_requests_total

# 带标签过滤的瞬时向量查询：查询生产环境、GET方法的5xx错误请求总数
http_requests_total{env="prod", method="GET", status=~"5.."}
```

### 4.1.2 区间向量（Range Vector）

#### 数学定义

区间向量是一组时序数据序列的集合，其中**每个时序包含指定时间窗口内的所有连续数据点**，每个数据点带有独立的毫秒级时间戳，时间窗口由方括号`[]`内的持续时间参数指定。

#### 核心特性

1. 包含一个时间范围内的多个数据点，**无法直接用于二元运算、聚合运算与告警规则**，必须通过速率类、聚合类函数转换为瞬时向量后方可使用；
2. 核心作用是为时序数据的趋势计算、速率计算提供时间范围内的原始数据支撑；
3. 时间窗口的大小必须至少覆盖 2 个抓取间隔，否则会因数据点不足导致计算失效。

#### 时间窗口单位规范

PromQL 支持的标准时间单位如下，禁止使用非标准单位：

|单位|含义|换算关系|
|---|---|---|
|`ms`|毫秒|1s = 1000ms|
|`s`|秒|1m = 60s|
|`m`|分钟|1h = 60m|
|`h`|小时|1d = 24h|
|`d`|天|1w = 7d|
|`w`|周|1y = 365d|
|`y`|年|基于公历年 365 天计算|

#### 基础语法与示例

区间向量的语法为在瞬时向量选择器后追加方括号包裹的时间窗口，语法如下：

```bash
<metric_name>{<label_matchers>}[<duration>]
```

标准示例：

```bash
# 区间向量查询：查询所有实例最近5分钟的HTTP请求总数的所有数据点
http_requests_total[5m]

# 带标签过滤的区间向量查询：查询生产环境实例最近10分钟的CPU空闲时间数据点
node_cpu_seconds_total{env="prod", mode="idle"}[10m]
```
