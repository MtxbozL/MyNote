
PromQL 提供了两种时间调整能力，用于实现非实时时间的时序数据查询：**`offset` 时间位移操作**（相对时间调整）与 **`@` 绝对时间修饰符**（绝对时间定点），二者均用于修改向量的时间戳，是环比、同比、定点查询的核心实现方式。

### 4.3.1 `offset` 时间位移操作

`offset`用于将向量的查询时间戳向**历史方向**位移指定的持续时间，实现与历史数据的对比分析，支持瞬时向量与区间向量。

#### 语法规范

`offset`必须紧跟在向量选择器的末尾，语法如下：

```bash
# 瞬时向量位移
<instant_vector> offset <duration>

# 区间向量位移
<range_vector> offset <duration>
```

#### 标准示例与适用场景

```bash
# 基础位移：查询5分钟前的HTTP请求总数瞬时值
http_requests_total offset 5m

# 同比查询：查询昨天同一时间的CPU使用率
100 - avg(rate(node_cpu_seconds_total{mode="idle"}[1m] offset 1d)) * 100

# 环比查询：对比当前1小时请求速率与上一小时的速率
rate(http_requests_total[1h]) / rate(http_requests_total[1h] offset 1h)

# 区间向量位移：查询10分钟前的5分钟窗口内的请求总数数据点
http_requests_total[5m] offset 10m
```

#### 核心约束

1. `offset`仅支持向历史方向位移，禁止使用负数值向未来方向位移；
2. `offset`必须紧跟向量选择器，不可放在函数或表达式的末尾；
3. 位移后的向量类型保持不变，瞬时向量位移后仍为瞬时向量，区间向量位移后仍为区间向量。

### 4.3.2 `@` 绝对时间修饰符

`@`修饰符用于将向量的查询时间戳固定为**指定的 Unix 时间戳**，实现定点时间的精准查询，支持瞬时向量与区间向量，Prometheus 2.25.0 及以上版本支持。

#### 语法规范

`@`必须紧跟在向量选择器的末尾，可与`offset`组合使用，语法如下：

```bash
# 固定绝对时间戳
<vector> @ <unix_timestamp>

# 内置时间函数：查询时间范围的起始/结束时间
<vector> @ start()
<vector> @ end()
```

#### 标准示例与适用场景

```bash
# 定点查询：查询2024-01-01 00:00:00（Unix时间戳1704067200）的磁盘剩余空间
node_filesystem_avail_bytes @ 1704067200

# 区间向量定点：查询指定时间点前5分钟的请求总数数据点
http_requests_total[5m] @ 1704067200

# 与offset组合使用：查询指定时间点前1天的CPU使用率
rate(node_cpu_seconds_total{mode="idle"}[1m] @ 1704067200 offset 1d)

# 内置函数：子查询中固定查询范围的结束时间
rate(http_requests_total[1h] @ end())
```

#### 核心约束

1. `@`修饰符的时间戳为毫秒级 Unix 时间戳，若传入秒级时间戳，Prometheus 会自动转换；
2. 禁止使用未来时间戳，会导致查询无结果；
3. `start()`与`end()`仅在 Grafana 范围查询、Prometheus 子查询中生效，瞬时查询中无意义。

### 4.3.3 `offset`与`@`的核心区别

|特性|`offset` 位移操作|`@` 绝对时间修饰符|
|---|---|---|
|时间调整方式|相对时间位移，基于查询时间向前偏移|绝对时间定点，固定为指定 Unix 时间戳|
|适用场景|环比、同比、历史趋势对比|定点时间回溯、固定时间窗口查询、历史快照对比|
|灵活性|无需提前知道历史时间戳，适配动态查询|需明确指定目标时间戳，适配精准定点查询|
