
Gauge 是与 Counter 互补的瞬时状态型指标，核心语义为**被观测对象的当前快照状态**，是基础设施监控、资源状态监控的核心指标类型。

### 3.3.1 核心定义与数学特性

Gauge 是一种**无单调性约束**的数值型指标，其数值可正向增长、负向下降，直接反映被监控对象在某一时刻的瞬时状态，无累积语义。

数学特性为：

与无固定大小关系

### 3.3.2 语义规范与适用场景

Gauge 的核心语义是**某一时刻的瞬时快照值**，适用于描述可任意波动的状态量，典型适用场景包括：

- 主机 CPU 使用率、内存占用率、磁盘剩余空间、磁盘使用率；
- 服务当前并发连接数、活跃请求数、在线用户数；
- 消息队列堆积长度、线程池活跃线程数、连接池空闲连接数；
- 环境温度、硬件负载、网络带宽实时使用率；
- 容器 CPU / 内存实时使用量、Pod 副本数。

### 3.3.3 核心使用规范与 PromQL 适配

Gauge 的原始值可直接用于查询、可视化与告警，无需速率函数转换，核心适配 PromQL 函数聚焦于趋势分析与预测：

- `deriv()`：计算 Gauge 在时间窗口内的每秒变化率，基于线性回归计算，可反映指标的增长 / 下降趋势；
- `predict_linear()`：基于时间窗口内的历史数据，预测指标未来的变化趋势，典型用于磁盘空间耗尽预警；
- `avg()`/`max()`/`min()`/`sum()`：直接对 Gauge 的瞬时值进行聚合计算，反映整体状态。

标准查询示例：

```bash
# 主机CPU使用率（1分钟平均）
100 - avg by (instance) (rate(node_cpu_seconds_total{mode="idle"}[1m])) * 100

# 预测磁盘空间将在24小时内耗尽的实例
predict_linear(node_filesystem_avail_bytes[6h], 24*3600) < 0

# 服务当前最大并发连接数
max by (service) (http_server_active_connections)
```

### 3.3.4 高频使用误区与禁忌

1. **绝对禁止**使用 Gauge 记录累积型事件总数，例如请求总数、错误次数，会导致无法计算事件发生速率，丢失历史变化信息；
2. **禁止**对 Gauge 使用`rate()`/`increase()`函数，该类函数专为 Counter 设计，仅能处理单调非递减数据，用于 Gauge 会产生完全错误的计算结果；
3. 禁止为 Gauge 添加频繁变化的标签值，避免时序基数持续增长；
4. 禁止使用 Gauge 记录高频波动的瞬时值，导致数据无统计意义，需结合合理的抓取间隔平衡精度与存储开销。
