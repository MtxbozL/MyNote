
Counter 是 Prometheus 最基础的累积型指标类型，核心语义为**事件发生的累积次数**，是监控体系中使用最广泛的指标类型之一。

### 3.2.1 核心定义与数学特性

Counter 是一种**严格单调非递减**的累积型指标，其数值仅能正向增长，或在被监控进程重启时重置为 0，**绝对不会主动下降**。

数学约束为：

∀t2​>t1​,Value(t2​)≥Value(t1​)

仅当被监控进程重启、实例重建时，Counter 值会被重置为 0，Prometheus 的 PromQL 速率函数可自动识别并处理该重置行为，不会导致计算结果异常。

### 3.2.2 语义规范与适用场景

Counter 的核心语义是**累积发生的事件总数**，仅适用于描述只会正向增长的统计量，典型适用场景包括：

- 服务请求总数、错误请求总数、成功请求总数；
- 接口调用次数、任务执行次数、异常触发次数；
- 网络传输字节总数、磁盘读写字节总数；
- 进程 CPU 使用时间、GC 总次数。

### 3.2.3 核心使用规范与 PromQL 适配

Counter 的原始值为累积总量，**无直接监控与查询意义**，必须通过 PromQL 速率函数计算其单位时间内的增长速率，才能反映事件的发生频率。

核心适配函数：

- `rate()`：计算时间窗口内 Counter 的平均每秒增长速率，适用于长期趋势监控、告警规则，是生产环境首选；
- `irate()`：计算时间窗口内最后两个数据点的瞬时每秒增长速率，适用于实时峰值监控，对数据点密度要求高；
- `increase()`：计算时间窗口内 Counter 的总增长量，本质是`rate() * 窗口时长`，适用于统计周期内的事件总数。

标准查询示例：

```bash
# 计算HTTP请求的每秒平均QPS（5分钟窗口）
rate(http_requests_total[5m])

# 统计5分钟内的HTTP错误请求总数
increase(http_requests_total{status=~"5.."}[5m])
```

### 3.2.4 高频使用误区与禁忌

1. **绝对禁止**使用 Counter 记录可增可减的瞬时状态量，例如 CPU 使用率、内存占用、当前在线人数、队列长度；
2. **禁止**直接使用 Counter 的原始值进行告警、可视化与趋势分析，例如`http_requests_total > 10000`，该表达式仅能反映进程启动后的总请求数，无任何监控意义；
3. **禁止**对 Counter 的速率计算结果再次使用速率函数，会导致逻辑错误与结果失真；
4. 禁止为 Counter 添加高基数标签，避免时序基数爆炸。
