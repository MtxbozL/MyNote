## 一、核心定位

Linux 基础终端操作命令，属于「基础系统与终端操作类」，核心作用是**查看、格式化输出系统日期与时间，也可手动修改系统时间**，是日常终端操作、Shell 脚本编写、日志管理、时间校准场景的高频核心命令。

## 二、语法格式

```bash
# 1. 查看/格式化输出时间（日常90%场景使用）
date [可选参数] [+格式控制符]

# 2. 设置系统时间（需root权限）
date [可选参数] --set="时间字符串"
```

> 格式说明：`+` 是格式化输出的必填前缀，格式控制符需紧跟在`+`后，多格式符建议用双引号包裹，避免空格导致解析异常。

## 三、高频核心参数与格式控制符

### 1. 核心功能参数

|参数|核心作用|高频使用场景|
|:--|:--|:--|
|`-d, --date=STRING`|按指定的时间字符串解析并输出时间，不修改系统当前时间|时间计算（昨天 / 7 天前 / 下个月）、日志时间筛选、脚本时间偏移处理|
|`-s, --set=STRING`|按指定字符串设置系统时间|手动校准系统时间（仅离线 / 无 NTP 场景使用）|
|`-u, --utc`|输出 / 设置 UTC 标准时间（忽略本地时区）|跨时区服务部署、日志统一时间戳|
|`-I, --iso-8601`|输出 ISO 8601 标准格式的日期|日志命名、文件归档、接口时间字段|
|`-R, --rfc-2822`|输出 RFC 2822 标准格式的时间|邮件头、HTTP 请求头、标准化日志输出|

### 2. 高频格式控制符（格式化输出核心）

|格式符|核心作用|输出示例|
|:--|:--|:--|
|`%Y`|4 位完整年份|2026|
|`%m`|2 位月份（01-12）|03|
|`%d`|2 位日期（01-31）|15|
|`%H`|24 小时制小时（00-23）|14|
|`%M`|2 位分钟（00-59）|30|
|`%S`|2 位秒数（00-59）|00|
|`%F`|快捷格式，等价于`%Y-%m-%d`|2026-03-15|
|`%T`|快捷格式，等价于`%H:%M:%S`|14:30:00|
|`%w`|星期几（0 = 周日，1-6 = 周一到周六）|0|
|`%Z`|系统当前时区|CST|
|`%s`|Unix 时间戳（从 1970-01-01 00:00:00 UTC 到当前的秒数）|1773642600|

## 四、实操示例

### 1. 基础入门示例

```bash
# 示例1：直接查看系统当前日期时间（默认格式）
date
# 执行效果：输出带时区的完整本地时间，例：Sun Mar 15 14:30:00 CST 2026
```

```bash
# 示例2：基础格式化输出，按「年-月-日」格式输出当前日期
date +"%Y-%m-%d"
# 执行效果：2026-03-15
```

```bash
# 示例3：用快捷格式符输出完整时间
date +"%F %T"
# 执行效果：2026-03-15 14:30:00
```

### 2. 高频场景示例

```bash
# 场景1：生成带时间戳的文件名（日志/备份文件最常用）
date +"app_log_%Y%m%d%H%M%S.log"
# 执行效果：app_log_20260315143000.log
```

```bash
# 场景2：相对时间计算（-d参数核心用法，日志排查高频）
# 输出昨天的日期
date -d "yesterday" +"%F"
# 输出7天前的日期
date -d "7 days ago" +"%F"
# 输出下个月今天的日期
date -d "next month" +"%F"
# 输出1小时后的时间
date -d "1 hour" +"%F %T"
# 执行效果：对应偏移后的时间，例：7天前输出2026-03-08
```

```bash
# 场景3：输出Unix时间戳，用于时间对比、接口传参
date +%s
# 执行效果：1773642600
```

```bash
# 场景4：手动设置系统时间（需root/sudo权限）
sudo date -s "2026-03-15 14:30:00"
# 执行效果：将系统时间修改为指定值，仅临时生效，重启后会被硬件时间/ NTP覆盖
```

## 五、避坑提示 & 注意事项

1. **格式符大小写极易混淆，新手高频踩坑**：
    
    - `%m`= 月份，`%M`= 分钟，切勿写反（错误示例：`date +%Y%m%d%H%m%S` 会导致月份重复）
    - `%H`=24 小时制，`%I`=12 小时制，日志 / 脚本建议固定用`%H`，避免上午 / 下午时间歧义
    
2. **修改系统时间的风险**：
    
    - 仅 root 用户可修改时间，普通用户必须加`sudo`；
    - 生产环境严禁随意手动修改时间，会导致日志时序混乱、定时任务异常、SSL 证书验证失败、数据库数据异常，优先使用 NTP/chrony 服务自动同步时间。
    
3. **系统时间与硬件时间的区别**：
    
    - `date`修改的是**系统时间（操作系统运行的时间）**，服务器重启后会失效；
    - 若需永久生效，修改后需执行`sudo hwclock -w`，将系统时间同步到服务器硬件时间。
    
4. **跨平台兼容性**：
    
    - `-d` 相对时间参数是 GNU date 的特性，主流 Linux 发行版（Ubuntu、CentOS、Rocky）均支持；MacOS 自带的 BSD date 不兼容该写法，需使用`date -v -1d +%F`实现 “昨天” 的效果。
    
5. **格式化前缀必填**：格式化输出必须加`+`前缀，否则会报错，`+`与格式符之间无空格，多格式符建议用双引号包裹。

## 六、同类命令对比 & 拓展

|命令|核心作用|与 date 的区别 & 适用场景|
|:--|:--|:--|
|`hwclock`|查看 / 设置服务器硬件时间（BIOS 时间）|date 管理系统运行时的时间，hwclock 管理断电后仍能保持的硬件时间，用于时间持久化|
|`timedatectl`|systemd 系统标配的时间、时区管理命令|可一站式查看 / 设置系统时间、时区、NTP 同步状态，替代 date+hwclock 的组合操作，现代 Linux 系统优先推荐|
|`cal`|查看日历（当月 / 指定年份月份）|与 date 互补，用于可视化查看日期分布，无时间修改 / 格式化能力|
