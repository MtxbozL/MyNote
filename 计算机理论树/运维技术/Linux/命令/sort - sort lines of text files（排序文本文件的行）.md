## 一、核心定位

`sort` 是 Linux 系统中**最核心、最高频的文本排序工具**，属于「文本处理类」，无任何权限要求，普通用户可直接处理自己有权限的文件。核心作用是**按行排序文本，支持多种排序规则（字典序、数值序、月份序、自然版本序、随机序）、指定排序字段、忽略大小写、反向排序、去重、合并已排序文件等**，是文本整理、数据排序、日志分析、报告生成的必备基础命令，也是 Linux 入门必学的文本处理命令。

> 补充说明：`sort` 默认按**字典序**排序（ASCII 码顺序），数值排序必须加`-n`参数；支持多个文件合并排序，是`uniq`（去重）的前置依赖（`uniq`仅对相邻重复行有效，需先排序）。

## 二、语法格式

bash

运行

```
# 核心语法1：排序单个/多个文件，输出到标准输出
sort [可选参数] 文件1 文件2 ...

# 核心语法2：排序标准输入（管道输入）
命令 | sort [可选参数]

# 核心语法3：直接输出到指定文件（可用于修改原文件）
sort [可选参数] 文件 -o 输出文件
```

> 关键说明：
> 
> 1. 无参数时，默认按**字典序**排序（ASCII 码顺序），区分大小写；
> 2. 无文件参数时，默认从 ** 标准输入（stdin）** 读取内容，即管道输入；
> 3. 多个文件时，默认合并所有文件后排序；
> 4. 默认输出所有行，包括重复行。

## 三、高频核心参数（按功能分类）

### 1. 基础排序类

表格

|参数|核心作用|高频使用场景|
|:--|:--|:--|
|无参数（直接执行`sort 文件`）|按**字典序**排序（ASCII 码顺序），区分大小写|日常简单排序文本，最基础、最常用的用法|
|`-n, --numeric-sort`|按**数值序**排序，将行首的数字视为数值比较|数值数据排序、价格 / 数量 / 大小排序，**数值排序必须加此参数**|
|`-r, --reverse`|**反向**排序，颠倒默认排序顺序|从大到小排序、从新到旧排序、反向字典序|
|`-f, --ignore-case`|排序时**忽略大小写**，将小写字母视为大写字母比较|模糊排序、不区分大小写的文本排序|
|`-u, --unique`|**去重**，仅输出排序键相同的唯一行（仅比较排序键，不是整行）|去除重复数据、唯一值统计|

### 2. 字段排序类

表格

|参数|核心作用|高频使用场景|
|:--|:--|:--|
|`-t, --field-separator=SEP`|指定**输入字段分隔符**，支持字符串或正则表达式，默认是任意空白字符（空格、制表符）|处理非空白分隔的文本（如逗号分隔的 CSV、冒号分隔的 /etc/passwd）|
|`-k, --key=KEYDEF`|指定**排序字段**，KEYDEF 格式为`F[.C][OPTS][,F[.C][OPTS]]`，F 为字段号，C 为字符位置，OPTS 为排序规则（n/r/f 等）|按指定列排序、表格数据排序、多字段组合排序|

### 3. 排序规则类

表格

|参数|核心作用|高频使用场景|
|:--|:--|:--|
|`-M, --month-sort`|按**月份序**排序，识别 Jan/Feb/.../Dec（不区分大小写）|月份数据排序、日志按月份排序|
|`-V, --version-sort`|按**自然版本序**排序，正确排序 file1、file2、file10（而非 file1、file10、file2）|版本文件排序、日志文件排序、带数字的文件名排序|
|`-h, --human-numeric-sort`|按**人类可读单位**排序，识别 K/M/G/T/P（如 1K、2M、3G）|磁盘空间排序、文件大小排序、带单位的数值排序|
|`-R, --random-sort`|按**随机序**排序，每次运行结果不同|随机打乱数据、随机抽样、随机排序测试|
|`-d, --dictionary-order`|按**字典序**排序，仅考虑字母、数字和空白字符，忽略其他字符|纯文本字典序排序、忽略特殊字符|

### 4. 输出类

表格

|参数|核心作用|高频使用场景|
|:--|:--|:--|
|`-o, --output=FILE`|输出到**指定文件**，可用于直接修改原文件（安全）|保存排序结果、批量修改文件|
|`-c, --check, --check=diagnose-first`|检查文件是否**已排序**，不输出排序结果，仅返回状态码|验证文件是否已排序、脚本前置检查|
|`-m, --merge`|合并**已排序的文件**，不重新排序，直接合并|合并多个已排序文件、提升性能|
|`-s, --stable`|**稳定排序**，保持排序键相同的行的原始相对顺序|多字段排序时保持次要键的原始顺序|
|`--help`|输出 sort 帮助信息|快速查看参数与用法|
|`--version`|输出 sort 版本信息|确认版本兼容性|

## 四、实操示例

### 1. 基础入门示例（新手必练，零门槛上手）

bash

运行

```
# 示例1：最基础用法，按字典序排序
sort test.txt
# 执行效果：输出test.txt的内容，按字典序排序
```

bash

运行

```
# 示例2：按数值序排序（数值排序必须加-n）
sort -n numbers.txt
# 执行效果：按数值大小排序，而非字典序
```

bash

运行

```
# 示例3：反向排序
sort -r test.txt
# 执行效果：颠倒默认排序顺序
```

bash

运行

```
# 示例4：忽略大小写排序
sort -f test.txt
# 执行效果：不区分大小写，将a和A视为相同
```

bash

运行

```
# 示例5：去重排序
sort -u test.txt
# 执行效果：仅输出唯一行，去除重复行
```

### 2. 高频日常场景示例

#### 场景 1：指定字段排序（结合 - t 和 - k）

bash

运行

```
# 处理/etc/passwd文件（冒号分隔），按第3字段（UID）数值排序
sort -t ':' -k 3n /etc/passwd

# 处理CSV文件（逗号分隔），按第2字段（姓名）字典序排序
sort -t ',' -k 2 data.csv

# 按第2字段数值排序，再按第1字段字典序排序（多字段组合）
sort -t ',' -k 2n -k 1 data.csv
```

#### 场景 2：自然版本序排序

bash

运行

```
# 正确排序带数字的文件名（file1、file2、file10）
sort -V files.txt
# 执行效果：file1、file2、file10，而非file1、file10、file2
```

#### 场景 3：人类可读单位排序

bash

运行

```
# 按文件大小排序（识别K/M/G）
sort -h sizes.txt
# 执行效果：1K、2M、3G，按实际大小排序
```

#### 场景 4：检查文件是否已排序

bash

运行

```
# 检查文件是否已按字典序排序
sort -c test.txt
# 执行效果：若已排序则无输出，返回状态码0；若未排序则输出错误信息，返回状态码1
```

#### 场景 5：合并已排序文件

bash

运行

```
# 合并两个已排序的文件，不重新排序
sort -m sorted1.txt sorted2.txt
# 执行效果：直接合并两个已排序文件，输出合并后的有序结果
```

### 3. 进阶奇妙用法

#### 奇妙用法 1：结合其他命令（管道输入）

bash

运行

```
# 结合ls，按文件大小排序
ls -l | sort -k 5n

# 结合ps，按CPU使用率排序
ps aux | sort -k 3nr

# 结合df，按磁盘使用率排序
df -h | sort -k 5hr

# 结合du，按目录大小排序
du -sh * | sort -hr
```

#### 奇妙用法 2：排序后去重（sort -u）

bash

运行

```
# 排序后去重，仅输出唯一行
sort -u test.txt
# 等价于先排序再用uniq
sort test.txt | uniq
```

#### 奇妙用法 3：随机排序（-R）

bash

运行

```
# 随机打乱文件内容，每次运行结果不同
sort -R test.txt

# 随机抽样，取前10行
sort -R test.txt | head -10
```

#### 奇妙用法 4：按多个字段排序（-k）

bash

运行

```
# 先按第2字段数值降序，再按第1字段字典序升序
sort -t ',' -k 2nr -k 1 data.csv

# 先按第3字段月份序，再按第1字段数值序
sort -t ',' -k 3M -k 1n data.csv
```

#### 奇妙用法 5：排序大文件（-s 稳定排序）

bash

运行

```
# 稳定排序，保持排序键相同的行的原始相对顺序
sort -s -k 2n data.csv
```

#### 奇妙用法 6：直接修改原文件（-o）

bash

运行

```
# 排序后直接输出到原文件，安全修改
sort test.txt -o test.txt
# 执行效果：test.txt被排序后的内容覆盖
```

## 五、避坑提示 & 注意事项

1. **新手最高频致命坑：数值排序必须加`-n`**
    
    - `sort` 默认按**字典序**排序（ASCII 码顺序），数值不加`-n`会导致排序错误（如 10 < 2，因为字典序 1 < 2）；
    - 强制要求：数值排序**必须加`-n`参数**；
    - 典型错误：`sort numbers.txt`（10 排在 2 前面）；
    - 正确示例：`sort -n numbers.txt`（2 排在 10 前面）。
    
2. **字段分隔符的默认行为**
    
    - `sort` 默认的输入字段分隔符是**任意空白字符（空格、制表符、换行符）**，且连续的空白字符会被视为一个分隔符；
    - 若文本以单个空格或制表符分隔，默认行为没问题；但若以其他字符（如逗号、冒号）分隔，**必须加`-t`参数指定分隔符**；
    - 典型错误：处理`/etc/passwd`时不加`-t ':'`，导致整行被视为一个字段。
    
3. **`-u`去重的坑**
    
    - `sort -u` 仅比较**排序键**，不是整行；若排序键相同但其他字段不同，也会被视为重复行而去除；
    - 解决方法：若需整行去重，先排序再用`uniq`，即`sort test.txt | uniq`；
    - 典型错误：`sort -u -k 2n data.csv`（仅比较第 2 字段，去除第 2 字段相同的行，不管其他字段）。
    
4. **`-o`直接修改原文件的坑**
    
    - `sort -o 文件` 会直接覆盖原文件，无任何提示；
    - 强制要求：使用`-o`前，**必须先不加`-o`执行一遍，预览排序效果**；
    - 进阶技巧：重要文件先备份，再用`-o`修改。
    
5. **大文件的性能优化**
    
    - 处理大文件时，`sort` 性能很高，但需注意避免不必要的操作；
    - 尽量用`-t`和`-k`指定字段，避免复杂的排序规则；
    - 合并已排序文件时，用`-m`参数，不重新排序，可大幅提升性能。
    
6. **排序规则的优先级**
    
    - 排序规则的优先级：`-n`（数值） > `-M`（月份） > `-V`（版本） > `-h`（人类可读） > 默认字典序；
    - 若同时指定多个排序规则，优先级高的规则生效；
    - 多字段排序时，每个字段可单独指定排序规则，如`-k 2n -k 1`。
    
7. **中文字符的排序**
    
    - 中文字符的排序受系统`locale`影响，可能会按拼音排序，也可能按 Unicode 编码排序；
    - 若需按 ASCII 码顺序排序（忽略 locale），可设置`LC_ALL=C`，即`LC_ALL=C sort test.txt`。
    
8. **`-R`随机排序的随机性**
    
    - `sort -R` 的随机排序是基于哈希值的，每次运行结果不同，但相同的输入会产生相同的输出（若不改变系统随机种子）；
    - 若需真正的随机排序，可结合`shuf`命令，即`shuf test.txt`。
    

## 六、同类命令对比 & 拓展

表格

|命令|核心作用|与 sort 的区别 & 适用场景|||
|:--|:--|:--|---|---|
|`uniq`|去除相邻重复行|优势：简单快速，仅用于去重；劣势：仅对相邻重复行有效，需先排序。去重必须先排序，简单去重用`sort -u`或 `sort|uniq`||
|`awk`|模式驱动的文本处理语言|优势：支持字段提取、数据计算、格式化输出；劣势：简单排序不如 sort 简洁。复杂文本处理用 awk，简单排序用 sort，两者常配合使用|||
|`head`/`tail`|取前 N 行 / 后 N 行|优势：简单快速，仅用于取首尾；劣势：无法排序。取排序后的前 N 行 / 后 N 行用 `sort|head`/`sort|tail`|
|`shuf`|随机打乱行|优势：专门用于随机排序，随机性更强；劣势：无其他排序功能。随机排序优先用 shuf，简单随机用 sort -R|||
|`vim`/`vi`|交互式文本编辑器|优势：支持交互式排序；劣势：非交互式、不适合脚本批量处理。交互式排序用 vim，脚本批量处理用 sort|||
|`cut`|简单字段提取|优势：简单快速，仅用于提取字段；劣势：无法排序。字段提取用 cut，排序用 sort，两者常配合使用|||

## 七、课后练习任务

1. 基础排序练习：创建一个`test.txt`文件，写入几行包含字母的内容，用`sort`按字典序排序。
2. 数值排序练习：创建一个`numbers.txt`文件，写入几行包含数字的内容（如 10、2、100、20），用`sort`默认排序，观察错误；再用`sort -n`按数值序排序，验证正确。
3. 反向排序练习：用`sort -r`反向排序`test.txt`，对比与默认排序的差异。
4. 忽略大小写练习：在`test.txt`中写入几行包含大小写字母的内容（如 Apple、banana、Cherry），用`sort -f`忽略大小写排序，验证效果。
5. 去重练习：在`test.txt`中添加几行重复内容，用`sort -u`去重排序，验证效果；再用`sort test.txt | uniq`对比，验证两者的差异。
6. 指定字段排序练习：创建一个`data.csv`文件（逗号分隔），包含 3 列（姓名、年龄、城市），用`sort -t ',' -k 2n`按年龄数值排序，用`sort -t ',' -k 1`按姓名字典序排序。
7. 自然版本序练习：创建一个`files.txt`文件，写入几行带数字的文件名（如 file1、file2、file10、file20），用`sort`默认排序，观察错误；再用`sort -V`按自然版本序排序，验证正确。
8. 人类可读单位练习：创建一个`sizes.txt`文件，写入几行带单位的大小（如 1K、2M、3G、100M），用`sort`默认排序，观察错误；再用`sort -h`按人类可读单位排序，验证正确。
9. 检查是否已排序练习：用`sort -c`检查排序后的`test.txt`，验证是否已排序；再检查未排序的`test.txt`，观察错误信息。
10. 合并已排序文件练习：创建两个已排序的文件`sorted1.txt`和`sorted2.txt`，用`sort -m`合并它们，验证合并后的结果是否有序。
11. 结合管道练习：用`ls -l | sort -k 5n`按文件大小排序，用`ps aux | sort -k 3nr`按 CPU 使用率降序排序，用`du -sh * | sort -hr`按目录大小降序排序。
12. 随机排序练习：用`sort -R`随机打乱`test.txt`，运行多次，观察结果是否不同；再用`sort -R test.txt | head -5`随机抽样前 5 行。
13. 按多个字段排序练习：在`data.csv`中添加更多行，用`sort -t ',' -k 3 -k 2n`先按城市字典序，再按年龄数值序排序。
14. 直接修改原文件练习：先不加`-o`预览排序效果，确认无误后用`sort test.txt -o test.txt`直接修改原文件，验证修改效果。
