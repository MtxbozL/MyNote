## 一、核心定位

`du` 是 Linux 系统中**定位大文件、分析目录空间占用的核心命令**，属于「02 - 文件目录与路径导航类」，普通用户可查看自己有权限的文件 / 目录。核心作用是**递归统计指定目录 / 文件的实际磁盘占用，支持人类可读单位、大小排序、深度限制、inode 统计、阈值过滤等多种场景**，是磁盘空间清理、大文件定位、目录增长分析、配额管理的必备工具，与`df`（文件系统整体空间监控）形成完美互补，也是 Linux 入门必学的磁盘分析命令。

## 二、语法格式

```bash
# 核心语法1：查看当前目录的磁盘占用（默认递归统计所有子目录）
du [可选参数]

# 核心语法2：查看指定文件/目录的磁盘占用
du [可选参数] 文件/目录路径1 文件/目录路径2 ...
```

> 关键说明：
> 
> 1. 无参数时，默认递归统计**当前目录**及其所有子目录的磁盘占用；
> 2. 输出单位默认是 1K 块，需配合`-h`参数转换为人类可读单位；
> 3. 统计的是**实际磁盘占用**（考虑块大小、稀疏文件、硬链接），而非文件的逻辑大小。

## 三、高频核心参数

| 参数                      | 核心作用                           | 高频使用场景                 |
| :---------------------- | :----------------------------- | :--------------------- |
| `-h, --human-readable`  | 以人类可读的单位（K/M/G/T/P）显示大小        | 日常快速查看目录大小，最核心、最常用的参数  |
| `-s, --summarize`       | 仅显示指定目录的总计大小，不显示子目录            | 快速查看单个目录的总大小，避免输出过多子目录 |
| `-a, --all`             | 显示所有文件的大小，不仅限于目录               | 定位具体大文件，而非仅看目录大小       |
| `-d, --max-depth=N`     | 限制递归统计的目录深度，N=0 等价于`-s`        | 避免深层目录输出过多，仅关注前 N 层目录  |
| `-c, --total`           | 统计所有指定目录 / 文件的 grand total（总计） | 同时统计多个目录时，显示总大小        |
| `-k, --kilobytes`       | 以 1KB 为单位显示大小                  | 脚本解析、统一单位统计            |
| `-m, --megabytes`       | 以 1MB 为单位显示大小                  | 中等大小目录的快速统计            |
| `-g, --gigabytes`       | 以 1GB 为单位显示大小                  | 大目录的快速统计               |
| `-i, --inodes`          | 统计 inode 的使用数量，而非磁盘块           | 排查小文件过多导致的 inode 占用问题  |
| `-t, --threshold=SIZE`  | 仅显示大小超过指定阈值的文件 / 目录            | 过滤小文件，快速定位大文件          |
| `-x, --one-file-system` | 不跨越文件系统边界，仅统计当前文件系统            | 避免统计挂载的其他分区 / 网络文件系统   |
| `-S, --separate-dirs`   | 统计目录大小时，不包含子目录的大小              | 仅看当前目录自身文件的大小，排除子目录    |
| `-L, --dereference`     | 跟随符号链接，统计符号链接指向的目标文件大小         | 统计符号链接指向的实际文件占用        |
| `-P, --no-dereference`  | 不跟随符号链接，仅统计符号链接本身大小（默认行为）      | 仅看符号链接自身的占用            |
| `--time`                | 显示文件 / 目录的最后修改时间               | 结合大小分析，定位最近增长的大文件      |
| `--exclude=PATTERN`     | 排除匹配指定模式的文件 / 目录               | 过滤日志、临时文件等不需要统计的内容     |

## 四、实操示例

### 1. 基础入门示例（新手必练，零门槛上手）

```bash
# 示例1：最基础用法，查看当前目录及其所有子目录的大小（默认1K块单位）
du
# 执行效果：输出当前目录下每个子目录的大小，最后一行是当前目录总计
```

```bash
# 示例2：以人类可读单位显示当前目录大小
du -h
# 执行效果示例：
# 4.0K    ./doc
# 16K     ./log
# 20K     .
```

```bash
# 示例3：仅查看指定目录的总计大小（不显示子目录）
# -s, --summarize  仅显示指定目录的总计大小，不显示子目录
du -sh /home/ubuntu
# 执行效果示例：5.2G    /home/ubuntu
```

```bash
# 示例4：限制递归深度，仅查看前2层目录
du -h -d 2 /data
# 执行效果：仅显示/data目录下前2层子目录的大小，避免输出过深
```

```bash
# 示例5：显示所有文件的大小，不仅限于目录
du -ah /home/ubuntu/linux_practice
# 执行效果：显示目录下每个文件和子目录的大小
```

### 2. 高频日常场景示例

#### 场景 1：定位大文件 / 目录，按大小降序排序（磁盘清理核心）

```bash
# 方式1：查看当前目录下所有子目录的大小，按降序排序，取Top10
du -h -d 1 | sort -hr | head -10

# 方式2：查看所有文件（含子目录）的大小，按降序排序，取Top10
du -ah | sort -hr | head -10

# 方式3：仅显示大于100M的文件/目录，按降序排序
du -ah -t 100M | sort -hr
```

#### 场景 2：同时统计多个目录，并显示总计

```bash
# 同时统计/home、/data、/var/log三个目录的大小，并显示总大小
du -shc /home /data /var/log
# 执行效果示例：
# 5.2G    /home
# 100G    /data
# 2.5G    /var/log
# 107.7G  total
```

#### 场景 3：统计 inode 使用情况，排查小文件过多问题

```bash
# 统计当前目录及其子目录的inode使用数量，按降序排序
du -i -d 1 | sort -nr | head -10

# 以人类可读方式统计inode（需配合awk转换）
du -i -d 1 | awk '{printf "%-10s %s\n", $1, $2}' | sort -nr
```

#### 场景 4：不跨越文件系统，仅统计当前分区

```bash
# 统计/目录的大小，但不统计挂载的其他分区（如/data、/home）
du -shx /
```

#### 场景 5：排除特定文件 / 目录，过滤不需要统计的内容

```bash
# 统计当前目录大小，排除所有.log文件
du -sh --exclude="*.log" .

# 统计当前目录大小，排除log和tmp目录
du -sh --exclude=./log --exclude=./tmp .
```

#### 场景 6：显示文件 / 目录的最后修改时间，结合大小分析

```bash
# 显示文件大小和最后修改时间，按时间排序
du -ah --time | sort -k2 -r
```

### 3. 进阶奇妙用法

#### 奇妙用法 1：一行命令找出当前目录下 Top10 大文件

```bash
# 找出当前目录及其子目录下Top10大文件，显示大小和路径
find . -type f -exec du -h {} + | sort -hr | head -10
```

#### 奇妙用法 2：定时记录目录增长，分析空间变化趋势

```bash
# 1. 编辑crontab定时任务
crontab -e

# 2. 添加配置：每天凌晨2点记录/data目录的大小
0 2 * * * echo "=== $(date +%F %T) ===" >> /var/log/dir_growth.log && du -sh /data >> /var/log/dir_growth.log

# 3. 查看记录的目录增长日志
tail -f /var/log/dir_growth.log
```

#### 奇妙用法 3：结合 xargs，批量删除大于指定大小的文件

```bash
# 找出当前目录下大于1G的.log文件，确认后批量删除
# 第一步：先确认要删除的文件
find . -name "*.log" -type f -size +1G -exec du -h {} +

# 第二步：确认无误后，批量删除
find . -name "*.log" -type f -size +1G -delete
```

#### 奇妙用法 4：统计稀疏文件的实际磁盘占用

```bash
# 稀疏文件：逻辑大小很大，但实际磁盘占用很小
# 创建一个1G的稀疏文件
dd if=/dev/zero of=sparse_file bs=1 count=0 seek=1G

# 查看逻辑大小（ls显示的是逻辑大小）
ls -lh sparse_file
# 执行效果：1.0G

# 查看实际磁盘占用（du显示的是实际占用）
du -h sparse_file
# 执行效果：0（或很小的数值）
```

#### 奇妙用法 5：统计目录中每种文件类型的总大小

```bash
# 统计当前目录下.jpg、.png、.mp4、.log等每种文件类型的总大小
echo "=== 文件类型大小统计 ==="
for ext in jpg png mp4 log txt; do
    SIZE=$(find . -name "*.$ext" -type f -exec du -k {} + | awk '{sum += $1} END {print sum}')
    SIZE_H=$(echo $SIZE | awk '{printf "%.2fM", $1/1024}')
    echo "$ext 文件总大小：$SIZE_H"
done
```

## 五、避坑提示 & 注意事项

1. **最大认知误区：du 显示的大小 ≠ ls 显示的大小**
    
    - `ls -l`：显示文件的**逻辑大小**（文件内容的实际字节数）；
    - `du`：显示文件的**实际磁盘占用**（考虑文件系统块大小、稀疏文件、硬链接）；
    - 典型差异场景：
        
        - 稀疏文件：ls 显示 1G，du 显示 0；
        - 小文件：文件系统块大小为 4K，1 字节的小文件，ls 显示 1 字节，du 显示 4K；
        - 硬链接：多个硬链接指向同一个文件，du 仅统计一次。
        
    
2. **新手高频踩坑：默认单位是 1K 块，不是字节**
    
    - 无参数时，`du` 默认以 1K 块为单位显示大小，新手极易误读；
    - 强制要求：日常查看**必须加`-h`参数**，转换为人类可读的 K/M/G/T 单位，避免误读。
    
3. **硬链接的重复统计坑**
    
    - 多个硬链接指向同一个文件时，`du` 默认仅统计一次，不会重复计算；
    - 若要统计所有硬链接的占用，需加`-l`参数（但实际不会增加磁盘占用）。
    
4. **符号链接的处理坑**
    
    - `du` 默认不跟随符号链接，仅统计符号链接本身的大小（几字节）；
    - 若要统计符号链接指向的目标文件大小，需加`-L`参数；
    - 注意：加`-L`后可能导致循环链接（A 链接 B，B 链接 A），使`du`陷入死循环。
    
5. **大目录的性能坑**
    
    - 统计包含数百万文件的大目录时，`du` 会非常慢，因为需要递归遍历所有文件；
    - 优化建议：
        
        - 加`-d`参数限制深度，先看上层目录；
        - 用`ncdu`（交互式磁盘分析工具）替代`du`，速度更快且支持交互；
        - 用`--exclude`参数排除不需要统计的目录（如日志、临时文件）。
        
    
6. **权限不足的坑**
    
    - 统计系统目录（如`/root`、`/var`）时，普通用户会因权限不足无法读取部分子目录，导致统计结果偏小；
    - 统计系统目录需加`sudo`，确保能读取所有子目录。
    
7. **`--max-depth`与`-d`的兼容性**
    
    - `-d`是`--max-depth`的简写，GNU du 支持，但老旧 Unix 系统可能不支持；
    - 跨平台脚本需使用`--max-depth`，保证兼容性。
    
8. **`-t`阈值的单位问题**
    
    - `-t`参数的阈值单位默认是 1K 块，需配合`-h`使用人类可读单位；
    - 示例：`du -h -t 100M`（仅显示大于 100M 的文件 / 目录）。
    

## 六、同类命令对比 & 拓展

表格

|命令|核心作用|与 du 的区别 & 适用场景|
|:--|:--|:--|
|`df`|查看文件系统整体磁盘空间|df 查看文件系统层面的整体空间（含保留空间、已删除但未释放的文件）；du 查看具体文件 / 目录的实际占用。整体空间监控用 df，定位大文件用 du，两者互补|
|`ncdu`|交互式磁盘空间分析工具|是 du 的进阶替代方案，支持交互式浏览、排序、删除大文件，速度更快，界面更直观，深度排查大文件优先用 ncdu，简单统计用 du|
|`ls -l`/`ls -lh`|查看文件的逻辑大小与元数据|ls 显示文件的逻辑大小，du 显示实际磁盘占用；ls 用于查看文件属性，du 用于统计磁盘占用，两者互补|
|`find`|查找文件 / 目录|find 可按大小、时间、类型等条件查找文件，结合`-exec du`可实现更灵活的统计；du 专注于空间统计，find 专注于文件查找，两者配合使用|
|`tree`|以树形结构显示目录|tree 可直观显示目录结构，结合`-h`可显示大小，但统计功能弱于 du；查看目录结构用 tree，统计空间用 du|
|`quota`|查看用户 / 组的磁盘配额|用于查看用户 / 组的配额使用情况；du 用于查看具体文件 / 目录的占用，配额管理用 quota，空间分析用 du|
|`filelight`/`baobab`|图形化磁盘空间分析工具|图形化界面，直观展示磁盘占用分布；du 是命令行工具，适合远程服务器、无图形界面环境|

## 七、课后练习任务

1. 基础查看练习：分别用`du`、`du -h`、`du -sh`三种方式查看你的家目录大小，对比输出结果的差异。
2. 指定深度练习：用`du -h -d 2`查看`/var`目录的前 2 层子目录大小，找出最大的子目录。
3. 排序找大文件练习：用`du -ah | sort -hr | head -10`找出当前目录下 Top10 大的文件 / 目录，记录结果。
4. 统计总计练习：同时统计`/home`、`/tmp`、`/var/log`三个目录的大小，并显示总大小。
5. inode 查看练习：用`du -i -d 1`查看当前目录下各子目录的 inode 使用数量，找出 inode 占用最多的子目录。
6. 不跨文件系统练习：用`du -shx /`统计根目录的大小，确认不包含挂载的其他分区。
7. 定时记录练习：配置一个 crontab 定时任务，每天凌晨 1 点记录你的家目录大小，保存到`/var/log/home_growth.log`日志文件中。
