## 一、核心定位

`telnet` 和 `nc`（netcat）是 Linux 上**两大经典的传输层网络测试工具**，均为 TCP/UDP 协议的前端，功能各有侧重：

- **telnet**：早期的远程登录工具，因**明文传输不安全**已很少用于登录，现在主要用来**快速测试 TCP 端口连通性**，语法极简。
- **nc**（netcat）：被称为 **“网络瑞士军刀”**，功能远强于 telnet，可用于端口测试、文件传输、端口监听、简单聊天、UDP 测试等，适合复杂网络场景。

---

## 二、telnet 详解

### 1. 核心概念

- **主要用途**：现在仅用于**测试 TCP 端口是否开放**，远程登录功能已被 SSH 替代。
- **默认状态**：多数发行版默认**不安装**，需手动安装。

### 2. 语法格式

bash

运行

```
telnet [选项] 主机 端口
```

> 关键说明：
> 
> 1. **主机**：必选，可以是 IPv4/IPv6 地址或域名。
> 2. **端口**：必选，TCP 端口号（如 22/SSH、80/HTTP、53/DNS）。
> 3. **退出方法**：按 `Ctrl+]` 进入 telnet 命令模式，然后输入 `quit` 退出（新手容易忘！）。

### 3. 高频核心命令

表格

|功能|命令|说明|
|:--|:--|:--|
|**测试 TCP 端口连通性**|`telnet 8.8.8.8 53`|测试 Google DNS 的 53/TCP 端口是否开放|
||`telnet 192.168.1.1 22`|测试局域网网关的 22/SSH 端口|
||`telnet www.baidu.com 80`|测试百度的 80/HTTP 端口|

## 三、nc 详解

### 1. 核心概念

- **主要用途**：端口测试、文件传输、端口监听、简单聊天、UDP 测试等，功能极丰富。
- **默认状态**：多数发行版默认**不安装**，需手动安装：
    
    - Debian/Ubuntu：`sudo apt install netcat-openbsd`（推荐）或 `netcat-traditional`。
    - RHEL/CentOS：`sudo yum install nc`。
    

### 2. 语法格式

bash

运行

```
nc [选项] 主机 端口
```

> 关键说明：
> 
> 1. **主机**：可选（监听模式 `-l` 不需要）。
> 2. **端口**：必选（监听模式 `-l` 为监听端口）。

### 3. 高频核心选项与命令

表格

|功能|选项 / 命令|说明|
|:--|:--|:--|
|**测试 TCP 端口连通性（最常用）**|`-z`（零 I/O 模式，仅测试不发送数据）、`-v`（详细）|`nc -zv 8.8.8.8 53` 测试 Google DNS 的 53/TCP 端口|
|**测试 UDP 端口**|`-u`（UDP 模式）|`nc -zuv 8.8.8.8 53` 测试 Google DNS 的 53/UDP 端口|
|**端口监听（服务端）**|`-l`（监听模式）|`nc -l 8888` 监听本地 8888/TCP 端口|
|**连接监听端口（客户端）**|无特殊选项|`nc 192.168.1.10 8888` 连接到 192.168.1.10 的 8888 端口|
|**文件传输（发送端）**|重定向输入|`nc 192.168.1.10 8888 < test.txt` 发送 test.txt 到 192.168.1.10 的 8888 端口|
|**文件传输（接收端）**|重定向输出|`nc -l 8888 > test.txt` 监听 8888 端口，接收数据保存为 test.txt|
|**简单聊天（双方）**|无特殊选项|服务端：`nc -l 8888`；客户端：`nc 192.168.1.10 8888`，双方输入文字即可聊天|

## 四、避坑提示与注意事项

### telnet

1. **不安全，不要用来远程登录**
    
    - telnet 是**明文传输**，密码和数据都能被抓包看到，远程登录已被 SSH 完全替代；
    - 推荐：仅用 telnet 测试 TCP 端口，不要用来登录。
    
2. **退出要按 `Ctrl+]` 再 `quit`，新手容易忘**
    
    - 直接按 `Ctrl+C` 可能无法退出 telnet；
    - 正确方法：先按 `Ctrl+]` 进入 telnet 命令模式，然后输入 `quit` 回车退出。
    
3. **不是默认安装，要先安装**
    
    - 多数发行版默认不安装 telnet；
    - 安装：Debian/Ubuntu `sudo apt install telnet`，RHEL/CentOS `sudo yum install telnet`。
    

### nc

1. **监听模式要注意安全，不要在公网随便监听**
    
    - 在公网监听端口可能被攻击或滥用；
    - 推荐：仅在局域网或测试环境使用监听模式，用完及时关闭。
    
2. **UDP 模式用 `-u`，和 TCP 不同**
    
    - 默认是 TCP 模式，测试 UDP 必须加 `-u`；
    - 典型错误：`nc -zv 8.8.8.8 53` 测试的是 TCP，不是 UDP；
    - 正确示例：`nc -zuv 8.8.8.8 53` 测试 UDP。
    
3. **文件传输要先启动接收端，再启动发送端**
    
    - 接收端要先监听端口，发送端再连接；
    - 典型错误：先启动发送端，接收端还没监听，连接失败；
    - 正确示例：先启动接收端 `nc -l 8888 > test.txt`，再启动发送端 `nc 192.168.1.10 8888 < test.txt`。

## 五、同类对比

表格

|特性|telnet|nc|
|:--|:--|:--|
|**核心用途**|仅测试 TCP 端口|端口测试、文件传输、监听、聊天、UDP 等|
|**功能丰富度**|低（仅端口测试）|高（网络瑞士军刀）|
|**语法复杂度**|极简|中等（选项多）|
|**默认安装**|多数不安装|多数不安装|
|**适用场景**|快速简单的 TCP 端口测试|复杂网络场景、多种功能需求|

## 六、课后练习

### telnet 练习

1. 安装 telnet（如果未安装）。
2. 测试本地回环地址的 22/SSH 端口：`telnet 127.0.0.1 22`。
3. 测试局域网网关的 53/DNS 端口：`telnet 你的网关IP 53`。
4. 测试公网百度的 80/HTTP 端口：`telnet www.baidu.com 80`。
5. 练习退出 telnet：按 `Ctrl+]` 然后 `quit`。

### nc 练习

1. 安装 nc（如果未安装）。
2. 测试 Google DNS 的 53/TCP 端口：`nc -zv 8.8.8.8 53`。
3. 测试 Google DNS 的 53/UDP 端口：`nc -zuv 8.8.8.8 53`。
4. 端口监听与连接：
    
    - 服务端：`nc -l 8888`。
    - 客户端：`nc 127.0.0.1 8888`，输入文字测试。
    
5. 文件传输：
    
    - 接收端：`nc -l 8888 > test.txt`。
    - 发送端：创建一个 test.txt 文件，然后 `nc 127.0.0.1 8888 < test.txt`，验证 test.txt 是否传输成功。
    
6. 简单聊天：
    - 服务端：`nc -l 8888`。
    - 客户端：`nc 127.0.0.1 8888`，双方输入文字聊天。
