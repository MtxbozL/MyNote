## 一、核心定位

`sed` 是 Linux 系统中**最核心、最强大的非交互式流文本编辑器**，属于「文本处理类」，无任何权限要求，普通用户可直接处理自己有权限的文件。核心作用是**按行扫描文本，支持正则表达式匹配、文本替换、行删除、行插入 / 追加、行替换、字符转换、打印行号等功能**，是 Shell 脚本中文本批量处理、格式转换、日志清洗、配置修改的核心工具，也是 Linux 入门必学的高级文本处理命令。

> 补充说明：`sed` 名称来源于 “stream editor”（流编辑器），按行处理文本，非交互式，支持管道输入，与`grep`、`awk`并称为 Linux 文本处理三剑客。

## 二、语法格式

```bash
# 核心语法1：处理文件，输出到标准输出（不修改原文件）
sed [可选参数] '编辑脚本' 文件1 文件2 ...

# 核心语法2：处理标准输入（管道输入）
命令 | sed [可选参数] '编辑脚本'

# 核心语法3：直接修改原文件（危险，需谨慎）
sed -i [可选参数] '编辑脚本' 文件
```

> 关键语法说明：
> 
> 1. **编辑脚本**：必选，用单引号包裹，包含地址范围和编辑命令，如`s/old/new/g`（全局替换）、`1,5d`（删除 1-5 行）；
> 2. **地址范围**：可选，指定编辑的行，如`1`（第 1 行）、`$`（最后一行）、`1,5`（1-5 行）、`/pattern/`（匹配 pattern 的行）；
> 3. **默认行为**：输出所有行（包括未编辑的行），仅修改匹配的行；
> 4. **单引号包裹**：`'编辑脚本'`必须用单引号包裹，避免 Shell 解析其中的`$`、`/`、`*`等特殊字符。

## 三、高频核心参数与编辑命令

### 1. 核心启动参数

| 参数                                   | 核心作用                                                  | 高频使用场景                                    |
| :----------------------------------- | :---------------------------------------------------- | :---------------------------------------- |
| 无参数（直接执行`sed '脚本' 文件`）               | 基本处理，输出所有行到标准输出，不修改原文件                                | 日常文本处理、预览修改效果，最核心、最常用的用法                  |
| `-n, --quiet, --silent`              | 仅输出**编辑命令显式打印的行**，不输出默认行                              | 配合`p`命令打印特定行、避免冗余输出                       |
| `-i[SUFFIX], --in-place[=SUFFIX]`    | 直接**修改原文件**，若指定 SUFFIX 则先备份原文件为`原文件SUFFIX`（如`-i.bak`） | 批量修改文件、配置更新，**高危操作，需谨慎**                  |

| `--help`                             | 输出 sed 帮助信息                                           | 快速查看参数与命令用法                               |

### 2. 核心编辑命令

|命令|核心作用|高频使用场景|
|:--|:--|:--|
|`s/旧/新/[g]`|**替换**命令，将每行的第一个 “旧” 替换为 “新”，加`g`则全局替换（每行所有匹配）|文本替换、格式转换、配置修改，最核心、最常用的命令|
|`d`|**删除**命令，删除匹配的行|删除空行、删除注释行、删除特定行|
|`i\内容`|**插入**命令，在匹配行的**前面**插入内容|在文件开头插入内容、在特定行前插入注释|
|`a\内容`|**追加**命令，在匹配行的**后面**追加内容|在文件末尾追加内容、在特定行后追加配置|
|`c\内容`|**替换行**命令，将匹配的行**整行替换**为内容|替换整行配置、更新特定行|

|`y/字符集1/字符集2/`|**字符转换**命令，将字符集 1 中的字符逐个替换为字符集 2 中的对应字符|大小写转换、字符替换|
|`=`|打印匹配行的**行号**|显示行号、定位问题行|
|`n`|读取**下一行**，覆盖当前模式空间|处理多行、隔行操作|

## 四、实操示例

### 1. 基础入门示例（新手必练，零门槛上手）

```bash
# 示例1：最基础用法，替换每行的第一个"old"为"new"
sed 's/old/new/' test.txt
# 执行效果：输出test.txt的内容，每行的第一个"old"被替换为"new"，原文件不修改
```

```bash
# 示例2：全局替换，替换每行的所有"old"为"new"
sed 's/old/new/g' test.txt
# 执行效果：每行的所有"old"都被替换为"new"
```

```bash
# 示例3：删除特定行
# 删除第5行
sed '5d' test.txt
# 删除1-5行
sed '1,5d' test.txt
# 删除最后一行
sed '$d' test.txt
# 删除包含"error"的行
sed '/error/d' test.txt
```

```bash
# 示例4：仅打印特定行（配合-n）
# 仅打印第10行
sed -n '10p' test.txt
# 仅打印1-5行
sed -n '1,5p' test.txt
# 仅打印包含"error"的行
sed -n '/error/p' test.txt
```

```bash
# 示例5：打印行号
# 打印所有行的行号
sed '=' test.txt
# 仅打印包含"error"的行的行号
sed -n '/error/=' test.txt
```

### 2. 高频日常场景示例

#### 场景 1：直接修改原文件（先备份）

```bash
# 方式1：直接修改原文件，不备份（危险，需谨慎）
sed -i 's/old/new/g' test.txt

# 方式2：修改原文件，先备份为test.txt.bak（推荐）
sed -i.bak 's/old/new/g' test.txt
# 执行效果：test.txt被修改，同时生成test.txt.bak备份文件
```

#### 场景 2：替换特定行的内容

```bash
# 仅替换第10行的"old"为"new"
sed '10s/old/new/g' test.txt

# 替换1-5行的"old"为"new"
sed '1,5s/old/new/g' test.txt

# 替换包含"error"的行的"old"为"new"
sed '/error/s/old/new/g' test.txt
```

#### 场景 3：删除空行

```bash
# 删除所有空行
sed '/^$/d' test.txt

# 删除包含空格或制表符的空行
sed '/^[[:space:]]*$/d' test.txt
```

#### 场景 4：删除注释行（以 #开头）

bash

运行

```
# 删除以#开头的注释行
sed '/^#/d' test.txt

# 删除以#开头的注释行和空行
sed '/^#/d;/^$/d' test.txt
```

#### 场景 5：在文件开头 / 末尾插入内容

bash

运行

```
# 在文件开头插入一行内容
sed '1i\# 这是文件开头' test.txt

# 在文件末尾追加一行内容
sed '$a\# 这是文件末尾' test.txt
```

#### 场景 6：结合其他命令（管道输入）

bash

运行

```
# 结合ps，删除包含"grep"的行
ps aux | sed '/grep/d'

# 结合df，替换"%"为" percent"
df -h | sed 's/%/ percent/g'

# 结合grep，先过滤再替换
grep 'error' test.log | sed 's/error/ERROR/g'
```

### 3. 进阶奇妙用法

#### 奇妙用法 1：正则分组与反向引用

bash

运行

```
# 分组匹配，反向引用（\1代表第一个分组，\2代表第二个分组）
# 将"first last"替换为"last, first"
echo "first last" | sed 's/\(.*\) \(.*\)/\2, \1/'
# 执行效果：last, first
```

#### 奇妙用法 2：字符转换（y 命令）

bash

运行

```
# 大小写转换，将所有小写字母转换为大写
sed 'y/abcdefghijklmnopqrstuvwxyz/ABCDEFGHIJKLMNOPQRSTUVWXYZ/' test.txt
# 或更简洁的写法
sed 'y/a-z/A-Z/' test.txt

# 将所有大写字母转换为小写
sed 'y/A-Z/a-z/' test.txt
```

#### 奇妙用法 3：批量修改多个文件

bash

运行

```
# 批量修改当前目录下所有.txt文件，先备份为.bak
sed -i.bak 's/old/new/g' *.txt
```

#### 奇妙用法 4：在每行开头 / 结尾添加内容

bash

运行

```
# 在每行开头添加"# "
sed 's/^/# /' test.txt

# 在每行结尾添加" [END]"
sed 's/$/ [END]/' test.txt
```

#### 奇妙用法 5：替换特定分隔符（避免 / 冲突）

bash

运行

```
# 当替换内容包含/时，使用其他分隔符（如#、@、|）
# 替换"/old/path"为"/new/path"
sed 's#/old/path#/new/path#g' test.txt
# 或
sed 's@/old/path@/new/path@g' test.txt
```

#### 奇妙用法 6：打印匹配行的前后行

bash

运行

```
# 打印包含"error"的行及其后2行
sed -n '/error/,+2p' test.txt

# 打印包含"error"的行及其前2行（需结合grep -A -B）
# 推荐直接用grep -C 2 'error' test.txt
```

## 五、避坑提示 & 注意事项

1. **新手最高频致命坑：编辑脚本必须用单引号包裹**
    
    - 编辑脚本中若包含`$`、`/`、`*`、`?`、`()`、`|`等特殊字符，**必须用单引号包裹**，否则 Shell 会先解析这些字符，导致 sed 接收到的脚本错误；
    - 错误示例：`sed s/$/ [END]/ test.txt`（Shell 会解析`$`为变量结尾，导致脚本错误）；
    - 正确示例：`sed 's/$/ [END]/' test.txt`。
    
2. **`-i`直接修改原文件无提示，无法恢复**
    
    - `-i` 会直接修改原文件，无任何提示，且 Linux 默认没有回收站，修改后数据几乎无法恢复；
    - 强制要求：
        
        - 使用`-i`前，**必须先不加`-i`执行一遍，预览修改效果**；
        - 优先使用`-i.bak`，先备份原文件，确认修改无误后再删除备份；
        - 重要数据必须提前备份。
        
    
3. **替换命令的分隔符选择**
    
    - `s`命令的默认分隔符是`/`，若替换内容包含`/`（如路径），必须使用其他分隔符（如`#`、`@`、`|`），避免冲突；
    - 错误示例：`sed 's//old/path//new/path/g' test.txt`（分隔符冲突，语法错误）；
    - 正确示例：`sed 's#/old/path#/new/path#g' test.txt`。
    
4. **正则表达式的特殊字符转义**
    
    - 若需匹配正则表达式的特殊字符（如`.`、`*`、`?`、`+`、`|`、`()`、`[]`、`^`、`$`），**必须用反斜杠`\`转义**；
    - 错误示例：`sed 's/error.log/error.txt/g' test.txt`（`.`会匹配任意字符）；
    - 正确示例：`sed 's/error\.log/error\.txt/g' test.txt`。
    
5. **默认输出所有行的坑**
    
    - 默认情况下，`sed` 会输出所有行（包括未编辑的行），仅修改匹配的行；
    - 若需仅输出编辑的行，必须加`-n`参数配合`p`命令；
    - 典型错误：`sed '10p' test.txt`会输出所有行，同时第 10 行输出两次；
    - 正确示例：`sed -n '10p' test.txt`仅输出第 10 行。
    
6. **地址范围的写法**
    
    - 地址范围用逗号分隔，如`1,5`（1-5 行）、`/start/,/end/`（从匹配 start 的行到匹配 end 的行）；
    - `$`代表最后一行，如`sed '$d' test.txt`删除最后一行；
    - 注意：地址范围是闭区间，包含起始和结束行。
    
7. **大文件的性能优化**
    
    - 处理大文件时，`sed` 性能很高，但需注意避免不必要的操作；
    - 尽量使用简单的编辑命令，避免复杂的正则表达式；
    - 直接修改原文件时，优先使用`-i`，避免管道的额外开销。
    
8. **二进制文件的处理**
    
    - `sed` 是文本编辑器，直接处理二进制文件会导致文件损坏；
    - 解决方法：不要用 sed 处理二进制文件，二进制文件处理用`hexdump`、`dd`等工具。
    
9. **扩展正则表达式的使用**
    
    - 基本正则（BRE）和扩展正则（ERE）的语法略有不同，ERE 支持`|`（或）、`+`（一次或多次）、`?`（零次或一次）、`()`（分组）、`{n,m}`（重复次数）等，无需转义；
    - 若需使用这些功能，必须加`-r`或`-E`参数。
    

## 六、同类命令对比 & 拓展

表格

|命令|核心作用|与 sed 的区别 & 适用场景|
|:--|:--|:--|
|`awk`|模式驱动的文本处理语言|优势：支持字段提取、数据计算、格式化输出、数组操作；劣势：简单文本替换不如 sed 简洁。简单文本替换、行删除用 sed，复杂文本处理、数据计算用 awk，两者常配合使用|
|`grep`|按模式匹配行|优势：速度极快，仅用于模式匹配；劣势：无法修改文本。简单模式匹配用 grep，模式匹配后修改用 sed，两者常配合使用|
|`tr`|字符转换、删除、压缩|优势：简单快速，仅用于字符级操作；劣势：无法行级操作、无法正则匹配。简单字符转换、删除用 tr，复杂文本替换、行操作用 sed|
|`cut`|简单字段提取|优势：简单快速，仅用于提取字段；劣势：无法修改文本、无法正则匹配。简单固定分隔符的字段提取用 cut，复杂处理用 sed/awk|
|`head`/`tail`|查看文件开头 / 结尾|优势：简单快速，仅用于查看开头 / 结尾；劣势：无法修改文本。查看开头 / 结尾用 head/tail，修改文本用 sed|
|`vim`/`vi`|交互式文本编辑器|优势：功能强大，支持交互式编辑；劣势：非交互式、不适合脚本批量处理。交互式编辑用 vim，脚本批量处理用 sed|
|`perl`|全能脚本语言|优势：功能比 sed 更强大，支持复杂数据结构、正则表达式更强；劣势：学习曲线更陡，脚本更复杂。简单文本处理用 sed，复杂任务用 perl|

## 七、课后练习任务

1. 基础替换练习：创建一个`test.txt`文件，写入几行包含 "old" 的内容，用`sed 's/old/new/' test.txt`替换每行的第一个 "old"，再用`sed 's/old/new/g' test.txt`全局替换。
2. 删除行练习：用`sed`删除`test.txt`的第 3 行，删除 1-5 行，删除包含 "old" 的行，删除所有空行。
3. 打印行练习：用`sed -n`仅打印`test.txt`的第 10 行，仅打印 1-5 行，仅打印包含 "old" 的行。
4. 直接修改文件练习：先不加`-i`预览修改效果，确认无误后用`sed -i.bak 's/old/new/g' test.txt`修改原文件，验证备份文件是否生成。
5. 插入内容练习：用`sed`在`test.txt`的开头插入一行 "# 这是开头"，在末尾追加一行 "# 这是末尾"。
6. 字符转换练习：用`sed 'y/a-z/A-Z/' test.txt`将所有小写字母转换为大写，再用`sed 'y/A-Z/a-z/' test.txt`转换回小写。
7. 正则分组练习：创建一个包含 "first last" 格式的文件，用`sed 's/\(.*\) \(.*\)/\2, \1/'`将其替换为 "last, first" 格式。
8. 结合管道练习：用`ps aux | sed '/grep/d'`删除包含 "grep" 的进程行，用`df -h | sed 's/%/ percent/g'`替换 "%" 为 "percent"。
9. 批量修改练习：创建`file1.txt`、`file2.txt`、`file3.txt`三个文件，用`sed -i.bak 's/old/new/g' *.txt`批量修改它们，验证修改效果和备份文件。
10. 替换分隔符练习：创建一个包含路径的文件，用`sed 's#/old/path#/new/path#g'`替换路径，验证分隔符选择的重要性。
