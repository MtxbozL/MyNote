## 一、核心定位

`awk` 是 Linux 系统中**最强大、最全能的文本处理与数据提取工具**，属于「文本处理类」，无任何权限要求，普通用户可直接处理文本文件或标准输入。核心作用是**按行扫描文本，自动分割字段，支持模式匹配、条件判断、数据提取、格式化输出、简单数值计算、数组操作等功能**，是 Shell 脚本中文本分析、日志处理、数据清洗、报告生成的核心工具，也是 Linux 入门必学的高级文本处理命令。

> 补充说明：`awk` 名称来源于其三位创始人 Alfred Aho、Peter Weinberger、Brian Kernighan 的姓氏首字母；Linux 系统中默认使用的是`gawk`（GNU awk），功能比标准 awk 更强大。

## 二、语法格式

```bash
# 核心语法1：处理文件
awk [可选参数] '模式{动作}' 文件1 文件2 ...

# 核心语法2：处理标准输入（管道输入）
命令 | awk [可选参数] '模式{动作}'
```

> 关键语法说明：
> 
> 1. **模式（Pattern）**：可选，用于匹配行，满足条件的行才执行动作，如`/error/`（匹配包含 error 的行）、`$3>100`（第 3 字段大于 100）；
> 2. **动作（Action）**：必选，用`{}`包裹，对匹配行执行的操作，如`print $0`（打印整行）、`print $1,$3`（打印第 1 和第 3 字段）；
> 3. **字段（Field）**：默认按 **空白字符（空格、制表符）** 分割行，`$0`代表整行，`$1`代表第 1 字段，`$2`代表第 2 字段，`$NF`代表最后一个字段，`NR`代表当前行号，`NF`代表当前行的字段数；
> 4. **单引号包裹**：`'模式{动作}'`必须用单引号包裹，避免 Shell 解析其中的特殊字符。

## 三、高频核心参数与内置变量

### 1. 核心参数

|参数|核心作用|高频使用场景|
|:--|:--|:--|
|`-F, --field-separator=FS`|指定**输入字段分隔符**，支持字符串或正则表达式|处理非空白分隔的文本（如逗号分隔的 CSV、冒号分隔的 /etc/passwd）|
|`-v, --assign=VAR=VAL`|在 awk 脚本执行前**定义变量**|向 awk 脚本传递外部变量（如 Shell 变量）|
|`-f, --file=PROGFILE`|从**文件中读取 awk 脚本**，而非命令行输入|复杂 awk 脚本的管理，避免命令行过长|
|`--help`|输出 awk 帮助信息|快速查看参数与内置变量用法|
|`--version`|输出 awk 版本信息|确认是否为 gawk 及版本兼容性|

### 2. 核心内置变量

|变量|核心作用|高频使用场景|
|:--|:--|:--|
|`$0`|当前行的完整内容|打印整行、处理整行文本|
|`$1, $2, ..., $n`|当前行的第 1、第 2、...、第 n 个字段|提取指定字段数据|
|`NR`|当前处理的**行号**（Number of Record）|显示行号、统计行数|
|`NF`|当前行的**字段数**（Number of Field）|显示字段数、提取最后一个字段（`$NF`）|
|`FS`|输入字段分隔符（Field Separator），默认空白|等价于`-F`参数，在脚本中动态修改|
|`OFS`|输出字段分隔符（Output Field Separator），默认空格|自定义输出字段的分隔符（如逗号、制表符）|
|`RS`|输入记录分隔符（Record Separator），默认换行符|处理非换行分隔的记录（如多行合并的记录）|
|`ORS`|输出记录分隔符（Output Record Separator），默认换行符|自定义输出记录的分隔符（如空格、分号）|
|`FILENAME`|当前处理的文件名|处理多文件时区分不同文件|

## 四、实操示例

### 1. 基础入门示例

```bash
# 示例1：打印整行（等价于cat）
awk '{print $0}' test.txt
# 或省略$0，默认打印整行
awk '{print}' test.txt
```

```bash
# 示例2：打印指定字段
# 打印第1和第3字段
awk '{print $1, $3}' test.txt
# 打印最后一个字段
awk '{print $NF}' test.txt
# 打印倒数第二个字段
awk '{print $(NF-1)}' test.txt
```

```bash
# 示例3：打印行号和字段数
# 打印行号、字段数、整行内容
awk '{print NR, NF, $0}' test.txt
```

```bash
# 示例4：指定输入字段分隔符（处理/etc/passwd文件）
# /etc/passwd以冒号分隔，打印第1（用户名）和第6（家目录）字段
awk -F ':' '{print $1, $6}' /etc/passwd
```

```bash
# 示例5：指定输出字段分隔符
# 输入以冒号分隔，输出以制表符分隔
awk -F ':' -v OFS='\t' '{print $1, $6}' /etc/passwd
```

### 2. 高频日常场景示例

#### 场景 1：模式匹配（包含 / 不包含某字符串）

```BASH
# 匹配包含"error"的行，打印整行
awk '/error/' test.log
# 等价于
awk '/error/{print $0}' test.log

# 匹配不包含"error"的行
awk '!/error/' test.log

# 匹配第2字段等于"GET"的行
awk '$2 == "GET"' access.log

# 匹配第3字段大于100的行
awk '$3 > 100' data.txt
```

#### 场景 2：条件判断与多条件组合

```BASH
# 单条件判断：第3字段大于100则打印第1和第3字段
awk '$3 > 100 {print $1, $3}' data.txt

# 多条件组合：第2字段等于"GET"且第3字段大于100
awk '$2 == "GET" && $3 > 100 {print $1, $3}' access.log

# 条件分支：第3字段大于100打印"高"，否则打印"低"
awk '{if ($3 > 100) print $1, "高"; else print $1, "低"}' data.txt
```

#### 场景 3：格式化输出（printf）

```BASH
# 格式化输出，指定宽度和对齐方式
# %-10s：左对齐10位字符串，%5d：右对齐5位整数，%.2f：2位小数
awk '{printf "%-10s %5d %.2f\n", $1, $2, $3}' data.txt
```

#### 场景 4：简单数值计算（求和、平均值、最大值、最小值）

```BASH
# 计算第3字段的总和
awk '{sum += $3} END {print "总和：", sum}' data.txt

# 计算第3字段的平均值
awk '{sum += $3; count++} END {print "平均值：", sum/count}' data.txt

# 计算第3字段的最大值
awk 'BEGIN {max=0} {if ($3 > max) max=$3} END {print "最大值：", max}' data.txt

# 计算第3字段的最小值
awk 'BEGIN {min=999999} {if ($3 < min) min=$3} END {print "最小值：", min}' data.txt
```

### 3. 进阶奇妙用法

#### 奇妙用法 1：处理 CSV 文件（逗号分隔）

```BASH
# 处理逗号分隔的CSV文件，打印第2和第4字段
awk -F ',' '{print $2, $4}' data.csv

# 处理带引号的CSV文件（字段包含逗号）
awk -F '","' '{gsub(/"/,"",$1); gsub(/"/,"",$NF); print $2, $4}' data.csv
```

#### 奇妙用法 2：统计文本中单词出现的频率

```BASH
# 统计文本中每个单词的出现次数，按次数降序排序
awk '{for (i=1; i<=NF; i++) count[$i]++} END {for (word in count) print count[word], word}' test.txt | sort -nr
```

#### 奇妙用法 3：结合其他命令（如 ls、ps、df）

```BASH
# 结合ls，统计当前目录下所有文件的总大小
ls -l | awk 'BEGIN {sum=0} /^-/{sum += $5} END {print "总大小：", sum/1024/1024, "MB"}'

# 结合ps，统计每个用户的进程数
ps aux | awk 'NR>1 {count[$1]++} END {for (user in count) print user, count[user]}'

# 结合df，提取磁盘使用率超过80%的分区
df -h | awk 'NR>1 {gsub(/%/,"",$5); if ($5 > 80) print $1, $5"%"}'
```

#### 奇妙用法 4：替换文本中的字符串

```BASH
# 替换每行中的"old"为"new"
awk '{gsub(/old/, "new"); print}' test.txt

# 仅替换第2字段中的"old"为"new"
awk '{gsub(/old/, "new", $2); print}' test.txt
```

#### 奇妙用法 5：生成格式化报告

```BASH
# 生成带表头、分隔线、汇总的格式化报告
awk '
BEGIN {
    print "=============================="
    printf "%-10s %5s %10s\n", "用户名", "ID", "家目录"
    print "------------------------------"
}
-F ":" {
    printf "%-10s %5s %10s\n", $1, $3, $6
    count++
}
END {
    print "------------------------------"
    print "总用户数：", count
    print "=============================="
}' /etc/passwd
```

## 五、避坑提示 & 注意事项

1. **新手最高频踩坑：字段分隔符的默认行为**
    
    - `awk` 默认的输入字段分隔符是**任意空白字符（空格、制表符、换行符）**，且连续的空白字符会被视为一个分隔符；
    - 若文本以单个空格或制表符分隔，默认行为没问题；但若以其他字符（如逗号、冒号）分隔，**必须加`-F`参数指定分隔符**；
    - 典型错误：处理`/etc/passwd`时不加`-F ':'`，导致整行被视为一个字段。
    
2. **单引号与双引号的混淆**
    
    - `'模式{动作}'`必须用**单引号**包裹，避免 Shell 解析其中的`$`、`{}`、`!`等特殊字符；
    - 若需在 awk 中使用 Shell 变量，必须用`-v`参数传递，不能直接在单引号内使用`$VAR`；
    - 错误示例：`awk '{print $VAR}'`（Shell 不会解析单引号内的`$VAR`）；
    - 正确示例：`awk -v var="$VAR" '{print var}'`。
    
3. **`BEGIN`和`END`块的作用**
    
    - `BEGIN{}`块：在**处理任何行之前**执行，仅执行一次，用于初始化变量、打印表头；
    - `END{}`块：在**处理完所有行之后**执行，仅执行一次，用于打印汇总、计算结果；
    - 注意：`BEGIN`和`END`块中没有`$0`、`$1`等字段变量，因为还没处理或已处理完行。
    
4. **数组的使用**
    
    - `awk` 的数组是**关联数组**，下标可以是数字或字符串；
    - 数组不需要提前声明，直接赋值即可使用，如`count[$1]++`；
    - 遍历数组用`for (key in array)`，注意遍历顺序是不确定的，若需排序需结合`sort`命令。
    
5. **处理大文件的性能**
    
    - `awk` 处理大文件时性能很高，但需注意避免不必要的操作；
    - 尽量在模式中过滤行，减少进入动作块的行数，如`awk '/error/{...}'`比`awk '{if(/error/) ...}'`性能更好；
    - 避免在动作块中调用外部命令（如`system()`），会严重降低性能。
    
6. **正则表达式的匹配**
    
    - `awk` 的正则表达式用`/ /`包裹，如`/error/`；
    - 若需匹配特殊字符（如`.`、`*`、`?`），必须用反斜杠转义，如`/error\.log/`；
    - 匹配字段用`~`（包含）或`!~`（不包含），如`$1 ~ /^test/`（第 1 字段以 test 开头）。
    
7. **`print`和`printf`的区别**
    
    - `print`：简单打印，自动添加换行符，字段间用`OFS`分隔；
    - `printf`：格式化打印，**不会自动添加换行符**，需手动加`\n`，支持指定宽度、对齐方式、小数位数等；
    - 典型错误：用`printf`时忘记加`\n`，导致所有输出连在一起。
    

## 六、同类命令对比 & 拓展

表格

|命令|核心作用|与 awk 的区别 & 适用场景|
|:--|:--|:--|
|`grep`|按模式匹配行|优势：速度极快，仅用于模式匹配；劣势：无法提取字段、无法计算。简单模式匹配用 grep，复杂文本处理用 awk|
|`sed`|流编辑器，用于文本替换、删除、插入|优势：擅长文本替换、行操作；劣势：字段提取、计算能力弱。文本替换、行编辑用 sed，字段提取、数据计算用 awk，两者常配合使用|
|`cut`|简单字段提取|优势：简单快速，仅用于提取字段；劣势：无法模式匹配、无法计算、分隔符灵活性差。简单固定分隔符的字段提取用 cut，复杂处理用 awk|
|`sort`|排序文本行|优势：专门用于排序，支持多种排序规则；劣势：无文本处理能力。排序用 sort，awk 处理后常结合 sort 排序|
|`uniq`|去重、统计重复次数|优势：专门用于去重、统计；劣势：无文本处理能力。去重用 uniq，awk 处理后常结合 uniq 去重|
|`perl`|全能脚本语言|优势：功能比 awk 更强大，支持复杂数据结构、正则表达式更强；劣势：学习曲线更陡，脚本更复杂。简单文本处理用 awk，复杂任务用 perl|

## 七、课后练习任务

1. 基础字段提取练习：处理`/etc/passwd`文件（冒号分隔），打印第 1（用户名）、第 3（UID）、第 6（家目录）字段，输出字段间用制表符分隔。
2. 模式匹配练习：处理一个日志文件`access.log`，匹配第 2 字段等于`"POST"`且第 3 字段大于`200`的行，打印第 1（IP）、第 3（状态码）、第 7（URL）字段。
3. 数值计算练习：处理一个数据文件`data.txt`（包含 3 个字段：名称、数量、单价），计算每个商品的总金额（数量 × 单价），并计算所有商品的总金额和平均单价。
4. 格式化输出练习：结合`ls -l`，统计当前目录下所有普通文件的总大小（以 MB 为单位，保留 2 位小数），并打印最大的 3 个文件的名称和大小。
5. 单词频率统计练习：处理一个文本文件`test.txt`，统计每个单词的出现次数，按次数降序排序，取 Top10。
6. CSV 处理练习：处理一个 CSV 文件`data.csv`（逗号分隔，包含表头），打印第 2（姓名）和第 4（成绩）字段，筛选成绩大于 80 的记录，并按成绩降序排序。
7. 结合命令练习：结合`df -h`，提取磁盘使用率超过 70% 的分区，打印分区名、挂载点、使用率，并生成一个带表头的格式化报告。
