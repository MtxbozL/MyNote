## 一、核心定位

`fdisk` 和 `parted` 是 Linux 上**两大经典的磁盘分区工具**，功能各有侧重：

- **fdisk**：传统的 **MBR 分区表工具**，适合小于 2TB 的磁盘，语法简单，交互友好，但不支持 GPT（GUID 分区表），且 MBR 最多支持 4 个主分区（或 3 主 + 1 扩展）。
- **parted**：支持 **MBR 和 GPT 两种分区表**，适合大于 2TB 的磁盘（GPT 无 2TB 限制，分区数也更多），功能更强大，但需注意**parted 是实时写入的**（操作立即生效，无 “确认写入” 步骤）。

---

## 二、fdisk 详解

### 1. 核心概念

- **主要用途**：MBR 分区表管理，适合 <2TB 磁盘。
- **交互模式**：进入交互式界面，操作后需 `w` 写入才生效，`q` 可放弃退出。

### 2. 语法格式

bash

运行

```
# 查看所有磁盘的分区表
fdisk -l

# 进入指定磁盘的交互式分区界面
fdisk /dev/sda
```

### 3. 高频交互式命令

表格

|命令|核心作用|说明|
|:--|:--|:--|
|`p`|**打印**分区表|查看当前磁盘的分区情况，操作前必看|
|`n`|**新建**分区|创建新分区，按提示选择主 / 扩展、分区号、起始扇区、大小|
|`d`|**删除**分区|删除指定分区，需输入分区号|
|`t`|更改**分区类型**|如改为 Linux LVM（8e）、Linux swap（82）等|
|`w`|**写入**分区表并退出|操作确认无误后写入，**写入后立即生效**|
|`q`|**放弃**操作并退出|不保存，直接退出|
|`m`|显示**帮助**|查看所有可用命令|

---

## 三、parted 详解

### 1. 核心概念

- **主要用途**：MBR/GPT 分区表管理，适合 >2TB 磁盘。
- **交互模式**：进入交互式界面，**操作实时写入**（无单独的 “写入” 命令，需非常小心！）。

### 2. 语法格式

bash

运行

```
# 查看所有磁盘的分区表
parted -l

# 进入指定磁盘的交互式分区界面
parted /dev/sda
```

### 3. 高频交互式命令

表格

|命令|核心作用|说明|
|:--|:--|:--|
|`print`|**打印**分区表|查看当前磁盘的分区情况、分区表类型（gpt/msdos），操作前必看|
|`mklabel gpt`/`mklabel msdos`|创建**分区表**|`gpt` 为 GPT，`msdos` 为 MBR，**会清空整个磁盘！**|
|`mkpart primary 起始 结束`|**新建**主分区|起始 / 结束为位置（如 `0% 50%`、`1MiB 100GiB`）|
|`rm 分区号`|**删除**分区|需输入分区号，**立即生效！**|
|`set 分区号 flag on/off`|设置**分区标志**|如 `boot`（启动）、`lvm` 等|
|`quit`|**退出**|退出 parted|
|`help`|显示**帮助**|查看所有可用命令|

## 四、核心示例

### fdisk 示例

bash

运行

```
# 1. 查看所有磁盘的分区表
sudo fdisk -l

# 2. 进入 /dev/sdb 的交互式分区（假设是新磁盘）
sudo fdisk /dev/sdb

# 3. 在 fdisk 交互界面中：
# p  # 先打印分区表，确认磁盘
# n  # 新建分区
# p  # 选择主分区
# 1  # 分区号 1
#    # 直接回车，使用默认起始扇区
# +50G  # 大小 50GB
# n  # 再新建一个分区
# p  # 主分区
# 2  # 分区号 2
#    # 默认起始
# +100G  # 大小 100GB
# p  # 再次打印，确认分区
# w  # 写入分区表并退出（确认无误后再按！）
```

### parted 示例

bash

运行

```
# 1. 查看所有磁盘的分区表
sudo parted -l

# 2. 进入 /dev/sdb 的交互式分区（假设是新磁盘）
sudo parted /dev/sdb

# 3. 在 parted 交互界面中：
# print  # 先打印分区表，确认磁盘和分区表类型
# mklabel gpt  # 创建 GPT 分区表（会清空磁盘！确认无误后再执行！）
# mkpart primary 0% 50%  # 新建主分区，从 0% 到 50%
# mkpart primary 50% 100%  # 新建主分区，从 50% 到 100%
# print  # 再次打印，确认分区
# quit  # 退出
```

## 五、避坑提示与注意事项

### fdisk

1. **仅适合 MBR，>2TB 磁盘用 parted**
    
    - MBR 分区表不支持 >2TB 的磁盘，且最多 4 个主分区；
    - 推荐：>2TB 磁盘必须用 parted + GPT。
    
2. **操作后需 `w` 写入才生效，`q` 可放弃**
    
    - fdisk 是 “先操作后写入”，`w` 前都可以用 `q` 放弃；
    - 推荐：操作完先用 `p` 确认，无误后再 `w`。
    
3. **不要在正在使用的磁盘 / 分区上操作**
    
    - 不要对已挂载的分区或系统盘进行删除 / 新建操作，会导致数据丢失或系统崩溃；
    - 推荐：仅对新磁盘或未挂载的磁盘操作，操作前备份数据。
    

### parted

1. **操作是实时写入的！没有 “确认写入” 步骤，非常危险！**
    
    - parted 的所有操作（`mkpart`、`rm`、`mklabel`）都是**立即生效**的，没有单独的 “写入” 命令；
    - 强制要求：操作前先用 `print` 确认磁盘，每一步都要小心，操作前备份数据！
    
2. **适合 GPT，>2TB 磁盘首选**
    
    - GPT 无 2TB 限制，分区数也更多；
    - 推荐：>2TB 磁盘必须用 parted + GPT。
    
3. **`mklabel` 会清空整个磁盘！**
    
    - `mklabel gpt` 或 `mklabel msdos` 会创建新的分区表，**清空整个磁盘的所有数据**；
    - 强制要求：仅对新磁盘使用，确认无误后再执行！

## 六、同类对比

表格

|特性|fdisk|parted|
|:--|:--|:--|
|**分区表支持**|仅 MBR|MBR + GPT|
|**最大磁盘支持**|<2TB|无限制（GPT）|
|**最大主分区数**|4|无限制（GPT）|
|**写入方式**|先操作后 `w` 写入|实时写入（操作立即生效）|
|**安全性**|较高（可 `q` 放弃）|较低（实时写入，需非常小心）|
|**适用场景**|<2TB 磁盘，MBR 分区|>2TB 磁盘，GPT 分区|

## 七、课后练习

### 重要提示：所有练习仅在测试环境或虚拟机中进行，绝对不要在生产环境或有重要数据的磁盘上操作！

1. **查看分区表**：

- `sudo fdisk -l` 查看所有磁盘的 MBR 分区表。
- `sudo parted -l` 查看所有磁盘的分区表（包括 GPT）。

2. **fdisk 练习（仅测试环境！）**：

- 找一个空磁盘或虚拟机中的新磁盘（如 `/dev/sdb`）。
- `sudo fdisk /dev/sdb` 进入交互界面。
- 用 `p` 打印分区表。
- 用 `n` 新建 2 个主分区。
- 用 `p` 确认。
- 用 `w` 写入并退出（仅测试环境！）。

3. **parted 练习（仅测试环境！）**：

- 找一个空磁盘或虚拟机中的新磁盘（如 `/dev/sdb`）。
- `sudo parted /dev/sdb` 进入交互界面。
- 用 `print` 打印分区表。
- 用 `mklabel gpt` 创建 GPT 分区表（仅测试环境！确认磁盘为空！）。
- 用 `mkpart primary 0% 50%` 和 `mkpart primary 50% 100%` 新建 2 个分区。
- 用 `print` 确认。
- 用 `quit` 退出。
