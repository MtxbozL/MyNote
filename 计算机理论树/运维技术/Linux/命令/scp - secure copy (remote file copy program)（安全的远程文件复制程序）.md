## 一、核心定位

`scp` 是 Linux 上**最经典、最基础的基于 SSH 的安全文件传输工具**，是旧工具 `rcp` 的安全替代品，用于**本地主机与远程主机之间、或远程主机与远程主机之间的文件 / 目录传输**，所有数据通过 SSH 加密传输，安全可靠。

> 补充说明：`scp` 名称来源于 “secure copy”（安全复制）；与 `rsync` 不同，`scp` 不支持断点续传和增量传输，但语法更简单，适合快速传输小文件；传输依赖 SSH，需确保 SSH 能正常登录目标主机。

---

## 二、语法格式

### 1. 本地 → 远程（上传）

```bash
scp [选项] 本地文件/目录 远程用户@远程IP:远程路径
```

### 2. 远程 → 本地（下载）

```bash
scp [选项] 远程用户@远程IP:远程文件/目录 本地路径
```

### 3. 远程 → 远程（中转）

bash

运行

```
scp [选项] 远程1用户@远程1IP:远程1文件/目录 远程2用户@远程2IP:远程2路径
```

> 关键说明：
> 
> 1. **顺序不能错**：本地在前是上传，远程在前是下载，新手容易搞混！
> 2. **远程路径**：可以是绝对路径（如 `/home/user/`）或相对路径（如 `./`，即远程用户的家目录）。
> 3. **默认端口**：默认使用 SSH 的 22 端口，非标准端口需加 `-P`（大写！）。

## 三、高频核心选项

表格

|选项|核心作用|高频使用场景|
|:--|:--|:--|
|`-r, --recursive`|**递归传输目录**（包括目录下的所有子目录和文件）|传输整个目录，**最常用的选项之一**|
|`-P, --port PORT`|指定 SSH **端口 PORT**（注意是大写 P！小写 p 是保留的）|目标主机 SSH 端口不是 22 时使用|
|`-v, --verbose`|**详细模式**，显示传输过程的详细信息（调试用）|排查传输问题|
|`-C, --compress`|**压缩传输**，传输前压缩数据，减少带宽占用|传输大文件或慢网络时使用|
|`-i, --identity FILE`|指定 SSH **私钥文件 FILE**（如 `~/.ssh/id_rsa`）|使用密钥认证而非密码登录时使用|
|`-p, --preserve-mtimes`|**保留文件的修改时间和权限**|保留文件的元数据|

## 四、核心示例

### 1. 本地 → 远程（上传）

bash

运行

```
# 上传本地文件到远程用户的家目录
scp localfile.txt user@192.168.1.10:./

# 上传本地文件到远程绝对路径
scp localfile.txt user@192.168.1.10:/home/user/data/

# 上传本地目录到远程（必须加 -r！）
scp -r localdir/ user@192.168.1.10:/home/user/data/
```

### 2. 远程 → 本地（下载）

bash

运行

```
# 下载远程文件到本地当前目录
scp user@192.168.1.10:/home/user/remotefile.txt ./

# 下载远程文件到本地指定路径
scp user@192.168.1.10:/home/user/remotefile.txt /home/localuser/data/

# 下载远程目录到本地（必须加 -r！）
scp -r user@192.168.1.10:/home/user/remotedir/ /home/localuser/data/
```

### 3. 进阶选项示例

bash

运行

```
# 指定 SSH 端口为 2222（大写 P！）
scp -P 2222 localfile.txt user@192.168.1.10:./

# 指定私钥文件
scp -i ~/.ssh/my_private_key localfile.txt user@192.168.1.10:./

# 压缩传输大文件
scp -C largefile.iso user@192.168.1.10:./

# 详细模式，显示传输过程
scp -v localfile.txt user@192.168.1.10:./
```

## 五、避坑提示与注意事项

1. **顺序绝对不能错！本地在前是上传，远程在前是下载**
    
    - 新手最容易搞混，搞反了会把文件传错位置；
    - 强制要求：仔细看顺序，确认是上传还是下载；
    - 典型错误：`scp user@192.168.1.10:remotefile.txt ./` 以为是上传，实际是下载；
    - 正确示例：上传 `scp localfile.txt user@IP:./`，下载 `scp user@IP:remotefile.txt ./`。
    
2. **指定端口是大写 `-P`，不是小写 `-p`！**
    
    - 小写 `-p` 是保留的（用于保留修改时间），指定端口必须用大写 `-P`；
    - 典型错误：`scp -p 2222 localfile.txt user@IP:./`，报错；
    - 正确示例：`scp -P 2222 localfile.txt user@IP:./`。
    
3. **传输目录必须加 `-r`，否则会报错**
    
    - 传输目录（包括子目录和文件）必须加 `-r`（递归）；
    - 典型错误：`scp localdir/ user@IP:./`，报错 `not a regular file`；
    - 正确示例：`scp -r localdir/ user@IP:./`。
    
4. **基于 SSH，需确保 SSH 能正常登录**
    
    - `scp` 依赖 SSH，传输前需先用 SSH 登录目标主机测试；
    - 典型错误：SSH 登录失败，scp 也失败；
    - 正确示例：先 `ssh user@192.168.1.10` 测试登录，成功后再用 scp。
    
5. **不支持断点续传和增量传输，大文件推荐用 `rsync`**
    
    - `scp` 传输中断后必须重新开始，且传输整个文件，不支持增量；
    - 解决方法：大文件或频繁同步推荐用 `rsync`（支持断点续传、增量传输、更高效）；
    - 典型错误：大文件传输中断后重新用 scp 传，浪费时间；
    - 正确示例：小文件用 scp，大文件 / 同步用 rsync。

## 六、同类对比

表格

|特性|scp|rsync|
|:--|:--|:--|
|**核心用途**|快速安全传输小文件|高效同步大文件 / 目录、断点续传、增量传输|
|**断点续传**|不支持|支持|
|**增量传输**|不支持（传输整个文件）|支持（仅传输变化的部分）|
|**语法复杂度**|极简|中等（选项多）|
|**默认安装**|多数发行版默认安装|多数发行版默认安装|
|**适用场景**|快速传输小文件|大文件传输、目录同步、备份|

## 七、课后练习

1. **本地传文件到远程**：`scp localtest.txt user@你的远程IP:./`。
2. **远程传文件到本地**：`scp user@你的远程IP:remotetest.txt ./`。
3. **本地传目录到远程**：创建一个 `localtestdir/` 目录，里面放几个文件，然后 `scp -r localtestdir/ user@你的远程IP:./`。
4. **远程传目录到本地**：`scp -r user@你的远程IP:remotetestdir/ ./`。
5. **指定 SSH 端口**：如果你的远程 SSH 端口不是 22，用 `scp -P 你的端口 localtest.txt user@你的远程IP:./`。
6. **指定私钥文件**：如果你用密钥认证，用 `scp -i ~/.ssh/你的私钥 localtest.txt user@你的远程IP:./`。
7. **压缩传输大文件**：找一个大文件，用 `scp -C largefile.iso user@你的远程IP:./`。
8. **同类对比**：同时用 `scp` 和 `rsync`（如果已安装）传输同一个大文件，对比速度和效率。
