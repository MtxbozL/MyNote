
Nginx 是一款开源、跨平台的高性能 HTTP 与反向代理服务器，同时支持邮件代理、TCP/UDP 四层代理能力，由 Igor Sysoev 于 2004 年首次公开发布。其核心设计目标为高并发承载、低内存占用、高服务稳定性，是目前互联网架构中应用最广泛的 Web 服务与流量网关组件。

### 1.1.1 高性能核心原理

Nginx 的高性能源于其**事件驱动 + 异步非阻塞 IO**的核心设计，彻底区别于传统 Web 服务器同步阻塞的进程 / 线程 - per-connection 模型。其核心逻辑为：将网络请求的全生命周期拆解为独立事件，通过 IO 多路复用接口统一管理所有连接与事件，单进程 / 单线程即可处理数万级并发连接，避免了传统模型中进程 / 线程创建、切换的巨额系统开销，同时实现了内存资源的极低占用。

### 1.1.2 与 Apache HTTP Server 的核心对比

表格

|对比维度|Nginx|Apache HTTP Server|
|---|---|---|
|并发模型|多进程单线程 + 异步非阻塞 IO，IO 多路复用驱动，单 Worker 可处理数万并发连接|多进程 / 多线程同步阻塞模型，prefork/worker/event 三种 MPM，单连接占用一个进程 / 线程|
|资源占用|极低，并发场景下内存与 CPU 开销远低于 Apache|较高，高并发下进程 / 线程切换开销显著，内存占用激增|
|静态资源处理|原生适配静态资源场景，性能极强|静态资源处理能力弱于 Nginx|
|动态请求处理|需通过反向代理转发至后端应用服务，无原生动态内容处理能力|原生支持 mod_php、mod_perl 等模块，可直接嵌入动态脚本执行|
|扩展性|模块化编译架构，第三方模块需编译集成，高版本支持动态模块加载|丰富的动态加载模块生态，无需重新编译即可启用 / 禁用模块|
|配置灵活性|配置语法简洁，支持精细化的流量、路由、缓存控制|配置体系庞大，基于目录的配置能力更强|

### 1.1.3 核心应用场景

1. **静态 Web 服务器**：高性能处理静态资源（HTML、CSS、JS、图片、视频等）的 HTTP 请求，支持目录浏览、压缩、缓存控制等优化能力。
2. **反向代理服务器**：接收客户端请求并转发至后端应用服务，实现请求转发、协议适配、客户端信息透传，是前后端分离架构的核心组件。
3. **负载均衡器**：基于 upstream 模块将客户端请求分发至多台后端服务节点，实现流量调度、故障隔离、服务高可用，支持多种负载均衡算法。
4. **四层代理网关**：基于 stream 模块实现 TCP/UDP 协议的反向代理与负载均衡，可用于 MySQL、Redis、MQ 等中间件的流量转发与高可用部署。
5. **HTTP 缓存服务器**：基于 proxy_cache 模块实现后端响应内容的本地缓存，降低后端服务压力，提升请求响应速度。
6. **安全防护与 API 网关**：支持 HTTPS 加密、IP 黑白名单、身份认证、防盗链、限流防攻击、请求过滤等安全能力，可作为轻量级 API 网关使用。
