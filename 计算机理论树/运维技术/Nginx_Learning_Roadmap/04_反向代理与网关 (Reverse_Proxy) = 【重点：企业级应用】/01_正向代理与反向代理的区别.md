# 第四章 反向代理与网关

Nginx 作为反向代理服务器与轻量级 API 网关，是企业级分布式架构的核心组件，能够实现客户端请求的转发、协议适配、跨域处理、真实 IP 透传等核心能力，有效解耦客户端与后端服务，提升系统的扩展性、安全性与高可用性。本章围绕 Nginx 反向代理的核心配置展开，涵盖正反向代理区别、基础代理配置、真实 IP 透传、代理优化、WebSocket 代理及跨域配置六大核心模块，是实现企业级网关能力的基础。

## 4.1 正向代理与反向代理的区别

代理本质是**请求转发的中间节点**，根据代理服务的**目标对象**与**部署位置**，分为正向代理（Forward Proxy）与反向代理（Reverse Proxy），二者核心职责、应用场景、部署架构存在本质区别，且均为网络架构中不可或缺的组件。

### 4.1.1 核心定义与核心职责

1. **正向代理**
    
    - 定义：代理**客户端**向目标服务器发起请求，客户端明确知道目标服务器地址，代理服务器为客户端提供中转服务，隐藏客户端真实 IP。
    - 核心职责：突破网络访问限制、隐藏客户端身份、缓存资源提升客户端访问速度。
    - 典型特征：**客户端知道代理服务器存在**，需在客户端配置代理地址与端口才能生效。
    
2. **反向代理**
    
    - 定义：代理**后端服务**接收客户端请求，客户端**不知道后端服务的真实地址**，仅需访问代理服务器的地址，由代理服务器将请求转发至后端可用的服务节点。
    - 核心职责：请求转发、负载均衡、隐藏后端服务架构、统一入口管理、协议转换、安全防护。
    - 典型特征：**客户端无感知代理存在**，所有配置均在服务端完成，对客户端透明。
    

### 4.1.2 核心对比与适用场景

表格

|对比维度|正向代理|反向代理|
|---|---|---|
|代理目标|为客户端服务，转发客户端请求至目标服务器|为后端服务服务，转发客户端请求至后端服务节点|
|客户端感知|客户端明确知道代理存在，需手动配置|客户端无感知，仅访问代理统一入口|
|地址隐藏|隐藏客户端真实 IP，目标服务器仅能看到代理 IP|隐藏后端服务真实 IP 与架构，客户端仅能看到代理 IP|
|部署位置|部署在客户端侧（如企业内网、个人主机）|部署在服务端侧（如公网边缘节点、后端服务集群前端）|
|核心场景|科学上网、企业内网访问公网、客户端缓存|微服务网关、负载均衡、统一入口、跨域处理、安全防护|
|典型实现|浏览器代理、Squid、Shadowsocks|Nginx、HAProxy、Apache APISIX|

### 4.1.3 反向代理的企业级价值

反向代理作为后端服务的**统一入口**，是企业级架构的核心枢纽，其核心价值体现在：

1. **解耦客户端与后端**：客户端仅需对接代理服务器，后端服务的扩容、缩容、地址变更对客户端透明，提升系统扩展性；
2. **负载均衡**：将客户端请求分发至多台后端服务节点，避免单节点压力过大，提升系统并发承载能力；
3. **安全防护**：作为公网与内网的隔离层，屏蔽后端服务的真实地址，同时可实现 IP 黑白名单、限流、防盗链等安全能力；
4. **协议适配**：实现 HTTP/HTTPS 与 WebSocket、TCP/UDP 的协议转换，适配不同后端服务的协议需求；
5. **性能优化**：提供缓存、压缩、静态资源分发等能力，降低后端服务压力，提升请求响应速度。
