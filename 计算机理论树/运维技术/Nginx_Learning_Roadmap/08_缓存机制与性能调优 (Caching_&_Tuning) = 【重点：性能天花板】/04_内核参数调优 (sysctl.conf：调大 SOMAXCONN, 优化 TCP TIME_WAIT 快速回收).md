
## 04 内核参数调优

Linux 内核默认参数针对通用桌面 / 服务器场景设计，未针对高并发 Web 服务做优化，是 Nginx 高并发场景下性能瓶颈的核心根源之一。内核参数调优通过修改`/etc/sysctl.conf`配置文件，优化 TCP/IP 协议栈、内存管理、网络队列等核心内核参数，适配 Nginx 异步非阻塞、高并发、大流量的业务特性，突破系统层面的性能限制。

### 4.1 核心优化参数分类与详解

所有参数均需配置在`/etc/sysctl.conf`文件中，修改完成后执行`sysctl -p`命令生效，生产环境需先在测试环境验证，避免配置错误导致系统异常。

#### 4.1.1 TCP 连接队列优化

该组参数优化 TCP 三次握手过程中的连接队列大小，解决高并发下「连接丢失」「握手失败」问题，是提升并发连接承载能力的核心配置。

表格

|参数|推荐值|核心功能与原理|
|---|---|---|
|`net.core.somaxconn`|`10240`|定义系统级 TCP 全连接队列的最大长度，默认值 128。Nginx `listen`指令的`backlog`参数默认值 511，受限于该参数，高并发下必须调大，避免全连接队列溢出导致的连接丢失。|
|`net.core.netdev_max_backlog`|`10240`|定义网卡接收数据包的队列最大长度，当内核接收数据包的速度快于用户态进程处理速度时，数据包会进入该队列缓存，高并发大流量场景下需调大，避免网卡队列溢出丢包。|
|`net.ipv4.tcp_max_syn_backlog`|`8192`|定义 TCP 半连接队列（SYN_RECV 状态）的最大长度，默认值 1024。高并发场景下，大量客户端发起 TCP 握手会产生大量半连接，调大该参数可抵御 SYN 洪水攻击，避免半连接队列溢出导致的正常握手失败。|

#### 4.1.2 TIME_WAIT 状态优化

TCP 连接关闭时，主动关闭连接的一方会进入 TIME_WAIT 状态，持续 2MSL（默认 120 秒）时间，用于确保 TCP 连接的可靠关闭。高并发短连接场景下，系统会产生大量 TIME_WAIT 连接，占用系统端口与内存资源，严重时会导致新连接无法建立。

表格

|参数|推荐值|核心功能与原理|
|---|---|---|
|`net.ipv4.tcp_timestamps`|`1`|开启 TCP 时间戳选项，RFC 1323 定义的 TCP 扩展特性，是 TIME_WAIT 复用的前置条件，必须开启。|
|`net.ipv4.tcp_tw_reuse`|`1`|允许将 TIME_WAIT 状态的连接重新用于新的 TCP 连接，完全符合 RFC 规范，安全无副作用，是解决 TIME_WAIT 过多问题的核心方案，NAT 环境下也可安全使用。|
|`net.ipv4.tcp_fin_timeout`|`30`|定义 TCP 连接关闭时，FIN_WAIT_2 状态的超时时间，默认值 60 秒。调小该参数可加快僵死连接的回收，减少系统资源占用。|
|`net.ipv4.tcp_max_tw_buckets`|`20000`|定义系统允许的最大 TIME_WAIT 连接数量，默认值 180000。超出该数量的 TIME_WAIT 连接会被直接清除，避免系统被大量 TIME_WAIT 连接占满内存。|

> ⚠️ 关键避坑提示：`net.ipv4.tcp_tw_recycle`参数已在 Linux 4.12 + 内核中完全移除，且该参数在 NAT 网络环境下会导致大量正常客户端连接失败，**生产环境绝对禁止使用**，所有老教程中该参数的推荐均为错误配置。

#### 4.1.3 TCP 缓冲区优化

该组参数优化 TCP 发送 / 接收缓冲区的大小，适配大带宽、高延迟的网络环境，提升大文件传输、高并发长连接场景的吞吐量。

表格

|参数|推荐值|核心功能与原理|
|---|---|---|
|`net.core.rmem_default`|`262144`|定义 TCP 接收缓冲区的默认大小，单位字节，推荐值 256KB。|
|`net.core.rmem_max`|`16777216`|定义 TCP 接收缓冲区的最大值，单位字节，推荐值 16MB。|
|`net.core.wmem_default`|`262144`|定义 TCP 发送缓冲区的默认大小，单位字节，推荐值 256KB。|
|`net.core.wmem_max`|`16777216`|定义 TCP 发送缓冲区的最大值，单位字节，推荐值 16MB。|
|`net.ipv4.tcp_moderate_rcvbuf`|`1`|开启 TCP 接收缓冲区自动调整功能，内核会根据网络延迟、带宽动态调整缓冲区大小，适配不同的网络环境，平衡吞吐量与延迟。|

#### 4.1.4 端口范围优化

表格

|参数|推荐值|核心功能与原理|
|---|---|---|
|`net.ipv4.ip_local_port_range`|`1024 65535`|定义系统本地端口的可用范围，默认值`32768 60999`。Nginx 作为反向代理时，每个上游连接都会占用一个本地端口，高并发场景下默认端口范围会导致端口耗尽，新的代理连接无法建立，需调大至全量可用端口范围。|

#### 4.1.5 安全与其他优化

表格

|参数|推荐值|核心功能与原理|
|---|---|---|
|`net.ipv4.tcp_syncookies`|`1`|开启 SYN Cookies 功能，当 SYN 半连接队列溢出时，内核会通过 SYN Cookies 机制验证客户端合法性，抵御 SYN 洪水攻击，同时保证正常连接的建立。|
|`net.ipv4.ip_forward`|`0`|关闭 IP 转发功能，仅当 Nginx 作为网关、负载均衡器需要跨网段转发时开启，普通 Web 服务场景关闭，提升系统安全性。|
|`net.ipv4.conf.all.rp_filter`|`1`|开启反向路径过滤，内核会验证数据包的源 IP 地址是否可达，避免 IP 欺骗攻击。|
|`net.ipv4.icmp_echo_ignore_all`|`0`|禁止关闭 ICMP 响应，关闭后会导致监控、网络诊断无法正常工作，如需防护 ICMP 洪水攻击，可通过防火墙限制速率，而非完全关闭。|
|`vm.swappiness`|`1`|调整内核使用 swap 分区的优先级，默认值 60。设置为 1 表示尽可能使用物理内存，仅当物理内存几乎耗尽时才使用 swap 分区，避免 swap 磁盘 IO 导致的性能急剧下降，高并发服务器必须配置。|
|`vm.overcommit_memory`|`1`|允许内核过量分配内存，适配 Nginx、Redis 等内存使用模式，避免因内存预留机制导致的进程启动失败、内存分配失败问题。|

### 4.2 生产环境完整 sysctl.conf 优化配置

conf

```
# ==============================================
# TCP连接队列优化
# ==============================================
net.core.somaxconn = 10240
net.core.netdev_max_backlog = 10240
net.ipv4.tcp_max_syn_backlog = 8192

# ==============================================
# TIME_WAIT状态优化
# ==============================================
net.ipv4.tcp_timestamps = 1
net.ipv4.tcp_tw_reuse = 1
net.ipv4.tcp_fin_timeout = 30
net.ipv4.tcp_max_tw_buckets = 20000

# ==============================================
# TCP缓冲区优化
# ==============================================
net.core.rmem_default = 262144
net.core.rmem_max = 16777216
net.core.wmem_default = 262144
net.core.wmem_max = 16777216
net.ipv4.tcp_moderate_rcvbuf = 1

# ==============================================
# 端口范围优化
# ==============================================
net.ipv4.ip_local_port_range = 1024 65535

# ==============================================
# 安全与防御优化
# ==============================================
net.ipv4.tcp_syncookies = 1
net.ipv4.conf.all.rp_filter = 1
net.ipv4.conf.default.rp_filter = 1
net.ipv4.conf.all.accept_source_route = 0
net.ipv4.conf.default.accept_source_route = 0
net.ipv4.conf.all.accept_redirects = 0
net.ipv4.conf.default.accept_redirects = 0
net.ipv4.ip_forward = 0

# ==============================================
# 内存管理优化
# ==============================================
vm.swappiness = 1
vm.overcommit_memory = 1
vm.dirty_ratio = 10
vm.dirty_background_ratio = 5
```

### 4.3 关键注意事项

1. **配置生效验证**：修改`/etc/sysctl.conf`后，必须执行`sysctl -p`命令使配置实时生效，重启服务器后配置仍会保留；通过`sysctl [参数名]`命令可验证参数是否生效。
2. **参数适配原则**：所有推荐值均为通用生产环境标准，需根据服务器硬件配置（CPU 核心数、内存大小、网卡带宽）、业务场景（静态资源 / 反向代理 / API 服务）、并发量级做针对性调整，不可盲目照搬。
3. **内核版本适配**：部分参数仅在特定 Linux 内核版本中支持，需根据服务器内核版本调整，可通过`uname -r`命令查看内核版本。
4. **灰度变更**：生产环境内核参数变更需先在测试环境完成全量压测验证，确认无性能异常、连接异常后，再在生产环境分批执行，避免配置错误导致线上故障。
