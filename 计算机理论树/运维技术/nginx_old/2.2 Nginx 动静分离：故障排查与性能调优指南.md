## 一、常见故障排查

### 1. 静态资源 404 错误

- **现象**：图片、CSS、JS 等静态文件返回 404。
- **排查步骤**：
    
    1. **检查路径**：确认 `root` 或 `alias` 指令指向的目录是否存在，文件是否存在。
    
    - `root /var/www/static;` 表示文件在 `/var/www/static/xxx.jpg`。
    - `alias /var/www/static/;` 表示文件在 `/var/www/static/xxx.jpg`（注意末尾的 `/`）。
    
    2. **检查权限**：确保 Nginx 进程（通常是 `nginx` 用户）对文件和目录有读取权限。
	
	```bash
	sudo chmod -R 755 /var/www/static
	sudo chown -R nginx:nginx /var/www/static
	```
    
    3. **检查 location 匹配**：开启 Nginx 的 debug 日志，查看请求到底匹配了哪个 location。

```nginx
    error_log /var/log/nginx/error.log debug;
```

### 2. 动态请求转发失败

- **现象**：访问动态页面返回 502 Bad Gateway 或 504 Gateway Timeout。
- **排查步骤**：
    
    1. **检查后端服务**：确认后端应用服务器（如 Tomcat, Node.js）是否正常运行，端口是否监听。

    ```bash
    curl -I http://127.0.0.1:8080
    ```
    
    2. **检查网络连通性**：确认 Nginx 服务器能否访问后端服务器。

    ```bash
    telnet 127.0.0.1 8080
    ```
    
    3. **检查超时设置**：适当调整 `proxy_connect_timeout`、`proxy_read_timeout` 等参数。

    ```nginx
    location / {
        proxy_pass http://backend;
        proxy_connect_timeout 30s;
        proxy_read_timeout 60s;
    }
    ```
    

### 3. 缓存不生效

- **现象**：静态资源每次访问都从服务器重新下载，响应头中没有 `Cache-Control` 或 `Expires`。
- **排查步骤**：
    
    1. **检查 expires 指令**：确保在处理静态资源的 location 块中配置了 `expires`。
    2. **检查响应头**：使用浏览器开发者工具或 `curl -I` 查看响应头。
    
    bash
    
    运行
    
    ```
    curl -I http://example.com/style.css
    ```
    
    3. **检查是否被覆盖**：确认是否有其他 location 或 add_header 指令覆盖了缓存设置。

## 二、核心性能调优

### 1. 开启高效文件传输

- **sendfile**：开启 `sendfile on;` 可以让 Nginx 直接在内核态传输文件，避免用户态和内核态之间的数据拷贝，大幅提升静态文件传输效率。
- **tcp_nopush** 和 **tcp_nodelay**：
    
    - `tcp_nopush on;`：配合 `sendfile` 使用，将多个响应头和文件起始部分合并成一个数据包发送，减少网络开销。
    - `tcp_nodelay on;`：对于短连接或需要快速响应的场景，禁用 Nagle 算法，确保数据立即发送。
    

nginx

```
http {
    sendfile on;
    tcp_nopush on;
    tcp_nodelay on;
    # ...
}
```

### 2. 优化缓存策略

- **浏览器缓存**：通过 `expires` 和 `Cache-Control` 头，让浏览器缓存静态资源，减少重复请求。
    
    nginx
    
    ```
    location ~* \.(jpg|png|css|js)$ {
        root /var/www/static;
        expires 30d;
        add_header Cache-Control "public, max-age=2592000";
    }
    ```
    
- **Nginx 本地缓存**：使用 `proxy_cache` 对后端返回的动态内容或静态资源进行缓存，减轻后端服务器压力。
    
    nginx
    
    ```
    http {
        proxy_cache_path /var/cache/nginx levels=1:2 keys_zone=my_cache:10m max_size=10g inactive=60m use_temp_path=off;
    
        server {
            location / {
                proxy_pass http://backend;
                proxy_cache my_cache;
                proxy_cache_key "$host$request_uri";
                proxy_cache_valid 200 30m;
            }
        }
    }
    ```
    

### 3. 压缩静态资源

- **gzip 压缩**：对文本类型的静态资源（如 CSS, JS, HTML）进行压缩，减少传输体积。
    
    nginx
    
    ```
    http {
        gzip on;
        gzip_types text/plain text/css application/json application/javascript text/xml application/xml application/xml+rss text/javascript;
        gzip_min_length 256;
    }
    ```
    

### 4. 调整工作进程和连接数

- **worker_processes**：设置为 CPU 核心数，充分利用多核 CPU。
    
    nginx
    
    ```
    worker_processes auto;
    ```
    
- **worker_connections**：设置每个 worker 进程能处理的最大并发连接数。
    
    nginx
    
    ```
    events {
        worker_connections 10240;
    }
    ```

## 三、监控与日志分析

### 1. 开启状态监控

- 使用 `ngx_http_stub_status_module` 模块获取 Nginx 的运行状态。
    
    nginx
    
    ```
    location /nginx_status {
        stub_status on;
        access_log off;
        allow 127.0.0.1;
        deny all;
    }
    ```
    
    访问 `http://yourdomain.com/nginx_status` 可以查看活跃连接数、请求数等关键指标。

### 2. 分析访问日志

- 定期分析 `access.log`，找出访问量最大的静态资源、响应时间过长的请求等。
    
    bash
    
    运行
    
    ```
    # 统计访问量前10的静态资源
    awk '{print $7}' /var/log/nginx/access.log | grep -E "\.(jpg|png|css|js)$" | sort | uniq -c | sort -nr | head -10
    ```

## 四、安全加固

### 1. 隐藏 Nginx 版本号

- 在响应头中隐藏 Nginx 版本，减少被攻击的风险。
    
    nginx
    
    ```
    server_tokens off;
    ```
    

### 2. 限制静态资源的访问

- 对敏感的静态资源（如配置文件、备份）进行访问控制。
    
    nginx
    
    ```
    location ~* \.(conf|bak|sql)$ {
        deny all;
    }
    ```
