## 一、反向代理的核心概念

反向代理（Reverse Proxy）是指代理服务器接收客户端的请求，然后将请求转发到内部网络中的后端服务器，并将后端服务器的响应结果返回给客户端。

### 核心作用

- **隐藏后端服务**：客户端只与代理服务器交互，后端服务的地址和结构对外不可见。
- **负载均衡**：将请求分发到多个后端服务器，提高系统吞吐量和可用性。
- **缓存加速**：缓存静态资源或频繁访问的动态内容，减轻后端压力。
- **安全防护**：统一入口，进行访问控制、WAF、SSL 终端等安全处理。
- **协议转换**：例如将 HTTP/HTTPS 请求转发给后端的 HTTP/1.1 或其他协议服务。

## 二、核心配置指令

### 1. `proxy_pass`

最核心的指令，用于指定请求转发的目标地址。

```nginx
location /app/ {
    proxy_pass http://127.0.0.1:8080/;
}
```

- **注意 URI 末尾的 `/`**：
    
    - `proxy_pass http://127.0.0.1:8080/;`：会将 `/app/xxx` 转发为 `/xxx`。
    - `proxy_pass http://127.0.0.1:8080;`：会将 `/app/xxx` 转发为 `/app/xxx`。
    

### 2. 请求头设置

为了让后端服务正确处理请求，需要传递客户端的真实信息。

```nginx
location / {
    proxy_pass http://backend;
    proxy_set_header Host $host;
    proxy_set_header X-Real-IP $remote_addr;
    proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    proxy_set_header X-Forwarded-Proto $scheme;
}
```

- `Host`：确保后端能识别正确的域名。
- `X-Real-IP` / `X-Forwarded-For`：传递客户端真实 IP。
- `X-Forwarded-Proto`：告知后端原始请求是 HTTP 还是 HTTPS。

### 3. 超时与缓冲

控制代理与后端的交互行为，提升稳定性和性能。

```nginx
location / {
    proxy_pass http://backend;
    proxy_connect_timeout 30s;   # 与后端建立连接的超时时间
    proxy_read_timeout 60s;      # 读取后端响应的超时时间
    proxy_send_timeout 30s;      # 向后端发送请求的超时时间

    proxy_buffering on;          # 开启缓冲
    proxy_buffer_size 4k;        # 响应头缓冲区大小
    proxy_buffers 8 4k;          # 响应体缓冲区数量和大小
}
```

## 三、典型配置示例

### 1. 基础反向代理

将所有请求转发到后端应用服务器。

```nginx
server {
    listen 80;
    server_name www.example.com;

    location / {
        proxy_pass http://127.0.0.1:8080;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }
}
```

### 2. 基于路径的反向代理

不同路径转发到不同的后端服务。

```nginx
server {
    listen 80;
    server_name www.example.com;

    # 博客服务
    location /blog/ {
        proxy_pass http://127.0.0.1:8081/;
    }

    # 电商服务
    location /shop/ {
        proxy_pass http://127.0.0.1:8082/;
    }

    # 管理后台
    location /admin/ {
        proxy_pass http://127.0.0.1:8083/;
        # 增加访问控制
        allow 192.168.1.0/24;
        deny all;
    }
}
```

- **`allow`**：只允许指定的 IP 地址或网段访问。
- **`deny`**：拒绝指定的 IP 地址或网段访问。

### 3. HTTPS 反向代理（SSL 终端）

Nginx 处理 HTTPS，转发 HTTP 到后端，简化后端配置。

```nginx
server {
    listen 443 ssl;
    server_name www.example.com;

    ssl_certificate /etc/nginx/ssl/example.com.crt;
    ssl_certificate_key /etc/nginx/ssl/example.com.key;

    location / {
        proxy_pass http://127.0.0.1:8080;
        proxy_set_header Host $host;
        proxy_set_header X-Forwarded-Proto https;
    }
}

# HTTP 重定向到 HTTPS
server {
    listen 80;
    server_name www.example.com;
    return 301 https://$host$request_uri;
}
```

## 四、常见问题与排查

### 1. 502 Bad Gateway

- **原因**：Nginx 无法连接到后端服务器，或后端服务无响应。
- **排查**：
    
    - 检查后端服务是否正常运行。
    - 检查 `proxy_pass` 地址和端口是否正确。
    - 检查防火墙是否阻止了 Nginx 与后端的通信。
    

### 2. 504 Gateway Timeout

- **原因**：Nginx 等待后端响应超时。
- **排查**：
    
    - 检查后端服务处理请求是否过慢。
    - 适当调大 `proxy_read_timeout` 的值。
    

### 3. 后端获取不到真实客户端 IP

- **原因**：未正确设置 `X-Real-IP` 或 `X-Forwarded-For` 头。
- **解决**：在 `location` 块中添加 `proxy_set_header X-Real-IP $remote_addr;`，并确保后端服务配置了读取该头。

## 五、进阶：与负载均衡结合

反向代理是负载均衡的基础，通过 `upstream` 块定义后端服务器组，Nginx 会自动分发请求。

```nginx
http {
    upstream backend_servers {
        server 10.0.0.1:8080 weight=3;  # 权重3，处理更多请求
        server 10.0.0.2:8080 weight=2;
        server 10.0.0.3:8080 backup;    # 备份服务器，仅在其他服务器不可用时使用
    }

    server {
        listen 80;
        server_name www.example.com;

        location / {
            proxy_pass http://backend_servers;
            proxy_set_header Host $host;
        }
    }
}
```
