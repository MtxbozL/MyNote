`redis.conf`是 Redis 服务端的核心配置文件，定义了实例运行行为、资源约束、持久化规则、安全管控、高可用参数等全维度核心配置。Redis 启动时通过指定文件路径加载配置，无指定则使用内置默认值；**生产环境必须通过自定义配置文件启动，禁止使用默认配置**。运行时可通过`CONFIG GET <参数名>`查询配置、`CONFIG SET <参数名> <值>`动态修改非持久化类配置，修改后需执行`CONFIG REWRITE`将变更持久化到配置文件中。

本文档聚焦基础核心配置项解析，进阶原理与调优策略在对应后续章节（持久化、内存管理、性能调优等）详细展开。

#### 一、基础通用与进程配置

|配置项|默认值|取值范围|核心作用与生产规范|
|---|---|---|---|
|`daemonize`|`no`|`yes/no`|控制 Redis 是否以守护进程（后台运行）模式启动。<br><br>生产环境必须设为`yes`，前台调试场景可保留`no`；守护进程模式必须配合`pidfile`配置使用。|
|`pidfile`|`/var/run/redis_6379.pid`|合法文件路径|守护进程模式下，存储 Redis 进程 PID 的文件路径。<br><br>多实例部署时必须按端口号唯一命名，避免进程管理冲突；需保证 Redis 进程对路径有读写权限。|
|`databases`|`16`|1~65535|设置实例的逻辑数据库数量。客户端可通过`SELECT <dbid>`切换数据库，`dbid`取值范围为`0~databases-1`，默认使用 0 号库。<br><br>注意：Redis Cluster 集群模式仅支持 0 号库，该功能自动禁用；单实例多库无权限隔离能力，不建议用于多业务数据拆分。|

#### 二、网络连接与安全基础配置

本模块为 Redis 网络访问的核心安全与性能配置，生产环境必须严格管控，禁止使用默认开放配置。

1. **核心监听配置**
    
| 配置项              | 默认值         | 核心作用与生产规范                                                                                                         |
| ---------------- | ----------- | ----------------------------------------------------------------------------------------------------------------- |
| `bind`           | `127.0.0.1` | 绑定服务监听的网卡 IP 地址，多 IP 用空格分隔。<br><br>核心安全配置：默认仅监听本地回环地址，外网无法访问；**生产环境禁止设为`0.0.0.0`（监听所有网卡）**，仅绑定内网业务网段 IP，最小化网络暴露面。 |
| `port`           | `6379`      | Redis 服务监听的 TCP 端口。<br><br>生产环境建议修改默认端口，降低自动化攻击风险；设为`0`时禁用 TCP 监听，仅支持 Unix 套接字通信。                                 |
| `unixsocket`     | 空（禁用）       | Unix 套接字文件路径。<br><br>本地进程通信场景下，Unix 套接字无 TCP 协议开销，延迟远低于 TCP 回环地址，适合本地服务高频调用场景。                                    |
| `protected-mode` | `yes`       | 保护模式兜底开关。<br><br>当 Redis 未配置`bind`监听 IP、未设置`requirepass`密码时，保护模式自动开启，仅允许本地回环地址访问，拒绝外网连接；生产环境必须保持开启，作为安全兜底防护。      |
    
2. **TCP 性能与连接管控**
    
| 配置项             | 默认值     | 核心作用与生产规范                                                                                                                        |
| --------------- | ------- | -------------------------------------------------------------------------------------------------------------------------------- |
| `tcp-backlog`   | `511`   | TCP 全连接队列最大长度。<br><br>高并发场景下，默认值不足会导致客户端连接被拒绝，需调高至 2048 以上，同时必须同步调高操作系统`net.core.somaxconn`内核参数，二者匹配生效。                          |
| `tcp-keepalive` | `300`   | TCP 保活探测时间间隔，单位秒。<br><br>内核定期向空闲连接发送 ACK 包检测客户端存活状态，避免死连接占用资源，同时防止网络中间设备断开空闲连接；生产环境建议设为 60~300。                                  |
| `timeout`       | `0`     | 客户端连接空闲超时时间，单位秒。<br><br>`0`表示禁用超时，永不主动断开空闲连接；生产环境建议设为 300~600，清理无效空闲连接，释放文件描述符资源。                                                |
| `maxclients`    | `10000` | 实例允许的最大客户端并发连接数。<br><br>受操作系统进程最大文件描述符限制（Redis 预留 32 个文件描述符用于内部逻辑，实际最大连接数 = 系统`ulimit -n` -32）；生产环境需根据业务规模设置，同步调整操作系统`ulimit`参数。 |

#### 三、内存管理核心基础配置

本模块为 Redis 内存资源管控的核心配置，直接决定服务稳定性与缓存可用性，生产环境必须按需配置，禁止无限制内存占用。

表格

|配置项|默认值|核心作用与生产规范|
|---|---|---|
|`maxmemory`|无（不限制内存）|实例最大可用内存上限，支持单位缩写（KB/MB/GB）。<br><br>核心生产配置：当内存使用达到该阈值时，触发`maxmemory-policy`定义的内存淘汰策略。<br><br>生产环境必须设置，禁止无限制占用系统内存，避免触发操作系统 OOM Killer 终止进程；建议设置为物理内存的 50%~70%，预留内存给系统、持久化 fork 子进程使用。|
|`maxmemory-policy`|`noeviction`|内存达到上限时的淘汰策略，可选值：<br><br>1. `noeviction`：不淘汰任何数据，新写入命令直接返回错误（默认值）<br><br>2. `volatile-lru`：对设置了过期时间的键，使用 LRU（最近最少使用）算法淘汰<br><br>3. `allkeys-lru`：对所有键，使用 LRU 算法淘汰<br><br>4. `volatile-lfu`：对设置了过期时间的键，使用 LFU（最不经常使用）算法淘汰<br><br>5. `allkeys-lfu`：对所有键，使用 LFU 算法淘汰<br><br>6. `volatile-random`：对设置了过期时间的键，随机淘汰<br><br>7. `allkeys-random`：对所有键，随机淘汰<br><br>8. `volatile-ttl`：优先淘汰 TTL 最小的即将过期键<br><br>生产规范：缓存场景优先`allkeys-lru/allkeys-lfu`；持久化固定数据场景用`volatile-lru/volatile-ttl`；**禁止生产环境使用默认`noeviction`**，高并发写入会直接导致服务写入不可用。|
|`maxmemory-samples`|`5`|LRU/LFU 算法的采样样本数。<br><br>Redis 采用近似 LRU/LFU 算法，而非全量键排序，通过随机采样 N 个键，从中淘汰符合规则的键；值越大，淘汰结果越接近精准算法，但 CPU 开销越高；生产环境建议设为 5~10，平衡精度与性能。|
|`activedefrag`|`no`|是否开启主动内存碎片整理。<br><br>Redis 长期运行会产生内存碎片，导致 RSS 内存占用远高于实际数据存储量；生产环境建议设为`yes`开启，进阶调优参数在性能调优章节详解。|

#### 四、持久化基础配置

本模块为 RDB、AOF 持久化的核心基础配置，进阶原理与优化策略在「持久化机制」章节详细展开，此处仅解析配置项本身。

1. **RDB 快照基础配置**
    
| 配置项              | 默认值                                                        | 核心作用与生产规范                                                                                                                                                               |
| ---------------- | ---------------------------------------------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `save`           | `save 3600 1`<br><br>`save 300 100`<br><br>`save 60 10000` | RDB 自动快照触发规则，格式为`save <seconds> <changes>`，表示指定秒数内数据发生指定次数修改，自动触发`bgsave`生成快照；多个规则为或关系，满足任一即触发。<br><br>生产环境需根据数据安全性要求调整，禁止完全关闭 RDB（除非开启 AOF）；需禁用自动 RDB 时，配置`save ""`即可。 |
| `dir`            | `./`（启动时当前目录）                                              | RDB、AOF、日志等所有持久化文件的存储目录。<br><br>生产环境必须设置为独立的磁盘目录，禁止使用启动目录，保证 Redis 进程有读写权限；建议使用高性能 SSD 磁盘，降低持久化 IO 阻塞风险。                                                                |
| `dbfilename`     | `dump.rdb`                                                 | RDB 快照文件名称。<br><br>多实例部署时必须按端口唯一命名，避免文件冲突。                                                                                                                              |
| `rdbcompression` | `yes`                                                      | 是否开启 RDB 文件 LZF 压缩。<br><br>开启后大幅降低 RDB 文件体积，减少磁盘 IO，仅消耗少量 CPU 资源；生产环境建议开启，除非 CPU 资源极度紧张。                                                                                |
| `rdbchecksum`    | `yes`                                                      | 是否开启 RDB 文件 CRC64 校验和。<br><br>开启后在 RDB 文件末尾写入校验和，启动时校验文件完整性，防止数据损坏；生产环境建议开启，保障数据安全性。                                                                                    |
    
2. **AOF 持久化基础配置**
    
| 配置项                           | 默认值              | 核心作用与生产规范                                                                                                                                                                                                                                                                                |
| ----------------------------- | ---------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `appendonly`                  | `no`             | 是否开启 AOF（Append Only File）持久化。<br><br>默认关闭，仅使用 RDB 持久化；生产环境建议开启，Redis 4.0 + 推荐使用 RDB+AOF 混合持久化方案，平衡数据安全性与恢复效率。                                                                                                                                                                           |
| `appendfilename`              | `appendonly.aof` | AOF 日志文件名称。<br><br>多实例部署时必须按端口唯一命名，避免文件冲突。                                                                                                                                                                                                                                               |
| `appendfsync`                 | `everysec`       | AOF 日志刷盘策略，核心持久化参数，可选值：<br><br>1. `always`：每次写入命令都执行 fsync 刷盘，数据安全性最高，性能最低，极端场景仅丢失 1 条命令<br><br>2. `everysec`：每秒执行 1 次 fsync 刷盘，平衡性能与安全性，默认值，极端场景最多丢失 1 秒数据<br><br>3. `no`：不主动执行 fsync，由操作系统控制刷盘时机，性能最高，安全性最低，宕机丢失数据量不可控<br><br>生产规范：绝大多数场景使用默认`everysec`；金融等强数据安全场景使用`always`；禁止使用`no`。 |
| `aof-use-rdb-preamble`        | `yes`            | 是否开启混合持久化（Redis 4.0 + 支持）。<br><br>开启后，AOF 重写时文件前半段为 RDB 格式全量快照，后半段为增量 AOF 命令日志，兼顾 RDB 的快速恢复能力与 AOF 的数据安全性，生产环境必须开启。                                                                                                                                                                      |
| `auto-aof-rewrite-percentage` | `100`            | AOF 自动重写的触发增长比例。<br><br>当 AOF 文件大小比上一次重写后增长了指定百分比，触发自动重写；`100`表示文件翻倍时触发，设为`0`禁用自动重写。                                                                                                                                                                                                     |
| `auto-aof-rewrite-min-size`   | `64mb`           | AOF 自动重写的最小文件阈值。<br><br>仅当 AOF 文件大小达到该值时，才会触发自动重写，避免小文件频繁重写；生产环境建议调高至 512mb 以上，降低重写频率。                                                                                                                                                                                                   |
| `no-appendfsync-on-rewrite`   | `no`             | AOF 重写期间是否暂停 fsync 刷盘。<br><br>AOF 重写会占用大量磁盘 IO，此时持续执行 fsync 可能导致主线程阻塞；高并发写入场景建议设为`yes`，权衡可用性与数据安全性。                                                                                                                                                                                      |

#### 五、安全管控进阶配置

|配置项|默认值|核心作用与生产规范|
|---|---|---|
|`requirepass`|空（无密码）|Redis 服务访问密码。<br><br>客户端连接后需执行`AUTH <password>`认证，否则无法执行任何命令；核心安全配置，生产环境必须设置高强度密码，禁止弱密码或空密码；注意：密码以明文存储在配置文件中，需严格控制文件权限为 600，仅 Redis 运行用户可读写。|
|`rename-command`|空（无重命名）|命令重命名或禁用配置，格式为`rename-command <原命令名> <新命令名>`。<br><br>核心安全配置：若新命令名为空字符串，则完全禁用该命令；生产环境必须禁用高危命令：`KEYS`、`FLUSHDB`、`FLUSHALL`、`CONFIG`、`MONITOR`、`DEBUG`等，避免误操作或恶意攻击导致服务不可用、数据泄露。|
|`aclfile`|空（禁用）|ACL 权限控制规则文件路径（Redis 6.0 + 支持）。<br><br>支持创建多用户，分配不同的命令、键空间权限；生产环境多业务共享实例时，必须使用 ACL 替代单一密码认证，实现细粒度权限隔离。|

#### 六、日志与可观测性基础配置

1. **系统日志配置**
    
| 配置项        | 默认值        | 核心作用与生产规范                                                                                                          |
| ---------- | ---------- | ------------------------------------------------------------------------------------------------------------------ |
| `loglevel` | `notice`   | 日志级别，可选值：`debug`、`verbose`、`notice`、`warning`。<br><br>生产环境建议使用`notice`，兼顾日志信息量与磁盘占用；故障排查使用`verbose`，调试场景使用`debug`。 |
| `logfile`  | 空（输出到标准输出） | 日志文件存储路径。<br><br>守护进程模式下，默认日志输出到`/dev/null`，生产环境必须设置日志文件路径，便于问题排查与审计；需配合操作系统`logrotate`实现日志轮转，避免日志文件占满磁盘。          |
    
2. **慢查询日志配置**
    
| 配置项                       | 默认值     | 核心作用与生产规范                                                                                                                            |
| ------------------------- | ------- | ------------------------------------------------------------------------------------------------------------------------------------ |
| `slowlog-log-slower-than` | `10000` | 慢查询阈值，单位微秒（1 秒 = 1000000 微秒）。<br><br>命令执行时间超过该阈值，会被记录到慢查询日志；默认 10ms，生产环境建议设为 1000~5000（1~5ms），捕获高耗时命令用于性能优化；设为 0 记录所有命令，设为负数禁用慢查询日志。 |
| `slowlog-max-len`         | `128`   | 慢查询日志最大存储条数。<br><br>日志采用先进先出队列存储，超过长度后旧日志自动淘汰；生产环境建议调高至 1024 以上，保留更多慢查询记录，便于问题排查。                                                    |

#### 七、客户端缓冲区基础配置

`client-output-buffer-limit` 用于控制不同类型客户端的输出缓冲区上限，避免大 Key、全量查询、复制中断等场景导致内存溢出，格式为：

```txt
client-output-buffer-limit <客户端类型> <hard_limit> <soft_limit> <soft_seconds>
```

- 硬限制`hard_limit`：缓冲区达到该值，立即断开客户端连接
- 软限制`soft_limit`：缓冲区达到该值并持续`soft_seconds`秒，断开客户端连接

表格

|客户端类型|默认配置|生产规范|
|---|---|---|
|普通客户端|`normal 0 0 0`|无限制，常规场景无需调整；大 Key 批量查询场景需设置合理阈值，避免 OOM|
|从节点复制客户端|`slave 256mb 64mb 60`|全量复制场景需调高阈值，避免缓冲区溢出导致复制中断，建议设为`1024mb 256mb 120`|
|Pub/Sub 客户端|`pubsub 32mb 8mb 60`|消息发布订阅场景，根据消息体量调整，避免消息堆积导致连接断开|

#### 配置生效与生产环境通用规范

1. 配置文件修改后，**需重启 Redis 服务才能永久生效**；仅支持`CONFIG SET`动态修改的参数，可运行时生效，需执行`CONFIG REWRITE`将变更持久化到配置文件。
2. 生产环境配置文件必须设置严格的文件权限（建议`600`），仅 Redis 运行用户可读写，避免配置泄露。
3. 多实例部署时，必须保证`port`、`pidfile`、`dbfilename`、`appendfilename`、`logfile`等参数唯一，避免实例间资源冲突。
4. 所有内存、磁盘相关配置，必须匹配服务器硬件资源，避免过度配置导致系统资源耗尽。
