
## 02_哨兵模式 (Sentinel)

Redis Sentinel（哨兵）是 Redis 官方提供的**主从架构自动化高可用解决方案**，基于主从复制架构，实现了主节点的自动化监控、故障检测与故障转移，解决了主从架构手动故障切换的痛点，是生产环境主从架构的必备组件。

Sentinel 本身是一个分布式的特殊 Redis 节点，不存储业务数据，仅负责集群监控与管理，生产环境必须部署 3 个及以上奇数个 Sentinel 节点，保证集群的可用性与选举合法性。

### 01_核心作用 (监控、通知、自动故障转移)

Sentinel 的核心职能分为三大块，覆盖了高可用架构的全流程需求：

1. **监控（Monitoring）**
    
    Sentinel 节点会持续监控所有主节点、从节点、其他 Sentinel 节点的运行状态，定期发送心跳检测，判断节点是否存活。核心监控内容包括：
    
    - 主从节点的 TCP 连接是否正常；
    - 节点是否正常响应 PING 命令；
    - 主节点的主从复制拓扑是否正常，从节点数量是否符合预期；
    - 其他 Sentinel 节点的存活状态。
    
2. **通知（Notification）**
    
    当 Sentinel 检测到某个节点出现故障时，会通过 API、邮件、短信等配置的通知方式，向运维人员发送告警信息；同时会将节点故障事件通知给所有其他 Sentinel 节点，同步集群状态。此外，故障转移完成后，Sentinel 会将新的主节点地址通知给客户端，客户端自动更新连接地址，无需人工修改配置。
    
3. **自动故障转移（Automatic Failover）**
    
    这是 Sentinel 最核心的职能，当 Sentinel 检测到主节点发生故障，且达到法定票数确认主节点客观下线后，会自动执行完整的故障转移流程，无需人工干预：
    
    1. 从故障主节点的所有从节点中，筛选出一个合格的从节点，提升为新的主节点；
    2. 让所有剩余的从节点，改为复制新的主节点；
    3. 若原故障主节点恢复上线，会将其设置为新主节点的从节点，保持主从拓扑一致性；
    4. 向客户端推送新的主节点地址，实现业务无感知切换。
