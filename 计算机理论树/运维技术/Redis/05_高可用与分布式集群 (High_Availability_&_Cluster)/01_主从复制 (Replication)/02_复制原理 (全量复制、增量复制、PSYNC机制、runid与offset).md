主从复制的核心是**主节点与从节点的数据一致性同步**，Redis 2.8+ 引入`PSYNC`（Partial SYNC，部分同步）机制替代旧版`SYNC`命令，实现了增量复制，大幅优化了网络中断后的复制性能。

#### 一、核心基础概念

1. **runid（运行 ID）**
    
    每个 Redis 节点启动时都会生成一个唯一的 40 位十六进制运行 ID，用于唯一标识节点身份。主从复制时，从节点会记录主节点的 runid，若主节点 runid 发生变化（如主节点重启、切换），从节点会判定主节点身份变更，触发全量复制。
    
2. **replication offset（复制偏移量）**
    
    主节点与从节点各自维护一个复制偏移量，单位为字节：
    
    - 主节点每向复制缓冲区写入 1 字节的写命令，自身偏移量 + 1；
    - 从节点每收到并执行 1 字节的主节点命令，自身偏移量 + 1；
    - 主从节点的偏移量差值，即为数据同步的延迟量，偏移量一致代表主从数据完全同步。
    
3. **复制积压缓冲区（replication backlog buffer）**
    
    主节点维护的一个固定长度的环形 FIFO 缓冲区，默认大小 1MB，用于缓存最近执行的写命令。当主从网络短暂中断重连时，从节点可通过偏移量从该缓冲区获取中断期间丢失的增量命令，实现增量复制，无需全量同步。缓冲区越大，可容忍的网络中断时间越长。
    

#### 二、PSYNC 复制机制

PSYNC 是 Redis 主从复制的核心协议，支持两种同步模式：**全量复制**与**增量复制**，核心逻辑是从节点向主节点发送`PSYNC <master_runid> <offset>`命令，主节点根据 runid 与 offset 判断同步模式：

1. 若从节点传入的 runid 与当前主节点 runid 不匹配，或 offset 超出复制积压缓冲区的范围，触发**全量复制**；
2. 若 runid 匹配，且 offset 在复制积压缓冲区范围内，触发**增量复制**。

Redis 4.0+ 优化为 PSYNC2 协议，新增了**主节点重启后、主从切换后的增量复制能力**，解决了旧版 PSYNC 主节点重启必须全量复制的问题，进一步降低了全量复制的触发概率。

#### 三、全量复制执行流程

全量复制是主从首次建立复制、或无法增量复制时的同步方式，核心是主节点生成 RDB 快照发送给从节点，从节点加载 RDB 后，再同步快照期间的增量命令，完整流程如下：

1. **建立复制连接与身份认证**：从节点执行`SLAVEOF <master_ip> <master_port>`命令，与主节点建立 TCP 连接，完成密码认证（若配置了`requirepass`），连接建立后从节点进入复制状态。
2. **PSYNC 协商同步模式**：从节点向主节点发送`PSYNC ? -1`命令，告知主节点首次复制，请求全量同步。
3. **主节点生成 RDB 快照**：主节点收到全量同步请求后，执行`BGSAVE`，后台 fork 子进程生成全量 RDB 快照，同时将快照期间收到的所有写命令写入**复制缓冲区**。
4. **主节点发送 RDB 文件到从节点**：RDB 生成完成后，主节点将 RDB 文件通过复制连接发送给从节点，从节点接收并保存到本地磁盘。
5. **从节点加载 RDB 文件**：从节点清空自身所有数据，加载 RDB 文件到内存，加载期间无法响应客户端的读请求。
6. **增量命令同步**：从节点加载 RDB 完成后，向主节点发送确认，主节点将复制缓冲区中缓存的快照期间的增量写命令发送给从节点，从节点执行所有命令，完成数据同步。
7. **持续命令传播**：全量同步完成后，进入持续复制阶段，主节点每执行一条写命令，都会异步发送给所有从节点，从节点执行命令，保持主从数据一致。

#### 四、增量复制执行流程

增量复制是主从网络短暂中断重连后的优化同步方式，仅同步中断期间丢失的增量命令，无需全量 RDB 同步，性能提升数十倍，完整流程如下：

1. **网络中断与复制断开**：主从网络波动导致 TCP 连接断开，主节点会将中断期间的写命令持续写入复制积压缓冲区。
2. **从节点重连主节点**：网络恢复后，从节点重新与主节点建立 TCP 连接，发送`PSYNC <master_runid> <last_offset>`命令，携带记录的主节点 runid 与自身最后同步的偏移量。
3. **主节点校验同步条件**：主节点校验 runid 是否匹配，同时检查 offset 是否在复制积压缓冲区的范围内。
4. **增量命令发送**：若校验通过，主节点向从节点发送`CONTINUE`响应，告知从节点可增量同步，然后将积压缓冲区中 offset 之后的所有增量写命令发送给从节点。
5. **从节点执行增量命令**：从节点接收并执行所有增量命令，更新自身复制偏移量，与主节点偏移量对齐，完成增量同步，进入持续命令传播阶段。
