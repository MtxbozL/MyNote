Redis ZSet（Sorted Set）是**元素唯一不可重复、按关联分数有序排序**的字符串元素集合，每个元素绑定一个 double 类型的`score`（分数），Redis 按 score 对元素升序排序，score 相同时按元素的字典序排序。单 ZSet 最大支持 2^32-1 个元素。

- 核心特性：元素唯一性继承自 Set，同时具备有序性；支持按分数范围、排名范围高效查询，平均时间复杂度 O (logN)；所有操作均为原子性，是排序、范围查找场景的核心数据结构。
- 逻辑定位：用户层面的有序唯一集合，底层编码为 ziplist（压缩列表）与 skiplist（跳跃表）+ dict（哈希表）双结构，Redis 根据元素数量与长度自动切换编码，平衡内存占用与查询性能。

### 01_常用命令

ZSet 命令按功能分为**元素增删改、分数与排名查询、范围查询、范围删除、增量遍历、集合运算**六大类，严格标注时间复杂度与生产规范。

#### 一、元素增删改核心命令

##### 1. ZADD 添加 / 更新有序集合元素

- **语法**：`ZADD key [NX|XX] [GT|LT] [CH] [INCR] score member [score member ...]`
- **核心功能**：向有序集合中添加一个或多个元素与对应分数；元素已存在则更新其分数，并重新调整排序位置。
- **核心参数详解**：
    
    表格
    
    |参数|功能说明|适用场景|
    |---|---|---|
    |NX|仅当元素不存在时，才执行添加操作，不更新已存在元素|仅新增元素，禁止覆盖已有分数的场景|
    |XX|仅当元素已存在时，才执行更新操作，不新增元素|仅更新已有元素分数，禁止新增的场景|
    |GT|仅当新分数大于当前分数时，才更新元素分数|仅提升分数的场景（如积分只增不减）|
    |LT|仅当新分数小于当前分数时，才更新元素分数|仅降低分数的场景（如倒计时）|
    |CH|修改返回值规则，返回**总变化元素数量**（新增 + 更新），默认仅返回新增数量|需要统计更新数量的场景|
    |INCR|原子性递增元素分数，等效于 ZINCRBY，仅支持单元素操作|原子性分数递增场景|
    
- **时间复杂度**：O (logN) 单个元素，O (M*logN) 多个元素，M 为添加的元素数量，N 为集合元素总数
- **返回值**：默认返回本次新增的元素数量；开启 CH 返回总变化元素数量；开启 INCR 返回更新后的分数。
- **生产规范**：必须根据业务场景选择 NX/XX/GT/LT 参数，避免并发场景下的分数竞态问题；单次添加元素数量建议控制在 1000 以内，避免主线程阻塞。

##### 2. ZREM 删除有序集合元素

- **语法**：`ZREM key member [member ...]`
- **核心功能**：原子性地从有序集合中删除一个或多个指定元素，不存在的元素会被忽略。
- **时间复杂度**：O (logN) 单个元素，O (M*logN) 多个元素
- **返回值**：整数类型，实际成功删除的元素数量。

##### 3. ZINCRBY 元素分数原子递增

- **语法**：`ZINCRBY key increment member`
- **核心功能**：原子性地为指定元素的分数加上增量 increment（支持正负值，负增量等效于递减）；元素不存在则先初始化为 0，再执行递增操作。
- **时间复杂度**：O(logN)
- **返回值**：字符串类型，执行递增后的最新分数。
- **生产规范**：排行榜积分更新、热度计数场景的核心命令，原子性避免并发竞态，无需业务层加锁。

#### 二、分数与排名查询命令

##### 1. ZSCORE 获取元素分数

- **语法**：`ZSCORE key member`
- **核心功能**：返回指定元素的分数值。
- **时间复杂度**：O (1)，基于底层 dict 哈希表实现，无需遍历跳跃表
- **返回值**：字符串类型，元素对应的分数；元素不存在或 key 不存在返回`(nil)`。

##### 2. ZRANK 获取元素正序排名

- **语法**：`ZRANK key member`
- **核心功能**：返回元素在集合中的**正序排名**（分数从小到大），排名从 0 开始计数，分数最小的元素排名为 0。
- **时间复杂度**：O(logN)
- **返回值**：整数类型，元素的排名；元素不存在返回`(nil)`。

##### 3. ZREVRANK 获取元素倒序排名

- **语法**：`ZREVRANK key member`
- **核心功能**：返回元素在集合中的**倒序排名**（分数从大到小），排名从 0 开始计数，分数最大的元素排名为 0。
- **时间复杂度**：O(logN)
- **返回值**：整数类型，元素的倒序排名；元素不存在返回`(nil)`。

##### 4. ZCARD 获取集合元素总数

- **语法**：`ZCARD key`
- **核心功能**：返回有序集合的元素总数量。
- **时间复杂度**：O (1)，底层结构原生维护元素计数器
- **返回值**：整数类型，集合元素总数；key 不存在返回 0。

##### 5. ZCOUNT 分数范围内元素计数

- **语法**：`ZCOUNT key min max`
- **核心功能**：返回集合中分数在`[min, max]`闭区间内的元素数量；支持`-inf`（负无穷）、`+inf`（正无穷），支持开区间语法（`(value`表示不包含该值）。
- **时间复杂度**：O (logN)，基于跳跃表的范围统计，无需遍历全量元素
- **返回值**：整数类型，指定分数范围内的元素数量。

#### 三、范围查询核心命令

##### 1. ZRANGE 按排名正序范围查询

- **语法**：`ZRANGE key start stop [WITHSCORES] [LIMIT offset count]`
- **核心功能**：返回集合中**正序排名**在`[start, stop]`闭区间内的元素，排名从 0 开始；支持负数索引（-1 为最后一个元素）。
    
    - `WITHSCORES`：可选，同时返回元素对应的分数；
    - `LIMIT`：可选，对结果集分页，offset 为偏移量，count 为返回数量。
    
- **时间复杂度**：O (logN + M)，M 为返回的元素数量
- **返回值**：元素数组；开启 WITHSCORES 时，数组交替存储元素与分数。
- **生产规范**：排行榜分页查询的核心命令，必须通过 LIMIT 分页，禁止执行`ZRANGE key 0 -1`全量查询大集合，避免主线程阻塞。

##### 2. ZREVRANGE 按排名倒序范围查询

- **语法**：`ZREVRANGE key start stop [WITHSCORES] [LIMIT offset count]`
- **核心功能**：与 ZRANGE 完全一致，唯一区别为按**倒序排名**（分数从大到小）查询，分数最大的元素排名为 0。
- **时间复杂度**：O(logN + M)
- **返回值**：与 ZRANGE 格式一致。
- **生产规范**：Top N 排行榜的核心命令，如`ZREVRANGE key 0 9 WITHSCORES`获取分数 Top10 的元素。

##### 3. ZRANGEBYSCORE 按分数正序范围查询

- **语法**：`ZRANGEBYSCORE key min max [WITHSCORES] [LIMIT offset count]`
- **核心功能**：返回集合中分数在`[min, max]`闭区间内的元素，按分数从小到大排序；支持`-inf`/`+inf`与开区间语法。
- **时间复杂度**：O(logN + M)
- **返回值**：元素数组，开启 WITHSCORES 时同步返回分数。

##### 4. ZREVRANGEBYSCORE 按分数倒序范围查询

- **语法**：`ZREVRANGEBYSCORE key max min [WITHSCORES] [LIMIT offset count]`
- **核心功能**：返回集合中分数在`[min, max]`闭区间内的元素，按分数从大到小排序；注意参数顺序为 max 在前、min 在后。
- **时间复杂度**：O(logN + M)
- **返回值**：与 ZRANGEBYSCORE 格式一致。

#### 四、范围删除命令

##### 1. ZREMRANGEBYRANK 按排名范围删除

- **语法**：`ZREMRANGEBYRANK key start stop`
- **核心功能**：删除集合中**正序排名**在`[start, stop]`闭区间内的所有元素。
- **时间复杂度**：O (logN + M)，M 为删除的元素数量
- **返回值**：整数类型，实际成功删除的元素数量。
- **生产规范**：用于固定长度排行榜的自动裁剪，如仅保留 Top1000 元素，定期删除排名 1000 以后的元素。

##### 2. ZREMRANGEBYSCORE 按分数范围删除

- **语法**：`ZREMRANGEBYSCORE key min max`
- **核心功能**：删除集合中分数在`[min, max]`闭区间内的所有元素，支持`-inf`/`+inf`与开区间语法。
- **时间复杂度**：O(logN + M)
- **返回值**：整数类型，实际成功删除的元素数量。
- **生产规范**：延迟队列场景中，用于删除已处理的过期任务。

#### 五、增量遍历与集合运算命令

##### 1. ZSCAN 增量遍历有序集合

- **语法**：`ZSCAN key cursor [MATCH pattern] [COUNT count]`
- **核心功能**：增量迭代有序集合中的所有元素与分数，参数规则与 SSCAN/HSCAN 完全一致，避免全量遍历阻塞主线程。
- **时间复杂度**：单次调用 O (1)，全量迭代总复杂度 O (N)
- **返回值**：数组格式，第一个元素为下一次迭代的游标，第二个元素为本次迭代的元素 - 分数对数组。
- **生产规范**：生产环境遍历大有序集合的唯一合规方案。

##### 2. ZINTERSTORE 有序集合交集运算并存储

- **语法**：`ZINTERSTORE destination numkeys key [key ...] [WEIGHTS weight [weight ...]] [AGGREGATE SUM|MIN|MAX]`
- **核心功能**：计算多个有序集合的交集，将结果存储到 destination 集合中，结果元素的分数按 AGGREGATE 规则聚合（默认 SUM 求和），WEIGHTS 参数为每个集合设置分数权重，聚合前先乘以对应权重。
- **时间复杂度**：O(N_K + M_logM)，N 为最小集合的元素数量，K 为集合数量，M 为结果集元素数量
- **返回值**：整数类型，结果集的元素数量。

##### 3. ZUNIONSTORE 有序集合并集运算并存储

- **语法**：`ZUNIONSTORE destination numkeys key [key ...] [WEIGHTS weight [weight ...]] [AGGREGATE SUM|MIN|MAX]`
- **核心功能**：计算多个有序集合的并集，将结果存储到 destination 集合中，分数聚合规则与 ZINTERSTORE 完全一致。
- **时间复杂度**：O (N + M*logM)，N 为所有集合的元素总数量，M 为结果集元素数量
- **返回值**：整数类型，结果集的元素数量。
