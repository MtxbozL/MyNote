Redis Hash 是**String 类型的 field（字段）与 value（值）的映射表**，等效于内存中的 HashMap，一个 Hash 对应一个 key，内部包含多个 field-value 对，适合存储结构化对象。单 Hash 最大支持 2^32-1 个 field-value 对。

- 核心特性：field 唯一不可重复，支持对单个 field 执行原子性读写、增减操作；相比将对象序列化存储为 String，Hash 支持按需读写单个字段，大幅减少网络开销，内存利用率更高。
- 逻辑定位：用户层面的结构化键值映射，底层编码为 ziplist（压缩列表）与 hashtable（哈希表），自动切换适配不同场景。

### 01_常用命令

Hash 命令按功能分为**基础读写、批量操作、原子增减、字段管理与遍历**四大类，严格标注时间复杂度与生产规范。

#### 一、基础读写核心命令

##### 1. HSET 设置哈希字段值

- **语法**：`HSET key field value [field value ...]`
- **核心功能**：为哈希的指定 field 设置 value，field 不存在则新增，存在则覆盖原有值；Redis 4.0 + 支持一次性设置多个 field-value 对，完全替代废弃的 HMSET 命令。
- **时间复杂度**：O (1) 单个 field，O (N) 多个 field，N 为设置的 field 数量
- **返回值**：整数类型，本次操作**新增的 field 数量**，覆盖更新的 field 不计入。

##### 2. HGET 获取哈希字段值

- **语法**：`HGET key field`
- **核心功能**：获取哈希中指定 field 对应的 value。
- **时间复杂度**：O(1)
- **返回值**：field 存在返回对应的 value；field 不存在或 key 不存在返回`(nil)`；非 Hash 类型返回类型错误。

##### 3. HSETNX 字段不存在时原子设置

- **语法**：`HSETNX key field value`
- **核心功能**：仅当哈希中指定 field 不存在时，才设置其 value；field 已存在则不执行任何操作，全程原子性。
- **时间复杂度**：O(1)
- **返回值**：1 = 设置成功；0=field 已存在，设置失败。
- **生产规范**：适合幂等性写入场景，避免覆盖已有字段值，无需业务层加锁。

---

#### 二、批量操作命令

##### 1. HMGET 批量获取字段值

- **语法**：`HMGET key field [field ...]`
- **核心功能**：按传入顺序，批量获取哈希中多个 field 对应的 value。
- **时间复杂度**：O (N)，N 为传入的 field 数量
- **返回值**：数组类型，长度与传入 field 数量一致；不存在的 field 对应位置返回`(nil)`。
- **生产规范**：批量读取多个字段时必须使用 HMGET，禁止循环执行 HGET，减少网络 RTT 开销。

##### 2. HGETALL 获取全量字段与值

- **语法**：`HGETALL key`
- **核心功能**：返回哈希中所有的 field-value 对，数组中交替存储 field 与 value。
- **时间复杂度**：O (N)，N 为哈希的 field 总数量
- **返回值**：field-value 对交替排列的数组；key 不存在返回空数组。
- **生产环境铁律**：**绝对禁止在生产环境对大哈希执行 HGETALL**，当 field 数量超过 1000 时，会导致主线程长时间阻塞；生产环境必须使用 HSCAN 增量遍历，完全替代 HGETALL、HKEYS、HVALS。

##### 3. HKEYS 获取所有字段名

- **语法**：`HKEYS key`
- **核心功能**：返回哈希中所有的 field 名称。
- **时间复杂度**：O (N)，N 为哈希的 field 总数量
- **返回值**：field 名称数组；key 不存在返回空数组。
- **生产规范**：大哈希场景禁止使用，必须用 HSCAN 替代。

##### 4. HVALS 获取所有字段值

- **语法**：`HVALS key`
- **核心功能**：返回哈希中所有 field 对应的 value。
- **时间复杂度**：O (N)，N 为哈希的 field 总数量
- **返回值**：value 数组；key 不存在返回空数组。
- **生产规范**：大哈希场景禁止使用，必须用 HSCAN 替代。

#### 三、原子增减命令

##### 1. HINCRBY 字段整数值原子递增

- **语法**：`HINCRBY key field increment`
- **核心功能**：将哈希中指定 field 的整数值，原子性地加上指定步长 increment（支持正负值，负步长等效于递减）；field 不存在则先初始化为 0，再执行递增操作。
- **时间复杂度**：O(1)
- **返回值**：整数类型，执行操作后的最新数值。
- **生产规范**：适合结构化对象的单字段计数场景（如商品库存、用户积分、文章点赞数），原子性无竞态，无需业务层加锁。

##### 2. HINCRBYFLOAT 字段浮点数值原子递增

- **语法**：`HINCRBYFLOAT key field increment`
- **核心功能**：将哈希中指定 field 的浮点数值，原子性地加上指定浮点步长 increment（支持正负值）；field 不存在则先初始化为 0，再执行操作。
- **时间复杂度**：O(1)
- **返回值**：字符串类型，执行操作后的最新浮点数值。
- **生产规范**：遵循 IEEE 754 浮点数标准，存在极小精度误差，金融级高精度计算场景禁止使用，需转为整数分位存储。

#### 四、字段管理与增量遍历命令

##### 1. HEXISTS 判断字段是否存在

- **语法**：`HEXISTS key field`
- **核心功能**：检查哈希中是否存在指定的 field。
- **时间复杂度**：O(1)
- **返回值**：1=field 存在；0=field 不存在或 key 不存在。

##### 2. HDEL 删除哈希字段

- **语法**：`HDEL key field [field ...]`
- **核心功能**：删除哈希中指定的一个或多个 field，不存在的 field 会被忽略。
- **时间复杂度**：O (1) 单个 field，O (N) 多个 field
- **返回值**：整数类型，实际成功删除的 field 数量。

##### 3. HLEN 获取哈希字段总数

- **语法**：`HLEN key`
- **核心功能**：返回哈希中 field 的总数量。
- **时间复杂度**：O (1)，底层结构原生维护字段计数器，无需遍历
- **返回值**：整数类型，field 总数量；key 不存在返回 0。

##### 4. HSCAN 增量遍历哈希

- **语法**：`HSCAN key cursor [MATCH pattern] [COUNT count]`
- **核心功能**：增量迭代哈希中的所有 field-value 对，避免 HGETALL 的主线程阻塞问题。
    
    - `cursor`：迭代游标，首次迭代传入 0，后续使用返回的游标继续迭代，游标返回 0 表示迭代完成；
    - `MATCH pattern`：可选，仅返回匹配指定模式的 field；
    - `COUNT count`：可选，提示单次迭代返回的元素数量，默认 10，仅为参考值，不保证严格匹配。
    
- **时间复杂度**：单次调用 O (1)，全量迭代总复杂度 O (N)，N 为哈希的 field 总数量
- **返回值**：数组格式，第一个元素为下一次迭代的游标，第二个元素为本次迭代的 field-value 对数组。
- **生产规范**：生产环境遍历大哈希的唯一合规方案，完全替代 HGETALL、HKEYS、HVALS，无主线程阻塞风险。
