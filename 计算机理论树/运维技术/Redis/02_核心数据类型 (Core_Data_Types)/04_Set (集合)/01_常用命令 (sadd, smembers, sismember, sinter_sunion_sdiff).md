Redis Set 是**无序、元素唯一不可重复**的字符串元素集合，等效于数学意义上的集合，支持集合间交、并、差运算，所有单元素操作均为原子性，单 Set 最大支持 2^32-1 个元素。

- 核心特性：元素唯一性天然实现去重能力；单元素存在性检查时间复杂度 O (1)，远超 List 类型；原生支持集合间原子性运算，是多维度关联业务场景的核心实现；元素无序，无索引概念，无法通过下标定位元素。
- 逻辑定位：用户层面的无序唯一集合，底层编码为 intset（整数集合）与 hashtable（哈希表），Redis 根据元素类型与数量自动切换编码，平衡内存占用与操作性能。

### 01_常用命令

Set 命令按功能分为**元素增删查与统计、全量读取与增量遍历、随机元素操作、集合间运算**四大类，严格标注时间复杂度与生产规范。

#### 一、元素增删查与统计核心命令（O (1) 高频操作）

##### 1. SADD 集合添加元素

- **语法**：`SADD key member [member ...]`
- **核心功能**：原子性地向集合中添加一个或多个元素，已存在的元素会自动忽略，不会重复存储。
- **时间复杂度**：O (1) 单个元素，O (N) 多个元素，N 为添加的元素数量
- **返回值**：整数类型，本次操作**实际新增成功的元素数量**，已存在的元素不计入。
- **生产规范**：天然支持去重，无需业务层判断元素是否存在，原子性避免并发竞态问题；单次添加元素数量建议控制在 1000 以内，避免主线程阻塞。

##### 2. SREM 集合删除元素

- **语法**：`SREM key member [member ...]`
- **核心功能**：原子性地从集合中删除一个或多个指定元素，不存在的元素会被自动忽略。
- **时间复杂度**：O (1) 单个元素，O (N) 多个元素
- **返回值**：整数类型，本次操作**实际成功删除的元素数量**，不存在的元素不计入。

##### 3. SISMEMBER 元素存在性检查

- **语法**：`SISMEMBER key member`
- **核心功能**：检查指定元素是否存在于集合中。
- **时间复杂度**：O(1)
- **返回值**：1 = 元素存在于集合中；0 = 元素不存在或 key 不存在。
- **核心价值**：O (1) 时间复杂度的存在性检查，是黑白名单、权限校验、去重场景的核心命令，性能远超 List 类型的遍历查找。

##### 4. SCARD 获取集合元素总数

- **语法**：`SCARD key`
- **核心功能**：返回集合中的元素总数量（基数）。
- **时间复杂度**：O (1)，底层结构原生维护元素计数器，无需遍历集合
- **返回值**：整数类型，集合元素总数；key 不存在返回 0。

#### 二、全量读取与增量遍历命令

##### 1. SMEMBERS 获取集合全量元素

- **语法**：`SMEMBERS key`
- **核心功能**：返回集合中所有的元素。
- **时间复杂度**：O (N)，N 为集合的元素总数量
- **返回值**：集合所有元素的数组；key 不存在返回空数组。
- **生产环境铁律**：**绝对禁止在生产环境对大集合执行 SMEMBERS**，当集合元素超过 1000 条时，会导致主线程长时间阻塞；生产环境必须使用`SSCAN`增量遍历，完全替代 SMEMBERS。

##### 2. SSCAN 增量遍历集合

- **语法**：`SSCAN key cursor [MATCH pattern] [COUNT count]`
- **核心功能**：增量迭代集合中的所有元素，避免 SMEMBERS 的主线程阻塞问题，参数规则与 HSCAN 完全一致：
    
    - `cursor`：迭代游标，首次迭代传入 0，后续使用返回的游标继续迭代，游标返回 0 表示迭代完成；
    - `MATCH pattern`：可选，仅返回匹配指定模式的元素；
    - `COUNT count`：可选，提示单次迭代返回的元素数量，默认 10，仅为参考值。
    
- **时间复杂度**：单次调用 O (1)，全量迭代总复杂度 O (N)，N 为集合元素总数量
- **返回值**：数组格式，第一个元素为下一次迭代的游标，第二个元素为本次迭代的元素数组。
- **生产规范**：生产环境遍历大集合的唯一合规方案，无主线程阻塞风险。

#### 三、随机元素操作命令

##### 1. SRANDMEMBER 随机获取元素

- **语法**：`SRANDMEMBER key [count]`
- **核心功能**：从集合中随机获取元素，不删除元素，count 参数控制返回规则：
    
    - 无 count：返回 1 个随机元素；
    - count>0：返回最多 count 个**不重复**的随机元素，count 超过集合总数时返回全部元素；
    - count<0：返回 | count | 个**允许重复**的随机元素。
    
- **时间复杂度**：O (N)，N 为返回的元素数量
- **返回值**：无 count 时返回单个元素；带 count 时返回元素数组；key 不存在返回`(nil)`。

##### 2. SPOP 随机弹出元素

- **语法**：`SPOP key [count]`
- **核心功能**：从集合中随机移除并返回元素，count 参数指定弹出数量，count 超过集合总数时返回全部元素。
- **时间复杂度**：O (N)，N 为弹出的元素数量
- **返回值**：无 count 时返回单个弹出元素；带 count 时返回弹出元素数组；key 不存在返回`(nil)`。
- **生产规范**：抽奖、盲盒场景的核心命令，原子性实现随机抽取与元素移除，无需业务层加锁。

#### 四、集合间交并差运算命令

该类命令是 Set 类型的核心特性，支持多个集合间的原子性运算，是多维度关联场景的核心实现。

##### 1. SINTER 交集运算

- **语法**：`SINTER key [key ...]`
- **核心功能**：返回所有给定集合的交集，即同时存在于所有集合中的元素。
- **时间复杂度**：O (N*M)，N 为最小集合的元素数量，M 为给定集合的数量
- **返回值**：交集元素的数组；任一集合不存在则返回空数组。

##### 2. SINTERSTORE 交集运算并存储结果

- **语法**：`SINTERSTORE destination key [key ...]`
- **核心功能**：计算所有给定集合的交集，并将结果存储到 destination 集合中，若 destination 已存在则覆盖。
- **时间复杂度**：O (N*M)，规则同 SINTER
- **返回值**：整数类型，交集结果的元素数量。
- **生产规范**：生产环境集合运算的首选方案，避免大结果集通过网络传输，直接存储到 Redis 本地，降低网络开销与阻塞风险。

##### 3. SUNION 并集运算

- **语法**：`SUNION key [key ...]`
- **核心功能**：返回所有给定集合的并集，即存在于任意一个集合中的元素，自动去重。
- **时间复杂度**：O (N)，N 为所有集合的元素总数量
- **返回值**：并集元素的数组。

##### 4. SUNIONSTORE 并集运算并存储结果

- **语法**：`SUNIONSTORE destination key [key ...]`
- **核心功能**：计算所有给定集合的并集，将结果存储到 destination 集合中，已存在则覆盖。
- **时间复杂度**：O (N)，规则同 SUNION
- **返回值**：整数类型，并集结果的元素数量。

##### 5. SDIFF 差集运算

- **语法**：`SDIFF key [key ...]`
- **核心功能**：返回第一个集合与后续所有集合的差集，即存在于第一个集合、但不存在于后续所有集合中的元素。
- **时间复杂度**：O (N)，N 为第一个集合的元素总数量
- **返回值**：差集元素的数组；第一个集合不存在则返回空数组。

##### 6. SDIFFSTORE 差集运算并存储结果

- **语法**：`SDIFFSTORE destination key [key ...]`
    
- **核心功能**：计算第一个集合与后续所有集合的差集，将结果存储到 destination 集合中，已存在则覆盖。
    
- **时间复杂度**：O (N)，规则同 SDIFF
    
- **返回值**：整数类型，差集结果的元素数量。
    
- **集合运算生产规范**：
    
    1. 生产环境禁止直接执行大集合的 SINTER/SUNION/SDIFF 运算，必须使用 STORE 系列命令将结果存储到新 Key，避免大结果集网络传输阻塞；
    2. 运算时优先将元素数量最少的集合放在第一个位置，降低运算时间复杂度；
    3. Redis Cluster 集群模式下，参与运算的集合必须位于同一哈希槽，否则无法执行跨槽集合运算。
