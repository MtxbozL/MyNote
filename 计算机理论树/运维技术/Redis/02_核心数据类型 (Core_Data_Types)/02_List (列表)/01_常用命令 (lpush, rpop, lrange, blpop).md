Redis List 是**有序、可重复、支持双向高效增删**的字符串元素集合，元素按插入顺序排序，底层基于快速列表实现，单 List 最大支持 2^32-1 个元素。

- 核心特性：列表两端的增删操作时间复杂度为 O (1)，具备原子性；支持阻塞式读写，是 Redis 原生轻量级消息队列的核心实现；元素为字符串类型，兼容 String 类型的所有格式规范。
- 逻辑定位：用户层面的有序列表结构，底层编码经历了 ziplist→linkedlist→quicklist 的演进，Redis 3.2 + 默认使用 quicklist 编码，平衡内存占用与操作性能。

### 01_常用命令

List 命令按功能分为**双向增删核心命令、范围读取与修剪、元素定位与修改、阻塞式读写**四大类，严格标注时间复杂度与生产规范。

#### 一、双向原子增删命令（O (1) 核心操作）

该类命令为 List 的核心高频操作，仅操作列表两端，时间复杂度稳定 O (1)，无性能风险。

##### 1. LPUSH 表头左端插入

- **语法**：`LPUSH key element [element ...]`
- **核心功能**：原子性地向列表的表头（左端）插入一个或多个元素，多元素按传入顺序从左到右依次插入表头。
- **时间复杂度**：O (1) 单个元素，O (N) 多个元素，N 为插入的元素数量
- **返回值**：整数类型，插入完成后列表的总元素数量
- **衍生命令**：`LPUSHX key element`，仅当列表 key 存在时才执行插入，key 不存在返回 0，不创建空列表。

##### 2. RPUSH 表尾右端插入

- **语法**：`RPUSH key element [element ...]`
- **核心功能**：原子性地向列表的表尾（右端）插入一个或多个元素，多元素按传入顺序从左到右依次插入表尾。
- **时间复杂度**：O (1) 单个元素，O (N) 多个元素
- **返回值**：整数类型，插入完成后列表的总元素数量
- **衍生命令**：`RPUSHX key element`，仅当列表 key 存在时才执行插入，key 不存在返回 0。

##### 3. LPOP 表头左端弹出

- **语法**：`LPOP key [count]`
- **核心功能**：移除并返回列表表头（左端）的元素，可选 count 参数指定弹出元素数量，count 超过列表长度时返回全部元素，不会报错。
- **时间复杂度**：O (N)，N 为弹出的元素数量
- **返回值**：无 count 参数时返回弹出的单个元素，key 不存在返回`(nil)`；带 count 参数时返回弹出元素的数组。

##### 4. RPOP 表尾右端弹出

- **语法**：`RPOP key [count]`
- **核心功能**：移除并返回列表表尾（右端）的元素，可选 count 参数指定弹出数量。
- **时间复杂度**：O (N)，N 为弹出的元素数量
- **返回值**：弹出的元素 / 元素数组，key 不存在返回`(nil)`。

##### 5. RPOPLPUSH 原子性元素迁移

- **语法**：`RPOPLPUSH source destination`
- **核心功能**：原子性地从 source 列表的表尾弹出元素，插入到 destination 列表的表头，并返回该元素。全程为原子操作，不会被其他命令中断。
- **时间复杂度**：O(1)
- **返回值**：被迁移的元素；source 列表不存在返回`(nil)`。
- **核心价值**：实现可靠消息队列的核心命令，消费前将消息迁移至待确认列表，避免消费者宕机导致消息丢失。

#### 二、范围读取与列表修剪

##### 1. LLEN 获取列表长度

- **语法**：`LLEN key`
- **核心功能**：返回列表的总元素数量。
- **时间复杂度**：O (1)，底层结构原生维护长度计数器，无需遍历列表
- **返回值**：整数类型，列表元素总数；key 不存在返回 0；非 List 类型返回类型错误。

##### 2. LRANGE 范围读取元素

- **语法**：`LRANGE key start stop`
- **核心功能**：返回列表中`[start, stop]`闭区间内的所有元素，索引规则如下：
    
    - 正数索引：从表头开始计数，0 为第一个元素，1 为第二个，以此类推；
    - 负数索引：从表尾开始计数，-1 为最后一个元素，-2 为倒数第二个，以此类推；
    - stop 超出列表最大索引时，自动截断至表尾，不会报错。
    
- **时间复杂度**：O (S+N)，S 为 start 索引到列表两端的偏移量，N 为返回的元素数量
- **返回值**：指定范围内的元素数组；key 不存在返回空数组。
- **生产环境铁律**：**禁止执行`LRANGE key 0 -1`全量读取大列表**，当列表元素超过 1000 条时，会导致主线程长时间阻塞；生产环境必须分页读取，单次读取元素数量控制在 1000 以内。

##### 3. LTRIM 列表定长修剪

- **语法**：`LTRIM key start stop`
- **核心功能**：仅保留列表中`[start, stop]`闭区间内的元素，删除区间外的所有元素，索引规则与 LRANGE 完全一致。
- **时间复杂度**：O (N)，N 为被删除的元素数量
- **返回值**：执行成功返回`OK`；key 不存在返回错误。
- **生产规范**：常与 LPUSH 配合实现固定长度列表（如最新 100 条动态），避免列表无限增长；修剪操作需避开高并发时段，防止大量元素删除阻塞主线程。

#### 三、元素定位、修改与删除

该类命令需遍历列表定位元素，时间复杂度随列表长度增长而升高，生产环境禁止对大列表高频使用。

##### 1. LINDEX 获取指定索引元素

- **语法**：`LINDEX key index`
- **核心功能**：返回列表中指定索引位置的元素，索引规则与 LRANGE 一致。
- **时间复杂度**：O (N)，N 为索引到列表两端的距离；正数索引从表头遍历，负数索引从表尾遍历
- **返回值**：索引对应的元素；索引超出范围或 key 不存在返回`(nil)`。
- **生产规范**：禁止对大列表频繁调用 LINDEX 查询中间索引，优先使用 LRANGE 读取两端连续元素，降低遍历开销。

##### 2. LSET 设置指定索引元素

- **语法**：`LSET key index element`
- **核心功能**：将列表中指定索引位置的元素值替换为新元素。
- **时间复杂度**：O (N)，N 为索引到列表两端的距离
- **返回值**：执行成功返回`OK`；索引超出范围或 key 不存在返回错误。

##### 3. LREM 删除指定匹配元素

- **语法**：`LREM key count element`
- **核心功能**：删除列表中与 element 匹配的元素，count 参数控制删除规则：
    
    - `count > 0`：从表头开始，最多删除 count 个匹配元素；
    - `count < 0`：从表尾开始，最多删除 | count | 个匹配元素；
    - `count = 0`：删除列表中所有匹配的元素。
    
- **时间复杂度**：O (N)，N 为列表总元素数量
- **返回值**：整数类型，实际成功删除的元素数量；key 不存在返回 0。

##### 4. LINSERT 指定位置插入元素

- **语法**：`LINSERT key BEFORE|AFTER pivot element`
- **核心功能**：在列表中基准元素 pivot 的前 / 后位置插入新元素 element。
- **时间复杂度**：O (N)，N 为基准元素 pivot 到列表两端的距离
- **返回值**：插入成功后列表的总元素数量；pivot 不存在返回 - 1；key 不存在返回 0。

#### 四、阻塞式读写命令

该类命令是 List 实现异步消息队列的核心，支持列表为空时客户端阻塞等待，避免轮询带来的 CPU 开销，所有操作均为原子性。

##### 1. BLPOP 阻塞式表头弹出

- **语法**：`BLPOP key [key ...] timeout`
- **核心功能**：监听一个或多个列表，按顺序检查列表是否存在元素：
    
    - 若至少有一个列表非空，立即从第一个非空列表的表头弹出元素并返回；
    - 若所有列表均为空，客户端阻塞等待，直到有列表新增元素，或达到 timeout 超时时间；
    - timeout 单位为秒，设置为 0 表示永久阻塞。
    
- **时间复杂度**：O(1)
- **返回值**：数组格式，第一个元素为弹出元素所属的列表 key，第二个元素为弹出的元素值；超时无元素返回`(nil)`。
- **核心特性**：支持多客户端同时监听同一个列表，元素弹出时仅会分配给一个客户端，天然实现负载均衡。

##### 2. BRPOP 阻塞式表尾弹出

- **语法**：`BRPOP key [key ...] timeout`
- **核心功能**：与 BLPOP 完全一致，唯一区别为从列表的表尾（右端）弹出元素。
- **时间复杂度**：O(1)
- **返回值**：与 BLPOP 格式一致。

##### 3. BRPOPLPUSH 阻塞式元素迁移

- **语法**：`BRPOPLPUSH source destination timeout`
- **核心功能**：RPOPLPUSH 的阻塞版本，当 source 列表为空时，客户端阻塞等待，直到 source 列表有元素可迁移；元素迁移全程为原子操作。
- **时间复杂度**：O(1)
- **返回值**：被迁移的元素；超时无元素返回`(nil)`。
- **生产规范**：实现可靠消息队列的核心命令，消费前将消息迁移至待确认列表，消费成功后删除待确认列表中的消息，避免消费者宕机导致消息丢失。
