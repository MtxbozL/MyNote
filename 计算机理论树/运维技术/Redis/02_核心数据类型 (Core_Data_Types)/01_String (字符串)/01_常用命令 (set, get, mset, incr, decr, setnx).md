String 是 Redis 最基础、最核心的键值数据类型，Key 为字符串格式，Value 可存储结构化字符串、整数、浮点数、二进制数据（图片、序列化对象等），单 Key 最大存储容量为**512MB**。

- 核心特性：所有单命令操作均具备原子性，基于单线程执行模型保证并发安全；支持二进制安全，可存储任意格式数据，无特殊字符截断问题；是 Redis 所有其他数据类型的底层基础。
- 逻辑定位：用户层面的字符串为逻辑数据类型，对应底层三种物理编码结构（int、embstr、raw），底层均基于 SDS（简单动态字符串）实现。

### 01_常用命令

#### 一、基础读写核心命令

##### 1. SET 键值设置

- **语法**：`SET key value [EX seconds | PX milliseconds | EXAT timestamp | PXAT timestamp] [NX | XX] [KEEPTTL] [GET]`
- **核心功能**：为指定 Key 设置字符串类型的 Value，支持丰富的可选参数实现条件设置、过期时间绑定、原子性读写等能力，是 String 类型最核心的命令，可完全替代 SETNX、SETEX、PSETEX 等衍生命令。
- **时间复杂度**：O(1)
- **核心参数详解**：
    
    表格
    
    |参数|功能说明|适用场景|
    |---|---|---|
    |EX seconds|为 Key 设置秒级过期时间，等效于 SETEX|缓存场景绑定过期时间，避免内存泄漏|
    |PX milliseconds|为 Key 设置毫秒级过期时间，等效于 PSETEX|分布式锁、高精度超时控制场景|
    |EXAT timestamp|设置 Key 的过期时间为指定秒级 Unix 时间戳|固定过期时间的业务场景（如活动结束时间）|
    |PXAT timestamp|设置 Key 的过期时间为指定毫秒级 Unix 时间戳|高精度固定过期场景|
    |NX|仅当 Key 不存在时，才执行 SET 操作，等效于 SETNX|分布式锁、幂等性控制场景，原子性避免竞态|
    |XX|仅当 Key 已存在时，才执行 SET 操作，覆盖原有 Value|仅更新已存在的缓存，禁止新增场景|
    |KEEPTTL|保留 Key 原有的过期时间，不覆盖|更新 Value 但不改变原有过期时间的场景|
    |GET|设置新 Value 的同时，返回 Key 原有的 Value，原子性读写|原子性数据替换、幂等性校验场景|
    
- **返回值**：
    
    - 无 GET 参数：设置成功返回`OK`，不满足 NX/XX 条件设置失败返回`(nil)`
    - 带 GET 参数：返回 Key 原有的 Value，Key 不存在返回`(nil)`；若 Key 原有 Value 非 String 类型，返回类型错误
    
- **生产规范**：
    
    1. 缓存场景必须通过 EX/PX 参数绑定过期时间，禁止先 SET 再 EXPIRE 的非原子操作，避免程序崩溃导致 Key 永久有效；
    2. 分布式锁场景必须使用`SET key value NX PX milliseconds`单命令原子性设置，禁止拆分 SETNX+EXPIRE 两条命令，避免竞态条件导致死锁；
    3. 禁止存储超大 Value（单 Value 超过 100KB），String 类型大 Value 会导致网络阻塞、内存碎片、主线程卡顿，生产环境单 Value 建议控制在 10KB 以内。
    

##### 2. GET 键值读取

- **语法**：`GET key`
- **核心功能**：获取指定 Key 对应的 Value，仅支持 String 类型的 Key，非 String 类型 Key 执行会返回类型错误。
- **时间复杂度**：O(1)
- **返回值**：Key 存在且为 String 类型，返回对应的 Value；Key 不存在返回`(nil)`；Key 存在但非 String 类型，返回类型错误。
- **生产规范**：无性能风险，可高频调用；禁止在循环中批量执行 GET，应使用 MGET 替代，减少网络 RTT 开销。

#### 二、批量操作命令

##### 1. MSET 批量键值设置

- **语法**：`MSET key value [key value ...]`
- **核心功能**：原子性地同时设置多个 Key-Value 对，所有 Key 的设置操作要么全部成功，要么全部失败，无中间状态。
- **时间复杂度**：O (N)，N 为设置的 Key-Value 对数量
- **返回值**：始终返回`OK`（语法错误除外）
- **核心特性与生产规范**：
    
    1. 原子性保证：MSET 为原子操作，执行过程中不会被其他命令中断，避免部分 Key 设置成功、部分失败的不一致问题；
    2. 性能优势：批量设置 N 个 Key 仅需 1 次网络 RTT，相比循环执行 SET 性能提升显著，适合批量缓存预热场景；
    3. 限制：MSET 不支持 NX/XX 条件参数与过期时间设置，需批量设置带过期时间的 Key 时，应使用 Pipeline 或 Lua 脚本保证原子性；
    4. 生产限制：单次 MSET 传入的 Key 数量建议控制在 1000 以内，避免单次命令传输数据量过大导致主线程阻塞、网络缓冲区溢出。
    

##### 2. MGET 批量键值读取

- **语法**：`MGET key [key ...]`
- **核心功能**：原子性地同时获取多个 Key 对应的 Value，按传入 Key 的顺序返回结果。
- **时间复杂度**：O (N)，N 为读取的 Key 数量
- **返回值**：数组类型，长度与传入 Key 数量一致；对应 Key 不存在或非 String 类型，对应位置返回`(nil)`，不影响其他 Key 的读取。
- **核心特性与生产规范**：
    
    1. 原子性保证：MGET 为原子操作，读取的是同一时间点的快照数据，避免并发场景下的数据不一致；
    2. 性能优势：批量读取 N 个 Key 仅需 1 次网络 RTT，是生产环境批量读取的唯一合规方案，禁止循环执行 GET；
    3. 生产限制：单次 MGET 传入的 Key 数量建议控制在 1000 以内，避免单次读取数据量过大导致网络阻塞、客户端内存溢出。

#### 三、数值增减原子命令

该系列命令仅支持 Value 为**整数 / 浮点数**的 String 类型 Key，所有操作均为原子性，无竞态条件，是高并发计数场景的核心实现。

##### 1. INCR 原子递增 1

- **语法**：`INCR key`
- **核心功能**：将 Key 对应的数值原子性加 1；Key 不存在则先初始化为 0 再执行加 1；Value 非整数类型返回类型错误。
- **时间复杂度**：O(1)
- **返回值**：整数类型，执行递增后的最新数值。
- **生产规范**：高并发计数器场景的首选，原子性避免多线程 / 多进程并发计数的竞态问题，无需业务层加锁。

##### 2. DECR 原子递减 1

- **语法**：`DECR key`
- **核心功能**：将 Key 对应的数值原子性减 1；Key 不存在则先初始化为 0 再执行减 1；Value 非整数类型返回类型错误。
- **时间复杂度**：O(1)
- **返回值**：整数类型，执行递减后的最新数值。
- **生产规范**：支持负数结果，无下限限制，业务层需自行控制数值范围，避免超预期的负数结果。

##### 3. INCRBY 指定步长原子递增

- **语法**：`INCRBY key increment`
- **核心功能**：将 Key 对应的数值原子性加上指定整数步长 increment；支持正 / 负步长，负步长等效于递减；Key 不存在则先初始化为 0 再执行操作。
- **时间复杂度**：O(1)
- **返回值**：整数类型，执行操作后的最新数值。

##### 4. DECRBY 指定步长原子递减

- **语法**：`DECRBY key decrement`
- **核心功能**：将 Key 对应的数值原子性减去指定整数步长 decrement；等效于`INCRBY key -decrement`；Key 不存在则先初始化为 0 再执行操作。
- **时间复杂度**：O(1)
- **返回值**：整数类型，执行操作后的最新数值。

##### 5. INCRBYFLOAT 浮点数原子递增

- **语法**：`INCRBYFLOAT key increment`
- **核心功能**：将 Key 对应的数值原子性加上指定浮点数步长 increment；支持正 / 负步长，支持整数 / 浮点数 Value；Key 不存在则先初始化为 0 再执行操作。
- **时间复杂度**：O(1)
- **返回值**：字符串类型，执行操作后的最新数值（浮点数格式）。
- **生产规范**：浮点数精度遵循 IEEE 754 标准，存在极小的精度误差，金融级高精度计算场景禁止使用，需使用整数分位存储。

#### 四、衍生核心命令

##### 1. SETNX 仅当 Key 不存在时设置（SET if Not eXists）

- **语法**：`SETNX key value`
- **核心功能**：仅当 Key 不存在时设置 Value；等效于`SET key value NX`。
- **时间复杂度**：O(1)
- **返回值**：整数类型，1 = 设置成功，0=Key 已存在，设置失败。
- **生产规范**：Redis 2.6.12+ 优先使用`SET key value NX PX`单命令替代，避免拆分 SETNX+EXPIRE 导致的非原子性问题。

##### 2. SETEX 设置带秒级过期时间的键值

- **语法**：`SETEX key seconds value`
- **核心功能**：原子性设置 Key 的 Value 并绑定秒级过期时间；等效于`SET key value EX seconds`。
- **时间复杂度**：O(1)
- **返回值**：设置成功返回`OK`，过期时间非法返回错误。
- **生产规范**：原子性操作，避免 SET+EXPIRE 拆分的非原子性问题，缓存场景优先使用。

##### 3. APPEND 字符串追加

- **语法**：`APPEND key value`
- **核心功能**：将指定 value 追加到 Key 原有 Value 的末尾；Key 不存在则等效于`SET key value`。
- **时间复杂度**：O (1)（追加操作仅修改 SDS 元数据，预分配内存避免频繁扩容）
- **返回值**：整数类型，追加完成后 Value 的总字节长度。
- **生产规范**：适合日志拼接、增量数据追加场景，避免频繁 SET 全量字符串导致的性能损耗。

##### 4. STRLEN 获取 Value 字节长度

- **语法**：`STRLEN key`
- **核心功能**：返回 Key 对应的 Value 的字节长度（非字符长度）。
- **时间复杂度**：O (1)，SDS 原生存储长度属性，无需遍历字符串计算
- **返回值**：整数类型，Value 的字节长度；Key 不存在返回 0；Key 非 String 类型返回类型错误。
- **注意事项**：中文等多字节字符，单个字符占多个字节，STRLEN 返回的是字节数，而非字符数。
