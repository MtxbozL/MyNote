Redis 2.6+ 内置了 Lua 5.1 解释器，支持客户端发送 Lua 脚本到服务器端执行，脚本内可调用任意 Redis 命令，**整个脚本的执行具备原子性**，执行期间不会被其他客户端的命令打断，是 Redis 实现复杂原子操作的核心方案。

---

### 01_优势 (保证原子性、减少网络开销)

1. **天然保证原子性**
    
    整个 Lua 脚本在 Redis 主线程中串行执行，执行期间不会被其他命令打断，脚本内的所有 Redis 命令要么全部执行，要么都不执行，完美替代 Redis 事务，解决了事务无回滚、无法逻辑判断的核心缺陷，是分布式锁、限流、库存扣减等并发场景的首选实现方案。
    
2. **大幅减少网络开销**
    
    复杂业务逻辑通常需要多次 Redis 命令交互，使用 Lua 脚本可将所有逻辑打包，一次性发送到服务器端执行，一次性返回结果，将多次 RTT 减少为 1 次；同时脚本内可根据前一个命令的执行结果，决定后续的逻辑分支，这是 Pipeline 无法实现的。
    
3. **脚本复用能力**
    
    脚本执行后，Redis 会将脚本的 SHA1 摘要缓存到服务器中，后续可通过`EVALSHA`命令直接调用缓存的脚本，无需重复发送完整脚本内容，进一步减少网络开销，提升性能。
    
4. **逻辑灵活，兼容性强**
    
    Lua 脚本支持条件判断、循环、函数、变量等完整的编程能力，可实现复杂的业务逻辑，无需在客户端处理中间结果；同时支持所有 Redis 命令，适配单机、主从、哨兵、集群模式，只要脚本内的 Key 位于同一个哈希槽，集群模式下即可正常执行。
