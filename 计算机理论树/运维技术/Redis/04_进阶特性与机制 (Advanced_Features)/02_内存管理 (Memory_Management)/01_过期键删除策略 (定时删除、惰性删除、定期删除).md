
## 02_内存管理 (Memory_Management)

Redis 作为内存数据库，内存管理直接决定服务稳定性、资源利用率与业务性能，核心分为**过期键删除策略**与**内存淘汰策略**两大模块，分别解决过期键内存回收与内存上限时的键淘汰问题。

---

### 01_过期键删除策略 (定时删除、惰性删除、定期删除)

Redis 为设置了过期时间的 Key，专门维护了一个**过期字典（expire dict）**：字典的 Key 为 Redis 键，Value 为该键的毫秒级 Unix 过期时间戳，仅设置了过期时间的键会进入该字典，不影响主字典的读写性能。Redis 采用**惰性删除 + 定期删除**结合的策略，平衡 CPU 开销与内存利用率。

#### 一、三种删除策略原理对比

表格

|策略|核心原理|优点|缺点|Redis 是否采用|
|---|---|---|---|---|
|定时删除|设置键过期时间时，同步创建定时器，过期时间到达时，定时器立即触发并删除过期键|内存友好，过期键立即被删除，内存立即释放，无内存泄漏风险|CPU 极不友好，大量过期键会创建大量定时器，CPU 会花费大量时间处理过期键删除，严重挤占主线程命令执行的 CPU 资源，导致性能暴跌|完全不采用|
|惰性删除|不主动删除过期键，仅当客户端访问该键时，先检查键是否过期，过期则立即删除并返回 nil，未过期则正常返回值|CPU 友好，仅访问时才检查删除，不会花费额外 CPU 时间处理未访问的过期键，对主线程性能影响极小|内存极不友好，大量过期键若永远不被访问，会一直占用内存，造成严重的内存泄漏，浪费大量内存资源|采用，作为基础删除策略|
|定期删除|每隔固定时间，执行一次过期键扫描，随机抽取部分键检查并删除其中的过期键，平衡 CPU 与内存开销|平衡 CPU 与内存，通过调整扫描频率与采样数，既不会占用过多 CPU 资源，也不会让过期键占用过多内存，是折中最优解|需精准调整扫描参数，参数过松会导致内存泄漏，参数过严会挤占 CPU 资源|采用，作为核心主动删除策略|

#### 二、Redis 定期删除执行规则

定期删除是 Redis 主动清理过期键的核心机制，执行规则由`hz`配置项控制（默认 10，即每秒执行 10 次，每 100ms 执行一次），单次执行流程如下：

1. 从过期字典中随机抽取 20 个键；
2. 检查并删除其中已过期的键；
3. 若本次删除的过期键占比超过 25%，则重复步骤 1-2，继续扫描；
4. 单次扫描的最大执行时间不超过 CPU 时间片的 25%，避免长时间阻塞主线程。

#### 三、主从架构下的过期键删除规则

为保证主从数据一致性，Redis 主从架构下过期键删除遵循严格规则：

- 从节点**绝对不会主动删除过期键**，即使键已过期，客户端访问从节点时，仍会正常返回该键的值；
- 仅当主节点删除过期键后，向从节点发送 DEL 命令，从节点收到命令后，才会删除对应的过期键；
- 该规则避免了主从节点数据不一致，同时也会导致从节点内存中存在大量过期键，生产环境需关注主从节点的内存占用差异。
