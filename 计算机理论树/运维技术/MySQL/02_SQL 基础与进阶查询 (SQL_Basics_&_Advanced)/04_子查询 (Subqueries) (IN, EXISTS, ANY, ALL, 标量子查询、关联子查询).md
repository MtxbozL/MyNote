
>子查询指嵌套在另一个 SQL 语句中的完整查询语句，也称为内部查询，外层的 SQL 语句称为外部查询。子查询用于实现复杂的条件过滤、数据计算，将复杂查询拆解为多个逻辑单元，提升 SQL 的可读性与可维护性。

### 2.4.1 子查询分类

根据返回结果集的结构，子查询可分为四类：

1. **标量子查询**：返回单行单列的单个值，可与比较运算符配合使用，适用于固定值匹配场景；
2. **列子查询**：返回单列多行的结果集，需配合`IN`、`EXISTS`、`ANY`、`ALL`运算符使用；
3. **行子查询**：返回单行多列的结果集，可与多列等值匹配运算符配合使用，使用频率较低；
4. **表子查询**：返回多行多列的结果集，通常用于`FROM`子句中作为派生表，必须指定别名。

根据执行方式与外部查询的依赖关系，子查询可分为两类：

1. **非关联子查询**：子查询的执行不依赖外部查询的任何字段，可独立执行，执行顺序为：先执行子查询，将结果集传入外部查询，再执行外部查询；
2. **关联子查询**：子查询的执行依赖外部查询的字段值，执行顺序为：外部查询先遍历每一行，将该行的关联字段值传入子查询执行，再根据子查询的返回结果判断是否保留该行。

### 2.4.2 核心运算符与子查询场景

#### 2.4.2.1 IN 子查询

`IN`运算符用于判断外部查询的表达式是否存在于列子查询的结果集中，是列子查询最常用的场景。

- 基础语法：`表达式 IN (列子查询)`
- 核心特性：
    
    1. 等价于多个`OR`条件的组合，适用于固定值集合匹配；
    2. 子查询结果集为 NULL 时，整个条件表达式结果为`UNKNOWN`，外部查询不会返回任何行；
    3. 非关联子查询场景下，MySQL 会将子查询结果集进行去重优化，提升匹配效率。
    

#### 2.4.2.2 EXISTS 子查询

`EXISTS`运算符用于子查询的存在性判断，子查询不返回具体的业务数据，仅返回`TRUE`或`FALSE`：当子查询能返回至少一行结果时，`EXISTS`返回`TRUE`，外部查询保留当前行；否则返回`FALSE`，剔除当前行。

- 基础语法：`WHERE EXISTS (关联子查询)`
- 核心特性：
    
    1. 典型的关联子查询场景，子查询通常会引用外部查询的关联字段；
    2. 子查询的`SELECT`列列表无实际意义，通常写为`SELECT 1`即可，数据库不会解析列内容，仅判断是否存在结果行；
    3. **IN vs EXISTS 核心区别**：当子查询结果集较小时，IN 性能更优；当外部表数据量小、子查询表数据量大时，EXISTS 性能更优；当子查询结果集包含 NULL 值时，IN 返回`UNKNOWN`无结果，EXISTS 仍能正常判断存在性。
    

#### 2.4.2.3 ANY 与 ALL 运算符

`ANY`与`ALL`运算符用于将外部表达式与列子查询的结果集进行批量比较，必须配合比较运算符使用。

- 基础语法：`表达式 比较运算符 ANY/ALL (列子查询)`
- 核心语义：
    
    1. `ANY`：满足子查询结果集中的任意一个值，条件即为`TRUE`；
        
        - `> ANY(子查询)`：大于子查询结果集中的最小值，等价于`> MIN(子查询结果)`；
        - `< ANY(子查询)`：小于子查询结果集中的最大值，等价于`< MAX(子查询结果)`；
        - `= ANY(子查询)`：等价于`IN`运算符。
        
    2. `ALL`：满足子查询结果集中的所有值，条件才为`TRUE`；
        
        - `> ALL(子查询)`：大于子查询结果集中的最大值，等价于`> MAX(子查询结果)`；
        - `< ALL(子查询)`：小于子查询结果集中的最小值，等价于`< MIN(子查询结果)`；
        - `!= ALL(子查询)`：等价于`NOT IN`运算符。
        
    

### 2.4.3 子查询的典型应用场景

1. **WHERE 子句条件过滤**：通过标量子查询、列子查询实现复杂的条件判断，如查询工资高于公司平均工资的员工；
2. **FROM 子句派生表**：将表子查询的结果集作为临时表，供外部查询再次关联、过滤，必须为派生表指定别名；
3. **SELECT 子句标量计算**：通过标量子查询实现行级的关联计算，如查询每个用户的订单总数，需注意关联子查询在 SELECT 子句中会产生逐行执行的性能损耗；
4. **DML 语句数据操作**：在`INSERT`、`UPDATE`、`DELETE`语句中通过子查询实现批量数据操作，如根据子查询结果更新表数据。
