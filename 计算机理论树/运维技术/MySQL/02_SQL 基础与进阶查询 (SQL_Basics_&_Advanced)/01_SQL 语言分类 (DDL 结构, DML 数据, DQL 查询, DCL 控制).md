>结构化查询语言（Structured Query Language, SQL）是关系型数据库的标准交互语言，遵循 ANSI/ISO SQL 标准，是 MySQL 数据操作、查询、管理的核心载体。本章从 SQL 语言分类体系出发，完整覆盖基础查询语法、核心多表联查、子查询体系，以及 MySQL 8.0 新增的现代 SQL 特性，是后续索引调优、执行计划分析、业务 SQL 开发的核心理论与实践基础。

## 2.1 SQL 语言分类

根据功能与操作对象的不同，SQL 语言可分为四大核心类别，覆盖数据库全生命周期的操作需求，各类别边界清晰，操作权限与作用域存在本质差异。

### 2.1.1 DDL 数据定义语言（Data Definition Language）

DDL 用于定义与管理数据库的结构元数据，操作对象为数据库、表、索引、视图等数据库对象，**操作会立即生效且无法通过事务回滚**。

- 核心作用：构建、修改、删除数据库的逻辑结构，不直接操作表内的业务数据。
- 核心语句：
    
    1. `CREATE`：创建数据库、表、索引、视图、存储过程等对象；
    2. `ALTER`：修改已有数据库对象的结构，如表字段增减、类型修改、索引调整；
    3. `DROP`：删除数据库对象，会同时删除对象对应的所有数据与结构；
    4. `TRUNCATE`：清空表内所有数据，重置自增计数器，本质是删除并重建表结构，属于 DDL 范畴；
    5. `RENAME`：重命名数据库对象。
    

### 2.1.2 DML 数据操纵语言（Data Manipulation Language）

DML 用于对表内的业务数据进行增删改操作，是业务系统数据写入的核心载体，**操作受事务控制，支持提交（COMMIT）与回滚（ROLLBACK）**。

- 核心作用：对数据库内的业务数据进行写入、修改、删除操作，不改变表结构。
- 核心语句：
    
    1. `INSERT`：向表中插入新的数据行；
    2. `UPDATE`：修改表中已存在的数据行，需通过`WHERE`条件限定修改范围；
    3. `DELETE`：删除表中的数据行，可通过`WHERE`条件限定删除范围，属于 DML 范畴，与`TRUNCATE`存在本质区别。
    

### 2.1.3 DQL 数据查询语言（Data Query Language）

DQL 用于从数据库中提取符合条件的业务数据，是 SQL 中使用频率最高、逻辑最复杂的部分，也是本章的核心讲解内容。

- 核心作用：根据业务需求，通过条件过滤、分组聚合、多表关联等逻辑，获取目标数据集，不修改数据库内的任何数据。
- 核心语句：`SELECT`，配合`FROM`、`WHERE`、`GROUP BY`、`HAVING`、`ORDER BY`、`LIMIT`等子句，实现全场景查询需求。

### 2.1.4 DCL 数据控制语言（Data Control Language）

DCL 用于管理数据库的用户权限、事务控制，保障数据库的访问安全与数据一致性。

- 核心作用：控制用户对数据库对象的访问权限，管理事务的提交与回滚。
- 核心语句：
    
    1. `GRANT`：为用户授予数据库对象的操作权限；
    2. `REVOKE`：撤销用户已有的操作权限；
    3. `COMMIT`：提交事务，永久保存事务内的 DML 操作结果；
    4. `ROLLBACK`：回滚事务，撤销事务内未提交的 DML 操作结果；
    5. `SAVEPOINT`：设置事务保存点，支持部分回滚。
