
## 2.5 MySQL 8.0 现代 SQL 特性【加分项】

MySQL 8.0 版本对 SQL 标准的支持实现了质的飞跃，新增了窗口函数、CTE 通用表表达式两大核心特性，补齐了与商业数据库的 OLAP 能力差距，大幅简化了复杂查询的实现逻辑，提升了 SQL 的可读性与执行效率。

### 2.5.1 窗口函数（Window Functions）

窗口函数是对查询结果集中的一组行（窗口）进行计算的函数，与聚合函数的核心区别是：**聚合函数会将多行合并为一行，窗口函数不会改变结果集的行数，每行都会返回计算结果**，实现了分组内的排名、累计、偏移、占比等复杂计算，是数据分析、复杂报表的核心工具。

#### 2.5.1.1 核心语法结构

```sql
函数名(参数) OVER (
  [PARTITION BY 分组列1, 分组列2, ...]
  [ORDER BY 排序列1 [ASC|DESC], 排序列2 [ASC|DESC], ...]
  [窗口框架]
)
```

- 语法核心解析：
    
    1. `OVER()`：窗口函数的核心标志，必须存在，空`OVER()`表示将整个结果集作为一个单一窗口；
    2. `PARTITION BY`：将结果集按指定列进行分组，每个分组为一个独立的窗口，窗口函数在每个分组内独立执行，分组间互不影响；
    3. `ORDER BY`：在每个窗口内对行进行排序，决定窗口函数的计算顺序，是排名、累计类函数的必备子句；
    4. **窗口框架**：指定窗口内参与计算的行范围，仅对聚合类窗口函数生效，语法为：
        
        ```sql
        ROWS|RANGE BETWEEN 边界1 AND 边界2
        ```
        
        常用边界值：`UNBOUNDED PRECEDING`（分组第一行）、`CURRENT ROW`（当前行）、`UNBOUNDED FOLLOWING`（分组最后一行）；默认框架为`RANGE BETWEEN UNBOUNDED PRECEDING AND CURRENT ROW`，即从分组第一行到当前行。
    

#### 2.5.1.2 核心窗口函数分类

1. **排名类窗口函数（高频核心）**
    
    用于实现分组内的排名计算，是面试与业务开发的高频场景，核心区别在于排名规则的差异。
    
| 函数名          | 核心规则                       | 排名示例（成绩：90,90,85,80） |
| ------------ | -------------------------- | -------------------- |
| ROW_NUMBER() | 行号，分组内按排序连续编号，即使值相同，编号也不重复 | 1,2,3,4              |
| RANK()       | 跳跃排名，值相同排名相同，后续排名跳过重复序号    | 1,1,3,4              |
| DENSE_RANK() | 密集排名，值相同排名相同，后续排名不跳过序号     | 1,1,2,3              |
    
2. **聚合类窗口函数**
    
    常规聚合函数配合`OVER()`子句使用，实现分组内的累计求和、移动平均、占比计算等需求，无需通过`GROUP BY`合并行。
    
    - 典型示例：累计求和`SUM(amount) OVER (PARTITION BY user_id ORDER BY order_time ROWS UNBOUNDED PRECEDING)`，计算每个用户按订单时间排序的累计消费金额；
    - 典型示例：移动平均`AVG(price) OVER (ORDER BY trade_date ROWS BETWEEN 2 PRECEDING AND CURRENT ROW)`，计算最近 3 个交易日的收盘价移动平均值。
    
3. **偏移类窗口函数**
    
    用于获取当前行相邻行的列值，无需通过自连接实现，是同比环比、相邻行对比场景的最优解决方案。
    
    - `LAG(列名, 偏移量, 默认值)`：获取当前行**前面第 N 行**的指定列值，偏移量默认为 1，无匹配行时返回默认值，无默认值则返回 NULL；
    - `LEAD(列名, 偏移量, 默认值)`：获取当前行**后面第 N 行**的指定列值，参数规则与`LAG`一致；
    - 典型示例：`TIMESTAMPDIFF(HOUR, order_time, LEAD(order_time) OVER (PARTITION BY user_id ORDER BY order_time))`，计算用户相邻两次下单的时间间隔。
    

### 2.5.2 CTE 通用表表达式（WITH 关键字）

通用表表达式（Common Table Expression, CTE）是在单个 SQL 语句执行范围内有效的临时命名结果集，通过`WITH`关键字定义，用于将复杂查询拆解为多个逻辑清晰的单元，大幅提升复杂 SQL 的可读性与可维护性，分为非递归 CTE 与递归 CTE 两类。

#### 2.5.2.1 非递归 CTE

- 基础语法：
    
    ```sql
    WITH cte_name1 AS (
      -- 第一个CTE子查询
      SELECT 列列表 FROM 表A WHERE 条件
    ),
    cte_name2 AS (
      -- 第二个CTE子查询，可引用前面定义的CTE
      SELECT 列列表 FROM cte_name1 INNER JOIN 表B ON 关联条件
    )
    -- 主查询，可引用所有定义的CTE
    SELECT 列列表 FROM cte_name2 WHERE 条件;
    ```
    
- 核心特性：
    
    1. 逻辑分层：将嵌套子查询拆解为多个按逻辑顺序定义的 CTE，查询逻辑自上而下清晰可读，替代传统的多层嵌套子查询；
    2. 可复用性：在主查询中可多次引用同一个 CTE，避免重复编写相同的子查询代码；
    3. 作用域：仅在当前 SQL 语句内有效，语句执行完成后自动释放，无需手动删除；
    4. 兼容性：支持在`INSERT`、`UPDATE`、`DELETE`、`CREATE TABLE`语句中使用。
    

#### 2.5.2.2 递归 CTE

递归 CTE 是指在 CTE 内部引用自身的 CTE，用于处理树形结构、层级结构数据，是 MySQL 8.0 的核心新增能力，典型场景包括部门树形结构、商品分类、菜单权限、无限级评论等。

- 固定语法结构，必须包含两部分：
    
    ```sql
    WITH RECURSIVE cte_name AS (
      -- 1. 锚点成员：递归的起始点，非递归查询，返回初始结果集
      SELECT 主键列, 父级列, 其他列, 1 AS level
      FROM 树形表
      WHERE 父级列 IS NULL -- 顶级节点条件
      UNION ALL
      -- 2. 递归成员：引用CTE自身，定义递归逻辑
      SELECT t.主键列, t.父级列, t.其他列, cte.level + 1 AS level
      FROM 树形表 t
      INNER JOIN cte_name cte ON t.父级列 = cte.主键列 -- 递归关联条件
    )
    -- 主查询，获取递归结果
    SELECT * FROM cte_name;
    ```
    
- 核心特性：
    
    1. 必须使用`WITH RECURSIVE`关键字声明递归 CTE；
    2. 锚点成员与递归成员必须通过`UNION ALL`连接，列数、数据类型必须完全一致；
    3. 递归终止条件：递归成员执行后无新的结果行返回，自动终止递归；
    4. 递归深度控制：MySQL 默认`cte_max_recursion_depth`参数值为 1000，限制最大递归深度，避免无限递归导致数据库性能问题。
