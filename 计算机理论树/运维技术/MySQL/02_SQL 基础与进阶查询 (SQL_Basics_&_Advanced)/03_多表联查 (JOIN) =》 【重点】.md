
>多表联查是关系型数据库的核心能力，通过关联条件将多个表的数据进行横向拼接，实现跨表数据查询，是业务复杂查询的核心实现方式。其数学基础为关系代数的连接运算，核心是通过关联条件确定表间行的匹配规则。

### 2.3.1 笛卡尔积现象（Cross Join）

笛卡尔积（交叉连接）是所有连接运算的数学基础，指两个表的所有行进行全量无差别组合，最终结果集的行数 = 左表行数 × 右表行数。

- 基础语法：`SELECT ... FROM 表A CROSS JOIN 表B;`，等价于`SELECT ... FROM 表A, 表B;`
- 核心特性：无任何关联条件的连接必然产生笛卡尔积，业务场景中极少直接使用，绝大多数情况下是未指定有效关联条件导致的错误，会产生海量无效数据，严重影响数据库性能；仅适用于全量组合的特殊场景（如商品与标签的全量匹配、日历维度表的全量组合）。

### 2.3.2 内连接（INNER JOIN）

内连接是最常用的连接类型，仅返回两个表中 **完全满足关联条件** 的行，是两个表结果集的交集。

- 基础语法：
    
    ```sql
    SELECT 列列表
    FROM 表A
    INNER JOIN 表B
    ON 表A.关联列 = 表B.关联列;
    ```
    
- 核心特性：
    
    1. `ON`子句：指定连接的关联条件，是内连接的核心，必须保证关联条件的有效性，避免笛卡尔积；
    2. 等值连接与非等值连接：关联条件为`=`的为等值连接，是最常用的场景；关联条件为`>`、`<`、`BETWEEN`等非等值运算符的为非等值连接，适用于范围匹配场景；
    3. 隐式内连接：可省略`INNER JOIN`关键字，通过`FROM 表A, 表B WHERE 关联条件`实现，可读性差，不符合 SQL 标准，生产环境不推荐使用；
    4. 多表内连接：支持连续多个`INNER JOIN`子句，依次关联多个表，关联条件需保证表间的关联逻辑闭环。
    

### 2.3.3 外连接（OUTER JOIN）

外连接的核心是 **驱动表** 与 **被驱动表** 的区分，会返回驱动表的所有行，即使被驱动表中没有满足关联条件的行，未匹配的列值自动填充为 NULL。外连接分为左外连接、右外连接、全外连接三类。

#### 2.3.3.1 左外连接（LEFT JOIN）

左外连接以 `LEFT JOIN` 左侧的表为驱动表，右侧的表为被驱动表，返回驱动表的所有行，被驱动表中未匹配的行填充 NULL。

- 基础语法：
    
    ```sql
    SELECT 列列表
    FROM 表A -- 驱动表
    LEFT JOIN 表B -- 被驱动表
    ON 表A.关联列 = 表B.关联列;
    ```
    
- 核心业务场景：需要保留驱动表全量数据的场景，如查询所有用户及其订单信息，即使部分用户没有订单，也需要保留用户信息。
- 核心注意事项：对被驱动表的列进行`WHERE`条件过滤时，若过滤条件为`列 IS NOT NULL`，会导致左外连接退化为内连接；需对被驱动表进行前置过滤时，应将过滤条件写在`ON`子句中，或通过子查询提前过滤被驱动表。

#### 2.3.3.2 右外连接（RIGHT JOIN）

右外连接以`RIGHT JOIN`右侧的表为驱动表，左侧的表为被驱动表，逻辑与左外连接完全对称，返回驱动表的所有行，被驱动表中未匹配的行填充 NULL。

- 基础语法：
    
    ```sql
    SELECT 列列表
    FROM 表A -- 被驱动表
    RIGHT JOIN 表B -- 驱动表
    ON 表A.关联列 = 表B.关联列;
    ```
    
- 核心特性：所有右外连接均可等价改写为左外连接，生产环境推荐统一使用左外连接，保证查询逻辑的一致性，提升代码可读性。

#### 2.3.3.3 全外连接的 MySQL 变通实现

全外连接（FULL OUTER JOIN）返回两个表的所有行，无论是否满足关联条件，未匹配的列值填充 NULL，是两个表结果集的并集。**MySQL 原生不支持全外连接语法**，需通过`LEFT JOIN UNION RIGHT JOIN`的方式变通实现。

- 实现语法：
    
    ```sql
    -- 左外连接，获取左表全量数据
    SELECT 列列表 FROM 表A LEFT JOIN 表B ON 表A.关联列 = 表B.关联列
    UNION
    -- 右外连接，获取右表中左表未匹配的数据
    SELECT 列列表 FROM 表A RIGHT JOIN 表B ON 表A.关联列 = 表B.关联列;
    ```
    
- 核心特性：
    
    1. 使用`UNION`而非`UNION ALL`，自动剔除两个结果集中重复的匹配行，保证结果集的正确性；
    2. 仅适用于需要保留两个表全量数据的特殊场景，业务中使用频率较低。
