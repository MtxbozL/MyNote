### 1.2.1 MySQL 基础定位

MySQL 是一款开源的关系型数据库管理系统（RDBMS），采用插件式存储引擎架构，核心面向联机事务处理（OLTP）场景，是当前互联网业务中应用最广泛的数据库之一，现由 Oracle 公司维护迭代。

### 1.2.2 MySQL 5.7 经典版核心特性

MySQL 5.7 是业界长期广泛使用的 LTS（长期支持）版本，已于 2025 年 10 月结束官方扩展支持，其核心能力如下：

1. 架构层面：InnoDB 作为默认存储引擎，完善了事务、行锁、崩溃恢复能力；引入原生系统库 sys，简化运维监控。
2. 功能层面：原生支持 JSON 数据类型；增强 GTID 复制与多源复制；优化器大幅提升子查询、多表联查性能；支持在线 DDL、虚拟列。
3. 安全层面：初始化默认生成随机 root 密码，禁用匿名用户与远程 root 登录，完善密码策略。

### 1.2.3 MySQL 8.0 核心新特性

MySQL 8.0 是当前官方主推的 LTS 版本，在架构、性能、功能、安全上实现了全面升级，核心突破如下：

1. 架构重构：实现事务型数据字典，废弃 MyISAM 存储的元数据，解决元数据事务一致性问题；重构 SQL 优化器，引入直方图统计信息，大幅提升复杂查询执行计划的准确性。
2. InnoDB 引擎增强：支持原子 DDL、自增列持久化、并行查询、缓冲池预热优化；原生支持 InnoDB Cluster 集群架构。
3. SQL 标准增强：原生支持窗口函数、递归 / 非递归 CTE（通用表表达式）、降序索引、不可见索引、CHECK 约束，补齐了与主流商业数据库的 SQL 能力差距。
4. 性能提升：读写性能较 5.7 提升 2 倍以上，优化了高并发场景下的锁竞争、事务提交、连接池性能。
5. 安全与运维：默认采用 caching_sha2_password 认证插件，原生支持数据库角色、细粒度权限控制；增强在线 DDL、并行复制、GTID 能力，简化高可用架构部署。
