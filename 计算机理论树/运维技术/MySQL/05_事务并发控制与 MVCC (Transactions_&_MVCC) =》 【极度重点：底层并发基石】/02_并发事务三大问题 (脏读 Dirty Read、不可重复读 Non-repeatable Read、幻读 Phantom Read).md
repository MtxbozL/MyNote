
>当多个事务并发访问数据库的同一行数据时，若没有有效的隔离机制，会产生三类数据一致性问题，这三类问题是 SQL 隔离级别定义的核心依据。

### 5.2.1 脏读（Dirty Read）

**定义**：一个事务读取了另一个事务**未提交**的修改数据，称为脏读。未提交的数据随时可能被回滚，读取该数据会导致业务逻辑基于错误的脏数据执行，产生严重的一致性问题。

- 典型场景示例：
    
    1. 事务 A 修改账户余额，将 1000 元改为 2000 元，尚未提交；
    2. 事务 B 读取到事务 A 修改后的 2000 元余额，基于该值执行业务逻辑；
    3. 事务 A 执行回滚，余额恢复为 1000 元；
    4. 事务 B 读取到的 2000 元即为脏数据，整个业务逻辑执行错误。
    
- 本质原因：事务读取了其他事务未提交的中间状态数据，违反了事务的原子性边界。

### 5.2.2 不可重复读（Non-repeatable Read）

**定义**：在同一个事务内，执行两次完全相同的查询语句，两次查询返回的结果不一致，称为不可重复读。核心原因是两次查询之间，有其他并发事务对目标数据执行了`UPDATE`或`DELETE`操作并提交，导致当前事务两次读取的结果出现偏差。

- 典型场景示例：
    
    1. 事务 A 执行`SELECT balance FROM account WHERE id = 1`，读取到余额 1000 元，事务未提交；
    2. 事务 B 将该账户余额修改为 2000 元，并提交事务；
    3. 事务 A 再次执行相同的 SELECT 语句，读取到余额 2000 元，与第一次读取结果不一致，无法重复读取到相同的值。
    
- 本质原因：并发事务的提交修改，破坏了当前事务内查询结果的一致性，重点针对数据的更新与删除操作。

### 5.2.3 幻读（Phantom Read）

**定义**：在同一个事务内，执行两次完全相同的范围查询，第二次查询返回了第一次查询中不存在的行，或丢失了第一次查询中存在的行，称为幻读。核心原因是两次查询之间，有其他并发事务对目标范围执行了`INSERT`操作并提交，导致当前事务两次范围查询的结果集不一致。

- 典型场景示例：
    
    1. 事务 A 执行`SELECT * FROM user WHERE age BETWEEN 18 AND 20`，查询到 2 条符合条件的记录，事务未提交；
    2. 事务 B 向 user 表插入一条 age=19 的记录，并提交事务；
    3. 事务 A 再次执行相同的范围查询，返回了 3 条记录，凭空出现了一条 "幻影行"。
    
- 核心区别：不可重复读 vs 幻读
    
| 特性    | 不可重复读               | 幻读                |
| ----- | ------------------- | ----------------- |
| 核心操作  | 针对`UPDATE`/`DELETE` | 针对`INSERT`        |
| 影响范围  | 已存在的单行数据            | 范围查询的结果集行数        |
| 解决方式  | 行锁即可解决              | 行锁无法解决，需间隙锁 / 临键锁 |
| 一致性影响 | 同一行数据两次读取结果不同       | 同一范围查询两次返回的行数不同   |
