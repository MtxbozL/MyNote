## 一、开篇：命令使用前置说明

本笔记聚焦 MySQL 基础开发、运维中**最常用、可直接落地**的命令，所有命令均基于 MySQL 5.7+ 版本（兼容 MySQL 8.0），遵循以下规范：

1. **关键字大写**：SQL 关键字（如 `SELECT`、`CREATE`）统一大写，库表字段名小写，便于区分；
2. **分号结尾**：所有 SQL 命令以分号 `;` 结尾，命令行模式下必须输入分号才会执行；
3. **注释清晰**：命令旁加 `--` 注释，说明命令用途、注意事项；
4. **安全提示**：危险操作（如删除、全表更新）会标注安全提示，避免误操作。

## 二、第一部分：连接与退出命令

### 2.1 连接 MySQL 服务器

#### 基础连接

```bash
# 本地连接（默认端口3306，默认用户root）
mysql -u root -p
# 输入密码后回车登录
```

#### 常用参数连接

```bash
# 完整参数连接示例
mysql -h 192.168.1.100 -P 3307 -u dev_user -p123456 my_database --default-character-set=utf8mb4
```

|参数|说明|示例|
|:--|:--|:--|
|`-h`|主机地址（IP 或域名），本地可省略|`-h 192.168.1.100`|
|`-P`|端口号，默认 3306，注意是大写 P|`-P 3307`|
|`-u`|用户名|`-u dev_user`|
|`-p`|密码，-p 后直接跟密码（无空格），或单独输入（更安全）|`-p123456` 或 `-p` 后输入|
|库名|直接指定登录后使用的数据库，可省略|`my_database`|
|`--default-character-set`|指定连接字符集，推荐 utf8mb4|`--default-character-set=utf8mb4`|

### 2.2 退出 MySQL

```sql
-- 方式1：推荐
EXIT;
-- 方式2：
QUIT;
-- 方式3：快捷键（命令行模式）
Ctrl + D
```

## 三、第二部分：数据库（库）操作命令

### 3.1 创建数据库

```sql
-- 基础创建
CREATE DATABASE my_database;

-- 完整创建（指定字符集和排序规则，推荐）
CREATE DATABASE IF NOT EXISTS my_database 
DEFAULT CHARACTER SET utf8mb4 
DEFAULT COLLATE utf8mb4_unicode_ci;
```

|关键字|说明|
|:--|:--|
|`IF NOT EXISTS`|避免重复创建报错，推荐加上|
|`DEFAULT CHARACTER SET`|指定库的默认字符集，统一用 utf8mb4|
|`DEFAULT COLLATE`|指定排序规则，推荐 utf8mb4_unicode_ci（排序准确）或 utf8mb4_general_ci（性能高）|

### 3.2 查看数据库

```sql
-- 查看所有数据库
SHOW DATABASES;

-- 查看当前使用的数据库
SELECT DATABASE();

-- 查看数据库的创建语句（可查看字符集等信息）
SHOW CREATE DATABASE my_database;
```

### 3.3 选择 / 切换数据库

```sql
-- 切换到指定数据库
USE my_database;
```

### 3.4 修改数据库

```sql
-- 修改数据库的字符集和排序规则
ALTER DATABASE my_database 
DEFAULT CHARACTER SET utf8mb4 
DEFAULT COLLATE utf8mb4_unicode_ci;
```

### 3.5 删除数据库

⚠️ **危险操作**：删除后数据无法恢复，务必确认！

```sql
-- 基础删除
DROP DATABASE my_database;

-- 安全删除（避免库不存在报错）
DROP DATABASE IF EXISTS my_database;
```

## 四、第三部分：表操作命令

### 4.1 创建表

#### 基础创建

```sql
-- 完整建表示例（遵循库表设计规范）
CREATE TABLE IF NOT EXISTS user_info (
    id BIGINT UNSIGNED PRIMARY KEY AUTO_INCREMENT COMMENT '主键ID',
    user_name VARCHAR(50) NOT NULL DEFAULT '' COMMENT '用户姓名',
    phone CHAR(11) NOT NULL DEFAULT '' COMMENT '手机号',
    age TINYINT UNSIGNED NOT NULL DEFAULT 0 COMMENT '年龄',
    gender TINYINT UNSIGNED NOT NULL DEFAULT 0 COMMENT '性别：0-未知，1-男，2-女',
    is_deleted TINYINT UNSIGNED NOT NULL DEFAULT 0 COMMENT '是否删除：0-未删除，1-已删除',
    create_time DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
    update_time DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
    UNIQUE INDEX uk_phone (phone) COMMENT '手机号唯一索引',
    INDEX idx_create_time (create_time) COMMENT '创建时间索引'
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='用户信息表';
```

|关键字|说明|
|:--|:--|
|`IF NOT EXISTS`|避免重复创建报错|
|`PRIMARY KEY`|主键，推荐自增 BIGINT UNSIGNED|
|`AUTO_INCREMENT`|自增，仅适用于整数类型主键|
|`NOT NULL`|非空约束，尽量给所有字段加上|
|`DEFAULT`|默认值|
|`COMMENT`|表 / 字段注释，必须清晰|
|`ENGINE`|存储引擎，统一用 InnoDB|
|`UNIQUE INDEX`|唯一索引|
|`INDEX`|普通索引|

### 4.2 查看表

```sql
-- 查看当前库的所有表
SHOW TABLES;

-- 查看表结构（最常用）
DESC user_info;
-- 等价写法
DESCRIBE user_info;

-- 查看表的创建语句（可查看索引、字符集、存储引擎等完整信息）
SHOW CREATE TABLE user_info;

-- 查看表的状态（可查看数据量、碎片率等）
SHOW TABLE STATUS LIKE 'user_info';
```

### 4.3 修改表

#### 4.3.1 新增字段

```sql
-- 新增单个字段
ALTER TABLE user_info ADD COLUMN address VARCHAR(255) NOT NULL DEFAULT '' COMMENT '地址' AFTER phone;

-- 新增多个字段
ALTER TABLE user_info 
ADD COLUMN province VARCHAR(20) NOT NULL DEFAULT '' COMMENT '省份' AFTER address,
ADD COLUMN city VARCHAR(20) NOT NULL DEFAULT '' COMMENT '城市' AFTER province;
```

|关键字|说明|
|:--|:--|
|`AFTER`|指定新增字段的位置，放在某个字段之后|
|`FIRST`|放在表的最前面（不推荐）|

#### 4.3.2 修改字段

```sql
-- 修改字段类型、默认值、注释（推荐用MODIFY）
ALTER TABLE user_info MODIFY COLUMN age INT UNSIGNED NOT NULL DEFAULT 0 COMMENT '年龄（修改后）';

-- 修改字段名、类型等（MySQL 8.0+ 推荐用RENAME COLUMN，旧版本用CHANGE）
-- MySQL 8.0+
ALTER TABLE user_info RENAME COLUMN old_name TO new_name;
-- 旧版本（需重新指定字段类型等所有属性）
ALTER TABLE user_info CHANGE COLUMN old_name new_name VARCHAR(50) NOT NULL DEFAULT '' COMMENT '新字段名';
```

#### 4.3.3 删除字段

⚠️ **危险操作**：删除后字段数据无法恢复！

```sql
ALTER TABLE user_info DROP COLUMN address;
```

#### 4.3.4 修改表名

```sql
-- 方式1
ALTER TABLE user_info RENAME TO user_info_new;
-- 方式2
RENAME TABLE user_info TO user_info_new;
```

#### 4.3.5 修改表的字符集

```sql
ALTER TABLE user_info CONVERT TO CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
```

### 4.4 清空表

⚠️ **危险操作**：清空后数据无法恢复！

```sql
-- 方式1：DELETE（逐行删除，慢，可回滚，不会重置自增ID）
DELETE FROM user_info; -- 全表清空，务必加WHERE条件！
DELETE FROM user_info WHERE id > 100; -- 条件删除

-- 方式2：TRUNCATE（快速清空，不可回滚，重置自增ID，推荐）
TRUNCATE TABLE user_info;
```

### 4.5 删除表

⚠️ **危险操作**：删除后表和数据无法恢复！

```sql
-- 基础删除
DROP TABLE user_info;

-- 安全删除
DROP TABLE IF EXISTS user_info;
```

## 五、第四部分：数据操作命令（CRUD，最常用）

### 5.1 插入数据（INSERT）

```sql
-- 1. 单条插入（指定字段，推荐）
INSERT INTO user_info (user_name, phone, age, gender) 
VALUES ('张三', '13800138000', 25, 1);

-- 2. 单条插入（不指定字段，需按表结构顺序赋值，不推荐）
INSERT INTO user_info 
VALUES (NULL, '李四', '13800138001', 30, 2, 0, NOW(), NOW());

-- 3. 批量插入（性能最高，推荐）
INSERT INTO user_info (user_name, phone, age, gender) 
VALUES 
('王五', '13800138002', 28, 1),
('赵六', '13800138003', 22, 2),
('孙七', '13800138004', 35, 1);

-- 4. 插入或更新（如果唯一索引冲突则更新，否则插入）
INSERT INTO user_info (user_name, phone, age, gender) 
VALUES ('张三', '13800138000', 26, 1)
ON DUPLICATE KEY UPDATE age = 26, gender = 1;
```

### 5.2 查询数据（SELECT，核心中的核心）

#### 基础查询

```sql
-- 1. 查询所有字段（不推荐，只查需要的字段）
SELECT * FROM user_info;

-- 2. 查询指定字段（推荐）
SELECT id, user_name, phone FROM user_info;

-- 3. 字段别名（AS可省略）
SELECT id AS '用户ID', user_name '用户名' FROM user_info;

-- 4. 去重查询
SELECT DISTINCT gender FROM user_info;
```

#### 条件查询（WHERE）

```sql
-- 1. 基础条件
SELECT * FROM user_info WHERE id = 1;
SELECT * FROM user_info WHERE age > 20;
SELECT * FROM user_info WHERE age >= 20 AND age <= 30;
SELECT * FROM user_info WHERE age BETWEEN 20 AND 30; -- 等价于上面

-- 2. IN/NOT IN
SELECT * FROM user_info WHERE id IN (1, 2, 3);
SELECT * FROM user_info WHERE id NOT IN (1, 2, 3);

-- 3. LIKE模糊查询（%匹配任意字符，_匹配单个字符）
SELECT * FROM user_info WHERE user_name LIKE '张%'; -- 前缀匹配，可命中索引
SELECT * FROM user_info WHERE user_name LIKE '%三'; -- 左模糊，无法命中索引
SELECT * FROM user_info WHERE user_name LIKE '张_'; -- 匹配两个字，姓张

-- 4. IS NULL/IS NOT NULL
SELECT * FROM user_info WHERE address IS NULL;
SELECT * FROM user_info WHERE address IS NOT NULL;

-- 5. 多条件组合（AND/OR，AND优先级高于OR，用括号明确优先级）
SELECT * FROM user_info WHERE age > 20 AND gender = 1;
SELECT * FROM user_info WHERE (age > 20 OR gender = 1) AND is_deleted = 0;
```

#### 排序（ORDER BY）

```sql
-- 1. 单字段排序（ASC升序，默认；DESC降序）
SELECT * FROM user_info ORDER BY age ASC;
SELECT * FROM user_info ORDER BY create_time DESC;

-- 2. 多字段排序
SELECT * FROM user_info ORDER BY age DESC, create_time ASC;
```

#### 分组（GROUP BY）与聚合函数

```sql
-- 常用聚合函数：COUNT、SUM、AVG、MAX、MIN
-- 1. 基础分组
SELECT gender, COUNT(*) AS '人数' FROM user_info GROUP BY gender;

-- 2. 分组后筛选（HAVING，分组后筛选，WHERE是分组前筛选）
SELECT gender, AVG(age) AS '平均年龄' 
FROM user_info 
GROUP BY gender 
HAVING AVG(age) > 25;
```

#### 分页（LIMIT）

```sql
-- 语法：LIMIT offset, size（offset是偏移量，从0开始；size是每页数量）
SELECT * FROM user_info ORDER BY id DESC LIMIT 0, 10; -- 第1页
SELECT * FROM user_info ORDER BY id DESC LIMIT 10, 10; -- 第2页
```

#### JOIN 关联查询

```sql
-- 准备表：order_info（订单表）
CREATE TABLE order_info (
    id BIGINT UNSIGNED PRIMARY KEY AUTO_INCREMENT,
    order_no VARCHAR(32) NOT NULL,
    user_id BIGINT UNSIGNED NOT NULL,
    order_amount DECIMAL(10,2) NOT NULL,
    create_time DATETIME NOT NULL
);

-- 1. 内连接（INNER JOIN，只返回两张表都匹配的记录，INNER可省略）
SELECT u.user_name, o.order_no, o.order_amount 
FROM user_info u
INNER JOIN order_info o ON u.id = o.user_id;

-- 2. 左连接（LEFT JOIN，返回左表所有记录，右表匹配不到的为NULL）
SELECT u.user_name, o.order_no, o.order_amount 
FROM user_info u
LEFT JOIN order_info o ON u.id = o.user_id;

-- 3. 右连接（RIGHT JOIN，返回右表所有记录，左表匹配不到的为NULL，少用）
SELECT u.user_name, o.order_no, o.order_amount 
FROM user_info u
RIGHT JOIN order_info o ON u.id = o.user_id;
```

### 5.3 更新数据（UPDATE）

⚠️ **危险操作**：务必加 WHERE 条件，避免全表更新！

```sql
-- 1. 单条更新
UPDATE user_info SET age = 26, update_time = NOW() WHERE id = 1;

-- 2. 批量更新
UPDATE user_info SET age = age + 1 WHERE id IN (1, 2, 3);

-- 安全提示：加LIMIT限制更新行数，避免误操作影响过大
UPDATE user_info SET age = 26 WHERE id = 1 LIMIT 1;
```

### 5.4 删除数据（DELETE）

⚠️ **危险操作**：务必加 WHERE 条件，避免全表删除！

```sql
-- 1. 单条删除
DELETE FROM user_info WHERE id = 1;

-- 2. 批量删除
DELETE FROM user_info WHERE id IN (1, 2, 3);

-- 安全提示：加LIMIT限制删除行数
DELETE FROM user_info WHERE id = 1 LIMIT 1;
```

## 六、第五部分：索引操作命令

### 6.1 创建索引

```sql
-- 1. 普通索引（CREATE INDEX）
CREATE INDEX idx_user_name ON user_info (user_name);

-- 2. 普通索引（ALTER TABLE）
ALTER TABLE user_info ADD INDEX idx_age (age);

-- 3. 唯一索引
CREATE UNIQUE INDEX uk_phone ON user_info (phone);
ALTER TABLE user_info ADD UNIQUE INDEX uk_phone (phone);

-- 4. 联合索引（注意列顺序）
CREATE INDEX idx_name_age ON user_info (user_name, age);
ALTER TABLE user_info ADD INDEX idx_name_age (user_name, age);

-- 5. 前缀索引（长字符串字段用）
CREATE INDEX idx_name_prefix ON user_info (user_name(5));

-- 6. 全文索引（MySQL 5.6+ InnoDB支持）
CREATE FULLTEXT INDEX idx_fulltext_name ON user_info (user_name);
```

### 6.2 查看索引

```sql
-- 查看表的所有索引（最常用）
SHOW INDEX FROM user_info;
```

### 6.3 删除索引

```sql
-- 方式1
DROP INDEX idx_user_name ON user_info;
-- 方式2
ALTER TABLE user_info DROP INDEX idx_user_name;
```

## 七、第六部分：用户与权限操作命令

### 7.1 创建用户

```sql
-- 创建用户（指定主机，%表示任意主机）
CREATE USER 'dev_user'@'%' IDENTIFIED BY '123456';
CREATE USER 'dev_user'@'192.168.1.100' IDENTIFIED BY '123456'; -- 仅允许指定IP访问
```

### 7.2 授权

```sql
-- 1. 全局权限（所有库所有表，谨慎授权）
GRANT ALL PRIVILEGES ON *.* TO 'dev_user'@'%';

-- 2. 库级权限（指定库的所有表）
GRANT SELECT, INSERT, UPDATE, DELETE ON my_database.* TO 'dev_user'@'%';

-- 3. 表级权限（指定库的指定表）
GRANT SELECT, INSERT ON my_database.user_info TO 'dev_user'@'%';

-- 4. 刷新权限（授权后必须执行）
FLUSH PRIVILEGES;
```

### 7.3 查看权限

```sql
-- 查看当前用户权限
SHOW GRANTS;

-- 查看指定用户权限
SHOW GRANTS FOR 'dev_user'@'%';
```

### 7.4 撤销权限

```sql
-- 撤销库级权限
REVOKE SELECT, INSERT, UPDATE, DELETE ON my_database.* FROM 'dev_user'@'%';

-- 刷新权限
FLUSH PRIVILEGES;
```

### 7.5 删除用户

```sql
DROP USER 'dev_user'@'%';
```

## 八、第七部分：事务控制命令

### 8.1 基础事务操作

```sql
-- 1. 开启事务
START TRANSACTION;
-- 等价写法
BEGIN;

-- 2. 执行SQL操作
INSERT INTO user_info (user_name, phone) VALUES ('测试', '13900000000');
UPDATE order_info SET order_amount = 100 WHERE id = 1;

-- 3. 提交事务（成功后执行，数据永久生效）
COMMIT;

-- 4. 回滚事务（失败后执行，撤销所有操作）
ROLLBACK;
```

### 8.2 自动提交

```sql
-- 查看自动提交状态（1表示开启，0表示关闭）
SHOW VARIABLES LIKE 'autocommit';

-- 关闭自动提交（当前会话有效）
SET autocommit = 0;

-- 开启自动提交
SET autocommit = 1;
```

## 十、核心要点总结

1. **安全第一**：DELETE、UPDATE、DROP、TRUNCATE 等危险操作务必加 WHERE 条件 / IF EXISTS，操作前确认，最好加 LIMIT 限制；
2. **规范优先**：SQL 关键字大写，库表字段小写，加清晰注释，字符集统一 utf8mb4，存储引擎统一 InnoDB；
3. **查询优化**：SELECT 只查需要的字段，避免 SELECT *；LIKE 前缀匹配可命中索引，左模糊不行；JOIN 关联字段必须建索引；
4. **批量操作**：INSERT 批量插入性能远高于单条插入，推荐使用；
5. **权限最小化**：用户权限按需授权，不要随便给 ALL PRIVILEGES；
6. **事务控制**：重要操作（如转账、订单）必须用事务，保证数据一致性；
7. **排查工具**：SHOW PROCESSLIST 排查正在执行的慢 SQL，SHOW INDEX 查看索引，EXPLAIN 分析执行计划。
