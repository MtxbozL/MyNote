## 一、开篇：排序与分页的作用

排序（`ORDER BY`）和分页（`LIMIT`）是 DQL（数据查询语言）中对**查询结果进行二次处理**的核心操作：

- **排序**：调整查询结果的显示顺序，让数据更符合业务需求（如按时间倒序查看最新订单）；
- **分页**：从大量查询结果中截取指定范围的数据，避免一次性返回过多数据（如电商商品列表的分页展示）。

---

## 二、排序操作（ORDER BY）

### 2.1 基础定义

`ORDER BY` 用于对查询结果按**一个或多个字段**排序，默认按**升序（ASC）** 排列，也可指定**降序（DESC）**。

### 2.2 核心语法

```sql
-- 单字段排序
SELECT 字段 FROM 表名 ORDER BY 排序字段 [ASC/DESC];

-- 多字段排序（先按第一个字段排序，相同则按第二个字段排序）
SELECT 字段 FROM 表名 ORDER BY 排序字段1 [ASC/DESC], 排序字段2 [ASC/DESC], ...;
```

### 2.3 常见示例

基于 `user_info`（用户表）和 `order_info`（订单表）演示：

| 需求                              | 示例代码                                                      | 说明             |
| :------------------------------ | :-------------------------------------------------------- | :------------- |
| **按年龄升序排列用户**                   | `SELECT * FROM user_info ORDER BY age ASC;`               | `ASC` 可省略，默认升序 |
| **按用户 ID 降序排列用户**               | `SELECT * FROM user_info ORDER BY user_id DESC;`          | 从大到小排列         |
| **多字段排序：先按年龄升序，年龄相同再按用户 ID 降序** | `SELECT * FROM user_info ORDER BY age ASC, user_id DESC;` | 第一个字段优先级更高     |
| **按订单金额降序排列订单**                 | `SELECT * FROM order_info ORDER BY order_amount DESC;`    | 金额从高到低         |

### 2.4 关键注意事项

1. **NULL 值的处理**：
    
    - MySQL 中，`NULL` 值在**升序**时会排在**最前面**，在**降序**时会排在**最后面**；
    - 示例：`SELECT * FROM user_info ORDER BY email ASC;`（`email` 为 `NULL` 的用户会排在最前）。
    
2. **排序字段的选择**：
    
    - 建议使用**有索引的字段**排序，可大幅提高查询效率；
    - 避免对大文本字段（如 `TEXT`）排序，会消耗大量内存和 CPU。
    
3. **多字段排序的优先级**：
    
    - 严格按 `ORDER BY` 后字段的顺序排序，只有前一个字段值相同时，才会按后一个字段排序。

## 三、分页操作（LIMIT）

### 3.1 基础定义

`LIMIT` 用于从查询结果中**截取指定范围的数据**，是实现分页功能的核心，有两种常用写法：

1. 仅指定条数：取前 N 条数据；
2. 指定偏移量和条数：从第 M 条开始，取 N 条数据（分页用）。

### 3.2 核心语法

```sql
-- 写法1：取前 N 条数据
SELECT 字段 FROM 表名 LIMIT 条数;

-- 写法2：分页用（从偏移量位置开始，取指定条数）
SELECT 字段 FROM 表名 LIMIT 偏移量, 条数;
```

### 3.3 分页公式（重中之重！）

实现分页时，**偏移量**的计算公式为：

```bash
偏移量 = (当前页 - 1) × 每页条数
```

### 3.4 常见示例

| 需求                  | 示例代码                                                     | 说明                  |
| :------------------ | :------------------------------------------------------- | :------------------ |
| **取前 5 个用户**        | `SELECT * FROM user_info ORDER BY user_id LIMIT 5;`      | 仅指定条数，偏移量默认为 0      |
| **查询第 1 页，每页 10 条** | `SELECT * FROM user_info ORDER BY user_id LIMIT 0, 10;`  | 偏移量 = (1-1)×10 = 0  |
| **查询第 2 页，每页 10 条** | `SELECT * FROM user_info ORDER BY user_id LIMIT 10, 10;` | 偏移量 = (2-1)×10 = 10 |
| **查询第 3 页，每页 20 条** | `SELECT * FROM user_info ORDER BY user_id LIMIT 40, 20;` | 偏移量 = (3-1)×20 = 40 |

### 3.5 关键注意事项

1. **LIMIT 必须放在 ORDER BY 之后**：
    
    - 逻辑上要先排序，再从有序结果中分页，否则分页结果无意义；
    - 错误示例：`SELECT * FROM user_info LIMIT 10, 10 ORDER BY user_id;`（语法会报错）。
    
2. **偏移量过大的性能问题**：
    
    - 当偏移量很大时（如 `LIMIT 1000000, 10`），MySQL 需要扫描前 1000010 条数据，然后丢弃前 1000000 条，效率极低；
    - **解决方法**：
        
        - 方法 1：用 `WHERE` 子句先过滤（如 `SELECT * FROM user_info WHERE user_id > 1000000 ORDER BY user_id LIMIT 10;`）；
        - 方法 2：延迟关联（先查主键，再关联原表）。
        
    
3. **LIMIT 后的参数必须是正整数**：
    
    - 偏移量和条数都不能是负数或小数，否则会报错。

## 四、排序与分页的综合实操

基于 `order_info`（订单表），完成以下需求：

1. 查询所有订单，按订单金额降序排列，取前 5 条；
2. 查询所有订单，按下单时间降序排列，查询第 2 页，每页 8 条；
3. 查询金额大于 100 的订单，按金额升序、下单时间降序排列，查询第 1 页，每页 3 条。

### 参考答案

```sql
-- 1. 按金额降序取前5条
SELECT * FROM order_info ORDER BY order_amount DESC LIMIT 5;

-- 2. 按时间降序，第2页每页8条
SELECT * FROM order_info ORDER BY create_time DESC LIMIT 8, 8;

-- 3. 金额>100，多字段排序，第1页每页3条
SELECT * FROM order_info 
WHERE order_amount > 100 
ORDER BY order_amount ASC, create_time DESC 
LIMIT 0, 3;
```

## 五、核心要点总结

1. **排序（ORDER BY）**：
    
    - 默认升序（ASC），降序需指定（DESC）；
    - 多字段排序时，前一个字段优先级更高；
    - NULL 值升序排最前，降序排最后；
    - 建议用索引字段排序，提高效率。
    
2. **分页（LIMIT）**：
    
    - 分页公式：`偏移量 = (当前页 - 1) × 每页条数`；
    - 必须放在 ORDER BY 之后；
    - 偏移量过大时，用 WHERE 子句或延迟关联优化性能。
    
3. **综合使用顺序**：
    
    完整的查询语句顺序为：`SELECT` → `FROM` → `WHERE` → `GROUP BY` → `HAVING` → `ORDER BY` → `LIMIT`。
