
文件操作是 Python 与本地文件系统交互的核心能力，用于实现文件的读取、写入、修改、追加等操作，是数据持久化、配置文件读写、日志存储、文件处理的基础能力。Python 通过内置`open()`函数创建文件对象，实现所有文件操作，同时结合上下文管理器保证文件资源的安全释放。

### 6.5.1 open () 函数的完整语法

`open()`函数用于打开文件，创建文件对象，完整语法如下：

```python
open(
    file,
    mode='r',
    buffering=-1,
    encoding=None,
    errors=None,
    newline=None,
    closefd=True,
    opener=None
)
```

核心参数详解：

1. **file**：必填参数，文件的路径（相对路径或绝对路径），字符串类型；
2. **mode**：可选参数，文件打开模式，字符串类型，默认值为`'r'`，核心模式见 6.5.2 节；
3. **encoding**：可选参数，文件读写的编码格式，仅文本模式下有效，**强制推荐显式指定`encoding='utf-8'`**，避免系统默认编码导致的跨平台乱码问题（Windows 系统默认编码为 GBK，Linux/macOS 为 UTF-8）；
4. **errors**：可选参数，编码错误的处理方式，常用值：`'strict'`（默认，抛出异常）、`'ignore'`（忽略错误）、`'replace'`（用？替换错误字符）；
5. **newline**：可选参数，换行符的处理方式，用于适配不同系统的换行符差异（Windows 为`\r\n`，Linux/macOS 为`\n`）。

### 6.5.2 文件打开模式

文件打开模式分为**文本模式**与**二进制模式**两大类，通过基础模式 + 修饰符组合实现，核心模式的功能与规则如下表：

|模式|模式类型|核心功能|指针位置|文件不存在时|已有文件处理|
|---|---|---|---|---|---|
|`r`|文本模式|只读模式，默认模式|文件开头|抛出`FileNotFoundError`|保留原有内容，仅可读|
|`w`|文本模式|只写模式|文件开头|创建新文件|清空原有全部内容，重新写入|
|`a`|文本模式|追加模式|文件末尾|创建新文件|保留原有内容，在末尾追加写入|
|`r+`|文本模式|读写模式|文件开头|抛出`FileNotFoundError`|保留原有内容，可读写、覆盖写入|
|`w+`|文本模式|读写模式|文件开头|创建新文件|清空原有全部内容，可读写|
|`a+`|文本模式|读写模式|文件末尾|创建新文件|保留原有内容，追加写入，可读|
|`rb`|二进制模式|只读模式，对应`r`的二进制版本|文件开头|抛出`FileNotFoundError`|保留原有内容，仅可读|
|`wb`|二进制模式|只写模式，对应`w`的二进制版本|文件开头|创建新文件|清空原有全部内容，重新写入|
|`ab`|二进制模式|追加模式，对应`a`的二进制版本|文件末尾|创建新文件|保留原有内容，在末尾追加写入|
|`rb+`/`wb+`/`ab+`|二进制模式|对应文本模式的读写版本，二进制模式|对应规则|对应规则|对应规则|

核心注意事项：

- 二进制模式用于读取非文本文件（图片、视频、音频、exe、压缩包等），**禁止指定 encoding 参数**；
- 文本模式用于读取文本文件（.txt、.py、.json、.csv 等），必须显式指定`encoding='utf-8'`；
- `w`/`wb`模式会直接清空文件原有内容，无任何提示，使用时必须谨慎，避免误删文件数据；
- `r+`模式的写入为覆盖写入，从指针当前位置开始覆盖后续字节，而非插入写入。

### 6.5.3 文件对象的核心方法

通过`open()`函数创建的文件对象，提供了文件读写、指针操作、关闭的核心方法，分为文本模式通用方法与二进制模式通用方法。

#### 1. 读操作方法

|方法|功能描述|适用场景|
|---|---|---|
|`read(size=-1)`|读取文件内容，size 为读取的字符数（文本模式）/ 字节数（二进制模式），默认 - 1 读取全部内容|小文件一次性读取，禁止用于超大文件|
|`readline(size=-1)`|读取一行内容，size 为该行读取的最大字符 / 字节数，返回字符串|大文件逐行读取，流式处理|
|`readlines(hint=-1)`|读取所有行，返回每行内容组成的列表，hint 为读取的最大字节数|小文件按行读取，需对所有行做批量处理|

#### 2. 写操作方法

|方法|功能描述|注意事项|
|---|---|---|
|`write(text)`|将字符串 / 字节串写入文件，返回写入的字符数 / 字节数|文本模式必须传入字符串，二进制模式必须传入 bytes 类型|
|`writelines(lines)`|将字符串 / 字节串的可迭代对象写入文件，不会自动添加换行符|需手动为每行添加换行符`\n`|
|`flush()`|强制刷新文件缓冲区，将缓冲区中的数据立即写入磁盘|避免程序异常退出时，缓冲区数据未写入磁盘丢失|

#### 3. 文件指针操作方法

|方法|功能描述|注意事项|
|---|---|---|
|`tell()`|返回文件指针当前的位置，单位为字节|文本模式下，多字节编码字符的位置计算以字节为单位|
|`seek(offset, whence=0)`|移动文件指针到指定位置，offset 为偏移量，whence 为参考位置|whence 可选值：0 - 文件开头（默认）；1 - 当前位置；2 - 文件末尾。文本模式下，仅支持 whence=0 的非负偏移量，二进制模式无限制|

#### 4. 关闭方法

|方法|功能描述|核心规则|
|---|---|---|
|`close()`|关闭文件对象，释放系统资源，刷新缓冲区数据到磁盘|文件操作完成后必须关闭，否则会导致资源泄漏、文件句柄占用、数据丢失。多次调用无副作用|

### 6.5.4 上下文管理器 with 语句的标准用法

文件操作的最佳实践是使用`with`语句（上下文管理器），其底层会自动调用`close()`方法关闭文件，无论代码正常执行还是发生异常，都能保证文件资源被安全释放，彻底避免资源泄漏，是 Python 官方推荐的文件操作标准写法。

标准语法：

```python
# 文本文件读取，标准写法
with open("test.txt", "r", encoding="utf-8") as f:
    # with块内执行文件操作，文件对象f有效
    content = f.read()
    print(content)
# 离开with块后，文件已自动关闭，无需手动调用close()
```

进阶用法：同时打开多个文件

```python
# 同时打开两个文件，实现文件复制
with open("source.txt", "r", encoding="utf-8") as src_f, open("target.txt", "w", encoding="utf-8") as tgt_f:
    for line in src_f:
        tgt_f.write(line)
```

### 6.5.5 大文件的流式读取最佳实践

对于 GB 级的超大文件，禁止使用`read()`、`readlines()`一次性读取全部内容，否则会导致内存溢出，必须使用**逐行流式读取**，每次仅读取一行内容到内存，处理完成后释放，内存占用始终固定。

示例：超大日志文件流式处理

```python
# 流式读取10GB的日志文件，统计错误日志数量
error_count = 0
# 逐行读取，内存仅占用一行内容的大小
with open("large_log.log", "r", encoding="utf-8") as f:
    for line in f:
        if "ERROR" in line:
            error_count += 1
            # 处理错误日志逻辑

print(f"错误日志数量：{error_count}")
```

二进制大文件读取：对于二进制大文件，使用固定块大小循环读取，示例：

```python
# 流式读取大视频文件，实现复制
chunk_size = 4096  # 4KB块大小
with open("large_video.mp4", "rb") as src_f, open("copy_video.mp4", "wb") as tgt_f:
    while chunk := src_f.read(chunk_size):
        tgt_f.write(chunk)
```
