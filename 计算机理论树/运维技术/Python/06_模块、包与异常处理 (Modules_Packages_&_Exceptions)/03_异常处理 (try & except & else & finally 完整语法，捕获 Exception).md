
异常是 Python 程序运行时发生的错误事件，会中断程序的正常执行流程。Python 采用**EAFP（Easier to Ask for Forgiveness than Permission）** 编程风格，即优先执行代码，捕获处理异常，而非提前做大量条件判断，这是 Python 区别于其他语言的核心特性之一。异常处理机制的核心作用是：捕获并处理运行时异常，保证程序不会意外崩溃，提升程序的健壮性与容错能力。

### 6.3.1 Python 异常体系

Python 中所有异常都是类，均继承自顶级基类`BaseException`，形成完整的异常继承体系，核心层级结构如下：

```python
BaseException（顶级基类）
├── Exception（所有业务异常的基类，自定义异常必须继承该类）
│   ├── ValueError、TypeError、AttributeError、KeyError等内置异常
│   ├── OSError（系统IO异常）
│   ├── RuntimeError（运行时异常）
│   └── 自定义异常
├── SystemExit（解释器退出异常）
├── KeyboardInterrupt（用户中断异常，Ctrl+C触发）
└── GeneratorExit（生成器关闭异常）
```

核心规范：

- 业务代码的异常捕获，**必须捕获`Exception`及其子类**，禁止直接捕获`BaseException`，否则会拦截`SystemExit`、`KeyboardInterrupt`等系统级异常，导致程序无法正常退出、无法响应中断信号；
- 禁止使用空`except:`语句捕获所有异常，必须精准捕获预期的异常类型，避免隐藏未知的程序 bug。

### 6.3.2 try /except/else /finally 完整语法

Python 提供完整的异常处理语法结构，每个代码块的执行顺序与触发条件严格定义，完整语法如下：

```python
try:
    # 可能抛出异常的业务代码，必选块
    执行语句
except 异常类型1 as 异常别名:
    # 捕获异常类型1，执行该块，可选块，可多个
    异常处理语句
except (异常类型2, 异常类型3) as 异常别名:
    # 同时捕获多个异常类型，执行该块
    异常处理语句
except Exception as 异常别名:
    # 捕获所有业务异常，兜底处理，可选块
    异常兜底处理语句
else:
    # try块无任何异常，正常执行完毕后，执行该块，可选块
    无异常时的执行语句
finally:
    # 无论是否发生异常、是否触发return/break，都会执行该块，可选块
    资源清理、释放语句
```

#### 各代码块的执行规则详解

1. **try 块**：异常监控的核心块，仅执行可能抛出异常的业务代码，代码应尽量精简，避免无关代码；
2. **except 块**：异常捕获与处理块，支持单个异常、多个异常元组、异常子类匹配，遵循**从上到下的匹配顺序**，找到第一个匹配的异常类型即停止匹配，执行对应处理代码。因此，子类异常必须写在父类异常前面，否则父类异常会提前拦截子类异常，导致子类异常块永远无法执行。
    
    反例（错误顺序）：

    ```python
    try:
        {}["key"]
    except Exception as e:
        print("兜底异常")
    except KeyError as e:  # 永远不会执行，KeyError是Exception的子类，已被前面的块拦截
        print("键不存在")
    ```
    
3. **else 块**：仅当 try 块**无任何异常、正常执行完毕**时才会执行，若 try 块抛出异常，else 块完全不执行。核心适用场景：仅当业务代码正常执行时才需要执行的逻辑，如成功日志、数据提交等，避免将这些逻辑放在 try 块中，导致无关代码的异常被意外捕获。
4. **finally 块**：**无论是否发生异常，无论 try/except/else 块中是否有 return、break、continue 语句，finally 块一定会执行**，是资源释放、清理操作的唯一安全位置。核心适用场景：文件关闭、数据库连接释放、锁释放、网络连接关闭等必须执行的资源清理逻辑，避免资源泄漏。

【面试高频考点】finally 块的执行优先级：

```python
def test_finally():
    try:
        return 1
    finally:
        # 即使try块中有return，finally块仍会执行，且在return之前执行
        print("finally块执行")

print(test_finally())
# 输出：
# finally块执行
# 1
```

注意：若 finally 块中存在 return 语句，会覆盖 try/except 块中的 return 值，导致非预期结果，因此禁止在 finally 块中编写 return 语句。

### 6.3.3 异常处理最佳实践

1. **精准捕获异常**：仅捕获预期会发生的异常类型，禁止宽泛捕获`Exception`，更禁止空`except`，避免隐藏程序 bug；
2. **最小化 try 块代码**：try 块中仅包含可能抛出异常的代码，避免将大量无关代码放入 try 块，导致异常范围失控；
3. **异常信息完整记录**：捕获异常时，必须记录完整的异常信息（异常类型、异常信息、异常栈），禁止静默捕获异常不做任何日志记录，否则会导致问题无法排查；
4. **异常修复与重试**：异常处理的核心是修复异常场景，而非仅记录日志，可根据异常类型执行重试、降级、兜底等修复逻辑，保证程序继续运行；
5. **禁止用异常处理控制正常业务流程**：异常仅用于处理错误场景，禁止将异常作为正常的业务逻辑分支判断，避免性能损耗与代码可读性下降。
