
`if __name__ == '__main__':`是 Python 程序的标准入口写法，是区分模块「作为脚本直接运行」和「作为模块被导入」的核心机制，是 Python 工程化开发的强制规范。

### 6.2.1 **name** 变量的核心本质

`__name__`是 Python 模块内置的字符串变量，由解释器自动赋值，其值完全取决于模块的运行方式：

1. **当模块作为顶层脚本直接运行时**：`__name__`被解释器赋值为固定字符串`'__main__'`；
2. **当模块被其他模块 import 导入时**：`__name__`被赋值为模块的**全限定名**（如`my_package.core`）。

### 6.2.2 核心语法与执行逻辑

标准语法结构：

```python
# 模块内的函数、类、全局变量定义
def main():
    """程序主逻辑入口函数"""
    print("程序执行主逻辑")

# 程序入口判断
if __name__ == '__main__':
    # 仅当模块作为脚本直接运行时，才会执行该代码块
    main()
```

执行逻辑：

- 直接运行该模块：`python module.py`，`__name__ == '__main__'`为`True`，执行`main()`函数，启动程序主逻辑；
- 该模块被其他模块导入：`import module`，`__name__ == '__main__'`为`False`，该代码块不会执行，仅加载模块内的函数、类定义，不会执行业务逻辑。

### 6.2.3 工程意义与最佳实践

1. **实现模块的双重身份**：一个模块既可以作为被导入的复用模块，也可以作为独立的脚本运行，代码块中可编写模块的测试代码、独立运行逻辑，示例：

    ```python
    # utils.py 工具模块
    def add(a, b):
        return a + b
    
    # 模块的单元测试代码，仅在直接运行该模块时执行
    if __name__ == '__main__':
        assert add(1, 2) == 3
        assert add(-1, 1) == 0
        print("所有测试用例通过")
    ```
    
2. **避免导入时的意外执行**：禁止在模块顶层编写业务执行代码，所有可执行逻辑都应放在函数 / 类中，仅在`if __name__ == '__main__':`代码块中触发执行，避免模块被导入时意外执行业务逻辑；
3. **统一程序入口规范**：Python 项目的主入口文件必须使用该语法，明确程序的执行起点，符合 Python 社区的通用规范，提升代码可读性；
4. **多进程兼容**：Windows 系统下，Python 多进程`multiprocessing`模块要求程序入口必须使用该语法，否则会导致子进程无限递归创建，触发运行时错误。
