
Python 函数的参数系统极其灵活，同时存在多个高频面试坑点，参数传递的核心本质是**传对象引用**：实参将对象的内存地址传递给形参，形参与实参指向同一个内存对象。

参数定义与传递的**强制顺序规则**（PEP3102 规范），从左到右必须遵循：

`位置参数 → *args → 默认参数 → 仅限关键字参数 → **kwargs`

顺序错误会直接触发语法错误。

### 3.4.1 位置参数（Positional Arguments）

位置参数是最基础的参数类型，定义时按顺序排列，调用时必须按**顺序、数量**严格匹配，传入的实参会按位置依次赋值给对应的形参，示例：

```python
def func(a, b, c):
    return a + b + c

# 正确调用：3个位置实参，顺序匹配
func(1, 2, 3)
# 错误调用：数量不匹配，抛出TypeError
func(1, 2)
```

### 3.4.2 默认参数（Default Arguments）

默认参数是在函数定义时为形参指定默认值，调用时若未传入对应实参，则自动使用默认值，实现可选参数功能，语法结构：

```python
def 函数名(位置参数, 形参名=默认值):
    函数体
```

核心规则与【必考坑点】：

1. 带默认值的参数必须定义在无默认值的位置参数右侧，否则触发语法错误；
2. 调用时，默认参数可省略，也可通过关键字指定传入，覆盖默认值；
3. **致命坑点：禁止使用可变对象（列表、字典、集合等）作为默认参数的默认值**
    
    底层原理：默认参数的默认值，是在**函数定义阶段（def 语句执行时）计算并初始化一次**，而非每次函数调用时重新创建。多次调用函数时，若修改了可变默认参数对象，其值会在多次调用之间累积，导致非预期结果。
    
    反例（错误用法）：

    ```python
    def func(a, lst=[]):
        lst.append(a)
        return lst
    
    # 多次调用，默认列表对象被累积修改
    print(func(1))  # 输出：[1]
    print(func(2))  # 非预期输出：[1, 2]，预期应为[2]
    print(func(3))  # 非预期输出：[1, 2, 3]
    ```
    
    正确用法：使用不可变对象`None`作为默认值，在函数体内部初始化可变对象，每次调用都会创建新的可变对象，避免累积：

    ```python
    def func(a, lst=None):
        if lst is None:
            lst = []
        lst.append(a)
        return lst
    
    # 多次调用，结果符合预期
    print(func(1))  # 输出：[1]
    print(func(2))  # 输出：[2]
    print(func(3))  # 输出：[3]
    ```
    

### 3.4.3 动态参数（可变参数）

动态参数用于接收任意数量的实参，分为位置动态参数`*args`与关键字动态参数`**kwargs`，是装饰器、通用函数封装的核心基础。

1. *_位置动态参数 _args__
    
    - 语法：形参名前加单个`*`，约定俗成命名为`args`，可自定义名称；
    - 功能：接收调用时传入的**任意数量的位置实参**，将其打包为一个**元组**，赋值给 args 形参；
    - 示例：

        ```python
        def sum_all(*args):
            # args是元组，包含所有传入的位置实参
            return sum(args)
        
        print(sum_all(1, 2, 3))  # 输出：6
        print(sum_all(1, 2, 3, 4, 5))  # 输出：15
        ```
        
    - 序列解包传参：调用函数时，在列表、元组、集合等可迭代对象前加`*`，可将其元素解包为独立的位置实参传入，示例：

        ```python
        nums = [1, 2, 3, 4]
        print(sum_all(*nums))  # 等价于 sum_all(1,2,3,4)，输出：10
        ```
        
    
2. ** 关键字动态参数 **kwargs**
    
    - 语法：形参名前加两个`*`，约定俗成命名为`kwargs`，可自定义名称；
    - 功能：接收调用时传入的**任意数量的关键字实参**，将其打包为一个**字典**，赋值给 kwargs 形参；
    - 示例：

        ```python
        def print_info(**kwargs):
            # kwargs是字典，包含所有传入的关键字实参
            for k, v in kwargs.items():
                print(f"{k}: {v}")
        
        print_info(name="Python", version=3.12, type="backend")
        # 输出：
        # name: Python
        # version: 3.12
        # type: backend
        ```
        
    - 字典解包传参：调用函数时，在字典前加`**`，可将其键值对解包为独立的关键字实参传入，示例：

        ```python
        info_dict = {"name": "Python", "version": 3.12}
        print_info(**info_dict)  # 等价于 print_info(name="Python", version=3.12)
        ```
        
    

### 3.4.4 仅限关键字参数（Keyword-only Arguments）

仅限关键字参数是 Python3 新增的语法特性，指必须通过**关键字实参**传递、不能通过位置实参传递的参数，用于强制调用方明确参数含义，提升代码可读性与安全性，避免位置参数顺序错误导致的 bug。

两种定义方式：

1. 定义在`*args`之后，`*args`接收了所有位置实参，后续的参数只能通过关键字传递：

    ```python
    def func(a, *args, c, d):
        # c、d为仅限关键字参数，必须用关键字传递
        print(a, args, c, d)
    
    # 正确调用
    func(1, 2, 3, c=4, d=5)
    # 错误调用：用位置传递c、d，抛出TypeError
    func(1, 2, 3, 4, 5)
    ```
    
2. 使用单独的`*`作为分隔符，`*`右侧的所有参数均为仅限关键字参数，不接收多余的位置实参：

    ```python
    def func(a, b, *, c, d):
        # *仅作为分隔符，c、d为仅限关键字参数
        print(a, b, c, d)
    
    # 正确调用
    func(1, 2, c=3, d=4)
    # 错误调用：位置传递c、d，抛出TypeError
    func(1, 2, 3, 4)
    ```
    

核心规则：仅限关键字参数可设置默认值，设置后调用时可省略，覆盖默认值仍需通过关键字传递。
