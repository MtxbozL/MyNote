
作用域指变量的可访问范围，Python 采用**静态作用域（词法作用域）**，变量的作用域在代码编写阶段（定义时）就已确定，而非运行时决定。作用域的核心规则为**LEGB 规则**，是 Python 变量查找的核心准则，也是面试超高频考点。

### 3.6.1 作用域的层级划分

Python 的作用域按层级分为 4 类，按变量查找的优先级从高到低排序，即为 LEGB 顺序：

|层级|全称|中文名称|定义|
|---|---|---|---|
|L|Local|局部作用域|函数内部定义的变量，仅在函数内部可访问，函数执行结束后销毁|
|E|Enclosing|闭包作用域（嵌套作用域）|外层嵌套函数的作用域，仅在嵌套函数场景中存在，对应闭包的外层函数变量|
|G|Global|全局作用域|模块（.py 文件）顶层定义的变量，在整个模块文件内可访问，模块加载完成后存在|
|B|Built-in|内置作用域|Python 解释器内置的变量、函数、常量，如 print ()、len ()、True、False 等，在所有模块中均可访问|

### 3.6.2 LEGB 变量查找规则

当代码中访问一个变量时，Python 解释器会严格按照**L → E → G → B**的顺序依次查找：

1. 首先在当前局部作用域（L）查找，找到则直接使用，停止查找；
2. 若未找到，在嵌套的外层闭包作用域（E）查找，找到则使用，停止查找；
3. 若未找到，在全局作用域（G）查找，找到则使用，停止查找；
4. 若仍未找到，在内置作用域（B）查找，找到则使用；
5. 所有层级均未找到，抛出`NameError`异常。

示例：

```python
# G：全局作用域
g_var = "全局变量"
def outer():
    # E：闭包作用域
    e_var = "外层函数变量"
    def inner():
        # L：局部作用域
        l_var = "内层函数变量"
        print(l_var)  # L层找到，输出：内层函数变量
        print(e_var)  # L未找到，E层找到，输出：外层函数变量
        print(g_var)  # L、E未找到，G层找到，输出：全局变量
        print(len)    # L、E、G未找到，B层找到，输出内置函数len
    inner()

outer()
```

核心补充：Python 的**代码块（if/for/while/try）不会创建新的作用域**，代码块内定义的变量，在代码块执行结束后，仍可在所属的作用域内访问，区别于 C/C++/Java 的块级作用域。

### 3.6.3 变量的赋值与作用域绑定

Python 中，对变量的赋值操作，会将变量绑定到当前作用域，而非修改外层作用域的变量，这是核心坑点。

示例：

```python
g_num = 10

def func():
    # 赋值操作，将num绑定到局部作用域，而非修改全局变量
    g_num = 20
    print(g_num)  # 输出：20，访问的是局部变量

func()
print(g_num)  # 输出：10，全局变量未被修改
```

上述代码中，函数内的`g_num = 20`是赋值操作，Python 会将`g_num`识别为局部变量，而非全局变量，若在赋值前访问该变量，会抛出`UnboundLocalError`异常。

### 3.6.4 global 关键字

`global`关键字用于在**函数内部声明变量为全局作用域的变量**，强制 Python 在全局作用域绑定变量，而非创建局部变量，实现函数内部对全局变量的修改。

核心规则：

1. 对全局变量的**重新绑定（赋值）**，必须使用`global`声明；
2. 对全局可变对象（列表、字典、集合）的**原地修改**（如 append、update），无需使用`global`，因为并未修改变量的引用指向，仅修改了对象内部的内容；
3. 一个 global 语句可声明多个全局变量，用逗号分隔。

示例：

```python
g_num = 10
g_list = [1, 2, 3]

def func():
    global g_num  # 声明为全局变量
    g_num = 20    # 修改全局变量的值
    g_list.append(4)  # 原地修改全局列表，无需global

func()
print(g_num)  # 输出：20，全局变量已修改
print(g_list)  # 输出：[1, 2, 3, 4]，全局列表已修改
```

### 3.6.5 nonlocal 关键字

`nonlocal`关键字是 Python3 新增的，用于**嵌套函数场景**，声明变量为外层（Enclosing）闭包作用域的变量，强制 Python 在外层嵌套函数的作用域绑定变量，而非创建局部变量，实现内层函数对外层函数变量的修改。

核心规则：

1. 仅能在嵌套函数中使用，不能用于全局作用域；
2. 声明的变量必须在外层嵌套函数的作用域中已存在，不能用于创建新变量；
3. 仅能修改最近一层嵌套作用域的变量，不能跨层级直接修改全局变量。

示例：

```python
def outer():
    e_num = 10  # 外层函数变量，Enclosing作用域
    def inner():
        nonlocal e_num  # 声明为外层作用域的变量
        e_num = 20      # 修改外层函数的变量
        print(e_num)    # 输出：20
    inner()
    print(e_num)  # 输出：20，外层变量已被修改

outer()
```

### 3.6.6 global 与 nonlocal 的核心区别

|特性|global|nonlocal|
|---|---|---|
|作用目标|全局作用域的变量|外层嵌套函数（Enclosing）作用域的变量|
|适用场景|所有函数内部|仅嵌套函数内部|
|变量创建|可用于声明不存在的全局变量|必须对应外层已存在的变量，不能创建新变量|
|跨层级能力|直接绑定全局作用域，无视嵌套层级|仅绑定最近一层的嵌套作用域，不能直接绑定全局|
