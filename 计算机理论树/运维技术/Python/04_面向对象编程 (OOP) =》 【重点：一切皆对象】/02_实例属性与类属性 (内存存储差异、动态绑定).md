
Python 类的属性分为**实例属性**与**类属性**，二者在内存存储、作用范围、生命周期上存在本质差异，是面试高频考点。

### 4.2.1 核心定义与内存差异

|特性|实例属性|类属性|
|---|---|---|
|定义位置|在`__init__`构造方法中，通过`self.属性名`定义；或在实例化后，通过`实例.属性名`动态绑定|在类体的顶层，方法外部直接定义，与构造方法、实例方法平级|
|绑定对象|绑定在**实例对象**上，每个实例拥有独立的属性副本，互不影响|绑定在**类对象**上，所有该类的实例共享同一份属性，全局唯一|
|内存存储|存储在每个实例自身的`__dict__`属性字典中，每个实例的`__dict__`相互独立|存储在类对象的`__dict__`属性字典中，全局仅一份，所有实例共享引用|
|生命周期|随实例的创建而生成，随实例的销毁而回收|随类的定义（解释器执行 class 语句）而生成，随程序结束而回收，早于实例属性|
|访问方式|仅能通过`实例.属性名`访问，类无法直接访问实例属性|可通过`类.属性名`访问，也可通过`实例.属性名`访问|

### 4.2.2 实例属性详解

1. **核心特性**：实例属性是每个实例独有的数据，修改一个实例的实例属性，不会影响其他实例的同名属性，实现了实例之间的数据隔离；
2. **动态绑定特性**：Python 作为动态类型语言，支持在实例化后，为单个实例动态绑定新的属性，该属性仅对当前实例有效，其他实例无法访问；
    
    示例：

    ```python
    stu1 = Student("张三", 18, "2024001")
    stu2 = Student("李四", 19, "2024002")
    
    # 为stu1动态绑定独有的实例属性
    stu1.major = "计算机科学与技术"
    print(stu1.major)  # 输出：计算机科学与技术
    print(stu2.major)  # 抛出AttributeError，stu2无该属性
    ```
    
3. **属性查找优先级**：当通过`实例.属性名`访问属性时，Python 解释器会优先在实例自身的`__dict__`中查找该属性，找到则直接返回；若未找到，再去类对象的`__dict__`中查找类属性。

### 4.2.3 类属性详解

1. **核心特性**：类属性是所有实例共享的全局数据，修改类属性会影响所有该类的实例，适用于存储类的通用常量、全局配置、实例计数等场景；
    
    示例：

    ```python
    class Student:
        # 类属性：学校名称，所有学生实例共享
        school_name = "XX大学"
        # 类属性：实例计数，统计创建的学生总数
        student_count = 0
    
        def __init__(self, name, age, student_id):
            self.name = name
            self.age = age
            self.student_id = student_id
            # 每创建一个实例，类属性计数+1
            Student.student_count += 1
    
    # 实例化
    stu1 = Student("张三", 18, "2024001")
    stu2 = Student("李四", 19, "2024002")
    
    # 类属性访问
    print(Student.school_name)  # 输出：XX大学
    print(stu1.school_name)     # 输出：XX大学
    print(Student.student_count) # 输出：2
    ```
    
2. **【高频坑点】实例修改类属性的陷阱**
    
    核心规则：通过`实例.类属性名 = 新值`的操作，**不会修改类属性**，而是为该实例动态绑定了一个与类属性同名的实例属性，根据属性查找优先级，后续访问会优先使用实例属性，屏蔽类属性。
    
    反例与正确用法：

    ```python
    class Student:
        school_name = "XX大学"
    
    stu1 = Student()
    stu2 = Student()
    
    # 错误操作：实例赋值，仅创建实例属性，不修改类属性
    stu1.school_name = "YY大学"
    print(stu1.school_name)  # 输出：YY大学（实例属性）
    print(stu2.school_name)  # 输出：XX大学（类属性，未被修改）
    print(Student.school_name) # 输出：XX大学（类属性未变）
    
    # 正确操作：通过类名修改类属性，所有实例同步生效
    Student.school_name = "ZZ大学"
    print(stu2.school_name)  # 输出：ZZ大学
    print(stu1.school_name)  # 仍输出：YY大学（实例属性优先级更高）
    ```
    
3. **可变类型类属性的特殊情况**：若类属性是列表、字典等可变对象，通过`实例.类属性名.方法()`的原地修改操作，会直接修改类属性本身，因为并未修改属性的引用指向，仅修改了引用对象的内部内容。
    
    示例：

    ```python
    class Student:
        # 可变类型类属性
        course_list = []
    
    stu1 = Student()
    stu2 = Student()
    
    # 原地修改可变类属性，所有实例同步生效
    stu1.course_list.append("Python编程")
    print(stu2.course_list)  # 输出：['Python编程']
    print(Student.course_list) # 输出：['Python编程']
    ```
    

### 4.2.4 动态绑定机制

Python 作为动态语言，支持在运行时为类或实例动态绑定属性与方法，无需修改类的原始定义：

1. **为类动态绑定属性 / 方法**：绑定后，所有已创建和后续创建的实例，均可访问该属性 / 方法；
2. **为实例动态绑定属性 / 方法**：仅对当前实例有效，其他实例无法访问，绑定方法时需通过`types.MethodType`将函数绑定为实例方法，保证`self`参数的自动传递。
