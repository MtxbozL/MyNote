
封装、继承、多态是面向对象编程的三大核心特性，是实现代码复用、解耦、扩展的核心基础。

### 4.3.1 封装

封装是面向对象的基础特性，核心思想是：**将对象的属性与实现细节隐藏在内部，对外仅暴露有限的、安全的访问接口，禁止外部直接访问和修改内部私有数据**，从而保证数据的安全性、完整性，降低代码耦合度。

Python 的封装并非语法强制的访问控制，而是基于**命名规范与名称改写机制**实现的约定式封装，分为三个访问级别：

#### 1. 公有成员（Public）

- 定义：无任何下划线前缀的属性与方法；
- 访问权限：类内部、子类内部、外部代码均可自由访问；
- 适用场景：对外暴露的公开接口、通用方法。

#### 2. 受保护成员（Protected）

- 定义：以**单下划线`_`** 为前缀的属性与方法，如`_name`、`_get_info()`；
- 访问权限：类内部、子类内部可正常访问，外部代码也可访问，但 PEP8 规范明确约定：单下划线前缀的成员为受保护成员，仅类内部与子类可访问，外部禁止直接访问；
- 核心本质：单下划线仅为命名约定，无任何语法层面的访问限制，解释器不会做任何拦截，仅用于提示开发者该成员为内部实现细节，不对外公开。

#### 3. 私有成员（Private）

- 定义：以**双下划线`__`** 为前缀的属性与方法，如`__id`、`__check_data()`；
- 核心机制：Python 解释器会对双下划线前缀的成员执行**名称改写（Name Mangling）**，底层将`__var`改写为`_类名__var`的格式，从而实现外部无法通过原名称直接访问；
- 访问权限：仅类内部可通过原名称访问，子类无法继承、无法直接访问，外部代码无法通过原名称访问；
- 核心本质：Python 的私有是**伪私有**，并非真正的访问权限控制，仅通过名称改写实现访问限制，外部代码若知道改写后的名称，仍可强制访问，示例：

    ```python
    class Student:
        def __init__(self, name, student_id):
            self.name = name
            # 私有属性：双下划线前缀
            self.__student_id = student_id
        
        def __check_id(self):
            # 私有方法
            return len(self.__student_id) == 7
    
    stu = Student("张三", "2024001")
    
    # 外部直接访问私有属性，抛出AttributeError
    # print(stu.__student_id)
    
    # 通过改写后的名称可强制访问
    print(stu._Student__student_id)  # 输出：2024001
    print(stu._Student__check_id())  # 输出：True
    ```
    
- 适用场景：存储核心敏感数据、内部实现细节方法，禁止外部直接修改与访问，保证数据安全。

#### 4. property 装饰器

`property`装饰器是 Python 实现封装的核心工具，可将类的方法转换为属性访问，实现对私有属性的**受控访问**，包括只读属性、属性值校验、修改 / 删除逻辑拦截，替代 Java/C# 风格的 Getter/Setter 方法，更符合 Pythonic 的编码风格。

核心语法与用法：

```python
class Student:
    def __init__(self, name, age):
        self.name = name
        # 私有属性，外部禁止直接访问
        self.__age = age
    
    # Getter：将方法转为属性访问，实现只读访问
    @property
    def age(self):
        """获取年龄属性，对外暴露只读接口"""
        return self.__age
    
    # Setter：实现属性修改的拦截与校验
    @age.setter
    def age(self, new_age):
        """修改年龄时，执行合法性校验"""
        if not isinstance(new_age, int):
            raise TypeError("年龄必须为整数")
        if not 0 < new_age < 120:
            raise ValueError("年龄必须在0-120之间")
        self.__age = new_age
    
    # Deleter：实现属性删除的拦截
    @age.deleter
    def age(self):
        raise PermissionError("年龄属性不允许删除")

# 外部使用
stu = Student("张三", 18)

# 像访问普通属性一样访问age，无需调用方法
print(stu.age)  # 输出：18

# 修改属性，自动触发setter的校验逻辑
stu.age = 20
print(stu.age)  # 输出：20

# 非法赋值，触发异常
# stu.age = 200  # 抛出ValueError
# del stu.age     # 抛出PermissionError
```

核心规则：

1. `@property`装饰的方法为 Getter 方法，方法名即为对外暴露的属性名，无额外参数，返回值为属性值；
2. `@属性名.setter`装饰的方法为 Setter 方法，用于拦截属性修改，接收一个参数为新的属性值，无返回值；
3. `@属性名.deleter`装饰的方法为 Deleter 方法，用于拦截属性删除操作；
4. 仅定义`@property`、未定义 setter 的属性为**只读属性**，外部无法修改，是封装只读数据的最佳实践。

### 4.3.2 继承

继承是实现代码复用的核心特性，核心思想是：**一个子类（派生类）可以继承一个或多个父类（基类 / 超类）的属性与方法，无需重复编写相同代码，同时可重写父类方法、扩展自身独有的功能**，实现代码的高效复用与层级化设计。

Python 支持**单继承**与**多继承**，所有类的顶级父类均为`object`类（Python3 中，所有类默认继承 object，无需显式声明）。

#### 1. 单继承

单继承指子类仅继承一个父类，是最基础、最常用的继承模式，语法结构如下：

```python
class 父类名:
    # 父类属性与方法定义
    pass

class 子类名(父类名):
    # 子类重写父类方法、扩展自身方法
    pass
```

核心规则：

- 子类继承父类所有的公有成员、受保护成员，私有成员因名称改写机制，子类无法直接继承与访问；
- 子类可重写父类的方法：子类定义与父类同名的方法，调用时优先执行子类的方法，实现对父类方法的覆盖；
- 子类重写`__init__`构造方法时，必须显式调用父类的构造方法，保证父类的初始化逻辑正常执行，调用方式有两种：
    
    1. 父类名直接调用：`父类名.__init__(self, 父类参数)`，需手动传入`self`参数，适用于单继承场景；
    2. super () 函数调用：`super().__init__(父类参数)`，Python3 推荐用法，无需手动传入`self`，自动处理继承关系。
    

示例：

```python
# 父类：Person
class Person:
    def __init__(self, name, age):
        self.name = name
        self.age = age
    
    def introduce(self):
        return f"我是{self.name}，今年{self.age}岁"

# 子类：Student，继承Person
class Student(Person):
    def __init__(self, name, age, student_id, major):
        # 调用父类的构造方法，初始化父类属性
        super().__init__(name, age)
        # 子类独有的属性
        self.student_id = student_id
        self.major = major
    
    # 重写父类的introduce方法
    def introduce(self):
        # 调用父类的方法，复用父类逻辑
        parent_intro = super().introduce()
        # 扩展子类逻辑
        return f"{parent_intro}，学号{self.student_id}，就读{self.major}专业"

# 子类实例化
stu = Student("张三", 18, "2024001", "计算机科学与技术")
print(stu.introduce())
# 输出：我是张三，今年18岁，学号2024001，就读计算机科学与技术专业
```

#### 2. 多继承

多继承指一个子类同时继承多个父类，可同时复用多个父类的代码，语法结构如下：

```python
class 父类1:
    pass

class 父类2:
    pass

class 子类名(父类1, 父类2):
    # 继承多个父类，按定义顺序排列
    pass
```

多继承带来了代码复用的灵活性，同时也带来了**命名冲突、菱形继承（钻石继承）** 等问题，Python 通过**MRO 方法解析顺序**与 C3 线性化算法，彻底解决了多继承的二义性问题。

#### 3. MRO 方法解析顺序与 C3 算法

MRO（Method Resolution Order，方法解析顺序）是 Python 中多继承场景下，属性与方法的查找顺序规则，Python3 采用**C3 线性化算法**生成 MRO 列表，保证方法查找的唯一性、确定性。

核心规则：

1. 每个类都有一个对应的 MRO 列表，可通过`类名.__mro__`属性、`类名.mro()`方法查看，列表的顺序即为方法查找的优先级顺序；
2. C3 算法的核心原则：
    
    - 子类优先于父类：查找时先找子类自身，再找父类；
    - 同层级父类按继承定义的从左到右顺序查找；
    - 任何类在 MRO 列表中仅出现一次；
    - 菱形继承场景下，公共父类仅在最后出现一次，避免重复查找。
    
3. 方法查找逻辑：当调用实例方法时，Python 会严格按照 MRO 列表的顺序，从左到右依次查找对应的方法，找到第一个匹配的方法立即执行，停止查找。

示例：菱形继承的 MRO 解析

```python
# 顶级父类
class A:
    def func(self):
        print("A.func")

# 子类B、C继承A
class B(A):
    def func(self):
        print("B.func")

class C(A):
    def func(self):
        print("C.func")

# 子类D多继承B、C
class D(B, C):
    pass

# 查看D的MRO列表
print(D.mro())
# 输出：[<class '__main__.D'>, <class '__main__.B'>, <class '__main__.C'>, <class '__main__.A'>, <class 'object'>]

# 调用func方法，按MRO顺序，优先执行B的func
d = D()
d.func()  # 输出：B.func
```

#### 4. super () 函数的本质

`super()`的核心本质**不是调用父类的方法**，而是获取当前类 MRO 列表中，当前类的下一个类的实例，从而调用对应类的方法。这一特性完美解决了多继承、菱形继承场景下的重复调用问题，保证每个父类的方法仅执行一次。

核心规则：

- Python3 中，`super()`无需传入任何参数，自动绑定当前类与`self`实例；
- `super()`的调用顺序完全遵循 MRO 列表，而非继承的父子关系；
- 多继承场景下，所有子类的`super()`调用会形成链式调用，严格按照 MRO 顺序执行。

### 4.3.3 多态

多态是面向对象的核心扩展特性，核心思想是：**同一种操作，作用于不同的对象，会产生不同的执行结果**。多态的核心价值是实现程序的「开闭原则」—— 对扩展开放，对修改关闭，无需修改原有代码，仅通过扩展新的子类，即可实现功能扩展。

Python 的多态是**基于鸭子类型（Duck Typing）** 实现的，与 Java/C# 等静态语言的基于继承、接口的多态完全不同，是 Python 动态类型的核心优势。

#### 1. 鸭子类型核心定义

鸭子类型的核心思想来自于 James Whitcomb Riley 的经典描述：**「当看到一只鸟走起来像鸭子、游泳起来像鸭子、叫起来也像鸭子，那么这只鸟就可以被称为鸭子。」**

在 Python 中，鸭子类型的具体表现为：**解释器不关心对象的具体类型，不关心对象是否继承了某个父类，只关心对象是否实现了对应的方法 / 属性。只要不同的对象实现了同名的方法，就可以统一调用该方法，实现多态。**

#### 2. 多态实现示例

```python
# 无需定义统一的父类/接口，不同类实现同名方法即可
class Cat:
    def speak(self):
        return "喵喵喵"

class Dog:
    def speak(self):
        return "汪汪汪"

class Duck:
    def speak(self):
        return "嘎嘎嘎"

# 统一的调用接口，不关心传入对象的类型，只关心是否有speak方法
def animal_speak(animal):
    print(animal.speak())

# 多态表现：同一种操作，不同对象产生不同结果
cat = Cat()
dog = Dog()
duck = Duck()

animal_speak(cat)  # 输出：喵喵喵
animal_speak(dog)  # 输出：汪汪汪
animal_speak(duck) # 输出：嘎嘎嘎
```

核心优势：

- 无需强制继承关系，代码耦合度极低，灵活性极强；
- 无需定义接口，仅需约定方法名与参数规范，即可实现多态；
- 完全符合开闭原则：新增动物类，仅需实现`speak`方法，无需修改`animal_speak`接口的任何代码。

#### 3. 与静态语言多态的核心区别

|特性|Python 鸭子类型多态|静态语言（Java）继承 / 接口多态|
|---|---|---|
|类型约束|无强制类型约束，仅关注方法是否实现|强制类型约束，必须继承父类或实现接口|
|实现方式|动态绑定，运行时确定方法调用|静态绑定，编译时确定方法调用|
|耦合度|极低，无继承依赖|较高，强依赖继承 / 接口体系|
|灵活性|极高，扩展无需修改原有体系|相对较低，扩展需遵循继承体系|
