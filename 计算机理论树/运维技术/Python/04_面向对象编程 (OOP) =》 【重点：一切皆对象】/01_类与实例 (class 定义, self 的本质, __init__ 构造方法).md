## 本章概述

Python 是**纯面向对象编程语言**，其核心设计哲学为「一切皆对象」—— 从基础的数值、字符串，到函数、类、模块，本质都是对象。面向对象编程（Object-Oriented Programming, OOP）是 Python 工程化开发、框架设计、大型项目构建的核心编程范式，通过封装、继承、多态三大特性，实现代码的高内聚、低耦合、可复用与可扩展。

本章将系统讲解 Python 面向对象的核心机制，从类与实例的本质出发，深入属性管理、三大特性、方法类型与魔术方法，所有内容严格遵循 Python 官方语法规范与 PEP8 代码规范，深度覆盖面试超高频考点，为后续高阶特性、Web 框架、底层源码阅读构建完整的面向对象知识体系。

### 4.1.1 核心定义

- **类（Class）**：是创建对象的**模板与蓝图**，是对具有相同属性、相同行为的一类事物的抽象封装。类定义了该类所有实例共享的属性（数据）与方法（行为），本质是一个自定义的类型对象（`type`的实例）。
- **实例（Instance）**：是类的**具体实体**，是类实例化后生成的独立对象，拥有类定义的属性与方法，同时可绑定自身独有的实例属性。每个实例拥有独立的内存地址，是类的具象化产物。

### 4.1.2 类的定义语法

Python 通过`class`关键字定义类，基础语法结构如下：

```python
class 类名:
    """类的文档字符串（docstring）"""
    # 类属性定义
    类属性名 = 属性值
    
    # 构造方法
    def __init__(self, 实例属性参数):
        # 实例属性初始化
        self.实例属性名 = 参数值
    
    # 实例方法定义
    def 方法名(self, 参数列表):
        # 方法体
        执行语句
```

核心规范：

1. 类名遵循**大驼峰命名法（CamelCase）**，每个单词首字母大写，禁止使用下划线分隔，与函数、变量的命名规范明确区分；
2. 文档字符串用于描述类的功能、用途，可通过`类名.__doc__`或`help(类名)`访问，是工业界工程化开发的强制规范；
3. 类体中主要包含属性定义与方法定义，禁止编写无关的顶层执行代码。

### 4.1.3 **init** 构造方法

`__init__`是 Python 类的**初始化方法**，属于魔术方法的一种，在类实例化创建对象时，由 Python 解释器自动调用，核心作用是为新创建的实例对象绑定实例属性，完成初始化操作。

核心规则：

1. `__init__`方法的第一个参数必须是`self`，后续参数由开发者自定义，用于接收实例化时传入的初始化参数；
2. `__init__`方法**禁止有非 None 的返回值**，否则会触发`TypeError`异常，其仅负责实例初始化，对象的创建由`__new__`方法完成；
3. 若类未定义`__init__`方法，Python 会自动提供一个空的默认构造方法，实例化时无需传入参数；
4. 子类继承父类时，若重写`__init__`方法，必须显式调用父类的构造方法，保证父类的初始化逻辑正常执行。

示例：

```python
class Student:
    """学生类，用于封装学生的基础信息与行为"""
    def __init__(self, name: str, age: int, student_id: str):
        # 为实例绑定属性，完成初始化
        self.name = name
        self.age = age
        self.student_id = student_id
    
    def introduce(self):
        """学生自我介绍方法"""
        return f"我是{self.name}，学号{self.student_id}，今年{self.age}岁"

# 类的实例化，创建具体的实例对象
stu1 = Student("张三", 18, "2024001")
stu2 = Student("李四", 19, "2024002")

# 调用实例方法
print(stu1.introduce())
# 输出：我是张三，学号2024001，今年18岁
```

### 4.1.4 self 的本质

`self`是 Python 面向对象中最核心的基础概念，其本质是**当前正在被操作的实例对象本身的内存引用**，是实例方法的强制第一个参数。

核心底层规则：

1. `self`是约定俗成的参数名，并非 Python 关键字，开发者可修改为其他名称（如`this`），但 PEP8 规范强制要求使用`self`，禁止自定义名称；
2. 调用实例方法时，Python 解释器会**自动将调用该方法的实例对象作为第一个参数**，传递给`self`，开发者无需手动传参；
    
    示例：`stu1.introduce()` 本质等价于 `Student.introduce(stu1)`，`stu1`被自动赋值给方法的`self`参数；
3. `self`的核心作用是绑定实例属性与实例方法：通过`self.属性名`绑定的属性，是每个实例独有的实例属性；通过`self.方法名()`可在类的内部调用其他实例方法；
4. `self`的生命周期与实例对象一致，仅在实例方法内部有效，实例方法执行结束后，`self`的引用会被回收。
