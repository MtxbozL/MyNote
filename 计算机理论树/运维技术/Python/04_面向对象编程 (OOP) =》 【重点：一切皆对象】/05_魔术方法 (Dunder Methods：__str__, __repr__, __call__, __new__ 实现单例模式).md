
魔术方法（也叫双下划线方法、特殊方法）是 Python 预先定义的、以双下划线`__`开头和结尾的方法，是 Python 面向对象的核心协议机制。这些方法会在特定的操作场景下，由 Python 解释器自动触发调用，无需手动调用，通过实现魔术方法，可让自定义类的对象具备与 Python 内置对象一致的行为，实现运算符重载、对象生命周期管理、类型转换等高级功能。

本节重点讲解目录指定的核心魔术方法，覆盖面试高频考点。

### 4.5.1 **str** 与 **repr**

二者均用于定义对象的字符串表示形式，触发场景与设计目标存在本质差异，是自定义类必须实现的基础魔术方法。

|特性|**str**|**repr**|
|---|---|---|
|设计目标|给**最终用户**看的，友好、易读的字符串描述，侧重展示对象的核心信息|给**开发者**看的，官方、准确、无歧义的字符串表示，侧重还原对象的创建方式|
|触发场景|1. `print(对象)` 调用；2. `str(对象)` 转换；3. 格式化字符串`f"{对象}"`|1. 交互式解释器中直接输入对象回车；2. `repr(对象)` 转换；3. 容器打印内部元素时调用；4. **str__未实现时，fallback 到__repr**|
|最佳实践|可选实现，用于友好展示|强制实现，保证所有场景下都有明确的字符串表示|

示例：

```python
class Student:
    def __init__(self, name, age, student_id):
        self.name = name
        self.age = age
        self.student_id = student_id
    
    def __str__(self):
        """用户友好的字符串表示"""
        return f"学生[name={self.name}, age={self.age}, student_id={self.student_id}]"
    
    def __repr__(self):
        """开发者友好的官方表示，可直接执行还原对象"""
        return f"Student(name='{self.name}', age={self.age}, student_id='{self.student_id}')"

stu = Student("张三", 18, "2024001")

print(stu)          # 触发__str__，输出：学生[name=张三, age=18, student_id=2024001]
print(str(stu))     # 触发__str__
print(repr(stu))    # 触发__repr__，输出：Student(name='张三', age=18, student_id='2024001')
```

### 4.5.2 **call**

`__call__`方法用于让**类的实例对象可以像函数一样被调用**，实现了该方法的对象，称为「可调用对象（Callable）」。

- 触发场景：当实例对象通过`实例()`的方式被调用时，自动触发`__call__`方法的执行；
- 核心作用：实现实例的函数式调用、状态保持的函数、类装饰器等高级功能；
- 补充：Python 中所有的函数、类本身，本质都是实现了`__call__`方法的可调用对象。

示例：

```python
class Adder:
    def __init__(self, base_num):
        # 实例保持基础数值状态
        self.base_num = base_num
    
    def __call__(self, *args):
        """实例被调用时，将所有传入的数值与基础值累加"""
        return self.base_num + sum(args)

# 创建实例，保持base_num=10的状态
adder = Adder(10)

# 实例像函数一样被调用，触发__call__
print(adder(5))          # 输出：15
print(adder(1, 2, 3))    # 输出：16
print(callable(adder))   # 输出：True，验证为可调用对象
```

### 4.5.3 **new** 与单例模式实现

`__new__`是 Python 类的**实例创建方法**，是静态方法（无需显式声明），负责创建并返回类的实例对象，是实例生命周期的第一个执行的方法。

#### 1. **new** 与 **init** 的核心区别

| 特性   | **new**                 | **init**                    |
| ---- | ----------------------- | --------------------------- |
| 核心作用 | 创建实例对象，分配内存空间           | 初始化实例对象，为实例绑定属性             |
| 执行顺序 | 先执行，实例创建的入口             | 后执行，接收__new__返回的实例作为 self   |
| 参数   | 第一个参数为`cls`，代表当前类       | 第一个参数为`self`，代表__new__创建的实例 |
| 返回值  | 必须返回一个类的实例对象，通常是当前类的实例  | 禁止返回非 None 值                |
| 重写场景 | 控制实例的创建过程，如单例模式、不可变类型定制 | 绝大多数场景下，仅需重写__init__完成初始化   |

实例创建的完整流程：

1. 执行`类名(参数)`实例化时，Python 解释器自动调用`__new__(cls, 参数)`；
2. `__new__`方法创建并返回当前类的实例对象；
3. 解释器将返回的实例作为`self`参数，传递给`__init__`方法，执行初始化逻辑。

#### 2. 基于 **new** 实现单例模式

单例模式是设计模式中最常用的创建型模式，核心目标是：**保证一个类在整个程序的生命周期中，仅有一个实例对象，每次实例化都返回同一个实例**，适用于全局配置、数据库连接池、日志管理器等场景。

基于`__new__`的单例模式核心实现：

```python
class Singleton:
    # 类属性：存储唯一的实例对象
    _instance = None

    def __new__(cls, *args, **kwargs):
        # 若实例未创建，则创建并存储
        if not cls._instance:
            # 调用父类object的__new__方法创建实例
            cls._instance = super().__new__(cls)
        # 始终返回唯一的实例
        return cls._instance

    def __init__(self, name=None):
        # 初始化逻辑，仅首次创建时执行赋值，避免重复初始化
        if name is not None:
            self.name = name

# 测试单例模式
s1 = Singleton("全局配置")
s2 = Singleton()

# 两个变量指向同一个内存对象，id完全一致
print(id(s1) == id(s2))  # 输出：True
print(s1.name)            # 输出：全局配置
print(s2.name)            # 输出：全局配置
```
