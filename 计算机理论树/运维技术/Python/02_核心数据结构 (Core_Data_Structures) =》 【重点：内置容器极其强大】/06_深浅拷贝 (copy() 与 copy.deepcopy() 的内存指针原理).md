
深浅拷贝是 Python 内存管理的核心知识点，也是面试超高频考点，其核心区别在于**对嵌套可变对象的复制深度不同**，本质是内存指针的复制范围差异。

### 2.6.1 Python 变量与内存模型基础

Python 中，变量本质是对象内存地址的引用（指针），而非存储值本身。例如`a = [1, 2, [3, 4]]`，变量`a`存储的是外层列表对象的内存地址，外层列表的元素存储的是：不可变对象`1`、`2`的地址，以及内层嵌套列表`[3,4]`的内存地址。

所有拷贝操作的核心差异，就在于是否复制嵌套的可变子对象的内存，还是仅复制其指针引用。

### 2.6.2 直接赋值（引用传递）

直接赋值`b = a`，本质是让变量`b`与`a`指向**同一个内存对象**，未创建任何新对象，`id(a) == id(b)`为`True`。

- 特性：修改任何一个变量的内容，另一个都会同步变化，因为二者操作的是同一个对象；
- 示例：

    ```python
    a = [1, 2, [3, 4]]
    b = a
    print(id(a) == id(b))  # True
    a.append(5)
    print(b)  # [1, 2, [3, 4], 5]
    a[2].append(6)
    print(b)  # [1, 2, [3, 4, 6], 5]
    ```

### 2.6.3 浅拷贝（Shallow Copy）

#### 核心定义

浅拷贝会创建一个新的外层容器对象，新对象的元素是原对象元素的**引用指针副本**，即仅复制最外层容器，内部嵌套的子对象仍共享同一块内存，未被复制。

#### 实现方式

1. 序列切片：`b = a[:]`；
2. 容器内置`copy()`方法：`b = a.copy()`；
3. `copy`模块的`copy()`函数：`import copy; b = copy.copy(a)`。

#### 核心特性

- 外层容器的内存地址不同，是全新的对象；
- 不可变元素（int、str、tuple 等）：因不可修改，修改时会创建新对象，原对象与拷贝对象互不影响；
- 可变子对象（list、dict、set 等）：仅复制指针，指向同一个子对象，修改其中一个的嵌套子对象，另一个会同步变化；
- 示例：

    ```python
    import copy
    a = [1, 2, [3, 4]]
    b = copy.copy(a)
    
    print(id(a) == id(b))  # False，外层是新对象
    print(id(a[2]) == id(b[2]))  # True，内层子对象共享引用
    
    # 修改外层不可变元素，互不影响
    a[0] = 100
    print(a)  # [100, 2, [3, 4]]
    print(b)  # [1, 2, [3, 4]]
    
    # 修改内层可变子对象，同步变化
    a[2].append(5)
    print(a)  # [100, 2, [3, 4, 5]]
    print(b)  # [1, 2, [3, 4, 5]]
    ```
    

### 2.6.4 深拷贝（Deep Copy）

#### 核心定义

深拷贝会创建一个完全独立的新对象，**递归复制原对象中所有层级的可变子对象**，新对象与原对象完全无内存共享，修改任何一个对象的任何层级内容，都不会影响另一个。

#### 实现方式

唯一标准实现：`copy`模块的`deepcopy()`函数，`import copy; b = copy.deepcopy(a)`。

#### 核心特性

- 外层容器与所有嵌套的可变子对象，均为全新创建的对象，内存地址完全不同；
- 递归遍历所有层级，直到不可变对象为止，实现完全的内存隔离；
- 对不可变对象做优化：不可变对象不会重复创建，仅复制引用，节省内存开销；
- 自动处理循环引用：可处理对象之间的循环引用场景，不会出现无限递归；
- 示例：

    ```python
    import copy
    a = [1, 2, [3, 4]]
    b = copy.deepcopy(a)
    
    print(id(a) == id(b))  # False
    print(id(a[2]) == id(b[2]))  # False，内层子对象也是全新创建
    
    # 修改内层可变子对象，完全互不影响
    a[2].append(5)
    print(a)  # [1, 2, [3, 4, 5]]
    print(b)  # [1, 2, [3, 4]]
    ```
    

### 2.6.5 核心注意事项与性能对比

1. **不可变对象的拷贝优化**：对于全不可变元素的对象（如纯数字元组、字符串、数字），浅拷贝与深拷贝均不会创建新对象，仅返回原对象的引用，因为不可变对象无法修改，无需内存隔离；
2. **性能差异**：浅拷贝仅复制外层容器，时间复杂度 O (1)（固定长度容器）；深拷贝需递归遍历所有层级，时间复杂度 O (n)，n 为所有层级的元素总数，内存开销与执行时间远高于浅拷贝；
3. **使用选型**：
    
    - 仅需复制外层容器，内部无嵌套可变对象，或无需隔离嵌套子对象，使用浅拷贝；
    - 需要完全隔离所有层级的可变对象，避免修改互相影响，使用深拷贝。
