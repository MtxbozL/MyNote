### 2.3.1 核心定义与底层特性

字典是 Python 中唯一的内置**映射型容器**，以键值对（key-value）为存储单元，Python3.7 + 官方规定字典**保留插入顺序**，键必须唯一且可哈希，底层为 C 实现的哈希表（散列表，`PyDictObject`结构体），是 Python 中最高效的随机访问容器，平均查询、插入、删除时间复杂度均为 O (1)。

核心底层特性：

1. **可哈希键要求**：字典的键必须是**可哈希对象**，需满足两个核心条件：
    
    - 实现`__hash__()`方法，生命周期内哈希值固定不变；
    - 实现`__eq__()`方法，可判断相等性，且两个相等的对象必须拥有相同的哈希值；
        
        因此，不可变对象（int、str、全不可变元素的 tuple、float 等）可作为键，可变对象（list、dict、set 等）不可哈希，无法作为键；值无类型限制，支持任意 Python 对象。
    
2. **有序性实现**：Python3.6 及之前版本字典为无序存储，Python3.7 + 重构了底层实现，采用「哈希索引数组 + 键值对有序数组」的双数组结构，既保留了哈希表 O (1) 的访问效率，又严格保留键值对的插入顺序。
3. **哈希冲突与扩容机制**：
    
    - 哈希冲突解决：Python 采用**开放寻址法**解决哈希冲突，当键的哈希槽位被占用时，按照固定探测序列寻找下一个空槽位，避免了链地址法的内存碎片化问题；
    - 负载因子与扩容：字典的负载因子（已用槽位 / 总容量）阈值为 2/3，当负载因子超过阈值时，自动触发扩容，新容量为原容量的 2 倍，扩容时会对所有键值对执行重哈希，重新分配槽位，保证查询效率。
    
4. **视图对象特性**：字典的`keys()`、`values()`、`items()`方法返回动态视图对象，实时反映字典的修改，不会复制数据，内存开销极低。

### 2.3.2 核心操作与时间复杂度

#### 增 / 改操作

|操作方法|功能描述|平均时间复杂度|注意事项|
|---|---|---|---|
|`dict[key] = value`|键存在则覆盖值，不存在则新增键值对|O(1)|最基础的增改操作，键不可哈希时抛出`TypeError`|
|`update(mapping, **kwargs)`|批量新增 / 修改键值对，原地修改字典|O (k)，k 为新增键值对数量|支持传入字典、可迭代键值对、关键字参数，重复键会覆盖原有值|
|`setdefault(key, default=None)`|键存在则返回对应值，不存在则新增`key:default`并返回 default|O(1)|避免键不存在的`KeyError`，实现「不存在则插入」的原子操作|

#### 查询操作

|操作方法|功能描述|平均时间复杂度|注意事项|
|---|---|---|---|
|`dict[key]`|返回键对应的值|O(1)|键不存在时抛出`KeyError`|
|`get(key, default=None)`|键存在返回对应值，不存在返回 default|O(1)|工业界推荐的安全查询方式，不会抛出异常|
|`keys()`|返回所有键的动态视图对象`dict_keys`|O(1)|视图对象实时同步字典修改，支持迭代、成员运算，不支持索引|
|`values()`|返回所有值的动态视图对象`dict_values`|O(1)|特性与`keys()`一致，成员运算时间复杂度为 O (n)|
|`items()`|返回所有键值对元组的动态视图对象`dict_items`|O(1)|遍历字典的首选方式，`for k, v in dict.items()`|
|`in` 成员运算|`key in dict`，判断键是否存在|O(1)|仅判断键，判断值是否存在需用`value in dict.values()`，O(n)|

#### 删除操作

|操作方法|功能描述|平均时间复杂度|注意事项|
|---|---|---|---|
|`pop(key, default)`|删除指定键，返回对应值|O(1)|键不存在时，指定 default 则返回 default，否则抛出`KeyError`|
|`popitem()`|删除并返回最后插入的键值对（LIFO）|O(1)|Python3.7 + 固定删除末尾元素，空字典抛出`KeyError`|
|`clear()`|清空字典所有键值对|O(1)|释放哈希表内存，保留字典对象|
|`del` 语句|`del dict[key]`，删除指定键值对|O(1)|键不存在时抛出`KeyError`|

### 2.3.3 哈希表核心工作原理

字典的高效性完全依赖哈希表的实现，核心执行流程如下：

1. **哈希计算**：插入 / 查询键值对时，首先对键执行`hash(key)`函数，生成 64 位（64 位系统）整型哈希值；
2. **槽位定位**：将哈希值对哈希表的容量取模，得到对应的槽位（bucket）索引；
3. **冲突处理**：
    
    - 插入时：若槽位为空，直接存储键值对；若槽位已被占用，且键相等，则覆盖值；若键不相等，则触发开放寻址，按探测序列寻找下一个空槽位；
    - 查询时：若槽位键匹配，返回对应值；若槽位键不匹配，按探测序列继续查找，直到找到匹配键或空槽位（键不存在）；
    
4. **扩容重哈希**：当负载因子超过 2/3 时，申请新的 2 倍容量内存，遍历所有键值对，重新计算槽位索引，写入新的哈希表，释放旧内存。

### 2.3.4 适用场景

字典是 Python 后端开发、数据处理的核心容器，适用于：键值映射关系存储、高频随机查询、数据去重、配置管理、JSON 数据解析与处理、需要通过唯一标识快速定位数据的场景。
