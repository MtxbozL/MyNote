### 2.2.1 核心定义与底层特性

元组是**不可变、有序、支持元素重复的线性容器**，底层为 C 实现的固定长度数组（`PyTupleObject`结构体），与列表的核心区别是**不可变性**：元组创建后，其内部存储的元素指针无法修改、新增、删除，任何修改元组的操作都会生成全新的元组对象。

核心底层特性与优势：

1. **不可变性带来的哈希安全**：仅当元组的所有元素均为不可变对象时，元组具备可哈希性，可作为字典的键、集合的元素，这是列表不具备的核心能力；
2. **性能与内存优化**：
    
    - 内存占用更小：64 位系统中，空元组仅占用 40 字节，空列表占用 56 字节，元组无需存储动态扩容所需的容量信息，内存结构更紧凑；
    - 创建速度更快：Python 对长度 0-20 的小元组实现了缓存池，减少内存分配开销，创建速度显著优于列表；
    - 解释器优化：不可变对象可被解释器安全缓存，减少运行时的内存检查开销；
    
3. **线程安全**：不可变对象无法被多线程并发修改，无竞态条件，可在多线程环境中安全共享；
4. **无副作用特性**：不可变对象作为函数参数时，不会被函数意外修改，避免可变对象带来的隐性副作用。

### 2.2.2 核心创建与操作

#### 创建规则

- 空元组：`()`；
- 单元素元组：必须添加末尾逗号，`(1,)`，否则括号会被识别为运算符，`(1)`本质为整型`1`，这是元组最核心的语法坑点；
- 多元素元组：`(1, 2, 3)`，括号可省略，`1,2,3`同样为合法元组；
- 类型转换：`tuple(iterable)`，从可迭代对象创建元组，如`tuple([1,2,3])`。

#### 核心操作

元组不可变，无增删改方法，仅支持查询类与通用序列操作：

- 索引访问：`tuple[index]`，O (1) 时间复杂度，支持负索引，索引越界抛出`IndexError`；
- 切片操作：`tuple[start:stop:step]`，生成新的元组对象，规则与列表完全一致；
- `index(obj, start, end)`：查找元素第一个匹配项的索引，O (n)，元素不存在抛出`ValueError`；
- `count(obj)`：统计元素出现次数，O (n)；
- `in` 成员运算：`obj in tuple`，判断元素是否存在，O (n)；
- 拼接`+`、重复`*`：均生成全新的元组对象，不修改原元组。

### 2.2.3 元组解包（Unpacking）

元组解包是 Python 的核心语法特性，本质是将可迭代对象的元素依次赋值给对应变量，元组是解包操作的首选载体，也是 Python 实现多返回值、平行赋值的底层基础。

核心用法：

1. **基础解包**：变量数量与元组元素数量严格一致

    ```python
    a, b, c = (1, 2, 3)
    # 等价于 a=1, b=2, c=3
    ```
    
2. **平行赋值**：无临时变量的变量值交换，底层为元组解包

    ```python
    a, b = 10, 20
    a, b = b, a
    # 执行后 a=20, b=10，先创建元组(b,a)，再解包赋值给a,b
    ```
    
3. ***号通配符解包**：接收剩余元素，生成列表，单个解包表达式中仅能出现一次`*`

    ```python
    a, *rest, b = (1, 2, 3, 4, 5)
    # a=1, rest=[2,3,4], b=5
    *head, tail = (1, 2, 3)
    # head=[1,2], tail=3
    first, *_, last = (1, 2, 3, 4, 5)
    # 下划线_为占位符，忽略中间元素，first=1, last=5
    ```
    
4. **嵌套解包**：支持多层嵌套结构的层级匹配

    ```python
    a, (b, c), d = (1, (2, 3), 4)
    # a=1, b=2, c=3, d=4
    ```
    

### 2.2.4 元组与列表的核心差异对比

|特性|元组（Tuple）|列表（List）|
|---|---|---|
|可变性|不可变，创建后无法修改元素指针|可变，支持原地增删改|
|可哈希性|全不可变元素的元组可哈希|不可哈希，无法作为字典键|
|内存占用|更小，结构紧凑|更大，需预留扩容空间|
|创建速度|更快，有缓存池优化|相对较慢|
|线程安全|不可变，天然线程安全|可变，多线程操作需加锁|
|适用场景|固定常量、字典键、函数多返回值、数据保护|动态数据、频繁增删、灵活修改的场景|

### 2.2.5 适用场景

元组适用于：存储固定不变的常量集合、作为字典的键、函数多返回值（Python 函数多返回值本质为元组）、解包赋值、需要保证数据不被意外修改的场景、作为不可变数据结构在多线程环境中共享。
