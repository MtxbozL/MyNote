## 本章导读

本章为 Docker 技术体系的理论奠基章节，旨在建立 Docker 技术的完整认知框架，明确其核心解决的工程问题、虚拟化技术定位、架构设计范式、核心实体定义与行业标准化规范。本章内容为后续实操操作、工程化实践与底层原理剖析提供理论支撑，需完整掌握所有知识点，无遗漏理解核心概念。

## 01 什么是 Docker

Docker 是基于 Linux 内核操作系统级虚拟化技术构建的开源容器引擎，核心解决**应用跨环境部署的一致性问题**，即软件工程中经典的「在我的机器上能跑」的环境异构痛点。

传统应用交付流程中，开发、测试、生产环境的操作系统版本、依赖库版本、系统配置、网络环境的差异，会导致应用出现「本地运行正常，部署后异常」的环境漂移问题。Docker 通过将应用及其运行所需的代码、运行时、依赖库、环境变量、配置文件等所有资源，打包为标准化的容器交付单元，彻底屏蔽底层环境差异，实现**一次构建，到处运行（Build once, Run anywhere）** 的工程化目标。

从技术本质来看，Docker 容器是被严格隔离与资源限制的进程集合，而非完整的操作系统实例，其所有隔离与资源控制能力均基于 Linux 内核原生机制实现，是操作系统级虚拟化技术的工程化落地产品。
