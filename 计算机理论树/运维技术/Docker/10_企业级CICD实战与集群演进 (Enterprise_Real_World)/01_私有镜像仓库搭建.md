## 本章导读

本章是 Docker 技术体系从单主机实操到企业级生产落地的收官章节，核心目标是构建 Docker 企业级应用的完整技术栈，覆盖生产环境必需的镜像分发、安全合规、可观测性、DevOps 工程化四大核心能力，同时明确容器技术向云原生集群编排的演进路径。

本章内容完全承接前序章节的底层原理与实操规范，严格遵循**OCI 容器开放标准**与云原生计算基金会（CNCF）技术规范，兼顾学术严谨性与企业级生产实践，无遗漏覆盖目录内所有知识点，同时为 Docker 向云原生进阶提供完整的理论与实践指引。

## 01 私有镜像仓库搭建

私有镜像仓库是企业容器化体系的核心基础设施，是容器镜像全生命周期管理的核心载体，遵循**OCI Distribution Specification（OCI 分发规范）**，实现容器镜像的集中存储、版本管控、安全分发与访问控制，彻底解决公共镜像仓库的供应链安全风险、带宽限制、访问权限缺失、合规性无法保障等核心问题。

Docker 镜像的拉取、推送全流程，本质是 Docker Daemon 与镜像仓库通过 OCI 分发 API 的标准化交互，与前序章节讲解的 OCI 镜像规范、镜像分层原理完全兼容。

### 1.1 官方 Registry（轻量级，基础无界面）

Docker Registry 是 Docker 官方开源的轻量级镜像仓库实现，是 OCI 分发规范的原生参考实现，无图形化管理界面，仅提供核心的镜像存储与分发 API，适用于小型测试环境、离线场景与简单镜像分发需求。

#### 1.1.1 核心特性与局限性

|核心特性|核心局限性|
|---|---|
|完全兼容 OCI 分发规范，与 Docker Daemon 原生适配|无 RBAC 细粒度访问控制，仅支持基础的全局认证|
|轻量级部署，资源占用极低，基于 Docker 容器一键启动|无图形化管理界面，仅能通过 API 与 CLI 操作|
|支持镜像分层存储、增量拉取 / 推送，完全兼容 Docker 原生命令|无镜像安全扫描、镜像签名、漏洞检测等安全能力|
|支持本地存储与 S3 等对象存储后端|无多实例主从同步、跨区域复制能力，无法适配大规模集群|
|开源免费，无商业授权限制|无审计日志、生命周期管理、垃圾回收等企业级能力|

#### 1.1.2 标准化部署与配置

基于 Docker Compose 实现一键部署（承接上一章编排规范），配置基础认证与持久化存储：

```yaml
# docker-compose.yml
version: '3.8'
services:
  registry:
    image: registry:2.8.3
    container_name: docker-registry
    restart: unless-stopped
    ports:
      - "5000:5000"
    volumes:
      # 镜像数据持久化
      - registry_data:/var/lib/registry
      # 认证配置
      - ./auth:/auth
      # TLS证书配置
      - ./certs:/certs
    environment:
      # 基础认证配置
      REGISTRY_AUTH: htpasswd
      REGISTRY_AUTH_HTPASSWD_REALM: Registry Realm
      REGISTRY_AUTH_HTPASSWD_PATH: /auth/htpasswd
      # TLS加密配置
      REGISTRY_HTTP_TLS_CERTIFICATE: /certs/domain.crt
      REGISTRY_HTTP_TLS_KEY: /certs/domain.key
      # 垃圾回收配置
      REGISTRY_STORAGE_DELETE_ENABLED: "true"
volumes:
  registry_data:
```

- 前置配置：通过`htpasswd`工具生成认证账号密码，配置受信任的 TLS 证书，生产环境严禁使用 HTTP 协议与无认证配置；
- 核心操作：镜像推送前需添加仓库地址标签`docker tag nginx:1.27.0 registry.example.com:5000/nginx:1.27.0`，通过`docker push/pull`实现镜像的上传与拉取。

### 1.2 Harbor（企业级镜像仓库）

Harbor 是 VMware 开源、现已捐赠给 CNCF 的**企业级镜像仓库项目**，是 CNCF 毕业项目，在官方 Registry 的基础上，扩展了完整的企业级能力，是国内企业生产环境私有镜像仓库的事实标准。

#### 1.2.1 核心企业级能力

|能力模块|核心功能与生产价值|
|---|---|
|RBAC 角色权限控制|基于项目的细粒度权限管理，支持访客、开发、维护、管理员等多角色，实现不同团队、不同项目的镜像权限隔离，符合最小权限原则|
|企业级身份认证|原生支持 LDAP/AD、OIDC、CAS 等企业统一身份认证体系，无缝对接企业现有账号体系|
|镜像安全与供应链安全|内置 Trivy 漏洞扫描引擎，支持镜像构建时、推送前的自动漏洞扫描，支持镜像签名与验证（Cosign）、SBOM 物料清单生成，实现供应链安全全链路管控|
|镜像生命周期管理|支持镜像保留策略、自动垃圾回收、标签不可变配置，避免历史镜像占用存储，防止镜像标签覆盖导致的版本漂移|
|多实例同步与容灾|支持主从同步、跨区域镜像复制、双向同步，实现多数据中心镜像分发与容灾备份，适配大规模跨地域集群|
|审计与合规|完整的操作审计日志，记录所有镜像推送、拉取、删除、扫描操作，满足等保 2.0、PCI-DSS 等合规要求|
|扩展能力|支持 Webhook 事件通知、OCI 制品管理（Helm Chart、Operator 等）、镜像代理缓存、REST API 二次开发|

#### 1.2.2 部署与生产规范

Harbor 基于 Docker Compose 实现一键部署，官方提供标准化的安装包与配置脚本，生产环境部署需遵循以下核心规范：

1. **高可用部署**：生产环境必须采用多节点集群部署，后端存储使用分布式对象存储（MinIO、Ceph、阿里云 OSS 等），避免单点故障；
2. **安全加固**：必须启用 TLS 加密传输，对接企业统一身份认证，配置细粒度 RBAC 权限，启用镜像漏洞扫描门禁，禁止未通过扫描的镜像被拉取；
3. **镜像同步规范**：建立公共镜像代理缓存机制，所有基础镜像必须经过安全扫描、合规审核后同步到私有仓库，严禁生产环境直接拉取 Docker Hub 等公共仓库的镜像，防范供应链攻击；
4. **生命周期管理**：制定镜像保留策略，自动清理过期的测试镜像、临时镜像，启用标签不可变规则，生产环境镜像标签禁止覆盖，保证版本可追溯。

### 1.3 生产级镜像仓库最佳实践

1. **镜像命名与版本管理规范**：建立企业统一的镜像命名规范，格式为`[仓库地址]/[项目名]/[服务名]:[版本号]`，版本号采用`语义化版本+Git Commit哈希`，严禁使用`latest`标签部署生产环境；
2. **分层镜像治理**：建立企业统一的基础镜像体系，分为操作系统基础镜像、语言运行时基础镜像、业务通用基础镜像三层，统一维护、定期更新与安全扫描，避免业务团队各自维护基础镜像导致的安全漏洞与版本混乱；
3. **合规门禁**：将镜像漏洞扫描、签名验证嵌入 CI/CD 流水线，未通过安全门禁的镜像禁止推送到生产仓库，实现安全左移；
4. **容灾备份**：定期备份镜像仓库元数据与镜像数据，建立跨区域镜像同步机制，保障极端场景下的业务恢复能力。
