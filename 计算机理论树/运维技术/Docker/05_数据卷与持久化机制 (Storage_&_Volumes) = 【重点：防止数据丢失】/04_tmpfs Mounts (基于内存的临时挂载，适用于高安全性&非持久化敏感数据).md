
tmpfs Mounts 是基于 Linux 内核原生 tmpfs 文件系统的临时存储方案，所有数据完全存储在宿主机的内存中，不会写入宿主机磁盘，也不会写入容器的 Overlay2 可写层，生命周期与容器完全绑定，是高安全、高性能的临时数据存储方案。

### 4.1 核心语法与用法

Docker 提供`--tmpfs`简写语法与`--mount`完整语法，生产环境优先使用`--mount`语法，支持精细化的参数配置。

#### 4.1.1 简写语法 `--tmpfs`

- 标准格式：`--tmpfs [容器内目标路径]:[权限选项]`
- 典型示例：

    ```bash
    docker run -d --tmpfs /tmp:rw,noexec,nosuid,size=100m nginx:1.27.0
    ```

#### 4.1.2 完整语法 `--mount`

- 标准格式：`type=tmpfs,destination=[容器内目标路径],[可选参数]`
- 典型示例：

    ```bash
    docker run -d --mount type=tmpfs,target=/tmp,tmpfs-size=100m,tmpfs-mode=1777,noexec,nosuid nginx:1.27.0
    ```

#### 核心配置参数

|参数|核心作用|生产规范|
|---|---|---|
|`tmpfs-size`|设置 tmpfs 挂载的最大内存上限，单位为字节，默认值为宿主机总内存的 50%|**生产环境必须显式设置**，避免容器占用过多宿主机内存，引发 OOM 故障|
|`tmpfs-mode`|设置 tmpfs 文件系统的权限模式，默认值为 1777|按需配置最小权限，避免权限过度开放|
|`noexec`|禁止在 tmpfs 中执行二进制程序|生产环境必须开启，防范恶意代码执行攻击|
|`nosuid`|禁止 setuid/setgid 权限位生效|生产环境必须开启，防范提权攻击|
|`nodev`|禁止创建设备文件|生产环境必须开启，防范设备文件滥用|

### 4.2 底层实现原理

tmpfs Mounts 基于 Linux 内核原生的**tmpfs 文件系统**实现，该文件系统完全驻留在宿主机的内存页缓存中，无磁盘 IO 操作：

1. **内存级存储**：所有数据存储在内存中，读写延迟极低，性能接近原生内存访问速度，无任何磁盘 IO 开销；
2. **生命周期绑定**：tmpfs 挂载的生命周期与容器完全绑定，容器停止或销毁时，内核自动释放对应的内存页，所有数据被永久清除，无法恢复；
3. **隔离性保障**：tmpfs 挂载在容器的 Mount Namespace 中，仅当前容器可访问，其他容器与宿主机进程无法访问，数据隔离性极强；
4. **资源管控**：通过 Cgroups 机制与`tmpfs-size`参数双重限制内存使用，避免内存资源滥用。

### 4.3 核心特点

1. **极致性能**：完全基于内存读写，无磁盘 IO 开销，延迟极低，吞吐量极高，是 Docker 所有存储方案中性能最高的；
2. **高安全性**：数据仅在容器运行时存在于内存中，容器销毁后数据完全清除，不会在磁盘上留下任何痕迹，彻底避免敏感数据泄露；
3. **无磁盘占用**：不会写入宿主机磁盘，也不会写入容器可写层，避免临时文件占用磁盘空间，减少磁盘碎片化；
4. **非持久化**：容器停止、重启、销毁时，数据完全丢失，无法恢复，仅适用于临时数据存储；
5. **内存资源占用**：占用宿主机的物理内存，大量使用会导致宿主机内存资源紧张，引发 OOM 故障。

### 4.4 适用场景与生产规范

#### 合法适用场景

1. **高敏感数据的临时存储**：如密钥、证书、访问令牌、会话信息、加密临时数据等，容器销毁后数据完全清除，避免敏感数据落盘泄露；
2. **高并发低延迟的临时计算**：如实时数据聚合、排序、缓存、中间计算结果等，无需持久化，对 IO 性能要求极高的场景；
3. **只读容器的临时写入需求**：生产环境将容器根文件系统设置为只读模式，仅开放 tmpfs 挂载点用于临时写入，大幅提升容器安全防护能力；
4. **高频临时文件写入场景**：如日志临时缓存、上传临时文件、锁文件等，避免写入容器可写层，减少磁盘 IO 开销，容器销毁后自动清理。

#### 生产环境严格规范

1. **严禁用于持久化数据存储**：tmpfs 数据在容器停止后完全丢失，无法恢复，仅可用于临时数据存储；
2. **强制设置内存上限**：必须显式配置`tmpfs-size`参数，限制最大内存使用量，避免容器占用过多宿主机内存，引发系统级 OOM 故障；
3. **安全加固配置**：必须开启`noexec、nosuid、nodev`三个安全选项，防范恶意代码执行与提权攻击；
4. **内存资源管控**：合理规划 tmpfs 的总内存占用，避免大量容器同时使用 tmpfs，导致宿主机内存资源耗尽；
5. **数据备份豁免**：tmpfs 中的临时数据无需备份，核心业务数据严禁写入 tmpfs。

|底层实现|Linux bind mount 机制|Linux bind mount 机制 + Docker 存储驱动|Linux tmpfs 内存文件系统|
|性能|极高，接近原生磁盘|高，接近原生磁盘|极致，接近原生内存|
|可移植性|极差，强依赖宿主机目录结构|极强，无宿主机环境依赖|极强，无宿主机环境依赖|
