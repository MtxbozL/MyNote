# 3.4 与 3.5 进阶探究：UnionFS 底层机制与 commit 容器变身记

在前面的学习中，“分层（Layer）”这个词被频繁提及。它是 Docker 能够做到轻量化、极速启动的核心密码。本节我们将掀开引擎盖，一窥这项被称为 **联合文件系统 (UnionFS)** 的底层黑科技，并通过 `docker commit` 命令实战验证这一理论。

### 1. 深度剖析：UnionFS 联合文件系统

**联合文件系统 (UnionFS)** 并非 Docker 独创，而是 Linux 内核的一项技术特性（目前 Docker 默认主流使用的存储驱动是其变种 `Overlay2`）。

它是一种**分层、轻量级并且高性能的文件系统**，其最大的特点就是：**支持将多个不同物理位置的目录，联合挂载（Union Mount）到同一个虚拟的挂载点下，对外呈现出一个统一的、合并后的文件系统视图。**

为了便于理解，我们用一个**“透明玻璃板叠放”**的模型来比喻 Docker 镜像的分层逻辑：

1.  **基础镜像层（最底层玻璃）**：比如 Ubuntu 镜像，这块玻璃上画着基础的 Linux 命令和文件系统（如 `/bin`, `/etc`）。
2.  **增量堆叠层（中间层玻璃）**：在这块玻璃之上，你再叠放一块玻璃，上面画着 Nginx 环境。透过这两块玻璃往下看，你既能看到 Nginx，也能看到 Ubuntu 的系统文件。
3.  **隐藏与遮挡机制**：如果上层玻璃（比如修改了 `/etc/hosts`）和下层玻璃有相同的文件，**上层会遮挡住下层**。对外展示的始终是最顶层的内容。但这仅仅是视觉上的遮盖，下层的原始文件依然完好无损地存在着。

**核心精髓：只读（Read-Only）与 可写层（Read-Write）**
*   **镜像 (Image)**：由多层**绝对只读**的“玻璃板”叠合而成。它们一旦被构建，就永远不会被改变。
*   **容器 (Container)**：当容器启动时，Docker 会在镜像的最顶层，额外放上一块**崭新的、完全空白的“可写玻璃板”（即可写层，Container Layer）**。

你在容器里执行的所有操作（例如 `touch 了一个新文件`，或者 `用 vim 修改了配置`），都仅仅是画在这块最顶层的“可写玻璃板”上。底层的镜像层没有发生一丝一毫的改变。
