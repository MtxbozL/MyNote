
本节讲解本地镜像的信息查看、状态识别与磁盘空间清理，核心解决镜像磁盘占用管控、无用镜像清理的生产级需求，重点掌握虚悬镜像的识别与处理规范。

### 2.1 镜像查看命令

Docker 提供结构化的镜像查看命令，新版规范推荐使用`docker image <子命令>`的管理式命令，旧版简写命令仍兼容可用。

#### 2.1.1 基础查看命令

`docker images` 与 `docker image ls` 完全等价，用于列出本地存储的所有镜像，是最常用的镜像查看命令。

- **核心语法**

    ```bash
    docker image ls [OPTIONS] [REPOSITORY[:TAG]]
    ```
    
- **核心参数**
    
    - `--all/-a`：显示所有镜像，包括默认隐藏的中间镜像层；
    - `--digests`：显示镜像的 SHA256 哈希摘要，用于完整性校验；
    - `--filter/-f`：按条件过滤镜像，常用过滤规则：`dangling=true`（仅显示虚悬镜像）、`before=IMAGE`（显示指定镜像之前创建的镜像）、`after=IMAGE`（显示指定镜像之后创建的镜像）、`reference=PATTERN`（按仓库名 / 标签模式匹配）；
    - `--format`：按指定模板格式化输出，适用于自动化脚本解析，示例：`docker image ls --format "{{.Repository}}:{{.Tag}} {{.ID}}"`。
    

#### 2.1.2 输出字段核心解读

|输出字段|核心含义与技术说明|
|---|---|
|REPOSITORY|镜像的仓库名称，对应镜像的标准化命名规范|
|TAG|镜像的标签，用于版本标识，一个镜像可对应多个标签|
|IMAGE ID|镜像的唯一标识，为镜像顶层分层的 64 位 SHA256 哈希值，默认显示前 12 位；内容完全一致的镜像，无论标签如何，IMAGE ID 完全相同|
|CREATED|镜像的创建时间，对应镜像构建完成的时间戳|
|SIZE|镜像的虚拟大小，为镜像所有分层的解压后总大小；由于分层共享机制，多个镜像共享的分层仅占用一次磁盘空间，因此所有镜像的 SIZE 总和不等于实际磁盘占用空间|

#### 2.1.3 镜像详细信息查看

`docker image inspect` 用于查看镜像的完整元数据，包括镜像分层、环境变量、启动命令、端口声明、卷声明、构建历史等全量信息，是镜像排障、合规性校验的核心命令。

- **核心语法**

    ```bash
    docker image inspect IMAGE_NAME[:TAG|@DIGEST]
    ```
    
- 输出为 JSON 格式，可通过`--format`参数提取指定字段，示例：提取镜像的启动命令`docker image inspect nginx:1.27.0 --format "{{.Config.Cmd}}"`。

### 2.2 虚悬镜像（Dangling Image）

#### 2.2.1 定义与产生原因

虚悬镜像是指仓库名与标签均为`<none>:<none>`的镜像，其核心特征是已无任何标签引用，无法被直接使用，仅占用磁盘空间。

产生的核心原因：当构建或拉取一个新镜像时，使用了与本地已有镜像完全相同的仓库名与标签，原有镜像的标签会被剥离并转移至新镜像，原有镜像失去标签引用，成为虚悬镜像。

#### 2.2.2 识别与清理

- 识别命令：`docker image ls --filter dangling=true`
- 单独清理：可通过`docker image rm`命令直接删除，也可通过`docker image prune`命令批量清理。

### 2.3 镜像删除命令

`docker rmi` 与 `docker image rm` 完全等价，用于删除本地镜像，底层基于镜像分层的引用计数机制实现。

- **核心语法**

    ```bash
    docker image rm [OPTIONS] IMAGE_NAME[:TAG|@DIGEST] [IMAGE...]
    ```
    
- **核心参数**
    
    - `--force/-f`：强制删除镜像，即使存在基于该镜像运行的容器；注意：强制删除仅会移除镜像的标签引用，正在运行的容器所使用的镜像分层不会被物理删除，直至容器完全销毁；
    - `--no-prune`：不删除无标签的父镜像，默认会自动删除已无引用的父镜像。
    

#### 底层删除逻辑

Docker 镜像采用分层引用计数机制：

1. 执行删除命令时，仅会先删除该镜像对应的标签引用，当镜像的所有标签均被删除后，镜像主体进入待清理状态；
2. 系统会校验镜像各分层的引用计数，仅当某一分层的引用计数降为 0 时，才会物理删除该分层，释放磁盘空间；
3. 若多个镜像共享同一底层分层，删除其中一个镜像不会影响其他镜像，也不会删除共享分层，保证分层复用机制的稳定性。

### 2.4 镜像批量清理命令

`docker image prune` 是专门用于清理无用镜像的标准化命令，是生产环境磁盘空间管控的核心工具，可精准过滤并清理无业务价值的镜像。

- **核心语法**

    ```bash
    docker image prune [OPTIONS]
    ```
    
- **核心参数**
    
    - `--all/-a`：清理所有未被任何容器引用的镜像，不仅限于虚悬镜像；
    - `--filter`：按条件过滤清理范围，常用规则：`until=24h`（清理 24 小时内未被使用的镜像）、`label!=key=value`（排除指定标签的镜像）；
    - `--force/-f`：强制清理，不弹出确认提示，适用于自动化定时清理脚本。
    

#### 生产环境最佳实践

1. 生产环境需配置定时任务执行`prune`命令，避免无用镜像持续占用磁盘空间；
2. 必须添加过滤条件，严禁直接使用`-a`参数无差别清理，避免误删备用镜像、历史版本镜像；
3. 核心业务镜像需添加固定标签，通过`label`过滤规则排除在清理范围外，保障业务稳定性。
