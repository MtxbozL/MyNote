随着开发的深入，你不断地 `pull`（拉取）和 `build`（构建）镜像，宿主机的磁盘空间会被悄悄吞噬。如果不进行科学的管理，磁盘最终会报警。

本节我们将聚焦于本地镜像的资产盘点、精准删除，并深度解析一个非常重要的高阶概念——**虚悬镜像 (Dangling Images)**。

### 1. `docker images`：本地资产盘点

要管理资产，首先得看清家底。`docker images`（或较新的规范命令 `docker image ls`）用于列出宿主机本地已存在的所有镜像。

*   **基础语法**：`docker images`
*   **核心输出字段解析**：
    *   **REPOSITORY**：镜像的仓库源名称（如 `nginx`、`ubuntu`）。
    *   **TAG**：镜像的标签版本号。
    *   **IMAGE ID**：镜像的唯一标识符（由 SHA256 哈希值计算得出，通常显示前 12 位）。
    *   **CREATED**：镜像被构建出来的相对时间（注意：这是镜像作者构建它的时间，而不是你 pull 下来的时间）。
    *   **SIZE**：镜像的逻辑体积。
*   **👨‍💻 学术建议 / 进阶参数**：
    *   `docker images -a`：显示所有镜像（包含隐藏的中间层镜像）。
    *   `docker images -q`：**极客必会参数**，仅显示 `IMAGE ID`。这个参数在我们需要进行“批量删除”结合脚本使用时具有极其重要的工程价值。

### 2. `docker rmi`：精准销毁镜像

当我们不再需要某个镜像时，应将其从宿主机删除以释放空间。

*   **基础语法**：`docker rmi [镜像名:Tag] 或 [IMAGE ID]`
    *(注：`rm` 代表 remove container，而 `rmi` 中的 `i` 代表 image，切勿混淆)*
*   **示例**：`docker rmi nginx:latest` 或 `docker rmi 605c77e624dd`
*   **经典避坑：为什么会报 `conflict: unable to delete` 错误？**
    *   **原因机制**：如果基于该镜像实例化的**任何容器**（即使容器处于 Stop 停止状态）依然存在于宿主机上，Docker 引擎出于数据安全考量，会锁定该镜像，拒绝删除。
    *   **正确处理路径**：先使用 `docker rm` 删除依赖它的所有容器，然后再使用 `docker rmi` 删除该镜像。
    *   *(暴力选项)*：你可以添加 `-f` 参数强制删除（`docker rmi -f [IMAGE ID]`），但强烈不推荐在生产环境使用。

### 3. 深度概念探究：虚悬镜像 (Dangling Image)

在执行 `docker images` 时，你有时会看到一些奇怪的镜像，它们的 `REPOSITORY` 和 `TAG` 列显示的都是 **`<none>`**。这类镜像在学术上被称为 **虚悬镜像 (Dangling Image)**。

*   **它们是怎么产生的？**
    假设你编写了一个 Dockerfile，构建了一个名为 `my-app:1.0` 的镜像。第二天，你修改了应用代码，再次执行构建命令，且**赋予了完全相同的名称和标签 `my-app:1.0`**。
    此时，旧的镜像并不会消失。Docker 会将 `my-app:1.0` 这个名字“剥夺”并赋予给新构建的镜像。那么，失去了名字和标签的旧镜像，就变成了 `<none>:<none>`，即虚悬镜像。
*   **工程影响**：
    虚悬镜像已经失去了被引用的逻辑入口，它们本质上就是过期的**“垃圾数据”**，毫无保留价值，且会白白占用大量的系统存储空间。

### 4. `docker image prune`：系统级优雅瘦身

面对堆积如山的虚悬镜像，如果通过 `docker rmi` 逐个去删，效率极低。Docker 提供了一个极其优雅的清理命令：`prune`（修剪）。

*   **清理虚悬镜像**：
    执行 `docker image prune`
    Docker 会自动扫描并**安全地一键清除所有悬空的（无标签的）且没有被任何容器引用的虚悬镜像**。执行完毕后，它会贴心地告诉你释放了多少 MB/GB 的空间。
*   **终极瘦身（核弹级命令，慎用）**：
    执行 `docker image prune -a`
    加上 `-a` (all) 参数后，清理逻辑将发生质变：它不仅清理虚悬镜像，还会清理**所有当前没有被任何运行中或停止中的容器所使用的镜像**。通常用于磁盘严重告警时的极限瘦身。
