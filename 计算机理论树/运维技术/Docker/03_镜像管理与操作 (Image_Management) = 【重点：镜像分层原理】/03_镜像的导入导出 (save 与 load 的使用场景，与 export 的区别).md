
本节讲解 Docker 镜像离线传输的两组核心命令，重点明确`save/load`与`export/import`的底层差异、适用场景与使用边界，严禁两组命令混用导致的元数据丢失、镜像不可用问题。

### 3.1 docker save 与 docker load

该组命令用于 Docker 镜像的完整导出与加载，是镜像离线备份、迁移的官方推荐方案，可完整保留镜像的所有结构与元数据。

#### 3.1.1 docker save 镜像导出

将一个或多个镜像完整导出为 tar 归档文件，完整保留镜像的所有分层、元数据、标签、历史记录与构建信息。

- **核心语法**

    ```bash
    docker save [OPTIONS] IMAGE [IMAGE...]
    ```
    
- **核心参数**
    
    - `--output/-o`：指定导出的 tar 文件路径，替代标准输出重定向。
    
- **典型示例**

    ```bash
    # 导出单个镜像并指定标签
    docker save -o nginx_1.27.0.tar nginx:1.27.0
    # 导出多个镜像至同一个归档文件
    docker save -o service_images.tar nginx:1.27.0 mysql:8.0 redis:7.2
    ```
    
- **底层逻辑**：导出的 tar 文件包含镜像的所有只读分层、配置元数据、标签信息，与镜像仓库中存储的镜像结构完全一致，无任何信息丢失。

#### 3.1.2 docker load 镜像加载

从`docker save`导出的 tar 归档文件中，完整加载镜像至本地，恢复镜像的所有结构、标签与元数据。

- **核心语法**

    ```bash
    docker load [OPTIONS]
    ```
    
- **核心参数**
    
    - `--input/-i`：指定加载的 tar 文件路径，替代标准输入重定向；
    - `--quiet/-q`：静默加载，不输出加载日志。
    
- **典型示例**

    ```bash
    docker load -i nginx_1.27.0.tar
    ```
    
- **核心特性**：加载后的镜像与原镜像完全一致，包括镜像 ID、标签、分层结构、历史记录与启动配置，可直接用于容器创建，无需任何额外配置。

#### 适用场景

1. 离线隔离环境的镜像传输，无网络连接的生产环境镜像部署；
2. 镜像的完整备份与归档，保留镜像的所有可追溯信息；
3. 多主机之间的镜像批量迁移，无需经过镜像仓库中转。

### 3.2 docker export 与 docker import

该组命令用于容器文件系统的导出与导入，仅针对容器的可写层快照，不保留镜像的分层结构与元数据，与`save/load`存在本质差异。

#### 3.2.1 docker export 容器文件系统导出

将**运行中或已停止的容器**的当前文件系统快照导出为 tar 归档文件，仅保留容器内的静态文件内容。

- **核心语法**

    ```bash
    docker export [OPTIONS] CONTAINER
    ```
    
- **核心参数**
    
    - `--output/-o`：指定导出的 tar 文件路径。
    
- **底层逻辑**：仅导出容器当前可写层的文件系统快照，完全丢弃原镜像的所有分层结构、元数据、历史记录、环境变量、启动命令、卷挂载信息等核心配置，仅保留文件系统的静态内容。

#### 3.2.2 docker import 快照导入为镜像

从`docker export`导出的 tar 文件中，将文件系统快照导入为一个全新的镜像。

- **核心语法**

    ```bash
    docker import [OPTIONS] FILE|URL|- [REPOSITORY[:TAG]]
    ```
    
- **核心参数**
    
    - `--change/-c`：导入时追加 Dockerfile 指令，为新镜像设置元数据，如`CMD`、`ENV`、`EXPOSE`、`ENTRYPOINT`等；
    - `--message/-m`：为导入的镜像设置提交说明。
    
- **底层逻辑**：将导入的文件系统快照作为一个全新的单一层镜像，无任何历史分层与元数据，生成的镜像 ID 与原容器的基础镜像完全不同，镜像体积更小。

### 3.3 两组命令的核心差异对比

|对比维度|docker save / docker load|docker export / docker import|
|---|---|---|
|操作对象|Docker 镜像（完整分层结构）|Docker 容器的文件系统快照|
|内容完整性|完整保留所有分层、元数据、标签、历史记录、启动配置|仅保留文件系统静态内容，丢弃所有元数据、分层历史、启动配置|
|镜像 ID 一致性|加载后的镜像与原镜像 ID 完全一致|导入后生成全新镜像，ID 与原镜像无关联|
|分层结构|完整保留原有分层结构，支持分层复用|仅生成单一层镜像，无分层结构|
|镜像体积|相对较大，包含全部分层数据|相对较小，仅包含当前文件系统快照|
|可恢复性|完整恢复镜像所有功能，可直接启动容器|需手动配置启动命令、环境变量等，否则无法正常启动|
|适用场景|镜像完整备份、离线迁移、合规归档|容器定制化快照、基础镜像精简制作|
|生产规范|生产环境镜像备份迁移的唯一推荐方案|严禁用于业务镜像的备份与迁移，仅适用于特殊定制场景|
