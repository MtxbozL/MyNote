
UnionFS（联合文件系统）是 Docker 镜像分层存储、容器文件系统隔离的核心底层技术，其完整的架构原理、Overlay2 驱动实现、写时复制机制，已在第三章「镜像管理与操作」中进行了深度讲解，本章重点补充其与 Namespace、Cgroups 的协同工作机制，完整还原容器运行时的文件系统底层实现。

### 3.1 与 MNT Namespace 的协同

1. Docker 为容器创建独立的 MNT Namespace 后，会在该 Namespace 内执行 Overlay2 联合文件系统的挂载操作：将镜像的所有只读分层（lowerdir）与容器的可写层（upperdir）进行联合挂载，构建容器的根文件系统视图；
2. 该挂载操作仅在当前容器的 MNT Namespace 内生效，对宿主机与其他容器完全不可见，实现了容器文件系统的隔离；
3. 通过`pivot_root`系统调用，将容器的根目录切换为 Overlay2 联合挂载的 mergedir 目录，容器内的进程仅能看到该合并视图，无法访问宿主机的文件系统，彻底实现了根文件系统的隔离。

### 3.2 与 Cgroups 的协同

容器可写层的磁盘 IO 使用量，会被 Cgroups 的 blkio 子系统统一管控，用户配置的 IO 带宽、IOPS 限制，会直接作用于容器可写层的所有读写操作，实现容器磁盘 IO 资源的配额管控。

### 3.3 三大内核机制的完整协同流程

当执行`docker run`启动一个容器时，Linux 内核层面会按顺序执行以下操作，完整串联三大核心机制：

1. Docker Daemon 接收容器创建请求，校验本地镜像是否存在，准备镜像的分层文件系统；
2. 调用`clone()`系统调用创建容器的 1 号进程，同时为该进程创建并启用 PID、NET、IPC、MNT、UTS、USER 六大独立 Namespace，构建隔离的运行环境视图；
3. 为容器创建独立的 Cgroups 控制组，配置 CPU、内存、IO 等资源限制规则，将容器的 1 号进程加入该控制组，内核开始强制执行资源配额；
4. 在容器的 MNT Namespace 内，执行 Overlay2 联合文件系统的挂载操作，通过`pivot_root`切换容器的根文件系统，构建完整的运行环境；
5. 在隔离的 Namespace 与 Cgroups 管控下，执行容器的 ENTRYPOINT/CMD 启动命令，启动业务进程，容器进入 Running 状态。
