
端口映射（端口发布）是实现外部网络访问容器内服务的核心机制，其底层基于容器的 Network Namespace 隔离特性：容器拥有独立的网络栈、IP 地址与端口空间，与宿主机网络栈完全隔离，外部网络无法直接访问容器内的端口，必须通过端口映射建立宿主机到容器的流量转发通道。

### 2.1 核心命令与语法

端口映射通过`-p/--publish`（指定映射）与`-P/--publish-all`（随机映射）两个参数实现，仅可在容器创建时配置，容器启动后无法修改，需重建容器才能变更端口映射规则。

#### 2.2.1 指定端口映射：-p（生产环境唯一推荐）

`-p`参数可精准定义宿主机与容器的端口绑定规则，支持 4 种标准化语法，覆盖不同安全场景：

1. **全网卡绑定（最常用）**：`宿主机端口:容器端口`
    
    - 示例：`docker run -d -p 8080:80 nginx:1.27.0`
    - 作用：将宿主机所有网卡（0.0.0.0）的 8080 端口，映射到容器的 80 端口，外部可通过宿主机任意 IP 的 8080 端口访问容器内的 Nginx 服务；
    
2. **指定 IP 绑定（生产安全加固推荐）**：`宿主机IP:宿主机端口:容器端口`
    
    - 示例：`docker run -d -p 192.168.1.100:8080:80 nginx:1.27.0`
    - 作用：仅绑定宿主机指定内网 IP 的 8080 端口，禁止公网访问，大幅缩小攻击面，符合生产环境最小权限原则；
    
3. **指定 IP 随机端口**：`宿主机IP::容器端口`
    
    - 示例：`docker run -d -p 192.168.1.100::80 nginx:1.27.0`
    - 作用：绑定宿主机指定 IP 的随机高位端口到容器 80 端口，端口范围为 32768-65535；
    
4. **协议指定**：`宿主机端口:容器端口/协议`
    
    - 示例：`docker run -d -p 53:53/udp coredns/coredns`
    - 作用：指定映射的协议类型，默认为 TCP 协议，UDP 服务需显式声明协议类型。
    

#### 2.2.2 随机端口映射：-P（仅临时测试可用）

- 作用：自动将镜像 Dockerfile 中通过`EXPOSE`声明的所有端口，映射到宿主机的随机高位端口（32768-65535）；
- 示例：`docker run -d -P nginx:1.27.0`
- 底层逻辑：Docker Daemon 自动扫描镜像的 EXPOSE 声明，为每个声明的端口分配未被占用的宿主机高位端口，创建 iptables 转发规则；
- 风险警示：**生产环境严禁使用 - P 参数**，随机端口无法配置防火墙规则、负载均衡转发与服务发现，会导致业务访问链路不可控，仅适用于本地临时测试场景。

### 2.2 底层实现原理

Docker 端口映射的底层基于 Linux 内核的 iptables/netfilter 框架实现，通过**DNAT（目标地址转换）** 规则完成流量转发：

1. 容器创建时，Docker Daemon 在宿主机的 iptables NAT 表中添加 DNAT 规则，将宿主机指定端口的入站流量，目标地址转换为容器的 IP 与对应端口；
2. 宿主机内核通过路由规则，将转换后的流量转发到容器的 Network Namespace 中；
3. 容器内的服务接收流量并处理后，按原路返回响应数据，完成整个通信流程。

可通过`iptables -t nat -L -n`命令查看 Docker 自动生成的端口映射 DNAT 规则，验证映射配置是否生效。

### 2.3 核心概念区分：EXPOSE 与 -p

`EXPOSE`是 Dockerfile 中的声明指令，仅用于告知镜像使用者，容器内服务监听的端口号，属于文档化声明，**无任何实际的端口映射与网络放行效果**。

- 仅当使用`-P`参数时，Docker 会基于 EXPOSE 声明的端口创建随机映射；
- 未通过`-p`参数显式映射的端口，即使声明了 EXPOSE，外部网络也无法访问；
- 生产规范：镜像必须通过 EXPOSE 声明服务监听的端口，便于运维识别端口配置，同时必须通过`-p`参数显式配置端口映射，禁止依赖随机映射。
