## 00 Docker 网络的底层核心基石

Docker 所有网络能力均基于 Linux 内核原生的网络虚拟化技术实现，四大核心基石构成了容器网络的底层技术底座，是理解所有网络模式的前提：

1. **Linux Network Namespace**
    
    容器网络隔离的核心机制，为每个容器创建独立的网络命名空间，包含专属的网卡设备、路由表、iptables 规则、端口空间、协议栈，实现容器与宿主机、容器与容器之间的网络栈完全隔离，是第一章六大隔离机制的核心组成部分。
2. **veth pair（虚拟以太网设备对）**
    
    容器网络的虚拟 "网线"，成对出现的虚拟网卡设备，核心特性是数据包从一端进入，会完整、无损耗地从另一端输出。Docker 为每个 Bridge 模式容器创建一对 veth pair：一端放入容器的 Network Namespace，重命名为`eth0`作为容器的主网卡；另一端接入宿主机的 Linux 网桥，作为网桥的端口，实现容器与网桥的二层数据互通。
3. **Linux Bridge（网桥）**
    
    工作在数据链路层的虚拟二层交换机，是 Bridge 网络模式的核心转发组件。Docker 通过网桥连接宿主机上的所有 veth pair 设备、物理网卡，实现同网段内的数据包二层转发、MAC 地址学习、广播域控制，是容器间通信的核心枢纽。
4. **iptables/netfilter**
    
    Linux 内核原生的数据包过滤与地址转换框架，Docker 通过 iptables 实现三大核心能力：容器访问公网的**SNAT（源地址转换）**、外部访问容器的**DNAT（目标地址转换，即端口映射）**、不同网络之间的隔离与流量管控，是 Docker 网络安全与跨网段通信的核心实现。

## 01 Docker 默认网络模式

Docker 安装完成后，会自动创建 3 个核心网络，可通过`docker network ls`命令查看，分别对应三大原生网络模式。所有容器默认使用`bridge`模式，可通过`--network`参数在容器创建时显式指定网络模式。

三大默认网络模式的核心定位与特性总览：

|网络模式|隔离核心特性|核心能力|生产推荐等级|
|---|---|---|---|
|bridge（默认网桥 docker0）|独立 Network Namespace，通过虚拟网桥与宿主机通信|容器间互通、容器与外部网络互通、端口映射|低，仅临时测试可用，生产严禁使用|
|host|共享宿主机 Network Namespace，无网络隔离|无端口映射，直接使用宿主机网络栈，性能无损耗|中，仅极致性能需求的特定场景使用|
|none|独立 Network Namespace，无任何网卡设备|仅保留本地回环接口 lo，完全无网络连接|低，仅高安全离线场景使用|
