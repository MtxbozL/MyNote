
## 1.3 MongoDB 核心概念与关系型数据库映射

### 1.3.1 核心层级概念

MongoDB 的数据组织遵循**实例 (Instance) → 数据库 (Database) → 集合 (Collection) → 文档 (Document) → 字段 (Field)** 的层级结构，各单元的严格定义如下：

1. **数据库（Database）**：MongoDB 实例中的顶层命名空间与数据隔离单元，一个独立实例可创建多个相互隔离的数据库，每个数据库拥有独立的集合、权限配置与存储文件。默认内置 admin（权限管理）、local（副本集本地数据）、config（分片集群元数据）三个系统数据库。
2. **集合（Collection）**：数据库内文档的逻辑分组单元，对应 RDBMS 中的数据表。集合无强制 Schema 约束，仅可在创建时设置可选约束（如文档大小上限、TTL 过期规则、唯一索引约束），分为普通集合、固定集合（Capped Collection）、视图（View）三类。
3. **文档（Document）**：MongoDB 中最小的数据存储与操作单元，对应 RDBMS 中的数据行。文档是 BSON 格式的有序键值对集合，**单文档存在 16MB 的硬大小上限**，超出限制的文档无法写入。每个文档必须包含`_id`字段作为唯一主键，写入时未指定则自动生成 ObjectId 类型的默认值。
4. **字段（Field）**：文档内的单个键值对，对应 RDBMS 中的数据列。字段无固定数据类型约束，同一字段在不同文档中可存储不同类型的数据，支持字符串、数值、日期、嵌套文档、数组等全量 BSON 数据类型。

### 1.3.2 与 RDBMS 的精准映射对比

|MongoDB 核心概念|RDBMS 对应概念|核心差异说明|
|---|---|---|
|数据库（Database）|数据库（Database）|逻辑定位基本一致，均为数据隔离的顶层单元|
|集合（Collection）|数据表（Table）|集合无强制 Schema 约束；表有严格的固定结构约束，结构修改需执行 DDL|
|文档（Document）|行（Row/Record）|文档为支持嵌套的 BSON 结构化数据，结构灵活；行是扁平结构化数据，严格遵循表结构|
|字段（Field）|列（Column/Field）|字段无固定数据类型约束，支持动态增减；列有严格的数据类型与约束定义|
|索引（Index）|索引（Index）|逻辑定位基本一致，均为加速查询的辅助数据结构，MongoDB 支持更多特殊索引类型|
|主键（_id）|主键（Primary Key）|MongoDB 强制每个文档必须有_id 主键，默认自动生成；RDBMS 主键可自定义，允许无主键表|
