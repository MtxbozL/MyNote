
## 1.2 MongoDB 简介

### 1.2.1 核心定位与技术实现

MongoDB 是由 MongoDB Inc. 开发维护的 **开源、分布式、面向文档的 NoSQL 数据库** ，核心采用 C++ 语言编写，2009 年首次发布，当前为全球应用最广泛的文档型数据库。其设计核心是在保留 RDBMS 核心企业级能力（复杂查询、索引、ACID 事务）的同时，实现灵活的动态数据模型、原生分布式架构与无缝水平扩展能力，适配现代应用的快速迭代与规模化部署需求。

### 1.2.2 核心设计特性

1. **Schema-less 动态模式**：MongoDB 采用**读时模式（Schema on Read）**，无强制的表结构约束，同一集合内的文档可拥有不同的字段结构与数据类型，业务迭代无需执行 ALTER TABLE 等 DDL 操作，模式约束由应用层灵活控制，而非数据库强制校验。
2. **原生分布式架构**：从底层设计原生支持副本集高可用架构与分片集群水平扩展架构，可实现从单节点开发环境到 PB 级海量数据生产集群的平滑扩展，无需依赖第三方中间件实现分布式能力。
3. **文档型数据模型**：采用 BSON（Binary JSON）作为存储格式，兼容 JSON 的人类可读性，同时提供更高的编解码效率、更小的存储体积，支持日期、二进制、嵌套文档、数组等丰富数据类型，可直接映射面向对象编程的实体模型，消除 ORM/ODM 的转换开销。
4. **企业级能力完备**：支持多维度索引、聚合管道数据分析、多文档 ACID 事务、细粒度权限控制、数据备份恢复等全栈企业级特性，可支撑核心业务系统的稳定运行。
