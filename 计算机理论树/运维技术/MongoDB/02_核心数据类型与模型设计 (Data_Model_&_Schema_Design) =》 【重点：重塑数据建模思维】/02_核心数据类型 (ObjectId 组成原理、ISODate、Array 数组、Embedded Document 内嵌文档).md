
## 2.2 MongoDB 核心数据类型

MongoDB 的所有数据均以 BSON 类型存储，核心数据类型可分为标量基础类型、复合结构类型、系统特殊类型三大类，以下为业务开发与建模中的核心类型详解。

### 2.2.1 标量基础类型

该类为单值原子类型，是文档字段的基础构成单元，核心类型如下：

1. **String 字符串**：BSON 唯一合法的字符串类型，强制采用 UTF-8 编码，是业务中最常用的基础类型。MongoDB 的字符串操作、文本索引均基于 UTF-8 编码规范实现。
2. **Numeric 数值类型**：
    
    - `Int32`：32 位有符号整数，取值范围`-2^31 ~ 2^31-1`，适用于小数值场景，存储体积最小；
    - `Int64`：64 位有符号整数，适用于大整数、主键序列号、金额分单位存储等场景，避免精度丢失；
    - `Double`：64 位双精度浮点数，适用于非高精度的数值计算场景，存在浮点精度误差，禁止用于金融账务场景；
    - `Decimal128`：128 位高精度十进制数，支持 34 位有效数字，完全符合 IEEE 754-2008 规范，专为金融、账务等零精度容忍的场景设计。
    
3. **Boolean 布尔类型**：仅支持`true`和`false`两个布尔值，无隐式类型转换。
4. **Null 空值类型**：代表字段值为空，与字段不存在为两个完全不同的语义，可通过`$exists`操作符进行区分。
5. **Binary Data 二进制类型**：用于存储非结构化二进制数据，如图片、文件、加密数据等，可避免文本转义带来的开销，是 GridFS 大文件存储的底层基础类型。
6. **Regular Expression 正则表达式类型**：原生支持正则表达式存储，可直接用于查询匹配操作。

### 2.2.2 复合结构类型

该类为多值组合类型，是 MongoDB 实现灵活数据模型的核心，也是区别于关系型数据库的关键特性。

1. **Embedded Document 内嵌文档**
    
    指一个字段的值为完整的 BSON 文档，支持多层嵌套，是 MongoDB 实现非规范化建模的核心载体。其核心特性为：
    
    - 天然适配面向对象的编程模型，可直接映射代码中的嵌套实体结构，消除 ORM 转换开销；
    - 关联数据与主数据集中存储，一次 IO 即可完成全量读取，无需多表 Join 操作；
    - 单文档内的内嵌文档与主文档的写操作具备原生原子性，无需事务即可保证数据一致性。
    
2. **Array 数组类型**
    
    BSON 原生支持数组结构，数组内的元素可包含任意 BSON 数据类型，支持同数组内异构类型元素、嵌套数组、嵌套文档混合存储。其核心特性为：
    
    - MongoDB 原生提供全量数组查询、更新、聚合操作符，支持数组元素的精准匹配、修改与计算；
    - 针对数组字段创建索引时，会自动为数组内的每一个元素生成独立的索引条目，即多键索引，可实现针对数组元素的高效查询；
    - 数组是桶模式、属性模式等经典建模模式的核心基础结构。
    

### 2.2.3 系统特殊类型

该类为 MongoDB 系统内置的专用类型，具备特殊的语义与系统能力，是数据库正常运行的核心基础。

1. **ObjectId**
    
    ObjectId 是 MongoDB 默认的`_id`主键字段类型，为 12 字节的 BSON 二进制数据，具备全局唯一性，无需中心化生成节点即可在分布式环境下无冲突生成。其 12 字节的组成结构严格定义如下：
    
    - 第 1-4 字节：Unix 时间戳（秒级，大端序），记录文档的创建时间；
    - 第 5-9 字节：随机生成的机器 / 进程唯一标识，保证不同机器、不同 MongoDB 实例生成的 ObjectId 不冲突；
    - 第 10-12 字节：自增计数器（大端序），保证同一秒内、同一进程下生成的 ObjectId 具备唯一性。
        
        其核心特性：可排序性（基于前缀时间戳，按`_id`排序等价于按文档创建时间排序）、可反解（可提取创建时间戳）、分布式无锁生成。
    
2. **ISODate 日期类型**
    
    BSON 原生的时间日期类型，采用 64 位有符号整数存储，记录 Unix 纪元（1970-01-01 00:00:00 UTC）以来的毫秒数，支持正负值，可覆盖公元前与公元后的全量时间范围。核心特性：强制采用 UTC 时区存储，避免跨时区业务的时间混乱，应用层需完成时区转换；原生支持丰富的日期查询、聚合计算操作符。
3. **Timestamp 时间戳类型**
    
    64 位 MongoDB 内部专用时间戳类型，与 ISODate 业务日期类型完全隔离，仅用于副本集 Oplog、分片集群等内部系统场景，禁止业务层使用。
4. **DBRef 数据库引用类型**
    
    标准化的跨集合、跨数据库文档引用规范，包含目标文档的`_id`、所属集合、所属数据库信息，用于复杂引用场景的标准化关联，区别于手动的`_id`外键引用。
