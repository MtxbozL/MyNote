#### 参考答案

#### 一、配置前提

1. 申请合法的 SSL 证书（通常包含公钥证书 `.crt`/`.pem`、私钥文件 `.key`）；
2. Nginx 编译时需开启 `--with-http_ssl_module` 模块（主流安装包默认已开启）；
3. 服务器开放 443 端口（HTTPS 默认端口）。

#### 二、极简核心配置

```nginx
# HTTP强制跳转HTTPS
server {
    listen 80;
    server_name example.com;
    return 301 https://$server_name$request_uri;
}

# HTTPS服务配置
server {
    listen 443 ssl http2;
    server_name example.com;

    # 证书配置
    ssl_certificate /etc/nginx/ssl/example.com.crt; # 公钥证书
    ssl_certificate_key /etc/nginx/ssl/example.com.key; # 私钥文件

    # SSL安全配置
    ssl_protocols TLSv1.2 TLSv1.3; # 禁用不安全的SSLv3、TLS1.0、TLS1.1
    ssl_ciphers ECDHE-RSA-AES128-GCM-SHA256:HIGH:!aNULL:!MD5:!RC4:!DHE;
    ssl_prefer_server_ciphers on; # 优先使用服务端的加密套件
    ssl_session_cache shared:SSL:10m; # 复用SSL会话，减少握手耗时
    ssl_session_timeout 10m;

    # HSTS配置，强制浏览器使用HTTPS访问
    add_header Strict-Transport-Security "max-age=31536000; includeSubDomains" always;

    # 业务配置（反向代理/静态资源等）
    location / {
        root /usr/share/nginx/html;
        index index.html;
    }
}
```

#### 三、核心注意事项

1. 证书文件权限：私钥文件需设置 600 权限，避免泄露；证书链必须完整，否则浏览器会报不安全警告。
2. 安全合规：必须禁用低版本协议和弱加密套件，避免 SSL 漏洞攻击；生产环境必须配置 HSTS，防止降级劫持。
3. 性能优化：开启 SSL 会话复用、HTTP2 协议，大幅降低 HTTPS 握手耗时，提升访问速度。
4. 证书有效期：提前监控证书过期时间，避免证书过期导致网站无法访问。
5. 域名匹配：证书的域名必须和配置的 server_name 一致，泛域名证书需匹配对应的泛域名规则。
