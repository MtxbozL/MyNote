**职级适配**：中级必问、高级进阶

#### 【核心考点】

考察 HTTPS 协议的理解、生产环境安全合规配置能力、安全与性能的平衡能力，是企业级生产环境的必备技能，全站 HTTPS 已成为行业标配。

#### 【专家级标准答案】

1. **前置配置要求**
    
    - 已申请合法的 SSL 证书（包含公钥证书`.pem/.crt`、私钥文件`.key`，需配置完整证书链）；
    - Nginx 已编译`--with-http_ssl_module`模块（主流安装包默认已集成）；
    - 服务器已开放 443 端口（HTTPS 默认端口），防火墙已放行对应流量。
    
2. **生产级标准化配置模板**

```nginx
    # HTTP强制跳转HTTPS，301永久重定向，保留SEO权重
    server {
        listen 80;
        server_name example.com www.example.com;
        return 301 https://$server_name$request_uri;
    }
    
    # HTTPS核心服务配置
    server {
        listen 443 ssl http2;
        server_name example.com www.example.com;
        root /usr/share/nginx/example.com;
        index index.html;
    
        # 1. SSL证书核心配置
        ssl_certificate /etc/nginx/ssl/example.com_fullchain.pem; # 完整证书链文件
        ssl_certificate_key /etc/nginx/ssl/example.com_private.key; # 证书私钥文件
    
        # 2. SSL安全合规配置（等保合规标准）
        # 禁用不安全的协议，仅支持TLS1.2和TLS1.3
        ssl_protocols TLSv1.2 TLSv1.3;
        # 配置安全加密套件，禁用弱加密算法
        ssl_ciphers ECDHE-ECDSA-AES128-GCM-SHA256:ECDHE-RSA-AES128-GCM-SHA256:ECDHE-ECDSA-AES256-GCM-SHA384:ECDHE-RSA-AES256-GCM-SHA384:ECDHE-ECDSA-CHACHA20-POLY1305:ECDHE-RSA-CHACHA20-POLY1305:DHE-RSA-AES128-GCM-SHA256:DHE-RSA-AES256-GCM-SHA384;
        # 优先使用服务端的加密套件，避免客户端使用弱加密算法
        ssl_prefer_server_ciphers on;
        # 禁用不安全的SSL会话重用机制
        ssl_session_tickets off;
    
        # 3. SSL性能优化配置
        # 配置SSL会话缓存，复用SSL会话，减少重复握手开销
        ssl_session_cache shared:SSL:20m;
        ssl_session_timeout 1d;
    
        # 4. 安全防护头部配置
        # HSTS强制浏览器使用HTTPS访问，有效期1年，包含子域名，预加载
        add_header Strict-Transport-Security "max-age=31536000; includeSubDomains; preload" always;
        # 防XSS攻击、MIME类型嗅探、点击劫持
        add_header X-Frame-Options SAMEORIGIN always;
        add_header X-Content-Type-Options nosniff always;
        add_header X-XSS-Protection "1; mode=block" always;
    
        # 5. OCSP装订配置，提升SSL握手速度与证书验证安全性
        ssl_stapling on;
        ssl_stapling_verify on;
        ssl_trusted_certificate /etc/nginx/ssl/example.com_fullchain.pem;
        resolver 223.5.5.5 114.114.114.114 valid=300s;
        resolver_timeout 5s;
    
        # 业务配置（动静分离/反向代理等，此处省略）
        location / {
            try_files $uri $uri/ /index.html;
        }
    }
```
    
3. **核心配置合规要求**
    
    - 证书配置：必须使用完整的证书链文件，避免浏览器报 “不安全证书” 警告；私钥文件权限必须设置为 600，仅 root 用户可读写，防止私钥泄露。
    - 安全合规：必须禁用 SSLv3、TLS1.0、TLS1.1 等不安全协议，禁用 NULL、MD5、RC4 等弱加密套件，满足网络安全等级保护合规要求。
    - 强制 HTTPS：必须配置 HTTP 到 HTTPS 的永久重定向，同时配置 HSTS 头部，防止 HTTPS 降级劫持攻击。
    

#### 【深度拓展（面试加分项）】

1. 双证书适配方案：生产环境可配置 RSA+ECC 双证书，兼容老旧客户端的同时，提升新客户端的握手性能，ECC 证书密钥长度更短，加解密速度更快，CPU 开销更低。
2. HTTP/3 QUIC 协议支持：Nginx 1.25.0 及以上版本已原生支持 HTTP/3，基于 QUIC 协议，采用 UDP 传输，彻底解决 TCP 队头阻塞问题，弱网环境下访问性能提升显著。
3. 证书自动化运维：通过 Let's Encrypt 免费证书 + Certbot 工具，实现证书的自动申请、续期、重载，避免证书过期导致网站无法访问，生产环境需配置证书过期监控告警。
4. 国密 SSL 支持：针对等保 2.0 国密合规要求，可通过第三方模块实现 Nginx 的国密 SM2/SM3/SM4 算法支持，满足政企行业的合规要求。

#### 【面试避坑提示】

1. 禁止启用 SSLv3、TLS1.0、TLS1.1 等不安全协议，这些协议存在严重的安全漏洞，同时无法满足等保合规要求，面试中提及直接扣分。
2. 禁止忽略证书链的完整性，仅配置域名证书，不配置中间证书，会导致移动端、老旧浏览器出现证书不信任的问题。
3. 禁止盲目配置超长 HSTS 有效期，配置前需确认全站所有子域名均已支持 HTTPS，否则会导致未配置 HTTPS 的子域名无法访问。
4. 禁止将证书私钥文件提交到代码仓库，或设置过高的文件权限，造成私钥泄露，引发严重的安全事故。
