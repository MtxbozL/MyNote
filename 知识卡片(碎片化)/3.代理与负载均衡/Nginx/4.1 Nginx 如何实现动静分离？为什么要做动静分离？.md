#### 参考答案

#### 一、实现原理

动静分离的核心是**将静态资源请求和动态接口请求拆分处理**：利用 Nginx 高性能的静态资源处理能力，直接响应图片、JS、CSS、HTML、字体等静态资源；将 PHP/Java/Python 等后端动态接口请求，通过反向代理转发给对应的应用服务器处理。

极简配置示例：

```nginx
server {
    listen 80;
    server_name example.com;
    root /usr/share/nginx/html;

    # 静态资源处理，Nginx直接响应
    location ~* \.(jpg|jpeg|png|gif|js|css|html|ico)$ {
        expires 7d; # 配置浏览器缓存
        access_log off; # 关闭静态资源日志，减少IO
    }

    # 动态接口请求，转发给后端Tomcat/PHP-FPM
    location /api/ {
        proxy_pass http://backend_server;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }
}
```

#### 二、核心目的

1. **大幅提升响应速度**：静态资源直接由 Nginx 返回，无需经过后端应用服务器的处理链路，减少响应耗时
2. **降低后端服务压力**：将占比极高的静态资源请求剥离，后端服务器只需专注处理业务逻辑，提升系统并发能力
3. **提升服务稳定性**：后端应用故障时，静态资源仍可正常访问，避免全站瘫痪；同时便于做静态资源缓存和 CDN 对接
4. **便于运维管控**：静态资源和动态服务可分别做扩容、降级、权限管控，架构更清晰
