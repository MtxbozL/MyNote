**职级适配**：初级必问、中级打底

#### 【核心考点】

考察对 Nginx 的核心定位、技术底座、业务价值的全局认知，判断候选人是否理解 Nginx 的技术边界与适用场景，而非仅停留在 “会用配置” 的层面。

#### 【专家级标准答案】

Nginx 是一款**基于异步非阻塞事件驱动模型的高性能、轻量级、高可靠的开源代理服务器**，同时具备 Web 服务、流量治理、安全防护能力，核心优势是高并发下极低的内存占用、极强的稳定性、极简的配置体系与丰富的扩展生态。

1. **核心技术特性**
    
    - 架构层：Master-Worker 多进程架构，支持热部署、平滑重启，7*24 小时无中断服务
    - 性能层：epoll 异步非阻塞 IO 模型，单 Worker 进程可支撑数万并发连接，CPU 与内存资源利用率极致
    - 能力层：原生支持 HTTP/HTTPS/HTTP2/HTTP3、TCP/UDP 四层代理、反向代理、负载均衡、静态资源服务
    - 扩展层：模块化设计，支持丰富的第三方扩展模块，可灵活适配限流、缓存、WAF 等定制化需求
    
2. **主流生产应用场景**
    
    - 反向代理与统一入口：隐藏后端服务集群，实现内外网隔离，统一业务流量入口
    - 负载均衡：多维度流量分发，提升系统并发能力与服务可用性
    - 动静分离与静态 Web 服务：直接托管静态资源，剥离动态请求，大幅降低后端业务服务器压力
    - HTTPS/SSL 终端：统一管理证书与加解密，简化后端服务配置，实现全站 HTTPS
    - 流量治理与安全防护：实现接口限流、IP 黑白名单、防 CC 攻击、基础 WAF 防护、请求重写与跳转
    - 缓存加速：实现静态资源与业务接口的本地缓存，提升响应速度，减少回源请求
    - 云原生场景：作为 K8s 集群的 Ingress Controller，实现容器化服务的流量治理与转发
    

#### 【深度拓展（面试加分项）】

1. 与同类产品的核心差异：相比 Apache 的同步阻塞进程模型，Nginx 在高并发场景下资源占用仅为 Apache 的 1/10；相比 Tomcat 等应用服务器，Nginx 专注于网络层与应用层的流量转发，不处理业务逻辑，性能有数量级优势。
2. 大厂落地最佳实践：通常采用「LVS 四层负载 + Nginx 七层代理」的集群架构，LVS 扛住海量并发连接，Nginx 实现精细化的七层流量治理，兼顾性能与灵活性。
3. 商业生态：除开源版本外，还有 Nginx Plus 商业版，提供主动健康检查、动态配置、高级监控、API 网关等企业级能力。

#### 【面试避坑提示】

1. 禁止仅将 Nginx 定义为 “反向代理服务器”，忽略其静态 Web 服务、四层代理、流量治理等核心能力，造成认知片面的扣分。
2. 禁止混淆 “反向代理” 与 “负载均衡” 的概念，负载均衡是反向代理的衍生能力，二者并非等同关系。
3. 禁止无边界夸大 Nginx 的能力，Nginx 不适合处理复杂业务逻辑，核心定位是流量转发与治理，而非应用服务器。
