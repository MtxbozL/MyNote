**职级适配**：初级必问

#### 【核心考点】

考察代理模式的本质理解、网络架构基础认知，判断候选人是否清晰 Nginx 在网络链路中的定位与作用，是后续所有配置与排障的基础。

#### 【专家级标准答案】

二者的**核心本质区别是：代理对象不同、服务的主体不同、链路中的透明性不同**，一句话总结核心差异：**正向代理代理客户端，对服务端透明；反向代理代理服务端，对客户端透明**。

1. **正向代理**
    
    - 核心定义：位于客户端与目标服务端之间，代理**客户端**发起请求，服务端仅能感知到代理服务器，无法获取真实客户端信息。
    - 核心特征：客户端必须明确感知并配置代理服务器，主动将请求发送给代理；代理服务器对服务端完全透明。
    - 典型场景：翻墙代理、企业内网访问权限管控、客户端网络加速、爬虫 IP 代理。
    
2. **反向代理**
    
    - 核心定义：位于客户端与后端服务集群之间，代理**服务端**接收请求，客户端仅能感知到反向代理服务器，无法获取后端服务集群的拓扑与信息。
    - 核心特征：客户端无感知，请求直接发送给反向代理服务器，由代理完成转发；后端服务集群对客户端完全透明。
    - 典型场景：网站统一入口、负载均衡、动静分离、后端服务安全隔离、HTTPS 统一终端。
    

#### 【深度拓展（面试加分项）】

1. 透明代理模式：属于正向代理的特殊形态，客户端无需配置代理，由网络设备强制将流量转发至代理服务器，客户端完全无感知，常见于企业内网行为管控、运营商网络优化。
2. 四层 vs 七层反向代理：Nginx 反向代理支持四层（TCP/UDP，基于 IP + 端口转发）和七层（HTTP/HTTPS，基于 URI、请求头等应用层信息转发），四层转发性能更高，七层转发灵活性更强。
3. 隧道代理：正向代理的核心能力之一，通过 HTTP CONNECT 方法建立隧道，实现 HTTPS 流量的端到端转发，代理服务器仅做数据透传，无法解密流量。

#### 【面试避坑提示】

1. 禁止仅罗列场景，不点明核心本质区别，面试官核心考察的是对代理模式底层逻辑的理解，而非场景背诵。
2. 禁止混淆代理对象，出现 “反向代理代理客户端” 这类根本性概念错误，直接判定为基础不达标。
3. 禁止将负载均衡与反向代理划等号，负载均衡只是反向代理的核心应用之一，不能代表反向代理的全部能力。
