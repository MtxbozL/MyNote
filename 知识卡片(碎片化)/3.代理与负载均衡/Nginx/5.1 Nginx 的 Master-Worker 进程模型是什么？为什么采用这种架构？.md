#### 参考答案

#### 一、模型核心原理

Nginx 采用**多进程单线程**的 Master-Worker 架构，默认启动两类进程：

1. **Master 进程（主进程）**：1 个，核心职责是管理和调度。负责读取并验证配置文件、管理 Worker 进程、接收运维信号（如重载配置、平滑重启、停止服务）、监控 Worker 进程状态，Worker 异常退出时自动重启新的 Worker 进程。
2. **Worker 进程（工作进程）**：多个，数量可配置，默认等于 CPU 核心数。核心职责是处理实际的客户端请求，所有网络请求、读写操作均由 Worker 进程处理，Worker 之间通过共享内存实现通信，采用 epoll 异步非阻塞事件模型处理并发请求，单 Worker 可支持数万并发连接。

#### 二、采用该架构的核心优势

1. **高可用性**：Worker 进程之间相互独立，单个 Worker 进程异常崩溃不会影响其他 Worker，Master 会快速重启新的 Worker，避免服务中断，实现 7*24 小时不间断运行，支持热部署（平滑重载配置、升级版本不中断服务）。
2. **极致的 CPU 利用率**：Worker 进程数可绑定 CPU 核心（worker_cpu_affinity），避免进程切换带来的 CPU 开销，充分利用多核 CPU 的性能，实现真正的并行处理。
3. **高并发低消耗**：单 Worker 进程采用单线程 + 异步非阻塞模型，无锁设计，避免了多线程切换的资源消耗，同时支持海量并发连接，内存占用极低。
4. **运维友好**：所有配置变更、服务启停操作，都通过 Master 进程统一管控，无需修改业务请求链路，操作风险极低。
