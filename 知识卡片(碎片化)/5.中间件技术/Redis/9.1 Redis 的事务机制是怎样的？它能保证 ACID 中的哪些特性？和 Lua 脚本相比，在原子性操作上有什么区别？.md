**标准答案**：

- Redis 事务核心机制：通过 MULTI、EXEC、DISCARD、WATCH 四个命令实现，将多个命令打包，按顺序一次性执行，执行期间不会被其他客户端的命令打断。
    
    1. MULTI：开启事务；
    2. 命令入队：事务内的命令不会立即执行，而是放入队列；
    3. EXEC：提交事务，按顺序执行队列中的所有命令；
    4. WATCH：监控 key，若事务执行前 key 被修改，事务自动放弃，实现乐观锁。
    
- ACID 特性保证：
    
    1. 原子性：**弱原子性**，仅保证事务整体提交 / 放弃，单条命令执行报错，后续命令依然会执行，不支持回滚；
    2. 一致性：满足一致性，事务执行前后数据的约束和完整性不变；
    3. 隔离性：完全满足，单线程执行模型保证事务执行期间不会被其他命令打断；
    4. 持久性：不保证，取决于持久化配置。
    
- 和 Lua 脚本的核心区别：
    1. 原子性：Lua 脚本是**强原子性**，整个脚本内的所有命令要么全部执行成功，要么全部失败，执行期间绝对不会被其他命令打断；Redis 事务仅保证命令顺序执行，不支持回滚；
    2. 灵活性：Lua 脚本支持复杂的逻辑判断、循环，可实现复杂业务的原子操作；事务仅支持简单命令打包，无逻辑处理能力；
    3. 易用性：Lua 脚本可复用，适合复杂批量操作；事务命令简单，适合轻量批量操作。
