**题目**：

**标准答案**：

- 哨兵核心作用：实现 Redis 主从架构的高可用，核心能力包括：节点监控、自动故障转移、配置通知、客户端服务发现。
- 故障转移完整流程：
    
    1. 主观下线：单个哨兵检测到主节点心跳超时，标记为主观下线（SDOWN）；
    2. 客观下线：该哨兵向其他哨兵发起投票，当超过法定人数（quorum）的哨兵确认主节点下线，标记为客观下线（ODOWN）；
    3. 哨兵 Leader 选举：通过 Raft 算法选举出一个哨兵 Leader，负责执行故障转移；
    4. 新主节点选举：Leader 从存活的从节点中，按优先级 > 复制偏移量 > runID 的规则，选出最优从节点；
    5. 主从切换：Leader 将新节点提升为主节点，让其他从节点同步新主节点；将旧主节点标记为从节点，恢复后自动同步新主数据。
    
- 哨兵与 Redis Cluster 核心区别：
    
| 维度   | 哨兵（Sentinel）        | Redis Cluster 集群             |
| :--- | :------------------ | :--------------------------- |
| 核心定位 | 主从架构的高可用管控，解决单点故障   | 分布式分片集群，解决海量数据存储和高并发写压力      |
| 数据存储 | 全量数据在每个节点都有完整副本     | 数据按 16384 个槽位分片存储，每个节点负责部分槽位 |
| 读写能力 | 仅支持读写分离，写能力受限于单主节点  | 支持分布式写，多主节点分摊写压力，水平扩展能力强     |
| 集群规模 | 3 个哨兵 + 1 主多从，小规模部署 | 至少 3 主 3 从，支持上千节点大规模集群       |
