# 一、一句话核心结论（面试开场必说）

1. **MSET / MGET 是 Redis 原生命令**，属于**服务端行为**；
2. **Pipeline 是客户端提供的批量提交机制**，属于**客户端 + 网络层行为**；
3. **MGET/MSET 只能执行同种类型命令**；
4. **Pipeline 可以打包任意多条不同命令**；
5. **MGET/MSET 是原子操作**；
6. **Pipeline 不是原子操作**。

---

# 二、详细对比（面试必背）

## 1. 本质区别

### ✔ MGET / MSET（原生批命令）

- 属于 **Redis 服务端内置命令**
- 一次网络请求 = 执行 **一条命令**（服务端内部批量处理）
- 服务端**原子执行**，中间不会插入其他命令

### ✔ Pipeline（管道）

- 属于 **Redis 客户端实现**
- 一次网络请求 = 打包 **N 条不同命令**
- 服务端**依次执行**，但**不保证原子性**

---

## 2. 原子性（最重要区别）

### MGET / MSET

**原子操作**

整条命令一次性执行，中间不会被其他客户端命令打断。

### Pipeline

**非原子操作**

Pipeline 只是把多条命令一次性发给 Redis，**服务端会逐条执行**，中间完全可以插入其他客户端的命令。

---

## 3. 支持的命令类型

### MGET / MSET

**只能执行 GET / SET 一类的字符串批量操作**，不能混合其他命令。

- 不能 MGET + HGET
- 不能 MSET + INCR

### Pipeline

**可以打包任意命令**

- GET + HGET + INCR + ZADD
- 任何命令都能塞进 Pipeline 一起发

---

## 4. 网络模型区别（底层原理）

### MGET / MSET

1 次网络 IO → 1 条命令 → 服务端批量执行

### Pipeline

1 次网络 IO → N 条命令 → 服务端依次执行并返回 N 个结果

**Pipeline 减少了 N-1 次来回网络 RTT**，性能提升巨大。

---

## 5. 事务性

### MGET/MSET

**原子性**

### Pipeline

**无事务，不原子**

中间可以被其他命令插队。

---

## 6. 使用场景

### MGET / MSET

- 需要**原子批量读写字符串**
- 批量获取 / 设置用户信息、配置

### Pipeline

- **大量小命令需要极高性能**
- 不需要原子性
- 例如：批量日志、批量埋点、批量写入、批量更新

# 四、面试标准答案（直接背）

## 问：MGET/MSET 与 Pipeline 的区别？

你可以这样回答（满分）：

1. **本质不同**
    
    MGET/MSET 是 Redis 服务端提供的**原生批处理命令**；
    
    Pipeline 是**客户端实现**的命令打包机制。
    
2. **原子性不同**
    
    MGET/MSET 是**原子操作**；
    
    Pipeline 只是批量发送，**不保证原子性**。
    
3. **命令范围不同**
    
    MGET/MSET 只能用于字符串批量操作；
    
    Pipeline 可以打包**任意多条不同命令**。
    
4. **网络模型不同**
    
    MGET/MSET 是一次请求一条命令；
    
    Pipeline 是一次请求发送 N 条命令，**减少网络 RTT，大幅提升性能**。
    
5. **使用场景不同**
    
    需要原子批量读写字符串 → 用 MGET/MSET；
    
    大量小命令需要高性能、不要求原子 → 用 Pipeline。
