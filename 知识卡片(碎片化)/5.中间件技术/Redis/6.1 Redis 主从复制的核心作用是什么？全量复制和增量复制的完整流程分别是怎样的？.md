**标准答案**：

- 核心作用：实现数据热备份；实现读写分离，分摊读压力，提升集群并发能力；为哨兵、集群的故障转移提供数据支撑。
- 全量复制流程（首次建立主从、无法增量复制时触发）：
    
    1. 从节点发送 SLAVEOF 命令，与主节点建立网络连接；
    2. 主节点执行 bgsave 生成 RDB 快照，同时将快照期间的新写命令写入复制缓冲区；
    3. 主节点将 RDB 文件发送给从节点，从节点清空本地数据并加载 RDB 文件；
    4. 主节点将复制缓冲区的写命令发送给从节点，从节点执行完成数据同步；
    5. 后续进入持续命令传播阶段，主节点的写命令实时同步给从节点。
    
- 增量复制流程（网络闪断重连后触发）：
    1. 主从断连期间，主节点将写命令写入复制积压缓冲区（环形缓冲区）；
    2. 从节点重连后，向主节点发送自身的复制 ID 和复制偏移量；
    3. 主节点校验，若复制 ID 匹配且偏移量在缓冲区范围内，执行增量复制；
    4. 主节点将缓冲区中偏移量之后的写命令发送给从节点，从节点执行完成同步。
