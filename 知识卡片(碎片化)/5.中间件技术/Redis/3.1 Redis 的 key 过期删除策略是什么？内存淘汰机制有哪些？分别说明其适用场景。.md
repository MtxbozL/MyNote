**标准答案**：

- 过期删除策略：Redis 采用**惰性删除 + 定期删除**的组合策略
    
    1. 惰性删除：key 被访问时才检查是否过期，过期则删除，节省 CPU 资源，但会造成内存浪费；
    2. 定期删除：每隔固定时间抽取一定数量的过期 key 检查并删除，平衡 CPU 和内存开销。
    
- 内存淘汰机制（内存达到 maxmemory 上限时触发），主流核心策略如下：
    1. **noeviction**（默认）：不淘汰任何 key，新写入直接报错，适合不允许数据丢失的场景；
    2. **allkeys-lru**：对所有 key，淘汰最近最少使用的 key，通用缓存场景首选；
    3. **volatile-lru**：仅对设置了过期时间的 key，淘汰最近最少使用的，适合需保留持久化核心数据的场景；
    4. **allkeys-lfu**：对所有 key，淘汰使用频率最低的 key，适合热点数据缓存场景；
    5. **volatile-lfu**：仅对设置了过期时间的 key，淘汰使用频率最低的 key。
