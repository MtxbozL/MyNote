#### 核心标准答案

软链接（符号链接）和硬链接，都是 Linux 中用于实现文件共享的机制，核心区别如下：

表格

|特性|硬链接|软链接（符号链接）|
|:--|:--|:--|
|**inode 号**|与源文件共用同一个 inode 号，本质是同一个文件|有独立的 inode 号，是一个全新的文件|
|**跨文件系统**|不支持，只能在同一个文件系统内创建|支持，可跨文件系统、跨磁盘分区创建|
|**目录支持**|不支持给目录创建硬链接|支持给目录创建软链接|
|**源文件删除影响**|源文件删除后，硬链接文件依然可用，文件数据不会被删除，直到所有硬链接都被删除|源文件删除后，软链接失效，成为无效链接|
|**相对路径**|源文件路径变化，硬链接不受影响|相对路径创建的软链接，源文件路径变化，软链接失效|
|**文件类型**|普通文件，和源文件类型一致|链接文件，有独立的文件类型|

#### 专家级拓展

- 底层原理：
    
    1. 硬链接：本质是给源文件的 inode 新增一个别名，多个文件名指向同一个 inode，系统为 inode 维护一个链接计数，每新增一个硬链接，计数 + 1，每删除一个链接，计数 - 1，计数为 0 时，inode 和对应的数据块才会被回收。
    2. 软链接：本质是一个独立的文本文件，文件中存储的是源文件的路径，访问软链接时，系统会自动解析路径，找到源文件，和 Windows 的快捷方式原理一致。
    
- 落地场景：
    
    1. 硬链接：用于重要文件的备份，防止误删源文件导致数据丢失；比如系统的`/lib`目录下，大量库文件使用硬链接实现多版本共享。
    2. 软链接：用于程序版本切换、目录快捷访问、跨分区文件共享；比如 Nginx 的多站点配置，用软链接管理启用 / 禁用的站点配置；软件升级时，修改软链接指向新版本目录，无需修改程序配置。
    

#### 面试避坑指南

- 严禁说「硬链接可以给目录创建」，Linux 系统为了避免目录循环引用，默认禁止给目录创建硬链接（只有 root 用户加特殊参数可创建，生产环境严禁使用）；
- 避免混淆「源文件删除后的影响」，这是面试最常问的核心区别，必须精准回答。
