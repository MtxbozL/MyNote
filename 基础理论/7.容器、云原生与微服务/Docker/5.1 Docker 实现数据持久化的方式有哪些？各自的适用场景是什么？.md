**标准答案**：

Docker 官方提供 3 种数据持久化方案，均实现容器数据与容器生命周期的解耦，容器销毁后数据不会丢失，具体如下：

1. **Docker 卷（Volumes）**
    
    - 特点：由 Docker 完全管理，存储在宿主机的 Docker 专属目录（/var/lib/docker/volumes/），非 Docker 进程无法修改，生命周期独立于容器。
    - 适用场景：**官方推荐的持久化方案**，适合生产环境数据库数据、应用持久化数据、需要跨容器共享的数据等核心业务数据存储。
    
2. **绑定挂载（Bind Mounts）**
    
    - 特点：将宿主机的任意文件 / 目录直接挂载到容器中，宿主机和容器可双向读写修改，权限完全开放。
    - 适用场景：开发环境，比如将本地代码挂载到容器中，实现代码修改实时生效；需要在宿主机和容器之间共享配置文件的场景。
    
3. **临时文件系统挂载（tmpfs Mounts）**
    - 特点：将数据挂载到宿主机的内存中，不会写入宿主机的文件系统，容器停止后数据立即丢失。
    - 适用场景：存储敏感临时数据、高并发场景的临时缓存，不需要持久化、且需要高性能读写的场景。
