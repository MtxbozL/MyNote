**标准答案**：

Docker 的核心组件分为 5 个，共同构成完整的运行体系：

1. **Docker 客户端（Client）**：用户与 Docker 交互的命令行工具（CLI），用户通过 docker 命令向 Docker Daemon 发送请求。
2. **Docker 守护进程（Docker Daemon）**：运行在宿主机后台的服务进程，负责监听 Docker API 请求，管理容器、镜像、网络、数据卷等核心资源。
3. **Docker 镜像（Image）**：容器的只读模板，包含运行应用所需的代码、运行环境、依赖、配置文件等所有内容，是容器创建的基础。
4. **Docker 容器（Container）**：镜像的运行实例，是一个独立的、可运行的轻量级进程，拥有自己的文件系统、网络、进程空间，是应用的实际运行载体。
5. **Docker 仓库（Registry）**：用于存储和分发 Docker 镜像的服务，分为公共仓库（如 Docker Hub、阿里云镜像仓库）和私有仓库，实现镜像的共享与版本管理。
