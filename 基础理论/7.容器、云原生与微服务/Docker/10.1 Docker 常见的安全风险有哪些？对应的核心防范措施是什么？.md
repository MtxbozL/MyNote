**标准答案**：

Docker 的安全风险主要集中在容器逃逸、镜像安全、权限过大、资源耗尽、网络攻击等方面，对应的防范措施贴合行业安全最佳实践，具体如下：

1. **容器逃逸风险**
    
    - 风险点：特权容器（--privileged）、挂载宿主机根目录 / 敏感目录、内核漏洞、Capabilities 权限过大，导致容器突破隔离，获取宿主机 root 权限。
    - 防范措施：**严格禁止使用特权容器**；仅挂载必要的目录，禁止挂载宿主机 /etc、/root、/dev 等敏感目录；使用`--cap-drop=ALL`默认禁用所有 Linux Capabilities，仅添加业务必需的权限；及时更新宿主机内核与 Docker 版本，修复已知逃逸漏洞。
    
2. **镜像安全风险**
    
    - 风险点：使用来源不明的恶意镜像、镜像存在高危漏洞、镜像未做签名验证，导致容器被植入后门、挖矿程序等恶意代码。
    - 防范措施：仅使用 Docker 官方镜像、厂商官方镜像等可信来源的镜像；使用 Trivy、Clair 等镜像扫描工具，在部署前扫描镜像漏洞；开启镜像签名验证，防止镜像被篡改。
    
3. **权限过大风险**
    
    - 风险点：容器默认以 root 用户运行，拥有宿主机 root 的等效权限，一旦容器被入侵，攻击者可获取高权限。
    - 防范措施：严格遵循最小权限原则，在 Dockerfile 中通过 USER 指令指定非 root 用户运行容器；禁止容器使用 root 用户启动。
    
4. **资源耗尽风险**
    
    - 风险点：未对容器设置资源限制，单个容器可占用宿主机所有 CPU、内存、IO 资源，导致宿主机宕机，引发拒绝服务攻击（DoS）。
    - 防范措施：启动容器时必须设置资源限制，通过`--cpus`限制 CPU 核心数、`--memory`限制内存上限、`--blkio-weight`限制 IO 权重，防止资源滥用。
    
5. **网络安全风险**
    
    - 风险点：容器端口过度暴露、网络隔离不足，导致容器被扫描、攻击，横向渗透风险高。
    - 防范措施：仅暴露业务必需的端口，禁止不必要的端口映射；使用自定义网络隔离不同业务的容器，通过防火墙限制容器的南北向、东西向访问；禁止容器直接绑定宿主机的公网 IP。
