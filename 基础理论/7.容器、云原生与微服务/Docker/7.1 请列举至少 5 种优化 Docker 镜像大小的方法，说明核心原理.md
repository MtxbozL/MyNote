**标准答案**：

镜像优化的核心目标是：减少镜像层数、剔除不必要的文件、降低基础镜像体积，同时利用构建缓存提升构建速度，常用方法如下：

1. **多阶段构建（最核心方案）**
    
    - 原理：将构建环境与运行环境分离，在第一阶段完成代码编译、依赖打包等操作，第二阶段仅复制最终的可执行文件 / 运行所需文件到精简的运行镜像中，彻底剔除构建工具、中间产物、源码等冗余文件。
    
2. **选用精简基础镜像**
    
    - 原理：替换大体积的全功能基础镜像，优先使用 alpine（5MB 左右，精简版 Linux）、distroless（无额外系统工具，仅保留运行依赖）等镜像，可将基础镜像体积降低 90% 以上。
    
3. **合并 RUN 指令，减少镜像层数**
    
    - 原理：Dockerfile 中每一条指令都会生成一个镜像层，镜像层越多，镜像体积越大。将多个连续的 RUN 命令通过`&&`合并为一条，同时在同一条 RUN 中完成安装、清理缓存的操作，避免缓存文件被提交到镜像层中。
    
4. **清理冗余依赖与缓存**
    
    - 原理：在安装依赖的同一条 RUN 指令中，执行缓存清理命令，比如`apt install && apt clean && rm -rf /var/lib/apt/lists/*`、`npm install && npm cache clean --force`，避免包管理工具的缓存残留到镜像中。
    
5. **使用.dockerignore 文件**
    
    - 原理：排除构建上下文中不需要的文件（比如.git、node_modules、本地日志、临时文件、测试代码等），避免这些文件被复制到镜像中，同时提升构建速度。
    
6. **优化构建层顺序，利用缓存**
    - 原理：将不常变动的指令（比如依赖安装）放在 Dockerfile 前面，频繁变动的指令（比如复制代码）放在后面，当代码变动时，仅重新构建后面的层，复用前面的缓存层，同时避免不必要的层重建。
