**标准答案**：

Docker 容器的生命周期是容器从创建到销毁的完整状态流转，核心分为 5 个状态，对应操作如下：

1. **创建状态（Created）**：容器已被创建，但未启动，对应命令：`docker create`
2. **运行状态（Running）**：容器内进程正常运行，对应命令：`docker run`（创建 + 启动）、`docker start`（启动已创建 / 已停止的容器）
3. **暂停状态（Paused）**：容器内进程被暂停，冻结运行状态，不释放资源，对应命令：`docker pause`，恢复命令：`docker unpause`
4. **停止状态（Stopped）**：容器内进程终止，释放 CPU、内存资源，保留文件系统与配置，对应命令：`docker stop`、`docker kill`（强制终止）
5. **删除状态（Deleted）**：容器被彻底销毁，释放所有资源，无法恢复，对应命令：`docker rm`

补充：`docker restart`可实现容器的停止→重启流程，回到运行状态。
