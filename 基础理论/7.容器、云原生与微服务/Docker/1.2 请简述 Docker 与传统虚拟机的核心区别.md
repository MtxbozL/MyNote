### 【面试官标准提问】

请你说一下 Docker 容器和传统虚拟机（VMware/KVM）的核心区别，以及各自的适用场景。

### 【专家级作答话术】

首先，两者最核心的本质差异是**虚拟化的层级不同**：Docker 是操作系统级虚拟化，直接共享宿主机的 OS 内核；传统虚拟机是硬件级虚拟化，需要在宿主机硬件上虚拟出完整的硬件层，再运行独立的 Guest OS，这是所有差异的根源。

基于这个核心差异，我从 4 个核心维度展开，同时结合我们生产环境的选型经验说明：

第一，**资源占用与启动速度**：Docker 容器是 MB 级的，秒级启动，通常启动时间在 500ms 以内，因为它只需要启动应用进程，不需要完整的系统开机流程；而虚拟机是 GB 级的，分钟级启动，需要完整的 Guest OS 启动流程，CPU 和内存的固定开销非常大。我们生产上，单台 8 核 16G 的服务器，最多跑 10 台虚拟机，但可以轻松跑 100 + 个 Docker 容器，资源利用率提升了 8 倍以上。

第二，**隔离与安全级别**：虚拟机基于硬件虚拟化技术实现完全隔离，隔离性是内核级的，一台虚拟机崩溃不会影响其他虚拟机；而 Docker 是通过 Linux Namespace 做进程级隔离，Cgroup 做资源限制，共享宿主机内核，隔离性更弱，存在容器逃逸的风险。

第三，**移植性与交付效率**：Docker 镜像把应用的代码、依赖、环境、配置全部打包，真正实现「一次构建，到处运行」，从开发到测试再到生产，环境一致性可以做到 100%，彻底解决了「我本地能跑，线上不能跑」的问题；而虚拟机的镜像移植性很差，受硬件、虚拟化平台、操作系统版本的限制非常大。

第四，**适用场景**：Docker 容器最适合无状态的微服务应用、CI/CD 持续集成、自动化测试、弹性扩缩容的场景；而虚拟机更适合有状态的核心数据库、需要强隔离的多租户业务、需要运行不同操作系统内核的场景，两者不是替代关系，而是互补关系。

### 【核心加分亮点】

1. 点出了「虚拟化层级不同」的核心本质，而不是单纯罗列差异，体现底层原理的理解
2. 补充了生产环境的实测数据（资源利用率、启动速度），证明有真实的线上运维经验
3. 明确了两者是「互补关系」，而不是非黑即白，体现技术选型的成熟度
4. 提到了 Namespace 和 Cgroup 两个核心底层技术，预埋了后续追问的伏笔

### 【面试避坑提醒】

❌ 绝对不要说「Docker 比虚拟机更好，完全可以替代虚拟机」，会被面试官认为认知片面

❌ 不要混淆隔离性的底层实现，不要说 Docker 是硬件级虚拟化

❌ 不要只罗列差异，不讲适用场景，面试官最关心你会不会做技术选型

### 【面试官高频追问 & 应答要点】

1. 追问：Docker 共享宿主机内核，那能不能在 Ubuntu 宿主机上运行 CentOS 容器？
    
    应答要点：可以。因为容器只需要 CentOS 的用户态文件系统（库、配置、命令等），内核直接复用宿主机的 Ubuntu 内核，只要内核版本兼容，就可以正常运行。
2. 追问：容器的隔离性弱，那生产上怎么提升容器的安全性？
    
    应答要点：可以从 4 个方面入手：禁用特权容器、使用非 root 用户运行容器、限制 Linux Capabilities、开启 AppArmor/SELinux 安全策略、使用安全基线扫描镜像，刚好可以衔接第 10 道题的安全内容。
