**标准答案**：

Docker 原生提供 5 种核心网络模式，适配不同的业务场景，具体如下：

1. **bridge 模式（默认模式）**
    
    - 特点：Docker 为容器创建独立的网络命名空间，分配独立的 IP，所有容器默认接入 docker0 网桥，通过端口映射对外提供服务，容器间通过 IP 通信。
    - 适用场景：单主机上的独立容器，开发测试环境的基础使用。
    - 补充：自定义 bridge 网络（推荐）支持 Docker 内置 DNS 解析，容器间可直接通过容器名 / 服务名通信，隔离性更好，是生产环境单主机通信的首选。
    
2. **host 模式**
    
    - 特点：容器与宿主机共享网络命名空间，直接使用宿主机的 IP、端口、网卡，无网络隔离，网络性能无损耗。
    - 适用场景：对网络性能要求极高的场景，比如网关、压测工具，需要直接占用宿主机端口的应用。
    
3. **none 模式**
    
    - 特点：容器无任何网络接口，仅保留 lo 本地回环接口，完全关闭网络，隔离性最高。
    - 适用场景：高安全要求的离线应用、不需要网络的计算任务、敏感数据处理容器。
    
4. **container 模式**
    
    - 特点：新创建的容器与指定的已有容器共享同一个网络命名空间，两个容器之间通过[localhost](https://localhost)通信，网络配置完全共享。
    - 适用场景：需要网络互通的边车容器（Sidecar），比如日志采集、监控代理、网络代理容器与主业务容器共享网络栈。
    
5. **overlay 模式**
    - 特点：Docker 原生的跨主机网络模式，构建在宿主机网络之上的虚拟二层网络，实现不同宿主机上的容器直接通信，无需端口映射。
    - 适用场景：Docker Swarm、Kubernetes 等容器集群，跨主机多容器的分布式应用部署。
