**标准答案**：

COPY 和 ADD 均用于将宿主机的文件 / 目录复制到容器镜像中，核心区别与最佳实践如下：

1. **核心功能差异**
    
    - **COPY**：仅支持将本地宿主机的文件 / 目录复制到镜像中，功能单一，无额外操作。
    - **ADD**：在 COPY 的基础上，额外支持两个功能：① 自动解压本地 tar/zip 等压缩包到镜像的目标路径；② 支持从 URL 远程下载文件到镜像中。
    
2. **关键注意点**
    
    - ADD 的远程 URL 下载功能不会被 Docker 构建缓存命中，每次构建都会重新下载，会增大镜像体积，官方不推荐使用。
    - ADD 仅能解压本地压缩包，无法解压远程下载的压缩包。
    
3. **最佳实践**
    - 绝大多数场景优先使用**COPY**指令，仅当需要自动解压本地压缩包时，才使用 ADD 指令。
