**标准答案**：

- 缓存穿透：大量请求查询**不存在于缓存和数据库中的 key**，请求直接打穿到数据库，导致数据库压力骤增。
    
    解决方案：布隆过滤器拦截非法 key；对空查询结果设置短过期时间的空缓存；接口层增加参数校验与限流。
- 缓存击穿：某一个**热点 key 过期**，瞬间大量并发请求直接打到数据库，导致数据库压力暴增。
    
    解决方案：热点 key 设置永不过期，后台异步更新；使用互斥锁，仅允许一个线程更新缓存；提前预热热点数据。
- 缓存雪崩：大量**key 同时过期**，或 Redis 集群宕机，导致大量请求直接打到数据库，引发数据库宕机。
    
    解决方案：给 key 的过期时间增加随机值，避免集中过期；搭建 Redis 高可用集群，避免单点故障；服务层增加熔断、降级、限流保护；增加本地缓存多级防护。
