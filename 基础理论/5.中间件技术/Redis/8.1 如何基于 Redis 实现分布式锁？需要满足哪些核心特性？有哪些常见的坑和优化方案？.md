**标准答案**：

- 分布式锁核心特性：互斥性、防死锁、可重入性、高可用、操作原子性。
- 标准基础实现：
    
    1. 加锁：使用**SET key unique_value NX PX expire_time** 原子命令，NX 保证互斥，PX 设置过期时间防死锁，unique_value 使用客户端唯一标识，避免误删锁；
    2. 解锁：使用 Lua 脚本实现「判断 value 匹配→删除锁」的原子操作，避免误删其他客户端的锁。
    
- 常见坑与优化方案：
    1. 坑 1：锁过期释放，业务还未执行完 → 优化：引入**看门狗（watchdog）**，锁快过期时自动续期（Redisson 已实现）；
    2. 坑 2：主节点加锁成功未同步到从节点，主节点宕机导致锁丢失 → 优化：使用 RedLock 红锁，过半节点加锁成功才算成功；
    3. 坑 3：不可重入，同一线程重复加锁导致死锁 → 优化：基于 Hash 结构 + Lua 脚本，记录重入次数，实现可重入锁；
    4. 坑 4：解锁非原子性，判断和删除分两次执行导致误删 → 优化：必须使用 Lua 脚本保证解锁原子性。
