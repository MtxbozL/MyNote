**标准答案**：

核心优化分为命令、内存、部署、持久化、网络五大方向，可落地的核心优化手段如下：

1. **命令优化**：禁用 KEYS、FLUSHALL 等高风险慢命令，生产环境 rename 禁用；避免 O (n) 复杂度命令的全量查询，分批操作；批量操作使用 MSET/MGET 替代多次单命令，减少网络 IO；开启慢日志监控，定期优化慢命令。
2. **内存优化**：合理设置 key 过期时间，清理无效数据；小数据量优先使用压缩编码的 Hash/List 结构，减少内存开销；拆分大 key，避免单个 value 过大导致内存碎片和网络阻塞；开启内存碎片整理，定期清理内存碎片。
3. **部署架构优化**：采用读写分离，读请求分摊到从节点；Redis 与 CPU 密集型 / IO 密集型服务隔离部署；绑定 CPU 核心，避免进程在多核间切换；单节点内存控制在 10GB 以内，超大集群使用 Cluster 分片存储。
4. **持久化优化**：生产环境优先使用 RDB+AOF 混合持久化；控制 RDB 快照频率，避免频繁 fork 阻塞主进程；AOF 刷盘使用默认 everysec 策略，避免 always 带来的 IO 瓶颈；在业务低峰期手动触发 AOF 重写，避免高峰阻塞。
5. **网络与系统优化**：客户端使用长连接 + 连接池，避免频繁创建销毁 TCP 连接；关闭透明大页（THP），减少 fork 耗时；禁用 Swap 分区，避免 Redis 内存被换入磁盘导致性能骤降；调整 TCP backlog 参数，提升高并发连接处理能力。
