#### 【核心得分点】

哨兵的核心作用是**实现 Redis 主从架构的高可用**，核心能力包括节点监控、自动故障转移、配置通知、客户端服务发现；故障转移核心分为主观下线→客观下线→哨兵 Leader 选举→新主节点选举→主从切换 5 个阶段；哨兵是主从架构的高可用管控组件，Redis Cluster 是分布式分片集群，核心定位和能力完全不同。

#### 【专家级完整作答】

##### 一、Redis Sentinel（哨兵）的核心作用

哨兵是 Redis 官方提供的主从架构高可用管控组件，由多个哨兵节点组成分布式集群，核心解决主从架构的主节点单点故障问题，核心能力有 4 点：

1. **节点监控**：持续监控主节点、从节点、其他哨兵节点的运行状态，检测节点是否存活；
2. **自动故障转移**：当主节点发生故障时，自动完成主从切换，将最优从节点提升为新的主节点，其余从节点同步新主节点，整个过程无需人工干预；
3. **配置通知**：主从切换完成后，哨兵将最新的主节点地址通知给客户端，客户端自动切换连接；
4. **事件通知**：节点故障、主从切换等事件，可通过回调通知给运维人员，实现告警能力。

##### 二、哨兵故障转移完整流程

哨兵故障转移基于 Raft 分布式一致性算法实现，保证集群内节点的决策一致性，完整流程分为 5 个核心阶段：

1. **主观下线（SDOWN）**
    
    单个哨兵节点检测到主节点心跳超时（超过 down-after-milliseconds 配置），会将该主节点标记为**主观下线**，该状态仅代表单个哨兵的判断，不代表集群共识。
2. **客观下线（ODOWN）**
    
    标记主节点主观下线的哨兵，向集群内其他哨兵节点发送 is-master-down-by-addr 命令，询问其他哨兵是否认为主节点已下线；当收到**超过法定人数（quorum）** 的哨兵确认主节点下线后，将该主节点标记为**客观下线**，进入故障转移流程。
3. **哨兵 Leader 选举**
    
    通过 Raft 算法，从集群内的哨兵节点中，选举出一个哨兵 Leader，由该 Leader 全权负责本次故障转移操作，保证同一时间只有一个哨兵执行故障转移。
4. **新主节点选举**
    
    哨兵 Leader 从存活的从节点中，按照优先级规则，选出最优的从节点作为新的主节点，选举优先级从高到低为：
    
    - 第一步：筛选出 replica-priority 配置值最低的从节点（优先级最高）；
    - 第二步：优先级相同，筛选出复制偏移量最大的从节点（数据最完整）；
    - 第三步：前两项均相同，筛选出 runID 最小的从节点。
    
5. **主从切换执行**
    
    - 哨兵 Leader 向选出的新主节点发送 SLAVEOF NO ONE 命令，将其提升为主节点；
    - 向其余所有从节点发送 SLAVEOF 命令，让它们同步新的主节点；
    - 将故障的旧主节点标记为从节点，待其恢复后，自动同步新主节点的数据，完成整个故障转移流程。
    

##### 三、哨兵与 Redis Cluster 集群的核心区别

表格

|核心维度|Redis Sentinel（哨兵）|Redis Cluster 集群|
|:--|:--|:--|
|核心定位|主从架构的高可用管控组件，解决主节点单点故障问题|分布式分片集群，解决海量数据存储、高并发写压力的水平扩展问题|
|数据存储|全量数据在主节点和所有从节点都有完整副本，无分片|数据按 16384 个槽位分片存储，每个主节点负责一部分槽位，数据分散存储|
|并发能力|仅支持读写分离，读能力可水平扩展，写能力受限于单主节点|支持分布式读写，多主节点分摊读写压力，写能力可水平扩展|
|集群规模|小规模部署，建议 3 个哨兵节点 + 1 主多从，最大支持 1 主数十从|大规模分布式部署，至少 3 主 3 从，支持上千节点的超大规模集群|
|适用场景|数据量不大、读多写少、核心需求是高可用的场景|数据量巨大、高并发写、需要水平扩展的大规模业务场景|

#### 【加分项延伸】

1. 生产部署最佳实践：哨兵集群必须部署奇数个节点，至少 3 个，且跨机房部署，避免单机房宕机导致哨兵集群无法达成共识；quorum 值建议配置为哨兵节点数的一半 + 1，保证集群决策的一致性；
2. 脑裂问题解决方案：生产环境必须配置 min-replicas-to-write 参数，避免主节点网络隔离时，仍接收客户端写入，导致主从切换后数据丢失；
3. 选型边界：数据量小于 10GB、QPS 小于 10 万，优先选择哨兵 + 主从架构，运维成本低；数据量大于 10GB、QPS 大于 10 万，优先选择 Redis Cluster 集群。

#### 【避坑 & 追问预判】

- 致命避坑：绝对不能说哨兵可以实现数据分片存储；绝对不能混淆主观下线和客观下线的触发条件；
- 高频追问预判：哨兵集群的脑裂问题是什么？如何解决？哨兵的 Raft 选举流程？
