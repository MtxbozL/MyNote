#### 核心标准答案

Linux 系统开机启动是一个分层初始化的过程，从硬件自检到用户登录，完整流程分为 7 个核心阶段：

1. **BIOS/UEFI 硬件自检**：开机后主板固件启动，完成硬件 POST 加电自检，检测 CPU、内存、磁盘等硬件是否正常，之后根据启动顺序，加载对应的启动磁盘。
2. **MBR/GPT 引导**：读取启动磁盘的主引导记录（MBR）或 GUID 分区表（GPT），找到并启动引导加载程序 GRUB。
3. **GRUB 引导菜单**：系统引导程序 GRUB 启动，展示内核启动菜单，让用户选择要启动的内核版本，之后加载选中的 Linux 内核文件到内存。
4. **内核加载与初始化**：Linux 内核启动，挂载根文件系统，初始化硬件设备，加载核心驱动，最终启动系统的 1 号进程（systemd，传统系统为 sysvinit）。
5. **systemd 初始化**：systemd 作为系统所有进程的父进程，读取配置文件，执行系统初始化，包括挂载文件系统、加载内核模块、设置主机名、配置网络等。
6. **运行级别 / 目标单元启动**：systemd 根据预设的运行级别（runlevel，对应 systemd 的 target 单元），启动对应级别的服务，比如多用户模式、图形化模式，启动对应的系统服务与自启程序。
7. **用户登录**：启动终端服务，展示登录界面，用户输入账号密码完成登录，进入系统 shell 环境。

#### 专家级拓展

- systemd 与传统 sysvinit 的核心差异：sysvinit 是串行启动，服务启动慢，一个服务卡住会影响后续所有服务；systemd 是并行启动，基于依赖关系并行启动服务，启动速度更快，同时支持服务的 socket 激活、路径激活、快照、依赖管理，是目前所有主流 Linux 发行版的默认初始化系统。
- 落地场景：运维排查系统启动故障时，严格遵循这个流程定位问题 —— 比如硬件自检失败，排查硬件故障；GRUB 菜单不显示，排查引导分区损坏；内核加载失败，排查内核文件、根文件系统损坏；服务启动异常，排查 systemd 的服务配置文件与日志。
- 核心配置文件：systemd 的默认启动 target 为`/etc/systemd/system/default.target`，服务配置文件存放在`/usr/lib/systemd/system/`和`/etc/systemd/system/`目录。

#### 面试避坑指南

- 严禁把流程顺序说反，或遗漏核心阶段，这是面试的基础扣分点；
- 避免只说流程名称，不说每个阶段的核心作用，会被判定为死记硬背，无故障排查能力；
- 注意区分传统 sysvinit 和 systemd 的差异，现在主流系统都是 systemd，不要只讲过时的 sysvinit。
