#### 参考答案

Nginx 主流的限流方案分为 **连接数限流** 和 **请求速率限流** 两大类，均基于 ngx_http_limit_conn_module 和 ngx_http_limit_req_module 模块实现（默认内置），核心用于防 CC 攻击、恶意请求、接口过载保护。

#### 1. 请求速率限流（核心常用，限制单位时间内的请求数）

基于**漏桶算法**实现，严格控制请求的处理速率，支持突发流量配置，是最常用的限流方案。

nginx

```
# 1. http块中定义限流规则
# limit_req_zone 定义限流区域；$binary_remote_addr 基于客户端IP限流；zone定义共享内存空间；rate限制速率
http {
    limit_req_zone $binary_remote_addr zone=req_limit:10m rate=10r/s;

    server {
        # 2. server/location块中启用限流
        # burst=20 允许20个突发请求；nodelay 突发请求立即处理，不延迟
        location /api/ {
            limit_req zone=req_limit burst=20 nodelay;
            proxy_pass http://backend_server;
        }
    }
}
```

- 核心说明：rate=10r/s 表示每秒最多处理 10 个请求；burst 用于应对突发流量，nodelay 避免突发请求排队等待，提升用户体验。

#### 2. 连接数限流（限制单个客户端的并发连接数）

用于限制单个 IP 同时建立的连接数，适合防爬虫、大文件下载并发控制。

nginx

```
# 1. http块中定义连接数限流规则
http {
    limit_conn_zone $binary_remote_addr zone=conn_limit:10m;

    server {
        # 2. 启用限流，限制单个IP最多同时建立10个连接
        location /download/ {
            limit_conn conn_limit 10;
        }
    }
}
```

#### 3. 扩展限流维度

除了基于 IP 限流，还可基于（域名）、uri（接口地址）、自定义请求头做限流，适配不同业务场景。
