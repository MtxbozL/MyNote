#### 参考答案

Nginx 负载均衡策略分为 **内置核心策略（默认编译自带）** 和 **第三方扩展策略（需额外安装模块）** 两大类，具体如下：

表格

|策略类型|核心原理|适用场景|
|:--|:--|:--|
|轮询（默认）|请求按顺序逐一分配到不同的后端服务器，后端服务宕机自动剔除|后端服务器硬件性能一致，请求处理时长均匀的场景|
|加权轮询|基于权重分配请求，权重越高的服务器被分配的请求越多|后端服务器硬件性能不均，需给高性能机器分配更多流量的场景|
|IP 哈希（ip_hash）|基于客户端 IP 的哈希结果分配服务器，同一个 IP 的请求固定转发到同一台后端服务器|解决 session 共享问题，需保持用户会话一致性的场景|
|最少连接数（least_conn）|请求优先转发到当前活跃连接数最少的后端服务器|请求处理时长差异大（如长请求、接口耗时不均）的场景，避免单台服务器过载|
|fair（第三方）|基于后端服务器的响应时间分配请求，响应时间越短优先级越高|对接口响应速度要求高，需优先保证用户访问体验的场景|
|url_hash（第三方）|基于请求 URL 的哈希结果分配服务器，同一个 URL 固定转发到同一台后端|后端有缓存服务的场景（如 CDN、静态资源缓存），提升缓存命中率|
