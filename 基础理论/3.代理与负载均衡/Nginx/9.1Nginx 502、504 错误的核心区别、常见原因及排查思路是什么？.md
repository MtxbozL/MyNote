#### 参考答案

#### 一、核心区别

- **502 Bad Gateway**：网关错误，Nginx 作为反向代理，向后端服务发起请求时，**收到了无效、拒绝的响应，或无法连接到后端服务**。
- **504 Gateway Time-out**：网关超时，Nginx 已成功将请求转发给后端服务，但**在超时时间内未收到后端服务的响应**。

#### 二、502 错误常见原因及排查思路

##### 常见原因

1. 后端服务完全宕机 / 未启动，端口未监听，进程崩溃退出；
2. 后端服务端口被防火墙 / 安全组拦截，Nginx 与后端网络不通；
3. PHP-FPM/FastCGI 服务未启动、进程耗尽、配置错误；
4. 后端服务的最大连接数 /backlog 满了，拒绝新的连接；
5. Nginx proxy_pass 配置的后端地址 / 端口错误。

##### 排查步骤

1. 先验证后端服务是否正常：在 Nginx 服务器上通过 `curl 后端IP:端口/接口` 测试，看是否能正常响应；
2. 检查后端服务进程状态、端口监听状态，确认服务正常运行；
3. 检查防火墙、安全组、服务器之间的网络连通性，确认端口可访问；
4. 查看 Nginx error.log 和后端服务的错误日志，定位具体的连接拒绝原因；
5. 核对 Nginx 配置文件中 proxy_pass 的后端地址、端口是否正确，重载配置验证。

#### 三、504 错误常见原因及排查思路

##### 常见原因

1. 后端接口执行超时，如慢 SQL、复杂业务逻辑、第三方接口调用超时；
2. 后端服务器 CPU / 内存 / IO 满载，无法及时处理请求；
3. Nginx 的超时配置阈值过低，小于后端接口的正常执行时间；
4. 后端服务出现死锁、阻塞，无法返回响应。

##### 排查步骤

1. 直接访问后端接口，测试接口响应耗时，确认是否为接口本身性能问题；
2. 查看后端服务器的 CPU、内存、磁盘 IO、数据库负载，排查资源瓶颈；
3. 检查 Nginx 的超时配置，按需调整阈值，核心配置如下：

```nginx
    proxy_connect_timeout 60s; # 与后端建立连接的超时时间
    proxy_read_timeout 120s; # 后端响应的超时时间，核心配置
    proxy_send_timeout 120s; # 向后端发送请求的超时时间
```

1. 查看后端服务的慢日志、数据库慢查询日志，定位耗时的业务逻辑，优化性能。
