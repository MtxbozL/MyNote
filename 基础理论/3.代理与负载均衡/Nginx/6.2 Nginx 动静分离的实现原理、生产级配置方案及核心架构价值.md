**职级适配**：中级必问

#### 【核心考点】

考察 Web 分层架构设计的理解、Nginx 核心能力的落地能力、基础性能优化思维，判断候选人能否通过 Nginx 实现业务架构的性能与稳定性提升。

#### 【专家级标准答案】

1. **核心实现原理**
    
    动静分离是 Web 架构的核心分层设计思想，**核心是利用 Nginx 极强的静态资源处理能力，将静态资源请求与动态业务请求拆分处理**：
    
    - 静态资源请求（图片、JS、CSS、HTML、字体、视频等）：直接由 Nginx 响应返回，无需转发到后端应用服务器；
    - 动态业务请求（PHP/Java/Python 等后端接口、动态页面）：通过反向代理转发给后端应用服务器处理，再由 Nginx 将结果返回给客户端。
    
2. **核心架构价值**
    
    - 极致提升响应速度：静态资源直接由 Nginx 返回，无需经过后端应用的业务处理链路，响应耗时从百毫秒级降至毫秒级；
    - 大幅降低后端服务压力：互联网业务中静态资源请求占比通常超过 70%，剥离后后端服务器仅需专注处理业务逻辑，系统并发能力可提升数倍；
    - 提升服务可用性：后端应用故障时，静态资源仍可正常访问，避免全站瘫痪；同时静态资源可对接 CDN，实现全国加速，进一步降低源站压力；
    - 便于架构管控：静态资源与动态服务可分别做扩容、降级、权限管控、缓存策略，架构分层清晰，运维与迭代成本大幅降低。
    
3. **生产级标准化配置模板**

```nginx
    server {
        listen 80;
        server_name example.com;
        # 站点根目录，静态资源根路径
        root /usr/share/nginx/example.com;
        index index.html index.htm;
        charset utf-8;
    
        # 1. 高频接口精确匹配，健康检查接口，直接返回，无需转发后端
        location = /health {
            access_log off;
            return 200 "healthy\n";
            default_type text/plain;
        }
    
        # 2. 静态资源非正则前缀匹配，优先级高于正则，Nginx直接处理
        location ^~ /static/ {
            # 浏览器缓存7天，静态资源不变更无需重复请求
            expires 7d;
            # 关闭静态资源访问日志，减少磁盘IO开销
            access_log off;
            # 静态资源防盗链配置，防止恶意盗用
            valid_referers none blocked server_names *.example.com;
            if ($invalid_referer) {
                return 403;
            }
        }
    
        # 3. 静态资源正则匹配，兜底所有静态资源后缀
        location ~* \.(jpg|jpeg|png|gif|ico|js|css|html|woff|woff2|ttf|svg)$ {
            expires 7d;
            access_log off;
        }
    
        # 4. 动态接口请求，反向代理转发到后端应用集群
        location /api/ {
            proxy_pass http://backend_server;
            # 传递真实客户端信息给后端
            proxy_set_header Host $host;
            proxy_set_header X-Real-IP $remote_addr;
            proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
            proxy_set_header X-Forwarded-Proto $scheme;
            # 超时配置，与后端服务对齐
            proxy_connect_timeout 30s;
            proxy_read_timeout 60s;
            proxy_send_timeout 60s;
        }
    
        # 5. 兜底通用匹配，单页应用SPA适配
        location / {
            try_files $uri $uri/ /index.html;
        }
    }
```
    

#### 【深度拓展（面试加分项）】

1. 静态资源进阶优化：开启`sendfile`零拷贝、`tcp_nopush`数据包合并，大幅提升静态资源传输性能；开启 gzip/brotli 预压缩，提前压缩静态资源，避免请求时实时压缩的 CPU 开销；对接 OSS 对象存储，通过 Nginx 反向代理实现静态资源的对象存储托管，无需本地存储。
2. 跨域配置一体化：动静分离场景下，可在 Nginx 层统一配置 CORS 跨域规则，无需后端服务逐个配置，简化跨域处理流程。
3. 动静分离的进阶架构：采用「CDN+Nginx + 对象存储」的三级静态资源架构，CDN 扛住 90% 的静态资源请求，仅回源少量未命中的请求，源站压力降至最低。

#### 【面试避坑提示】

1. 禁止仅提供配置模板，不讲解原理与架构价值，面试官核心考察的是架构设计思维，而非配置复制能力。
2. 禁止错误配置静态资源匹配规则，导致动态请求被拦截，或静态资源请求被转发到后端，失去动静分离的意义。
3. 禁止忽略静态资源的缓存策略与防盗链配置，生产环境极易出现静态资源被恶意盗用、带宽被打满的问题。
