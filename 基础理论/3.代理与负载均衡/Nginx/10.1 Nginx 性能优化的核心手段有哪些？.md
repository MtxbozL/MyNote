#### 参考答案

Nginx 性能优化围绕**进程、IO、网络、缓存、配置、内核**六大维度，核心优化手段如下，适配高并发生产环境：

#### 1. 进程与 CPU 优化（最大化利用硬件性能）

- `worker_processes auto;`：自动匹配 CPU 核心数，避免进程切换开销；
- `worker_cpu_affinity`：绑定 Worker 进程到指定 CPU 核心，提升 CPU 缓存命中率；
- `worker_rlimit_nofile 65535;`：调高单个 Worker 进程的最大文件打开数，避免 “too many open files” 错误。

#### 2. 事件模型与并发优化

- `use epoll;`：启用 epoll 事件模型（Linux 系统默认），适配高并发异步非阻塞处理；
- `worker_connections 10240;`：调高单个 Worker 进程的最大并发连接数，单台 Nginx 最大并发 = worker_processes * worker_connections。

#### 3. IO 与网络传输优化

- `sendfile on;`：开启零拷贝，直接在内核空间传输文件，减少用户态与内核态的切换，大幅提升静态资源传输性能；
- `tcp_nopush on;`：配合 sendfile 使用，合并数据包，减少网络 IO 次数；
- `tcp_nodelay on;`：禁用 Nagle 算法，减少小包传输延迟，提升实时性；
- `keepalive_timeout 65;`：开启长连接复用，减少 TCP 三次握手 / 四次挥手的开销。

#### 4. 静态资源与缓存优化

- 开启 gzip 压缩：对 JS、CSS、HTML 等文本资源压缩，减少传输体积，降低带宽消耗；
- 配置浏览器缓存：通过`expires`指令给静态资源设置缓存过期时间，减少重复请求；
- 关闭静态资源访问日志：`access_log off;`，减少磁盘 IO 开销；
- 开启静态资源本地缓存：通过`proxy_cache`配置后端接口 / 资源的本地缓存，减少后端请求。

#### 5. 超时与安全优化

- 合理设置各类超时阈值，避免无效连接占用资源，提升连接复用率；
- 按需开启限流、IP 黑白名单，过滤恶意请求，减少无效资源占用；
- 禁用不必要的模块，减少 Nginx 内存占用，降低安全风险。

#### 6. 系统内核参数优化

- 调高文件描述符限制：`fs.file-max = 1024000`；
- 优化 TCP 内核参数：调整 TCP 连接复用、超时、缓冲区大小，适配高并发场景；
- 关闭防火墙不必要的规则，关闭 SELinux，减少网络转发开销。
